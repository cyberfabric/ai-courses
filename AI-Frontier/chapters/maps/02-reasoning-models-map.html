Chapter 2

Reasoning Models: Thinking Out Loud

Memory Map
🧠

The Big Idea

A reasoning model is a model that talks to itself before answering. It spends 10–100× more tokens on internal monologue, wins big on math/code/logic, and — in a strange paradox — hallucinates more than base models on grounded tasks like summarization. Match the model to the task, don’t reflexively reach for the slow one.

🚨 The Hallucination Paradox

The contrarian centerpiece. Reasoning makes models worse on grounded tasks.

10–100× more tokens
More inference steps
Chain-of-thought drift
Adds context beyond source
Confident verbose output

Mechanism: a 30-token base answer can only be wrong in 30 tokens. A 30,000-token reasoning chain has 1,000× more surface area for drift — and reasoning models are trained to produce confident, detailed reasoning even when their underlying knowledge is shaky.

📚 Key Terms

Test-time compute
Compute spent during inference (not training). The new scaling axis.
Chain-of-thought (CoT)
The model’s internal step-by-step monologue before producing the final answer. Usually hidden from the user.
Thinking budget
An API-exposed knob (Anthropic) or effort flag (OpenAI’s reasoning_effort) that controls how long the model deliberates per query.
Depth-wise scaling
Longer chain-of-thought, single trajectory. More tokens, deeper reasoning.
Breadth-wise scaling
Multiple parallel solutions scored against each other. Powers o3’s ARC-AGI breakthrough.
Chain-of-thought drift
Errors compounding over a long reasoning chain. Why reasoning models hallucinate more on summarization.
Hallucination paradox
The strange finding that reasoning-enabled models score worse on faithful summarization than non-reasoning baselines. R1 at 14.3%, base V3 at 3.9%.
Hybrid routing
The production pattern of using a cheap classifier to decide whether to send a query to a reasoning or base model.

🧪 Quick Quiz

Self-check

Why are reasoning models worse on summarization than base models?

They think through the source rather than just extracting from it — adding inferences and connections that aren’t in the original. On faithfulness benchmarks those additions score as hallucinations. R1: 14.3% vs V3: 3.9%.

What are the two dimensions of test-time compute?

Depth-wise (longer chain-of-thought, single trajectory) and breadth-wise (multiple parallel solutions with a scoring function). o3 combines both, which cracked ARC-AGI.

How many internal reasoning tokens can o1-preview burn on a single query?

Up to 32,768 tokens. Most queries don’t hit the cap, but on hard math or coding it’s common to see 4K–16K hidden tokens before the first user-visible token streams.

When should you reach for a base model instead of a reasoning model?

Summarization, faithful extraction, latency under 2 seconds, high-volume cost-sensitive workloads, simple lookups, and any “answer is in the input” task. Default to base; switch to reasoning only when deliberation will measurably help.

What three improvements should we expect in reasoning models by end of 2026?

(1) Reasoning + retrieval as default (cuts grounded hallucination). (2) Reasoning + tools as default (Python, DBs mid-chain). (3) Models that know when to stop — adaptive budgeting from inside the model, not from the API caller.

🔑 Key Takeaways

  • Reasoning models = base models that talk to themselves in 10–100× more tokens before answering.
  • Test-time compute is the new scaling axis — pre-training plateaued; inference compute is alive (projected 118× over training by 2026).
  • Wins: math, code, logic, graduate science. GPT-5.4 hits ~99% AIME, frontier reasoners cluster at ~94% GPQA Diamond.
  • Loses: summarization, faithful extraction, latency-sensitive UX, high-volume cheap workloads.
  • The paradox: reasoning models hallucinate more on grounded tasks. R1 14.3% vs V3 3.9%; Grok-4-fast-reasoning 20.2%.
  • The thinking-budget knob is now a permanent API surface: Anthropic budget (100ms–5s+), OpenAI reasoning_effort, Google deep_think.
  • The 80/20 rule: default to base, switch to reasoning only with a specific reason.
  • Production winners route per-query with a cheap classifier — not commit to one mode.
