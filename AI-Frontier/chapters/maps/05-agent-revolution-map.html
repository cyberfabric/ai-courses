The Agent Revolution (and Its Discontents)

Memory Map

The Big Idea

An agent is an LLM in a loop with tools. As of April 2026, capability is doubling every ~7 months. Reliability is roughly flat. That decoupling explains why developer adoption is exploding (4% of GitHub commits) and Gartner predicts 40%+ of enterprise agent projects will be canceled by 2027 — both at once.

📈 METR’s Moore’s Law for Agents

  1. 2022: ~30 seconds

    Frontier agents could complete tasks taking a human about half a minute.

  2. 2026: ~2 hours

    50% success rate at 2-hour tasks. ~100% at <4-minute tasks. <10% at >4-hour tasks.

  3. 2027: ~8-hour workday

    If the doubling continues, full workday tasks become reliable.

  4. 2028–29: Weeks → months

    Extrapolation says month-long initiatives by 2029. Bear case: skewed dataset, the curve flattens on messy real work.

Doubling every ~7 months — data goes back to 2019. More than 3× the rate of classical Moore’s Law.

📜 Key Terms

Agent loop
Observe → think → act → observe result → repeat. The fundamental control flow of every agent.
Tool calling
The model emits a structured request (JSON) describing a function and arguments. The harness executes it and feeds the result back.
Agent harness
The orchestration layer (Claude Code, Codex CLI, Cursor) that runs the loop, manages tools, enforces budgets, and persists state.
Long-horizon RL
Training on multi-step trajectories rather than single-shot completions. The technique behind every frontier agent.
Agent washing
Rebranding chatbots, RPA, or AI assistants as “agents” without substantive agentic capability. Gartner: only ~130 of thousands of vendors are real.
Step budget
A hard cap on agent loop iterations. Without it, silent loops can rack up thousands of dollars in tokens.
Demo trap
The pattern where 4-minute demo videos cherry-pick happy paths while production runs the long tail. The 100th task is the product, not the 1st.
Capability/reliability decoupling
Capability (can it do this once?) is doubling. Reliability (can it do this every time?) is roughly flat. The defining tension of 2026.

💥 The Six Failure Modes

🧩
Long-tail tool errors
Wrong-shaped API call → retries → $40 burn → corrupted state.
📦
Context-window blowouts
Multi-hour tasks accumulate logs & outputs until context overflows.
🔁
Silent loops
Two tools return contradictory state → agent oscillates indefinitely.
👻
Hallucinated tools
Model calls a function that’s not in the tool set, retries, fails.
🌬️
Drift on multi-step plans
Step 13 forgets the constraint set at step 1 (“don’t modify outside src/”).
🎯
Reward hacking
“Make tests pass” → agent deletes the failing tests.

10× lower token spend, order-of-magnitude better reliability. Frameworks: LangGraph, Temporal-AI, BAML, Inngest. The LLM is one expensive function call — use it where the work is genuinely linguistic.

✅ What To Build

  • Scope: 5–30 minutes of agent runtime — that’s where reliability lives
  • Step budgets + cost circuit-breakers: always
  • Structured outputs (JSON schemas): for every tool call
  • Trace every step: you’ll need replay
  • Human-in-the-loop: for non-reversible actions
  • Pick harness for threat model: local vs cloud vs programmatic
  • Demand private-benchmark numbers: public ones are contaminated

🧪 Quick Quiz

What does “capability is doubling but reliability is flat” mean for product design?

Use the narrow + supervised pattern: scope to 5–30 min tasks the agent can reliably complete, with a human reviewing non-reversible output. Full autonomy is the failure mode.

Why does GPT-5 score 83% on SWE-Bench Verified but only 23% on SWE-Bench Pro?

Verified is public — tasks are likely contaminated in training data. Pro uses 1,865 tasks from 41 private repositories the model hasn’t seen. The gap shows how much “benchmark performance” depends on memorization vs generalization.

What is “agent washing” and how big is the problem?

Vendors rebranding chatbots, RPA, and AI assistants as “agents” without substantive agentic capability. Gartner estimates only ~130 of thousands of self-described agentic AI vendors are real. ~95%+ of the vendor market is reheated 2023 chatbots.

What does the METR doubling curve predict for end-2027?

Reliable agent completion of full 8-hour workday tasks. Bear case: the curve is skewed toward verifiable coding, real-world messiness flattens it. Both predictions could be partly right.

When should you choose programmatic+LLM workflows over a pure agent loop?

Whenever the workflow has a known structure with deterministic steps. The LLM is an expensive function call — use it only for the natural-language parts. Result: 10× cheaper, much more reliable, debuggable.

If You Remember Nothing Else

  • An agent is an LLM in a loop with tools — observe, think, act, repeat.
  • METR: agent task length doubles every ~7 months. ~2-hour tasks reliable in 2026.
  • Capability doubling. Reliability roughly flat. The defining tension of 2026.
  • Production winners: Claude Code (4% of GitHub commits), Codex (3M weekly users), Cursor 3.
  • Public benchmarks saturated; private ones (SWE-Bench Pro at 64.3% top score) tell a different story.
  • Gartner: 40%+ of agentic projects canceled by 2027. ~95% of vendors are “agent washing.”
  • Six failure modes: long-tail tool errors, context blowout, silent loops, hallucinated tools, drift, reward hacking.
  • The shipping pattern: narrow scope + human-in-the-loop. Always step budgets.
  • The quiet alt-path winning operations: programmatic workflows with LLMs only as leaves.
  • Don’t buy from anyone who can’t show you private-benchmark numbers.
