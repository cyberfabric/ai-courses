Chapter 8

Rate Limits, Tiering & Capacity Planning

Memory Map
⚡

The Big Idea

Rate limits are the fuse box of your AI stack. In 2026 they’re measured in tokens, they unlock by cumulative spend, and a single agent task can multiply your traffic 10–80×. Plan capacity or the next HN write-up will plan it for you.

🧩 Core Concepts

🎛️
Token-based limits
RPM, ITPM, OTPM, TPD — all apply at once. You can pass TPM and still fail RPM.
🪜
Spend-gated tiers
Auto-promote as your bill grows. Top tiers need prepaid deposits or $1,000+ cumulative spend.
🤖
Agent multiplier
One agent task = 5–30 calls. A 1,000-QPH feature quietly becomes 82× heavier.
🚦
429 vs 529
429 = you went over (throttle). 529 = they’re saturated (retry with backoff).
🧰
Graceful degradation
Concurrency caps, routing fallback, feature shrink, cached defaults — never a raw 500.
🗺️
Capacity forecast
peak_RPM × avg_tokens × growth = future TPM. Alert at 70 % ceiling.

🧮 Capacity Forecasting in 3 Numbers

  1. Peak RPS

    95th-percentile hour from last month’s logs. Multiply by 60 for peak RPM.

  2. Avg tokens/request

    Input + output, weighted by model. Remember the agent multiplier.

  3. Growth factor (6 months)

    If you’re growing 3×/quarter, use 9×. Compare result to your tier ceiling.

  4. Act early

    Under 30 % headroom? Start the tier-bump conversation 6–12 weeks ahead of projected breach.

🕵️ Hidden Gotchas

🔐
Gemini per-project quota
Rotating keys inside one project doesn’t multiply throughput. Separate projects = separate buckets.
🌊
Streaming holds sockets
TPM counts the full response even if the user closes the tab. Wire up abort-on-disconnect.
🧵
Thundering-herd retries
Fixed-delay retries reproduce the spike. Use exponential backoff + random jitter.
🎚️
Unbounded max_tokens
A chatty reasoning model can emit 30K+ tokens you never asked for. Always set an explicit cap.

🗝️ Key Terms

RPM / TPM
Requests per minute / tokens per minute — the two main rate-limit dimensions in 2026.
ITPM / OTPM
Input / output tokens per minute, counted separately on Anthropic & others.
Leaky bucket
Perplexity’s algorithm: fixed capacity that drains at a constant rate, allowing short bursts up to capacity.
Dynamic rate limit
Together AI’s 2026 model: your limit grows if your sustained usage is steady, shrinks if you spike.
Usage tier
Spend-gated level that unlocks higher limits. Auto-promotes, or upgrade via deposit.
Agent multiplier
The factor by which agent-shaped traffic exceeds chat-shaped traffic. Commonly 10–80×.
429 vs 529
429 = your usage exceeded the limit; 529 = the provider’s fleet is overloaded.
Token budget
Hard cap on tokens allowed per feature per call. Protects against runaway loops.

📣 Tier-Bump Signals

🟢
Clean history
Low 429/5xx rate. If you’re already flooding retries, fix that first.
📊
Steady pattern
Smooth growth beats spiky bursts. Together AI literally rewards this.
💳
Prepay / deposit
Tier 4 at Anthropic = $400 deposit. Tier 5 at OpenAI = $1,000+ spend.
📅
Ask early
Approval takes 3–10 business days. A week before launch is already too late.

🧠 Quick Quiz

Why did rate limits shift from RPM to TPM in 2026?

Because token count is the real driver of provider compute cost — a 100K-token call and a 200-token call should not count the same.

You get an HTTP 529 — should you raise your tier?

No. 529 is the provider’s fleet being overloaded. Retry with exponential backoff and jitter; consider failover if it persists.

Why is the agent multiplier such a big deal?

A user-facing feature that looks like “1,000 requests/hour” on the surface can actually be 80,000+ API calls if each task refines through 8 steps — blowing through tier ceilings silently.

Your app has 5 Gemini API keys, all in one project. Does that 5× your throughput?

No. Gemini’s quota is per-project, not per-key. Every key in the project shares the same bucket. You need separate projects to multiply throughput.

What’s the one rate-limit metric to put on your dashboard?

1 - (remaining_tokens / ceiling) over the past hour — “ceiling proximity.” Alert at 80 % and you’ll prevent most 429 outages.

🎓 Key Takeaways

  • Limits are token-based in 2026 — RPM, ITPM, OTPM, TPD all apply at once.
  • Tiers auto-unlock by cumulative spend; top tiers often need a prepaid deposit.
  • 429 = your problem (throttle). 529 = their problem (retry with backoff + jitter).
  • A single agent task can multiply token usage 10–80× over chat-shaped traffic.
  • Forecast with peak_RPM × avg_tokens × 6-month growth. Act at 70 % ceiling, not 100 %.
  • Always read the rate-limit response headers and log “remaining” alongside cost.
  • Gemini quotas are per-project, not per-key — rotating keys inside one project buys you zero extra capacity.
  • Streaming holds sockets and burns TPM even if the user disconnects — wire up abort semantics.
  • Tier bumps take 3–10 business days; request them 6–12 weeks before projected breach.
