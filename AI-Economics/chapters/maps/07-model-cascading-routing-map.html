Chapter 7

Model Cascading & Routing: Right Model, Right Job

Memory Map
🧭

The Big Idea

Don’t make the CEO pick up every call. Send 80 % of traffic to the cheapest model that works, reserve the premium tier for the 20 % that truly needs it — and watch your bill drop 60–80 % without a visible quality hit.

🧩 Core Concepts

🪜
Cascading
Try cheap first, retry on expensive if the cheap answer fails a confidence or validator check.
🚦
Routing
Classify up front — keyword, embedding, or tiny-LLM — then dispatch to the right tier.
🎯
Escalation Signal
How you know the cheap answer isn’t good enough: confidence, schema, consistency vote, or downstream eval.
💰
Break-even Math
A router earns its keep when classifier cost < savings. Usually pays back at 10–50× price asymmetry between tiers.
🪢
Cascade Depth
2-tier is simplest. 3-tier (Haiku→Sonnet→Opus) catches rare hard cases. 4-tier adds free OSS — but with rate-limit ceilings.
🧱
Stacking
Routing stacks with caching, batching, and compression — routinely 80 %+ total savings on support workloads.

📉 Savings Cascade (1M req/month, 2K in + 300 out)

  1. Baseline — Sonnet only

    $10,500/month. One tier. The “safe” default that wastes money on FAQ traffic.

  2. 2-tier cascade (Haiku→Sonnet, 20 % escalation)

    $5,600/month. Cheap model handles 100 % first, Sonnet retries on failures. 47 % saved.

  3. 3-tier router (Haiku 80 % / Sonnet 18 % / Opus 2 %)

    $5,345/month. Tagger + dispatch by difficulty. 49 % saved. Flat latency.

  4. 3-tier router + caching + compression

    ~$2,000/month. Stacking the three earlier levers gets you to ~80 % total saving.

🚨 Failure Modes

🥽
Silent quality drift
Escalation signal weakens over time; the cheap model creeps into harder traffic. Fix: weekly dual-run sampling.
📐
Miscalibrated confidence
Cheap model says 0.9 and is still wrong. Pair confidence with deterministic checks (schema, test).
🧊
Cache miss on escalation
Forgetting to cache the expensive model’s prompt costs a write on every retry. Cache both tiers symmetrically.
⏱️
Latency tail
Cascade’s p95 balloons on the 20 % that escalate. Time-box cheap attempts; fire both in parallel on high-stakes.

🗝️ Key Terms

Cascade
Run cheap model; escalate to premium model only if the cheap answer fails a signal.
Router
Classify each request up front and dispatch to the right model tier.
Tagger
A tiny (Haiku-scale) LLM call that assigns a category label to a request for routing.
Confidence threshold
Score below which the cheap model’s answer is rejected and sent up the chain.
Auto Router
OpenRouter’s meta-model that picks a provider per request and bills at the routed model’s rate.
Meta-model
Not Diamond’s concept: an ensemble that beats any single LLM on quality while reducing cost.
Cascade depth
Number of rungs in the model ladder (2, 3, or 4 tiers). More rungs = more savings + more complexity.
Gateway
A single endpoint (e.g., Portkey) in front of many providers that handles routing, retries, and logging.

🎯 Four Escalation Signals

🔢
Self-reported confidence
Ask the cheap model to score 0–1. Easy but poorly calibrated — never use alone.
🧱
Structured-output check
JSON/YAML/SQL validator fails → escalate. Deterministic and cheap — the cleanest signal you can get.
🗳️
Consistency vote
Call the cheap model twice with slight variation. Disagreement = escalate. Doubles the cheap-tier cost.
📏
Downstream eval
Run the output through a rubric (small LLM or deterministic test). Slower, but catches semantic failures.

🚀 Deploying a Router in Four Steps

  1. Profile your traffic

    Export a week of logs. Tag each request trivial / medium / hard. Count the distribution.

  2. Ship a hand-rolled router

    Ten lines of Python — length, keyword, code-marker heuristics. Log model, cost, and quality proxy on every request.

  3. Sample both tiers in parallel

    Run ~1 % of traffic through the expensive model alongside the router’s pick. Compare outputs weekly.

  4. Upgrade if evals demand it

    Only graduate to Not Diamond / Martian / Portkey when the eval gap justifies the vendor cost and lock-in.

🧠 Quick Quiz

What’s the difference between cascading and routing?

Cascading decides after the cheap model runs (retry on failure). Routing decides before any model runs (classify and dispatch).

Why is self-reported confidence a bad standalone escalation signal?

LLM confidence is poorly calibrated — models routinely say 0.9 and are still wrong. Pair it with a deterministic check (schema, test, similarity).

When does a router pay for its classifier cost?

When the classifier cost is smaller than the inference savings it creates — usually when the price gap between tiers is 10× or more.

Where does cascading NOT work well?

Multi-step agentic workflows, complex code reasoning, and legal/medical edge cases — small errors compound, and silent quality drops cost more than routing saves.

How do you prevent silent quality drift in a cascade?

Sample ~1 % of traffic weekly and run both the cheap and premium model in parallel, comparing outputs on a quality proxy.

🎓 Key Takeaways

  • Cascading = retry on output signal; routing = dispatch on input signal. Pick based on what you can measure.
  • Self-reported confidence alone is brittle — always pair it with a deterministic check.
  • A 3-tier router (Haiku 80 % / Sonnet 18 % / Opus 2 %) cuts a 1M-request/month support bot from $10,500 to $5,345.
  • OpenRouter for breadth, Martian for polish, Not Diamond for tuned routers, RouteLLM for OSS, Portkey for full gateway.
  • Free OSS as the first rung on high-volume workloads adds another 80 % savings — but mind the 200-reqs/day ceiling.
  • The 60–80 % savings claim is real on FAQ / classification / extraction, fake on agentic / deep-reasoning workloads.
  • Log model + cost + quality proxy on every request. Without logs, you can’t tell if the router is helping or silently degrading.
  • Routing stacks with caching, batching, and compression. Together they routinely clear 80 % total savings.
