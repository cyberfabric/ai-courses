The Big Idea

The mindset shift is simple: from “call the API and hope” to “engineer the economics.” Every API call is a purchase order. Every feature has unit economics. Run the ten-item checklist, watch the bill shrink, teach the next engineer. That’s FinOps for AI.

💸 Per-feature budgets — the 4-step habit

  1. Tag

    Every call carries feature=X, user_id=Y at the gateway/SDK.

  2. Set

    Dollar ceiling per feature per month. Start generous, tighten over time.

  3. Alert

    50% informational · 80% review · 100% page & degrade. Never a silent cutoff.

  4. Review

    Monthly 30-minute cost review. Same cadence as any business review.

💥 The $90K → $13K transformation

  1. Naive: $90,000/mo

    5M conversations, Sonnet on every turn, no optimization.

  2. + Prompt caching

    90% discount on repeated system prompt → massive input-token savings.

  3. + Context compression

    Living summaries: 3,500 → 1,200 avg input tokens.

  4. + Routing (70% Haiku, 30% Sonnet)

    Blended rate drops to ≈ $2.80 / 1M weighted.

  5. Optimized: $13,000/mo

    85% savings · $924K/year back to the business.

📖 Key terms

FinOps for AI
The discipline of managing AI spend as a measurable, accountable, engineered dimension — like latency and reliability.
Unit economics
Cost per request × requests per user × users, compared to revenue. The sentence every AI feature needs to pass.
Cost-of-AI margin
Share of feature revenue consumed by inference cost. The number finance will actually ask about.
Showback
Visibility into which customer/team costs what, without changing their bill yet. Easy first step.
Chargeback
Variable billing to customers based on their actual AI usage. Requires metering infra; aligns price-to-value.
Budget alert thresholds
50% informational, 80% review, 100% page-and-degrade. Never a silent cutoff.
Graceful degrade
When budget trips, route to a cheaper model or cached response instead of failing. Keeps the feature alive.
Cost-deflation forecast
200× median annual price drop since 2024. But usage typically outruns deflation — plan for bills to grow.

🧪 Quick quiz

What’s the #1 challenge FinOps teams cite for AI spend in 2026?

Visibility — knowing who’s spending what. Cost isn’t the blocker; attribution is.

If prices are falling 200×/year, why are bills growing?

Usage grows faster. Agents, reasoning tokens, and more AI surfaces outpace deflation. Plan for 1.5–3× bills next year, not smaller.

What’s the first step when the invoice spikes?

Check the feature tag breakdown. Is it one feature, one user, or everything? The diagnosis path depends entirely on which.

Showback or chargeback first?

Showback. Get clean data for 3 months before shipping metering infra you haven’t proven you’ll use.

What’s the single most useful habit from this chapter?

Tagging every call with a feature name. Everything else — budgets, dashboards, alerts, showback — depends on that tag existing.
