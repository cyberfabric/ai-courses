Why Cost Engineering Makes or Breaks AI Features

Memory Map
💡

The Big Idea

Every AI API call is a purchase order — it has a price, a quality, and an alternative. Cost engineering is the craft of picking the right one, every time. Features that ignore this silently turn into five-figure invoices.

🧩 Core Concepts

💸
The $10k Surprise
A naive chatbot hits 50k users, gets one big invoice, and Finance slides it into Slack. Unit economics would have predicted it.
📐
Unit Economics
Cost per request, per user per month, at 10x scale. A feature at $0.05/request can be viable; $5/request rarely is.
⚖️
The 2026 Paradox
Token prices fell ~280x in two years — but average enterprise AI budgets went from $1.2M to $7M. Usage is outrunning deflation.
🤖
Agent Multiplier
One "user action" now often means 5–30 model calls. Capacity plans built for chatbots break the first time an agent ships.
🎛️
Five Cost Levers
Model choice, caching, batching, compression, routing. Every optimization in the course pulls one. They compound.
🎲
Stochastic Costs
A prompt usually returns 400 output tokens, occasionally 4,000. Budget against distributions, not averages.

❓ The Three Cost-Discovery Questions

  1. What's the cost per request?

    Include input, output, reasoning tokens, and embeddings. Ballpark it in dollars, not percentages.

  2. What's the cost per user per month?

    Multiply by expected usage. Then compare against revenue per user. This tells you if the feature is viable.

  3. What's the cost at 10x scale?

    Linear is best case. Agents and recursive loops can scale faster than linear — verify that yours doesn't.

📖 Key Terms

Unit economics
Per-unit cost vs per-unit revenue — the financial truth of every feature.
Purchase order (mental model)
Think of every API call as having a price, a quality, and an alternative.
Agent multiplier
The number of model calls behind one user action — typically 5–30x for agentic workflows.
LLMflation
A16z's term for the aggressive year-over-year price drop at fixed model quality.
Cost-discovery question
One of three questions (per request / per user / at 10x) you should answer before shipping any AI feature.
Reasoning tokens
"Internal thinking" tokens reasoning models bill you for but never return to the user.
Stochastic cost
A cost that varies per request because the model's output length isn't deterministic.

🧠 Quick Quiz

Why are AI bills exploding in 2026 if token prices are falling?

Usage is growing faster than prices are falling. Agentic workflows use 5–30x more tokens per task, and enterprises are rolling AI out far more broadly. Per-token deflation is real; per-month bills still climb.

What are the five cost levers?

Model choice, caching, batching, compression, routing. Each chapter in Parts I–II of this course dives into one or more of them.

What's the "purchase order" mental model?

Every API call has three fields worth tracking: price, quality, and alternative. A good engineer never runs one without knowing all three.

Why is AI FinOps different from cloud FinOps?

Quality is a cost lever (not just throughput), costs are stochastic, reasoning tokens are invisible, pricing drifts in weeks, and caching is a first-class API feature.

What are the three cost-discovery questions?

Cost per request, cost per user per month, cost at 10x scale. Answer them before shipping — not after Finance sends the invoice.

🚩 Red Flags in Your Codebase

❌
No feature tag on API calls
If you can't group spend by feature, you can't optimize the right one.
❌
Same model everywhere
One premium model for every task is the "buying organic" of AI cost mistakes.
❌
System prompt rebuilt per call
Prompt caching discounts can reach ~90% on repeated input — don't leave that on the table.
❌
No per-request cost logged
If cost isn't in your logs, you'll find it in a surprise invoice instead.

Key Takeaways

  • Cost is a first-class engineering concern in 2026 — not a post-launch worry.
  • Every API call = a purchase order with a price, a quality, and an alternative.
  • The five levers (model, caching, batching, compression, routing) compound — together they cut bills 60–85% without hurting quality.
  • Deflation is real (~280x in two years) but bills still rise because usage grows faster.
  • Agent multipliers silently break capacity plans built for chatbots.
  • Before shipping, answer three questions: per request, per user, at 10x.
  • Tag every AI call with feature and user_id — attribution is impossible without them.
