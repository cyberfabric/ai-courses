Chapter 2

Tokens Are Money: The Cost Equation

Memory Map
💡

The Big Idea

cost = input_tokens × input_price + output_tokens × output_price. Every optimization in this course is a variation on shrinking one of those four factors. Output is the expensive one — 4–8x per token.

🧩 Core Concepts

🧮
The Cost Equation
Four factors — input count, input price, output count, output price. Shrink any one and you save money.
🔤
What's a Token?
~4 characters / ~0.75 English words. Different tokenizers give different counts on the same text.
⚖️
Output >> Input
Output is typically 4–8x more expensive per token than input. Trimming responses usually beats trimming prompts.
🧠
Reasoning Tokens
"Thinking" tokens billed as output but never shown. 2–10x visible output. The invisible budget buster.
📏
Context Tax
Gemini ≥200K and GPT-5.4 ≥272K roughly 2x input price. Anthropic holds a flat rate.
💸
Token Budget
Max input, max output, max reasoning, derived max cost — every feature gets all four in its design doc.

📖 Key Terms

Token
The billable unit. ~4 chars in English, different count per provider.
Tokenizer
The program that slices text into tokens. tiktoken (OpenAI), Anthropic's own, SentencePiece (Gemini).
Reasoning tokens
Internal "thinking" tokens billed as output, never returned in the response.
Extended thinking / reasoning effort
Provider dials that cap how much reasoning the model performs. Anthropic: thinking.budget_tokens. OpenAI: reasoning_effort.
Context tax
Higher per-token rate once a prompt crosses a size threshold (200K for Gemini Pro, 272K for GPT-5.4).
max_tokens
Per-request output cap. The #1 missed setting in the wild.
Token budget
Max input + max output + max reasoning, derived per-call dollar cap. Belongs in every AI feature design doc.
Cost-per-1M intuition
The mental shortcut of reading every price sheet in "per 1M tokens" units.

🚨 The Base64 Disaster (Case Study)

  1. Team inlines base64 image in prompt

    Prototype worked on the laptop. Prompt averaged ~28,000 tokens per call.

  2. Launch weekend hits 10K req/min

    At $0.12/call on Sonnet 4.6, the meter spins hard.

  3. $4,300 before anyone notices

  4. Fix: use the vision endpoint

    4 hours of work. Cost drops to $0.004/call. 30x improvement.

⚠️ Phantom-Cost Gotchas

📡
Streaming ≠ cheaper
Same billing as non-streamed. UX feature, not cost lever.
🔁
Retries compound
Blind retries on successful-but-botched responses double the bill. Use idempotency + max attempts.
✂️
Cancel ≠ refund
You pay for tokens already generated before you hit cancel.
🛠️
Function calls count
Tool-call JSON bills as output tokens. Agent turns triple fast.

🧠 Quick Quiz

Why is output usually 4–8x more expensive than input?

Because output generation is sequential — each token forces another forward pass through the model. Input is processed in one chunk. Providers price for compute, not characters.

What are reasoning tokens and how are they billed?

"Thinking" tokens the model generates internally before answering. Billed as output, never returned. Typically 2–10x the visible output count; sometimes 40x.

When does Gemini 2.5 Pro's input price roughly double?

When the prompt exceeds 200K input tokens. OpenAI's GPT-5.4 has a similar tier at 272K (2x input, 1.5x output).

What's the #1 missed setting that causes surprise bills?

max_tokens (a.k.a. max_completion_tokens). Uncapped output is the single most common cause of a five-figure invoice.

Does streaming save money?

No. Streaming is a UX feature. Billing is identical to non-streamed responses with the same token counts.

Key Takeaways

  • Memorize the equation: cost = in × in_price + out × out_price.
  • Output is 4–8x more expensive than input — trimming answers beats trimming prompts.
  • Reasoning tokens are the invisible bill multiplier. Watch reasoning_tokens in every trace.
  • Set max_tokens on every request. Default to 800. Justify anything higher.
  • Mind context thresholds: 200K (Gemini Pro) and 272K (GPT-5.4) roughly double input price.
  • Never inline base64 blobs into prompts — use the vision/files API.
  • Count tokens with the provider's free tool before shipping any variable-length prompt.
  • Four price brackets: nano, mini, standard, premium. Most features only need mini.
