Cost Observability: You Can’t Optimize What You Can’t See

Memory Map
🔌

The Big Idea

Your AI bill is like a house with no electricity meter. Until every request carries a feature tag and logs its own cost, your monthly invoice is a mystery number. Instrument first; optimize second.

📋 The Non-Negotiable Ten

🆔
Request ID
Unique, joins LLM logs back to your app logs.
🏷️
Feature tag
The single highest-leverage field. No tag = one giant line called “LLM spend.”
👤
User / tenant ID
Hashed. Enables per-customer showback and abuse detection.
🤖
Model + provider
Even after routing — the final model that served the request.
🔢
Input / output tokens
Separate fields, plus cache-read / cache-write counts.
💵
Cost USD
Computed per request from a model-rate table. Never inferred from the invoice.
⏱️
Latency
Per request — aggregate to p50/p95/p99 later.
✅
Status + quality proxy
Success / 429 / 529 / 5xx + schema-valid? thumbs-up? eval score?

🕵️ The Worked Example: Finding the Hog

  1. Before: $12,000/month, no breakdown

    Team assumed “users grew, bill grew.” Actually a tiny feature was eating the budget.

  2. Instrument with feature tags

    Add feature + user_id metadata. 80 lines of code. Wait a week.

  3. Dashboard reveals the outlier

    draft-assistant: 3 % of traffic → 62 % of the bill. Uncached 30K-token context on Opus.

  4. Apply one fix: Sonnet + cache

    Monthly bill $12,000 → $4,800. $7,200 saved in one sprint.

🚨 Alert Types Worth Wiring Up

📐
Per-feature anomaly
3σ over 30-day rolling mean. Catches prompt regressions & retry loops.
🏃
Per-user runaway
Single user > 5× their historical daily p99. Agent loop or abuse.
🪣
Cache hit-rate collapse
Drops below floor (e.g. 70 %). Usually a whitespace change broke the prefix.
🔁
Retry ratio spike
429/529 > 1 %. Capacity ceiling or upstream outage.
🧭
Model-mix drift
Cascade silently escalating everything to Opus. Signal that your cheap tier isn’t working.
📊
Budget burn-rate
Feature on pace to blow monthly cap with days left. Act before 100 %, not at 100 %.

🗝️ Key Terms

Feature tag
Metadata field like feature=summarizer that attributes every request to a specific product area.
Gateway + eval-tool pattern
The 2026 architecture: gateway (Portkey/OpenRouter) captures traffic + eval tool (Langfuse/Braintrust) adds quality.
Showback
Per-customer cost reports — transparency without charging back. Common in B2B SaaS.
Anomaly alert
Fires when a metric deviates from its rolling baseline (e.g., 3σ), not a fixed threshold.
Cache hit rate
% of requests served from prompt cache. Collapses silently when prefixes drift.
Quality proxy
A cheap signal that an answer was usable: schema valid, thumbs-up, eval passed.
Maintenance mode
Still alive (security + bugfixes) but no new features. Helicone since March 2026 acquisition.
OpenTelemetry (OTEL)
Vendor-neutral tracing standard. Phoenix, TruLens, OpenLLMetry all build on it.

🧠 Quick Quiz

Why is a feature tag on every request so important?

Without it, your dashboards collapse to one giant line called “LLM spend.” With it, you can ask “what does feature X cost this month?” and get an answer in seconds.

Should you build new integrations on Helicone in 2026?

No. Helicone is in maintenance mode since the Mintlify acquisition on March 3, 2026. Security fixes only; no new features. Mintlify is guiding customers to migrate.

What’s the gateway + eval-tool pattern?

A gateway (Portkey/OpenRouter) captures every API call with routing + retries + logs; an eval tool (Langfuse/Braintrust/Phoenix) adds quality scoring on top. Together they cover cost + quality.

Why are anomaly alerts better than threshold alerts?

Thresholds stay flat while traffic grows — false positives or silent misses. Anomaly alerts scale with the baseline and catch drift before it becomes a bill.

What’s the one fact cost observability almost always reveals?

The distribution of spend is weirder than the distribution of traffic. One small feature usually eats a disproportionate share of the bill — fix that one first.

🎓 Key Takeaways

  • Tag every request with feature, user_id, environment. Everything else is filtering on top.
  • Log the non-negotiable ten: request ID, feature, user, model, in/out tokens, cache, cost, latency, status, quality proxy.
  • Langfuse Core $29/mo is the cheapest serious tool. Braintrust Pro $249 for evals-heavy teams.
  • Helicone is maintenance-mode — don’t build new integrations; plan migration if you’re on it.
  • Gateway + eval-tool is the 2026 reference architecture. Portkey in front, Langfuse/Braintrust behind.
  • Hash user IDs before they leave your app. Observability PII is a compliance time-bomb.
  • Anomaly alerts scale; threshold alerts don’t. 3σ over a 30-day rolling mean works.
  • Build a 5-panel weekly dashboard. Glance Monday, act Monday, save Monday.
  • Start with the 5-line SDK wrapper. Graduate to a SaaS when evals or per-user dashboards become the bottleneck.
