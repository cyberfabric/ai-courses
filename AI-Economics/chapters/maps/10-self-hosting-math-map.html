Chapter 10

Self-Hosting Math: When OSS + GPUs Beat APIs

Memory Map
🚗

The Big Idea

Self-hosting LLMs is like owning a car versus taking Uber. Cheap per mile if you drive a lot; wildly expensive if it sits in the driveway. In 2026, APIs have fallen so far that the break-even only flips above ~10 billion tokens per month of steady traffic.

🎯 The four drivers

🔒
Privacy
PII, medical, legal — data can’t leave your VPC. Cost isn’t the point.
⚡
Latency
On-prem or edge for sub-50ms. APIs add 100–300ms.
💰
Cost
The hardest case to win in 2026. Needs huge steady volume.
🎛️
Control
Custom fine-tunes, guaranteed throughput, exotic sampling.

📖 Key terms

MoE (Mixture of Experts)
Activates only a fraction of total params per token. DeepSeek V3.2 = 37B active of 685B. ~85–95% compute reduction vs dense.
Ops multiplier
1.3–1.6× applied to raw GPU bill to account for engineering, failover, monitoring, on-call.
Utilization
Fraction of GPU-hours actually serving traffic. Bursty workloads = 20–40%; batch = 70–90%.
Speculative decoding
Small draft model guesses tokens, big model verifies in parallel. 2–3× speedup, no quality loss.
Quantization
Run the model in INT8 or 4-bit (GGUF, AWQ, GPTQ). ~2× throughput, 4× VRAM headroom, minimal quality loss.
Reserved capacity
1-year GPU commit at discounted hourly rate. Great for steady load, painful for scaling down.
Hybrid architecture
Self-host cheap traffic, API for hard traffic, API overflow when GPUs saturate.
Break-even volume
The monthly token volume at which self-hosting beats API on cost. In 2026: ~10B tokens/month steady.

⚠️ What the GPU sticker price hides

🔧
Reliability

🔄
Model updates
Every new model = re-quantize, re-benchmark, re-eval.
📡
Serving infra

🛡️

🧪 Quick quiz

Why doesn’t self-hosting usually beat APIs on cost in 2026?

API providers have driven prices below specialist-cloud GPU rental + ops multiplier at anything under ~10B tokens/month. DeepSeek V3.2 API is cheaper than renting your own hardware to run it.

What ops multiplier should I apply to raw GPU-hour cost?

1.3 to 1.6×. Covers engineering, failover, monitoring, on-call. Anything lower is wishful thinking.

Why is every 2026 OSS flagship a MoE?

You pay compute for active parameters, not total. An MoE activates 5–15% of its weights per token, giving ~85–95% compute reduction vs an equivalent dense model.

Which serving framework should I start with?

vLLM. Shortest path from GPU to an OpenAI-compatible endpoint. Move to SGLang or TensorRT-LLM later for the last 30% of throughput.

What’s the “50% utilization trap”?

Your GPU sits idle at nights, weekends, and dips. Real utilization is often 20–40%, not the 50% you planned for — which pushes self-hosted cost back above API pricing.

🎯 If you remember nothing else

  • 🚗 Self-hosting is like owning a car — cheap only if you drive it a lot.
  • 💰 2026 break-even: ~10B tokens/month steady before self-hosting wins on cost.
  • 🧠 Every OSS flagship in 2026 is a MoE — 85–95% compute reduction vs dense.
  • ⚡ vLLM / SGLang / TensorRT-LLM are the only serious serving choices.
  • 🏎️ Speculative decoding + 4-bit quantization can halve your $/token — stack them.
  • 🔧 Multiply raw GPU bill by 1.3–1.6× for realistic ops overhead.
  • 📉 DeepSeek V3.2 API ($0.28/$0.42) undercuts self-hosting for most workloads.
  • 🏗️ Hybrid (self-host easy + API for hard + API overflow) beats either extreme.
  • 🔒 If privacy, latency, or control drives the decision, cost math barely matters.
