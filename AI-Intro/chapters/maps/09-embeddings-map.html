🧩 Core Concepts

📍
Embeddings
Lists of 768–3,072 numbers representing the meaning of text. Similar meanings → similar numbers → close in "meaning space."
👑
Vector Arithmetic
king − man + woman ≈ queen. Relationships are encoded as directions in embedding space. The model learned this from data alone.
📐
Cosine Similarity
Score from -1 to 1 measuring how close two embeddings are. 1.0 = identical meaning. 0 = unrelated. The universal "how similar?" metric.
🔍
Semantic Search
Search by meaning, not keywords. "fix array sorting bug" finds "resolved list ordering issue." Different words, same concept = match.
🗄️
Vector Databases
Databases built to store and search embeddings fast. Pinecone (cloud), Chroma (lightweight), Weaviate (hybrid). Find nearest neighbors in milliseconds.
🔗
RAG
Retrieval-Augmented Generation: embed your docs, find relevant ones via similarity, feed them to the LLM as context. How AI "knows" your data.

📐 The Cosine Similarity Scale

Score Meaning Example
1.0 Identical meaning "dog" vs. "dog"
0.85–0.99 Very similar "dog" vs. "puppy"
0.5–0.85 Somewhat related "dog" vs. "cat"
0.0–0.5 Unrelated "dog" vs. "democracy"
< 0 Opposite / very different "hot" vs. "cold"

🔗 The RAG Pipeline

  1. 📄 Embed Your Documents

    Convert docs, code, knowledge base into vectors via an embedding model API. Do this once.

  2. 🗄️ Store in Vector Database

    Pinecone, Chroma, or Weaviate. Indexed for fast nearest-neighbor search.

  3. ❓ User Asks a Question

    Embed the question into the same vector space.

  4. 🔍 Retrieve Relevant Documents

    Find the closest documents by cosine similarity. Typically the top 3–10 results.

  5. 🧠 Generate Answer with LLM

    Pass the retrieved docs + question to the LLM as context. The response is grounded in your actual data.

This is how AI chatbots "know about your company" — and how Cursor finds the right files in your codebase.

🎨 Analogies Worth Remembering

🗺️ GPS Coordinates for Meaning
GPS gives every location 2 numbers (lat, long). Embeddings give every concept 768–3,072 numbers. Nearby coordinates = nearby meanings. More dimensions = more nuance than a flat map could ever capture.
👑 Vector Arithmetic
king − man + woman ≈ queen. Paris − France + Japan ≈ Tokyo. Relationships are directions in meaning space. The model discovered these geometric patterns from data alone.
🗺️ The Map of Meaning
If you squish 768 dimensions down to 2D, you see clusters: programming words together, food words together, sports words together. A visual confirmation that embeddings really capture meaning.

💡 The Big Idea

💡

Embeddings Turn Meaning into Math

By converting text into lists of numbers where similar meanings get similar numbers, embeddings let computers do something they never could before: understand "similar." This one idea powers semantic search, vector databases, RAG, and how coding tools find relevant files. It's the bridge between raw text and AI that can reason about meaning.

🧪 Quick Quiz

Test Yourself

What is an embedding?

A list of 768–3,072 numbers (a vector) that represents the meaning of text. Similar meanings produce similar numbers, so "dog" and "puppy" are numerically close while "dog" and "democracy" are far apart. Each dimension captures some learned aspect of meaning.

Why is king − man + woman ≈ queen significant?

It shows that embeddings encode relationships as directions in vector space. The "gender direction" points from king to queen, man to woman, uncle to aunt. Nobody programmed this — the model discovered these geometric patterns purely from reading text.

How does semantic search differ from keyword search?

Keyword search matches exact words — "fix sorting bug" won't find "resolved ordering issue." Semantic search converts both query and documents into embeddings and finds matches by meaning, so different words expressing the same concept still match.

What is RAG and why does it need embeddings?

Retrieval-Augmented Generation: embed your documents, store in a vector DB, then at query time embed the question, retrieve the most similar documents, and pass them to the LLM as context. Without embeddings, there's no way to figure out which documents are relevant — the LLM would have to guess and hallucinate.

What does a cosine similarity of 0.92 mean?

The two pieces of text are very similar in meaning. Cosine similarity ranges from -1 to 1, where 1.0 = identical meaning and 0 = unrelated. A score of 0.92 indicates the texts are in the "very similar" range — like "dog" and "puppy" or two paraphrases of the same question.

📌 Key Takeaways

If You Remember Nothing Else

  • Embeddings are GPS coordinates for meaning — lists of 768+ numbers where similar meanings get similar coordinates.
  • Relationships are directions in vector space: king − man + woman ≈ queen. Learned from data, not programmed.
  • Cosine similarity (0 to 1) measures how close two meanings are. The universal "how similar?" metric.
  • Semantic search beats keyword search: finds results by meaning, not exact words. Powers Cursor's file search.
  • Vector databases (Pinecone, Chroma, Weaviate) store and search embeddings at scale in milliseconds.
  • RAG = embed docs + store + retrieve similar + generate with LLM. The standard way to connect AI to your private data.
