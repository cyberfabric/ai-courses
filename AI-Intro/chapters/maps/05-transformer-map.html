The Transformer

Memory Map

⚔️ RNN vs. Transformer — Why It Changed Everything

Aspect RNN (Before 2017) Transformer (After 2017)
How it reads One token at a time, left to right All tokens at once, simultaneously
Memory Forgets early tokens over long text Every token can "see" every other token
Speed Sequential — can't parallelize Parallel — perfect for GPUs
Scalability Hits a wall at larger sizes Gets disproportionately better at scale
Analogy 🔑 Reading through a keyhole 📖 Seeing the entire page at once

The transformer didn't just make things a little better — it made entirely new things possible. No GPT-4, no Claude, no Gemini without it.

🧩 Core Concepts

👁️
Self-Attention
Each token looks at ALL other tokens and decides which ones matter most for understanding it in context. The core innovation.
🔍
Query / Key / Value
Like a search engine: Query = "what am I looking for?", Key = "what do I offer?", Value = "my actual content." Match queries to keys, pull values.
🎭
Multi-Head Attention
32–128 attention mechanisms running in parallel, each looking for a different type of relationship: grammar, semantics, structure, temporal.
📍
Positional Encoding
Since the transformer sees all tokens at once, it needs explicit position tags. Modern models use relative positions (RoPE) instead of absolute.
🔄
Encoder vs. Decoder
Encoder = understand text (BERT). Decoder = generate text (GPT, Claude). Every chatbot and code assistant is a decoder-only transformer.
📈
Scaling Laws
Bigger transformers don't just get a little better — new capabilities emerge at scale. 1B: complete sentences. 100B+: reason and write code.

🚀 Scaling: What Emerges at Size

  1. 1B Parameters

    Basic text completion. Can finish sentences sensibly but can't follow complex instructions.

  2. 10B Parameters

    Instruction following emerges. The model can understand and execute basic requests.

  3. 100B+ Parameters

    Complex reasoning, coding, puzzle-solving. These abilities weren't programmed — they emerged from scale.

  4. 1T+ Parameters (GPT-4 scale)

    Nuanced reasoning, multi-step problem solving, understanding subtle context across long documents.

🎨 Analogies Worth Remembering

🔑 The Keyhole (RNN)
Reading a novel through a keyhole, one word at a time. By page 50, you've forgotten page 1. Transformers see the whole page at once.
🔍 The Search Engine (Q/K/V)
Query = your search term, Key = page titles, Value = page content. Attention runs thousands of searches simultaneously for every token.
💻 Multiple Grep Searches (Multi-Head)
Each attention head is like a different grep pattern running on the same text — one finds grammar, another finds semantics, another tracks code structure.
📖 GPT Decoded
Generative (decoder — it generates text), Pre-trained (trained on massive data first), Transformer (this architecture). Every word matters.

💡 The Big Idea

💡

Attention Is the Reason Modern AI Understands Context

Every time Claude Code infers a variable's type, ChatGPT follows a chain of reasoning, or Cursor understands your codebase conventions — it's attention heads at work, connecting relevant tokens across the entire input from dozens of perspectives simultaneously. One simple question powers it all: "Which other tokens should I pay attention to?"

🧪 Quick Quiz

Test Yourself

What were the two main problems with RNNs that transformers solved?

Forgetting — RNNs lost information about early tokens as the sequence grew longer. Speed — RNNs processed tokens sequentially, so you couldn't parallelize them with more GPUs. Transformers solve both by processing all tokens simultaneously.

How does attention resolve the ambiguity in "I went to the bank"?

The attention mechanism compares "bank" against all other tokens in the sentence. If the surrounding tokens include "deposit" and "money," the attention scores are high for those financial-context words, so "bank" gets enriched with financial meaning. If instead the context includes "river" and "flow," the attention shifts to nature-related meaning.

What do Query, Key, and Value represent in the attention mechanism?

Query = "What am I looking for?" (what context does this token need). Key = "What do I offer?" (what context can this token provide). Value = "What information do I carry?" (the actual content to pass along). High query-key match → high attention → that token's value gets pulled in.

Why is generation (output) slower than reading input in a transformer?

Input tokens can all be processed in parallel since they're known upfront. But output tokens must be generated one at a time — each new token depends on all the previous ones. Reading is parallel; writing is sequential.

Are GPT-4 and Claude encoder or decoder models?

They're decoder-only transformers. "Decoder" means they generate text token by token. "Only" means there's no separate encoder — the same architecture both understands input and generates output. Encoder-only models like BERT are used for understanding tasks (search, classification) but can't generate text.

📌 Key Takeaways

If You Remember Nothing Else

  • Transformers replaced RNNs by letting every token look at every other token simultaneously — no more forgetting, no more sequential bottleneck.
  • Self-attention is the core: for each token, the model dynamically decides which other tokens are most relevant to understanding it.
  • Query/Key/Value works like a search engine — find what's relevant, pull the information.
  • Multi-head attention runs 32–128 perspectives in parallel: grammar, meaning, code structure, and more.
  • GPT, Claude, Gemini, and Llama are all decoder-only transformers — they generate text one token at a time.
  • Bigger transformers get disproportionately better. New capabilities emerge at scale — reasoning, coding, and creativity weren't programmed, they appeared.
