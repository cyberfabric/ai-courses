🧩 Core Concepts

🚫
No Memory
Zero persistent state between requests. Every response is generated from scratch. The "conversation" is an illusion built by the app, not the model.
🔧
System Prompt
Hidden instructions before everything else. CLAUDE.md contents go here. You "pay" these tokens before typing your first message — but they're worth it.
📦
Context Window
Max tokens in a single request: system + history + your message + response. Shared budget. It's RAM, not a hard drive.
✂️
Truncation
When conversations exceed the window, oldest messages are silently dropped. The model literally can't see them anymore. This is why AI "forgets."
📍
Lost in the Middle
Models are best at info at the beginning and end of context. Details buried in the middle get overlooked. Put important instructions first or last.
🗄️
KV Cache
Stored keys & values for every token across every layer. Grows with conversation length. Major reason longer contexts cost more GPU memory.

📈 Context Window Evolution

  1. 4K tokens — GPT-3.5 (2022)

    ~3,000 words. A few pages. Conversations got truncated quickly.

  2. 32K tokens — GPT-4 (2023)

    ~24,000 words. A long article. Meaningful conversations possible.

  3. 128K–200K tokens — GPT-4 Turbo, Claude 3.5 (2024)

    ~96K–150K words. A short novel. Full codebases start to fit.

  4. 1M tokens — Gemini 1.5, Claude (2025)

    ~750K words. Several novels. But bigger ≠ always better.

🎨 Analogies Worth Remembering

🧠 The Amnesia Friend
Your friend has complete amnesia. Every time you talk, you hand them your full conversation journal. They read it, give a great response, then immediately forget. Next time, you hand them the journal again — now one entry longer.
💾 RAM, Not a Hard Drive
The context window is working memory — what the model can actively think about right now. Fixed size, nothing persists after the session. More RAM helps, but doesn't make the CPU smarter.
📄 Page 237 of 500
Giving the model 500 pages when the answer is on page 237 is worse than just giving it page 237. More context = more to sort through = more chance of missing what matters.

💡 The Big Idea

💡

AI Conversations Are an Illusion — And That's OK

The model has zero memory. Every "conversation" is the app re-sending the entire history as one giant prompt. When the history outgrows the context window, old messages are silently dropped — that's why AI "forgets." Understanding this illusion changes how you use AI: start fresh when things go sideways, curate your context carefully, and put important information at the beginning or end — never buried in the middle.

🧪 Quick Quiz

Test Yourself

Does the AI model "remember" your previous messages?

No. The model has zero memory between requests. The application (ChatGPT, Claude, etc.) re-sends the entire conversation history as a single prompt each time. The model processes everything from scratch — it doesn't even know it's in a "conversation."

What are the three layers of the conversation stack?

1. System prompt — hidden instructions (behavior, CLAUDE.md). 2. Conversation history — all previous messages and responses replayed in order. 3. Your latest message — appended at the end so the model has maximum context when generating.

Why does AI "forget" things in long conversations?

Because old messages are literally removed from the prompt (truncated) when the conversation exceeds the context window. The model can't reference what it can't see. It's a space constraint, not confusion.

What is the "lost in the middle" problem?

Models are best at using information at the beginning and end of the context window. Information buried in the middle of a long prompt is more likely to be overlooked. This is why important instructions should go at the start (system prompt) or end (just before your question).

Why does CLAUDE.md work?

CLAUDE.md contents are included in the system prompt — the very first part of the context — with every single request. The model sees it every time, giving it permanent context about your project's conventions, architecture, and preferences.

📌 Key Takeaways

If You Remember Nothing Else

  • AI has zero memory. "Conversations" work by re-sending the entire history with every message.
  • The context window is a shared token budget: system prompt + history + your message + response. It's RAM, not storage.
  • When conversations exceed the window, old messages are silently dropped — that's why AI "forgets."
  • Bigger context ≠ better results. Cost, speed, and "lost in the middle" attention degradation all push back.
  • Curated context beats dumped context. The right 5 files outperform 50 irrelevant ones.
  • When a long conversation goes sideways, start fresh — summarize key findings in your first message of the new chat.
