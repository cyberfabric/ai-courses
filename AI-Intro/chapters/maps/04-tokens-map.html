The model never sees text. It works entirely in numbers. The tokenizer is the translator between your world and the model's.

🧩 Core Concepts

✂️
Tokenization
Breaking text into chunks the model can process. Tokens aren't words — they can be whole words, word parts, or single characters.
🔗
BPE (Byte Pair Encoding)
Start with characters, find the most common pair, merge it into one token. Repeat ~100K times. Common patterns get short codes.
🍓
The Strawberry Problem
AI can't count R's in "strawberry" because it sees ["str", "aw", "berry"] — tokens, not letters. Character-level tasks fight the architecture.
💰
Token = Currency
Everything in AI is measured, priced, and limited by tokens. Input tokens are cheaper; output tokens cost more (each needs a full network pass).
📦
Context Window
Max tokens in a single request — input + output combined. 128K tokens ≈ 96K words, and it's a shared budget between prompt and response.
🌍
Language Inequality
English is most efficient (~6 tokens for a greeting). Japanese or Arabic can cost 2-3× more tokens for the same meaning.

🏗️ How BPE Builds a Vocabulary

  1. 📝 Start with Characters

    Every letter, digit, and symbol is its own token. "cat" = 3 tokens: "c", "a", "t".

  2. 🔍 Find Most Common Pair

    Scan all training data. Maybe "t" + "h" appear together millions of times.

  3. 🔗 Merge Into New Token

    "th" becomes a single token. Vocabulary grows by one.

  4. 🔁 Repeat ~100K Times

    Keep merging the most common pairs. Result: "the" = 1 token, "unconditionally" = 3 tokens, rare words = many tokens.

📏 Token Sizes in Practice

Content Approx. Tokens Key Insight
Short prompt ~50 ~35 words
Typical email ~200 ~150 words
Page of text ~400 ~300 words
Source file 1,000–3,000 Code is token-hungry
A novel (80K words) ~100K Nearly fills a 128K context

Rule of thumb: 1 token ≈ 0.75 English words. Or: 100 words ≈ 130–140 tokens.

🎭 Analogies Worth Remembering

📦 The Compression Dictionary
BPE is like zipping a file — common patterns get short codes (1 token), rare patterns are spelled out (many tokens). The tokenizer is optimized for the most frequent text patterns.
🍓 Counting From Across the Street
"How many R's in strawberry?" is like "how many bricks in this wall?" when you can only see it from across the street. The model sees chunks, not individual characters.
🪙 "don't" = Two Concepts
Tokenization is like how you read "don't" — one written form, but actually two concepts (do + not). The model splits text at conceptual boundaries.

💡 The Big Idea

💡

Tokens Are the Fundamental Currency of AI

Everything is measured in tokens: pricing, speed, context limits, rate limits, model comparisons. AI doesn't see your text — it sees chunks converted to numbers. Understanding tokens explains why some tasks are cheap and fast, others expensive and slow, and why letter-counting stumps a system that can write code.

🧪 Quick Quiz

Test Yourself

Why can't AI reliably count the letters in "strawberry"?

Because the model sees tokens like ["str", "aw", "berry"], not individual characters. It has to reason about what letters exist inside each token based on pattern memory, not direct inspection. Character-level tasks fundamentally fight the tokenization architecture.

What does BPE stand for and how does it work?

Byte Pair Encoding. It starts with individual characters, finds the most common adjacent pair in the training data, merges it into a single token, and repeats until reaching ~100K tokens. The result: common words are single tokens, rare words are split into many.

Why are output tokens more expensive than input tokens?

Each output token requires a full forward pass through the entire neural network (billions of parameters, dozens of layers). Input tokens are processed together in parallel, but output tokens must be generated one at a time sequentially.

If you have a 128K context window, how many words is that roughly?

About 96,000 words (128K × 0.75). And that's the total budget shared between your prompt AND the model's response. If your prompt uses 100K tokens, only 28K tokens remain for the output.

Why does the same sentence in Japanese cost more than in English?

Tokenizers are trained mostly on English text, so English gets the most efficient tokenization (common words = single tokens). Japanese characters are rarer in training data and may each need their own token, resulting in 2-3× more tokens for the same meaning.

📌 Key Takeaways

If You Remember Nothing Else

  • AI reads tokens, not words or characters. A token can be a word, part of a word, or a single character.
  • BPE builds the vocabulary by merging common pairs. Common text = few tokens (cheap). Rare text = many tokens (expensive).
  • The "strawberry problem": AI can't see individual characters inside tokens, so letter-counting is architecturally hard.
  • 1 token ≈ 0.75 English words. Code uses more tokens than prose. Non-English languages use 2-3× more than English.
  • Context window is tokens, shared between input and output. 128K tokens ≠ 128K words.
  • Tokens are the currency of AI — pricing, speed, limits, and capabilities all revolve around them.
