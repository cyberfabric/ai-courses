Chapter 7

Temperature, Top-p & Friends

Memory Map

🌡️ The Temperature Scale

  1. Temperature 0 — Deterministic

    Always picks the #1 token. Same input = same output every time. Best for code, facts, JSON.

  2. Temperature 0.3–0.5 — Slight Variety

    Mostly predictable with minor variation. Good for technical writing, code review.

  3. Temperature 0.7–1.0 — Balanced

    Natural, varied text. The "default" range. Good for conversation and explanations.

  4. Temperature 1.0–1.5 — Creative

    Surprising, unexpected choices. Good for brainstorming and creative writing.

  5. Temperature >1.5 — Wild

    Often incoherent. Rarely useful in practice. Experimental territory only.

⚔️ The Three Sampling Strategies

Parameter What It Does Adaptive? Best Use
🌡️ Temperature Reshapes all probabilities (sharper or flatter) ❌ Same effect everywhere Direct creativity control
🎯 Top-p Only considers tokens within top p% probability ✅ Adjusts per token Generally preferred
🔢 Top-k Only considers the top k tokens ❌ Fixed number always Simple but less flexible

Key insight: Top-p is adaptive — few candidates when confident, many when uncertain. Temperature and top-k are not.

🧩 All the Parameters

🌡️
Temperature
The "creativity dial." Low = predictable, high = wild. Controls how sharp or flat the probability distribution is before sampling.
🎯
Top-p (Nucleus)
Smart filter: only consider tokens whose cumulative probability reaches p. Adapts automatically — fewer options when confident, more when uncertain.
🔢
Top-k
Simple filter: only consider the top k tokens. k=1 is deterministic, k=50 allows variety. Not adaptive — always exactly k candidates.
📏
Max Tokens
Upper limit on output length. Model may stop earlier (stop token), but never exceeds this. Only counts output, not input.
🛑
Stop Sequences
Specific strings that halt generation instantly. Useful for structured output — e.g., stop at "\n\n" to get exactly one paragraph.
🔁
Frequency / Presence Penalty
Frequency: penalizes tokens proportional to how often they appeared. Presence: penalizes any token that appeared at all. Both fight repetition.

🎨 Analogies Worth Remembering

🗺️ GPS Adventure Level (Temperature)
Low = fastest highway every time. Medium = scenic alternative routes. High = wild detours through unknown villages. The "adventure level" of token selection.
🎭 Same Actor, Different Roles (Parameters)
The same model can be a precise engineer (temp 0) or a wild artist (temp 1.5). The only difference is these generation parameters. One model, infinite personalities.
⚠️ Don't Mix Carelessly
Setting temperature high AND top-p low makes them fight each other. Adjust one, leave the other at default. When in doubt: top-p = 0.95, temperature = 1.

💡 The Big Idea

💡

Temperature Is the Single Most Impactful AI Setting

Most AI "quality issues" are really temperature problems — too high produces incoherent output, too low produces repetitive output. The same model produces wildly different results just by turning this one knob. For code: keep it low. For brainstorming: crank it up. For everything else: 0.7–1.0 is your sweet spot.

🧪 Quick Quiz

Test Yourself

What does temperature 0 do?

Always picks the single most likely token. No randomness at all — deterministic. Same prompt always produces the same output. Best for code generation, factual Q&A, and structured data extraction where correctness matters more than variety.

Why is top-p generally preferred over top-k?

Top-p is adaptive. When the model is confident (one token dominates), very few tokens pass the filter. When uncertain (many plausible options), more tokens pass. Top-k always considers exactly k tokens regardless of the probability distribution, which can be too many or too few.

What temperature would you use for writing Python code vs. brainstorming startup ideas?

Python code: 0–0.3 (low — you want correct syntax, predictable patterns, and conventional structure). Brainstorming: 1.0–1.3 (high — you want maximum variety and unexpected ideas, and the occasional "wrong" path might spark creativity).

What's the difference between frequency penalty and presence penalty?

Frequency penalty scales with how many times a token has appeared — used 5 times gets penalized more than used once. Presence penalty is binary — any token that has appeared at all gets the same penalty. Frequency reduces overused words; presence encourages entirely new topics.

Why shouldn't you set both temperature and top-p to extreme values simultaneously?

They can fight each other. High temperature (more randomness) + low top-p (fewer candidates) produces conflicting signals. The recommendation is to adjust one and leave the other at its default. When in doubt: top-p ≈ 0.95, temperature ≈ 1.

📌 Key Takeaways

If You Remember Nothing Else

  • Temperature is the creativity dial: 0 = deterministic, 1 = default, >1 = increasingly wild and risky.
  • Top-p is the smart filter: only considers tokens within the top p% of probability. Adaptive and generally preferred over top-k.
  • Code tools use low temperature (0–0.3) for correctness. Creative tools use higher temperature for variety.
  • Most AI "quality issues" developers report are really temperature problems.
  • Don't combine temperature and top-p aggressively — adjust one, leave the other at default.
  • The same model can be a precise engineer or a wild artist — the only difference is these parameters.
