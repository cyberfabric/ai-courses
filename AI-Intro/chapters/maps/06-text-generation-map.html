🧩 Core Concepts

1️⃣
One Token at a Time
LLMs predict a single next token, append it, and repeat. Every response you've ever seen was built this way — no planning ahead.
📊
Probability Distribution
At each step, the model scores all ~100K tokens. "Paris" might get 92%, "Lyon" 0.4%, "banana" 0.00001%. The shape depends on context.
🎲
Sampling
Randomly picking from top candidates (weighted by probability). This randomness is a feature — it's what makes AI output feel natural and creative.
🦋
The Butterfly Effect
One different token early on cascades into a completely different response. Same prompt, different answers — every time.
👻
Hallucination
The model generates plausible-sounding tokens, not necessarily correct ones. It has no mechanism to prefer truth over plausibility.
⏩
Streaming
Words appearing one by one isn't animation — it's the actual generation happening in real time. Each word = one full forward pass.

⚔️ Greedy vs. Sampling

Aspect Greedy Decoding Sampling

👻 Why AI Hallucinates

  1. You ask about something obscure

    A rare API, a niche historical event, or something not well-represented in training data.

  2. The model must still generate tokens

    It has no "I don't know" circuit. It always produces the most plausible-sounding continuation.

  3. Plausible ≠ correct

    The generated method name follows naming conventions and fits the context — but doesn't actually exist.

  4. Confidence without knowledge

    The model has no uncertainty detector. It delivers the hallucination with the same confidence as a real fact.

This isn't a bug — it's the system working as designed. Always verify facts, API references, and library names.

⚡ Speed: Reading vs. Writing

Operation How Speed
Processing your prompt All tokens in parallel ⚡ Fast (even 10K tokens)
Generating the response One token at a time 🐢 Slow (~50-100 tokens/sec)

The brief pause after hitting Enter = fast parallel processing. The gradual text appearance = slow sequential generation.

🎨 Analogies Worth Remembering

📱 Autocomplete on Steroids
Your phone suggests 3 next words. An LLM scores 100,000 — then picks one automatically based on learned probabilities. That's the entire mechanism behind every AI response.
🎷 Jazz Improvisation
A jazz musician knows the "correct" note but deliberately plays unexpected ones for tension and surprise. AI sampling works the same way — slight randomness creates more interesting output.
🦋 The Butterfly Effect
Picking "However" instead of "But" as the first word cascades into a completely different response. One token changes everything that follows.
🗺️ GPS Always Taking the Popular Road
Greedy decoding (always pick #1 token) is like a GPS that only takes the most popular route — you'd drive the same path every time. Sampling explores alternatives.

💡 The Big Idea

💡

LLMs Predict Plausible Next Tokens — Not Truth

The model's entire job is to generate the most plausible continuation, one token at a time. Most of the time, "plausible" and "correct" overlap. When they don't, you get hallucination — delivered with the same confidence as a real fact. Randomness in sampling is what makes output creative and varied, but also means the same prompt gives different answers every time. Understanding this one mechanism explains creativity, hallucination, streaming, and variability — all at once.

🧪 Quick Quiz

Test Yourself

What is autoregressive generation?

Generating text one token at a time, where each new token is predicted based on everything before it (original prompt + all previously generated tokens). The token is appended and the process repeats until a stop token or max length is reached.

Why does greedy decoding (always picking the #1 token) produce bad output?

It gets stuck in repetitive loops. The most probable next token after a sentence tends to restart the same pattern, so the model keeps generating the same phrases over and over. Randomness through sampling breaks these loops and produces natural, varied text.

Why does the same prompt give different responses each time?

Because sampling is random. At each step, the model randomly selects from the probability distribution. A different choice at any step cascades — since the next token's probabilities depend on the previous one, one early difference leads to a completely different response.

Is hallucination a bug that will be fixed?

No. It's a fundamental property of next-token prediction. The model generates plausible tokens, not necessarily correct ones. It has no built-in mechanism to prefer truth over plausibility. Models are improving at saying "I don't know" through fine-tuning, but the underlying architecture will always generate something.

Why is generating a response slower than processing the prompt?

The prompt is processed with all tokens in parallel (one pass). But each output token needs its own full forward pass through the entire network, and you can't generate token #5 until tokens #1–4 exist. Generation is inherently sequential — that's why output tokens also cost more.

📌 Key Takeaways

If You Remember Nothing Else

  • LLMs generate one token at a time — predict, append, repeat. No planning ahead, no global structure. The response emerges step by step.
  • At each step, the model scores all ~100K tokens with probabilities. Sharp distributions = confident answers. Flat = creative variety.
  • Sampling (controlled randomness) is what makes AI feel alive. Greedy decoding produces repetitive garbage.
  • Same prompt, different answers — because one different random token choice cascades into a completely different response.
  • Hallucination is inherent: the model generates plausible tokens, not correct ones. Always verify facts and API references.
  • Reading your prompt is fast (parallel). Generating the response is slow (sequential). Streaming shows the real-time generation, not an animation.
