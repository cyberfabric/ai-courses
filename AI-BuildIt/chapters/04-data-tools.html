Chapter 4

Data Wrangling & Analysis Tools

Build tools that let you talk to your data instead of wrestling with pandas and SQL

The Data Accessibility Problem

Everyone has data. CSVs from marketing. JSON dumps from APIs. Database exports from the analytics team. That spreadsheet your PM swears contains "everything we need."

The problem? Getting answers out of data requires knowing pandas, SQL, or Excel formulas. And most of the time, the question is simple: "What were our top 10 products last month?" or "How many users signed up from Europe?"

Simple questions. But to answer them, you need to remember whether it's GROUP BY or groupby(), whether the date column is a string or a datetime, and why pandas is throwing a KeyError on a column that definitely exists.

73% of data questions are simple lookups
~20 min to write a "quick" pandas query
10 sec to ask the question in English

In Chapter 3, we generated code from descriptions. Now we're generating answers from data. Same AI, different superpower. These tools let anyone — developers, PMs, designers, whoever — ask questions in plain English and get real answers.

The Projects

CSV Whisperer
Weekend
Python Claude API pandas matplotlib

Load any CSV file and ask questions in plain English. "What's the average order value by country?" "Show me a chart of signups per month." "Which customers haven't ordered in 90 days?" Just ask — the tool writes and runs the pandas code for you.

What it does: Reads a CSV, sends the schema (column names, types, sample rows) to Claude, translates your English question into pandas code, executes it safely, and returns the answer — optionally with a chart.

Why it's cool: You can hand this tool to anyone on your team. Your PM can query the sales data. Your designer can explore user behavior. No pandas knowledge required. Just questions.

How to build it:

  1. Load the CSV with pandas and extract: column names, dtypes, row count, and 5 sample rows
  2. Send the schema to Claude along with the user's question
  3. System prompt: "You are a data analyst. Given this DataFrame schema, write pandas code to answer the question. Output ONLY executable Python code. Use the variable df for the DataFrame."
  4. Execute the generated code in a sandboxed environment with exec()
  5. Capture the output (print statements, return values) and display the result
  6. If the question asks for a chart, detect that and save/display a matplotlib figure

Running AI-generated code with exec() is powerful but dangerous. Always sandbox it: restrict imports, set memory limits, and timeout long-running queries. Never let generated code access the filesystem or network. Use RestrictedPython or run in a subprocess with resource limits.
Send the schema, not the data. Don't dump 15,000 rows into the prompt. Send column names, types, a few sample rows, and basic stats (min, max, unique counts). Claude generates better code from a clean schema than from a wall of raw data — and it's cheaper too.
Data Cleaner
Weekend
Python Claude API pandas

Feed it messy data and describe what "clean" looks like. "Remove duplicates, standardize phone numbers, fill missing zip codes from city names, and flag rows where the email looks invalid." It does all of it in one pass.

What it does: Analyzes dirty data, identifies problems (duplicates, inconsistent formats, missing values, outliers), and generates a cleaning pipeline that you can review, approve, and execute.

Why it's cool: Data cleaning is 80% of every data project. It's tedious, error-prone, and different every time. This tool turns it from a multi-hour slog into a 5-minute conversation.

How to build it:

  1. Load the data and run automatic profiling: missing values, duplicates, type mismatches, outliers
  2. Send the profile to Claude: "Here's a data quality report. Suggest a cleaning plan."
  3. Display the plan to the user for approval (never auto-clean without consent)
  4. On approval, generate and execute the pandas cleaning code
  5. Show a before/after comparison: rows removed, values fixed, columns standardized
  6. Export the cleaned data and save the cleaning script for reproducibility

SQL Query Builder
Weekend
Python Claude API SQLite / PostgreSQL

Describe what you want in English, get the SQL query. Optionally connect to a database, run it, and explain the results. "Show me all users who signed up in March but haven't made a purchase" becomes a working query in seconds.

What it does: Connects to your database (or reads a schema file), translates English questions into SQL, optionally runs the query, and explains the results in plain language.

Why it's cool: SQL is a language you use just often enough to forget the syntax. This tool remembers it for you. And it doesn't just write the query — it explains what it did and why.

How to build it:

  1. Connect to a database or read a schema definition file
  2. Extract table names, column names, types, and relationships
  3. Send the schema + question to Claude: "Write a SQL query for this database to answer this question"
  4. Display the query for review before executing
  5. With --run, execute the query and display results as a formatted table
  6. With --explain, get a plain English explanation of what the query does and why

Never run AI-generated SQL against a production database without review. Add these safety guardrails: read-only mode by default (no INSERT/UPDATE/DELETE unless --write flag is set), query timeout limits, and row count limits. Better yet, point the tool at a read replica.

What it does: Connects to spreadsheets, understands the structure, and performs analysis and transformations through conversation. It reads, computes, and writes back — all from English instructions.

How to build it:

Detect chart requests by looking for keywords: "chart", "plot", "graph", "visualize", "show me". When you see one, switch to the chart-generating system prompt.

For interactive charts, use plotly instead of matplotlib. It generates HTML files with zoom, hover, and pan. Your PM will think you're a wizard when you hand them an interactive revenue chart they can explore.

The Safety Layer

Data tools that modify things need guardrails. Here's the safety pattern every data tool in this chapter should follow:

Analyze → Preview → Confirm → Execute → Backup
  1. Analyze — understand the data and the request
  2. Preview — show what will change before changing it (first 5 rows, summary stats)
  3. Confirm — require explicit user approval for any write operation
  4. Execute — perform the operation
  5. Backup — always save the original data before modifying (auto-save a .bak file)

This pattern applies to the Data Cleaner (modifying rows), the Spreadsheet Automator (writing cells), and the SQL Builder (if you enable write mode). Never auto-execute writes.

Handling Large Datasets

Claude has a context window, and your CSV might have 500,000 rows. Here's how to handle data that doesn't fit:

Chunking

Process data in chunks of 1,000-10,000 rows. Aggregate results across chunks:

import pandas as pd

chunks = pd.read_csv("huge_file.csv", chunksize=5000)
results = []
for chunk in chunks:
    # Process each chunk with the AI-generated code
    result = process_chunk(chunk)
    results.append(result)

final = pd.concat(results)

Schema-First Approach

Don't send data to the AI. Send the schema and let it generate code that runs locally on the full dataset. This is what the CSV Whisperer does — Claude never sees your actual data, just the column names and a few sample rows.

Summarization

For analysis questions, pre-compute summaries (aggregations, distributions, correlations) and send those to Claude. "Here are the monthly totals. What trends do you see?" is a small prompt that produces smart analysis.

What if you combined the CSV Whisperer and SQL Builder into one universal tool? Imagine a single command: talkdata. It auto-detects the data source — CSV, JSON, SQLite, PostgreSQL, Google Sheets — loads the schema, and lets you ask questions in English regardless of format. "Talk to your data" in the most literal sense. Add conversation history so follow-up questions understand context: "Now filter that to just European customers" works because it remembers the previous query. Add chart generation and you've got a personal data analyst that works with anything.

Build It: CSV Whisperer in 20 Minutes

Let's build the CSV Whisperer. Grab any CSV file you have — a bank export, a spreadsheet download, or a sample dataset — and let's make it talk.

Step 1: Set up (2 min)

mkdir csv-whisperer && cd csv-whisperer
python -m venv .venv && source .venv/bin/activate
pip install anthropic pandas matplotlib

Step 2: Create csvwhisper.py (12 min)

import click
import anthropic
import pandas as pd
import sys, io

client = anthropic.Anthropic()

SYSTEM = """You are a data analyst. You have a pandas DataFrame
called `df`. Given the schema below, write Python code to answer
the user's question.

Output ONLY executable Python code. Use print() to show results.
For charts, use matplotlib and call plt.savefig('chart.png').

Schema:
{schema}"""

def get_schema(df):
    buf = io.StringIO()
    df.info(buf=buf)
    info = buf.getvalue()
    sample = df.head(3).to_string()
    stats = df.describe().to_string()
    return f"{info}\n\nSample rows:\n{sample}\n\nStats:\n{stats}"

@click.command()
@click.argument("csvfile")
def main(csvfile):
    df = pd.read_csv(csvfile)
    schema = get_schema(df)
    click.echo(f"Loaded: {csvfile} ({len(df)} rows, {len(df.columns)} columns)")
    click.echo(f"Columns: {', '.join(df.columns)}\n")

    while True:
        question = input("> ").strip()
        if question.lower() in ("quit", "exit", "q"):
            break

        system = SYSTEM.format(schema=schema)
        response = client.messages.create(
            model="claude-sonnet-4-20250514",
            max_tokens=1024,
            system=system,
            messages=[{"role": "user", "content": question}]
        )

        code = response.content[0].text
        # Strip markdown code fences if present
        code = code.replace("```python", "").replace("```", "").strip()

        try:
            exec(code, {"df": df, "pd": pd, "plt": __import__("matplotlib.pyplot")})
        except Exception as e:
            click.echo(f"Error: {e}")
        print()

if __name__ == "__main__":
    main()
