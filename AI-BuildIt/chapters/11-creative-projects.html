Build creative tools for fun, art, games, and writing — because not everything needs to be productive

AI as a Creative Partner

We've spent 10 chapters building serious, useful tools. CLI utilities, dashboards, APIs, bots, automation pipelines. All valuable. All solving real problems.

Now let's build something fun.

Creative projects are where AI gets magical. Not because the technology is different — it's the same Claude API you've been using. But because the output is delightful. A story that surprises you. A dungeon that tries to kill your character. A commit message written as a haiku.

Here's the important thing: AI is a creative partner, not a replacement. The best creative AI tools don't generate art while you watch. They generate art with you. You provide the spark, AI amplifies it, you refine it, and the result is something neither of you would have created alone.

100% fun-to-work ratio

In Chapter 10, we automated serious workflows. Now we're automating joy. Let's build things that make people smile.

The Projects

Story Collaborator
Weekend
Python Claude API Flask SQLite

You and AI take turns writing a story. You write a paragraph, AI writes the next, back and forth. But this isn't random generation — the AI maintains plot threads, character voices, and story arc. It remembers that the detective has a limp and the villain's motive is revenge.

What it does: A collaborative writing tool where human and AI alternate paragraphs. The system maintains a "story bible" — characters, plot points, setting details, unresolved threads — so the AI writes coherently over hundreds of paragraphs.

Why it's cool: Writing fiction alone is hard. Writer's block is real. But when you have a partner who never runs out of ideas, who can take your "the detective enters the warehouse" and turn it into a scene you didn't expect? That's when writing becomes play.

How to build it:

  1. Build a Flask web app with a simple text editor interface
  2. Maintain a "story bible" in the system prompt: character list, plot summary so far, setting, tone, unresolved threads
  3. After each AI turn, update the story bible by asking Claude to extract new characters, plot developments, and open questions
  4. Add controls: genre selection, tone slider (serious ↔ comedic), pacing (slow burn ↔ action)
  5. Save stories to SQLite with chapter breaks and the ability to branch (try different paths)
  6. Add a "surprise me" button that introduces an unexpected plot twist
Story Collaboration Session
Genre: Noir Detective | Tone: Dark humor You: The phone rang at 3am. Nobody calls at 3am with good news. Detective Mara Chen picked it up anyway, because bad news was her business. AI: "Someone stole the mayor's cat," said the voice on the other end. Chen almost hung up. Almost. But something in the caller's voice — a tremor that had nothing to do with a missing tabby — made her reach for her coat instead. In twenty years on the force, she'd learned that stolen cats were never just stolen cats. You: The mayor's mansion smelled like old money and new lies. A butler who looked like he'd been embalming himself led her to the study. AI: Mayor Rodriguez sat behind a desk that cost more than Chen's apartment. He wasn't crying about a cat. He was sweating. And when a man with three offshore accounts and a secret second family sweats, it's not about the cat. "Detective," he said, "I need you to find Mr. Whiskers before Friday." He didn't say why. He didn't have to — Chen already noticed the ransom note barely hidden under his whiskey glass.
The key to great collaborative fiction is the story bible. Without it, the AI "forgets" details by turn 10. Update it after every exchange: new characters, revealed information, open questions. Think of it as the AI's memory of the story so far. Keep it under 2,000 tokens and it fits comfortably in the system prompt.
Dungeon Master Bot
Week
Python Claude API Discord.py SQLite

A text-based RPG game master that runs dynamic adventures in Discord. It describes scenes, manages combat, tracks inventory, generates ASCII maps, and adapts the story based on player choices. Full persistent world state — your character carries over between sessions.

What it does: Acts as a dynamic tabletop RPG game master: generates adventure scenarios, describes environments, resolves player actions with dice-roll mechanics, manages NPCs, and maintains a persistent world state.

Why it's cool: D&D without needing to find a DM. The AI adapts to your play style: cautious players get mystery and intrigue, aggressive players get epic battles. And because it's in Discord, your whole friend group can play together.

How to build it:

  1. Create a Discord bot with channels for different game areas (#tavern, #dungeon, #marketplace)
  2. Build a world state system in SQLite: player stats, inventory, location, quest progress, NPC relationships
  3. System prompt includes: world lore, current scene, player stats, available actions
  4. Parse player commands: "I search the room", "I attack the goblin", "I talk to the innkeeper"
  5. Resolve actions with simulated dice rolls for skill checks and combat
  6. Generate ASCII maps for dungeons and towns
  7. Save world state between sessions — come back tomorrow and continue the adventure

Commit Message Poet
Weekend
Python Claude API Git hooks

A git hook that rewrites your boring commit messages as haikus, limericks, Shakespearean sonnets, or movie trailer narration. git commit -m "fixed null pointer" becomes a dramatic haiku about the void.

What it does: Intercepts commit messages via a prepare-commit-msg hook, sends the original message and diff summary to Claude, and rewrites it in a creative format while preserving the technical meaning.

Why it's cool: Your git log becomes entertainment. Imagine scrolling through commit history and reading limericks about bug fixes. It's silly, it's fun, and it makes code review meetings 100% more enjoyable.

How to build it:

  1. Create a prepare-commit-msg hook that runs after you type your message
  2. Read the original commit message and a summary of staged changes
  3. Send to Claude with a style parameter: haiku, limerick, sonnet, movie trailer, pirate speak
  4. Preserve the original message as a comment below the creative version
  5. Add a GIT_POET_STYLE environment variable to control the format
  6. Add a --no-poet flag for when you need to be serious (rare)

Trivia Game Generator
Weekend
Python Claude API Flask / Discord.py

Generates custom trivia games on any topic with varying difficulty levels. Play solo or compete with friends. "Generate 20 questions about 90s hip-hop, medium difficulty" — instant trivia night, no Googling questions for an hour.

What it does: Creates trivia quizzes on any topic, with adjustable difficulty and categories. Supports multiple choice and open-ended formats, tracks scores across games, and generates leaderboards.

Why it's cool: Trivia nights are fun but someone has to write the questions. Now nobody does. Pick any topic — "Marvel movies," "obscure programming languages," "types of cheese" — and get a complete quiz in seconds. Play in Discord with friends or via a web interface.

How to build it:

  1. Accept a topic, difficulty (easy/medium/hard), and number of questions
  2. Generate questions with Claude using tool use for structured output: question, 4 choices, correct answer, fun fact
  3. Build a turn-based game flow: show question, accept answers, reveal correct answer + fun fact, track scores
  4. For Discord: use reactions for answers (A/B/C/D), time limits with countdown
  5. For web: Flask app with real-time updates via WebSocket
  6. Add a leaderboard and category tracking: "You're best at Science, worst at Geography"
Music Practice Companion
Weekend
Python Claude API Flask

Tell it what instrument you play and what you're working on. It generates practice plans, explains music theory concepts in plain English, suggests exercises for specific techniques, and tracks your progress over weeks.

What it does: Acts as a personalized music teacher: creates structured practice plans, explains theory adapted to your level, generates exercises targeting your weak spots, and adjusts difficulty as you improve.

Why it's cool: Music lessons are expensive and practice is hard without direction. This companion gives you the structure of a teacher — warm-up, technique drill, new material, review — customized to what you're actually working on. "I'm learning jazz guitar and struggling with ii-V-I progressions" gets you a targeted practice session.

How to build it:

  1. Onboarding: instrument, experience level, current goals, available practice time
  2. Generate weekly practice plans with daily sessions: warm-up, technique, repertoire, theory
  3. After each session, ask for a quick self-assessment: "How did scales feel today?" (1-5)
  4. Adjust upcoming sessions based on progress: struggle areas get more time, mastered areas rotate out
  5. Explain theory on demand: "What's a tritone substitution?" gets an explanation with audio theory, not textbook jargon
  6. Suggest songs that use techniques you're practicing: "Here are 5 songs that use the ii-V-I progression"

Maintaining Consistency Over Long Sessions

Creative AI has a unique challenge: consistency. A story that forgets a character's name by chapter 3. A dungeon master that contradicts its own world rules. Here's how to fix it:

The Living Document Pattern

Maintain a structured document that gets updated after every interaction:

story_bible = {
    "characters": [
        {"name": "Detective Chen", "traits": "dry humor, limp in left leg",
         "goals": "find the mayor's cat, uncover the real conspiracy"},
    ],
    "plot_points": [
        "Ransom note found under mayor's whiskey glass",
        "Butler acting suspicious, avoided eye contact",
    ],
    "unresolved": [
        "Why is the mayor sweating?",
        "What's the Friday deadline about?",
    ],
    "setting": "Noir city, perpetual rain, 1940s aesthetic",
    "tone": "Dark humor, hardboiled dialogue"
}

# After each AI turn, update:
update_prompt = f"""Based on the latest story development,
update this story bible. Add new characters, plot points,
and questions. Remove resolved items.
Current bible: {json.dumps(story_bible)}
Latest text: {latest_paragraph}"""

The Window + Summary Pattern

For very long sessions, keep the full history in storage but only send Claude the last 5-10 exchanges plus a rolling summary of everything before that:

messages = [
    {"role": "system", "content": story_bible + "\n\nStory so far: " + summary},
    # Last 5 exchanges only
    *recent_messages[-10:]
]

The Temperature Dial

Temperature controls randomness. For creative projects, it matters more than anywhere else:

Temperature Effect Use For
0.0 - 0.3 Predictable, focused Trivia answers, music theory explanations, consistent game rules
0.5 - 0.7 Balanced creativity Story writing, game narration, most creative tasks
0.8 - 1.0 Wild, surprising Plot twists, comedy, experimental writing, brainstorming
Use different temperatures within the same project. In the Dungeon Master Bot: low temperature for combat resolution (consistent rules), medium for scene descriptions (creative but coherent), high for NPC dialogue and unexpected events (surprise the player).

Persistence: Saving Creative State

Creative projects need persistence more than any other type. Nobody wants to start their story over because the server restarted.

  • Stories: Save every exchange + the story bible. Allow branching (fork the story at any point).
  • Games: Save player stats, inventory, world state, quest progress. Resume anywhere.
  • Practice plans: Save session history, assessments, and generated plans.
  • Trivia: Save scores, topic performance, and question history (don't repeat questions).

SQLite is perfect for all of these. One file, no server, easy to back up.

Multiplayer: Creative AI for Groups

The most fun creative AI projects are multiplayer. Discord is the ideal platform: the Story Collaborator becomes a round-robin where 4 friends and the AI write together. The Dungeon Master Bot runs a party adventure. The Trivia Generator hosts pub quiz night. Build for groups first — solo mode is just a group of one.

What if you combined the Story Collaborator and Dungeon Master into a full interactive fiction engine? A web-based platform where anyone can create and share interactive stories. The creator writes the opening scene and sets the world rules. Players make choices that branch the narrative. The AI generates scenes based on player decisions while maintaining the creator's world. Think "Choose Your Own Adventure" meets D&D meets collaborative fiction. Add illustrations with image generation, ambient music links, and you've built a creative platform, not just a tool.

Build It: Commit Message Poet

Let's build the Commit Message Poet and install it right now. Your git log will never be boring again.

Step 1: Create the hook script (5 min)

Create a file called commit-poet.py in a convenient location (e.g., ~/tools/):

#!/usr/bin/env python3
"""Git commit message poet — rewrites commits as art."""
import sys, os, subprocess
import anthropic

STYLES = {
    "haiku": "Rewrite as a haiku (5-7-5 syllables). Keep technical meaning.",
    "limerick": "Rewrite as a limerick (AABBA rhyme). Keep technical meaning.",
    "movie": "Rewrite as a dramatic movie trailer narrator. Keep technical meaning.",
    "pirate": "Rewrite in pirate speak. Keep technical meaning.",
    "shakespeare": "Rewrite in Shakespearean English. Keep technical meaning.",
}

def main():
    commit_msg_file = sys.argv[1]
    with open(commit_msg_file) as f:
        original = f.read().strip()

    # Skip merges and already-poetic messages
    if original.startswith("Merge") or "🎭" in original:
        return

    style = os.environ.get("GIT_POET_STYLE", "haiku")
    if style not in STYLES:
        style = "haiku"

    client = anthropic.Anthropic()
    response = client.messages.create(
        model="claude-haiku-4-5-20251001",
        max_tokens=150,
        messages=[{"role": "user",
            "content": f"{STYLES[style]}\n\nOriginal: {original}"}]
    )

    poetic = response.content[0].text.strip()
    with open(commit_msg_file, "w") as f:
        f.write(f"🎭 {poetic}\n\n# Original: {original}")

if __name__ == "__main__":
    main()

Step 2: Install the git hook (2 min)

# In your project's .git/hooks/ directory:
cat > .git/hooks/prepare-commit-msg << 'EOF'
#!/bin/bash
python3 ~/tools/commit-poet.py "$1"
EOF
chmod +x .git/hooks/prepare-commit-msg

Step 3: Try it (1 min)

# Make a change and commit
echo "test" >> test.txt
git add test.txt
git commit -m "added test file"

# Check the log
git log --oneline -1
# 🎭 A file appears / test content joins the repo / simple yet complete

# Try different styles:
GIT_POET_STYLE=limerick git commit -m "fixed the login bug"
GIT_POET_STYLE=movie git commit -m "refactored database layer"
GIT_POET_STYLE=pirate git commit -m "removed deprecated API"

Your git history is now poetry. Show your team. Watch their reaction. You're welcome.
