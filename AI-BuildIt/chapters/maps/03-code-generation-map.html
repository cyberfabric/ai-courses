The 80/20 Rule of Code Generation

AI generates 80% of the code perfectly — the boilerplate, the wiring, the patterns. The remaining 20% needs your expertise: business logic, edge cases, domain-specific patterns. Don't aim for perfection — aim for speed, then polish.

📖 Key Terms

Scaffolding
Generating the boilerplate file structure for a feature — the boring 80% that follows patterns.
Validation Loop
Generate → Compile → Test → Fix Errors → Ship. The tool verifies its own work automatically.
Convention Files
A .codegen/ directory with team rules, templates, and style guides that the generator reads before generating.
Project Scan
Auto-detecting framework, architecture, naming conventions, and dependencies from existing code before generating new code.
Vertical Slice
All layers of a feature at once — UI, ViewModel, Repository, Model, DI, Navigation, Tests — wired together and ready to run.
Mermaid ER Diagram
Text-based entity-relationship diagrams that render visually — great for visualizing generated schemas.
TDD Mode
The Test Suite Generator's --style tdd flag: generate tests first, then write the implementation to make them pass.

🧪 Quick Quiz

Test Your Memory

How much time does the typical boilerplate phase take per feature?

~2 hours — creating files, copying patterns, wiring imports, setting up tests. AI cuts this to ~5 minutes.

What's the "secret weapon" for making generated code match your project?

Feed an existing file from your project as an example in the prompt. "Generate a new endpoint like this one, but for orders." Claude copies your patterns exactly.

What does the Validation Loop do after generating code?

Automatically runs the compiler and tests. If something fails, sends the error back to Claude for a fix. Most issues resolve in one retry.

Why should AI-generated tests be treated as a starting point?

They cover the obvious cases (happy paths, boundaries, error handling) but miss subtle business logic edge cases that require domain knowledge. Review and add the tricky cases yourself.

What goes in a .codegen/ directory?

conventions.md with naming/structure/error-handling rules, plus templates/ with skeleton files for endpoints, models, tests, etc. Ensures team-wide consistency.

📋 Key Takeaways

If You Remember Nothing Else

  • Scaffolding wastes ~2 hours per feature — AI cuts it to 5 minutes by generating the boring 80%.
  • The Validation Loop (Generate → Test → Fix) makes code generation reliable — the tool verifies its own work.
  • Feed examples from your project to match your conventions — "like this file, but for X."
  • Create .codegen/ convention files once — get consistent team-wide output forever.
  • Start with the Database Schema Designer — it helps at the very start of every project.
  • AI-generated tests are a starting point, not the finish line — add domain-specific edge cases yourself.
  • Chain Schema + API + Tests for the ultimate project bootstrap: one paragraph in, running skeleton out.
