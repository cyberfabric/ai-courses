The CLI Wrapper Pattern

    Create directory, virtual environment, pip install anthropic click.

📖 Key Terms

System Prompt
The hidden instruction that defines your tool's personality, output format, and behavior — the secret sauce of step 2.
Structured Output
Forcing the LLM to return JSON, tables, or runnable commands instead of free-form text.
Streaming
Printing tokens as they arrive from the API instead of waiting for the full response — feels instant.
Composability
Tools that read stdin and write stdout can be chained with pipes — the Unix philosophy for AI.
Dry Run
Showing what would happen without actually doing it — critical for tools that modify files or run commands.
Cache
Storing previous API responses keyed by input hash — same question twice shouldn't cost twice.
Model Selection
Haiku for speed (simple translations), Sonnet for balance (most tasks), Opus for deep reasoning (complex analysis).

⚡ Performance Cheat Sheet

🌊
Stream Responses
Use client.messages.stream() — users see output immediately instead of waiting 2-3 seconds.
💾
Cache Results
Hash the input, store the response in ~/.cache/your-tool/. Same question = instant answer.
🎚️
Right-Size the Model
Add --fast (Haiku) and --think (extended thinking) flags. Let users choose the speed/quality tradeoff.
📦
Chunk Big Input
For huge logs, send the last 500 lines + all ERROR/WARN/FATAL lines instead of everything.

🧪 Quick Quiz

Test Your Memory

What are the 5 steps in the anatomy of an AI CLI tool?

Parse Input → Build Prompt → Call LLM → Structure Output → Take Action

Why should file-modifying tools default to --dry-run?

To show the user what would happen before actually doing it — builds trust and prevents disasters like accidentally moving or deleting important files.

What's the "golden rule" of AI CLI tools?

Read from stdin, write to stdout, use stderr for status messages. This makes your tool composable with every other tool in the terminal.

Which step in the 5-step anatomy is "the magic"?

Step 2: Build Prompt. A well-crafted system prompt turns a generic LLM call into a specialized tool.

What's the fastest way to reduce API latency in a CLI tool?

Stream responses — use client.messages.stream() so users see output immediately instead of waiting for the full response.

📋 Key Takeaways

If You Remember Nothing Else

  • The CLI is the best starting point for AI tools — fast to build, easy to distribute, infinitely composable.
  • Every AI CLI tool follows the same pattern: Parse → Prompt → LLM → Structure → Act.
  • The system prompt is the magic — it turns a generic API call into a specialized tool.
  • All 5 projects are weekend builds: Git Companion, Log Detective, Config Generator, File Organizer, Terminal Translator.
  • Always default to --dry-run for tools that modify files or run commands.
  • Read stdin, write stdout — follow Unix conventions and your tools compose with pipes.
  • Start with the Terminal Translator — simplest build, daily use, 30 minutes to ship.
