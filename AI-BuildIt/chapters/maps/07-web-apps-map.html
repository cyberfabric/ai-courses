💰 Cost Management

  1. 📦 Cache Responses

    Same input → return cached output. Use Redis or a simple dict with input hashes. Don't pay twice for the same answer.

  2. 🤖 Right-Size the Model

    Haiku (~$0.001/request) for categorization and formatting. Sonnet for most features. Opus only for deep reasoning.

  3. 📏 Set max_tokens Wisely

    A tweet needs 100 tokens, not 4096. Lower limits = lower costs = faster responses.

  4. ♻️ Prompt Caching

    Long, unchanging system prompts? Claude's prompt caching saves 90% on that portion.

  5. 📦 Batch Requests

    "Generate all 5 social posts" in one call instead of 5 separate calls.

The $5/Month Sweet Spot

Haiku at 100 requests/day = ~$3/month. You can build a real, useful AI web app and keep costs under $5/month by using Haiku for simple tasks and Sonnet for the ones that need it.

📖 Key Terms

AI-Native App
A web app where the LLM is the core engine — without it, the app doesn't exist. Contrast with "bolt-on" AI features.
Server-Sent Events (SSE)
A protocol for streaming data from server to browser — used to show AI responses word by word instead of waiting for the full response.
Structured Output
Having Claude return JSON data instead of raw HTML — your template renders it consistently, the AI just provides the content.
Rate Limiting
Capping requests per user/IP per time period — essential to prevent one bot from running up a $500 API bill in an hour.
Prompt Caching
Claude's feature that caches long system prompts, saving 90% on repeated portions — crucial for cost control in production.
HTMX
A library for server-rendered interactivity with minimal JavaScript — great for Python-based AI web apps (Flask + HTMX).

🧪 Quick Quiz

Test Your Knowledge

What's the difference between an AI bolt-on and an AI-native web app?

A bolt-on adds AI to an existing app (e.g., a chatbot widget). An AI-native app can't exist without the LLM — the AI is the core engine, not a feature. Content Repurposer is AI-native; adding a chatbot to your SaaS is a bolt-on.

Why should you generate structured JSON instead of raw HTML from Claude?

JSON gives you consistent, beautiful output every time. The AI writes the words (content), your template handles the design (layout, CSS). Raw HTML from an LLM is unpredictable and hard to style.

How does streaming (SSE) improve the user experience?

Instead of a loading spinner for 3-5 seconds, text appears word by word as it's generated. The app feels instant even when the full response hasn't finished. Uses Server-Sent Events via EventSource API.

Why must you NEVER put your Claude API key in client-side JavaScript?

Anyone can view page source or inspect network requests and steal your key. They'd own your API bill. All LLM calls must go through your backend using environment variables.

How can you keep AI API costs under $5/month for a personal web app?

Cache responses, use Haiku for simple tasks (~$0.001/request), set appropriate max_tokens, use prompt caching for system prompts, and batch multiple outputs into single API calls.
