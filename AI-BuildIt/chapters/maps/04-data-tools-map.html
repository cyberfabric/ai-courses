⚠️ Critical Safety Rules

🔒
Sandbox exec()
AI-generated code running via exec() is powerful but dangerous. Restrict imports, set memory limits, timeout long queries. Use RestrictedPython.
📖
Read-Only by Default
SQL Builder: no INSERT/UPDATE/DELETE unless --write flag is set. Point at a read replica for production DBs.
👀
Preview Before Write
Data Cleaner and Spreadsheet Automator must show what will change (first 5 rows, summary stats) before modifying anything.
📋
Schema, Not Data
Send column names, types, and samples — not 15K raw rows. Better code, cheaper API calls, and your data stays private.

⏱️ CSV Whisperer in 20 Minutes

  1. Set Up (2 min)

    Create project, venv, pip install anthropic pandas matplotlib.

  2. Create csvwhisper.py (12 min)

    Load CSV → extract schema (info, sample, stats) → REPL loop → Claude writes pandas code → exec() locally.

  3. Get Sample Data (1 min)

    Use any CSV you have, or generate a quick 200-row sales dataset with random products and countries.

  4. Ask Questions (5 min)

    "Total revenue by product?" "Which country buys the most Gadgets?" "Bar chart of revenue by country."

📖 Key Terms

Schema-First
Send column names, types, and sample rows to the AI — not raw data. Claude generates code that runs locally on the full dataset.
Sandboxing
Restricting what AI-generated code can do when running via exec() — no filesystem access, no network, limited imports.
Data Profiling
Auto-analyzing a dataset: missing values, duplicates, type mismatches, outliers. The Data Cleaner does this before suggesting fixes.
E.164
International phone number standard (+1234567890). The format the Data Cleaner normalizes phone numbers to.
Chunking
Processing large files in batches of 1K-10K rows, aggregating results across chunks to stay within memory and context limits.
Read Replica
A read-only copy of your database. Point the SQL Builder here instead of production to prevent accidental writes.

🧪 Quick Quiz

Test Your Memory

Why send the schema instead of raw data to Claude?

Three reasons: (1) cheaper — fewer tokens, (2) better code — Claude generates cleaner code from a clean schema than from a wall of raw data, (3) privacy — your actual data never leaves your machine.

What are the 5 steps of the safety layer for data tools?

Analyze → Preview → Confirm → Execute → Backup. Never auto-execute writes. Always save a .bak file of the original.

Why is running exec() on AI-generated code dangerous?

The generated code could access your filesystem, network, or perform destructive operations. Always sandbox: restrict imports, set memory/time limits, use RestrictedPython or run in a subprocess.

What's the difference between matplotlib and plotly for charts?

matplotlib generates static PNG images. plotly generates interactive HTML with zoom, hover, and pan — more impressive for sharing with non-technical stakeholders.

How do you handle a 500K-row CSV that won't fit in a prompt?

Schema-first: send column names, types, and sample rows — Claude writes code that runs locally. For operations needing actual values, chunk into 1K-10K row batches. For analysis, pre-compute summaries and send those.

📋 Key Takeaways

If You Remember Nothing Else

  • Schema-first: send column names and samples to Claude, not raw data. Cheaper, faster, private.
  • The CSV Whisperer is the gateway drug — anyone can query any CSV in plain English. 20-minute build.
  • Safety layer: Analyze → Preview → Confirm → Execute → Backup. Never auto-execute writes.
  • Sandbox exec() — restrict imports, set limits, never let AI-generated code touch filesystem or network.
  • SQL Builder must be read-only by default. Point at a read replica for production databases.
  • For large datasets: chunk (1K-10K rows) or summarize (pre-compute aggregates) before sending to AI.
  • Use plotly instead of matplotlib for interactive charts your PM can explore.
