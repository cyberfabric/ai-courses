💡 The Core Insight

🔌

A Thin Wrapper Around Intelligence

An AI microservice is ~50 lines of code: accept input, build a prompt, call Claude, return structured JSON. The consumer never knows there's an LLM behind the API. Every messy if/else chain, regex, or keyword matcher in your backend is a candidate for an AI microservice that actually understands the data.

🎯 Why AI Beats Rules

Input Keyword Matching AI Understanding
"I'm going to kill this presentation" ❌ BLOCKED (contains "kill") ✅ ALLOWED (figurative, positive)
"You're so worthless, everyone hates you" ❌ ALLOWED (no banned words) ✅ BLOCKED (harassment, severity 8/10)
"Nice job, Einstein" (sarcastic) ❌ ALLOWED (positive words) ✅ WARNING (sarcastic bullying, 5/10)

The same principle applies everywhere: webhook routing, email classification, translation — AI understands meaning, rules only match patterns.

🔧 Structured Output: Tool Use

The gold standard for guaranteed JSON from Claude. Define a schema, force the model to use it.

📋 Define Schema
properties, types, enums
→
🔒 tool_choice

→
📦 response.content[0].input

The schema acts as a contract — just like a typed API response. Your consumers always get the fields they expect, in the types they expect. No parsing markdown, no hoping the JSON is valid.

🛡️ Production Hardening

  1. 🔄 Retry on Failure

    Most LLM errors are transient. Use tenacity with exponential backoff: 3 attempts, 1-10 second waits.

  2. ✅ Validate Responses

    Check required fields exist, enum values are valid, types are correct. Even with tool use, validate before returning.

  3. 🔒 Safe Defaults

    If all retries fail, return a safe fallback (e.g., "unknown" intent, "medium" urgency) rather than crashing.

  4. 📊 Log Everything

    Log every LLM call: input, output, latency, failures. Essential for debugging and quality monitoring.

📖 Key Terms

AI Microservice
A thin API wrapper (~50 lines) around a structured LLM call — consumers get JSON, never knowing there's an LLM behind it.
Tool Use / Structured Output
Claude's feature for forcing responses into a defined JSON schema — the gold standard for API-grade structured output.
tool_choice
API parameter that forces Claude to use a specific tool, guaranteeing your response matches the defined schema.
Evaluation Set
A curated set of 20-50 inputs with expected outputs, used to track AI service accuracy over time and catch regressions.
Exponential Backoff
Retry strategy with increasing waits (1s, 2s, 4s...) — prevents hammering a failing API while still recovering from transient errors.
Data Enrichment
Adding structured information (industry, size, tech stack) to a minimal data point (company name) — traditionally expensive, AI makes it $0.001/lookup.

🧪 Quick Quiz

Test Your Knowledge

What makes an AI microservice different from a regular API?

It uses an LLM for the core processing instead of hard-coded logic. The consumer sends data and gets structured JSON — the exact same interface as a regular API. The difference is that the processing understands meaning, not just patterns.

Why is tool use the "gold standard" for structured output?

It forces Claude to return data matching your JSON schema — specific fields, types, and enum values. The response is already a dict (not a string you have to parse). The schema acts as a contract, just like a typed API response.

Why does AI content moderation beat keyword-based filtering?

Keywords cause false positives ("I'm going to kill this presentation" → blocked) and miss real abuse ("You're worthless" → no banned words). AI understands context, sarcasm, and intent, making better moderation decisions.

How do you test a non-deterministic AI API?

Use snapshot tests to verify response structure (not exact text), evaluation sets (20-50 inputs with expected outputs), and accuracy tracking — alert if accuracy drops below your threshold (e.g., 90%).

What's the 4-step error handling pattern for AI microservices?

Retry (most failures are transient), validate (check required fields and types), fallback (safe defaults if all retries fail), log (every call for debugging and monitoring).

📌 Key Takeaways

If You Remember Nothing Else

  • An AI microservice is a thin API wrapper around a structured LLM call — ~50 lines, $0.001/request, consumers never know there's an LLM.
  • Use tool use (not JSON parsing) for guaranteed structured output — the schema is your API contract.
  • AI replaces messy if/else chains and keyword matching — it understands meaning, context, and intent.
  • Production hardening: retry → validate → fallback → log. LLMs are non-deterministic; build for it.
  • Use queues and concurrency limits to prevent rate limit exhaustion under load.
  • Test with evaluation sets — check structure and accuracy, not exact text.
  • Start with the Email Intelligence API (Weekend) — it teaches tool use, the most important technique in this chapter.
