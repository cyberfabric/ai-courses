💡 The Core Insight

🚀

Shipped Beats Perfect

A working tool with rough edges that you use every day is infinitely more valuable than a polished masterpiece sitting in a private repo. The best AI tool is the one you build. You don't need to wait for someone else — you have Claude Code, an API key, and 60+ ideas.

🎯 The "What Should I Build?" Framework

🔧
F — Feasibility
Can you build it this weekend? Rate 1-5. If it needs 3 weeks, scope it down.
💎
V — Value
Will you or others actually use it? Personal utility beats demo appeal.
📚
L — Learning
Will it teach you something new? New pattern, new API, new domain?
🎉
Fun — Enjoyment
Will you have fun building it? Boring projects get abandoned.

Multiply: F × V × L × Fun. Pick the highest score. A feasible(4) × useful(3) × educational(5) × fun(4) project scores 240 — way better than a feasible(5) × useful(5) × boring(1) × tedious(1) = 25.

⏰ The Weekend MVP

The #1 project killer isn't difficulty — it's scope creep. Force yourself to build the smallest useful version in 48 hours.

If you can't describe your project in one sentence, the scope is too big. "Send a receipt photo to Telegram, get the amount and category back." That's a weekend. Ship it, then add features over time.

🛠️ Tech Stack Decision

If You Need... Use This Why
CLI tool Python + Click Fastest to build, easiest to distribute
Web app Python + FastAPI Async, typed, auto-docs
Dashboard Python + Streamlit Zero frontend code
Bot Python + platform SDK Best SDK support everywhere
Rich UI TypeScript + Next.js When the UI IS the product

Pattern: Python for almost everything. Only reach for TypeScript when you need a rich interactive frontend.

🧠 Universal Prompt Patterns

  1. 👤 Persona

    "You are a senior database architect." Sets tone, vocabulary, and expertise level.

  2. 🚫 Constraint

    "Output ONLY valid JSON. No markdown." Tell the AI what NOT to do — prevents the most common failures.

  3. 📋 Example

    "Here's an existing endpoint. Generate one following the same patterns." One example beats 100 words of description.

  4. 🔧 Structured Output

    Use tool use for anything parsed programmatically. Don't "ask for JSON" — guarantee the schema.

Don't over-engineer prompts. Start with the simplest possible system prompt. Add constraints only when you see real failures, not hypothetical ones.

🧪 Quick Quiz

Test Your Knowledge

What are the four dimensions of the project selection framework?

Feasibility (can you build it this weekend?), Value (will you actually use it?), Learning (will it teach something new?), Fun (will you enjoy building it?). Multiply all four — highest score wins.

What's the "Weekend MVP" and why does it work?

Build the smallest useful version in 48 hours. Friday: one-sentence scope. Saturday: core feature. Sunday: polish + ship. It works because the #1 project killer is scope creep, not technical difficulty.

Why does "Python for almost everything" work for AI projects?

Python has the best AI libraries (anthropic, chromadb, pandas), simplest deployment, and Claude Code generates excellent Python. Only reach for TypeScript when the UI is the product.

What are the 4 universal prompt engineering patterns?

Persona (who the AI should be), Constraint (what NOT to do), Example (show, don't tell), Structured Output (tool use for guaranteed schemas). Start simple, add constraints only after seeing real failures.

When does a side project deserve to become a product?

When 3+ are true: you use it daily, others ask for it, you keep adding features because you want to, someone says "I'd pay for this," and competitors charge for something worse.

📌 Key Takeaways

If You Remember Nothing Else

  • Pick projects using F × V × L × Fun. Personal utility beats demo appeal.
  • Weekend MVP: one-sentence scope, core feature Saturday, ship Sunday. Scope creep kills projects, not difficulty.
  • Python for almost everything. TypeScript only when the UI is the product.
  • 4 prompt patterns: Persona, Constraint, Example, Structured Output. Start simple, iterate on failures.
  • Shipped beats perfect. Push to GitHub, deploy, tell someone. Improve later.
  • 5 small shipped projects beat 1 big unfinished one — build a portfolio in 30 days.
  • The only thing left is to pick one and start. Open a terminal. Now.
