💡 The Core Insight

💬

Meet People Where They Already Are

3B+ people use messaging apps daily. Instead of building another web app nobody bookmarks, build a bot that lives in Slack, Discord, or Telegram. Zero new apps to install. Just type a message and get an answer. Bots are the most natural interface for AI because talking to AI should feel like talking.

✅ Good Bot vs. ❌ Annoying Bot

Trait ❌ Annoying Bot ✅ Good Bot
Timing Messages you unsolicited Responds when spoken to (or on schedule)
Output Walls of text nobody reads Concise, actionable answers
Questions Asks 5 when 1 would do Clarifies only when necessary
Personality Cutesy tone that gets old fast Consistent, professional tone
Errors "I don't understand" on repeat Handles unexpected input gracefully
Rule #1: Be useful, not chatty. Nobody wants "Great question! Let me think about that..." before every answer. Get to the point. Deliver value. Shut up.

🧠 Conversation State Management

📝 Message History
last 10 messages
🔑 Key by user_id
or channel_id + user_id
💾 Conversation State
remember context
⏰ TTL: 30 min
clear stale context
💿 Persist: SQLite/Redis
survive restarts

A bot that forgets what you said 5 seconds ago is useless. Store conversation history, but clear after 30 min of inactivity — fresh conversations beat confused ones.

🔄 Multi-Turn Pattern

When a bot needs to ask follow-up questions, use a state machine:

📩 User Request
→
❓ Bot Clarifies
→
💬 User Answers
→
✅ Bot Acts

Track each user's position in a flow: {user_id: {flow: "expense", step: "confirm", data: {...}}}. No active flow → handle normally.

🚢 Deployment: Always-On vs. Serverless

Approach Best For Cost
Always-On Server Discord & Slack (WebSocket required). VPS on Hetzner, DigitalOcean, or Railway $5-10/month
Serverless / Webhooks Telegram (natively webhook-based). Lambda, Cloud Functions, Vercel Often free tier

📖 Key Terms

Conversation State
Stored message history and flow position per user — lets the bot remember context across messages.
State Machine
A pattern for multi-turn interactions: track which step of a flow each user is in and route responses accordingly.
Spaced Repetition (SM-2)
An algorithm that resurfaces quiz questions at optimal intervals based on how well you answered — used by the Learning Buddy Bot.
Conversation TTL
Time-to-live for bot context — clear after 30 min of inactivity to prevent stale context from confusing future interactions.
Webhook Mode
The platform pushes messages to your endpoint (vs. polling). Telegram supports this natively, enabling serverless bot hosting.
Meta-Bot
A single unified bot that classifies requests and routes to specialized handlers — one @assistant for standups, expenses, quizzes, and briefings.

🧪 Quick Quiz

Test Your Knowledge

What's the #1 rule of bot design?

Be useful, not chatty. Get to the point, deliver value, shut up. No "Great question! Let me think..." preambles. Concise, actionable answers only.

Why build a bot instead of a web app?

Your users already have messaging apps open all day. Zero new apps to install, zero URLs to remember. Bots meet people where they already are — Slack, Discord, Telegram.

How do you maintain conversation context across messages?

Store message history keyed by user_id, keep the last 10 messages, persist in SQLite/Redis for restart survival. Set a 30-minute TTL — clear stale context so fresh conversations aren't confused by old ones.

When should you use an always-on server vs. serverless for a bot?

Always-on for Discord/Slack (need WebSocket connections). Serverless/webhooks for Telegram (natively webhook-based, works perfectly with Lambda/Cloud Functions at near-zero cost).

What safety consideration applies specifically to on-call companion bots?

AI guidance should assist, not replace human judgment. Always include a disclaimer. For destructive actions (restarting services, scaling down), require human confirmation before executing.

📌 Key Takeaways

If You Remember Nothing Else

  • Meet users where they are — 3B+ people use messaging apps daily. Zero installs, zero bookmarks.
  • Bot design rule #1: be useful, not chatty. Concise answers, no filler.
  • Pick your platform: Slack Bolt for work, Discord.py for community, python-telegram-bot for personal.
  • Conversation state: store history, key by user, 30-min TTL, persist in SQLite/Redis.
  • Use state machines for multi-turn flows — track each user's position in a conversation flow.
  • Deploy: always-on for Discord/Slack (WebSocket), serverless for Telegram (webhooks).
  • Start with the Daily Briefing Bot (Weekend) — 20-minute build, you'll use it literally every day.
