💡 The Core Insight

🎨

AI is a Creative Partner, Not a Replacement

The best creative AI tools don't generate art while you watch — they generate art with you. You provide the spark, AI amplifies it, you refine it, and the result is something neither of you would have created alone. 100% fun-to-work ratio, 0 stakeholders to impress, infinite replay value.

📖

📜 The Story Bible Pattern

Creative AI's biggest challenge: consistency. The AI forgets the detective's limp by chapter 3. The fix:

✍️ Human Writes
or plays a turn
→
🤖 AI Responds
using story bible context
→
📜 Update Bible
new chars, plots, questions
→
🔄 Next Turn
bible in system prompt
👤 Characters
names, traits, goals
📍 Plot Points
what's happened so far
📜 Story Bible
<2,000 tokens
❓ Unresolved
open questions, threads
🌆 Setting & Tone
world, mood, style
For very long sessions, use the Window + Summary pattern: store full history but only send the last 5-10 exchanges plus a rolling summary of everything before. Keep the story bible under 2,000 tokens in the system prompt.

🌡️ The Temperature Dial

Temperature Effect Use For
0.0 – 0.3 Predictable, focused Trivia answers, game rules, music theory
0.5 – 0.7 Balanced creativity Story writing, game narration, most creative tasks
0.8 – 1.0 Wild, surprising Plot twists, comedy, experimental writing, brainstorming

Mix temperatures within one project: DM Bot uses low for combat rules, medium for descriptions, high for NPC dialogue and surprises.

💾 Persistence: Save Everything

  1. 📖 Stories

    Save every exchange + story bible. Allow branching — fork at any point to try alternate paths.

  2. 🎮 Games

    Save player stats, inventory, world state, quest progress. Resume between sessions seamlessly.

  3. 🎵 Practice Plans

    Save session history, self-assessments, and generated plans. Track improvement over weeks.

  4. 🧠 Trivia

    Save scores, topic performance, question history. Don't repeat questions. Track strengths and weaknesses.

SQLite handles all of these perfectly. One file, no server, easy to back up.

👥 Build for Groups First

The Most Fun Creative AI Projects Are Multiplayer

Discord is the ideal platform: Story Collaborator becomes round-robin with 4 friends + AI. DM Bot runs party adventures. Trivia Generator hosts pub quiz night. Build for groups first — solo mode is just a group of one.

📖 Key Terms

Story Bible
A structured document tracking characters, plot points, setting, and unresolved threads — the AI's memory of the creative work so far.
Window + Summary
Sending only recent exchanges plus a rolling summary of older ones — keeps context fresh without exceeding token limits.
Temperature
Controls output randomness (0.0 = deterministic, 1.0 = wild). Creative projects use different temperatures for different aspects of the same project.
Story Branching
Forking a story at any point to explore alternate paths — like "Choose Your Own Adventure" with infinite branches.
prepare-commit-msg
A git hook that runs after you write a commit message but before it's saved — the Commit Message Poet intercepts at this stage.

🧪 Quick Quiz

Test Your Knowledge

What's the "story bible" and why is it essential for creative AI?

A structured document tracking characters, plot points, setting, tone, and unresolved threads. Without it, the AI "forgets" details by turn 10. Updated after every exchange, kept under 2,000 tokens, included in the system prompt.

How should you use temperature differently within a single creative project?

Mix temperatures by purpose: low (0.0-0.3) for rules and facts (combat, trivia answers), medium (0.5-0.7) for scene descriptions and narration, high (0.8-1.0) for surprises, comedy, and unexpected twists.

What's the "Window + Summary" pattern for long creative sessions?

Store the full history in your database, but only send Claude the last 5-10 exchanges plus a rolling summary of everything before. Keeps context fresh without exceeding token limits.

Why should you build creative AI projects for groups first?

The most fun creative AI projects are multiplayer. Round-robin story writing, party RPG adventures, pub quiz nights. Solo mode is just a group of one — the architecture supports both.

What git hook does the Commit Message Poet use, and why?

prepare-commit-msg — it runs after you type your message but before it's saved. This lets the hook rewrite the message while preserving the original as a comment. Set GIT_POET_STYLE env var to control the format.

📌 Key Takeaways

If You Remember Nothing Else

  • AI is a creative partner, not a replacement — the best tools create with you, not for you.
  • The story bible is essential: track characters, plots, setting, and open threads in <2,000 tokens.
  • Mix temperature within one project: low for rules, medium for narration, high for surprises.
  • Persist everything in SQLite: stories, game state, scores, practice plans. Nobody wants to start over.
  • Build for groups first — Discord is ideal. Solo mode is just a group of one.
  • For long sessions, use Window + Summary: recent exchanges + rolling summary of everything older.
  • Start with the Commit Message Poet (10 min install) — instant payoff, immediate laughs, gateway to bigger creative projects.
