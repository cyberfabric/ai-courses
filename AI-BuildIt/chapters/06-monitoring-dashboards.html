Chapter 6

Monitoring & Analytics Dashboards

Build dashboards that explain what the numbers mean and tell you what to do about them

The Dashboard Problem

Here's an uncomfortable truth: 90% of dashboards are looked at once and then forgotten.

Someone spends a week building a beautiful Grafana board with 37 panels. The team oohs and aahs. Then nobody looks at it again until something is on fire — and by then they're reading raw logs anyway.

Why do dashboards fail? Because they show data without telling you what it means. A graph going up could be great or terrible. A number turning red might be critical or routine. Without context and interpretation, dashboards are just expensive wallpaper.

90% of dashboards abandoned within a month
15 min average time to interpret a complex dashboard
3 sec to read a plain-English summary

The fix? AI-narrated dashboards. Instead of just showing metrics, these dashboards explain them. "Revenue is up 12% this week, driven by a 3x spike in EU signups after the Product Hunt launch. However, churn increased 8% — new users are dropping off at the onboarding step. Recommended action: investigate the onboarding flow for EU users."

That's not a dashboard. That's a data analyst. And you can build one.

In Chapter 5, we made knowledge searchable. Now we're making data narrated — dashboards that talk to you instead of making you squint at charts.

The Projects

Smart App Monitor
Week
Python Claude API Firebase / Crashlytics Streamlit

Monitors crash reports from Crashlytics or Sentry, groups related crashes into root cause clusters, identifies which crashes matter most, and suggests specific code fixes. Not just "you had 200 crashes" — but "200 crashes from 3 root causes, the top one is a null pointer in PaymentFragment.kt line 47, here's the likely fix."

What it does: Pulls crash data from your crash reporting tool, uses AI to cluster related stack traces, identify root causes, assess user impact, and suggest fixes with confidence ratings.

Why it's cool: Crash reports are noisy. The same bug creates 50 slightly different stack traces. This tool cuts through the noise and tells you exactly what to fix first and how.

How to build it:

  1. Connect to Crashlytics (Firebase Admin SDK) or Sentry (REST API) to pull recent crashes
  2. Group stack traces using embedding similarity — crashes with similar traces are likely the same bug
  3. For each cluster, send representative stack traces to Claude: "Identify the root cause and suggest a fix"
  4. Rank by impact: crash count × affected users × frequency trend
  5. Build a Streamlit dashboard showing clusters, trends, and AI-generated fix suggestions
  6. Add Slack notifications for new crash clusters above a threshold

Personal Finance Tracker
Weekend
Python Claude API pandas Streamlit

Import your bank CSV, and the AI auto-categorizes every transaction (groceries, subscriptions, dining out, transport...), tracks your budget, and gives you monthly spending insights. "You spent 23% more on dining this month. Your three biggest subscriptions total $87/month — you haven't used Audible since February."

What it does: Reads bank transaction exports, automatically categorizes each transaction using AI, computes budget tracking, and generates plain-English spending insights with recommendations.

Why it's cool: Every budgeting app requires manual categorization. This one does it automatically and gives you the kind of insights a financial advisor would — without the $200/hour fee.

How to build it:

  1. Parse bank CSV exports (handle different bank formats with a format detector)
  2. Send transactions in batches to Claude: "Categorize each transaction into: Groceries, Dining, Transport, Subscriptions, Utilities, Shopping, Income, Transfer, Other"
  3. Build monthly summaries: spending by category, trends vs previous months, top merchants
  4. Generate AI insights: unusual spending, forgotten subscriptions, savings opportunities
  5. Build a Streamlit dashboard with charts and the AI narrative
  6. Add budget targets and track progress
Financial data is sensitive. Never send raw bank data to external APIs without considering privacy. Options: redact account numbers before sending, run categorization locally with a smaller model, or only send merchant names (not amounts) for categorization and handle the numbers locally.
Server Health Narrator
Week
Python Claude API Prometheus / CloudWatch Slack API

Instead of 47 Grafana panels, get a plain-English health report: "All systems nominal except the payments service, which is running at 89% CPU (normally 45%) due to a retry storm from the failed database migration at 3am. Recommendation: roll back migration, then investigate the retry logic."

What it does: Collects server metrics (CPU, memory, latency, error rates, request counts), compares to historical baselines, and generates plain-English health reports with root cause analysis and recommendations.

Why it's cool: On-call engineers get paged at 3am and have to stare at dashboards while half-asleep. This tool tells them what's wrong, why, and what to do — in two paragraphs they can read on their phone.

How to build it:

  1. Pull metrics from Prometheus (PromQL API), CloudWatch, or Datadog
  2. Compute: current values, 7-day averages, standard deviations, trend direction
  3. Flag anomalies: anything >2 standard deviations from the baseline
  4. Send metrics + anomalies to Claude: "Generate a server health report. For each anomaly, suggest a root cause and remediation."
  5. Post to Slack or send via email on a schedule (every 4 hours, or on anomaly detection)
  6. Include a severity rating: ALL CLEAR, WATCH, WARNING, CRITICAL
Include historical context in the prompt. "CPU is at 89%" means nothing in isolation. "CPU is at 89%, normal baseline is 45%, it spiked at 3:12am, and the last deploy was at 3:00am" gives Claude everything it needs to tell a useful story. More context in the prompt = better narrative out.

Building Quick Dashboards With Streamlit

Every project in this chapter uses Streamlit for the visual layer. Here's why: it turns a Python script into a web dashboard in minutes. No HTML, no CSS, no JavaScript.

import streamlit as st
import pandas as pd

st.title("My AI Dashboard")

# Sidebar for controls
period = st.sidebar.selectbox("Period", ["This Week", "This Month", "Last 30 Days"])

# Metrics row
col1, col2, col3 = st.columns(3)
col1.metric("Crashes", "847", "-12%")
col2.metric("Users Affected", "312", "+5%")
col3.metric("Avg Fix Time", "4.2 hrs", "-30%")

# AI narrative
st.subheader("AI Analysis")
st.write(ai_narrative)  # The Claude-generated summary

# Charts
st.bar_chart(crash_data)
st.line_chart(trend_data)

Run it with streamlit run dashboard.py and you've got a live web dashboard. Add AI narratives alongside every chart and you've got something people will actually read.

Streamlit has a caching decorator (@st.cache_data) that prevents re-running expensive API calls on every page refresh. Use it for the Claude API calls — generate the narrative once, cache it until new data arrives.

Scheduled Reports: AI Digests

The most useful dashboard is one you don't have to open. Build scheduled reports that come to you:

Cron Trigger → Fetch Data → AI Analysis → Format Report → Send to Slack 
import requests, anthropic, schedule, time

client = anthropic.Anthropic()

def send_weekly_report():
    metrics = fetch_metrics()  # Your data gathering function
    narrative = client.messages.create(
        model="claude-sonnet-4-20250514",
        max_tokens=1024,
        system="You are a data analyst. Summarize these metrics...",
        messages=[{"role": "user", "content": str(metrics)}]
    ).content[0].text

    # Send to Slack
    requests.post(SLACK_WEBHOOK, json={
        "text": f"*Weekly Report*\n{narrative}"
    })

schedule.every().monday.at("09:00").do(send_weekly_report)
while True:
    schedule.run_pending()
    time.sleep(60)

Alert Intelligence

Traditional alerts are noisy. CPU above 80%? Alert. Memory above 90%? Alert. Error rate above 1%? Alert. You get 50 alerts and ignore all of them.

AI-powered alert triage changes this. Instead of raw threshold alerts, the AI evaluates context:

  • Is this spike normal for this time of day?
  • Is it correlated with a recent deployment?
  • Are multiple services affected (indicating a systemic issue) or just one?
  • What's the user impact?

The result: instead of "CPU > 80%", you get "CPU spike is expected — the nightly batch job runs at this time every Tuesday. No action needed." Or: "CPU spike is unusual and correlated with the deploy 10 minutes ago. Error rate is climbing. Recommend rollback."

The Intelligence Multiplier

A dashboard shows you data. An AI-narrated dashboard shows you meaning. The difference is the same as getting a blood test result sheet vs. having a doctor explain it. Numbers become decisions. Trends become actions. And you stop ignoring your dashboards because they finally speak your language.

What if you built a "morning briefing" that combines all your dashboards? Every morning at 8am, you get one message: server health (all clear), app crashes (2 new clusters, 1 critical), GitHub activity (3 PRs waiting for your review), personal spending (on track for budget), and social media (your Tuesday post is taking off). Five data sources, one AI-narrated summary, read in 30 seconds while you sip coffee. That's not a dashboard — that's a personal data narrator.

Build It: Personal Finance Tracker

Let's build the Personal Finance Tracker. We'll categorize transactions and get spending insights from a bank CSV.

Step 1: Set up (2 min)

mkdir finance-tracker && cd finance-tracker
python -m venv .venv && source .venv/bin/activate
pip install anthropic pandas streamlit

Step 2: Create sample data (1 min)

python -c "
import pandas as pd, random
merchants = [
  ('Whole Foods', 'Groceries', 45), ('Spotify', 'Subscriptions', 11),
  ('Shell Gas', 'Transport', 55), ('Netflix', 'Subscriptions', 16),
  ('Uber Eats', 'Dining', 32), ('Target', 'Shopping', 67),
  ('Starbucks', 'Dining', 7), ('AWS', 'Subscriptions', 23),
  ('Costco', 'Groceries', 120), ('PG&E', 'Utilities', 89),
]
rows = []
for i in range(60):
    m = random.choice(merchants)
    rows.append({'date': f'2026-03-{(i%28)+1:02d}', 'description': m[0],
                 'amount': round(m[2] * random.uniform(0.8, 1.3), 2)})
pd.DataFrame(rows).to_csv('bank_export.csv', index=False)
print('Created bank_export.csv')
"

Step 3: Create finance.py (10 min)

import streamlit as st
import anthropic
import pandas as pd
import json

client = anthropic.Anthropic()

st.title("Personal Finance Tracker")

uploaded = st.file_uploader("Upload bank CSV", type="csv")
if not uploaded:
    st.info("Upload a bank CSV to get started.")
    st.stop()

df = pd.read_csv(uploaded)
st.write(f"Loaded {len(df)} transactions")

# Categorize with AI
@st.cache_data
def categorize(descriptions):
    response = client.messages.create(
        model="claude-haiku-4-5-20251001",
        max_tokens=2048,
        messages=[{"role": "user", "content":
            f"Categorize each merchant into: Groceries, Dining, "
            f"Transport, Subscriptions, Utilities, Shopping, Other. "
            f"Return JSON array of categories.\n{json.dumps(descriptions)}"}]
    )
    text = response.content[0].text
    return json.loads(text[text.index("["):text.rindex("]")+1])

cats = categorize(df["description"].tolist())
df["category"] = cats

# Summary
col1, col2 = st.columns(2)
col1.metric("Total Spent", f"${df['amount'].sum():.2f}")
col2.metric("Transactions", len(df))

st.bar_chart(df.groupby("category")["amount"].sum())

# AI Insights
@st.cache_data
def get_insights(summary):
    r = client.messages.create(
        model="claude-sonnet-4-20250514", max_tokens=512,
        messages=[{"role": "user", "content":
            f"Analyze spending and give 3 actionable tips:\n{summary}"}]
    )
    return r.content[0].text

summary = df.groupby("category")["amount"].agg(["sum","count"]).to_string()
st.subheader("AI Spending Insights")
st.write(get_insights(summary))

Step 4: Run it (1 min)

streamlit run finance.py
# Upload bank_export.csv in the browser

You've got an AI-powered finance dashboard. Try it with a real bank export — the categorization and insights get much more interesting with real data.
