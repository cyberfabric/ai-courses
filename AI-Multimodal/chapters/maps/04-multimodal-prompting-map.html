Chapter 4

Multimodal Prompting: Sending Images to LLMs

Memory Map
💡

The Big Idea

Sending images to LLMs follows a universal pattern: image + text in one message. The quality of your results depends almost entirely on how specific your prompt is — tell the model what the image is, what you want to know, and what format you want the answer in.

🧩 Core Concepts

📤
Base64 Encoding
Convert image to a text string and embed it directly in the API request. Works everywhere, no hosting needed.
🔗
URL Reference
Pass a public URL — the model's server fetches the image. Smaller requests but requires hosting.
🎯
The Golden Rule
Tell the model: (1) what the image is, (2) what you want to know, (3) what output format. All three = great results.
🖼️
Multi-Image Prompting
Send multiple images for before/after comparison, sequence analysis, consistency checks, or multi-page docs.
💰
Token Cost
Images cost tokens based on pixel dimensions. A 1080p image ≈ 1,600 tokens in Claude. Resize and crop to save money.
👻
Confident Hallucinations
Models don't say "I'm not sure" — they invent plausible details. Always explicitly give permission to say "I can't read this."
📊
Structured Output
Request JSON, tables, or specific fields from images. Include an example schema in your prompt for consistency.
🤝
Universal API Pattern
Claude, GPT-4, Gemini all use the same concept: a message with mixed content types (image + text together).

✏️ Prompt Engineering Patterns

  1. Be Specific, Not Vague

    ❌ "Look at this image." → generic surface description
    ✅ "List every form field, its value, and whether it has a validation error." → focused, useful analysis

  2. Give Context

    Tell the model what the image is — "architecture diagram," "checkout page," "error dialog." This activates relevant domain knowledge.

  3. Request Structure

    Ask for JSON, numbered lists, or Markdown tables. Specify the exact schema you want. Include a few-shot example for complex outputs.

  4. Prevent Hallucinations

    Explicitly say: "If you can't read it, say UNREADABLE. Do not guess." Give the model permission to admit uncertainty.

💸 Cost Optimization Checklist

📐
Resize First
4K adds cost, not understanding. Resize to 1080p or 720p for most tasks.
✂️
Crop to ROI
Only care about one button? Crop to just that region. Fewer tokens, better focus.
🔍
Low Detail First
Use GPT-4o's low mode (85 tokens) for triage, then high-res only for matches.
📊
Two-Pass Batch
Processing 100 images? Filter at low-res first, analyze hits at high-res. Save 80%+ costs.

🧠 The Analogy to Remember

📚

The Well-Read Friend

Sending an image to an LLM is like showing a photo to a friend who has read every book ever written — but has never actually been outside. They can identify and describe almost everything, but sometimes they "fill in" details that aren't there, because their understanding comes from reading about the world, not experiencing it.

❓ Quick Quiz

What are the two ways to send an image to an LLM via API?

Base64 encoding (image embedded in JSON payload) and URL reference (model fetches the image from a public URL).

What are the three things the "golden rule" says to include in every image prompt?

(1) What the image is, (2) what you want to know about it, (3) what format you want the answer in.

How many tokens does a 1080p image cost in Claude?

Approximately 1,600 tokens. GPT-4o ranges from 85 (low detail) to 1,105 (high detail). Gemini Flash uses ~258 tokens.

Why are "confident hallucinations" dangerous with vision models?

Vision models don't say "I'm not sure" — they'll invent plausible details (like a fake serial number) that look exactly like correct answers. You must explicitly give permission to say "I can't read this."

What's the best cost optimization trick for batch image processing?

Two-pass approach: first send each image at low resolution for triage, then only send matching images for high-resolution analysis. Can cut costs by 80%+.

📖 Key Terms

Base64 Encoding
Converting binary image data to a text string for embedding in API requests (~33% larger than the original file).
Vision Model
An LLM with image understanding capabilities — can describe, analyze, extract, compare, and reason about visual content.
Multi-Image Prompting
Sending multiple images in a single API request for comparison, sequence analysis, or multi-page document processing.
Structured Output
Requesting model responses in a specific format (JSON, tables, lists) that can be parsed programmatically.
Confident Hallucination
When a model invents plausible-sounding details instead of admitting it can't read or understand something in the image.
Few-Shot Prompting
Including an example of the expected output format in your prompt to improve consistency of structured extraction.
Token Cost
The number of input tokens an image consumes, determined by pixel dimensions — directly affects API pricing and latency.

Key Takeaways

  • All major APIs (Claude, GPT-4, Gemini) use the same concept: text + image in one message, just different syntax.
  • The golden rule: tell the model what the image is, what you want, and what format — specificity drives quality.
  • Vision models are great at description, analysis, and extraction — but struggle with precise counting, tiny text, and spatial reasoning.
  • Always guard against confident hallucinations by explicitly allowing "I don't know" responses.
  • Resize and crop images before sending — 4K doesn't improve understanding but doubles your token cost.
  • Multi-image prompting unlocks before/after, sequence, consistency, and multi-page document analysis.
  • For batch processing: two-pass (low-res triage → high-res analysis) can save 80%+ on API costs.
