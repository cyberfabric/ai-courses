Chapter 6

Image Generation & Editing with AI

Memory Map
💡

The Big Idea

Image generation models don't "draw" — they start with pure random noise and gradually remove noise until a coherent image emerges. Your text prompt steers which patterns survive. It's sculpting from static, not painting from scratch.

🎨 How Diffusion Works

Pure Noise (TV static)
→
Predict Noise
→
Subtract It
→
Less Noisy Image
→
Repeat 20-50×
→
Final Image ✨
📸
Training
Model sees millions of images progressively destroyed by noise. Learns to predict and reverse each noise step.
🧭
CLIP Guidance
Your text prompt becomes a CLIP embedding. At each step, noise is removed in the direction that matches your description.
🔄
Reverse Direction
Vision encoders go Image → Numbers. Generators go Numbers → Image. Same embedding space, opposite direction.
🪨
The Sculptor Analogy
Diffusion = sculptor chipping away marble (noise) until the shape described by the blueprint (prompt) emerges.

✂️ Image Editing Techniques

🖌️
Inpainting
Mask out a region → describe replacement → model fills it seamlessly. Great for removing objects or swapping backgrounds.
🔲
Outpainting
Extend the canvas beyond original borders. Square headshot → landscape banner. Inpainting's bigger sibling.
🔀
img2img
Transform existing images: sketch → render, photo → painting. Strength param controls how much changes (0.3 = subtle, 0.8 = major).
🎯
ControlNet
Precise structural control via edge maps, depth, poses, or scribbles. Structure stays locked, style changes freely. SD ecosystem only.
Canny Edge
Extracts edge outlines — maintains shapes while changing style
Depth Map
Captures near/far distances — preserves 3D spatial relationships
OpenPose
Detects body joint positions — generates characters in specific poses
Scribble
Uses rough hand-drawn outlines — turns sketches into polished images
Segmentation
Labels regions (sky, ground, building) — controls what goes where

Prompt Templates
Don't let users write raw prompts. Build templates with slots to control quality and prevent misuse.
🛡️
Safety Filtering
Add your own content checks on both input prompts and output images, beyond API-level filters.
💾
Caching
Hash prompts as cache keys. Serve cached images for repeated requests instead of regenerating.
⏱️
Async + Cost Controls
Generation takes 5-15s — use job queues, never block UI. Set per-user rate limits ($0.04-0.08/image adds up).

🧪 Quick Quiz

Test Yourself

How does a diffusion model generate an image from a text prompt?

It starts with pure random noise and iteratively removes noise over 20-50 steps, guided by the text prompt's CLIP embedding, until a coherent image emerges.

What's the difference between inpainting and outpainting?

Inpainting fills in a masked region inside an existing image. Outpainting extends the canvas and generates content beyond the original borders.

What does the "strength" parameter do in img2img?

It controls how much the output deviates from the input. Low strength (0.3) = subtle changes, keeps most of the original. High strength (0.8) = major transformation, uses original as loose inspiration.

When would you choose Stable Diffusion over DALL-E 3?

When you need full control (self-hosting), custom fine-tuning (LoRA), structural guidance (ControlNet), cost optimization at scale, or data privacy. DALL-E 3 is better for quick prototyping and simple API integration.

What are the five layers of a good image prompt?

Subject, Setting, Style, Lighting, and Composition — describe what's in the scene, where it is, how it looks, how it's lit, and from what angle.

Key Takeaways

  • Diffusion models generate images by starting from noise and iteratively denoising, guided by CLIP text embeddings.
  • Prompt engineering for images uses five layers: subject, setting, style, lighting, composition.
  • Three editing techniques: inpainting (fill regions), outpainting (extend borders), img2img (transform style).
  • ControlNet adds precise structural control via edge maps, depth, and pose — Stable Diffusion ecosystem only.
  • DALL-E 3 for quick API work, Stable Diffusion for control/customization, Midjourney for aesthetics, Firefly for copyright safety.
  • Production pipelines need prompt templates, safety checks, caching, async processing, and cost controls.
  • Known weak spots: hands, text in images, consistent characters, and precise spatial layouts.
