The Big Idea

The demo is the easy part. Production multimodal = architecture + cost control + fallbacks + input/output validation + UX feedback + privacy + monitoring. Every extra modality multiplies the things that can go wrong.

🔧 Preprocessing
Normalize chaos: validate format, check size, resize images, strip EXIF metadata, detect blur. The bouncer at the club door.
✅ Postprocessing
Validate model output: parse JSON, check schema, sanity-check values, flag low confidence. Never trust model output blindly.
⏱️ Sync vs Async
User waits (<5s): image description, receipt scan, voice commands. Background (minutes): video summarization, batch processing, image generation.

🍽️ Analogy: Like a restaurant with video menu, live music, and a chef on camera — each modality adds complexity, cost, and failure points.

7 Cost Control Strategies

  1. 📐 Resize Aggressively

    512×512 for classification. 1024px for most tasks. 4K image = 4x the tokens of 1024px. 75% savings.

  2. 💾 Cache Identical Requests

    Hash image + prompt as cache key. Great for product catalogs and standard documents.

  3. 🎯 Route by Complexity

    GPT-4o-mini for classification. Claude Sonnet for OCR. Opus for complex reasoning. Don't send everything to the expensive model.

  4. 📦 Batch Where Possible

    Batch APIs are 50% cheaper. 50 product photos = 1 batch, not 50 individual calls.

  5. 🎬 Sample Video Frames Smartly

    Scene detection → 20-40 frames vs 600 frames at 1fps. Same understanding, fraction of the cost.

  6. 🚦 Set Per-User Limits

    "5 free image analyses per day." Prevents abuse, controls costs predictably.

  7. 🔔 Monitor Daily + Cost Alerts

    One rogue user = hundreds of dollars in a day. Alert if daily spend exceeds 2x average.

🔒 Privacy: Images & Audio Are PII Magnets

📸 Images
Contain faces, locations, documents, credit card digits. A "receipt photo" leaks more than you think.
🎤 Audio
Voice = biometric data. Storing recordings triggers GDPR/BIPA. Treat like fingerprints.
📍 EXIF Data
GPS coordinates embedded in smartphone photos. Always strip before processing or storing.
🗑️ Minimize Retention
Delete raw inputs after processing. Store only extracted structured data. Set clear TTLs.
⚖️ Involve Legal Early
Don't build the feature then ask if it's OK. Privacy controls cost 10x more to add after launch.

🧪 Quick Quiz

Test Your Understanding

Why should you resize images before sending to vision APIs?

Image tokens are expensive. A 4000×3000 image may cost ~1,600 tokens; resized to 1024×768 it's ~400 tokens. 75% cheaper with the same quality for most use cases. Always resize to the model's optimal resolution.

What's the fallback chain pattern?

Primary model → retry with smaller image → fallback model (e.g., GPT-4o-mini) → OCR only (Tesseract) → ask user to re-upload. Graceful degradation: always provide some value, never fail silently.

Why is "perceived latency" more important than actual latency?

A 3-second response with immediate feedback ("Analyzing your image... Step 2/3: Extracting text...") feels faster than a 2-second response with no feedback. Stream responses, show progress steps, and render results progressively.

Why are images and audio "PII magnets"?

Images contain faces, GPS coordinates (EXIF), documents, and credit card digits. Audio is biometric data (voice = identity). A "simple receipt photo" can leak location, payment info, and timestamp. Strip metadata, minimize retention, involve legal early.

Name three things to monitor that are unique to multimodal features.

1) Cost per request (multimodal = 5-10x text cost, can spike). 2) Input rejection rate (if >30%, UX problem). 3) Output validation failures (model updates can silently degrade quality). Plus cache hit rate and token usage per request.

Key Takeaways

  • Production pipeline: preprocess (normalize inputs) → model → postprocess (validate outputs). Never trust raw user input or raw model output.
  • Multimodal costs 5-50x more than text. Resize images, cache responses, route by complexity, batch requests, set per-user limits. Run the math before launch.
  • Build fallback chains: primary model → retry → fallback model → basic extraction → user re-upload. Never fail silently.
  • Perceived latency > actual latency. Stream responses, show step-by-step progress, display confidence levels, give actionable error messages.
  • Images and audio are PII magnets. Strip EXIF, minimize retention, treat voice as biometric data, involve legal early.
  • Monitor: latency (p50/p90/p99), error rate by modality, daily cost, cache hit rate, output validation failures. Watch for silent quality degradation.
