The Multimodal Toolkit: What to Use When

The Big Idea

Every multimodal choice maps along four axes: Task (what?), Latency (how fast?), Budget (how much?), and Privacy (can data leave?). Get these right and the tool choice almost makes itself. Use each model for what it's best at, then connect them with clean interfaces.

🎬 Video & ⚡ Real-Time

Gemini native video upload, or keyframe extraction → Claude/GPT-4o.

⚡ Voice + camera
Gemini Live (~300-600ms). Native multimodal streaming with video input and screen sharing.
⚡ Custom pipeline
Whisper + LLM + TTS (~1-3s). Full control, any model combo. Use LiveKit or Pipecat for plumbing.

🔓 The "Whisper Rule"

📏

When Open Source = No-Brainer

If an open-source model is genuinely as good as the best proprietary option, use open source. You get cost savings, privacy, and control with no quality trade-off. But don't use mediocre open source just to avoid paying — the quality gap matters to users.

🧪 Quick Quiz

Test Your Understanding

What are the four axes of the decision matrix?

Task (what are you doing?), Latency (how fast?), Budget (how much per request?), and Privacy (can data leave your infrastructure?). Privacy often overrides all other considerations.

When should you use YOLO vs Claude for vision?

YOLO for object detection (bounding boxes) — <50ms, runs locally, free. Claude for reasoning about images (describe, extract structured data, answer questions) — needs cloud API but far more capable for understanding. Use YOLO for "what objects are here" and Claude for "what does this mean."

What's the "Whisper rule" for choosing open source vs proprietary?

If open source is genuinely as good as proprietary (Whisper for STT, YOLO for detection, SAM for segmentation), use open source — free, private, controllable with no quality trade-off. But don't use mediocre open source just to avoid API costs; the quality gap matters to users.

What's the recommended approach for video with mostly talking heads?

Focus on the audio track: extract audio → Whisper → LLM summarize. Only sample a few frames for slide content or speaker identification. The real information is in what people say, not what they look like sitting still.

Why is hybrid (on-device + cloud) often the best architecture?

On-device gives instant feedback and privacy (e.g., YOLO detects a receipt in <50ms). Cloud provides deep reasoning (e.g., Claude extracts structured data in 2-3s). Users get immediate acknowledgment while heavy processing happens in the background. Best of both worlds.

Key Takeaways

  • Decision matrix: Task × Latency × Budget × Privacy → tool choice. Privacy often overrides everything else.
  • Vision: Claude/GPT-4o for reasoning (80% of use cases). YOLO for detection, SAM for segmentation (open source, no contest).
  • Audio: Whisper for batch STT (free, multilingual). Deepgram for real-time. OpenAI TTS as default, ElevenLabs for premium.
  • The "Whisper rule": if open source matches proprietary quality (Whisper, YOLO, SAM), always choose open source.
  • Hybrid on-device + cloud is the winning architecture. Instant local feedback + deep cloud reasoning.
  • Re-evaluate quarterly. Models that are best today may not be best in six months. Automate your benchmarks.
  • Multimodal AI isn't a separate field — it's where all of AI is heading. Every major model is becoming multimodal.
