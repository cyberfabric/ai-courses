The Fusion Problem

Memory Map
🔀

The Big Idea

Fusion is how models combine embeddings from different modalities into one understanding. Three approaches — stew (early), bento box (late), jazz band (cross-attention) — each with distinct trade-offs. Most modern LLMs use cross-attention: the sweet spot of richness and efficiency.

🍳 The Three Fusion Strategies

🥘
Early Fusion: The Stew
Mix all raw data into one pot from the start. Deepest understanding, but most expensive. Example: Gemini.
🍱
Late Fusion: The Bento Box
Cook each modality separately, combine results at the end. Cheap, modular, but no cross-modal nuance. Example: CLIP search.
🎵
Cross-Attention: The Jazz Band
Each modality processes separately but listens to the others. Text guides where to look in the image. Example: GPT-4V, Claude.

🧭 Which Fusion for Which Task?

  1. Image / Audio Search

    Late fusion — encode everything in advance, compare at query time. Fast and scalable.

  2. Visual Q&A / Document Analysis

    Cross-attention — the question must guide where the model looks in the image.

  3. Content Moderation at Scale

    Late fusion — pre-encode, compare against flag categories. Needs speed at scale.

  4. Real-Time Voice + Vision Assistants

    Cross-attention or early fusion — needs deep, simultaneous understanding of audio and visual input.

📖 Key Terms

Fusion
The process of combining embeddings from different modalities into a unified representation.
Early Fusion
Mix raw data from all modalities before processing — one model handles everything from the start.
Late Fusion
Process each modality separately, combine only the final results. Modular and efficient.
Cross-Attention
Tokens from one modality "attend to" tokens from another during processing — text guides image understanding.
CLIP
Contrastive Language-Image Pre-training — places images and text in the same embedding space via contrastive learning.
Contrastive Learning
Training by pushing matching pairs (image + correct caption) close and non-matching pairs far apart in embedding space.
Shared Embedding Space
A vector space where different modalities coexist — enabling direct comparison between images and text.
Zero-Shot Classification
Classifying inputs without task-specific training data — just compare against text descriptions.
Projection Layer
A small neural network that maps one encoder's embeddings into another model's embedding space.

❓ Quick Quiz

What's the core difference between early, late, and cross-attention fusion?

Early: combine raw data before processing. Late: process separately, combine results. Cross-attention: process separately but let modalities "look at" each other during processing.

Why is cross-attention so good for visual Q&A?

Because the text question can guide which parts of the image the model focuses on. "What color is the car?" makes the model attend to car-containing patches — something late fusion can't do.

What did CLIP's contrastive learning achieve?

It trained image and text encoders to produce embeddings in the same space — matching image-caption pairs are close, non-matching are far. This enables zero-shot classification and cross-modal search.

When should you prefer late fusion over cross-attention?

When you need speed and scale: image search, content moderation, recommendation systems. Pre-encode everything, compare at query time. No deep cross-modal reasoning needed.

📌 Key Takeaways

  • Fusion = how you combine embeddings from different modalities. It's the core challenge of multimodal AI.
  • Three strategies: stew (early — deep but expensive), bento box (late — cheap but no cross-modal nuance), jazz band (cross-attention — sweet spot).
  • Cross-attention lets text guide where the model looks in an image — that's why visual Q&A works so well.
  • CLIP placed images and text in the same embedding space using contrastive learning on 400M pairs. This powers image search, zero-shot classification, and generation guidance.
  • GPT-4V and Claude use bolted-on vision (modular). Gemini uses native multimodal (deep). Both work well — choose by task performance, not architecture.
