CHAPTER 6

Image Generation & Editing with AI

From noise to masterpiece — how diffusion models create images from words

From Understanding to Creating

For the last two chapters, we've been teaching AI to look at images — describing them, extracting data, reading documents. Now we flip the script entirely. Instead of AI understanding images, we're going to make AI create them.

Type a sentence, get a picture. That's the promise of modern image generation, and it works astonishingly well. You've probably seen it in action — DALL-E, Midjourney, Stable Diffusion — but how does it actually work? And more importantly, how do you use it in real software?

The Big Idea

Image generation models don't "draw" pictures the way an artist does. They start with pure random noise — TV static — and gradually remove the noise until a coherent image emerges. Your text prompt guides which noise gets removed and which patterns get kept. It's sculpting, not painting.

If that sounds wild, stay with me. By the end of this chapter, you'll understand the core mechanism, know how to use the major APIs, and have a clear picture of what image generation can and can't do for your projects.

How Diffusion Models Work (No Math, Promise)

The technique behind DALL-E, Stable Diffusion, and most modern image generators is called diffusion. Here's the intuition:

Training: Learning to Un-Noise

During training, the model sees millions of real images. For each image, the training process adds noise — a little at first, then more and more, until the image is completely destroyed into random static. The model's job is to learn the reverse: given a noisy image, predict what the noise looks like so it can be subtracted.

Think of it like this: you show someone a clear photo of a dog, then progressively smear Vaseline on the lens. At each step, you ask them, "What got blurry? What was there before?" After seeing millions of photos get progressively blurred, they get extremely good at un-blurring.

The Diffusion Process

Pure Noise → Predict Noise → Subtract It → Less Noisy Image → Repeat 20-50x → Final Image

Generation: Sculpting from Static

When you ask the model to generate an image, it starts with a grid of pure random noise. Then it runs the denoising process step by step — typically 20 to 50 steps. At each step, the model looks at the current noisy image and predicts what noise to remove. After enough steps, a clean image emerges.

But how does the model know to make a dog instead of a car? That's where your text prompt comes in.

Text Guidance: CLIP Steers the Ship

Remember CLIP from Chapter 3? The model that puts images and text in the same embedding space? It's back, and it's the secret sauce of text-to-image generation.

Your text prompt gets converted into a CLIP embedding. At each denoising step, the model doesn't just remove noise randomly — it removes noise in a direction that makes the emerging image more similar to your text embedding. The text acts like a compass: "This way lies 'a golden retriever playing in autumn leaves,' so remove noise in that direction."

The Vision Modality in Reverse

In Chapters 4 and 5, vision encoders converted images into embeddings so language models could understand them. Image generation does the exact opposite: it starts with a text embedding and uses it to construct an image. Same embedding space, reverse direction. The encoder goes Image → Numbers. The generator goes Numbers → Image. This is why multimodal AI is so powerful — once everything lives in the same numerical space, you can translate freely between modalities.

The analogy from the chapter outline nails it: diffusion is like a sculptor starting with a block of marble (noise) and gradually chipping away everything that doesn't look like the description. The text prompt is the blueprint.

Generating Images via API

Let's get practical. Here's how to generate images with the two most developer-friendly APIs:

OpenAI (DALL-E 3)

from openai import OpenAI

client = OpenAI()
response = client.images.generate(
    model="dall-e-3",
    prompt="A cozy coffee shop interior with warm "
           "lighting, plants on shelves, and a cat "
           "sleeping on a windowsill. Watercolor style.",
    size="1024x1024",
    quality="standard",
    n=1
)
image_url = response.data[0].url
print(image_url)

That's it. One API call, one image URL. The quality parameter can be "standard" or "hd" (more detail, higher cost). The size can be "1024x1024", "1792x1024", or "1024x1792".

Stability AI (Stable Diffusion)

import requests, base64

resp = requests.post(
    "https://api.stability.ai/v2beta/stable-image"
    "/generate/core",
    headers={"Authorization": f"Bearer {API_KEY}",
             "Accept": "application/json"},
    files={"none": ""},
    data={"prompt": "A cozy coffee shop, watercolor",
          "output_format": "png"}
)
img_data = base64.b64decode(resp.json()["image"])
with open("coffee_shop.png", "wb") as f:
    f.write(img_data)
DALL-E 3 rewrites your prompts. When you send a prompt to DALL-E 3, it actually uses GPT-4 to expand and improve your prompt before generating the image. You can see the revised prompt in the API response (response.data[0].revised_prompt). This usually helps, but if you need exact control, it can be surprising. Stable Diffusion and Flux use your prompt verbatim.

Prompt Engineering for Image Generation

Writing prompts for image generation is a different skill than writing prompts for text LLMs. With text, you're having a conversation. With images, you're writing a scene description. Here's what works:

The Anatomy of a Good Image Prompt

Think in layers: subject, setting, style, lighting, composition.

Vague prompt
A dog

You'll get a generic dog. The model fills in every missing detail with its own "average" choices. No personality, no context.

Detailed prompt
A golden retriever puppy sitting in a field of wildflowers at golden hour. Soft bokeh background, eye-level perspective. Photography style, warm tones, natural lighting.

Subject (golden retriever puppy), setting (wildflower field), timing (golden hour), composition (eye-level), style (photography), mood (warm).

Style Keywords That Actually Work

Category Keywords Effect
Medium watercolor, oil painting, pencil sketch, digital art, 3D render Changes the overall art style
Photography DSLR, macro, portrait, wide-angle, aerial, drone shot Realistic photographic look
Lighting golden hour, dramatic lighting, neon, backlit, studio lighting Sets mood and atmosphere
Detail highly detailed, minimalist, intricate, simple, abstract Controls level of complexity
Mood serene, dramatic, whimsical, dark, vibrant, pastel Emotional tone of the image
Negative prompts matter (for Stable Diffusion and Flux). You can tell the model what to avoid: "blurry, low quality, distorted hands, extra fingers." DALL-E 3 doesn't support negative prompts directly, but Stable Diffusion and Flux do, and they make a significant difference in output quality.

Beyond Generation: Editing Existing Images

Generating images from scratch is flashy, but editing existing images is often more useful in practice. There are three core editing techniques you should know:

Inpainting: Fill in the Blanks

Inpainting lets you mask out part of an image and have the model fill it in. Imagine you have a product photo but the background is ugly — mask the background, describe what you want instead, and the model generates a new background that blends seamlessly with the product.

Inpainting Pipeline

Original Image → Create Mask → Describe Replacement → Model Fills Masked Area → Blended Result

Use cases that ship today: removing unwanted objects from photos (Google's Magic Eraser), replacing product backgrounds for e-commerce, fixing damaged or corrupted image regions.

Outpainting: Extend Beyond the Frame

Outpainting is inpainting's bigger sibling. Instead of filling in a masked region inside the image, you extend the canvas and have the model generate what's beyond the original borders. Got a square headshot but need a landscape banner? Outpaint the sides.

Image-to-Image (img2img): Transform What Exists

Image-to-image takes an existing image as a starting point and transforms it according to your prompt. Instead of starting from pure noise, the diffusion process starts from a noisy version of your input image. The result preserves the overall composition and structure while changing the style or content.

Great for img2img

Sketch to render: Hand-drawn wireframe → polished UI mockup

Style transfer: Photo → watercolor, pencil sketch, anime

Season change: Summer photo → same scene in winter

Quick concepts: Rough 3D model → photorealistic preview

Not ideal for img2img

Precise edits: "Move this object 3 inches left" — too spatial

Text changes: "Change the sign to say X" — text is hard

Identity preservation: "Same person, different pose" — inconsistent

Exact replication: "Exactly this, but blue" — too imprecise

The "strength" parameter controls how much changes. In img2img, a strength of 0.3 means "keep most of the original, just adjust the style." A strength of 0.8 means "use the original as loose inspiration and mostly regenerate." Start at 0.4-0.5 and adjust from there.

ControlNet: Precision Guidance

Regular img2img preserves the general structure of your input. ControlNet preserves the exact structure. It's the difference between "roughly this layout" and "exactly these edges and poses."

ControlNet works by extracting a structural map from your input image — edges, depth, human poses, or other structural features — and using that map to constrain the generation process. The model can change colors, textures, and styles freely, but the structure stays locked.

Types of ControlNet Guidance

Control Type What It Extracts Best For
Canny Edge Edge outlines of objects Maintaining shapes while changing style
Depth Map Near/far distance information Preserving 3D spatial relationships
OpenPose Human body joint positions Generating characters in specific poses
Scribble Rough hand-drawn outlines Turning sketches into polished images
Segmentation Region labels (sky, ground, building) Controlling what goes where in the scene

ControlNet is currently a Stable Diffusion ecosystem feature. DALL-E 3 and Midjourney don't offer this level of structural control (yet). If you need precision guidance in your pipeline, Stable Diffusion + ControlNet is the stack.

ControlNet requires local setup or specialized hosting. Unlike the simple API calls for DALL-E, running ControlNet means self-hosting Stable Diffusion with the ControlNet extension, or using a service like Replicate or Fal.ai that offers ControlNet endpoints. It adds complexity but the control is unmatched.

Practical Use Cases (That Aren't Art)

Image generation isn't just for creating pretty pictures. Here are use cases that actually ship in production software:

Placeholder and Prototype Images

Need realistic images for a UI prototype before the photo shoot? Generate them. Product mockups, user avatars, background images, hero banners — all can be generated for design reviews and demos. Way better than gray rectangles or Lorem Ipsum for images.

E-commerce Product Photos

Take a product photo on a white background, use inpainting to place it in lifestyle contexts. A backpack on a white background becomes a backpack on a hiking trail, at a coffee shop, or in a dorm room — without a photographer, studio, or location shoot.

Documentation Illustrations

Technical docs with custom illustrations are more engaging. Generate diagrams, conceptual illustrations, or icons that match your brand style. Some teams generate all their blog post header images this way.

Data Augmentation for ML

Training a computer vision model but don't have enough training images? Generate synthetic variations. A model trained on both real and generated images often performs better than one trained on limited real data alone.

Try It Now: Generate UI Mockup Assets

Open ChatGPT (with DALL-E) or use the OpenAI API and try generating assets for a hypothetical app:

  1. Generate an app icon: "A flat design app icon for a meditation app. Calm blue gradient background, minimalist white lotus symbol, rounded corners. iOS app icon style."
  2. Generate a hero image: "A serene mountain lake at sunrise, soft pastel colors, gentle mist. Wide aspect ratio, suitable as a mobile app header. Photography style."
  3. Generate a set of category icons: "A set of 4 minimalist line icons on white background: sleep, focus, relax, breathe. Consistent thin line style, blue color, simple shapes."

Compare the results. Which prompts produced usable output? Which needed iteration? This is the reality of working with image generation — it's a conversation, not a one-shot command.

What Image Generation Still Can't Do

For all its power, current image generation has clear, consistent failure modes. Knowing them saves you hours of frustration:

Hands and Fingers

The meme is real. Models routinely generate six fingers, merged fingers, impossible hand positions, and other anatomical horrors. It's gotten much better (DALL-E 3 and Flux handle hands reasonably well), but it's still the most common artifact. Always inspect hands in generated images.

Text in Images

Generating readable, correctly spelled text inside images is surprisingly hard. Models improve every generation — DALL-E 3 can handle short words fairly well — but longer text, unusual fonts, or specific formatting still fail frequently. If you need text in images, consider generating the image without text and compositing the text on top programmatically.

Consistent Characters

Generating the same character across multiple images (same face, same outfit, same style) is one of the hardest unsolved problems. Each generation is independent, so "the same person" in two images will look different. Solutions exist (fine-tuning, IP-Adapter, reference images) but none are turnkey.

Exact Spatial Relationships

"A red ball on top of a blue cube, with a green pyramid to the left" sounds simple, but models frequently get spatial relationships wrong. They're great at mood and style, weaker at precise layout.

Exact Specifications

"Make the logo exactly 200x50 pixels, with the text 'Acme Corp' in Helvetica Bold" — this is not what generative models are for. They're creative tools, not precision layout engines. Use Figma or Photoshop for pixel-perfect work.

Check generated images before using them. Models can generate offensive content, copyrighted characters, or recognizable real faces even when not intended. Most APIs have safety filters, but they're not perfect. Always review generated images before publishing, especially in production.

Ethics and Copyright: The Elephant in the Room

Image generation raises genuine ethical and legal questions that you should understand as a developer building with these tools:

Training Data

These models were trained on billions of images scraped from the internet, including copyrighted artwork. Artists have raised legitimate concerns, and lawsuits are ongoing. The legal landscape is still being defined. Some models (like Adobe Firefly) are trained exclusively on licensed or public domain images to avoid this issue.

Deepfakes and Misuse

The same technology that generates beautiful landscapes can generate fake photos of real people. All major APIs have safeguards against generating images of public figures or explicit content, but the open-source models can be used without these restrictions. If you're building a product that lets users generate images, you need content moderation.

Watermarking and Provenance

The industry is moving toward invisible watermarks and metadata standards (like C2PA) that mark AI-generated images. DALL-E adds metadata to generated images. This helps with transparency — people should know when they're looking at AI-generated content.

Practical policy for your team: If you're using generated images in a product, establish clear guidelines. Can generated images be used in marketing? In user-facing UI? Do they need to be labeled as AI-generated? Check your company's legal and brand policies before shipping generated content.

Building a Generation Pipeline

In a real application, you rarely just call the API and display the result. Here's a practical pipeline pattern:

from openai import OpenAI
import requests

def generate_product_image(description, style="photo"):
    client = OpenAI()
    prompt = (f"Product photo: {description}. "
              f"Clean white background, professional "
              f"{style} style, soft studio lighting. "
              f"No text or watermarks.")
    resp = client.images.generate(
        model="dall-e-3", prompt=prompt,
        size="1024x1024", quality="hd", n=1)
    # Download and save
    img_resp = requests.get(resp.data[0].url)
    return img_resp.content  # raw PNG bytes

Production Image Generation Pipeline

User Request → Build Prompt → Generate Image → Safety Check → Post-Process → Cache & Serve

Key things to add in production:

  • Prompt templates: Don't let users write raw prompts. Build templates with slots they can fill ("Generate a [product type] in [scene]"). This controls quality and prevents misuse.
  • Safety filtering: Even with API-level filters, add your own content checks on both the input prompt and the output image.
  • Caching: If the same prompt is requested multiple times, serve the cached image instead of regenerating. Hash the prompt for cache keys.
  • Async processing: Image generation takes 5-15 seconds. Never block the UI. Use a job queue and notify the user when the image is ready.
  • Cost controls: Set per-user rate limits. At $0.04-0.08 per image, a user generating 100 images costs $4-8. Multiply by your user base.

Hands-On Exercise

Exercise: Image Generation for a Hypothetical App

You're building a travel app. Generate the following assets using DALL-E (via ChatGPT or API) or any other image generation tool:

  1. App icon: Generate a travel app icon. Try at least 3 different prompt styles (flat design, 3D, photorealistic) and pick the best.
  2. Hero images: Generate hero images for 3 destination pages: Tokyo, Paris, and Iceland. Each should have a distinct mood but consistent style.
  3. Category icons: Generate a consistent set of icons for: flights, hotels, restaurants, and activities.
  4. Before/after comparison: Take any photo and try img2img to transform it into a different style (sketch, painting, different season).

Reflection questions:

  • How many attempts did each image take to get a usable result?
  • What prompt changes had the biggest impact on output quality?
  • Which generated assets would you actually use vs. which need a designer's touch?
  • What's the total cost if you generated all of these via API?
