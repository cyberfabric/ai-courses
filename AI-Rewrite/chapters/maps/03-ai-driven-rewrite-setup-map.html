⚡ The Four Rewrite Skills

Skill What It Does Key Steps
/migrate-to-module Move a legacy file to its target module Read tracker → apply rules → create file → write tests → update tracker
/extract-domain Pull domain logic from a God Activity Identify data classes → create repo interfaces → create use cases → tests
/add-di Generate Hilt module for a Gradle module Read public API → @Binds for interfaces → match scope to module type
/generate-tests-for-migration Tests for code about to be deleted Read legacy file → test behavior (not implementation) → fakes over mocks

🚧 The Three Quality Gate Hooks

  1. 🚫 Block New Legacy Code

    PreFileWrite hook blocks any new files in app/src/main/java/legacy/**. The most critical hook.

  2. 🔍 Forbidden Pattern Scanner

    Post-creation scan for LiveData, raw SQLiteDatabase, Fragment usage in new code. Auto-blocks violations.

  3. 🧪 Migration Test Gate

    Before deleting legacy files, checks that replacement tests exist. No tests = no deletion.

🧪 Quick Quiz

What are the four systems in the rewrite command center?

CLAUDE.md (context), rewrite skills (automation), quality gate hooks (enforcement), migration tracker (progress).

Which is the most critical quality gate hook?

Block New Legacy Code — prevents AI from adding any new files to the monolith module. All new code goes to target modules.

What's the branch strategy for an AI-driven rewrite?

One branch per architecture layer (not per file). Each branch compiles independently. Never rewrite on main. PR review per layer against its checklist.

What are the status transitions in the migration tracker?

REMAINING → IN_PROGRESS → MIGRATED. No legacy file is deleted until status is MIGRATED and replacement tests pass.

Why run a dashboard prompt at the start of every session?

Takes 10 seconds. Ensures you and AI are aligned on what's done, what's in progress, and what to migrate next. Prevents duplicate work.

🗝️ Key Takeaways

  • Build four systems: CLAUDE.md, skills, hooks, tracker — your rewrite command center.
  • Rewrite CLAUDE.md adds: migration status, forbidden patterns, skills list, naming conventions.
  • Four core skills: /migrate-to-module, /extract-domain, /add-di, /generate-tests-for-migration.
  • Three hooks: block legacy writes, scan forbidden patterns, require tests before deletion.
  • Migration tracker maps every file: legacy path → target module → status → new path → tests.
  • Branch per layer, not per file. Each branch compiles independently. Never rewrite on main.
  • Start every session with a dashboard prompt. 10 seconds to align on status and next action.
  • Session loop: Dashboard → Pick File → Run Skill → Hooks Check → Tracker Update. Repeat 10-15x.
