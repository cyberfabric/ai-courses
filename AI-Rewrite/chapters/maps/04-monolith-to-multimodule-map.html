Chapter 4

From Monolith to Multi-Module

Memory Map
💡

The Big Idea

Create 10 empty Gradle modules with correct dependency wiring, convention plugins, and a version catalog — the skeleton that compiles with zero source files. Get the structure right before any code moves in.

📋 The 7-Step Module Generation Process

  1. 🔍 Dependency Graph

    Prompt AI to map internal file-to-file coupling. Classify: UI→Logic, Logic→Data. Find circular deps.

  2. 🏗️ Generate Modules

    settings.gradle.kts with all 10 modules. Each gets a build.gradle.kts with approved dependencies only.

  3. 🔌 Convention Plugins

    4 plugins in build-logic/: android.library, android.feature, android.application, kotlin.library.

  4. 📦 Version Catalog

    libs.versions.toml — single source of truth for all dependency versions. No hardcoded strings.

  5. 📐 Add Graph to CLAUDE.md

    Dependency graph + FORBIDDEN section as permanent AI context.

  6. ✅ Validate

    5-point check: circular deps, forbidden deps, missing deps, visibility leaks, plugin consistency.

  7. ⚙️ First Compile

    ./gradlew assembleDebug on the empty project. Fix errors. Commit to rewrite/setup branch.

recipes.kotlin.library
Pure Kotlin, no Android. Used by :core:domain. Ensures zero Android deps.
recipes.android.library
Shared Android config (compileSdk, minSdk, Kotlin options). Base for data/core modules.
recipes.android.feature
Extends library + adds Compose + Hilt. Applies org.jetbrains.kotlin.plugin.compose (separate Kotlin plugin since Kotlin 2.0). Used by all :feature:* modules.
recipes.android.application
App module config. Full Android application setup.

⚠️ Common AI Mistakes

Mistake Why It Happens How to Catch
:feature → :data:local shortcut AI "helps" by adding direct data access PKG-005: features depend on :core:domain only
Circular dependency Shared utility references create cycles Extract interface to :core:domain
Hardcoded versions AI inlines versions despite catalog GRADLE-001: all versions in libs.versions.toml
KAPT instead of KSP AI uses older patterns from training data Explicit "KSP only" in CLAUDE.md
Inline plugin application AI writes id("com.android.library") directly Convention plugins are non-negotiable

📏 Rules Introduced

PKG-004
build.gradle.kts MUST declare only approved dependencies. No transitive shortcuts.
PKG-005
Feature modules depend on :core:domain for interfaces, NEVER on :data:* directly. Hilt wires implementations via :app.
GRADLE-001
All dependency versions in libs.versions.toml. No hardcoded versions. No version ranges.

🧪 Quick Quiz

Why run a dependency graph analysis before creating modules?

To map internal coupling and find circular dependencies that must be broken before files can be split across modules. Without this, the module boundaries won't be clean.

What does :core:domain depend on?

Nothing. It's pure Kotlin with zero dependencies. Everything else depends on it, but it depends on nothing.

Why can't :feature:recipe depend on :data:repository directly?

Features depend on repository interfaces in :core:domain, not implementations in :data:repository. Hilt wires the concrete implementation at runtime through :app's DI module.

What are the four convention plugins and what modules use them?

kotlin.library (:core:domain), android.library (:core:*, :data:*), android.feature (:feature:*), android.application (:app). Each eliminates config duplication.

What should the project look like after this chapter?

10 Gradle modules, all compiling, with zero source files in new modules. Convention plugins, version catalog, and dependency graph in CLAUDE.md. The monolith is untouched.

🗝️ Key Takeaways

  • Map internal dependencies first — circular deps must be broken before modularization.
  • Generate all 10 modules with explicit dependency graph as AI constraint.
  • Convention plugins eliminate duplication: 4 plugins cover all module types.
  • Version catalog (libs.versions.toml) is the single source of truth. No hardcoded versions.
  • Add the dependency graph + FORBIDDEN section to CLAUDE.md as permanent context.
  • Validate before coding: 5-point check catches structural errors while they're cheap to fix.
  • The sneakiest AI mistake: adding :feature → :data shortcuts. Features use domain interfaces only.
  • End state: 10 compiling modules, 0 source files, monolith untouched. Skeleton up, rooms empty.
