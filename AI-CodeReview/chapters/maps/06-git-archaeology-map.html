Chapter 6

Understanding What Changed

Memory Map

💡 The Big Idea

🪨

Git History = Sedimentary Rock Layers

Each commit is a layer. A geologist reads rock layers to understand what happened millions of years ago. You read git history to understand what happened last Tuesday. AI agents deposit layers fast — sometimes 20 commits in a session — and the layers don't always explain themselves.

🔧 The 4 Essential Git Commands

  1. git diff --stat — The Shape

    Which files changed? How much? Where's the weight? Start here before reading any code.

  2. git log --oneline --graph — The Story

    See the sequence. Did the AI build logically (bottom-up) or dump everything in one commit?

  3. git show <commit> — The Detail

    Full diff for one commit. Good messages tell you what and why. Bad ones tell you nothing.

  4. git blame -L start,end file — The Origin

    Who wrote each line, and when? Spot scope creep: lines added in unrelated commits.

These 4 commands cover 90% of AI code archaeology.

🔬 Advanced Diff Strategies

📝
--word-diff
Highlights specific words that changed, not entire lines. Perfect for AI renames and small refactors.
🎨
--color-moved
Moved code colored differently from new code. Instantly see: "this was relocated, not written."
📂
--name-only
Just file names, no content. Add --diff-filter=A for added-only or =M for modified-only.
🔍
-S "string"
Pickaxe search: find the commit that introduced or removed a specific function or string.

🚨 Watch Out For

🚩 Red Flag: Massive Unexplained Commit
Hundreds of changed lines, commit message says "AI changes." A black box you'll have to reverse-engineer when it breaks at 2 AM.
🤖 AI Tell: Scope Creep Across Commits
You asked for rate limiting. The commit also refactored logging, renamed a variable in orders.ts, and added a helper "for future use." Only 1 of 4 changed files was requested.
💬 Ask AI to Explain
"Explain the diff between HEAD and HEAD~5. For each commit: what changed, why, and what assumptions about the codebase?" Better than "What did you change?"
🕐 The Friday Afternoon Scan
10 minutes every Friday: git log --oneline --since="5 days ago". For any AI commit you don't recognize, run git show. Catches comprehension debt early.

📖 Git Command Quick Reference

git diff --stat HEAD~5..HEAD
Shape of changes: files, insertions, deletions across a range.
git log --oneline --graph -10
Story of recent commits: sequence, branching, structure.
git blame -L 45,55 -w file.ts
Who wrote lines 45-55, ignoring whitespace changes.
git log -S "functionName" --oneline
Pickaxe: find the commit that introduced or removed a specific string.
git log --author="Claude" --since="2 weeks ago"
All AI-generated commits in a time range.
git diff main...HEAD --stat
Full scope of a feature branch vs main.

❓ Quick Quiz

Test Your Understanding

What are the 4 essential git commands for AI code archaeology?

git diff --stat (shape), git log --oneline --graph (story), git show (detail), git blame (origin). They cover 90% of investigation needs.

What's the "refactor diff stack"?

Three commands in order: git diff --stat (shape) → --color-moved=dimmed-zebra (separate moved from new) → --word-diff (actual changes within lines). Turns 500-line refactors into 10-minute reads.

How do you spot AI scope creep in git history?

Run git diff --stat and check if the commit touches files unrelated to its stated purpose. If you asked for rate limiting and the commit also modified logging and orders, that's scope creep.

What are git trailers and why use them for AI code?

Machine-parseable metadata at the end of commit messages: Generated-with: Claude Code, Reviewed-by: Name. They let you filter AI commits with git log --grep="Generated-with:".

What's the "Friday afternoon scan"?

10 minutes every Friday: run git log --since="5 days ago", scan AI commits you don't recognize, git show anything suspicious. Catches comprehension debt before it compounds.

✅ Key Takeaways

If You Remember Nothing Else

  • Git history is your map — read it like sedimentary rock layers to understand what AI did and when.
  • 4 commands cover 90%: diff --stat (shape), log --oneline --graph (story), show (detail), blame (origin).
  • The refactor diff stack: stat → color-moved → word-diff. Turns intimidating diffs into readable ones.
  • Use git trailers (Generated-with:, Reviewed-by:) to track AI provenance in commits.
  • Watch for scope creep: AI commits that touch unrelated files beyond the requested task.
  • 100+ changed lines + vague message = must be broken up before merging.
  • The Friday scan: 10 minutes weekly reviewing the week's AI commits keeps your mental model current.
