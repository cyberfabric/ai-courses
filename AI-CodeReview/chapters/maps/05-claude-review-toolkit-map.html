💡 The Big Idea

🔧

A Mechanic's Guide to the Tool on Your Workbench

Claude Code Review dispatches a fleet of specialized agents (running on Opus 4.7 as of April 2026, with expanding multi-repo context) that analyze your diff in parallel, verify findings to filter false positives, and post inline comments ranked by severity. It's already at your fingertips — no extra subscription needed.

🤖 The Specialized Agents

🎯
Correctness Agent
Logic errors, off-by-one, edge cases. Verifies code does what the PR description claims.
🛡️
Security Agent
Injection vulns, auth bypasses, insecure data handling, dependency risks.
⚡
Performance Agent
N+1 queries, unnecessary re-renders, memory leaks, algorithmic inefficiencies.
📐
Style & Consistency
Project conventions, naming patterns, documentation standards.

The verification step filters false positives before you see them — <1% of findings flagged as incorrect.

🚦 Severity Markers

Marker Severity Action

Tip: Always expand the collapsible reasoning section. Rate findings with 👍/👎 to improve future reviews.

⚙️ Trigger Modes

Mode When It Runs Best For
Once after PR creation When PR opens or marked ready Most repos — one review, one cost
After every push On each push to the PR Security-critical repos, auto-resolves fixed issues
Manual @claude review or @claude review once High-traffic repos, draft PRs, spot checks
@claude review
Starts a review AND subscribes the PR to push-triggered reviews going forward.
@claude review once
Single review, no subscription. Best for long-running PRs or one-off second opinions.

📄 REVIEW.md vs. CLAUDE.md

File Purpose Example Content
REVIEW.md Review-only guidance. What to flag/skip during review. "Always check: new endpoints have integration tests." "Skip: generated files under src/gen/"
CLAUDE.md General project instructions for all tasks including review. "packages/auth owns all permissions." "apps/api is the only service that mutates billing."

CLAUDE.md = how your system is shaped. REVIEW.md = what to care about during review.

🔄 The Optimal Combined Workflow

  1. PR Opens

    Claude Review triggers automatically. Fleet of agents analyzes in parallel.

  2. Read Findings

    Check severity table, expand reasoning sections. Rate with 👍/👎.

  3. Fix & Push

    Address 🔴 Important findings first. Claude auto-resolves fixed threads.

  4. Human Review

    Apply 10-point checklist. Focus on intent, logic, integration, explain-it-back (points 1, 2, 7, 10).

Claude handles checklist points 3-6. Humans handle 1, 2, 7, 10. No redundant work.

🚨 Watch Out For

🚩
Red Flag
Silencing noisy findings without investigating. Repeated flags often mean a systemic problem — fix the root cause, don't silence the reviewer.
🤖
AI Strength
Cross-file bugs that humans miss: a renamed function in one PR causes a ReferenceError in code that still references the old name.
⚠️
Still Can't Catch
Intent mismatches, institutional context, architectural decisions, cross-service deps, undocumented business rules.

❓ Quick Quiz

Test Your Understanding

What are the 4 specialized agents in the review fleet?

Correctness (logic/edge cases), Security (vulns/auth), Performance (N+1/leaks), Style & Consistency (conventions/naming).

What's the difference between @claude review and @claude review once?

@claude review starts a review AND subscribes to future pushes. @claude review once is a single review with no subscription — best for one-off checks.

What goes in REVIEW.md vs. CLAUDE.md?

REVIEW.md: what to flag/skip during review (review-only rules). CLAUDE.md: how the system is shaped (architectural context for all tasks).

Which checklist points does Claude handle, and which need a human?

Claude handles points 3-6 (edge cases, security patterns, performance, duplication). Humans handle 1, 2, 7, 10 (intent, logic correctness, integration, explain-it-back).

Why shouldn't you silence noisy review findings without investigating?

Repeated findings often indicate a systemic problem (missing auth middleware, N+1 query patterns). Fix the root cause instead of silencing the signal.

✅ Key Takeaways

If You Remember Nothing Else

  • Claude Code Review uses a fleet of specialized agents (correctness, security, performance, style) with a verification layer that keeps false positives under 1%.
  • Three severity markers: 🔴 Important (fix it), 🟡 Nit (nice to fix), 🟣 Pre-existing (not your fault).
  • Write a REVIEW.md with "Always check," "Style," and "Skip" sections. 15 minutes of setup pays for itself on the first review.
  • CLAUDE.md = system architecture. REVIEW.md = review rules. Both matter.
  • Optimal workflow: Claude handles first pass (points 3-6), humans handle second pass (points 1, 2, 7, 10).
  • Claude's superpower: cross-file bugs — tracing renamed functions, broken references, stale callers across the codebase.
  • Investigate before you silence. Repeated findings = systemic problem, not noisy reviewer.
