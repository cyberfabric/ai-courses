🎯 OWASP Top 5 in AI Code

  1. 💉 Injection Flaws (33%)

    SQL, command, code injection. AI knows query shape but not threat model. String interpolation instead of parameterized queries.

  2. 🔗 SSRF

    Highest single-count vulnerability (32 instances). AI fetches user-controlled URLs without allowlists or IP blocking.

  3. 🔐 Broken Auth & Authz

    The most dangerous: AI writes correct middleware, then generates endpoints without attaching it. The vulnerability is in the integration gap.

  4. 🌐 XSS (86% failure rate)

    AI renders user input as HTML without escaping. Uses dangerouslySetInnerHTML without understanding why "dangerously" is in the name.

  5. 🔑 Hardcoded Secrets

    API keys, JWT secrets, default passwords in source code. AI optimizes for "code that works" over secure deployment patterns.

❓ The 4-Question Threat Model

📦
1. What Data?
User input? Credentials? PII? Financial data?
🎭
2. Who Are the Threat Actors?
Unauthenticated users? Limited-access users? Internal services?
💥
3. What's the Blast Radius?
One user? All users? Entire database? Financial loss?
🚧
4. Trust Boundaries?
Where does trusted data end and untrusted begin?

5 minutes. Write answers before AI generates the code. Check the output against them.

🔑 Key Terms

Security Sandwich
Human threat model → AI implementation → human security review. The human bookends are non-negotiable.
Integration Gap
AI writes correct auth middleware but doesn't attach it to new endpoints. The vulnerability is between components, not in them.
SAST
Static Application Security Testing. Scans source code for vulnerability patterns without running it.
DAST
Dynamic Application Security Testing. Scans running applications (e.g., OWASP ZAP). Catches runtime issues SAST misses.
Trust Boundary
Where trusted data ends and untrusted data begins. AI doesn't reason about these — you must.
High-Speed Intern Standard
For auth, payments, secrets, and untrusted input: threat model + SAST + line-by-line review + explicit middleware verification.

❓ Quick Quiz

Test Your Understanding

Why is AI code "syntactically clean but logically flawed" from a security perspective?

Traditional scanners catch sloppy code — missing brackets, obvious injection. AI code passes those checks but fails on subtler issues: missing trust boundaries, incomplete authorization, and insecure defaults that look intentional. The code looks secure but isn't.

What is the "integration gap" in AI-generated auth code?

AI writes correct auth middleware in one file, then generates a new endpoint without attaching it. Both look fine when reviewed separately. The vulnerability lives in the gap between them — the middleware exists but isn't applied.

Why do you need at least three SAST tools?

In AppSec Santa’s 2026 study (1,173 raw findings deduplicated to 175 confirmed positives across 5 SAST tools), the majority of confirmed AI vulnerabilities were caught by only one of the five tools — meaning any single tool misses most issues the others would catch. Three complementary tools (e.g., CodeQL + Semgrep + a language-specific scanner) materially improves coverage.

What are the three layers of the Security Sandwich?

1) Human threat model (identify what could go wrong before AI writes code), 2) AI implementation (generate with security constraints), 3) Human security review (verify against threat model + run SAST tools).

What are the four questions in the lightweight threat model?

1) What data does this code touch? 2) Who are the threat actors? 3) What's the blast radius? 4) What are the trust boundaries? Write answers before generation, verify after.

📌 Key Takeaways

If You Remember Nothing Else

  • 25-45% of AI code has confirmed vulnerabilities. It looks secure but isn't.
  • Top AI security failures: injection (33%), SSRF, broken auth, XSS (86% failure), hardcoded secrets.
  • The Security Sandwich: human threat model → AI implementation → human security review.
  • Use the 4-question threat model (5 min) before AI writes security-relevant code.
  • Run at least 3 SAST tools — the majority of confirmed AI vulnerabilities are caught by only one of five tools (AppSec Santa 2026).
  • Watch for the integration gap: auth middleware that exists but isn't wired up to endpoints.
  • Reject deprecated crypto immediately: no MD5, SHA-1, ECB, or hardcoded keys.
  • Auth, payments, secrets, untrusted input = high-speed intern standard. Every time. No exceptions.
