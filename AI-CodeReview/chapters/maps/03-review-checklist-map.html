🔟 The 10-Point Checklist

1️⃣
Intent Clarity
Does the change match what was requested? Can you state the problem it solves in one sentence?
2️⃣
Logic Correctness
Trace through with a concrete example, not just a code read. Does the algorithm do what it claims?
3️⃣
Edge Cases
Null, empty, boundary values, concurrent access. Did the AI handle them or pretend they don't exist?
4️⃣
Security
Auth checks? Input validated? Secrets hardcoded? XSS? SQL injection? AI gets this wrong 2.74× more.
5️⃣
Performance
Unnecessary loops? N+1 queries? Memory leaks? AI produces 8× more excessive I/O operations.
6️⃣
Duplication
Does this duplicate existing code? AI doesn't know your utils folder exists.
7️⃣
Integration
Works with the rest of the system? Respects existing error handling, logging, and config patterns?
8️⃣
Tests
Testing behavior or just achieving coverage? Tests should verify what code does, not how.
9️⃣
Maintainability
Will someone understand this in 6 months? Clear names, reasonable complexity, no magic numbers.
🔟
Explain It Back
Can you explain every function's purpose without re-reading the AI chat? If not, the PR isn't ready.

🔍 The "Explain It Back" Test

  1. State the Intent

    What problem does this solve, in your own words?

  2. Name the Edge Cases

    What does it handle? What doesn't it handle?

  3. Justify the Approach

    Why this implementation over alternatives?

If you can't fill in all three — you don't understand what you're shipping.

🚨 Watch Out For

Pattern What It Looks Like Why It's Dangerous
🚩 Tests mirror implementation Test spies on Array.reduce() instead of checking the result Breaks on any refactor. Confirms code does what code does — circular.
🚩 Tautological tests Test calls the same DB query as the function and compares results Always passes. Tests nothing. Gives false confidence.
🚩 Missing auth guards AI builds the feature but forgets authenticate() middleware Endpoint exposes sensitive data to anyone who guesses the URL.
🚩 Unnecessary caching Redis layer added to a 45ms endpoint nobody complained about Adds invalidation complexity and stale-data risk for no gain.

❓ Quick Quiz

Test Your Understanding

What are the 4 parts of the PR Contract Framework?

Intent (what & why), Proof (tests/screenshots), Risk + AI disclosure (risk tier + what's AI-generated), Focus (where the reviewer should spend time).

What's the difference between testing behavior and testing implementation?

Behavior tests check the output: "does calculateTotal return 20?" Implementation tests check how it works: "does it call Array.reduce()?" Implementation tests break on refactors even when behavior is unchanged.

Why is "explain it back" the most important checklist point?

It's a comprehension check, not a code check. If you can't explain intent, edge cases, and implementation choice — you don't understand the code, and approving it builds comprehension debt.

What's the pilot checklist analogy about?

Pilots with 10,000 hours still use pre-flight checklists. Not because they're inexperienced — because the human brain cuts corners under pressure. A review checklist prevents expertise from skipping steps.

Why should you break AI output into smaller commits?

You can't effectively review 800 lines at once. PRs under 400 lines see 30-50% faster first review and 40-60% shorter cycle time. Each commit gets the full 10-point check.

✅ Key Takeaways

If You Remember Nothing Else

  • Use a checklist every time — instinct fails under pressure, fatigue, and routine.
  • Every AI PR needs the PR Contract: intent, proof, risk + AI disclosure, focus areas.
  • The 10-point checklist: intent, logic, edge cases, security, performance, duplication, integration, tests, maintainability, explain it back.
  • AI tests often test implementation, not behavior. If refactoring breaks the test but output is identical — the test is wrong.
  • Break AI output into small commits — review each one separately. Under 400 lines each.
  • The "explain it back" test: state the intent, name the edge cases, justify the approach. Can't do all three? Don't approve.
  • Security-critical AI code (auth, payments, secrets) = high-speed intern rule. Every assumption questioned.
