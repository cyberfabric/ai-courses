Chapter 14

Subagents — Specialized Workers in .claude/agents/

Memory Map
💡

The Big Idea

A subagent is a specialist that runs in its own context window. Hand it a verbose, parallelizable, or differently-permissioned job; only the summary returns. Skills are what, hooks are when, subagents are where.

🛠️ The Three Built-in Subagents

🔍
Explore
Read-only codebase search. Haiku, fast, cheap. Three thoroughness levels: quick / medium / very thorough.
🗺️
Plan
Read-only research used during plan mode. Inherits parent model. Gathers context without infinite nesting.
⚙️
General-purpose
Full tools, full power. Inherits parent model. Use for multi-step work that needs explore + modify + iterate.

📝 Frontmatter Cheat Sheet

name
Required. Lowercase, hyphenated. Used in @-mentions and --agent flag.
description
Required. When Claude should delegate. Add "use proactively" for aggressive auto-delegation.
tools / disallowedTools
Allowlist / denylist. Deny is applied first, then allow. Omit both to inherit everything.
model
sonnet, opus, haiku, full ID, or inherit (default).
permissionMode
One of default, acceptEdits, auto, dontAsk, bypassPermissions, plan.
skills
Skills preloaded into the subagent's system prompt at startup. Subagents don't inherit parent skills.
mcpServers
MCP servers scoped to this subagent. Keeps their tool descriptions out of the parent's context.
hooks
Lifecycle hooks scoped to this subagent only. Stop auto-converts to SubagentStop.
memory
user | project | local. Persistent MEMORY.md across sessions.
isolation: worktree
Spawns the subagent in a temporary git worktree. Auto-cleaned if no edits made.
background: true
Always run concurrently. Permissions pre-approved at launch; failures auto-deny.

⚠️ Common Pitfalls

🤹
Over-delegating
Cold starts cost latency. Use subagents for isolation, parallelism, or context savings — not for trivial edits.
🗑️
Lost context
Only the summary returns. If you need a list of files, tell the subagent to include it explicitly.
💸
Model mismatch
Don't put Opus on a job Haiku can do. Match model to job — that's why Explore runs on Haiku.
🪜
Nested delegation
Subagents can't spawn subagents. Chain from the main thread, use Skills, or graduate to Agent Teams.

🧪 Quick Quiz

What's the one-sentence difference between skills, hooks, and subagents?

Skills run procedures, hooks run on events, subagents run in their own context window.

Why does the Explore subagent run on Haiku?

Codebase scanning is high-volume, low-complexity work. Haiku is fast and cheap; using Sonnet or Opus there is paying flagship rates for grep.

Where do you put a subagent that should be shared with your team?

.claude/agents/<name>.md — project scope, checked into git so the team gets it.

What does setting isolation: worktree do?

Spawns the subagent in a temporary git worktree so its file edits don't touch your checkout. Auto-cleaned if it makes no changes.

What's the difference between a fork and a named subagent?

A fork inherits your full conversation history (same prompt, tools, model, messages). A named subagent starts cold with its own definition.

Can a subagent spawn other subagents?

No. If you need nested delegation, chain subagents from the main thread, use Skills, or look at Agent Teams (CLAUDE_CODE_EXPERIMENTAL_AGENT_TEAMS=1).

🎯 Key Takeaways

  • Subagents are specialists running in their own context window — verbose work happens over there, only summaries come back.
  • Three built-ins: Explore (Haiku, read-only search), Plan (read-only planning), General-purpose (full tools).
  • Custom subagents are Markdown files in .claude/agents/ (project) or ~/.claude/agents/ (user). Two required fields: name, description.
  • Use /agents to create, browse, and manage interactively. Restart the session to load hand-edited files.
  • Four invocation patterns: natural language, @-mention, /agents UI, --agent <name> for whole-session role.
  • Parallel execution is the killer feature: code-review fan-outs (style + security + tests) cut wall time and keep main context clean.
  • Permission inheritance flows down — you can relax security from the parent but never tighten it via subagent frontmatter.
  • Persistent memory turns one-shot subagents into ones that learn your codebase across conversations.
  • Forks (experimental) are subagents that inherit full conversation history — perfect for cheap "side quest" delegations.
  • Pitfalls: over-delegating, lost summaries, model mismatch, forgetting subagents can't spawn subagents.
