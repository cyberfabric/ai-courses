Building Your Claude Playbook

Memory Map
💡

The Big Idea

Knowledge without a system is trivia. A playbook turns everything you've learned into daily routines, reusable skills, and tuned configuration that compounds over time — making you measurably faster every month.

🏛️ The Three Pillars of a Playbook

🔁
Routines
What you do with Claude every day without thinking — morning context load, dev loop, end-of-day cleanup.
⚡
Skills
5-10 custom slash commands that cover 80% of your work — consistent quality, zero effort.
⚙️
Configuration
Your CLAUDE.md, auto memory, settings, and hooks — all tuned to your coding style.

🛠️ The Essential Skill Set

📝
/daily-standup
Git activity → standup summary. Use every morning.
🔍
/review-pr
Review your branch before opening a PR. Bugs, security, conventions.
🧪
/generate-tests
Unit tests following your project's patterns. Use after implementing features.
🐛
/debug
Structured investigation: locate → diagnose → fix → test.
📖
/update-docs
Check and update project documentation. Use before releases.

💰 Model Selection Strategy

🧠
Opus (15%)
Architecture decisions, complex debugging, multi-file refactoring, subtle code review.
⚡
Sonnet (80%)
Feature implementation, test generation, documentation, standard code review.
🏎️
Haiku (5%)
Quick lookups: "Where is this function?" "What does this flag do?" Simple formatting.

📈 The Compound Effect

  1. Month 1

    Basic usage — Claude for coding and simple questions. Initial productivity gain.

  2. Month 3

    CLAUDE.md and skills are tuned. Claude's output matches your standards consistently.

  3. Month 6

    Auto memory has learned your patterns. Claude anticipates your needs. Bugs caught pre-push.

  4. Month 12

    Fully mature playbook. Ship faster, fewer bugs, less cognitive load. The system runs itself.

⚠️ Pitfalls to Avoid

🫣
Over-reliance
If you can't explain what Claude generated, you don't understand it. Your name is on the commit.
😵
Context Fatigue
8-hour sessions without /compact degrade quality. Compress often, start fresh for new problems.
😴
Prompt Laziness
"Fix it" gets lazy results. 30 seconds of good prompting saves 10 minutes of back-and-forth.
✅
Not Verifying
Always read the diff, run the tests, check edge cases. Trust, but verify.

📅 The 30-Day Challenge

  1. Week 1: Foundation

    Install/update Claude Code, create personal CLAUDE.md, build /daily-standup and /review-pr skills, set up auto-format hook.

  2. Week 2: Workflows

    Build /generate-tests and /debug skills. Use on real PRs and bugs. Generate docs for one module.

  3. Week 3: Optimization

    Practice context management. Try model switching. Run a skill audit. Clean up auto memory.

  4. Week 4: Team & Polish

    Create /onboard skill. Set up CI review. Share techniques with team. Write your personal playbook.

🧪 Quick Quiz

What are the three components of a Claude playbook?

Routines (daily habits), Skills (custom slash commands), and Configuration (CLAUDE.md, memory, settings, hooks).

When should you start a fresh session vs continue?

Continue when working on the same feature, iterating on designs, or doing multi-step refactoring. Start fresh when switching tasks, when Claude seems confused, or when context is bloated.

Which model handles 80% of typical developer work?

Sonnet — for feature implementation, test generation, documentation, and standard code review. Reserve Opus for genuinely hard problems.

What's the "Skill Audit" power tip?

After a week of using Claude, ask it to review your recent conversations and identify the 3 prompts you type most often. Build custom skills to automate those patterns.

What's the 80/20 rule of Claude?

80% of your productivity gain comes from 20% of Claude's features. Find your personal 20% and build your playbook around those — ignore the rest until the core is automatic.

🎯 Key Takeaways

  • A playbook is a living system of routines + skills + configuration, not a one-time document.
  • Audit your workflow first — find the 2-3 hours/day Claude can accelerate, then build around those.
  • Build 5-10 custom skills covering your most repeated tasks — consistent quality, zero effort.
  • Manage context intentionally: /compact after tasks, fresh sessions for fresh problems.
  • Use Sonnet for 80% of work, Opus for hard problems, Haiku for quick lookups.
  • Small optimizations compound: 5 skills × 5 min/day = 100 hours saved per year.
  • Avoid over-reliance, context fatigue, prompt laziness, and skipping verification.
  • The 30-day challenge builds your playbook incrementally — foundation → workflows → optimization → team.
