The Big Idea

The IDE extension is the same Claude Code as the CLI — same settings, same CLAUDE.md, same auth. The difference: visual diffs before changes, @-mentions for precise context, and automatic diagnostics sharing. Use the panel for visual work, the CLI for automation.

⚡ Power Features

📌
@-Mentions
Reference specific files and line ranges: @src/auth.ts#15-45. Type @ or press Alt+K.
📝
Inline Diffs
Changes appear in native diff viewer with accept/reject buttons. Review before any file is touched.
🔴
Auto Diagnostics
Errors and warnings from the Problems panel are shared automatically. Just say "fix this error."
⚡
Auto-Accept
Edits applied in real time as Claude generates them. Fast but risky — use on code you understand well.
📑
Multi-Tab Chats
Open multiple conversations in separate tabs or windows. Each maintains its own history and context.
👁️
Editor Context
Claude sees your open files, cursor position, selected text, and workspace structure automatically.

⌨️ Essential Keybindings

Ctrl+Esc / Cmd+Esc
Toggle focus between editor and Claude panel. Your most-used shortcut.
Alt+K / Option+K
Insert an @-mention reference (including any selected line range).
Ctrl+N / Cmd+N
New conversation (when panel is focused).
Shift+Tab
Cycle permission modes (default → acceptEdits → plan → auto).
Ctrl+K, Ctrl+S
Open VS Code keyboard shortcuts editor. Search "Claude" to customize.
~/.claude/keybindings.json
Claude Code's own keybinding config. Supports chord sequences (multi-key shortcuts).

🏗️ Large Project Survival Guide

🎯
Focus with @-mentions
Don't let Claude search everything. Point it at the exact files that matter.
📄
Directory CLAUDE.md
Put a CLAUDE.md in src/api/ to describe that module without loading the whole project.
🧹
Close irrelevant tabs
Open tabs are context signals. Lots of unrelated tabs dilute Claude's focus.
🔀
One task per chat
Keep conversations focused. Compact or restart between unrelated tasks.

🎯 Quick Quiz

What's the main advantage of the IDE panel over the CLI for code editing?

You see inline diffs before changes are applied. In the CLI, edits are applied directly and you review after the fact.

Your editor shows a red squiggly on line 42. What's the fastest way to fix it with Claude?

Press Ctrl+Esc to open the Claude panel and just say "fix this error." The extension automatically sends the diagnostic, file, and surrounding code.

How do you reference specific lines of a file in the VS Code panel?

Use @-mentions: type @src/file.ts#15-45 or select code and press Alt+K to insert the reference with line range.

When should you use the CLI instead of the IDE panel?

For automation (--print mode), CI/CD pipelines, complex git workflows, SSH/remote servers, and when you need full CLI flag control.

📌 Key Takeaways

  • The IDE extension and CLI are the same Claude Code — same settings, CLAUDE.md, auth, and capabilities.
  • IDE panel advantage: inline diffs (review before accept), @-mentions (precise context), and automatic diagnostics sharing.
  • CLI advantage: --print for automation, full flag control, works over SSH.
  • VS Code has a custom graphical panel; JetBrains uses integrated terminal + native diff viewer. Both launch with Ctrl+Esc.
  • JetBrains users get two paths to Claude Agent: the JetBrains AI Pro/Ultimate subscription (since March 2026) or Bring Your Own Key (since December 2025) with an Anthropic API account — no JetBrains AI subscription needed for BYOK.
  • For large projects: focus with @-mentions, use directory CLAUDE.md files, close irrelevant tabs, one task per chat.
  • Many power users switch between CLI and IDE panel depending on the task. There's no single "best" — use both.
