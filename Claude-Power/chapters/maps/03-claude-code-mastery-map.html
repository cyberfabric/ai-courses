The Big Idea

Claude Code is a race car — most people drive it in first gear. The --print flag turns it into an automation engine, /compact keeps long sessions sharp, and permission modes let you control the throttle.

🚩 The Big Five Flags

🖨️
--print (-p)
Headless mode. Process prompt, output result, exit. The key to automation, scripts, and CI/CD pipelines.
🧠
--model
Choose your engine: opus for hard problems, sonnet for daily work, haiku for quick lookups.
📊
--output-format
Machine-readable output. Use json or stream-json for parsing in scripts and pipelines.
🔒
--permission-mode
Control the throttle: default (ask everything), acceptEdits, plan (read-only), or auto.
🔧
--allowedTools
Fine-grained tool access. Restrict to Read,Grep,Glob for safe exploration, or exclude Bash for safer sessions.

🔐 Permission Modes

Mode Edits Commands Best For
default ⚠️ Asks ⚠️ Asks Learning, sensitive repos
acceptEdits ✅ Auto ⚠️ Asks Everyday coding
plan ❌ No ❌ No Architecture review, pre-flight
auto ✅ Auto 🤖 Decides Experienced users, trusted projects
dontAsk ❌ Denies ❌ Denies Locked-down CI & scripts
bypassPermissions ⚡ All ⚡ All Containers/VMs only

Shift+Tab cycles through modes mid-session — no restart needed. Auto needs Max/Team/Enterprise/API + Sonnet 4.6 / Opus 4.6 / Opus 4.7 (Max plan: 4.7 only).

⚡ Essential Slash Commands

/compact
🏆 Secret weapon. Compresses conversation history when Claude starts forgetting earlier context.
/cost
Shows token usage and spend for the current session. Run periodically for budget awareness.
/model
Switch models mid-session. Start with Sonnet, escalate to Opus for hard problems.
/review
Quick code review of recent changes. Great before committing.
/commit
Git commit with AI-generated message. Use after Claude makes changes you're happy with.
/resume
Pick up a previous session where you left off. Your conversation history is preserved.
/init
Auto-generate a CLAUDE.md file for your project. Best first step in a new repo.
/clear
Fresh start without exiting. Use when switching to an unrelated task.

🎯 Quick Quiz

What flag turns Claude Code into an automation engine for scripts and CI/CD?

--print (or -p). Claude processes the prompt and exits with output — no interactive session.

Your session is getting long and Claude is forgetting earlier context. What do you do?

Run /compact to compress conversation history, keeping important parts while freeing context space.

You want Claude to analyze a codebase but not change any files. Which permission mode?

plan mode. Claude reads and analyzes but cannot make edits or run commands.

How do you run a shell command without leaving a Claude Code session?

Prefix it with ! — e.g., ! git status. The output goes into Claude's context.

What's the difference between /clear and /compact?

/clear wipes everything — fresh start. /compact compresses but keeps important context. Use /compact to continue; /clear to switch tasks.

📌 Key Takeaways

  • --print is the most important flag — it turns Claude Code into a scriptable automation engine for CI/CD and shell scripts.
  • /compact is your secret weapon for long sessions — compresses history without losing important context.
  • Permission modes control the safety throttle: default → acceptEdits → plan → auto (plus dontAsk and bypassPermissions for CI/sandbox). Cycle with Shift+Tab.
  • Plan mode first, execute second — especially for big refactors and unfamiliar codebases.
  • Use Sonnet as default (90% of tasks), Opus 4.7 (the new flagship, same price as 4.6) for hard problems, Haiku for quick lookups. Switch mid-session with /model.
  • The .claude/ directory holds your settings, commands, and instructions — project-level and user-level.
