Chapter 2

Claude.ai Power Features

Projects, Artifacts, Styles, and the features most users never touch

In the last chapter, we mapped the full Claude ecosystem. Now it's time to zoom in on the product you probably use most: claude.ai.

Most developers treat Claude.ai like a search bar with better grammar. They open a new chat, ask a question, get an answer, close the tab. Next time? Start from scratch. No context, no memory, no system.

That's like buying a smartphone and only using it to make phone calls.

Claude.ai has a stack of power features hiding in plain sight. This chapter shows you every one of them — and how to use them like a developer who's been doing this for years.

Projects: Your Persistent Workspace

Projects are the single most impactful Claude.ai feature that most people ignore. A Project is a workspace where every conversation shares the same context — your instructions, your files, your knowledge base.

What a Project gives you

  • Project Instructions — rules Claude follows for every conversation in that project (think of it like a CLAUDE.md for the web interface)
  • Knowledge Base — uploaded files that Claude can reference in every chat (PDFs, code files, docs, CSVs, up to 30MB each)
  • Organized conversations — all chats within a project are grouped together
  • RAG scaling — on paid plans, projects use Retrieval Augmented Generation to handle knowledge bases up to 10x larger than the context window

Without Projects:

"Here's my API spec [paste 200 lines]. And here's our coding standards [paste 50 lines]. And remember, we use TypeScript with strict mode and..."

You repeat this every. single. conversation.

With Projects:

Upload your API spec and coding standards once. Write project instructions once. Every new conversation in that project already knows everything.

Just ask your question.

How to create a Project

  1. Open Claude.ai and find Projects in the left sidebar
  2. Click Create Project (or the + button)
  3. Give it a descriptive name — "Android App Codebase", "Q2 Migration Planning", "API Design Reviews"
  4. Add Project Instructions — the rules Claude should follow
  5. Upload files to the Knowledge Base — specs, docs, code samples, anything Claude should know

Writing great Project Instructions

The difference between vague and powerful instructions is enormous. Here's a real comparison:

Vague instructions
You are a helpful coding assistant. Help me with my project. Use best practices.
This tells Claude nothing specific. It would behave almost identically without these instructions.
Powerful instructions
You're helping me build a Kotlin Android app targeting API 26+. Tech stack: Jetpack Compose for UI, Hilt for DI, Coroutines + Flow for async, Room for local DB. Code style: - Kotlin with explicit return types on public functions - Compose: stateless components with hoisted state - Error handling: sealed Result types, never raw try-catch in ViewModels - Tests: JUnit 5 + Turbine for Flow testing When reviewing code, check for: memory leaks in coroutine scopes, missing null checks from Java interop, Compose recomposition issues. Respond concisely. Show code, not prose. Skip explanations I didn't ask for.
Claude now knows your stack, your conventions, what to watch for, and how to respond. Every chat in this project starts with this context loaded.
Treat Project Instructions like onboarding docs for a new teammate. What would a skilled developer need to know on day one to write code that fits your project? That's what goes in here. Include: language/framework, naming conventions, testing approach, common pitfalls, and your preferred response format. Update it monthly as your project evolves.

Knowledge Base best practices

The Knowledge Base is your project's reference library. Claude reads these files to answer your questions with project-specific accuracy.

Upload this Why
API specs (OpenAPI, GraphQL schema) Claude generates code that matches your actual endpoints
Architecture decision records Claude understands why things are the way they are
Style guides / coding standards Generated code follows your conventions automatically
Key source files Claude sees your patterns and extends them consistently
Bug report templates / past incidents Claude knows what goes wrong and what to watch for
Don't upload your entire codebase into a Project Knowledge Base. That's what Claude Code is for. Projects work best with reference documents — specs, standards, key examples. Think curated library, not file dump.

Artifacts: Interactive Outputs That Live Outside the Chat

Artifacts are one of Claude's most impressive features — and probably the most underused by developers. When Claude generates something substantial (code, a document, a diagram, an interactive app), it can render it as an Artifact — a live, interactive panel alongside the chat.

What Artifacts can create

  • React components — fully functional, running in the browser
  • HTML/CSS pages — complete web pages with styling
  • Interactive charts and data visualizations — including, in Claude Cowork, Live Artifacts: dashboards and trackers that stay connected to your apps and files and refresh with current data each time you open them
  • SVG graphics — icons, diagrams, illustrations
  • Mermaid diagrams — flowcharts, sequence diagrams, architecture diagrams
  • Interactive calculators and tools — forms, converters, mini-apps
  • Document drafts — markdown, formatted text
Build me an interactive color palette generator. I want to enter a primary color and see complementary, analogous, and triadic colors with their hex codes. Make it look professional.
I'll create a React component for you. [Generates a full interactive app as an Artifact with color pickers, live preview swatches, copy-to-clipboard for hex codes, and a clean modern UI — all running live in the browser panel.]

That's not a mockup. It's a working app, right there in your Claude conversation. You can interact with it, iterate on it, and copy the code to use in your own project.

The iteration workflow

The real power of Artifacts is iteration. You don't get it perfect on the first try — you refine it through conversation:

Describe what you want → Claude creates Artifact → You give feedback → Claude updates it → Repeat until perfect

Each version is saved, so you can go back to a previous iteration if the latest change doesn't work out.

Artifacts now include persistent storage across sessions and direct API calls on Pro, Max, Team, and Enterprise plans. Your Artifact mini-apps can save state between visits and even connect to external services via MCP integrations. The line between "quick prototype" and "real tool" is getting very blurry.

Custom Instructions: Teach Claude Once

Custom Instructions (called Profile Preferences in your Claude settings) apply to every conversation you have — not just within a project. Think of them as your global defaults.

What to put in Custom Instructions

  • Your role and expertise level ("I'm a senior Android developer")
  • Your preferred response format ("concise, show code first, explain after")
  • Languages you work in ("Kotlin, TypeScript, Python")
  • Things to always do ("include error handling in code examples")
  • Things to never do ("don't add comments explaining obvious code")

Every conversation:

"Remember, I'm using Kotlin and I prefer concise answers with code examples. Don't explain basic concepts I already know..."

With Custom Instructions set:

"How should I handle this API error?"

Claude already knows your stack, your level, and your style. Every time.

Custom Instructions vs Project Instructions

Feature Custom Instructions Project Instructions
Scope Every conversation, globally Only within a specific project
Best for Your identity, preferences, defaults Task-specific context, tech stack, conventions
Can include files? No Yes (Knowledge Base)
How to set Settings → Profile Inside each Project

Use both. Custom Instructions for "who you are." Project Instructions for "what you're working on."

Styles: Control Claude's Tone and Format

Styles let you change how Claude communicates without rewriting your instructions. They control tone, verbosity, formatting, and overall feel.

Claude comes with built-in presets you can switch from the style selector near the chat input area:

  • Normal — the default balanced response style
  • Concise — stripped-down, just-the-essentials (great for quick coding questions)
  • Explanatory — more detail, step-by-step breakdowns
  • Formal — professional tone for documentation or client-facing content

You can also create custom styles that capture your exact preferences.

Quick style switching: Click the style selector icon next to the chat input (or look for the style name displayed near the model selector). Switch between presets without interrupting your conversation. The style applies immediately to Claude's next response.

File Uploads: What Works and What Doesn't

You can upload files directly into a conversation or into a Project's Knowledge Base. Here's what you need to know:

Supported formats

  • Documents: PDF, DOCX, TXT, HTML, RTF, ODT, EPUB
  • Data: CSV, JSON, XML
  • Code: pretty much any text-based source file
  • Images: PNG, JPG, GIF, WebP (Claude has vision capabilities)

Limits to know

  • Individual files: up to 30MB
  • Knowledge Base: unlimited files, but total content must fit within the context window (RAG helps on paid plans)
  • Images: Claude can analyze screenshots, diagrams, charts, UI mockups, handwritten notes
Upload screenshots of errors. Instead of describing a bug, just screenshot the error dialog, the stack trace in your IDE, or the broken UI. Claude's vision can read and analyze images, often catching details you might miss when describing the problem verbally.

Search: Finding Past Conversations

Claude.ai has built-in search that lets you find past conversations by keyword. If you've ever had that moment of "I asked Claude about this last week, where did that go?" — search is your answer.

Claude also has web search capabilities. When you ask about recent events, current documentation, or up-to-date information, Claude can search the web and incorporate the results into its response. You'll see citations with links when web search is used.

Starred Conversations and Management

As your conversation list grows, organization matters:

  • Star conversations you want to find again quickly — they appear in a dedicated "Starred" section
  • Rename conversations for clarity — "API design discussion" beats "New Chat (3)"
  • Use Projects to group related conversations rather than relying on search alone
  • Delete old chats you won't need — less clutter means faster navigation

Mobile App: Claude in Your Pocket

The Claude mobile app (iOS and Android) gives you the same core experience with a few differences:

  • Voice input — speak your questions instead of typing (great for brainstorming while walking)
  • Camera integration — snap a photo and send it directly to Claude for analysis
  • Projects access — your Projects and their knowledge bases sync across devices
  • Artifacts — work but are tighter on a small screen; complex interactive apps are better on desktop
Mobile + camera is a debugging superpower. See a weird error on a test device? Snap a photo with the Claude app, ask "What's causing this error and how do I fix it?" Claude reads the screenshot, identifies the error, and gives you a fix — all while you're standing at someone's desk. No need to reproduce it on your machine first.

Where Claude.ai Genuinely Excels

You've probably used ChatGPT and maybe Gemini too. Here's where Claude.ai has a genuine edge for developers:

Strength Why it matters for devs
Long context (1M tokens) Upload entire codebases, full specs, long documents — Claude keeps track
Projects + Knowledge Base Persistent context across conversations — no other chat AI does this as well
Artifacts Interactive React components and visualizations rendered live — not just code blocks
Instruction following Claude is unusually good at following complex, multi-part instructions precisely
Code quality Sonnet 4.6 hits 79.6% on SWE-bench Verified at $3/MTok — the code is genuinely good (Opus 4.7 leads at 87.6% when you need more)
Honest about limitations Claude says "I don't know" instead of confidently making things up

This isn't about brand loyalty. It's about using the right tool. Claude.ai's combination of Projects, Artifacts, and long context make it the best conversational AI for sustained development work. Other tools might be better for web search (Gemini) or image generation (ChatGPT with DALL-E). Know the strengths.

Putting It All Together: A Power User Workflow

Here's how an experienced developer uses Claude.ai on a typical day:

The Power User's Claude.ai Day

Morning: Open your "API Design" Project. Start a new chat. "Based on the spec in the knowledge base, design the endpoint for user preferences. Include request/response types."

Midday: Create an Artifact — an interactive API documentation page for the endpoint you just designed. Iterate until it looks right. Share the link with your team.

Afternoon: Switch to your "Code Review" Project. Upload a diff. "Review this for security issues, focusing on the auth middleware changes."

Quick break: Open the mobile app. Snap a photo of the whiteboard from the architecture meeting. "Summarize the decisions on this whiteboard and create a Mermaid diagram of the data flow."

Create your first power Project right now.

  1. Go to claude.ai and create a new Project
  2. Name it after your current main project (e.g., "MyApp Android")
  3. Write Project Instructions: include your tech stack, coding conventions, and preferred response format (use the "powerful instructions" example above as a template)
  4. Upload 2-3 key files: your main README, an API spec, or a core source file
  5. Start a conversation and ask a question you'd normally ask in a generic chat
  6. Notice the difference — Claude already knows your context

Once you experience the difference between a project-aware conversation and a blank-slate conversation, you'll never go back.
