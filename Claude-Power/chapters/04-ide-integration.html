Chapter 4

IDE Integration Deep Dive

VS Code and JetBrains extensions — setup, keybindings, and making Claude feel native

In the last chapter, you mastered Claude Code from the terminal. But let's be honest — most of your day is spent in your IDE, not a standalone terminal. Switching between your editor and a CLI tool breaks your flow.

That's why Claude Code lives inside VS Code and JetBrains too. Same capabilities, embedded right where you work. This chapter shows you how to set it up, learn the keybindings, and figure out when the IDE panel beats the CLI (and when it doesn't).

VS Code Extension: Setup

Installation

  1. Make sure Claude Code CLI is installed (npm install -g @anthropic-ai/claude-code)
  2. Open VS Code
  3. Open the Extensions view: Ctrl+Shift+X (or Cmd+Shift+X on Mac)
  4. Search for "Claude Code" by Anthropic
  5. Click Install
  6. Authenticate: the extension will prompt you to log in (same auth as the CLI)

Once installed, you'll see a Claude icon in the activity bar (left sidebar). Click it to open the Claude Code panel.

The VS Code extension and the CLI share the same authentication, settings, and CLAUDE.md files. Anything you configure for the CLI works in the extension too — they're the same tool with different interfaces.

First use

Click the Claude icon in the sidebar. The panel opens. Type a question. That's it. Claude can already see your open files, your workspace, and your project structure.

But opening the panel and typing questions is just the beginning. Let's get into the features that make this extension powerful.

The Claude Code Panel

The panel is your primary interaction surface. It's more than a chat window — it's a full Claude Code session running inside VS Code.

Panel positioning

By default, the panel opens in the sidebar. But you can drag it anywhere:

  • Sidebar — good for quick questions while coding (doesn't take much space)
  • Bottom panel — good for longer conversations (wider space for code blocks)
  • Separate tab — use "Open in New Tab" from the Command Palette for a full-width Claude experience
  • New window — use "Open in New Window" to put Claude on a second monitor

Multiple conversations

You can open multiple Claude conversations in separate tabs or windows. Each conversation maintains its own history and context. This is great when you're working on two different tasks and want separate contexts.

Essential VS Code keybindings:
  • Ctrl+Esc / Cmd+Esc — Toggle focus between editor and Claude panel
  • Ctrl+N / Cmd+N — Start a new conversation (when panel is focused)
  • Alt+K / Option+K — Insert an @-mention reference
  • Ctrl+Shift+P / Cmd+Shift+P — Command Palette, then type "Claude Code" for all commands

@-Mentions: Pointing Claude at Specific Code

This is the feature that makes the IDE extension genuinely better than the CLI for certain tasks. You can @-mention files with specific line ranges directly in your prompt.

@src/auth/token-service.ts#15-45 refactor this to use async/await instead of callbacks

Claude gets the exact file and lines as context. No need to say "look at lines 15-45 in..." — the extension handles it.

How to insert @-mentions

  • Type @ in the chat input — you'll get an autocomplete dropdown of your project files
  • Press Alt+K — inserts an @-mention reference at your cursor (including any selected line range)
  • Select code in the editor, then switch to the panel — the selection is automatically included as context

CLI approach:

"Read src/auth/token-service.ts lines 15-45 and refactor to use async/await"

You have to remember the file path and line numbers from memory.

IDE approach:

Select the code block in the editor. Press Alt+K. Type "refactor to async/await."

Claude sees exactly what you're looking at.

Inline Diffs: Review Before You Accept

When Claude proposes changes, they appear as inline diffs in VS Code's native diff viewer. You see exactly what's being added, removed, and changed — with accept/reject buttons for each change.

The review flow

Ask Claude to change code → Diff appears in editor → Review changes → Accept or reject

This is a significant advantage over the CLI, where changes are applied directly and you review after the fact. In VS Code, you see the diff before any file is modified.

Auto-accept mode

If you trust Claude and want to move fast, you can enable auto-accept. Claude's edits are applied in real time as they're generated — no diff review step. This is great for rapid iteration when you're working on something low-risk.

Toggle it with the permission mode selector (same Shift+Tab cycling from Chapter 3).

Auto-accept is fast but risky in unfamiliar code. Use it when you're iterating on code you understand well. Switch to default mode (with diff review) when working on critical paths, shared code, or code you didn't write.

Context from Your Editor

The extension doesn't just give you a chat panel — it feeds Claude information from your editor automatically.

What Claude receives from the extension

  • Open files — Claude knows which files are open in your editor tabs
  • Current file and cursor position — Claude knows where you're looking
  • Selected text — any code you've highlighted is sent as context
  • Diagnostics — errors, warnings, and lint issues from VS Code's Problems panel are shared automatically
  • Workspace root — Claude understands your project structure
[With red squiggly on line 42 in the editor] Fix this error
I can see the TypeScript error on line 42: "Property 'userId' does not exist on type 'Request'." This is because Express's Request type doesn't include your custom properties by default. Here's the fix — you need to extend the Request interface: [Proposes a diff adding a type declaration file with the custom Request interface]

Notice: you didn't paste the error message, didn't explain the context, didn't even say which file. The extension sent all of that automatically. Claude saw the diagnostic, the file, the line, and the surrounding code.

Let VS Code diagnostics do the talking. When you have red or yellow squiggles in your code, just ask "fix this error" or "fix the warnings in this file." Claude reads the diagnostics panel automatically and knows exactly what's wrong. This is faster than copying error messages from the terminal.

Terminal Integration vs Panel

You can run Claude Code two ways in VS Code:

Feature Panel (Extension) Terminal (CLI in VS Code)
@-mentions Yes No
Inline diffs Yes (native diff viewer) No (applies directly)
Editor diagnostics Yes (automatic) No
Multi-tab conversations Yes No (one per terminal)
All CLI flags Some via settings Yes (full control)
Shell command execution Yes Yes (native terminal)
Headless mode (-p) No Yes
Session resume Yes (history) Yes (/resume)

Rule of thumb: Use the panel for interactive coding work where you want visual diffs and editor context. Use the terminal for automation, headless tasks, and when you need full CLI flag control.

JetBrains Extension: Setup and Differences

The JetBrains plugin works across all JetBrains IDEs: IntelliJ IDEA, Android Studio, WebStorm, PyCharm, GoLand, PhpStorm, and more.

Installation

  1. Make sure Claude Code CLI is installed
  2. Open your JetBrains IDE
  3. Go to Settings → Plugins → Marketplace
  4. Search for "Claude Code"
  5. Install and restart

How JetBrains differs from VS Code

The JetBrains plugin takes a different approach. Instead of a custom graphical panel, it runs Claude Code in the IDE's integrated terminal and uses JetBrains' native diff viewer for change review.

Aspect VS Code JetBrains
Interface Custom graphical panel Integrated terminal + native diff
Diff review Extension's inline diff view JetBrains' native diff viewer
Code selection context @-mentions with line ranges Selected code auto-shared
IDE diagnostics Auto-shared from Problems panel Auto-shared from inspections
Quick launch Ctrl+Esc Ctrl+Esc
Feel More graphical, chat-app-like More terminal-native, developer-raw
If you're an Android developer using Android Studio, the JetBrains plugin gives you full Claude Code access inside your existing workflow. Select code in the editor, press Ctrl+Esc, and start chatting. The selected code is sent as context automatically. Since March 2026, Claude Agent is also available through JetBrains AI as one of the built-in agents — included in the JetBrains AI Pro / Ultimate subscription. As of December 2025 there’s also a Bring Your Own Key (BYOK) path: log in via Anthropic Console (API Billing) and you can use Claude Agent with no JetBrains AI subscription. Note: BYOK requires an Anthropic API account — Pro/Max/Team consumer plans aren’t permitted for this auth path per Anthropic’s Terms of Service.

Working with Multiple Files

Real development rarely involves a single file. Here's how to point Claude at multiple files effectively in the IDE.

In VS Code

  • @-mention multiple files: @src/api/routes.ts @src/api/middleware.ts add rate limiting to all routes
  • Open relevant files in tabs — Claude sees your open tabs as context signals
  • Use the file explorer to reference paths — right-click a file and look for Claude-related options

In JetBrains

  • Select code across files using JetBrains' multi-cursor or find/replace with context sharing
  • Reference files by path in your prompt — Claude still reads them from disk
  • Use the Project tool window to navigate, then describe what you need

Integration with VS Code Features

Problems panel

When your project has errors or warnings in the Problems panel, Claude can see them. Ask "fix all the TypeScript errors in this project" and Claude will work through them systematically.

Source control

Claude Code integrates with VS Code's git features. You can ask Claude to stage files, create commits, and even create PRs — all from the panel. The git operations happen through Claude Code's tool system, not through VS Code's SCM API, so the behavior is identical to the CLI.

Terminal

Claude can run commands in VS Code's integrated terminal. When Claude needs to run npm test or git status, it executes in the same terminal environment you'd use manually. Output appears in the conversation.

Tips for Large Projects

Large monorepos and projects with thousands of files need some extra care.

  • Use @-mentions to focus Claude — don't let Claude search the entire project when you know which files are relevant
  • Create directory-specific CLAUDE.md files — a CLAUDE.md in src/api/ can tell Claude about that module without loading everything
  • Keep conversations focused — one task per conversation, compact or restart between tasks
  • Close unrelated tabs — since Claude uses open tabs as context signals, lots of irrelevant open files can dilute focus
Extension conflicts. If you have other AI coding extensions (Copilot, Codeium, Continue, etc.), they can occasionally conflict with Claude Code. Common symptoms: duplicate suggestions, keybinding conflicts, or slow performance. If you notice issues, try disabling other AI extensions temporarily to isolate the problem.

Common Pitfalls and Fixes

Problem Cause Fix
Extension doesn't start CLI not installed or outdated Run npm install -g @anthropic-ai/claude-code
Authentication failed Token expired or missing Run claude login in terminal
Slow response times Too many extensions or large workspace Disable unused extensions, focus context with @-mentions
Keybinding conflicts Other extensions using same shortcuts Remap in Ctrl+K, Ctrl+S — search "Claude"
Diffs not appearing Permission mode or extension glitch Check permission mode, restart extension

The Big Comparison: CLI vs VS Code vs JetBrains

When Each Surface Shines

CLI: Automation, CI/CD, headless scripts, multi-step tasks with full flag control, complex git workflows, working over SSH.

VS Code panel: Interactive coding with visual diffs, @-mentioning specific code, fixing diagnostics, rapid iteration with auto-accept, working across multiple files visually.

JetBrains plugin: Same as VS Code but in IntelliJ/Android Studio/WebStorm/PyCharm, native diff viewer, strong for Android developers, integrated with JetBrains inspections.

There's no "best" option. Many power users switch between all three depending on the task. The key insight: they're all the same Claude Code underneath. Your CLAUDE.md, settings, and custom commands work identically everywhere.

Customizing Keybindings

If the default shortcuts don't fit your muscle memory, customize them:

In VS Code

  1. Open Keyboard Shortcuts: Ctrl+K, Ctrl+S
  2. Search for "Claude"
  3. Click the pencil icon next to any command to rebind it

In Claude Code (CLI and extensions)

Claude Code has its own keybinding system via ~/.claude/keybindings.json. You can configure shortcuts per context and create chord sequences (multi-key shortcuts).

Set up your IDE extension and complete a real task — no terminal allowed.

  1. Install the Claude Code extension for your IDE (VS Code or JetBrains)
  2. Open a real project you're working on
  3. Select a block of code that could use improvement (a function that's too long, error handling that's missing, types that are wrong)
  4. Open the Claude panel (Ctrl+Esc) and ask Claude to improve it
  5. Review the diff before accepting
  6. Try @-mentioning a file (VS Code) or selecting code and referring to "this code" (JetBrains)
  7. Ask Claude to fix any diagnostics showing in your Problems panel

The goal: get comfortable with the IDE workflow so it feels as natural as the terminal. Once it clicks, you'll reach for the panel instinctively for visual tasks and the CLI for automation tasks.
