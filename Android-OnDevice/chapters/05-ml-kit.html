Chapter 5

ML Kit — Google's Plug-and-Play AI

Text recognition, barcode scanning, translation, and more — zero ML expertise required

The Highest-Level AI Toolkit on Android

In Chapter 3 you built an image classifier from scratch with TFLite. In Chapter 4 you used MediaPipe for real-time vision tasks with much less code. Now we're going even higher up the abstraction ladder.

ML Kit is Google's highest-level on-device ML SDK. It wraps common AI features into APIs so simple they feel like regular Android SDK calls. Want to read text from a photo? Three lines. Scan a barcode? Three lines. Translate a sentence offline? Three lines.

If TFLite is cooking from raw ingredients and MediaPipe is a meal kit, ML Kit is ordering takeout — you describe what you want and it just appears.

When to Use Which

Framework Best For Flexibility Setup Time
ML Kit Common tasks (OCR, barcode, translation, face detection) Low — use their APIs as-is Minutes
MediaPipe Real-time tracking (hands, face mesh, pose, objects) Medium — configurable but fixed pipelines 30 min
TFLite Custom models, specialized tasks, full control High — any model, any pipeline Hours
Start at the top. Check ML Kit first — if it does what you need, you're done. If you need more control or real-time landmark tracking, try MediaPipe. Only drop to raw TFLite when you need a custom model that neither ML Kit nor MediaPipe supports.

On-Device vs. Cloud Features

ML Kit has two categories of APIs. Know which is which before you ship:

Fully on-device (no network needed)

Text recognition (OCR) Barcode / QR scanning Face detection & face mesh On-device translation (with downloaded packs) Smart reply Entity extraction Digital ink recognition Image labeling Object detection

These work in airplane mode. Models are bundled or downloaded once. No per-request cost.

Requires network / Firebase

Cloud text recognition (document AI) Cloud image labeling (more categories) Cloud landmark recognition Custom model hosting via Firebase ML

These send data to Google Cloud. Higher accuracy but require connectivity and may incur API costs.

This chapter focuses entirely on the on-device features — the ones that work offline and cost nothing.

Text Recognition (OCR) — Building a Receipt Scanner

This is ML Kit's killer feature. Point the camera at text — any text — and ML Kit extracts it with structure preserved. It understands blocks, lines, words, and even the bounding box of each element.

OCR pipeline — from camera to structured text:

Camera Frame → InputImage → TextRecognizer → Text Blocks → Lines → Words → Compose UI

Add the dependency:

dependencies {
    implementation("com.google.mlkit:text-recognition:16.0.1")
}

And recognize text from a camera frame:

val recognizer = TextRecognition.getClient(TextRecognizerOptions.DEFAULT_OPTIONS)

// From CameraX ImageProxy:
analysis.setAnalyzer(executor) { proxy ->
    val mediaImage = proxy.image ?: return@setAnalyzer proxy.close()
    val inputImage = InputImage.fromMediaImage(
        mediaImage, proxy.imageInfo.rotationDegrees
    )

    recognizer.process(inputImage)
        .addOnSuccessListener { text ->
            // text.textBlocks → paragraphs
            // each block has .lines → lines of text
            // each line has .elements → individual words
            for (block in text.textBlocks) {
                for (line in block.lines) {
                    Log.d("OCR", line.text)  // "Total: $42.50"
                }
            }
        }
        .addOnCompleteListener { proxy.close() }
}

The result is hierarchical: Text contains TextBlocks (paragraphs), which contain Lines, which contain Elements (words). Each element has a boundingBox — a Rect telling you exactly where that text appears in the image.

Building the Receipt Scanner UI

@Composable
fun ReceiptResults(blocks: List<TextBlock>) {
    LazyColumn(modifier = Modifier.padding(16.dp)) {
        items(blocks) { block ->
            Card(modifier = Modifier.fillMaxWidth().padding(vertical = 4.dp)) {
                Column(modifier = Modifier.padding(12.dp)) {
                    block.lines.forEach { line ->
                        Text(
                            text = line.text,
                            fontSize = 14.sp,
                            fontFamily = FontFamily.Monospace
                        )
                    }
                }
            }
        }
    }
}

Barcode and QR Code Scanning

ML Kit's barcode scanner is fast, reliable, and handles every format you'll encounter. It's the one feature where ML Kit is clearly the best choice on Android — better than ZXing, faster than any custom solution.

dependencies {
    implementation("com.google.mlkit:barcode-scanning:17.3.0")
}

// Configure which formats to scan (narrower = faster)
val options = BarcodeScannerOptions.Builder()
    .setBarcodeFormats(
        Barcode.FORMAT_QR_CODE,
        Barcode.FORMAT_EAN_13,
        Barcode.FORMAT_CODE_128
    )
    .build()

val scanner = BarcodeScanning.getClient(options)

scanner.process(inputImage)
    .addOnSuccessListener { barcodes ->
        for (barcode in barcodes) {
            when (barcode.valueType) {
                Barcode.TYPE_URL -> openUrl(barcode.url!!.url!!)
                Barcode.TYPE_WIFI -> connectToWifi(barcode.wifi!!)
                Barcode.TYPE_PRODUCT -> lookupProduct(barcode.rawValue!!)
            }
        }
    }
Restrict barcode formats for speed. If you only need QR codes, don't scan for all 13 formats. Each additional format adds processing time. Scanning just QR codes is about 2x faster than scanning all formats.

On-Device Translation

ML Kit can translate between 50+ language pairs entirely on-device. Each language model is small (2-5MB), downloaded on demand, and runs without any network connection once downloaded.

dependencies {
    implementation("com.google.mlkit:translate:17.0.3")
}

// Create a translator for English → Spanish
val options = TranslatorOptions.Builder()
    .setSourceLanguage(TranslateLanguage.ENGLISH)
    .setTargetLanguage(TranslateLanguage.SPANISH)
    .build()
val translator = Translation.getClient(options)

// Download the language model if not already cached
val conditions = DownloadConditions.Builder()
    .requireWifi()  // don't burn mobile data
    .build()

translator.downloadModelIfNeeded(conditions)
    .addOnSuccessListener {
        // Model ready — now translate!
        translator.translate("Hello, how are you?")
            .addOnSuccessListener { translated ->
                // translated = "Hola, ¿cómo estás?"
            }
    }

Managing Downloaded Language Models

Each language pair downloads a small model (~2-5MB). On a device with limited storage, you might want to manage these:

val modelManager = RemoteModelManager.getInstance()

// List all downloaded translation models
modelManager.getDownloadedModels(TranslateRemoteModel::class.java)
    .addOnSuccessListener { models ->
        models.forEach { model ->
            Log.d("Translation", "Downloaded: ${model.language}")
        }
    }

// Delete a model to free space
val spanishModel = TranslateRemoteModel.Builder(TranslateLanguage.SPANISH).build()
modelManager.deleteDownloadedModel(spanishModel)
Translation quality is "travel-grade," not "professional-grade." On-device translation models are small and optimized for speed, not nuance. They handle everyday phrases, menus, signs, and basic communication well. They struggle with idioms, technical jargon, and literary text. For critical translations, fall back to a cloud API.

Smart Reply

Smart Reply generates contextual reply suggestions for messaging apps — like the "quick reply" chips in Gmail or Messages. You feed it a conversation history, and it suggests 1-3 appropriate responses.

dependencies {
    implementation("com.google.mlkit:smart-reply:17.0.4")
}

val smartReply = SmartReply.getClient()

// Build conversation history
val conversation = mutableListOf(
    TextMessage.createForRemoteUser(
        "Hey, are we still meeting for lunch?", System.currentTimeMillis(), "Alice"
    ),
    TextMessage.createForLocalUser(
        "Yes! Where should we go?", System.currentTimeMillis()
    ),
    TextMessage.createForRemoteUser(
        "How about that new Thai place?", System.currentTimeMillis(), "Alice"
    )
)

smartReply.suggestReplies(conversation)
    .addOnSuccessListener { result ->
        if (result.status == SmartReplySuggestionResult.STATUS_SUCCESS) {
            result.suggestions.forEach { suggestion ->
                Log.d("SmartReply", suggestion.text)
                // "Sounds great!", "Sure!", "I'm in!"
            }
        }
    }
Smart Reply works best with 3-10 messages of context. Too few messages and it can't understand the conversation. Too many and older messages add noise. Feed it the most recent, relevant messages.

Entity Extraction — Detecting Actionable Items in Text

Entity extraction finds structured data in unstructured text: dates, addresses, phone numbers, emails, flight numbers, tracking numbers, payment card numbers, and more. It's like a smart clipboard that knows what to do with copied text.

dependencies {
    implementation("com.google.mlkit:entity-extraction:16.0.0-beta5")
}

val extractor = EntityExtraction.getClient(
    EntityExtractorOptions.Builder(EntityExtractorOptions.ENGLISH).build()
)

extractor.downloadModelIfNeeded()
    .addOnSuccessListener {
        val params = EntityExtractionParams.Builder("Call me at 555-0123 on March 15")
            .build()

        extractor.annotate(params)
            .addOnSuccessListener { annotations ->
                annotations.forEach { annotation ->
                    val text = annotation.annotatedText  // "555-0123"
                    annotation.entities.forEach { entity ->
                        when (entity) {
                            is PhoneEntity -> dialPhone(text)
                            is DateTimeEntity -> addToCalendar(entity)
                            is AddressEntity -> openMaps(text)
                        }
                    }
                }
            }
    }

Digital Ink Recognition — Handwriting to Text

Digital ink recognition converts touch-drawn strokes into text. Unlike OCR (which reads text from photos), this works with live drawing input — perfect for note-taking apps, signature fields, or handwriting input in any language.

dependencies {
    implementation("com.google.mlkit:digital-ink-recognition:18.1.0")
}

// Collect strokes from touch events
val strokeBuilder = Ink.Stroke.builder()
// In your Canvas onDrag handler:
strokeBuilder.addPoint(Ink.Point.create(x, y, System.currentTimeMillis()))

// When the user lifts their finger:
val ink = Ink.builder().addStroke(strokeBuilder.build()).build()

val model = DigitalInkRecognitionModelIdentifier.fromTag("en-US")!!
val recognizer = DigitalInkRecognition.getClient(
    DigitalInkRecognizerOptions.builder(
        DigitalInkRecognitionModel.builder(model).build()
    ).build()
)

recognizer.recognize(ink)
    .addOnSuccessListener { result ->
        val topCandidate = result.candidates[0].text
        // topCandidate = "Hello world"
    }

ML Kit on-device feature availability by Android version:

  • Android 5.0+ (API 21) — Text recognition, barcode scanning, face detection, image labeling
  • Android 6.0+ (API 23) — On-device translation, smart reply, entity extraction
  • Android 7.0+ (API 24) — Digital ink recognition, all remaining features
  • Google Play Services required — ML Kit on-device models are delivered via Google Play Services. Devices without GMS (some Huawei phones, Amazon Fire tablets) cannot use ML Kit. For those devices, fall back to raw TFLite.

Custom Models via ML Kit

ML Kit isn't limited to its built-in features. You can run your own TFLite models through ML Kit's infrastructure, getting model download management, versioning, and OTA updates for free.

Local Model (Bundled in APK)

val localModel = LocalModel.Builder()
    .setAssetFilePath("my_custom_model.tflite")
    .build()

val labeler = ImageLabeling.getClient(
    CustomImageLabelerOptions.Builder(localModel)
        .setConfidenceThreshold(0.5f)
        .build()
)

Remote Model (Firebase-Hosted, OTA Updates)

val remoteModel = CustomRemoteModel.Builder(
    FirebaseModelSource.Builder("my_model_v2").build()
).build()

// Check if model is downloaded, fetch if needed
val downloadConditions = DownloadConditions.Builder()
    .requireWifi()
    .build()

RemoteModelManager.getInstance()
    .download(remoteModel, downloadConditions)
    .addOnSuccessListener {
        // Model ready — create labeler with remote model
        val labeler = ImageLabeling.getClient(
            CustomImageLabelerOptions.Builder(remoteModel)
                .setConfidenceThreshold(0.5f)
                .build()
        )
    }

Why Use Firebase ML Model Hosting?

Hosting models on Firebase lets you update your model without shipping an app update. Train a better model? Upload it to Firebase, and users get it automatically on next app launch. This is critical for features that improve over time — recommendation models, spam filters, content classifiers. We'll dive deeper into this in Chapter 11 (Production Deployment).

Combining ML Kit Features

The real power of ML Kit emerges when you combine multiple features. A "smart scanner" that reads text, scans barcodes, and extracts entities from a single camera feed gives users a unified scanning experience:

analysis.setAnalyzer(executor) { proxy ->
    val image = InputImage.fromMediaImage(
        proxy.image!!, proxy.imageInfo.rotationDegrees
    )

    // Run all three in parallel
    val textTask = textRecognizer.process(image)
    val barcodeTask = barcodeScanner.process(image)

    Tasks.whenAllComplete(textTask, barcodeTask)
        .addOnCompleteListener {
            val text = textTask.result
            val barcodes = barcodeTask.result

            // Extract entities from recognized text
            if (text.text.isNotEmpty()) {
                entityExtractor.annotate(
                    EntityExtractionParams.Builder(text.text).build()
                ).addOnSuccessListener { entities ->
                    updateResults(text, barcodes, entities)
                }
            }
            proxy.close()
        }
}

Try It: Build a Smart Scanner

Build a "smart scanner" that combines three ML Kit features in one camera view.

  1. Set up ML Kit with text recognition, barcode scanning, and entity extraction dependencies
  2. Create a CameraX preview with ImageAnalysis
  3. Run text recognition and barcode scanning on each frame (in parallel using Tasks.whenAllComplete)
  4. When text is found, run entity extraction on the recognized text
  5. Display results in a Compose UI:
    • Recognized text blocks with bounding box highlights
    • Detected barcodes with format and value
    • Extracted entities (phone numbers, dates, addresses) as tappable chips

Test it on:

  • A restaurant receipt — extract total, date, phone number
  • A business card — extract name, phone, email, address
  • A product with a barcode — scan the barcode and read the product label

Note how fast each feature processes and whether running all three simultaneously causes any frame drops.
