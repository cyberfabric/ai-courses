Chapter 6

Gemini Nano — On-Device Generative AI

Summarize, rewrite, and generate text — entirely on the user's phone

From Recognition to Generation

Everything we've built so far — classifying images, detecting objects, reading text, scanning barcodes — is recognition. The model looks at input and tells you what it sees. Useful, but limited.

Gemini Nano changes the game. It's a small large language model (LLM) that runs directly on the phone and can generate text. Summarize an article. Rewrite a paragraph in a different tone. Suggest contextually aware replies. Create content that didn't exist before — all without a single network request.

This is the same technology behind ChatGPT and Claude, shrunk down to fit on a phone. It's not as capable as the cloud models — but for focused, single-task generation, it's surprisingly good.

What Gemini Nano Is (and Isn't)

~3.25B Parameters (estimated)
~30 Tokens per second on-device
$0 Per-request cost

Gemini Nano is a distilled version of Google's Gemini model family, optimized to run on mobile hardware. Unlike the TFLite models we've used (2-25MB), Gemini Nano is around 2GB — but you don't manage it. The Android OS downloads, stores, updates, and serves the model through a system service called AICore.

Think of it like Google Play Services: it's there, it's shared across apps, and you just call the API.

Device Compatibility

Gemini Nano requires specific hardware and software:

  • Google Pixel 8 Pro, Pixel 9 series — Tensor G3/G4 with dedicated TPU. First-class support.
  • Samsung Galaxy S24 series and newer — Snapdragon 8 Gen 3 with Hexagon NPU. Full support via Samsung's AICore integration.
  • Other flagships with 8GB+ RAM — expanding support through Android 15+ and OEM partnerships. Check at runtime.
  • Android 14+ required — AICore system service is available starting with Android 14.
  • Not available on: mid-range phones (insufficient RAM/NPU), older flagships, emulators.

The supported device list is expanding with each Android release. Always check availability at runtime — never assume.

The AICore System Service

Unlike TFLite models that live inside your APK, Gemini Nano is managed by the AICore system service. Here's how it works:

  • The model is shared across all apps — one copy on the device, not one per app
  • The OS handles downloading — the model is fetched via Google Play Services, not by your app
  • The OS handles updates — improved models arrive through system updates, not app updates
  • Memory is managed — AICore loads and unloads the model based on demand and available resources

From your app's perspective, you just ask "is the model available?" and if yes, send prompts and receive generated text. You never touch a model file.

Setting Up Gemini Nano

Add the AI Edge dependency:

dependencies {
    implementation("com.google.ai.edge.aicore:aicore:0.0.4")
}

Declare AICore capability in your AndroidManifest.xml:

<application ...>
    <meta-data
        android:name="com.google.ai.edge.aicore.FEATURE"
        android:value="text_generation" />
</application>

Check availability and create the model:

// Check if Gemini Nano is available on this device
val isAvailable = GenerativeModel.isAvailable(context)

if (isAvailable) {
    val generativeModel = GenerativeModel(
        generationConfig = GenerationConfig.builder()
            .setTemperature(0.7f)
            .setTopK(40)
            .setMaxOutputTokens(256)
            .build()
    )
    // Ready to generate!
} else {
    // Fall back to cloud API or disable the feature
    showUnsupportedMessage()
}
Always check isAvailable() before creating a GenerativeModel. On unsupported devices, attempting to create the model will throw an exception. Even on supported hardware, the model might not be downloaded yet (first boot, after a factory reset). Handle both cases gracefully.

Text Generation — Building a "Rewrite This" Feature

Let's build a practical feature: a text rewriter that transforms user input into different tones. The user writes something, taps a button, and sees it rewritten — all on-device.

On-device text generation pipeline:

User Text → Prompt Template → Gemini Nano → Token Stream → Compose UI 
suspend fun rewriteText(
    model: GenerativeModel,
    text: String,
    tone: String  // "formal", "casual", "shorter", "simpler"
): Flow<String> = flow {
    val prompt = "Rewrite the following text in a $tone tone. " +
        "Output only the rewritten text, nothing else.\n\n$text"

    val response = model.generateContentStream(prompt)
    response.collect { chunk ->
        chunk.text?.let { emit(it) }
    }
}

Streaming Tokens for Responsive UX

Gemini Nano generates text token by token. Instead of waiting for the complete response (which could take 2-5 seconds), stream tokens to the UI as they arrive — just like ChatGPT does:

@Composable
fun RewriteScreen(model: GenerativeModel) {
    var inputText by remember { mutableStateOf("") }
    var outputText by remember { mutableStateOf("") }
    var isGenerating by remember { mutableStateOf(false) }
    val scope = rememberCoroutineScope()

    Column(modifier = Modifier.padding(16.dp)) {
        OutlinedTextField(
            value = inputText,
            onValueChange = { inputText = it },
            label = { Text("Your text") },
            modifier = Modifier.fillMaxWidth()
        )

        Row(horizontalArrangement = Arrangement.spacedBy(8.dp)) {
            listOf("Formal", "Casual", "Shorter", "Simpler").forEach { tone ->
                Button(
                    onClick = {
                        isGenerating = true
                        outputText = ""
                        scope.launch {
                            rewriteText(model, inputText, tone.lowercase())
                                .collect { token -> outputText += token }
                            isGenerating = false
                        }
                    },
                    enabled = !isGenerating
                ) { Text(tone) }
            }
        }

        Text(text = outputText, modifier = Modifier.padding(top = 16.dp))
    }
}
Streaming is essential for perceived performance. Generating 100 tokens at ~30 tokens/sec takes ~3.3 seconds. If you wait for the full response, the user stares at a spinner for 3 seconds. With streaming, the first token appears in ~200ms and text fills in progressively — the feature feels instant.

Summarization — The "TL;DR" Button

Summarization is one of Gemini Nano's strongest on-device use cases. The model is good at compressing text into key points:

suspend fun summarize(model: GenerativeModel, article: String): String {
    // Keep the prompt short and direct for tiny models
    val prompt = "Summarize this in 3 bullet points:\n\n$article"

    val response = model.generateContent(prompt)
    return response.text ?: "Unable to summarize"
}

Good prompts for Gemini Nano

"Summarize this in 3 bullet points: {text}" "Rewrite this in a formal tone: {text}" "List the key dates mentioned: {text}" "Is this review positive or negative? Answer in one word: {text}"

Short, single-task prompts with clear output format. One instruction, one expected result.

Prompts that overwhelm Nano

"You are a senior editor. First analyze the writing style, then identify the thesis, then summarize in exactly 5 bullet points of no more than 20 words each, and finally suggest 3 improvements." "Compare these two articles and write a balanced synthesis that..." "Generate a 500-word blog post about..."

Multi-step reasoning, complex instructions, and long-form generation exceed Nano's capabilities. These need a cloud model.

Smart Reply Generation

Unlike ML Kit's pattern-matching Smart Reply (Chapter 5), Gemini Nano can generate contextually rich replies that understand conversation nuance:

suspend fun generateReplies(
    model: GenerativeModel,
    conversation: List<Message>
): List<String> {
    val context = conversation.takeLast(5).joinToString("\n") {
        "${it.sender}: ${it.text}"
    }

    val prompt = "Given this conversation:\n$context\n\n" +
        "Suggest 3 short reply options for me. " +
        "Output only the replies, one per line."

    val response = model.generateContent(prompt)
    return response.text?.lines()
        ?.filter { it.isNotBlank() }
        ?.take(3)
        ?: emptyList()
}

The MediaPipe LLM Inference API — Beyond Gemini Nano

What if you want to run a different LLM on-device? Or target devices that don't support Gemini Nano? MediaPipe's LLM Inference API lets you run open-source models like Gemma, Phi, and StableLM directly on Android.

G
Gemma 2B
Google's open-source small LLM. 2B parameters, ~1.5GB quantized. Runs on 6GB+ RAM devices. Good for summarization and simple generation. Available from Kaggle.
P
Phi-2 / Phi-3 Mini
Microsoft's efficient small LLMs. Strong reasoning for their size. Phi-3 Mini (3.8B) fits on 8GB+ devices. Good alternative when Gemma struggles with reasoning tasks.
S
StableLM 2
Stability AI's compact LLM. 1.6B parameters, very fast inference. Works on 4GB+ devices. Best for simple text generation and completion tasks.
// MediaPipe LLM Inference API setup
dependencies {
    implementation("com.google.mediapipe:tasks-genai:0.10.22")
}

// Load a Gemma 2B model
val llmInference = LlmInference.createFromOptions(context,
    LlmInference.LlmInferenceOptions.builder()
        .setModelPath("/path/to/gemma-2b-it-q4.bin")
        .setMaxTokens(512)
        .setTemperature(0.8f)
        .build()
)

// Generate text
val result = llmInference.generateResponse("Summarize: $text")
Log.d("LLM", result)
Gemini Nano vs. MediaPipe LLM Inference: Use Gemini Nano when available — it's faster, better optimized, and OS-managed. Use MediaPipe LLM Inference when you need a specific open-source model, want broader device compatibility, or need to run offline on devices without AICore.

Prompt Engineering for Tiny Models

Prompts that work beautifully on GPT-4 or Claude will often fail on Gemini Nano. Here's why and what to do about it:

Cloud Model Habit Why It Fails on Nano What to Do Instead
Long system prompts (200+ words) Eats into the small context window Keep prompts under 50 words
Multi-step instructions Model loses track after step 2 One task per prompt
"Think step by step" Generates rambling filler text Ask for direct output only
Complex output formats (JSON, XML) Frequently produces malformed output Use plain text, one item per line
Nuanced tone instructions "Witty yet professional" is too subtle Use one clear adjective: "formal" or "casual"

Limitations and Guardrails

Be honest with your users about what on-device AI can and can't do. Gemini Nano is impressive for its size, but it has real limits:
  • No long-form generation — anything over 200-300 words degrades in quality
  • No complex reasoning — multi-step logic, math, or code generation is unreliable
  • Small context window — can't process documents longer than ~4,000 tokens
  • No image understanding — text only (no multimodal input on-device yet)
  • Occasional hallucinations — may generate plausible-sounding but incorrect information
  • Limited languages — best in English, reasonable in major languages, poor in minority languages

Fallback Strategies — On-Device First, Cloud Second

The best user experience combines on-device speed with cloud capability. Try Gemini Nano first; if the device doesn't support it, fall back to a cloud model:

class SmartTextGenerator(private val context: Context) {

    private var onDeviceModel: GenerativeModel? = null

    suspend fun initialize() {
        if (GenerativeModel.isAvailable(context)) {
            onDeviceModel = GenerativeModel(
                generationConfig = GenerationConfig.builder()
                    .setMaxOutputTokens(256)
                    .build()
            )
        }
    }

    suspend fun generate(prompt: String): String {
        // Try on-device first
        onDeviceModel?.let { model ->
            return try {
                model.generateContent(prompt).text ?: fallbackToCloud(prompt)
            } catch (e: Exception) {
                fallbackToCloud(prompt)
            }
        }
        // No on-device model available
        return fallbackToCloud(prompt)
    }

    private suspend fun fallbackToCloud(prompt: String): String {
        // Use Gemini Pro/Flash via the Generative AI SDK
        val cloudModel = com.google.ai.client.generativeai.GenerativeModel(
            modelName = "gemini-2.0-flash",
            apiKey = BuildConfig.GEMINI_API_KEY
        )
        return cloudModel.generateContent(prompt).text ?: "Generation failed"
    }
}

The Hybrid Pattern

The most robust approach: on-device for speed and privacy, cloud for capability and coverage. Use Gemini Nano when available (instant, free, private). Fall back to Gemini Flash when not (fast cloud, cheap, wider device support). Escalate to Gemini Pro only for complex tasks. The user gets the best experience on every device — and you pay cloud costs only when truly needed.

Text generation performance — on-device vs. cloud:

Model Location Time to First Token Generation Speed Cost per 1K Requests
Gemini Nano On-device (Pixel 9 Pro) ~150 ms ~30 tokens/sec $0
Gemini Nano On-device (Galaxy S24) ~200 ms ~25 tokens/sec $0
Gemini 2.0 Flash Cloud ~400 ms ~150 tokens/sec ~$0.10
Gemini 1.5 Pro Cloud ~800 ms ~80 tokens/sec ~$3.50

Gemini Nano's time-to-first-token is 2-5x faster than cloud models because there's no network latency. For short generations (50-100 tokens), on-device feels nearly instant. Cloud models catch up on longer responses where their faster token rate compensates for network overhead.

Try It: Build an AI Writing Assistant

Build an "AI Writing Assistant" with 4 text transformations running on-device.

  1. Set up the AI Edge SDK with Gemini Nano (or MediaPipe LLM Inference with Gemma as fallback)
  2. Create a Compose UI with a text input field and 4 action buttons:
    • Summarize — condense the text to 2-3 key points
    • Formalize — rewrite in professional/business tone
    • Simplify — rewrite for a general audience, plain language
    • Expand — add more detail and explanation
  3. Stream generated tokens to the UI for responsive feedback
  4. Add a cloud fallback (Gemini Flash) for unsupported devices
  5. Measure and display: time-to-first-token, total generation time, token count

Compare: Run the same text through both on-device and cloud. Note the differences in quality, speed, and tone accuracy. Where does Nano match the cloud? Where does it fall short?
