The Big Idea

MediaPipe is the meal kit of on-device AI — pre-built, optimized ML solutions with all preprocessing, postprocessing, and threading built in. What took an entire chapter with raw TFLite takes ~30 lines with MediaPipe. Start here, drop to TFLite only when you need a custom model.

🎯 The Six Key Tasks

📦
ObjectDetector
Bounding boxes + labels around detected objects. Camera apps, warehouse scanning, scene description.
😀
FaceLandmarker
468 face landmarks (eyes, nose, lips, jawline). AR filters, face-controlled UI, drowsiness detection.
✋
HandLandmarker
21 landmarks per hand (fingertips, knuckles, wrist). Gesture recognition, sign language, touchless control.
🏃
PoseLandmarker
33 body landmarks (head to toes). Fitness rep counting, physical therapy, gesture-controlled games.
🏷️
ImageClassifier
Same as Ch. 3 but simpler API. No Interpreter or ImageProcessor — MediaPipe handles it all.
👋
GestureRecognizer
Built-in gesture detection (thumbs up, peace, pointing). Higher level than raw hand landmarks.

⚙️ Running Modes

Mode Use When Call Pattern Threading
IMAGE Single photo detect(image) Synchronous, blocks
VIDEO Recorded video detectForVideo(img, ts) Synchronous, ordered
LIVE_STREAM ⭐ Camera feed detectAsync(img, ts) Async callback, drops frames

⚠️ Always use LIVE_STREAM for camera feeds. IMAGE/VIDEO block the thread → backpressure → stutter.

🎮 Gesture-to-Action Design Rules

✅
Distinct Gestures
Open palm, peace sign, fist, thumbs up — poses that are hard to trigger accidentally.
✅
Safe Actions
Map gestures to reversible actions (pause, screenshot, like). Never to destructive ops (delete, send, pay).
❌
Avoid Ambiguity
Subtle finger curls and wrist rotations = false positives. "Any hand visible" = always firing.
❌
Avoid Destructive Triggers
Never wire "delete file" or "send message" to a gesture alone. Require confirmation for irreversible actions.

💾 Multi-Task Memory Budget

  1. 1 Task (~30-50 MB)

    Fine on any device with 3 GB+ RAM. The typical setup.

  2. 2 Tasks (~60-90 MB)

    Comfortable on 4 GB+. Can run face + hand simultaneously on the same camera feed.

  3. 3+ Tasks (~100-150 MB)

    Flagship only (6 GB+). Watch onTrimMemory callbacks. Unload idle tasks.

On budget devices (2-3 GB): run one task at a time and switch based on user need.

🚀 Performance Tuning Checklist

GPU Delegate
.setDelegate(Delegate.GPU) in BaseOptions — reduces CPU load and heat for sustained inference.
setMaxResults
Only request as many detections as you need. Fewer results = less postprocessing work.
setScoreThreshold
Filter low-confidence results early (e.g., 0.5f). Saves postprocessing and reduces UI noise.
Frame Skipping
On slow devices, process every 2nd or 3rd frame instead of every one. UX stays smooth.
Coordinate Scaling
Model input is ~320×320 but preview fills the screen. Scale bounding box coords: box.left * (screenWidth / modelWidth).
NormalizedLandmark
Landmark x/y/z in [0, 1] range. Multiply by view dimensions for pixels. z = depth (negative = closer).
Close When Done
Call detector.close() when leaving screen to free GPU/NPU resources immediately.

🧪 Quick Quiz

What's the best analogy for MediaPipe vs raw TFLite?

MediaPipe is a meal kit (pre-portioned, pre-prepped, just assemble). Raw TFLite is cooking from scratch (buy ingredients, chop, season, cook). Same quality result, fraction of the work.

Which RunningMode should you use for live camera feeds?

LIVE_STREAM. It's asynchronous — results arrive via callback. It automatically drops frames when busy. IMAGE and VIDEO modes block the thread, causing backpressure and stutter.

How many landmarks does FaceLandmarker track? HandLandmarker? PoseLandmarker?

Face: 468 landmarks. Hand: 21 per hand. Pose: 33 body landmarks. All include x, y, z coordinates normalized to [0, 1].

How do you detect if a finger is "up" in hand tracking?

Check if the fingertip's y-coordinate is less than its base knuckle's y-coordinate. E.g., indexTip.y() < landmarks[6].y() means the index finger is extended (y-axis is inverted — lower value = higher position).

When should you drop to raw TFLite instead of using MediaPipe?

Only when you need a custom model that MediaPipe doesn't support — a model trained on your own data, or a specialized architecture not in MediaPipe's catalog. Start with MediaPipe; drop to TFLite only when forced.

📌 Key Takeaways

  • MediaPipe = batteries-included ML. ~30 lines for a complete feature vs an entire chapter with raw TFLite.
  • Six key tasks: ObjectDetector, FaceLandmarker (468), HandLandmarker (21), PoseLandmarker (33), ImageClassifier, GestureRecognizer.
  • Always use LIVE_STREAM mode for camera. IMAGE/VIDEO are synchronous and will stutter.
  • Three model variants: Lite (fast), Full (balanced, default), Heavy (precise). Pick based on device and accuracy needs.
  • Gesture design: use distinct poses, map to safe/reversible actions, never wire destructive ops to gestures alone.
  • Start with MediaPipe. Drop to raw TFLite only when you need a custom model it doesn't support.
  • Scale bounding box coordinates from model space to screen space — or your overlays will be misaligned.
