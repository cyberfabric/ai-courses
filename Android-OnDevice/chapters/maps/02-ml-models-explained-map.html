How ML Models Work

Memory Map
💡

The Big Idea

A model is a compiled function you didn't write — like a .so native library. You call it with an input (image, text) and get an output (label, score). You don't read its source; the "code" is millions of learned numerical weights.

🧠 What a Model Is (and Isn't)

✅
IS: A Black-Box Function
A 2-50 MB file of numerical weights. Load it, feed input, read output. Read-only at runtime — you never modify it.
❌
NOT: Code You Write
Not a rules engine, not if/else chains, not a database. It multiplies matrices — very fast — that's all.
📦
Metadata = README
Good .tflite files embed metadata: input/output shapes, normalization params, labels. Your contract with the model.
🔍
Netron = Layout Inspector
Open any model in netron.app to see input/output shapes, layers, and metadata. Like Android Studio's inspector, but for ML.

🖼️ Image Preprocessing (Steps 2 in Detail)

  1. Resize to Expected Dimensions

    Camera gives 4032×3024. Model wants 224×224 (or 320, 512). Use Bitmap.createScaledBitmap() or TFLite's ResizeOp.

  2. Normalize Pixel Values

    Bitmap pixels: 0-255 integers. Model wants: 0.0-1.0 floats (divide by 255). Some models need ImageNet mean/std normalization.

  3. Arrange Channels

    Android Bitmap = ARGB. Models expect RGB (no alpha) or BGR. TFLite's ImageProcessor handles all three steps in one pipeline.

📐 Tensors = Multi-Dimensional Arrays

You already use tensors. A Bitmap is a 3D tensor. An IntArray is a 1D tensor.

Android Concept Tensor Shape Dimensions
Confidence score [1] Scalar
Class probabilities [1000] 1D
Grayscale image [224, 224] 2D
RGB image [224, 224, 3] 3D
⭐ Standard model input [1, 224, 224, 3] 4D (batch, H, W, channels)

⚠️ Shape mismatches = #1 beginner mistake. Always check the model's expected input shape before feeding data.

📁 Model Formats

Format Extension When to Use
TensorFlow Lite .tflite ⭐ Default choice. Best acceleration, biggest ecosystem, widest Android support.
MediaPipe Bundle .task Pre-packaged model + metadata + config. Easiest to use — zero preprocessing code.
ONNX .onnx PyTorch-origin models, cross-platform (iOS+Android). Fallback when TFLite conversion fails.
PyTorch Mobile .ptl Rare on Android. Usually converted to TFLite or ONNX first.

⚖️ The Size-Accuracy Tradeoff

Model Size Accuracy Speed Verdict
MobileNetV3-Small 2.4 MB 67.5% ~3 ms Budget devices, max speed
MobileNetV3-Large 5.4 MB 75.2% ~6 ms ⭐ Sweet spot for most apps
EfficientNet-Lite4 13 MB 80.4% ~20 ms When accuracy matters more
EfficientNet-B7 66 MB 84.3% ~120 ms Usually too big for mobile

Rule of thumb: pick the smallest model that meets your accuracy needs, then fine-tune for your domain.

📱 Model Size → Device Compatibility

  1. Under 5 MB (MobileNet, MediaPipe)

    Runs on virtually any device from the last 5 years, including 2 GB RAM budget phones.

  2. 5-25 MB (EfficientNet-Lite, MobileBERT)

    Comfortable on mid-range (4 GB+ RAM). May need fallbacks on budget devices.

  3. 25-100 MB (Whisper small, larger EfficientNet)

    Flagship territory (8 GB+ RAM). Consider dynamic download instead of bundling in APK.

  4. Gemini Nano (~2 GB, system-managed)

    Pixel 8 Pro+, Galaxy S24+. The OS manages the model; your app just calls the API.

📖 Key Terms

Tensor
A multi-dimensional array. A Bitmap is a 3D tensor (H × W × channels). Standard model input: [1, 224, 224, 3].
Inference
Running a trained model on new input to get a prediction. The model multiplies matrices — that's all.
Preprocessing
Transforming raw input (camera Bitmap) into model-ready format: resize, normalize pixels (0-255 → 0.0-1.0), arrange channels.
Postprocessing
Interpreting raw model output (array of floats) into meaningful results (label + confidence score).
Logits / Probabilities
The raw numbers a model outputs — one float per class. Find the max index, look up the label.
Model Metadata
Embedded info inside .tflite files: input/output shapes, normalization params, labels file. Always check before coding.
ImageProcessor
TFLite helper class that chains resize → normalize → channel ops into a clean preprocessing pipeline.
Netron
Free browser tool (netron.app) to visually inspect model files — see shapes, layers, and metadata.

🧪 Quick Quiz

What's the best analogy for an ML model?

A compiled function you didn't write — like a .so native library. You call it with inputs and get outputs, but you can't read its source code. The "source" is millions of learned numerical weights.

What three preprocessing steps do you do for almost every image model?

1) Resize to expected dimensions (e.g., 224×224). 2) Normalize pixel values (0-255 → 0.0-1.0). 3) Arrange channels correctly (ARGB → RGB).

What does [1, 224, 224, 3] mean?

A 4D tensor: batch of 1 image, 224 pixels tall, 224 pixels wide, 3 color channels (RGB). This is the standard input shape for image classification models.

What's the #1 beginner mistake with on-device models?

Shape mismatches — feeding a 640×480 camera frame to a model expecting 224×224. Always check the model's expected input shape and resize your data to match.

Which model format should you default to on Android?

TensorFlow Lite (.tflite). It has the best hardware acceleration support (NNAPI, GPU delegate), the most mature tooling, and the largest collection of pre-optimized mobile models.

📌 Key Takeaways

  • A model = a black-box function (input → numbers → output). You don't write it — you load and call it.
  • Every feature follows one loop: load → preprocess → inference → postprocess → display.
  • Your new skills are preprocessing (resize + normalize) and postprocessing (find max + lookup label).
  • Tensors are just multi-dimensional arrays. Standard image input: [1, 224, 224, 3].
  • Default to .tflite on Android. Use Netron (netron.app) to inspect any model file.
  • MobileNetV3-Large (5.4 MB, 75% accuracy, 6ms) is the sweet spot for most mobile apps.
  • Always check model metadata first — shape mismatches are the #1 beginner mistake.
