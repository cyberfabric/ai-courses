The Big Idea

The inference pattern is identical to vision — load model, preprocess, infer, postprocess — but the input is text or audio instead of pixels. Tokenization is the NLP equivalent of resizing an image. Use traditional NLP (MobileBERT) for classification tasks that need speed and consistency, Gemini Nano for generation tasks that need creativity.

📖 Key Terms

Tokenization
Converting text into integer IDs a model understands. The NLP equivalent of resizing an image to 224×224.
WordPiece
BERT's tokenizer. Splits unknown words into subwords: "playing" → "play" + "##ing". 30,522 vocabulary tokens.
Attention Mask
Binary array telling the model which tokens are real (1) vs padding (0). Prevents the model from attending to empty positions.
MobileBERT
Compressed BERT for mobile. 25MB, ~12ms on NPU. The go-to model for on-device text classification and sentiment.
Mel Spectrogram
Audio converted to a 2D image of frequency vs time. How audio models "see" sound — it's the audio equivalent of a Bitmap.
Cosine Similarity
Measures how similar two embedding vectors are. Range: -1 (opposite) to 1 (identical). Used for semantic search.
YAMNet
Google's audio classification model. 521 sound categories, 3.7MB. Identifies speech, music, sirens, dog barks, and more.
Keyword Spotting
Always-on wake word detection using a tiny (<1MB) model. Runs continuously with negligible battery impact.

🧪 Quick Quiz

Why is tokenization critical — what happens if you use the wrong tokenizer?

Each model expects specific token IDs from its matching tokenizer. BERT uses WordPiece, GPT uses BPE. The wrong tokenizer maps words to different IDs, producing meaningless input vectors and garbage output. Always use the tokenizer that shipped with your model.

How can a keyword spotter run 24/7 without draining the battery?

The model is tiny (<1MB) and inference takes ~1ms. At 16kHz audio, processing 1-second windows continuously costs almost no CPU or power. The heavy speech-to-text model only activates after the keyword is detected — that's the gated pattern.

When should you use MobileBERT vs Gemini Nano for an NLP task?

MobileBERT for classification tasks (spam, sentiment, intent) — it's fast (~12ms), deterministic, and reliable. Gemini Nano for generation tasks (summarization, rewriting) — it's creative but variable. Same input to MobileBERT always gives the same output; Gemini Nano may vary.

What does all-MiniLM give you that MobileBERT doesn't?

all-MiniLM produces 384-dimensional embedding vectors that capture semantic meaning. You can compute cosine similarity between vectors to find related texts without predefined categories. MobileBERT classifies into fixed categories; MiniLM enables open-ended semantic search and similarity.

What's the multilingual strategy recommendation?

Start with one multilingual model for broad coverage (100+ languages in 50-100MB). If a specific language underperforms, add a per-language specialist model (15-25MB) downloaded on demand. This gives instant global coverage with targeted accuracy improvements.

📌 Key Takeaways

  • Same pattern as vision: load model → preprocess (tokenize text / extract spectrogram) → infer → postprocess. The pipeline is universal.
  • MobileBERT is the NLP workhorse: 25MB, ~12ms, handles classification and sentiment. Tokenizer must match the model (WordPiece for BERT).
  • Text embeddings enable semantic search: all-MiniLM converts text to 384-dim vectors. Cosine similarity finds related content without predefined categories.
  • Gated audio pattern: tiny keyword spotter (<1MB, always-on) gates the heavy STT model. Never run full speech recognition continuously.
  • Three STT options: SpeechRecognizer (easiest), Vosk (always-on), Whisper.cpp (most accurate). Pick by your latency and accuracy needs.
  • Traditional NLP + generative AI: use MobileBERT for classification (deterministic), Gemini Nano for generation (creative). They complement each other.
