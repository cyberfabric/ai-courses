The Big Idea

A complete on-device image classifier is just 7 steps: project setup, add model, load interpreter, preprocess, infer, connect CameraX, display in Compose. The core inference is ~5 lines of real code.

🛠️ The 7-Step Build Process

  1. Set Up Project

    Add TFLite deps (tensorflow-lite, -support, -metadata) + CameraX deps. Set noCompress += "tflite" in Gradle.

  2. Add Model to Assets

    Place mobilenet_v3.tflite (5.4 MB) + labels.txt (1,000 classes) in src/main/assets/.

  3. Load the Model

    FileUtil.loadMappedFile() → memory-maps from APK. Create Interpreter with numThreads = 4.

  4. Build Preprocessing

    ImageProcessor: ResizeOp(224, 224) + NormalizeOp(0.0f, 255.0f). Wrap Bitmap in TensorImage.

  5. Run Inference

    interpreter.run(input, output). Output = FloatArray[1000]. Sort by score, take top 5.

  6. Connect CameraX

    ImageAnalysis with STRATEGY_KEEP_ONLY_LATEST. Analyzer runs on Dispatchers.Default.

  7. Display in Compose

    Box with AndroidView (camera preview) + overlay Column showing labels + confidence.

⚙️ Critical Setup Details

📦
noCompress "tflite"
TFLite memory-maps model files directly from the APK. Compressed files can't be memory-mapped — they'd need full decompression into RAM.
🗺️
Memory-Mapped Loading
FileUtil.loadMappedFile() reads the model without copying to RAM. Faster startup, lower memory footprint.
🔥
Warm-Up Runs
First inference is 2-5x slower (memory allocation, op scheduling). Run 2-3 dummy inferences before measuring or showing results.
🔄
KEEP_ONLY_LATEST
Drop frames while processing. Without this, frames queue up → memory balloons → crash. Essential for real-time ML.

🚨 Common Pitfalls Cheat Sheet

Symptom Cause Fix
💥 "Cannot load model" Wrong path or case Check src/main/assets/, exact filename
💥 "Shape mismatch" Raw camera res, not 224×224 Always ResizeOp first
🗑️ Everything is "digital clock" Forgot normalize (0-255 ≠ 0-1) Add NormalizeOp(0.0f, 255.0f)
🧊 ANR dialog Inference on main thread Dispatchers.Default.asExecutor()
↩️ Image rotated 90° Ignoring ImageProxy rotation Apply rotationDegrees via Matrix
💾 OutOfMemoryError New Bitmap every frame Reuse + STRATEGY_KEEP_ONLY_LATEST

📖 Key Classes and APIs

Interpreter
TFLite's core class. Load a model, call .run(input, output), close when done.
FileUtil.loadMappedFile()
Memory-maps a .tflite file from assets — avoids copying the entire model into heap.
ImageProcessor
Chains preprocessing ops: ResizeOp + NormalizeOp. Builds a reusable pipeline.
TensorImage / TensorBuffer
Wrappers for input (image → tensor) and output (tensor → float array). Bridge between Android and TFLite.
ImageAnalysis
CameraX use case that delivers frames for processing. Use STRATEGY_KEEP_ONLY_LATEST for ML.
ProcessCameraProvider
Binds camera use cases (Preview + ImageAnalysis) to a lifecycle owner. One provider per activity.
Classification
Simple data class: (label: String, score: Float). The result type for top-N predictions.

🧪 Quick Quiz

Why do you need noCompress += "tflite" in Gradle?

So TFLite can memory-map the model file directly from the APK without decompressing it. Compressed files must be fully decompressed into RAM first, wasting memory and time.

What backpressure strategy should you use for real-time ML with CameraX?

STRATEGY_KEEP_ONLY_LATEST. It drops frames if you're still processing the previous one, preventing queue buildup and OOM crashes.

Why does the first inference take 2-5x longer?

TFLite performs internal optimizations on the first run: memory allocation, operation scheduling, and JIT compilation. Run 2-3 warm-up inferences with a dummy bitmap before measuring or showing results.

Your model outputs "digital clock" for every image. What went wrong?

You forgot to normalize pixel values. The model expects 0.0-1.0 but is getting 0-255. Add NormalizeOp(0.0f, 255.0f) to your ImageProcessor.

Which thread should inference run on, and how do results reach the UI?

Inference runs on Dispatchers.Default (background). Results update a mutableStateOf, which triggers Compose recomposition on the main thread automatically.

📌 Key Takeaways

  • The complete classifier is 7 steps: setup → add model → load → preprocess → infer → CameraX → Compose overlay.
  • Set noCompress += "tflite" so TFLite can memory-map models directly from the APK.
  • Use ImageProcessor to chain resize + normalize — never feed raw camera bitmaps to a model.
  • Always use STRATEGY_KEEP_ONLY_LATEST and run inference on a background thread.
  • Warm up the model with 2-3 dummy inferences before measuring or showing results.
  • MobileNetV3-Large runs at 15-80ms CPU-only depending on device. Hardware acceleration (Ch. 7) improves 3-10x.
  • Shape mismatch and missing normalization are the two most common beginner mistakes.
