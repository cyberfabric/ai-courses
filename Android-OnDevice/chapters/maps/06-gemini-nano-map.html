Gemini Nano — On-Device Generative AI

The Big Idea

Gemini Nano is a small LLM that runs directly on the phone — the same technology behind ChatGPT and Claude, shrunk to fit in your pocket. It can generate text (summarize, rewrite, suggest replies), not just recognize it. Zero network, zero cost, zero data leaving the device.

✏️ Prompt Engineering for Tiny Models

Cloud Model Habit Why It Fails on Nano Do This Instead
Long system prompts (200+ words) Eats the small context window Under 50 words
Multi-step instructions Loses track after step 2 One task per prompt
"Think step by step" Rambling filler text Ask for direct output only
Complex formats (JSON, XML) Frequently malformed Plain text, one item per line
"Witty yet professional" Too subtle for a tiny model One clear adjective: "formal"

🚧 What Nano Can't Do

📏
No Long-Form
Quality degrades past 200-300 words. Keep generation tasks short and focused.
🧮
No Complex Reasoning
Multi-step logic, math, and code generation are unreliable. Single-task only.
📄
Small Context (~4K tokens)
Can't process long documents. Chunk or summarize input before sending.
🖼️
Text Only
No image understanding on-device yet. No multimodal input.
🎭
Hallucinations
May generate plausible-sounding but incorrect information. Don't use for factual queries.
🌐
Limited Languages
Best in English. Reasonable in major languages. Poor in minority languages.

🔀 The Hybrid Fallback Pattern

  1. Check Availability

    GenerativeModel.isAvailable(context) — is AICore present and model downloaded?

  2. Try On-Device (Gemini Nano)

    Instant, free, private. ~150ms to first token. Best for short generation tasks.

  3. Catch Failures

    Model not available, exception during generation, or output is empty/garbage.

  4. Fall Back to Cloud (Gemini Flash)

    Fast cloud model, cheap, wider device support. ~400ms to first token.

  5. Escalate if Needed (Gemini Pro)

    Only for complex tasks that Flash can't handle. ~800ms, higher cost.

The rule: on-device for speed & privacy, cloud for capability & coverage. Pay cloud costs only when truly needed.

🔧 Alternative On-Device LLMs (MediaPipe LLM Inference)

Gemma 2B
Google's open-source small LLM. ~1.5GB quantized. Runs on 6GB+ RAM. Good Nano alternative via MediaPipe.
Phi-2 / Phi-3 Mini
Microsoft's efficient LLMs. Strong reasoning for their size. Phi-3 Mini (3.8B) fits on 8GB+ devices.
StableLM 2
Stability AI's compact LLM. 1.6B params, very fast. Works on 4GB+ devices. Simple text tasks.
MediaPipe LLM Inference API
Run any compatible open-source LLM on Android. Use when Gemini Nano isn't available or you need a specific model.
When to use which?
Gemini Nano when available (fastest, OS-managed). MediaPipe LLM Inference for broader device coverage or specific open-source models.

🧪 Quick Quiz

What's the key difference between Chapters 3-5 and Chapter 6?

Chapters 3-5 are recognition — the model looks at input and tells you what it sees. Chapter 6 is generation — Gemini Nano creates new text that didn't exist before (summarization, rewriting, replies).

Do you manage the Gemini Nano model file yourself?

No. The AICore system service handles downloading, storing, updating, and memory management. The model is shared across all apps. You just call GenerativeModel.isAvailable() and send prompts.

Why does "think step by step" fail on Gemini Nano?

Nano generates rambling filler text instead of useful reasoning. It's too small for chain-of-thought. Instead, ask for direct output only — one task, short prompt, plain text output.

Why should you always stream tokens instead of waiting for the full response?

At ~30 tokens/sec, 100 tokens takes 3.3 seconds. With streaming, the first token appears in ~150ms and text fills in progressively — feels instant. Without streaming, the user stares at a spinner for 3+ seconds.

What's the hybrid fallback pattern?

Try Gemini Nano first (free, fast, private). If unavailable or fails, fall back to Gemini Flash (cheap cloud). Escalate to Gemini Pro only for complex tasks. On-device for speed, cloud for capability.

📌 Key Takeaways

  • Gemini Nano = on-device LLM. Generates text (summarize, rewrite, reply) — not just recognition.
  • ~3.25B params, ~30 tokens/sec, $0 cost. First token in ~150ms. Stream for responsive UX.
  • AICore manages everything — download, storage, updates, memory. You just call the API.
  • Always check isAvailable() first. Pixel 8 Pro+, Galaxy S24+, expanding. Never assume support.
  • Prompt engineering for tiny models: short prompts (<50 words), one task, plain text output, no complex reasoning.
  • Real limits: no long-form, no complex reasoning, ~4K token context, text only, hallucinations possible.
  • Hybrid pattern: on-device first → Gemini Flash fallback → Gemini Pro escalation. Pay cloud only when needed.
  • MediaPipe LLM Inference API for alternative models (Gemma, Phi, StableLM) on devices without AICore.
