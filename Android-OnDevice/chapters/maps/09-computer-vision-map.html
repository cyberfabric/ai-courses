The Big Idea

Computer vision is a progression: classification tells you what, detection tells you where, segmentation tells you which pixels. SSD-MobileNet at 6ms on NPU gives you real-time object detection — the workhorse of mobile vision. Combine it with the producer-consumer pattern to hit 30fps.

👁️ The Vision Task Ladder

🏷️
Classification
"What is it?" — one label for the whole image. Simplest task. MobileNetV3, 2.7 MB.
📦
Detection
"Where is it?" — bounding boxes + labels + confidence. SSD-MobileNet or YOLOv8-nano.
🎭
Segmentation
"Which pixels?" — per-pixel class masks. Powers portrait mode and background blur. DeepLabV3.
🔍
NMS (Non-Maximum Suppression)
Filters 20 overlapping boxes down to 1 per object. Sort by confidence, remove high-IoU duplicates.
🎯
Transfer Learning
Fine-tune a pre-trained model on 200-500 of your own images. TF Model Maker makes it a 5-line script.
🔄
Producer-Consumer Pattern
Camera produces frames at 30fps, inference consumes latest-only. Never queue frames — skip stale ones.

🎓 Custom Model Training — The Fast Path

  1. 📸 Collect & Label Images

    200-500 images per class for production MVP. Use LabelImg for bounding box annotation (Pascal VOC or COCO format).

  2. 🔧 TensorFlow Model Maker

    High-level Python library. Load labeled data with DataLoader.from_pascal_voc(). Five lines of code in Google Colab.

  3. 🏋️ Fine-Tune EfficientDet-Lite

    Transfer learning on a pre-trained base. 50 epochs, batch size 8. Augmentation (flip, rotate, crop) applied automatically.

  4. 📦 Export to TFLite

    model.export() — automatically quantized. Drop the .tflite file into your Android project's assets/ folder.

  5. 📱 Deploy & Benchmark

    Load with Interpreter, add NNAPI delegate, run warm-up inferences, measure p50/p95 latency on target devices.

📖 Key Terms

IoU (Intersection over Union)
Overlap ratio between two bounding boxes. 0 = no overlap, 1 = perfect match. Used as threshold in NMS (typically > 0.5).
NMS (Non-Maximum Suppression)
Algorithm that removes duplicate detections by keeping the highest-confidence box and discarding overlapping ones.
SSD (Single Shot Detector)
Detection architecture that predicts boxes and classes in a single forward pass. Fast enough for real-time mobile use.
DeepLabV3
Semantic segmentation model. Outputs a class label for every pixel. 21 PASCAL VOC classes including person, car, background.
Segmentation Mask
An integer array where each value is a class ID. Pixel at position (x, y) with value 15 = that pixel belongs to a person.
Producer-Consumer
Pattern where camera pushes frames continuously and inference grabs only the latest. Prevents frame queue buildup and OOM.
AtomicReference
Thread-safe container for the latest frame. Camera sets it, inference reads and clears it. No locks, no queues.
Normalized Coordinates
Bounding box values in [0, 1] range representing percentages of image dimensions, not pixel positions.

🧪 Quick Quiz

What's the difference between detection and segmentation?

Detection gives you bounding boxes (rectangles) around objects. Segmentation gives you per-pixel masks — the exact outline of each object. Detection is faster; segmentation is more precise.

Why do object detectors return 20 boxes for one cat?

Detectors evaluate many candidate regions at different scales. Multiple overlapping regions all score high for the same object. NMS filters these down to one box by removing any box that overlaps > 0.5 IoU with a higher-confidence box.

Why shouldn't you queue every camera frame for inference?

At 30fps you'd queue 30 frames per second. If inference takes 10ms, the queue grows faster than you can drain it — causing memory pressure, GC thrashing, and eventually OOM. Instead, use AtomicReference to always process only the latest frame.

How many labeled images do you need for a production custom detector?

200-500 images per class with data augmentation is usually enough for a production MVP (85-92% accuracy). 50-100 works for prototypes. 1,000+ for high-accuracy requirements.

Why can't you draw SSD output boxes directly on screen?

SSD outputs normalized coordinates [0, 1] relative to the model's 320×320 input. You need to scale them to the actual screen dimensions: screenX = box.left * modelWidth * (screenWidth / modelWidth). Also, the camera aspect ratio differs from the model's square input.

📌 Key Takeaways

  • Three levels of vision: classification (what?) → detection (where?) → segmentation (which pixels?). Each builds on the one before.
  • SSD-MobileNetV2 is the go-to detection model: 4.4 MB, 6ms inference on NPU, 90 COCO classes. Use NNAPI delegate.
  • NMS is essential: sort by confidence, keep the best box, discard any overlapping box with IoU > 0.5. Many TFLite models include it built-in.
  • DeepLabV3 gives you per-pixel segmentation for portrait mode and background effects. Class 15 = person.
  • Producer-consumer pattern: never queue frames. Use AtomicReference for latest-only processing. Run inference on Dispatchers.Default, push results via StateFlow.
  • Custom models are accessible: 200-500 labeled images + TF Model Maker + Colab = a production-ready custom detector in an afternoon.
