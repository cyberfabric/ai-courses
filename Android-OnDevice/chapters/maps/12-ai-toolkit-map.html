The Big Idea

Always start from the highest-level framework: ML Kit → MediaPipe → Gemini Nano → TFLite/ONNX. The further right you go, the more work you take on. Most features can ship with ML Kit or MediaPipe in an afternoon. Only reach for raw TFLite when you genuinely need a custom model.

📊 The 6 Metrics Before Shipping

🎯
Accuracy
Target >90% on 200+ labeled examples from your domain.
⚡
Latency
p95 < 33ms for real-time (30fps). Measure 100 runs, report p50/p95/p99.
📦
Model Size
<10MB bundled, <50MB downloaded. Post-quantization.
💾
Memory
<50MB runtime on mid-range. Check via Android Profiler.
🔋
Power
<5% battery per hour of continuous use. Battery Historian.
📱
Device Coverage
>80% of your user base. Check RAM/API in Play Console.

A 95%-accurate model on 30% of devices loses to an 88%-accurate model on 95% of devices. Always evaluate accuracy + coverage together.

📅 Your 30-Day Action Plan

  1. 📅 Week 1: Prototype

    Pick one feature. Build with ML Kit or MediaPipe. Get it running on your phone. Don't optimize yet.

  2. 📅 Week 2: Optimize

    Int8 quantization (Ch 8) + hardware acceleration (Ch 7). Benchmark on 2-3 devices. Target <15ms inference.

  3. 📅 Week 3: Harden

    Device capability detection, error handling, confidence thresholds, graceful degradation (Ch 11). Add telemetry.

  4. 📅 Week 4: Ship

    Deploy to beta. Monitor for one week. Fix device-specific issues. Roll out to all users. Celebrate. 🎉

🧪 Quick Quiz

You need barcode scanning in your app. Which framework do you pick?

ML Kit. It's the highest-level option for barcode scanning — production-ready, no model management, done in an hour. Don't use TFLite or MediaPipe for something ML Kit already handles perfectly.

Your model is 95% accurate but only runs on 30% of devices. Is it ready to ship?

No. A model that's 88% accurate and runs on 95% of devices is better. Always evaluate accuracy AND device coverage together. Consider quantizing to a smaller variant, or using a simpler framework that has broader device support.

How many small models can you run simultaneously on a 4GB mid-range phone?

2-3 small models (15-30MB each) comfortably, with ~2GB available RAM. But load on demand and release when the pipeline step completes. Never keep all models loaded at once — that's flagship territory.

Can you share your .tflite model file between Android and iOS?

Yes! The .tflite file is identical on both platforms. TFLite, ONNX, MediaPipe, and ML Kit all work cross-platform. Only hardware acceleration APIs differ (NNAPI/GPU on Android vs Core ML/Metal on iOS).

📌 Key Takeaways

  • Framework priority: ML Kit → MediaPipe → Gemini Nano → TFLite/ONNX. Start from the left. Most features don't need raw TFLite.
  • 6 metrics before shipping: accuracy (>90%), latency (p95 <33ms), model size (<10MB bundled), memory (<50MB), power (<5%/hr), device coverage (>80%).
  • Abstract the pattern: OnDeviceModel<I, O> wraps init → preprocess → infer → postprocess → release. Every new model is a subclass.
  • Multi-model pipelines: chain models for complex features. Load on demand, release per step. Budget memory: 15-80MB per model.
  • Cross-platform: share .tflite/.onnx/.task model files between Android and iOS. Only preprocessing code differs.
  • 30-day plan: Week 1 prototype → Week 2 optimize → Week 3 harden → Week 4 ship. You have all the knowledge — now execute.
