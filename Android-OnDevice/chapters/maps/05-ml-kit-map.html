The Big Idea

ML Kit is ordering takeout — you describe what you want and it just appears. Three lines of code for text recognition, barcode scanning, or translation. If ML Kit does what you need, you're done. Don't overthink it.

🧰 Six On-Device Features

📝
Text Recognition (OCR)
Extract text from photos with structure — blocks, lines, words, bounding boxes. Receipt scanner, business card reader.
📊
Barcode & QR Scanning
Fast, reliable scanning of QR, EAN, Code128, and more. Restrict formats for 2x speed boost.
🌍
On-Device Translation
50+ language pairs, 2-5 MB per language model, downloaded on demand. Works fully offline once cached.
💬
Smart Reply
1-3 contextual reply suggestions from conversation history. Best with 3-10 messages of context.
🔍
Entity Extraction
Detects dates, phone numbers, addresses, flight numbers, tracking numbers in unstructured text. Smart clipboard.
✍️
Digital Ink Recognition
Handwriting → text from touch strokes. 300+ languages. Note-taking, signature fields, handwriting input.

📖 Key Terms

TextBlock → Line → Element
OCR result hierarchy: paragraphs → lines of text → individual words. Each has a boundingBox.
InputImage
ML Kit's universal image wrapper. Create from MediaImage, Bitmap, Uri, or ByteBuffer.
DownloadConditions
Controls when models download — .requireWifi() to avoid burning mobile data.
RemoteModelManager
Lists, downloads, and deletes on-device language/model files. Use to manage storage.
BarcodeScannerOptions
Restrict scanned formats (FORMAT_QR_CODE, FORMAT_EAN_13) — fewer formats = faster scanning.
EntityExtractionParams
Feed raw text in, get structured annotations out — dates, phones, addresses, flight numbers.
Ink / Stroke / Point
Digital ink building blocks: collect touch points into strokes, strokes into an Ink object, then recognize.
Tasks.whenAllComplete
Run multiple ML Kit recognizers in parallel on the same frame — OCR + barcode + entity extraction.

🏗️ Building the Smart Scanner

  1. Add Dependencies

    text-recognition + barcode-scanning + entity-extraction in Gradle.

  2. Create Recognizers

    Initialize TextRecognition.getClient(), BarcodeScanning.getClient(options), and EntityExtraction.getClient().

  3. Set Up CameraX

    Preview + ImageAnalysis use case. Convert ImageProxy → InputImage with rotation.

  4. Run in Parallel

    Tasks.whenAllComplete(textTask, barcodeTask) — process OCR and barcodes on the same frame simultaneously.

  5. Extract Entities

    Feed recognized text into entity extractor — detect phone numbers, dates, addresses automatically.

  6. Display Results

    Text blocks with bounding boxes, barcode values with format labels, entities as tappable action chips.

🧪 Quick Quiz

If TFLite is cooking from scratch and MediaPipe is a meal kit, what is ML Kit?

Ordering takeout. You describe what you want (OCR, barcode, translation) and it just appears — three lines of code, zero ML knowledge needed.

How do you make barcode scanning faster?

Restrict the barcode formats with BarcodeScannerOptions. If you only need QR codes, don't scan for all 13 formats — scanning just QR is about 2x faster.

What's the quality level of ML Kit's on-device translation?

"Travel-grade," not "professional-grade." Handles everyday phrases, menus, signs well. Struggles with idioms, technical jargon, and literary text. Fall back to cloud API for critical translations.

Can you run your own custom TFLite model through ML Kit?

Yes! Use LocalModel (bundled in APK) or CustomRemoteModel (Firebase-hosted with OTA updates). You get ML Kit's download management and versioning infrastructure for free.

What's the main limitation of ML Kit on non-GMS devices?

ML Kit requires Google Play Services. Devices without GMS (some Huawei phones, Amazon Fire tablets) cannot use ML Kit at all. For those devices, fall back to raw TFLite.

📌 Key Takeaways

  • ML Kit = highest-level Android ML SDK. Check it first — if it does what you need, you're done in minutes.
  • Six on-device features: OCR, barcode scanning, translation, smart reply, entity extraction, digital ink. All free, all offline.
  • OCR returns structured results: TextBlock → Line → Element, each with a boundingBox for precise location.
  • Restrict barcode formats for 2x speed. Don't scan all 13 formats when you only need QR.
  • Translation is "travel-grade" — great for menus and signs, not for legal documents. Models are 2-5 MB each.
  • Custom models work through ML Kit too — LocalModel (in APK) or CustomRemoteModel (Firebase, OTA updates).
  • Combine features with Tasks.whenAllComplete — run OCR + barcode + entity extraction on the same frame.
  • ML Kit requires Google Play Services. No GMS = no ML Kit. Fall back to raw TFLite on those devices.
