Context Engineering for Real Projects

The Big Idea

Context engineering is mise en place for AI — every ingredient measured, prepped, and within reach before cooking starts. Know the load order, know what survives /compact, and use path-scoped rules so context costs zero until it's relevant.

🏗️ What Loads Before You Type

#ComponentTokens
1System prompt~4,200
2Auto memory MEMORY.md (200 lines / 25KB)~680
3Environment info (cwd, git, OS)~280
4MCP tool names (deferred schemas)~120
5Skill descriptions~450
6User CLAUDE.md (~/.claude/)~320
7Project CLAUDE.md~1,800

~8K tokens loaded before your first message — about 4% of a 200K window. Order is fixed; sizes vary with your setup.

💰 The Token Math

📊

Code is ~4x denser than prose.
⏱️

Stable context is cached and reused — essentially free.
📏
<5K Always-Loaded
Cap CLAUDE.md + unscoped rules under 5K tokens. Path-scoped rules cost zero until matched.

🌪️ What Survives /compact

MechanismAfter compaction
System prompt + output style✅ Unchanged (not in message history)
Project-root CLAUDE.md + unscoped rules✅ Re-injected from disk
Auto memory✅ Re-injected from disk
Path-scoped rules (paths:)❌ LOST until matching file is read again
Nested CLAUDE.md (subdirs)❌ LOST until file in that dir is read again
Invoked skill bodies⚠️ Re-injected, capped 5K/skill, 25K total
Skill descriptions (the index)❌ NOT re-injected — only invoked skills survive
HooksN/A (run as code, not context)

If a rule MUST persist across compaction, drop the paths: frontmatter or move it into project-root CLAUDE.md.

🧠 Auto Memory: Claude's Notes to Itself

🧠

The Other Memory

CLAUDE.md is what you write. Auto memory is what Claude writes between sessions — build commands, debugging patterns, corrections. Stored at ~/.claude/projects/<project>/memory/, machine-local, one dir per git repo.

📥
Only MEMORY.md Auto-Loads
First 200 lines or 25KB of MEMORY.md. Topic files (debugging.md, etc.) load on demand.
💻
Machine-Local
All worktrees of one repo share one memory dir. Not synced across machines or cloud.
⚙️
Toggle With Settings
autoMemoryEnabled, autoMemoryDirectory, or env var CLAUDE_CODE_DISABLE_AUTO_MEMORY=1.
🔢
v2.1.59+ Required
Older versions don't support auto memory. Check with claude --version.

🎯 Path-Scoped Rules (.claude/rules/)

🎯

The Condensed-Rules Contract

Max ~25 lines body. One topic. One-line rules with IDs. paths: frontmatter declares which files trigger the rule. Zero startup cost — loads only when Claude reads a matching file.

📄 .claude/rules/<topic>.md
~25 lines, paths: glob
Auto-loads on match

Loaded by /run-review

🚨 The 5 Context Polluters

📅
Outdated Rules
CLAUDE.md says "use LiveData" but you migrated. AI follows old rule confidently.
⚔️
Conflicting Rules
Root says "sealed class" but a rule file says "data class." AI merges both into Frankenstein code. Most dangerous.
🗑️
Irrelevant Context
Backend API docs for a UI task. Wastes tokens and causes over-engineering.
📚
Verbose Documentation
5-page ADR for a bug fix. Actual rules get buried under narrative.
🏚️
Legacy Patterns
Old code examples using deprecated approaches. AI treats them as patterns to follow.

🔍 The Context Audit (Quarterly)

  1. List Auto-Loaded Files

    CLAUDE.md, unscoped rules, auto memory MEMORY.md, ARCHITECTURE.md.

  2. Check Accuracy

    For each file: "Is every statement still accurate?"

  3. Detect Conflicts

    Same topic, different rules across files? Fix immediately.

  4. Prune & Condense

    Remove outdated refs, condense verbose sections, delete dead files.

  5. Run /context

    Confirm utilization <80% before starting heavy work. Adjust if memory or rules are eating too much.

📖 Key Terms

Context Engineering
Project-level discipline of controlling what AI reads, in what format, and at what size.
Path-Scoped Rule
A markdown file in .claude/rules/ with paths: frontmatter. Loads automatically when Claude reads a matching file. Zero startup cost.
Dual-Format Pattern
Each topic has two files: a condensed path-scoped rule (for development) and a verbose review checklist (for review).
Auto Memory
Notes Claude writes for itself between sessions, in ~/.claude/projects/<project>/memory/. Machine-local, repo-scoped, first 200 lines of MEMORY.md auto-load.
/context
Slash command that breaks down current context window usage by category. Use to spot hot zones before heavy tasks.
/compact survival
What carries through compaction: system, root CLAUDE.md, auto memory, invoked skill bodies. Lost: path-scoped rules, nested CLAUDE.md, skill index.
Subagent Context Isolation
Each subagent runs in its own window. File reads stay in the subagent's context; only the summary returns to the main session.
Context Pollution
When AI reads information that actively hurts output — outdated, conflicting, irrelevant, or verbose.
Mise en Place
Chef's prep analogy: every ingredient measured and ready before cooking starts = every context file prepped before AI codes.

🧪 Quick Quiz

You added a rule but Claude ignores it. What do you check first?

Run /memory. It lists every CLAUDE.md, CLAUDE.local.md, and rules file currently loaded. If your file isn't there, Claude can't see it.

What's the most dangerous type of context pollution?

Conflicting rules — where CLAUDE.md and a path-scoped rule disagree. AI tries to satisfy both, producing Frankenstein code that follows neither pattern.

A rule must always apply, even after /compact. Where does it go?

In project-root CLAUDE.md or as an unscoped rule (no paths: frontmatter). Path-scoped rules and nested CLAUDE.md are LOST after compact until a matching file is re-read.

You need to research a large codebase. How do you avoid blowing your context?

Delegate to a subagent. Its file reads stay in its own separate window — you get back only a 400-token summary, not the 6,000+ tokens of files it had to read.

How is auto memory different from CLAUDE.md?

CLAUDE.md is what you write (team-shared, in git). Auto memory is what Claude writes for itself across sessions (machine-local, gitignored by default location). Different audiences, different lifecycles.

What gets loaded before your first prompt, and how much does it cost?

~8K tokens: system prompt, auto memory MEMORY.md, environment, MCP names, skill descriptions, user CLAUDE.md, project CLAUDE.md. About 4% of a 200K window.

📌 Key Takeaways

  • Context engineering = mise en place: prep everything before AI starts coding.
  • ~8K tokens load before your first prompt (system, memory, MCP names, skills index, CLAUDE.md). Order is fixed.
  • Path-scoped rules in .claude/rules/ cost zero until matched — chapter 7 covers them in depth.
  • Dual-format: condensed rule (paths:, ~25 lines) + review checklist (verbose, on demand).
  • /compact survival matters: root CLAUDE.md and unscoped rules survive; path-scoped rules and nested CLAUDE.md don't.
  • Auto memory is Claude's own notes between sessions — review with /memory, prune what's stale.
  • Diagnostics: /context for live breakdown, /memory for what's loaded, subagents to keep large reads out of your window.
  • Conflicting rules are the deadliest polluter — one source of truth, multiple formats.
