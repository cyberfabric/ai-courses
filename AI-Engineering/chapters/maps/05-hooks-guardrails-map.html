Hooks and Guardrails: Automated Quality Gates

Memory Map

The Big Idea

Skills define what AI should do. Hooks enforce what AI must NOT do. Together — proactive guidance plus reactive safety — they form a complete quality system. Hooks are CI/CD checks running locally on every AI action.

🛡️ PreToolUse: The Safety Gates

💣
Dangerous Command Blocker
Blocks rm -rf, DROP TABLE, force push to main, git reset --hard, chmod 777. Exit 1 = action blocked.
🔒
Sensitive File Protector
Blocks writes to .env, credentials, .pem/.key, google-services.json, CI workflows.

PreToolUse hooks run before the action. Exit code 1 = block entirely. Your first line of defense.

✨ PostToolUse: Automatic Quality

🎨
Auto-Formatter
Runs ktlint/prettier/black per file after Write or Edit. Always exit 0 — format issues don't block.
🧪
Module Test Runner
Runs affected module's unit tests after edits. Exit 1 = warns AI, which self-corrects in same session.
👁️
Subagent Reporter
Counts changed/added/deleted files. Flags >20 file changes or infrastructure file modifications.

PostToolUse hooks run after the action. Exit 1 = feedback to AI (self-correct), not blocking.

🔧 Hook Debugging: Top 4 Failures

  1. Missing Execute Permission

    Most common failure. Fix: chmod +x .claude/hooks/*.sh

  2. Wrong jq Path for Input

    Hooks receive JSON via stdin. Test: echo '{"command":"test"}' | bash .claude/hooks/your-hook.sh

  3. Matcher Doesn't Match Tool Name

    Use exact capitalized names: Bash, Write, Edit. Not lowercase. Pipe-separate combos: Write|Edit.

  4. Overly Broad Pattern Matching

    Hook blocks legitimate operations. Narrow the matcher or refine the pattern list. Test with real inputs before deploying.

Golden rule: Never disable hooks to "get past" a block. Fix the hook's pattern — don't remove the safety net.

⚙️ Hook Configuration Anatomy

📋 Event (PreToolUse / PostToolUse / SubagentStop)
🎯 Matcher (Bash, Write|Edit, or empty = all)
🔧 .claude/settings.json
📜 Hook Script (.claude/hooks/*.sh)
📥 Input (JSON via stdin, parse with jq)

Hook reads tool input via stdin as JSON. Output goes to stderr (>&2).
Exit code determines action: 0 = allow/pass, 1 = block (PreToolUse) or warn (PostToolUse).

Pro tip: Document hooks in CLAUDE.md — AI reads "blocks destructive commands" and proactively avoids triggering them. The documentation becomes a teaching tool, not just enforcement.

📖 Key Terms

Hook
Shell command that Claude Code runs automatically at specific moments — before, after, or on agent completion.
PreToolUse
Runs before a tool executes. Exit 1 blocks the action entirely. Your safety gate.
PostToolUse
Runs after a tool completes. Exit 1 provides feedback to AI for self-correction.
SubagentStop
Runs when a background agent completes. Provides oversight for unattended work.
Matcher
Filter that determines which tool triggers a hook: Bash, Write, Edit, or pipe-separated combos.
Layered Defense
Hooks (fast, local) + CI/CD (thorough, remote) = issues caught at the earliest possible moment.

🧪 Quick Quiz

What's the difference between PreToolUse and PostToolUse exit codes?

PreToolUse exit 1 = blocks the action entirely (before it happens). PostToolUse exit 1 = signals feedback to AI for self-correction (action already completed).

Why implement the dangerous command blocker before the auto-formatter?

Prioritize by damage potential. Missing formatter = messy code you can clean up. Missing command blocker = deleted databases you can't recover.

Why should PostToolUse test hooks only run for the affected module?

Performance: module tests take 10 seconds, full suite takes 3 minutes. Full suite per-edit kills your workflow. Scope tests to the modified module.

Why document hooks in CLAUDE.md?

It teaches AI about the safety net proactively. When AI reads "blocks destructive commands," it avoids triggering them. Documentation becomes a teaching tool, not just enforcement.

What are the 5 essential hooks every project should have?

1) Dangerous command blocker, 2) Sensitive file protector, 3) Auto-formatter, 4) Module test runner, 5) Subagent change reporter.

📌 Key Takeaways

  • Skills = what to do. Hooks = what AI can't get away with. Both needed for production quality.
  • 3 hook events: PreToolUse (block before), PostToolUse (warn after), SubagentStop (agent oversight).
  • Prioritize hooks by damage potential: critical safety gates first, quality automation second.
  • 5 essential hooks: command blocker, file protector, formatter, test runner, subagent reporter.
  • Hooks + CI/CD = layered defense. Hooks catch fast and local; CI catches thorough and remote.
  • Document hooks in CLAUDE.md — AI reads it and proactively avoids triggering blocked patterns.
  • Never disable hooks to "get past" a block. Fix the pattern, don't remove the safety net.
