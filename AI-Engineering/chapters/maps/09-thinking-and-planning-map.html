Chapter 9

Extended Thinking and Plan Mode

Memory Map

💡 The Big Idea

🧠

Thinking Is for Decisions, Not Execution

Extended thinking earns its cost when AI faces trade-offs and unknowns. If the decision is already made — by a rule in .claude/rules/, a skill template, or a previous plan — skip the thinking and just do the work.

✅

⏭️
Skip Thinking

Rule of thumb: One file + clear pattern = skip. Multiple files + trade-offs = think.

🚫 Anti-Patterns

🐌
Overthinking Simple Edits
20 seconds of "thinking" for a one-line fix. Just execute directly.
📄
Planning Boilerplate
Plan says "create file, add imports, add class." Use a scaffold skill instead.
🔁
Re-planning Known Patterns
Planning how to write a ViewModel when .claude/rules/ already defines it. The rule file IS the plan.
👥
Planning in Parallel
Multiple devs plan the same refactor. One plan, shared via PR, executed collaboratively.

📋 Key Terms

Extended Thinking
Private reasoning scratchpad before generating a response. Costs more, produces higher quality on complex tasks.
Plan Mode
Multi-step workflow: create a plan, get human approval, then execute. For high-risk operations.
Plan-Execute-Validate
Three-phase pattern: AI plans, human approves, AI executes, then validates with tests and rules.
ADR (Architecture Decision Record)
Document capturing an architecture decision, alternatives considered, and consequences. AI context for future work.
Thinking Budget
The token cost of extended thinking. 3–5x standard cost. Encode task categories to control spending.
"DO NOT write code yet"
Critical boundary instruction in plan-mode skills. Without it, AI tends to skip planning and jump to execution.

❓ Quick Quiz

Test Yourself

What's the difference between extended thinking and plan mode?

Extended thinking is a private scratchpad within a single response — the AI reasons before answering. Plan mode is a multi-step workflow — AI creates a plan, human reviews it, then AI executes. One is automatic reasoning, the other is a collaborative process.

Which phase of plan-execute-validate benefits most from extended thinking?

The Plan phase. That's where trade-offs are weighed and dependencies discovered. Execute follows the plan (straightforward), and Validate is mechanical (tests, linting).

When should you NOT use extended thinking?

Single-file edits, formatting/linting, boilerplate scaffolding, test generation, and any task where the pattern is already defined by a rule in .claude/rules/ or a skill template. The decision is already made — just execute.

Why do plan-mode skills need "DO NOT write code yet"?

Without an explicit boundary, AI tends to skip planning and jump straight to execution. The instruction forces a pause between the planning and execution phases, giving the human a chance to review and approve.

What's the ROI math for extended thinking?

3–5x token cost, but 60–80% reduction in rework on complex tasks. If thinking prevents a 45-minute cleanup, the extra tokens are a bargain. On simple tasks, it's pure waste.

📌 Key Takeaways

If You Remember Nothing Else

  • Extended thinking = private scratchpad for complex reasoning. Plan mode = collaborative workflow with human approval gates.
  • Think for decisions, skip for execution. Multiple files + trade-offs = think. Single file + clear pattern = just do it.
  • Plan-Execute-Validate is the pattern: plan with thinking, execute without, validate with hooks and tests.
  • Encode in CLAUDE.md which tasks need thinking and which don't. Don't leave it to individual judgment.
  • "DO NOT write code yet" is the most important line in any plan-mode skill. Without it, AI skips planning.
  • 3–5x cost, 60–80% less rework. The math works for complex tasks. It doesn't for simple ones. Configure the boundary.
