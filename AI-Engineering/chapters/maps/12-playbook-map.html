💡 The Big Idea

📖

Configuration over Conversation

Every rule in CLAUDE.md prevents a hundred correction prompts. Every path-scoped rule in .claude/rules/ prevents wrong patterns. Every skill prevents rework. Invest in configuration — it compounds with every AI interaction.

📋 The 5-Pillar Checklist (Condensed)

📄
P1: CLAUDE.md
Root + directory-level files. 5+ NEVER rules. Build commands. Naming conventions. Stable-first ordering. Monthly review.
🧠
P2: Context
ARCHITECTURE.md. Path-scoped rules in .claude/rules/ (~25 lines body each). Auto memory reviewed. /context habit.
⚡
P3: Skills & Hooks
scaffold + review + test skills. Safety hooks (dangerous commands, sensitive files). Plan-mode skills for high-risk ops.
📐
P4: Rules
MUST/SHOULD/PREFER severity. Rule IDs. 5+ of 10 areas. /check-architecture skill. Updated during migrations.
👥
P5: Team
Onboarding checklist. MCP with ${ENV_VAR}. Adoption metrics. 50%+ contributing skills/rules. Cost monitoring.

✊ The Manifesto

1
Configuration > Conversation
Every CLAUDE.md rule prevents a hundred correction prompts.
2
Prevention > Review
A path-scoped rule that guides generation beats a review that catches mistakes.
3
Shared > Personal
Config in the repo benefits everyone. Config on your laptop benefits only you.
4
Specific > Vague
"NEVER call repos from composables" beats "follow Clean Architecture."
5
Maintained > Created
Stale config is worse than no config. Monthly reviews are non-negotiable.
6
Balanced > Deep
Score 3 across all 5 pillars before scoring 4 on any single one.
7
Evidence > Opinion
Track metrics. Run audits. Let the numbers guide investment.
8
Context > Prompts
Rich cached context produces correct output. No prompt engineering compensates for missing rules.

❓ Quick Quiz

Test Yourself

What are the 8 principles of the AI-Ready Manifesto?

Configuration > Conversation. Prevention > Review. Shared > Personal. Specific > Vague. Maintained > Created. Balanced > Deep. Evidence > Opinion. Context > Prompts.

What's the maintenance cadence?

Weekly: sprint context + hook check (15 min). Monthly: CLAUDE.md + .claude/rules/ + skills + auto memory review (1 hr). Quarterly: full audit + MCP review (1.5 hr). On migration: update rules immediately (10 min).

What score should you target at Week 8 of the 90-day plan?

15+ out of 20. Week 8 is the midpoint audit. If below 12, focus remaining weeks on filling gaps rather than adding advanced features. Depth in basics beats breadth in advanced topics.

What's the most dangerous configuration state?

Outdated configuration — worse than no configuration. Stale rules give AI wrong instructions, producing code that looks correct but violates your current architecture. Monthly reviews prevent this drift.

What's the first thing to create when starting from zero?

CLAUDE.md with 5 NEVER rules. It's the single highest-impact artifact. It gets cached, applies to every interaction, and immediately constrains AI from your project's most common mistakes.

📌 Key Takeaways

If You Remember Nothing Else

  • 5 pillars, 1 checklist. CLAUDE.md, Context, Skills/Hooks, Architecture Rules, Team Config. Check what you have, build what you don't.
  • Maintain weekly, review monthly, audit quarterly. Configuration drifts. Stale config is worse than no config. Put it on the calendar.
  • 90 days from zero to AI-native. Foundation (W1-2) → Skills (W3-4) → Rules & Team (W5-8) → Maturity (W9-12).
  • Start with CLAUDE.md + 5 NEVER rules. It's the single highest-impact artifact. Everything else builds on it.
  • Configuration > Conversation. Every rule prevents correction prompts. Every path-scoped rule in .claude/rules/ prevents wrong patterns. Invest in config — it compounds.
  • This course → AI-Rewrite (transform existing code) → AI-Factory (build from scratch). Your AI-ready project is a platform for everything that follows.
