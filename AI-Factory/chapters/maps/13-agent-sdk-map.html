Running the Factory Programmatically — The Claude Agent SDK

Memory Map
💡

The Big Idea

The Factory you built interactively in Claude Code runs unchanged through the Claude Agent SDK — same agent loop, same skills, same sub-agents, just headless. Use Claude Code for design iteration, the SDK for CI/cron/webhook execution, and share one .claude/ directory between them.

⚖️ Claude Code (CLI) vs Agent SDK

Aspect Claude Code (CLI) Claude Agent SDK
Surface Terminal session Library (Python / TypeScript)
Best for Interactive work, design iteration CI/CD, batch jobs, production automation
Skills .claude/skills/ Same files, auto-loaded
Sub-agents .claude/agents/ Same files, OR programmatic AgentDefinition
Auth Browser login or API key API key only (or Bedrock/Vertex/Foundry)

Same engine, different surfaces. Workflows translate directly — no rewrite required.

🤖 Three Headless Patterns

  1. CI Pipeline

    GitHub Actions, GitLab CI, Jenkins. Trigger on push, scheduled cron, or release branch. Skills directory ships with the repo.

  2. Cron / Scheduled

    Daily idea-to-prototype runs. Pull seed from Linear or backlog file, spin up a prototype branch overnight.

  3. Webhook

    HTTP service wraps orchestrate.py. POST a payload (idea, platform, budget) and the service runs the pipeline.

⚠️ Headless = no humans in the loop. Re-add approval gates as code.

🤝 Hybrid Workflow

🛠️
CLI for Design
Architect-stage iteration runs in Claude Code. Humans iterate on rules with interactive feedback.
⚙️
SDK for Build
Once architecture is locked, dev/review/QA loop runs overnight in CI via the SDK.
📂

A skill update made interactively is live for the next CI run. No library version drift.
📨
Approvals via Slack/PR
SDK hooks post intermediate artifacts to a channel; humans approve, the loop continues.

🔒 Security Caveats: Re-add the Humans

  1. After Architect (end Phase 2)

    Ship architecture.md + rules.md to a review channel. Block until human responds.

  2. After Security verdict (Phase 4)

    If any HIGH/CRITICAL finding exists, do NOT auto-merge. Send to security oncall.

  3. Before deploy / external API call

    Use permission_mode: "acceptEdits" — auto-approves edits, prompts for shell commands.

  4. Tighten allowed_tools

    Strip WebFetch/WebSearch from a build pipeline. OWASP LLM06 = Excessive Agency (see Ch 10).

📖 Key Terms

Claude Agent SDK
The agent loop, tools, sub-agents, hooks, skills, and MCP support that power Claude Code, packaged as a Python and TypeScript library. Renamed in 2026 from "Claude Code SDK".
query()
Async iterator entry point. Streams messages (tool calls, results, reasoning, final response) as Claude works through the agent loop.
ClaudeAgentOptions
Configuration: allowed_tools, model, cwd, setting_sources, hooks, agents, mcp_servers, permission_mode.
AgentDefinition
Programmatic sub-agent definition (description + prompt + tools). Alternative to filesystem .claude/agents/<name>.md.
Hooks
Callbacks at PreToolUse / PostToolUse / Stop / Session events. Used for budget enforcement, audit logging, validation, blocking.
Headless mode
Non-interactive execution — CLI's --print flag or SDK's query(). No human in the loop unless explicitly wired.
Approval gate
A human-decision point re-added as code (Slack prompt, PR review, stdin block) to compensate for losing the CLI's interactive checkpoints.
Three-lane model strategy
From Ch 12: Opus 4.7 (judgment), Sonnet 4.6 (pattern-following), Haiku 4.5 (high-volume). The SDK's model option picks the lane.

🎓 CLI vs SDK Decision

Stay on CLI when… Move to SDK when…
Pipeline runs < once a week Pipeline runs daily or per push
Each run is bespoke Same skills run across projects/clients
Human in the loop is desired Cost ceilings + audit logs needed
Team is 1–2 engineers Factory is part of a product

Graduation is gradual. Most teams keep CLI for design and switch build/review/QA to SDK once the pattern stabilizes.

🧪 Quick Quiz

What does the SDK auto-load from .claude/?

Skills (.claude/skills/<name>/SKILL.md), sub-agents (.claude/agents/<name>.md), legacy slash commands (.claude/commands/<name>.md, merged with skills), and CLAUDE.md memory. Restrict scope with setting_sources=["project"] for CI safety.

Why does the Factory's CLI work need zero rewrites for the SDK?

The CLI and SDK share an engine — the same agent loop, tools, hooks, and skill loader. The SDK reads the same .claude/ directory the CLI does. The 7 role skills you wrote in chapters 4–11 run unchanged through query().

What three checkpoints disappear in headless mode?

(1) The Architect-review pause after Phase 2; (2) the Code Reviewer's interactive block-then-confirm; (3) the Security Reviewer's escalation prompt. Re-add as code: Slack/PR posts, stdin gates, or a Stop-hook that emails a human.

How does the SDK enforce a per-pipeline budget cap?

PostToolUse hook tracks each call's cost into a running total. PreToolUse hook checks the total before the next call and returns {decision: "block"} when the cap is exceeded. The pipeline halts cleanly with the cap as the reason.

When should you stay on the CLI rather than move to the SDK?

When the pipeline runs less than once a week, each run is bespoke, you want a human in the design loop, and the team is one or two engineers. The interactive cost is a rounding error and the CLI's safety rails are free.

📌 Key Takeaways

  • Same engine, two surfaces: Claude Code (CLI) and the Agent SDK share an agent loop. Workflows translate directly.
  • Zero-rewrite migration: SDK auto-loads .claude/skills/, .claude/agents/, and CLAUDE.md from your working directory.
  • Install: pip install claude-agent-sdk or npm install @anthropic-ai/claude-agent-sdk. Opus 4.7 needs v0.2.111+.
  • Three headless patterns: CI, cron, webhook. Bring the Factory wherever a script can run.
  • Self-governing cost: PreToolUse / PostToolUse hooks enforce per-pipeline budget caps. The Factory reports its own spend.
  • Hybrid is the norm: CLI for design iteration, SDK for build/review/QA. Both share .claude/.
  • Re-add humans as code: Slack hooks, PR gates, stdin pauses. Headless ≠ unsafe if you wire approvals.
  • The end-state: a 7-role pipeline, a script that runs it, and a ledger that proves what it cost.
