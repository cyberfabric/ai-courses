Chapter 1

The AI Factory: Roles, Not Prompts

Memory Map
💡

The Big Idea

Stop prompting AI and start hiring it. Give each AI invocation a specialized role with a defined contract — inputs, outputs, and quality criteria — then orchestrate them as a pipeline.

🧩 Core Concepts

🏭
AI Factory
A reproducible AI development pipeline with formalized roles, structured handoffs, and quality gates.
🎭
Roles, Not Prompts
Each AI invocation gets a specific job title, defined inputs/outputs, and quality criteria — not a vague request.
📝
Role Contract
The interface every role implements: what it receives, what it must produce, and what "done" means.
⚡
Skills as Roles
Roles are implemented as Claude Code skill files in .claude/commands/ — versioned, shareable, invokable by name.
🎯
Orchestration
The Agent tool dispatches to specialized subagents in sequence, passing artifacts between them automatically.
🔗
Artifact Handoffs
Each role's output becomes the next role's input: idea → vision → specs → design → code → review → tests.

👥 Roles at a Glance

  1. Product Owner

    Idea → vision document, personas, MoSCoW priorities, release roadmap

  2. Product Manager

    Vision → feature specs, acceptance criteria, edge cases

  3. Architect

    Specs → module graph, data models, architecture rules

  4. Developer

    Design + rules → compilable Kotlin, passing unit tests

  5. Code Reviewer

    Code + rules → structured findings with severity and fixes

  6. Security Reviewer

    Code → OWASP findings, secret detection, remediation steps

  7. QA Engineer

    Code + specs → test suite, coverage report, edge case matrix

⚖️ Ad-Hoc Prompting vs. Role-Based Pipeline

Aspect ❌ Ad-Hoc Prompting ✅ Role-Based Pipeline
Consistency Different output every run Same quality every time
Reproducibility Lives in your clipboard Git-versioned skill files
Quality gates None — hope for the best Each role has pass/fail criteria
Improvement Rewrite the whole prompt Fix one role independently
Auditability No trace of how output was made Every artifact has a source role
Team sharing Tribal knowledge Shared skill library in repo

📖 Key Terms

Role
A specialized AI identity with one job, defined inputs/outputs, and quality criteria.
Contract
The interface a role implements: inputs it receives, outputs it produces, criteria for "done."
Skill
A markdown file in .claude/commands/ that implements a role — invokable by name (e.g., /product-owner).
Artifact
A structured output from a role (vision doc, specs, code, etc.) that becomes input for the next role.
Orchestrator
A master skill that dispatches to role skills in sequence, passing artifacts between them.
Agent Tool
Claude Code's mechanism for dispatching work to isolated subagents — each gets only the context it needs.
Pipeline
The full sequence of roles from seed idea to tested, reviewed code.
Handoff
The structured passing of one role's output as the next role's input.

🧪 Quick Quiz

What are the three parts of a role contract?

Inputs (what the role receives), Outputs (what it must produce), and Quality Criteria (what "done" means).

Why do skills beat prompts?

Skills are files — version-controlled, shareable, invokable by name. Prompts live in your clipboard and die when you close the tab.

What does the Agent tool do in the pipeline?

It dispatches work to specialized subagents in isolation. Each role gets only the context it needs and returns a structured artifact.

Why is isolation between roles important?

It prevents roles from overstepping their boundaries. The PO doesn't see code, the Developer doesn't see brainstorming — each stays in their lane.

What's the one-sentence seed for the course's running example?

"I want an offline-first Android app for managing recipes, organizing them into books, and generating grocery lists."

📌 Key Takeaways

  • Roles, not prompts: Give each AI invocation a specialized identity with a defined contract.
  • 7-role pipeline: PO → PM → Architect → Developer → Reviewer → Security → QA.
  • Contract = Interface: Every role defines inputs, outputs, and quality criteria.
  • Skills = Infrastructure: Roles are markdown files in .claude/commands/, version-controlled and shareable.
  • Agent tool = Dispatcher: Orchestrator dispatches to specialists, passing artifacts between them.
  • One sentence → full app: The pipeline transforms a seed idea into reviewed, tested production code.
