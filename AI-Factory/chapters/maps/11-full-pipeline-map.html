The Full Pipeline: All Roles in Concert

Memory Map
💡

The Big Idea

Seven roles, four phases, three human checkpoints. The Orchestrator coordinates everything autonomously between checkpoints — you spend ~17 minutes of active time while the pipeline produces a reviewed, secured, tested codebase in ~90 minutes for ~$6.50 on Opus 4.7 + Sonnet 4.6 (down from ~$9.20 on the older Opus rate card).

🚀 Ship Readiness

  1. Phase 1: Discovery (~$0.10 Opus 4.7, ~6 min)

    PO generates vision → Human checkpoint → PM writes specs (15 stories, 39 behaviors, 33 edge cases).

  2. Phase 2: Design (~$0.13 Opus 4.7, ~6 min)

    Architect designs 10 modules + 7 rules → Human checkpoint → Planner decomposes into 20 tasks.

  3. Phase 3: Build (~$6.20, ~60 min)

    4 sprints: Developer generates → Code Reviewer checks → Security Reviewer scans → QA generates tests. Repeat per task.

  4. Phase 4: Ship Readiness (~2 min)

    Full compile, 128 tests pass, 0 warnings, 0 TODOs → Human checkpoint (ship decision).

👤 Three Human Checkpoints

1️⃣
After PO Vision (~2 min)
Is this the right product? Right priorities? Remove features that don't belong in MVP. Prevents building the wrong thing for $9.
2️⃣
After Architect Design (~5 min)
Sound module structure? Correct dependencies? Wrong architecture is the most expensive mistake. Fix here, not during Sprint 3.
3️⃣
Ship Readiness (~10 min)
Review the report, spot-check files, run the app. Pipeline ensures correctness; human ensures quality of experience.

Total human time: ~17 minutes out of 90 minutes pipeline execution.

📖 Key Terms

Orchestrator
The master skill that coordinates all 7 roles, validates output contracts, and routes decisions (advance, re-dispatch, escalate).
Human Checkpoint
Pipeline pauses for human judgment at 3 points: after PO vision, after Architect design, and at ship readiness.
Ship Readiness Report
Final pipeline output: all reviews passed, all tests pass, 0 warnings, 0 TODOs. Human reviews the report and makes the ship decision.
Error Recovery
Re-dispatch with structured findings (max 2 cycles). If still failing, escalate to human with full context.
Phase 3 Parallelism
Sprint 3 features have no cross-dependencies. All 8 tasks (4 ViewModels + 4 screens) can be dispatched simultaneously.
The 80/20 Split
The pipeline replaces ~80% of mechanical work. Humans focus on the ~20% requiring judgment: product, UX, performance, ship decision.

🧪 Quick Quiz

Why does the orchestrator pause after the PO vision but not after the PM specs?

The PO vision sets the direction — wrong direction means building the wrong product for $9. PM specs are derived from the approved vision; if the vision is right, specs are unlikely to be fundamentally wrong. Save human time for the highest-impact decisions.

What would happen without the quality stack in the RecipeVault run?

The type converter bug in Task 7 would cascade into compile failures in Tasks 9-16 (all features depend on the database). That's 8 failing tasks instead of 1 re-dispatch. Under 2026 pricing the quality stack costs ~$3.80 (Code Review + Security + QA), but the saved rework is still ~$10-15 because the cascade pattern is independent of pricing.

Why is the Build phase ~95% of total cost?

Discovery and Design run once each (4 dispatches, ~$0.23 total). Build runs 20+ Developer dispatches, 16 Code Reviews, 4 Security Reviews, and 20 QA dispatches. Volume drives cost. With Opus 4.7 making review cheap and Sonnet 4.6 / Haiku 4.5 driving Developer + QA costs down, the per-dispatch cost is small but the dispatch count is large.

How much human time does the full pipeline require?

~17 minutes: ~2 min reviewing PO vision, ~5 min reviewing architecture, ~10 min for ship readiness (review report, spot-check files, run the app). Out of 90 minutes total, the rest runs autonomously.

What can't the pipeline catch that a human must verify?

Product judgment (do features matter?), UX quality (does it feel right?), runtime performance (needs profiling), business edge cases ("0 servings" makes no sense), and end-to-end flows across features.

📌 Key Takeaways

  • 4 phases: Discovery → Design → Build (4 sprints) → Ship Readiness.
  • 3 checkpoints: After PO vision, after Architect design, at ship readiness. ~17 min human time.
  • ~$6.50 total (Opus 4.7 + Sonnet 4.6): Discovery ~$0.10 + Design ~$0.13 + Build ~$6.20 (Dev $2.40, Code Review $1.40, Security $0.40, QA $2.00). Drops to ~$5.10 if QA runs on Haiku 4.5. Down from ~$9.20 on the older Opus rate card.
  • ~90 minutes: From one sentence to reviewed, secured, tested code. Human equivalent: 2-4 days.
  • 128 tests: 48 Developer + 80 QA. 92% acceptance criteria coverage.
  • All failures recovered: 6 issues caught by quality roles, all fixed in 1 re-dispatch cycle.
  • 80/20 split: Pipeline handles mechanical work. Human handles judgment: product, UX, performance, ship.
