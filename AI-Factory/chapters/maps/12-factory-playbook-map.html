The Big Idea

RecipeVault was the demo. The real value is the factory itself — portable skills, structured contracts, and orchestration patterns you can adapt to any platform, scale to any team size, and share as a team asset. Under Opus 4.7 pricing, the model-assignment decision shifted from cost-driven to judgment-vs-mechanical.

📏 Three Pipeline Sizes

👤
Solo Dev (3 roles, ~$2-3)
PO + Architect + Developer. Skip PM, Reviewer, Security, QA. Personal projects where manual review is fine.
👥
Small Team (5 roles, ~$5-7)
PO + PM + Architect + Developer + Code Reviewer. Team does manual security review and thorough testing.
🏢
Enterprise (7+ roles, ~$12-18)
All 7 core roles + custom roles (Designer, DevOps, Tech Writer, Perf Engineer). Full quality stack + documentation.

⚠️ Never skip the Architect. Without rules, 20 Developer dispatches produce 20 inconsistent modules.

🧩 Adding Custom Roles

  1. Designer

    After PM, before Architect. Wireframes, color scheme, component specs.

  2. DevOps

    After Architect. CI/CD config, Dockerfile, deploy scripts.

  3. Tech Writer

    After QA. README, API docs, contributing guide.

  4. Performance Engineer

    After QA. Optimization plan, benchmarks from profiling data.

💰 Cost & Time Levers (2026)

The 2026 reality: Opus 4.7's price drop made wall-clock time the bigger lever, not dollars.

⚡ Three-Lane Model Strategy

🧠
Opus 4.7 (Judgment)
Architect, Code Reviewer, Security Reviewer, Orchestrator. Deep reasoning for tradeoffs, rule interpretation, severity decisions. $5/$25 per MTok.
⚡
Sonnet 4.6 (Pattern-Following)
PO, PM, Planner, Developer, QA Engineer. Template-following, pattern-matching, daily-driver. $3/$15 per MTok. 79.6% SWE-bench.
🚀
Haiku 4.5 (High-Volume)
Mechanical Developer tasks (entities, fakes, DAOs), high-volume QA test generation. $1/$5 per MTok. 4–5× faster, 73.3% SWE-bench.

Axis is judgment vs mechanical, not cost. Opus and Sonnet are now near cost-parity once Opus 4.7's larger tokenizer is factored in.

🧪 Quick Quiz

What three things change to adapt the pipeline to iOS?

Architect rules (SwiftData, @Observable, NavigationStack), Developer context (Swift, SPM), and Security checks (Keychain, App Transport Security). PO, PM, Planner, Reviewer structure, QA strategy, and Orchestrator stay identical. 2026 alternative: use Compose Multiplatform and share 80–96% of the UI code with Android instead of duplicating skills.

Why is the Architect the only non-negotiable role?

The Architect produces architecture rules. Without rules, 20 Developer dispatches make their own style decisions, producing 20 modules that look like 20 different codebases. Rules are what keep the factory's output consistent.

Where do skills, sub-agents, and slash commands live in 2026?

.claude/skills/<name>/SKILL.md for skills (auto-loaded or /invoke). .claude/agents/<name>.md for sub-agents (delegated, isolated context). .claude/commands/<name>.md for legacy slash commands — they still work and are merged with skills, but new roles should go in skills/. The Factory uses skills for the 7 generative roles and sub-agents for the 2 reviewers.

How did Opus 4.7 pricing change the model-selection axis?

Opus 4.7 dropped to $5/$25 per MTok (down from $15/$75) — roughly cost-parity with Sonnet 4.6 ($3/$15) once Opus 4.7's larger tokenizer is factored in. The decision is now judgment vs mechanical, not cost. Use Opus where you need genuine reasoning, Sonnet for pattern-following, Haiku 4.5 for high-volume mechanical work.

What's the recommended way to start using the pipeline?

Start with Discovery + Design (Phase 1-2). Validate the architecture at checkpoint 2. Then run one sprint of Phase 3. Tune skills based on results. Scale to full 4-sprint execution only after Sprint 1 is solid.

📌 Key Takeaways

  • Platform-agnostic structure: Change Architect rules + Developer context + Security checks. Everything else transfers. Compose Multiplatform now collapses Android+iOS into one stack.
  • Three sizes: Solo (3 roles), Small Team (5 roles), Enterprise (7+ roles). Never skip the Architect.
  • Custom roles: Designer, DevOps, Tech Writer, Perf Engineer. Same contract pattern: input → process → output.
  • Three-lane model strategy: Opus 4.7 (judgment), Sonnet 4.6 (pattern-following), Haiku 4.5 (high-volume). Axis is task type, not cost.
  • Three locations in .claude/: skills/ (auto-loaded), agents/ (delegated, isolated), commands/ (legacy, merged with skills).
  • Time, not dollars: Opus 4.7's price drop made wall-clock the bigger optimization lever.
  • Start small: Discovery + Design first, then one sprint, then scale. Build confidence before committing to full execution.
  • The 80/20 split: Factory handles mechanical work. You handle product vision, UX, performance, and the ship call.
