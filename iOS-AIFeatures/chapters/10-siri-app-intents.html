Chapter 10

Siri, App Intents & System Integration

Make your AI features accessible everywhere — Siri, Shortcuts, Spotlight, widgets, and the Lock Screen

You’ve built amazing AI features. They summarize text, understand images, transcribe speech, and generate creative content. But here’s the problem: they only work when the user opens your app.

That’s like building a great restaurant and hiding it in a basement with no sign. The food is incredible — but nobody can find it.

System integration is the sign, the menu in the window, and the delivery service. It makes your AI features discoverable and accessible from anywhere on iOS — Siri, Shortcuts, Spotlight, widgets, the Lock Screen, and even the Action button.

The Multiplier Effect

An AI feature inside your app has one entry point. The same feature exposed through App Intents has six: Siri voice, Siri Type to Siri, Shortcuts automations, Spotlight suggestions, widgets, and your app’s own UI. Same code, 6× the surface area.

User Story: AI Without Opening the App

Marcus is driving. He says, “Hey Siri, summarize my latest note in SmartNotes.” Siri reads back a 3-bullet summary. When he arrives at the office, a Shortcut automation triggers: summarize unread notes, extract action items, and push them to a Home Screen widget. At lunch, he searches Spotlight for “meeting action items” — his AI-generated summaries appear alongside system results. He never opened the app once.

App Intents Framework 101

App Intents is Apple’s modern framework for exposing your app’s functionality to the system. It replaced the old SiriKit with something dramatically simpler. Instead of conforming to Apple’s predefined domains, you define any action your app can perform.

The Three Building Blocks

AppIntent
Actions
What your app can do. “Summarize a note,” “Classify a photo,” “Generate a cover image.”
AppEntity
Data
What your app operates on. Notes, photos, recipes, tasks — the nouns of your app.
EntityQuery
Discovery
How the system finds your data. Search by ID, property, or suggested entities.

API Recipe: Your First App Intent for an AI Feature

Goal: Make “Summarize my latest note” available to Siri and Shortcuts.

Step 1: Define the entity

import AppIntents

struct NoteEntity: AppEntity {
    static var typeDisplayRepresentation = TypeDisplayRepresentation(name: "Note")
    var displayRepresentation: DisplayRepresentation {
        DisplayRepresentation(title: "\(title)")
    }

    let id: UUID
    let title: String
    let body: String

    static var defaultQuery = NoteQuery()
}

Step 2: Define the query

struct NoteQuery: EntityQuery {
    func entities(for identifiers: [UUID]) async throws -> [NoteEntity] {
        NoteStore.shared.notes(for: identifiers)
    }

    func suggestedEntities() async -> [NoteEntity] {
        NoteStore.shared.recentNotes(limit: 5)
    }
}

Step 3: Define the intent

struct SummarizeNoteIntent: AppIntent {
    static var title: LocalizedStringResource = "Summarize Note"
    static var description = IntentDescription(
        "Summarizes a note using on-device AI."
    )

    @Parameter(title: "Note") var note: NoteEntity

    func perform() async throws -> some ReturnsValue<String> {
        let session = LanguageModelSession()
        let summary = try await session.respond(
            to: "Summarize in 3 bullets: \(note.body)"
        )
        return .result(value: summary.content)
    }
}

Step 4: Register Siri phrases

struct SmartNotesShortcuts: AppShortcutsProvider {
    static var appShortcuts: [AppShortcut] {
        AppShortcut(
            intent: SummarizeNoteIntent(),
            phrases: [
                "Summarize my note in \(.applicationName)",
                "Summarize \(\.$note) in \(.applicationName)"
            ],
            shortTitle: "Summarize Note",
            systemImageName: "doc.text.magnifyingglass"
        )
    }
}

Result: “Hey Siri, summarize my note in SmartNotes” works immediately. The intent also appears in Shortcuts and Spotlight suggestions.

Tip: openAppWhenRun

By default, App Intents open your app when run. For AI features that return a simple result (a summary, a classification), set static var openAppWhenRun = false so the intent runs headlessly. Siri shows the result inline without launching your app — much better UX.

The 12 App Intent Domains

Apple defines 12 domains — categories of app functionality where Siri gets deeper integration. If your app fits a domain, conform to its pre-defined intent types and Siri handles natural language resolution with minimal custom code.

DomainExample Siri RequestAI Feature Opportunity

Browsers“Open [URL] in [App]”Page summarization

Tip: Don’t Force a Domain

If your app doesn’t fit a domain, don’t force it. Generic AppIntent works perfectly for custom AI actions. Domains add deeper Siri integration, but your intent is still fully functional without one.

Writing Tools: Free AI for Text Views

Here’s the best deal in iOS development: if your app uses standard text views (UITextView, TextEditor, WKWebView), it gets Apple’s Writing Tools for free. No code. No setup. Your users can Rewrite, Proofread, and Summarize any text in your app.

Product Pattern: Custom Writing Tools Actions

The pattern: Extend Apple’s Writing Tools with app-specific actions that appear alongside Rewrite, Proofread, and Summarize.

Why it works: Users already know the Writing Tools menu. Adding your actions there means zero learning curve. “Convert to Action Items,” “Simplify for Client,” or “Translate to Spanish” feel like native system features.

Where to use it: Any app with text editing — notes, email, documents, messaging, project management.

How to implement: Define an AppIntent that conforms to the Word Processor domain’s writing intent types. Your custom action appears in the Writing Tools menu alongside Apple’s built-in options.

Spotlight: Making AI Content Discoverable

Every AI-generated summary, classification, or extraction your app creates is content. If it stays locked inside your app, users have to remember it exists and navigate to find it. Spotlight fixes this.

IndexedEntity: The Modern Approach

struct NoteEntity: IndexedEntity {
    // ... existing AppEntity properties ...

    static var defaultQuery = NoteQuery()

    @Property(indexingKey: .title)
    var title: String

    @Property(indexingKey: .description)
    var summary: String  // AI-generated summary

    @Property(indexingKey: .keywords)
    var tags: [String]  // AI-extracted topics
}

Then index your entities at app launch or when content changes:

import CoreSpotlight

CSSearchableIndex.default().indexAppEntities(
    NoteStore.shared.allNotes()
)

Now when a user searches Spotlight for “meeting action items,” your AI-generated summaries and tags make those notes discoverable — even with semantic search matching.

Tip: Index AI Outputs, Not Just Titles

Index the AI-generated summary and extracted tags alongside the original content. A note titled “Monday standup” might not match a Spotlight search for “API deadline.” But the AI summary that says “API migration deadline moved to Friday” will.

Widget Intelligence

Widgets are your AI features’ storefront. A well-designed widget shows AI-generated insights without the user lifting a finger.

AI Widget Ideas

Great AI Widgets

  • “Today’s Summary” — AI-generated daily briefing from your app’s content
  • “Smart Suggestions” — proactive recommendations based on usage patterns
  • “Action Items” — extracted from recent voice notes or meetings
  • “Quick Capture” — one-tap to voice-record + AI-summarize

Avoid These

  • “AI Chatbot” — widgets aren’t interactive enough for conversations
  • “Full Transcript” — too much text for a widget. Show the summary, link to the full content
  • “Live AI Processing” — widgets refresh on a timeline, not in real time. Don’t show stale “processing” states
  • “Settings” — don’t waste widget space on configuration

Warning: Widget Timeline vs Real-Time

Widgets refresh on a system-managed timeline, not in real time. Generate AI content in your main app and pass it to the widget via a shared App Group container. Don’t try to run Foundation Models inside a widget extension — the memory budget is too small and the runtime too short.

Live Activities for Long AI Tasks

Some AI features take time: batch-analyzing 120 photos, processing a 30-minute meeting transcript, or generating multiple cover images. Live Activities let you show real-time progress on the Lock Screen and Dynamic Island.

1

Start

User triggers batch AI processing. Start a Live Activity with initial state.

→
2

Update

Push progress updates: “Analyzing photos… 47/120.” Dynamic Island shows compact progress.

→
3

Complete

Final update with result summary. Lock Screen shows “Done — 8 categories found.” Tap to open.

import ActivityKit

struct AIProcessingAttributes: ActivityAttributes {
    let taskName: String  // "Analyzing photos"

    struct ContentState: Codable, Hashable {
        var progress: Int      // 47
        var total: Int         // 120
        var currentStep: String  // "Classifying..."
        var isComplete: Bool
    }
}

// Start the activity
let attributes = AIProcessingAttributes(taskName: "Analyzing photos")
let state = AIProcessingAttributes.ContentState(
    progress: 0, total: 120,
    currentStep: "Starting...", isComplete: false
)
let activity = try Activity.request(
    attributes: attributes, content: .init(state: state, staleDate: nil)
)

Tip: Don’t Overuse Live Activities

Reserve Live Activities for tasks that take 10+ seconds and where the user genuinely cares about progress. A 2-second text summary doesn’t need a Live Activity — it needs a loading spinner. Batch photo analysis or long meeting processing? That’s Live Activity territory.

Focus Modes & Smart Notifications

Not all AI insights are relevant all the time. Focus modes let you customize what your app surfaces based on context.

Work Focus

  • Show project summaries and action items
  • Surface meeting-related notes
  • Prioritize work-tagged content
  • Widget shows “Today’s priorities”

Personal Focus

  • Show recipe suggestions and personal notes
  • Surface weekend plans
  • Prioritize personal-tagged content
  • Widget shows “Weekend ideas”

Implement Focus filters by conforming to SetFocusFilterIntent:

struct SmartNotesFocusFilter: SetFocusFilterIntent {
    static var title: LocalizedStringResource = "SmartNotes Filter"

    @Parameter(title: "Category")
    var category: NoteCategory?

    func perform() async throws -> some IntentResult {
        NotificationCenter.default.post(
            name: .focusCategoryChanged,
            object: category
        )
        return .result()
    }
}

On-Screen Awareness (iOS 26)

iOS 26 introduces a powerful new capability: Siri can understand what’s visible on screen in your app. Users can ask questions about content they’re looking at — “What’s this recipe’s calorie count?” or “Summarize this article” — and Siri routes to your intent.

This works by associating AppEntity instances with SwiftUI views using the .appIntentBindings() modifier. When Siri is invoked while your view is on screen, it can access the bound entity and run the appropriate intent.

Warning: Privacy with On-Screen Entities

On-screen awareness means Siri can read your app’s visible content. Only bind entities that the user would expect Siri to access. Sensitive content (financial details, health data, private messages) should be excluded or require explicit opt-in.
