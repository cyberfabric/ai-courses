Chapter 7

Image Understanding & Document Intelligence

Vision framework OCR, RecognizeDocumentsRequest, photo intelligence, and structured data extraction

Your Camera Is an AI Input Device

Every iPhone has a camera. That camera is also the world’s most convenient scanner, translator, and data entry tool — if your app knows how to use it. This chapter is about teaching your app to see: read text from images, understand document structure, classify photos, and extract structured data from visual content.

The magic pipeline is: camera captures image → Vision extracts text → Foundation Models structures it. Vision handles the seeing. Foundation Models handles the understanding. Together, they turn a photo of a receipt into a typed Swift struct with merchant, date, total, and line items — all on-device, for free.

Scenario: Aisha is at a business lunch. She snaps a photo of the receipt with her expense app. In under 2 seconds, the app shows: “Cafe Milano — $47.80 — March 12, 2026 — 3 items.” Each field is editable in case the AI got something wrong. She taps “Save” and the expense is logged. No typing. No data entry. Just point, shoot, done.

Vision Framework: The Eyes

The Vision framework has been on iOS since iOS 11, and it keeps getting better. It provides specialized request types for different visual tasks — each optimized, each running on-device:

26 Languages supported for text recognition
0ms Cloud latency — everything runs on-device
iPhone 8+ Minimum device for Vision framework

The Key Vision Requests

  • RecognizeDocumentsRequest (iOS 26) — The new star. Understands document structure: tables, lists, paragraphs, QR codes, data types (emails, phone numbers, URLs, dates, addresses). Not just “here’s some text” but “here’s a table with 4 rows and 3 columns.”
  • RecognizeTextRequest — Classic OCR. Extracts raw text with per-character confidence scores. Two modes: .fast for real-time camera preview, .accurate for still images. Still useful when you just need the text without structure.
  • VNClassifyImageRequest — Multi-label image classification. “This photo contains: beach, sunset, people.” No custom model needed.
  • VNDetectFaceRectanglesRequest — Finds faces in images. Returns bounding boxes, landmarks (eyes, nose, mouth), and face quality scores.
  • GeneratePersonSegmentationRequest — Creates a matte that isolates people from the background. Three quality levels. Works on video frames too.

RecognizeDocumentsRequest: The Game Changer

Before iOS 26, scanning a table was painful. RecognizeTextRequest returned each cell as a separate, unrelated text observation. You had to figure out which cells belonged to which row and column by analyzing bounding box positions. It was fragile and error-prone.

RecognizeDocumentsRequest changes everything. It returns a hierarchical DocumentObservation that understands structure:

Recipe: Scan a Document with Structure

Goal: Extract structured content from a document image — paragraphs, tables, and detected data types.

Step 1: Create and perform the request:

import Vision

let request = RecognizeDocumentsRequest()
let observations = try await request.perform(on: documentImage)

for document in observations {
    for block in document.blocks {
        switch block {
        case .paragraph(let paragraph):
            print("Paragraph: \(paragraph.text)")
        case .table(let table):
            for row in table.rows {
                let cells = row.cells.map(\.text)
                print("Row: \(cells.joined(separator: " | "))")
            }
        case .list(let list):
            for item in list.items {
                print("• \(item.text)")
            }
        }
    }
}

Step 2: Extract detected data types automatically:

// RecognizeDocumentsRequest also detects:
// - Email addresses
// - Phone numbers
// - URLs
// - Dates
// - Postal addresses
// - Tracking numbers
// - Flight numbers
// - Payment card numbers

for dataItem in document.detectedDataItems {
    switch dataItem {
    case .email(let email): handleEmail(email)
    case .phoneNumber(let phone): handlePhone(phone)
    case .url(let url): handleURL(url)
    case .date(let date): handleDate(date)
    default: break
    }
}

No regex. No custom parsing. The framework detects data types and hands them to you as typed values. A business card becomes a contact. A receipt becomes structured data. A boarding pass becomes a flight number and gate.

RecognizeDocumentsRequest vs RecognizeTextRequest

Use RecognizeDocumentsRequest when you need structure (tables, lists, data types). Use RecognizeTextRequest when you just need raw text fast (real-time camera overlay, simple OCR). For most new features, start with RecognizeDocumentsRequest — it’s the modern path.

The Snap-and-Understand Pipeline

Here’s where it gets powerful: Vision extracts text, Foundation Models structures it. Two frameworks, one pipeline, entirely on-device.

The Snap-and-Understand Pattern

  1. Capture — User points camera at a document (receipt, business card, menu, invoice). Use DataScannerViewController for a polished camera UI with live text overlay.
  2. Extract — Vision’s RecognizeDocumentsRequest pulls out raw text with structure. Tables stay as tables. Lists stay as lists.
  3. Structure — Foundation Models with @Generable parses the raw text into your typed Swift struct. Receipt text becomes ReceiptData(merchant: "Cafe Milano", total: 47.80, items: [...]).
  4. Verify — Show results in an editable form with confidence indicators. Users can fix anything the AI got wrong.
  5. Save — One tap to save. The 30-second manual data entry is now a 2-second photo.

This pattern works for receipts, business cards, invoices, menus, boarding passes, prescriptions — any structured document.

Receipt Parsing with @Generable

Let’s build the Foundation Models side of the pipeline. Vision gives us raw text; we need typed data:

@Generable
struct ReceiptData {
    @Guide(description: "The merchant or store name")
    var merchant: String

    @Guide(description: "The total amount paid")
    var total: String

    @Guide(description: "The date of the transaction")
    var date: String

    @Guide(description: "Individual line items with name and price")
    var items: [LineItem]
}

@Generable
struct LineItem {
    @Guide(description: "Item name or description")
    var name: String

    @Guide(description: "Item price")
    var price: String
}

// After Vision extracts the text:
let session = LanguageModelSession()
let receipt = try await session.respond(
    to: "Extract receipt details from this text: \(ocrText)",
    generating: ReceiptData.self
)
// receipt.merchant → "Cafe Milano"
// receipt.total → "$47.80"
// receipt.items → [LineItem(name: "Margherita Pizza", price: "$18.00"), ...]

Why not just use RecognizeDocumentsRequest alone? It detects structure beautifully (tables, data types), but it doesn’t interpret. It knows “this is a table with prices” but doesn’t know which number is the total vs a subtotal vs a tax amount. Foundation Models adds the understanding layer. Use both together.

Photo Library Intelligence

Beyond documents, Vision can make your entire photo library smarter. Here are the building blocks:

Smart Photo Features You Can Build

Auto-categorization (vacation/food/pets), duplicate detection, “find photos like this,” accessibility descriptions for VoiceOver, face grouping for contacts, scene detection for filters.

Each uses a different Vision request, all on-device, all private. Photos are deeply personal — on-device processing is essential.

Don’t Do This

Upload photos to a cloud API for analysis, process the entire library on the main thread, show classification labels without letting users correct them, auto-delete “bad” photos.

Photos trigger the strongest privacy reactions. Never upload without explicit consent. Never auto-delete. Always let users override AI decisions.

Image Classification

import Vision

func classifyImage(_ image: CGImage) async throws -> [String] {
    let request = VNClassifyImageRequest()
    let handler = VNImageRequestHandler(cgImage: image)
    try handler.perform([request])

    guard let results = request.results else { return [] }
    return results
        .filter { $0.confidence > 0.5 }
        .prefix(5)
        .map(\.identifier)
    // → ["food", "pizza", "indoor", "restaurant", "plate"]
}

The built-in classifier recognizes hundreds of categories without any custom model. It’s perfect for auto-tagging, smart albums, and content filtering.

Person Segmentation for Portrait Features

let request = GeneratePersonSegmentationRequest()
request.qualityLevel = .accurate  // .fast for video, .accurate for stills

let handler = VNImageRequestHandler(cgImage: photo)
try handler.perform([request])

if let result = request.results?.first {
    let matteMask = result.pixelBuffer
    // Use matteMask to blur background, change background color,
    // create sticker cutouts, or apply effects to people only
}

This is how the Photos app creates portrait effects without a depth sensor on older devices. The segmentation mask isolates people pixel-by-pixel, letting you blur backgrounds, swap backgrounds, or create sticker-style cutouts.

Batch Processing Photos

When you analyze a photo library, you’re potentially processing hundreds or thousands of images. This requires careful resource management:

func analyzeLibrary(photos: [PHAsset]) async {
    // Process in batches to avoid memory pressure
    let batchSize = 10

    for batch in photos.chunked(into: batchSize) {
        await withTaskGroup(of: PhotoAnalysis?.self) { group in
            for asset in batch {
                group.addTask {
                    guard let image = await self.loadImage(asset) else { return nil }
                    let categories = try? await self.classifyImage(image)
                    let text = try? await self.extractText(image)
                    return PhotoAnalysis(assetID: asset.localIdentifier,
                                        categories: categories ?? [],
                                        extractedText: text ?? "")
                }
            }
            for await result in group {
                if let analysis = result {
                    await saveAnalysis(analysis)  // Store in SwiftData
                }
            }
        }
    }
}

Never block the main thread. Image processing is CPU-intensive. Always use Task.detached(priority: .utility) or a background TaskGroup. Process in batches of 10–20 images to control memory. Show a progress indicator: “Analyzing photos… 47/120.”

Image Preprocessing

Before feeding images to Vision, handle iOS-specific format considerations:

  • HEIC format — iOS’s default photo format. Vision handles it natively, but if you’re sending to a cloud API, convert to JPEG first.
  • Live Photos — Extract the still image component. Don’t accidentally process the video portion.
  • Orientation metadata — iOS stores rotation in EXIF metadata, not in pixel data. Always pass the image orientation to VNImageRequestHandler via the orientation parameter, or your bounding boxes will be wrong.
  • Resize for performance — A 48MP ProRAW photo is overkill for classification. Downscale to 1024px on the long edge before processing. Vision’s accuracy is nearly identical at lower resolutions.
  • Memory management — CGImage objects are large. Release them as soon as processing completes. Use autoreleasepool blocks in tight loops.

Accessibility: AI-Generated Image Descriptions

One of the most impactful uses of image AI is generating VoiceOver descriptions for visually impaired users. Combine Vision classification with Foundation Models to create natural descriptions:

func generateAccessibleDescription(for image: CGImage) async throws -> String {
    // Step 1: Vision classifies the image
    let categories = try await classifyImage(image)  // ["beach", "sunset", "people"]

    // Step 2: Foundation Models creates a natural description
    let session = LanguageModelSession(
        instructions: "Create brief, helpful image descriptions for VoiceOver users."
    )
    let description = try await session.respond(
        to: "Describe a photo containing: \(categories.joined(separator: ", "))"
    )
    return description.content
    // → "A beach at sunset with people visible in the scene"
}

This is high-impact accessibility. Photos without descriptions are invisible to VoiceOver users. Auto-generating descriptions makes your entire photo experience accessible. Set them via .accessibilityLabel() in SwiftUI. Apple does this in the Photos app — you should too.

Privacy: The Non-Negotiable Rule

Photos are the most personal data on someone’s phone. More personal than messages, more personal than health data. The privacy rules for image AI are absolute:

The Photo Privacy Compact

  • Process on-device. Vision + Foundation Models both run on-device. Use them. No photo should leave the device for AI analysis unless the user explicitly opts in.
  • Request minimal access. Use PHPickerViewController for user-selected photos (no permission dialog needed). Only request full library access if your feature genuinely needs it.
  • Be transparent. If a feature requires photo library access, explain exactly why in the permission dialog string. “To organize your photos by category” is specific. “To improve your experience” is not.
  • Never upload without consent. If you must send a photo to a cloud API (for capabilities beyond on-device), show a clear dialog first: “This photo will be sent to our servers for analysis. Continue?”

Real-World Examples

Image understanding powers some of the most beloved features on iOS:

  • Live Text — Apple’s built-in feature that lets you select, copy, and act on text in any photo. It uses Vision under the hood and surfaces phone numbers, URLs, and addresses as tappable actions.
  • Notes Scanner — The document scanner in Notes uses Vision for OCR and perspective correction. RecognizeDocumentsRequest makes this even more powerful in iOS 26.
  • Signeasy — Extracts contract clauses, dates, and obligations from scanned documents using on-device AI. Legal privacy makes on-device processing essential.
  • Receipts by Wave — Photograph receipts, auto-extract merchant, total, and date. Exactly the snap-and-understand pipeline we built above.
  • CamScanner / Scanner Pro — Professional document scanning with OCR, table detection, and export to searchable PDF.

Your Turn

Exercise: Build a Receipt Organizer

Build a feature where users photograph receipts and the app extracts structured data:

  1. Camera capture: Use PHPickerViewController or the camera to capture a receipt image.
  2. Vision OCR: Use RecognizeDocumentsRequest to extract text with structure. If targeting older iOS versions, fall back to RecognizeTextRequest.
  3. Foundation Models parsing: Define a @Generable ReceiptData struct with merchant, date, total, and items. Pass the OCR text to Foundation Models to fill it.
  4. Editable form: Show the extracted data in a SwiftUI form where every field is editable. Add a colored confidence dot next to each field (green for high confidence, yellow for medium).
  5. Save & organize: Store receipts in SwiftData, grouped by month and category.
  6. Graceful fallback: If Foundation Models isn’t available on this device, still show the raw OCR text and let the user fill in the fields manually.

Test with: A restaurant receipt, a grocery receipt, and a gas station receipt. Each has different layouts — your @Generable struct should handle all three because Foundation Models understands context, not just format.
