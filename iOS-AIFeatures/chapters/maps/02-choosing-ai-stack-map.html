The Big Idea

iOS gives you four AI paths — each with a sweet spot. Foundation Models is free and private but needs new hardware. Apple frameworks cover more devices. Cloud APIs are powerful but costly. The best apps use a hybrid architecture that routes each task to the right provider.

🎯 The Five-Factor Decision Framework

  1. Latency

    Need <100ms? On-device only. Autocomplete and camera features can’t tolerate cloud round trips.

  2. Privacy

    Sensitive data (health, finance, photos)? On-device only. Users see your privacy nutrition label before downloading.

  3. Cost at Scale

    100K DAU × 3 req/day = 300K/day. Foundation Models = $0. Cloud API = $3K–$90K/month depending on model.

  4. Device Coverage

    Foundation Models = A17 Pro+. Vision/NLP = iPhone 8+. Cloud = any device with internet.

  5. Capability

    Complex reasoning or image generation? You may need cloud. Text classification? On-device handles it fine.

📖 Key Terms

@Generable
Swift macro that constrains Foundation Models to output typed structs/enums — guaranteed structural correctness.
@Guide
Restricts model output values to your predefined set (e.g., category must be from your enum).
LanguageModelSession
Stateful session for multi-turn conversations with Foundation Models. Maintains context automatically.
Private Cloud Compute
Apple’s secure cloud for complex tasks. No data stored, no remote access, cryptographically auditable.
Hybrid Architecture
Routing pattern: on-device for fast/private tasks, cloud for heavy tasks. Best of both worlds.
API Proxy Pattern
Your server holds the API key, not the app. The only production-safe way to use cloud APIs on iOS.
App Attest
Apple’s API that proves a request comes from a legitimate copy of your app, not a script.
Graceful Degradation
Check if Foundation Models is available at runtime. If not, fall back to Apple frameworks or cloud.

🧪 Test Yourself

Quick Quiz

When should you use Foundation Models vs Vision framework for text extraction?

Use Vision (VNRecognizeTextRequest) for raw OCR — it’s faster, lighter, and supports more devices. Use Foundation Models with @Generable to structure the extracted text into typed Swift data (e.g., parse a receipt into merchant, date, total).

What are the five factors in the decision framework?

Latency (how fast?), Privacy (sensitive data?), Cost (at scale?), Device Coverage (which iPhones?), and Capability (can on-device handle it?).

Why should you never hardcode API keys in your iOS app?

An IPA is a zip archive — anyone can extract it and find string literals. They use your key, you get the bill. Use a backend proxy: your app talks to your server, your server talks to the API.

What is Private Cloud Compute?

Apple’s secure cloud infrastructure for complex AI tasks. Runs on Apple silicon, data is never stored, no remote access, cryptographically auditable. Same privacy guarantees as on-device. Transparent to your code.

How do you handle older iPhones that don’t support Foundation Models?

Three strategies: Graceful degradation (skip the AI feature), Framework fallback (use Vision/NLP instead), or Cloud fallback (route to API on older devices). Check SystemLanguageModel.default.isAvailable at runtime.

Key Takeaways

  • Four paths: Foundation Models, Apple Frameworks, Core ML, Cloud APIs — each has a sweet spot.
  • Foundation Models = $0 per inference. This changes which features are financially viable.
  • Apple frameworks (Vision, NLP, SpeechAnalyzer) are often faster and support more devices than Foundation Models.
  • Cloud APIs are most powerful but cost money, need internet, and trigger privacy nutrition labels.
  • Use the Five-Factor Framework: Latency, Privacy, Cost, Device Coverage, Capability.
  • Most production apps use a hybrid architecture — on-device for fast/private, cloud for heavy tasks.
  • Never ship API keys in your app binary. Use a backend proxy with App Attest.
  • Make AI features optional enhancements, not hard requirements — graceful degradation is key.
