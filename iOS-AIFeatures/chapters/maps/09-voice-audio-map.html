The Big Idea

SpeechAnalyzer hears. Foundation Models understands. The new modular Speech framework (iOS 26) replaces SFSpeechRecognizer with Swift Concurrency-native, on-device transcription that’s 2.2× faster than Whisper. Chain it with Foundation Models to turn spoken words into structured data and app actions — all without leaving the device.

🎙️ SpeechAnalyzer: Three Modules

🗣️
SpeechTranscriber
Clean transcription for commands and structured input. No extra punctuation. Best for voice commands, search queries, form filling.
📝
DictationTranscriber
Natural dictation with automatic punctuation, paragraph breaks, formatting. Best for note-taking, email, long-form text.
👂
SpeechDetector
Detects voice activity without full transcription. Lightweight. Best for push-to-talk, voice activation, silence detection.

Composable: add only the modules you need. All stream results via AsyncSequence.

🖥️ The Five States of a Voice UI

  1. Idle

    Microphone button with clear “Tap to speak” affordance. No ambiguity.

  2. Listening

    Animated waveform + streaming text appearing word-by-word. Button shows “Tap to stop.”

  3. Processing

    Brief finalization. SpeechAnalyzer is fast enough that this is often invisible.

  4. Result

    Final transcript displayed. If chaining with Foundation Models, show “Analyzing…”

  5. Error

    Mic denied, no speech, or unsupported device. Always offer a text-input fallback.

🔒 Privacy & Battery

🎤
Microphone Is Sacred
Explain before asking. “To transcribe your voice notes” not “To improve your experience.”
📱
100% On-Device
SpeechAnalyzer + Foundation Models both local. Lead with: “Your voice never leaves your iPhone.”
🔋
~5% Battery/Hour
Continuous transcription drain. Use SpeechDetector first, stop promptly, respect Low Power Mode.
🗑️
Don’t Store Raw Audio
Transcribe, extract, discard buffers. Keep the transcript, not the recording (unless user opts in).
🟠
Orange Dot
iOS shows the recording indicator. Embrace it. Add your own in-app indicator too. Stop when backgrounded.
⌨️
Always Text Fallback
Voice is an accelerator, not a requirement. Noisy environments, public spaces, accessibility — always let users type.

📖 Key Terms

SpeechAnalyzer
iOS 26 modular speech framework. Session manager — add modules (transcribers, detectors) as needed. Replaces SFSpeechRecognizer.
SpeechTranscriber
Module for clean command-style transcription. No auto-punctuation. Good for voice commands and form filling.
DictationTranscriber
Module for natural dictation with punctuation, paragraph breaks. Good for notes, email, long-form text.
SpeechDetector
Lightweight voice-activity detection without full transcription. Use to gate heavier analysis.
SNClassifySoundRequest
Sound Analysis framework. 300+ categories (sirens, music, dog barks). On-device, free, iOS 15+.
SNAudioStreamAnalyzer
Real-time stream analyzer. Feed audio buffers from AVAudioEngine, get classification results via observer.
AVSpeechSynthesizer
Text-to-speech engine. Use for short confirmations only — show long content on screen instead.
Voice-to-Action
Pipeline: speech → SpeechTranscriber → Foundation Models @Generable → structured app action.

🧪 Test Yourself

Quick Quiz

When should you use SpeechTranscriber vs DictationTranscriber?

SpeechTranscriber for commands and structured input (clean text, no auto-punctuation). DictationTranscriber for natural text composition (notes, email — adds punctuation, paragraph breaks automatically).

How does the voice-to-action pipeline work?

User speaks → SpeechTranscriber converts to text → Foundation Models extracts structured data via @Generable (e.g., CalendarAction with title, attendees, date) → app executes the action → user confirms or edits.

What are three key improvements of SpeechAnalyzer over SFSpeechRecognizer?

1. Modular architecture (add only what you need). 2. Long-form audio support (meetings, lectures — no 60s limit). 3. Automatic language detection (no manual locale config). Also: AsyncSequence streaming, distant mic support, 2.2× faster.

How do you handle long meeting transcripts that exceed Foundation Models’ context window?

Map-reduce chunking: split the transcript into ~2,000-token segments, summarize each independently, then combine the summaries into a final cohesive summary. Two good summaries merged beats one truncated summary.

Why should you use SpeechDetector before SpeechTranscriber?

Battery efficiency. SpeechDetector is much lighter — it only detects voice activity without full transcription. Run it first, then activate SpeechTranscriber only when speech is detected. Saves significant battery for features that listen continuously.

Key Takeaways

  • SpeechAnalyzer (iOS 26) replaces SFSpeechRecognizer. Modular, AsyncSequence-native, 2.2× faster than Whisper.
  • Three modules: SpeechTranscriber (commands), DictationTranscriber (dictation), SpeechDetector (voice activity).
  • Voice-to-action: speech → transcribe → Foundation Models @Generable → structured app action. Always confirm before acting.
  • Sound classification: SNClassifySoundRequest identifies 300+ categories on-device. Free, iOS 15+.
  • Audio pipeline: AVAudioEngine → installTap → SNAudioStreamAnalyzer → SNResultsObserving.
  • Meeting notes pipeline: DictationTranscriber → Foundation Models → MeetingSummary @Generable (key points, action items, decisions).
  • Voice assistant loop: listen → understand (tool calling) → execute → speak response (keep it short).
  • Microphone is the most sensitive permission. Explain before asking. Process on-device. Don’t store raw audio.
  • Battery: ~5%/hour for continuous transcription. Use SpeechDetector to gate, stop promptly, respect Low Power Mode.
  • Voice is an accelerator, not a requirement. Always provide a text-input fallback.
