Chapter 8

Image Generation & Creative AI

Memory Map
💡

The Big Idea

Apple gives you on-device image generation that’s free, private, and requires zero cloud infrastructure. ImagePlaygroundSheet for Apple’s system UI. ImageCreator for programmatic control. Genmoji for custom emoji. All on A17 Pro+, all on-device. No photorealism — and that’s a feature, not a limitation.

🎨 The Three Creative Tools

🖼️
ImagePlaygroundSheet
Present Apple’s full image generation UI in your app. One SwiftUI modifier. Users get the familiar Playground interface. Easiest path.
⚙️
ImageCreator
Programmatic API (iOS 18.4+). Generate images without system UI. Returns AsyncSequence of images. Full custom workflow control.
😀
Genmoji
Custom emoji from text descriptions. Free with standard text views. NSAdaptiveImageGlyph for custom text engines. Genmoji Mixes in iOS 26.

🖌️ The “Generate Cover Image” Pattern

  1. Context

    User creates content (note, event, playlist). You know the theme from context.

  2. Suggest

    Show “Generate Cover Image” button. Pre-fill the concept from content title/theme.

  3. Generate

    ImageCreator generates 3–4 options. Show each as it arrives (AsyncSequence).

  4. Choose

    User picks favorite. Offer “Regenerate” if none fit.

  5. Apply

    Set as cover. Show subtle “Created with Image Playground” attribution.

✨ Generation UX Checklist

Progress Communication
“Creating your image…” with contextual animation. Never a bare spinner. Generation takes 3–10 seconds.
Show As They Arrive
ImageCreator returns AsyncSequence. Show each image immediately — first image at 4s feels fast; all four at 15s feels slow.
Multiple Options
Always generate 3–4. One image is a demand. Multiple are a choice. Users feel ownership when they pick.
Regenerate Button
“Try Again” or “More Options.” First batch might not hit the mark.
Attribution
“Created with Image Playground” shown subtly. Transparency about AI origin is Apple HIG.
Prompt Engineering
Scene + subject + mood + setting works best. Avoid abstract adjectives and negative prompts.

🧪 Test Yourself

Quick Quiz

What’s the difference between ImagePlaygroundSheet and ImageCreator?

ImagePlaygroundSheet presents Apple’s full system UI inside your app — one SwiftUI modifier, users interact with the familiar Playground interface. ImageCreator is a headless API for programmatic generation without any system UI — full control over the workflow, returns an AsyncSequence of images.

Why can’t Image Playground generate photorealistic images?

Apple explicitly blocks photorealism to prevent deepfakes and celebrity impersonation. This is a deliberate design decision for safety, not a technical limitation. If you need photorealistic generation, use cloud APIs like DALL-E 3 or Stability AI.

How should you handle devices that don’t support image generation?

Three-tier fallback: on-device (ImageCreator, A17 Pro+) → cloud API (DALL-E, costs money, needs internet) → stock images (curated library, no generation). Check with try? await ImageCreator() — it throws on unsupported devices.

Why chain Foundation Models with ImageCreator?

Users type vague descriptions (“birthday party”). Foundation Models expands that into vivid visual descriptions with specific details. Better prompts produce better images. The user never sees the expanded prompt — they just get better results.

How does Genmoji work in your app?

Automatically if you use standard text views (UITextView, TextEditor). For custom text engines, use NSAdaptiveImageGlyph which provides multiple resolutions, a stable identifier, accessibility descriptions, and alignment metrics.

Key Takeaways

  • Three tools: ImagePlaygroundSheet (system UI), ImageCreator (programmatic), Genmoji (custom emoji). All on-device, all free.
  • Three on-device styles: Animation, Illustration, Sketch. iOS 26 adds ChatGPT-powered styles (cloud, opt-in).
  • No photorealism by design. Prevents deepfakes. Use cloud APIs (DALL-E, Stability AI) when photorealism is required.
  • Pre-fill the concept from app context. Don’t make users start from scratch.
  • Chain Foundation Models + ImageCreator: AI writes better prompts than users, producing better images.
  • Generate 3–4 options, not one. Multiple options = user choice = ownership.
  • Show images as they arrive via AsyncSequence. Don’t wait for all to complete.
  • Genmoji is free with standard text views. NSAdaptiveImageGlyph for custom text engines.
  • Fallback strategy: on-device → cloud API → stock images. Never make generation a hard requirement.
  • Generated images should support the user’s goal (notes, events, playlists), not become the goal.
