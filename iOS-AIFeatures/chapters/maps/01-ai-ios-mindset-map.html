The Big Idea

AI features are probabilistic, not deterministic. That one word — probably — changes everything about how you design, build, test, and ship on iOS. Apple gives you a free, private, on-device LLM. Your job is to make it feel magical.

🔀 The Core Shift

Aspect Traditional Features AI Features
Behavior Deterministic — same input, same output Probabilistic — "probably" a cat, 94%
Error handling Bugs are fixable defects Wrong answers are expected & designed for
Testing Assert exact outputs Measure accuracy over many cases
Lifecycle Build → Ship → Done Build → Ship → Monitor → Improve → Forever
UX design Design the happy path Design the error case first

⭐ The Four Patterns of Great AI Features

👻
Invisible Intelligence
AI works behind the scenes — users never think about it. Siri Suggestions surface the right app at the right time.
🦸
Augmented Capability
AI gives users superpowers. Live Text lets you copy text from any photo — impossible without AI.
⚡
Reduced Friction
AI removes tedious steps. QuickType completes your sentence. Mail suggests one-tap replies.
🔮
Proactive Assistance
AI anticipates needs before you ask. Maps says "Leave by 5:15 PM" without you requesting a route.

📊 The AI Feature Spectrum

  1. Classification (Low Risk)

    Is this photo a receipt? Yes/No. Start here — simple, safe, and delightful when done well.

  2. Autocomplete (Low Risk)

    QuickType predictions. Wrong suggestions cost the user nothing — they just keep typing.

  3. Summarization (Medium Risk)

    Safari page summaries. Useful but can miss key points — always show the original.

  4. Content Generation (Medium-High Risk)

    Mail suggested replies. The AI writes for the user — mistakes carry the user's name.

  5. Conversational AI (High Risk)

    In-app chatbots. Open-ended input = open-ended failure modes. Needs guardrails.

  6. Autonomous Agents (Very High Risk)

    Siri performing multi-step actions. The AI acts on behalf of the user — stakes are highest.

🚫 Five Anti-Patterns to Avoid

🎪 AI for AI's Sake
Adding AI where a UIPickerView would work. If it's 5 options, just show a list.
⬛ The Black Box
AI makes decisions with zero explanation. Users need context to trust recommendations.
💤 Set and Forget
Shipping an AI feature and never checking analytics. Quality degrades silently.
✨ The Over-Promise
"AI-powered" label on a keyword match. Users trained by Apple Intelligence will notice.
🍳 The Kitchen Sink
Five AI features at once instead of nailing one. Ship one excellent feature, then iterate.
🏆

The Golden Rule

If you remove the AI and the feature still makes sense as a manual workflow, you probably have a good AI feature. The user need comes first; the AI is the accelerant.

🧪 Test Yourself

Quick Quiz

What's the fundamental difference between traditional features and AI features?

Traditional features are deterministic (same input always gives same output). AI features are probabilistic (the model gives its best guess with some confidence level).

Name the four patterns of great AI features.

Invisible Intelligence (works behind the scenes), Augmented Capability (gives users superpowers), Reduced Friction (removes tedious steps), Proactive Assistance (anticipates needs).

Why is privacy a competitive advantage on iOS, not just a compliance checkbox?

On-device processing lets you work with health data, financial records, and personal photos that users would never send to a cloud API. Privacy nutrition labels are visible in the App Store — "No Data Collected" drives adoption.

What makes Foundation Models framework unique for iOS developers?

It gives you access to Apple's ~3B parameter on-device LLM with zero cost per inference, no API key, no internet needed, and a Swift-native API. It's the same model powering Writing Tools and Siri.

What's the litmus test for whether an AI feature is worth building?

Does it save the user time, effort, or frustration? If not, it doesn't matter how technically impressive it is. Impressive demos ≠ useful features.

Key Takeaways

  • AI features are probabilistic — design for the error case first, not just the happy path.
  • Apple's Foundation Models gives you a free, on-device ~3B LLM — $0 per inference, 3 lines of Swift.
  • Privacy is your competitive moat: on-device processing unlocks data categories cloud APIs can't touch.
  • Great AI features follow four patterns: Invisible Intelligence, Augmented Capability, Reduced Friction, Proactive Assistance.
  • Start low on the complexity spectrum — a well-executed classifier beats a half-baked chatbot.
  • The Golden Rule: if you remove the AI and the workflow still makes sense, you have a good AI feature.
  • AI features are gardens, not buildings — ship them, but never stop tending them.
  • The best AI features have a low cost of error — when the AI is wrong, it should be a minor inconvenience, not a disaster.
