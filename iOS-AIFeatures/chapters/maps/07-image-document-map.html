The Big Idea

Vision sees. Foundation Models understands. Vision extracts text and structure from images. Foundation Models with @Generable turns that raw text into typed Swift data. Together they create the snap-and-understand pipeline: point your camera at a receipt → get a typed ReceiptData struct. All on-device. All free.

👁️ Vision Framework Requests

📄
RecognizeDocumentsRequest
iOS 26. The new star. Understands structure: tables, lists, paragraphs, QR codes, emails, phones, URLs, dates, addresses.
🔤
RecognizeTextRequest
Classic OCR. Raw text with per-character confidence. Two modes: .fast (camera) and .accurate (stills).
🏷️
VNClassifyImageRequest
Multi-label classification. “beach, sunset, people” — hundreds of categories, no custom model needed.
👤
GeneratePersonSegmentationRequest
Pixel-level matte isolating people from background. Three quality levels. Works on video frames.
😀
VNDetectFaceRectanglesRequest
Finds faces with bounding boxes, landmarks (eyes, nose, mouth), and quality scores.

📊 Which Request to Use?

Factor RecognizeDocumentsRequest RecognizeTextRequest
Structure Tables, lists, paragraphs — hierarchical Raw text observations — flat
Data detection Emails, phones, URLs, dates, addresses None (DIY regex)
iOS version iOS 26+ iOS 13+
Best for Documents, receipts, forms, tables Quick OCR, real-time camera overlay
Languages 26 26

For most new features, start with RecognizeDocumentsRequest. Fall back to RecognizeTextRequest for older iOS versions.

🔧 Building the Pipeline

  1. Capture

    Use DataScannerViewController for a polished camera UI with live text overlay. Or PHPickerViewController for gallery selection.

  2. Extract (Vision)

    RecognizeDocumentsRequest returns hierarchical DocumentObservation with structured blocks: paragraphs, tables (rows → cells), lists, and auto-detected data types.

  3. Structure (Foundation Models)

    Pass OCR text to Foundation Models with @Generable struct (e.g., ReceiptData). Vision detects structure; Foundation Models adds understanding (which number is the total vs tax).

  4. Verify & Save

    Show in an editable form with confidence indicators. One tap to save. 30 seconds of data entry → 2 seconds.

🖼️ Image Preprocessing Checklist

📸
HEIC Format
Vision handles natively. Convert to JPEG only if sending to cloud APIs.
🔄
Orientation Metadata
Always pass orientation to VNImageRequestHandler. EXIF stores rotation, not pixel data.
📐
Resize First
Downscale to 1024px on long edge. 48MP ProRAW is overkill. Accuracy is nearly identical.
🧹
Memory Management
Release CGImage after processing. Batch in groups of 10–20. Use autoreleasepool in tight loops.

🔒 The Photo Privacy Compact

Process on-device
Vision + Foundation Models both run locally. No photo should leave the device for AI analysis without explicit opt-in.
Request minimal access
Use PHPickerViewController for user-selected photos (no permission dialog). Only request full library access if genuinely needed.
Be transparent
“To organize your photos by category” is specific. “To improve your experience” is not. Explain exactly why.
Never upload without consent
If cloud processing is needed, show a clear dialog first. The user decides when their photos leave the device.
Never auto-delete
AI can suggest, classify, and organize. It must never delete photos on its own. Users override AI decisions, always.

🧪 Test Yourself

Quick Quiz

What’s the key advantage of RecognizeDocumentsRequest over RecognizeTextRequest?

Structure. RecognizeDocumentsRequest returns hierarchical DocumentObservation with tables (rows → cells), lists, paragraphs, and auto-detected data types (emails, phones, URLs, dates). RecognizeTextRequest returns flat, unrelated text observations.

Why do you need Foundation Models after Vision OCR?

Vision detects structure (this is a table with numbers) but doesn’t interpret it (which number is the total vs subtotal vs tax). Foundation Models with @Generable adds the understanding layer, turning raw OCR text into typed Swift data.

How should you handle batch photo processing?

Use TaskGroup with batches of 10–20 images to control memory. Process on a background queue with Task.detached(priority: .utility). Show a progress indicator. Release CGImage objects immediately after processing.

Why is orientation metadata important for Vision?

iOS stores image rotation in EXIF metadata, not in pixel data. If you don’t pass the orientation parameter to VNImageRequestHandler, bounding boxes and text positions will be rotated incorrectly.

How can you generate VoiceOver descriptions for photos?

Two-step pipeline: VNClassifyImageRequest identifies scene categories (“beach, sunset, people”), then Foundation Models generates a natural-language description from those categories. Set via .accessibilityLabel() in SwiftUI.

Key Takeaways

  • Vision sees, Foundation Models understands. Use both together for the snap-and-understand pipeline.
  • RecognizeDocumentsRequest (iOS 26) understands structure: tables, lists, paragraphs, and auto-detected data types.
  • RecognizeTextRequest is still useful for quick OCR and real-time camera overlays on older iOS versions.
  • @Generable structs turn OCR text into typed data: ReceiptData, BusinessCard, BoardingPass — not Strings.
  • Vision detects structure but doesn’t interpret it. Foundation Models adds the understanding layer.
  • Always show extracted data in an editable form. AI gets it right most of the time — but not every time.
  • Photos are deeply personal. Process on-device, request minimal access, never upload without explicit consent.
  • Batch photo processing: TaskGroup, groups of 10–20, background queue, release images after processing.
  • Generate VoiceOver descriptions with Vision classification + Foundation Models. High-impact accessibility.
  • Always pass image orientation to VNImageRequestHandler or your bounding boxes will be wrong.
