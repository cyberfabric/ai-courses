Siri, App Intents & System Integration

Memory Map
💡

The Big Idea

One App Intent = six entry points. Define your AI feature as an AppIntent once, and it becomes available through Siri voice, Type to Siri, Shortcuts automations, Spotlight suggestions, widgets, and your app’s own UI. Same code, 6× the surface area. Your AI features shouldn’t require opening the app.

🧱 The Three Building Blocks

⚡
AppIntent
What your app can do. “Summarize a note,” “Classify a photo,” “Generate a cover image.” The action.
📦
AppEntity
What your app operates on. Notes, photos, recipes, tasks. The nouns. Conforms to IndexedEntity for Spotlight.
🔍
EntityQuery
How the system finds your data. Search by ID, property, or suggested entities. Powers Siri parameter resolution.

📋 System Integration Priority

  1. App Intent

    Define the core action. Foundation for everything else. Set openAppWhenRun = false for headless execution.

  2. Siri Phrases

    Register via AppShortcutsProvider. Users discover with “Hey Siri, what can [App] do?”

  3. Shortcuts

    Automatic once you have an intent. Users chain your AI with other apps in automations.

  4. Spotlight

    Conform AppEntity to IndexedEntity. Index AI summaries and tags — not just titles.

  5. Widget

    Show AI insights passively. Pre-compute in main app, pass via App Group. Don’t run Foundation Models in widget extensions.

  6. Live Activity

    For 10+ second AI tasks only. Progress on Lock Screen and Dynamic Island.

  7. Focus Filters

    SetFocusFilterIntent customizes AI content per context (work vs personal).

  8. On-Screen Awareness

    iOS 26. Bind entities to views with .appIntentBindings(). Siri understands visible content.

🔦 Spotlight: Index AI Outputs

📄
IndexedEntity
Conform AppEntity to IndexedEntity. Annotate with @Property(indexingKey:) for title, description, keywords.
🧠
Index AI Content
Index the AI summary and tags, not just the title. “Monday standup” won’t match “API deadline” — the AI summary will.
🔎
Semantic Search
Once indexed, Spotlight applies semantic matching automatically. Meaning-based, not just keyword.
🔄
Re-index on Change
Call CSSearchableIndex.default().indexAppEntities() at launch and when content changes.

📖 Key Terms

AppIntent
Protocol defining an action. Implement perform() async throws -> some IntentResult. The foundation of system integration.
AppEntity
Protocol for your app’s data objects. The nouns that intents operate on. Needs a defaultQuery.
AppShortcutsProvider
Registers Siri phrases that work without user setup. Users discover via “Hey Siri, what can [App] do?”
IndexedEntity
AppEntity subprotocol for Spotlight indexing. Annotate with @Property(indexingKey:) for search fields.
openAppWhenRun
Set to false for headless AI intents. Siri shows results inline without launching your app.
SetFocusFilterIntent
Lets users customize your AI content per Focus mode. Work shows project summaries; personal shows recipes.
App Intent Domains
12 predefined categories (Books, Mail, Photos, etc.) with deeper Siri integration. Optional — generic intents work fine.
.appIntentBindings()
iOS 26 SwiftUI modifier. Associates entities with views so Siri understands on-screen content.

🧪 Test Yourself

Quick Quiz

What are the three building blocks of App Intents?

AppIntent (actions — what your app can do), AppEntity (data — what it operates on), and EntityQuery (discovery — how the system finds your data). Define these three, and your AI feature works in Siri, Shortcuts, and Spotlight.

Why set openAppWhenRun to false for AI features?

So the intent runs headlessly. Siri shows the result (e.g., a note summary) inline without launching your app. Much better UX for features that return a simple result — the user stays in context.

Why should you index AI-generated summaries in Spotlight, not just titles?

A note titled “Monday standup” won’t match a search for “API deadline.” But the AI summary that says “API migration deadline moved to Friday” will. Indexing AI outputs makes content discoverable by meaning.

When should you use a Live Activity for an AI feature?

Only for tasks taking 10+ seconds where the user genuinely cares about progress — batch photo analysis, long meeting processing. A 2-second text summary needs a spinner, not a Live Activity. Show progress on Lock Screen and Dynamic Island.

How do Writing Tools work in your app for free?

Use standard text views (UITextView, TextEditor, WKWebView). Apple’s Rewrite, Proofread, and Summarize appear automatically. No code needed. Add custom actions by conforming to the Word Processor domain’s intent types.

Key Takeaways

  • One App Intent = six entry points: Siri, Type to Siri, Shortcuts, Spotlight, widgets, and in-app UI.
  • Three building blocks: AppIntent (actions), AppEntity (data), EntityQuery (discovery).
  • Set openAppWhenRun = false for AI features that return simple results. Siri shows them inline.
  • Register Siri phrases via AppShortcutsProvider. No user setup needed.
  • 12 App Intent domains for deeper Siri integration. Don’t force a domain — generic intents work fine.
  • Writing Tools are free with standard text views. Add custom actions via Word Processor domain.
  • Index AI-generated summaries and tags in Spotlight via IndexedEntity. Match by meaning, not just titles.
  • Pre-compute widget content in the main app. Don’t run Foundation Models in widget extensions.
  • Live Activities for 10+ second AI tasks only. Lock Screen + Dynamic Island progress.
  • iOS 26: on-screen awareness via .appIntentBindings(). Siri understands visible content in your app.
