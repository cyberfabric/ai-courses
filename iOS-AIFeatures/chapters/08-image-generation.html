Chapter 8

Image Generation & Creative AI

Image Playground, Genmoji, ImageCreator API, and building creative features that let users generate images on-device

Creating Images with AI

Chapter 7 taught your app to see — reading text and understanding images. This chapter goes the other direction: creating images from nothing but a text description. And unlike cloud image generation services that cost money per image and send data to remote servers, Apple’s Image Playground runs entirely on-device, for free, with no privacy concerns.

The creative AI landscape on iOS gives you three tools:

Image Playground System UI for image creation — present Apple’s familiar interface in your app
ImageCreator Programmatic API — generate images in your own custom workflow, no system UI
Genmoji Custom emoji — automatically available in apps with standard text views

Scenario: Mika is planning a birthday party using an event planning app. She types “tropical beach party at sunset” as the event theme. The app generates four concept images in different styles — a playful animation, a bold illustration, and a quick sketch — all on her phone in seconds. She picks her favorite for the invitation card, tweaks the text, and shares it with friends. No stock photo search. No design skills. No cloud API costs.

ImagePlaygroundSheet: The Easy Path

The simplest way to add image generation is to present Apple’s system Image Playground UI. Users get the same familiar interface they know from Messages and Notes, right inside your app. One SwiftUI modifier is all it takes:

import ImagePlayground

struct EventEditorView: View {
    @State private var showPlayground = false
    @State private var generatedImage: Image?

    var body: some View {
        VStack {
            if let image = generatedImage {
                image.resizable().scaledToFit()
            }
            Button("Create Event Image") {
                showPlayground = true
            }
        }
        .imagePlaygroundSheet(
            isPresented: $showPlayground,
            concept: .text("tropical beach party at sunset")
        ) { url in
            generatedImage = Image(uiImage: UIImage(contentsOfFile: url.path)!)
        }
    }
}

That’s it. The .imagePlaygroundSheet modifier handles the entire generation flow — the UI, the model, the progress, the style picker. You provide an optional concept (a text description to start with), and you get back a URL to the generated image.

Pre-fill the concept. Don’t make users start from scratch. If you know the context (an event theme, a note title, a recipe name), pass it as the concept: parameter. The user can always modify it in the Playground UI, but a good starting point saves them time and shows them what’s possible.

ImageCreator: Full Programmatic Control

ImagePlaygroundSheet is great when you want Apple’s UI. But what if you need to generate images in the background, as part of an automated workflow, or in your own custom interface? That’s ImageCreator — the headless API introduced in iOS 18.4.

Recipe: Generate Images Programmatically

Goal: Generate concept images from a text description without showing the system UI.

Step 1: Check availability and create the generator:

import ImagePlayground

// ImageCreator throws if the device doesn't support generation
guard let creator = try? await ImageCreator() else {
    // Device doesn't support image generation (pre-A17 Pro)
    showFallbackUI()
    return
}

Step 2: Define concepts and generate:

let concepts: [ImagePlaygroundConcept] = [
    .text("a cozy coffee shop on a rainy evening")
]

// Check available styles before using one
let style: ImagePlaygroundStyle = ImageCreator.availableStyles.contains(.animation)
    ? .animation : .illustration

let images = creator.images(for: concepts, style: style, limit: 4)

for try await image in images {
    // Each image arrives as it's generated
    displayImage(image)
}

Step 3: The images(for:style:limit:) method returns an AsyncSequence, so images arrive one at a time as they’re generated. Show each one as it appears — don’t wait for all four.

Image Styles

Image Playground offers three on-device styles, each with a distinct visual personality:

The Three On-Device Styles

Animation — Whimsical 3D cartoon look. Playful, colorful, great for social and casual contexts. Illustration — Strong outlines, bold colors, simple shapes. Professional yet approachable, good for presentations and cards. Sketch — Hand-drawn feel with lighter detail. Perfect for brainstorming, quick concepts, and informal content.

All three run on-device, require no internet, and are free.

What You Can’t Do

Photorealistic images, specific real people or celebrities, brand logos, explicit or violent content. Apple explicitly blocks photorealism to prevent deepfakes and impersonation.

These are design decisions, not bugs. If you need photorealistic generation, you’ll need a cloud API like DALL-E or Stability AI.

New in iOS 26: ChatGPT-powered styles expand beyond the three on-device options. These use OpenAI’s models through Apple’s privacy-preserving integration, offering more dynamic and varied art styles. Users opt in to these when selecting a style in Image Playground. Check ImageCreator.availableStyles at runtime to see what’s available on the user’s device.

Genmoji: Custom Emoji

Genmoji lets users create custom emoji from text descriptions. It’s not just a feature — it’s a new content type that lives alongside standard emoji in the keyboard.

Good News: Genmoji Works for Free

If your app uses standard text views (UITextView, TextEditor), Genmoji works automatically. Users can create and insert custom emoji without you writing a single line of code. The emoji render at the correct size, align properly with text, and support accessibility descriptions.

Custom Text Engines

If your app uses a custom text rendering engine (like a game, a drawing app, or a rich text editor), you need to handle Genmoji manually via NSAdaptiveImageGlyph:

// NSAdaptiveImageGlyph provides:
// - Multiple resolutions for different display contexts
// - A unique stable identifier for each Genmoji
// - Content description for VoiceOver accessibility
// - Alignment metrics for proper text layout

// Render in your custom text engine:
if let glyph = attributedString.attribute(.adaptiveImageGlyph, at: index) as? NSAdaptiveImageGlyph {
    let image = glyph.imageContent  // Get the image data
    let description = glyph.contentDescription  // "smiling cat with party hat"
    renderGlyphInline(image, description: description)
}

New in iOS 26: Genmoji Mixes let users combine existing emoji as input prompts to create mashup Genmoji. Your app gets this for free with standard text views.

Building a Creative Feature

The “Generate Cover Image” Pattern

  1. Context — The user has created something (a note, an event, a playlist, a recipe collection). You know the theme from the content.
  2. Suggest — Show a “Generate Cover Image” button. Pre-fill the concept from the content: note title, event theme, playlist mood.
  3. Generate — Use ImageCreator to generate 3–4 options. Show them in a grid as they arrive (AsyncSequence).
  4. Choose — The user picks their favorite. Offer “Regenerate” if none fit.
  5. Apply — Set the chosen image as the cover. Show a subtle “Created with Image Playground” attribution.

This pattern works for notes, playlists, recipes, albums, projects, events — anything that benefits from a visual identity.

Multi-Model Pipeline: Text + Image

For even smarter creative features, chain Foundation Models with Image Playground. Foundation Models generates creative descriptions; ImageCreator turns them into visuals:

// Step 1: Foundation Models creates variations
let session = LanguageModelSession(
    instructions: "You create vivid, visual descriptions for Image Playground."
)

@Generable
struct ImageConcepts {
    @Guide(description: "Four vivid visual descriptions, each a different interpretation")
    var descriptions: [String]
}

let concepts = try await session.respond(
    to: "Create 4 visual concept descriptions for: \(userTheme)",
    generating: ImageConcepts.self
)

// Step 2: ImageCreator generates an image for each
let creator = try await ImageCreator()
for description in concepts.descriptions {
    let images = creator.images(
        for: [.text(description)],
        style: .illustration,
        limit: 1
    )
    for try await image in images {
        addToGallery(image)
    }
}

The user types “birthday party” and gets four distinct interpretations: a sunset beach celebration, a garden tea party, a neon-lit dance floor, and a cozy living room surprise. Foundation Models provides creative diversity; ImageCreator provides the visuals.

Prompt Design for Image Generation

The on-device model responds best to clear, descriptive text. Here are the patterns that work:

Do This Not This
“A golden retriever wearing a birthday hat in a sunny park” “Dog party”
“A cozy coffee shop interior with warm lighting and bookshelves” “Cafe”
“Mountain landscape at sunrise with a winding trail” “Make it look really cool and epic”
Scene + subject + mood + setting Abstract adjectives, technical jargon, negative prompts

Let Foundation Models write the prompts. Users don’t know how to write good image descriptions. Have Foundation Models expand “beach party” into “a tropical beach party at sunset with tiki torches, colorful paper lanterns, and friends gathered around a bonfire.” Better prompts = better images. The user never sees the expanded prompt.

UX Patterns for Generated Images

Image generation takes seconds, not milliseconds. Your UX needs to account for this:

  • Progress communication — Show “Creating your image…” with a contextual animation. Never just a spinner with no text. Generation typically takes 3–10 seconds.
  • Show images as they arrive — ImageCreator.images() returns an AsyncSequence. Show each image as it’s ready, don’t wait for all of them. The first image appearing after 4 seconds feels fast. Four images appearing after 15 seconds feels slow, even though total time is the same.
  • Multiple options — Always generate 3–4 options. One image is a take-it-or-leave-it demand. Multiple options are a choice. Users feel more ownership when they pick.
  • Regenerate — A clear “Try Again” or “More Options” button. The first batch might not hit the mark.
  • Attribution — Show “Created with Image Playground” subtly below generated images. Transparency about AI origin is part of the Apple HIG.
  • Save to Photos — Offer a “Save to Photos” option so users can use the image outside your app. Use UIImageWriteToSavedPhotosAlbum or the Photos framework.

Content Safety

Apple’s Image Playground has strong built-in safety filters. As a developer, this simplifies your life considerably:

What Apple Blocks Automatically

  • No photorealism — Prevents deepfakes and impersonation entirely.
  • No real people — Can’t generate images of specific celebrities, public figures, or identifiable individuals.
  • No explicit content — Violence, nudity, and harmful content are filtered automatically.
  • No brand logos — Won’t reproduce trademarked imagery.

You don’t need to build your own content filter for Image Playground. Apple handles it at the model level. Your app cannot produce harmful images through this API — period.

This changes your feature design. If your app needs photorealistic images (e.g., virtual staging for real estate, product mockups), Image Playground isn’t the right tool. You’ll need a cloud API like DALL-E 3 or Stability AI. Be transparent about the privacy and cost tradeoffs when using cloud generation.

When Image Playground Isn’t Enough

Image Playground is free, private, and on-device — but it has clear limitations. Here’s when you need cloud alternatives:

Image Playground Is Enough

Cover images, decorative illustrations, concept art, greeting cards, social media graphics, event invitations, presentation backgrounds, avatar creation, brainstorming visuals.

Stylized, non-photorealistic content where the artistic interpretation IS the point.

You Need Cloud APIs

Photorealistic product mockups, virtual home staging, fashion try-on, architectural visualization, medical imaging, brand-accurate marketing materials.

When accuracy to reality matters more than artistic expression. DALL-E 3, Midjourney API, or Stability AI.

Device Requirements and Fallbacks

Image generation requires an A17 Pro chip or later (iPhone 15 Pro+, M1+ Macs). That’s the same requirement as Foundation Models. For older devices:

func checkImageGenerationSupport() -> ImageGenStrategy {
    if let _ = try? await ImageCreator() {
        return .onDevice  // Full image generation
    } else if hasCloudAPIAccess() {
        return .cloud  // Fall back to DALL-E / Stability AI
    } else {
        return .stockImages  // Pre-curated image library
    }
}

enum ImageGenStrategy {
    case onDevice    // Free, private, instant
    case cloud       // Costs money, needs internet
    case stockImages // No generation, pick from library
}

Stock images are a valid fallback. Not every user needs AI-generated images. A curated library of beautiful stock photos, organized by category, is a perfectly good alternative for older devices. Don’t make image generation a hard requirement for your feature.

Combining Image Generation with App Context

The most useful creative features aren’t standalone “generate an image” buttons. They’re woven into existing workflows where visual content adds genuine value:

  • Event invitations — User creates an event, app generates a themed invitation image automatically. The event details (date, location, theme) become the concept.
  • Note covers — Each note gets a visual identity based on its content. A travel note gets a destination illustration. A recipe note gets a food visual.
  • Contact avatars — Generate a stylized avatar for contacts who don’t have a photo. Use the contact’s name and your app’s context to create a personalized illustration.
  • Onboarding illustrations — Generate tutorial images that match your app’s current theme or the user’s content, instead of using generic stock illustrations.
  • Social media cards — Turn a blog post, article summary, or product listing into a shareable social image with AI-generated visuals and overlaid text.

The key insight: generated images should support the user’s goal, not become the goal. Nobody opens your app just to generate images — they open it to plan events, take notes, or manage content. The images make those experiences better.

Real-World Examples

Creative AI features are appearing across the iOS ecosystem:

  • Messages — Image Playground integration lets users create custom images in conversations. The style options match the playful context of messaging.
  • Notes — Image Wand transforms rough sketches into polished illustrations using Image Playground. Circle a sketch, and it becomes a proper image.
  • Keynote — Generate slide illustrations that match your presentation theme. Type a description, get a visual.
  • Freeform — Create brainstorming visuals directly on the canvas. Quick concept images for mood boards and ideation.
  • Localframe — An open-source SwiftUI app demonstrating the ImageCreator API for fully local image generation with a clean SwiftUI interface.

Your Turn

Exercise: Build an AI Mood Board

Build a feature where users describe a project theme and get AI-generated concept images:

  1. Theme input: A text field where the user describes their project (e.g., “minimalist Japanese garden”).
  2. Expand with Foundation Models: Use @Generable to generate 4 visual description variations from the user’s theme. Each should be a different interpretation (different setting, mood, or angle).
  3. Generate images: Use ImageCreator to create an image for each description. Show images in a 2×2 grid as they arrive via AsyncSequence.
  4. Style picker: Let the user switch between Animation, Illustration, and Sketch styles. Regenerate all images in the new style.
  5. Save & share: Tap an image to save to Photos or share via the system share sheet.
  6. Fallback: If ImageCreator isn’t available, show the ImagePlaygroundSheet as a fallback. If neither is available, show a message explaining the device requirement.

Bonus: Add a “Surprise Me” button that picks a random theme from a curated list (nature, city, space, food, abstract) and generates a mood board automatically.
