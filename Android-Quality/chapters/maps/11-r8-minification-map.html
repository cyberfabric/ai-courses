Chapter 11

R8: Minification Without Mysteries

Memory Map
📦

The Big Idea

R8 strips, optimizes, and renames everything not reachable from a manifest entry or a keep rule. Your job: tell it about reflection, JNI, and string-named lookups via proguard-rules.pro. The reward is a 20-40% smaller APK and a faster cold start — if you keep mapping.txt, every release-only crash stays readable.

⚙️ The Four Phases of R8

🗑️ Shrink
strip unreachable
→
⚡ Optimize
inline, merge, propagate
→
🎭 Obfuscate
rename to a.b.C.d
→
✅ Preverify
runtime metadata

🔓 The Two Magic Lines

🪓
isMinifyEnabled = true
Turns on the full R8 pipeline: shrink + optimize + obfuscate.
🖼️
isShrinkResources = true
Strips unused PNG/XML/strings. Requires minifyEnabled to be on first.

👀 The Three Things R8 Can't See

🔍 Reflection
Gson, Moshi, Hilt, Retrofit
🌉 JNI
native ↔ JVM bridge
Needs Keep Rules
📄 XML / strings
custom views, manifests
🏷️ Annotation lookups
@JsonClass, @Inject

Anything reflected, JNI'd, or named-by-string needs a keep rule. Everything else, R8 figures out automatically.

🛡️ Keep-Rule Patterns

PatternWhenExample
-keep class X.** { *; } Sledgehammer; last resort API DTOs, models
-keepclasseswithmembernames Preserve named members of matched classes native <methods>
-keep,allowobfuscation,allowshrinking Signature-only preservation, can still rename Retrofit interface markers
-keepattributes Preserve bytecode metadata SourceFile,LineNumberTable

🗺️ The mapping.txt Pipeline

  1. Build produces mapping.txt

    Located at app/build/outputs/mapping/release/mapping.txt after every R8 build.

  2. Auto-upload to crash reporter

    Crashlytics, Sentry, Bugsnag — all auto-upload via their Gradle plugins. Belt: also commit to a private bucket per release.

  3. Crash arrives obfuscated

    at a.b.C.d(:0) — meaningless on its own.

  4. Retrace via Studio or retrace.jar

    Build → Analyze APK → ProGuard ReTrace, OR run retrace.jar mapping.txt trace.txt.

  5. Read the retraced trace

    First frame in your package = entry point. Walk down to framework boundary.

🚨 Crash → Keep-Rule Diagnostic

Crash typeWhat R8 strippedFix shape
NoSuchMethodError A method name needed by reflection -keepclassmembers class X { methodName(...); }
ClassNotFoundException An entire class looked up by string -keep class com.example.X
NoClassDefFoundError A class referenced by another class but stripped together -keep class com.example.pkg.** { *; }
Retrofit "Unable to invoke ..." API interface methods -keep interface com.example.api.** { *; }

📚 Key Terms

R8
The bundled minifier in Android Gradle Plugin since AGP 3.4 (replaced ProGuard).
Keep rule
A directive in proguard-rules.pro telling R8 not to remove or rename specific classes/members.
mapping.txt
The build-specific file mapping original symbol names to obfuscated names — required to retrace any release-only stack.
Retrace
The act of converting an obfuscated stack back to readable form using mapping.txt.
Consumer ProGuard rules
Keep rules a library ships inside its AAR (under META-INF/proguard/) so its consumers don't have to write them.
Full mode
R8's more aggressive optimization profile, enabled via android.enableR8.fullMode; default in AGP 8+.
Baseline Profile
An AOT-compilation hint shipped alongside an APK that pre-compiles hot code paths for a faster cold start.
fullMode
R8's stricter mode that drops some legacy ProGuard compatibility for better optimization.

🚫 Pitfalls

⏳
"We'll turn it on later"
Later never comes. Enable in a branch, fix the rules, ship. Two hours of work.
🔨
Wildcard -keep
Sledgehammer keeps work but waste bytes and reveal class names. Narrow when you can.
🗑️
Lost mapping.txt
No mapping = unreadable crashes forever. Auto-upload AND commit to private store.
📁
Editing the wrong file
Your rules go in proguard-rules.pro, not proguard-android-optimize.txt (overwritten on AGP upgrades).

🧪 Quick Quiz

What's the minimum config to enable R8 in a release build?

isMinifyEnabled = true + isShrinkResources = true in the release buildType, with proguard-android-optimize.txt + your own proguard-rules.pro in proguardFiles.

Why must mapping.txt be saved per release?

Because the obfuscation is build-specific. A mismatched mapping retraces to wrong lines, not "no line." Lose it and every crash from that build is unreadable forever.

What two attributes should you keep so Crashlytics shows file/line info?

-keepattributes SourceFile,LineNumberTable — without them the retraced trace shows class & method but no line numbers.

What does -keep class com.example.api.dto.** { *; } protect?

All classes in the com.example.api.dto package and its subpackages, with all their fields/methods, from being shrunk or obfuscated — the typical pattern for Gson/Moshi DTOs accessed by reflection.

Where do most modern libraries ship their keep rules?

Inside the AAR at META-INF/proguard/*.pro — R8 picks them up automatically. Inspect via unzip -p library.aar META-INF/proguard/library.pro.

Key Takeaways

  • Two lines turn it on: isMinifyEnabled = true + isShrinkResources = true in the release buildType.
  • R8 strips everything not reachable from a manifest entry or keep rule. Reflection / JNI / string-name lookups need keep rules.
  • Always keep SourceFile,LineNumberTable attributes — or your retraced traces have no line numbers.
  • Most modern libraries ship consumer rules in their AAR. The libraries you write keep rules for are old, small, or your own DTOs/APIs.
  • mapping.txt is non-negotiable. Auto-upload to your crash reporter on every release build. Lose it = crashes unreadable forever.
  • Prefer narrow keeps (-keepclasseswithmembers, package boundaries) over wildcards. Wildcards waste bytes and leak names.
  • Three crash patterns map cleanly to keep rules: NoSuchMethodError → member name; ClassNotFoundException → class; NoClassDefFoundError → whole package.
  • Reusable skill: /draft-keep-rules — turns retraced stacks into the precise rules that fix them.
  • Once R8 is solid, the next perf lever is Baseline Profiles — 20-40% startup gains.
