Chapter 1

Welcome to the CCA-F

Your roadmap to Anthropic's first official certification — what it tests, how it works, and how to pass it

Why This Certification Matters

On March 12, 2026, Anthropic launched the Claude Certified Architect — Foundations (CCA-F). It's the first official technical credential in the Claude ecosystem, and it's not just a badge — it's proof that you can design and ship production Claude systems.

Think of it like this: lots of developers use Claude. Some copy-paste API calls from tutorials. The CCA-F separates the people who understand the architecture from those who just call the endpoint. If you've been building with Claude for six months or more, this is how you prove it.

What Makes the CCA-F Different

This isn't a trivia test about API parameters. Every question is anchored to a real-world scenario — you'll be asked how to design components, integrations, and data flows for production Claude systems. It tests architectural thinking, not memorization.

The certification is available through the Claude Partner Network, which any organization can join for free. The exam costs $99 (free for the first 5,000 Claude Partner Network employees). Anthropic has confirmed additional certifications are coming later in 2026, making CCA-F the entry point of a credential stack.

Exam Format: The Numbers

Let's get the logistics out of the way. Here's exactly what you're signing up for:

60 Multiple-choice questions
120 Minutes to complete
720 Passing score (out of 1,000)

Each question has one correct answer and three distractors. The exam is live proctored and closed-book — no AI assistance, no documentation, no second screen. Just you and 60 architecture decisions.

At 2 minutes per question, you have a comfortable pace — but don't dawdle. Some questions require reading a scenario context and a code snippet, so the time adds up.

The passing score is 720 out of 1,000 on a scaled score. This means not every question is weighted equally — harder questions carry more weight. Don't panic if you're unsure on a few questions. Focus on getting the high-value domain questions right (especially Domain 1 at 27%).

The 5 Competency Domains

The exam organizes everything into five domains. Each domain has a weight that tells you how many questions come from it. Here's the breakdown:

Domain Topic Weight ~Questions
1 Agentic Architecture & Orchestration 27% ~16
2 Tool Design & MCP Integration 18% ~11
3 Claude Code Configuration & Workflows 20% ~12
4 Prompt Engineering & Structured Output 20% ~12
5 Context Management & Reliability 15% ~9

Notice the lopsided weighting. Domain 1 alone is over a quarter of the exam. If you only have a weekend to cram, spend it on agentic architecture. Domains 3 and 4 together are 40% — another huge chunk. Domain 5 is the smallest, but don't skip it — context management questions are tricky because they test nuance.

Study strategy: weight your prep time like the exam weights its questions. If Domain 1 is 27%, roughly 27% of your study time should go there. Many first-time candidates over-study prompt engineering (Domain 4) because it feels familiar, and under-study orchestration patterns (Domain 1) because they seem abstract. Don't make that mistake.

The 6 Exam Scenarios

Here's what makes the CCA-F unique: every question is anchored to one of six real-world scenarios. You won't see generic "which API parameter does X" questions. Instead, you'll see "In the Customer Support Agent scenario, which orchestration pattern best handles..."

The six scenarios are:

1. Customer Support Resolution Agent

A multi-step agent that handles customer tickets — classifying issues, looking up account info, searching knowledge bases, and routing to human agents when needed. Tests routing, escalation patterns, tool design, and long-conversation context management.

2. Code Generation with Claude Code

Using Claude Code to generate, refactor, and review code in a development workflow. Tests CLAUDE.md configuration, worktree usage, permission models, and how to structure prompts for code output.

3. Multi-Agent Research System

A system where multiple specialized agents collaborate to research a topic — one searches, one analyzes, one summarizes. Tests task decomposition, map-reduce patterns, agent communication, and result aggregation.

4. Developer Productivity with Claude

Integrating Claude into a developer's daily workflow — MCP servers for internal tools, hooks for automation, custom skills. Tests MCP integration, Claude Code configuration, and workflow automation.

5. Claude Code for CI/CD

Running Claude Code in headless mode (claude -p) inside CI/CD pipelines — automated code review, test generation, deployment checks. Tests non-interactive execution, security considerations, and pipeline integration.

6. Structured Data Extraction

Extracting structured information from unstructured documents — invoices, contracts, reports. Tests JSON schema enforcement, validation loops, extraction pipelines, and error handling for malformed output.

Critical detail: you won't see all 6 scenarios. Each exam sitting randomly selects 4 of the 6 scenarios. This means you can't skip studying two scenarios and hope for the best — you need to know all six. The randomization ensures no two exam sittings are identical.

How Scenarios Map to Domains

Each scenario touches multiple domains. Here's a rough map to help you see the connections:

Scenario Primary Domains
Customer Support Agent Domain 1 (orchestration), Domain 2 (tools), Domain 5 (context)
Code Generation Domain 3 (Claude Code), Domain 4 (prompts), Domain 1 (architecture)
Multi-Agent Research Domain 1 (orchestration), Domain 5 (reliability), Domain 2 (tools)
Developer Productivity Domain 3 (Claude Code), Domain 2 (MCP), Domain 4 (prompts)
CI/CD Domain 3 (Claude Code), Domain 5 (reliability), Domain 1 (architecture)
Data Extraction Domain 4 (structured output), Domain 2 (tools), Domain 5 (context)

See the pattern? Domain 1 (Agentic Architecture) shows up everywhere. That's why it's 27% of the exam. If you deeply understand orchestration patterns, you'll have a foundation for answering questions across most scenarios.

Prerequisites: What You Need Before Starting

The CCA-F isn't a beginner certification. Anthropic expects candidates to have:

  • 6+ months of hands-on experience with the Claude API, Claude Code, or the Agent SDK
  • Python familiarity — code examples in the exam are Python, JSON, and YAML
  • Practical experience building production Claude applications — including RAG systems, tool-enabled agents, and real integrations
  • Understanding of LLM limitations and AI system evaluation

If you've been using Claude through the web interface but haven't touched the API, you'll want to build 2–3 real projects first. The exam tests architectural decisions that only make sense when you've felt the pain of context window limits, flaky tool calls, and agent orchestration gone wrong.

If you're not sure whether you're ready, try this: can you explain the difference between tool_use and tool_result message types in the API? Can you describe when you'd use a hub-and-spoke pattern vs a pipeline? If those feel foreign, spend a few weeks building before booking the exam.

Study Timeline Recommendations

How long should you study? It depends on your starting point:

If you build with Claude daily (2–4 weeks)

You already know the API, you've configured Claude Code, and you've built agent workflows. Focus on the exam format — take the practice exam, identify gaps in your domain knowledge, and do targeted review. Most developers who passed with 2–4 weeks of prep fall in this category.

If you've used Claude but haven't built production systems (6–10 weeks)

You know the basics but need to deepen your architectural understanding. Work through this course chapter by chapter, build at least one multi-agent project, and take the practice exam at the 4-week mark to calibrate.

If you're newer to Claude (2–4 months)

Start with the Anthropic Academy courses, build 2–3 real projects (a RAG system, a tool-enabled agent, a Claude Code workflow), then come back to this course for exam-specific prep.

The #1 study mistake: reading about architecture without building anything. The exam tests decision-making, not recall. You need to have felt the pain of choosing the wrong orchestration pattern or designing a tool schema that confuses the model. Build first, study second.

Exam-Day Logistics

Here's what to expect on exam day:

  • Proctoring: Live proctored via webcam. You'll need a quiet room, a working webcam, and a government-issued ID.
  • Closed-book: No documentation, no AI tools, no second monitor. Close everything except the exam window.
  • No going back: Once you answer a question and move on, you typically can't go back. Read carefully before clicking.
  • Scenario context: Each question includes its scenario context, so you don't need to memorize scenario details — but knowing them speeds up your reading.
  • Results: You'll get your score shortly after completing the exam. The breakdown shows your performance per domain.
Time management matters. With 60 questions in 120 minutes, you have exactly 2 minutes per question. Some scenario-based questions have long setups that eat into your time. If a question is taking more than 3 minutes, mark your best guess and move on. You can't afford to spend 5 minutes on a single question.

Your Exam-Day Strategy Cheat Sheet

Here's a quick strategy framework to keep in mind as you study:

The 3-Pass Approach

Pass 1 (first 60 min): Answer every question you're confident about. Skip anything that takes more than 2 minutes. This builds momentum and ensures you don't miss easy points.

Pass 2 (next 40 min): Return to skipped questions. With the pressure of "first pass" gone, you'll often see answers more clearly.

Pass 3 (final 20 min): Review flagged questions. Double-check that you didn't misread any scenario contexts. Trust your first instinct unless you find a clear reason to change.

Common Misconceptions

Before we dive into the technical chapters, let's clear up some things that trip people up:

  • "It's just an API quiz" — Wrong. The CCA-F tests system design and architectural tradeoffs. Knowing every parameter of the Messages API won't save you if you can't reason about orchestration patterns.
  • "I need to memorize MCP specs" — You need to understand MCP architecture (hosts, clients, servers, transports) and know when to use which transport type. You don't need to memorize JSON-RPC message formats.
  • "Prompt engineering is the biggest domain" — It's 20%, tied with Claude Code. Agentic Architecture at 27% is the biggest. Adjust your study time accordingly.
  • "I can skip Claude Code — I use the API directly" — Claude Code is 20% of the exam. You need to know CLAUDE.md, hooks, worktrees, and CI/CD integration even if you don't use them daily.
Exercise: Map Your Knowledge Gaps

Before reading further, do a quick self-assessment. For each domain below, rate yourself 1–5 (1 = "never heard of it", 5 = "I could teach this"):

  1. Agentic Architecture & Orchestration (hub-and-spoke, pipelines, map-reduce)
  2. Tool Design & MCP Integration (tool schemas, MCP servers, transports)
  3. Claude Code Configuration & Workflows (CLAUDE.md, hooks, worktrees, CI/CD)
  4. Prompt Engineering & Structured Output (few-shot, chain-of-thought, JSON schemas)
  5. Context Management & Reliability (token budgets, compaction, escalation)

Multiply each score by the domain weight (27%, 18%, 20%, 20%, 15%). Your lowest weighted scores are where to focus first. For example, if you scored 2 on Domain 1: 2 × 27 = 54 — that's a critical gap in the highest-weighted domain.
