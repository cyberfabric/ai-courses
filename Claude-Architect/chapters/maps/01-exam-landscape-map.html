The Big Idea

The CCA-F tests architectural thinking, not memorization. Every question is anchored to a real-world scenario — you need to design systems, not recall API parameters.

📅 Study Timeline

  1. 🚀 Daily Claude Builders — 2–4 Weeks

    You already use the API, Claude Code, and agent workflows daily. Focus on exam format, take the practice exam, fill domain gaps.

  2. 🔧 Intermediate Users — 6–10 Weeks

    You know the basics but need architectural depth. Work through each chapter, build a multi-agent project, practice exam at week 4.

  3. 🌱 Newer to Claude — 2–4 Months

    Start with Anthropic Academy courses. Build 2–3 real projects (RAG, tool-enabled agent, Claude Code workflow), then do exam prep.

🧪 Quick Quiz

How many questions are on the CCA-F exam?

60 multiple-choice questions, each with one correct answer and three distractors.

Which domain has the highest weight on the exam?

Domain 1: Agentic Architecture & Orchestration at 27% (~16 questions).

How many of the 6 scenarios appear on each exam sitting?

4 out of 6, randomly selected. You can't skip studying any scenario.

What's the passing score?

720 out of 1,000 (scaled scoring — harder questions are worth more).

What prerequisites does Anthropic recommend?

6+ months hands-on experience with Claude API, Claude Code, Agent SDK, and MCP. Plus Python familiarity.

⚠️ Common Misconceptions

❌
"It's just an API quiz"
Nope. It tests system design and architectural tradeoffs, not parameter recall.
❌
"Memorize MCP specs"
Understand MCP architecture (hosts, clients, servers, transports) — not JSON-RPC formats.
❌
"Prompts are biggest domain"
Prompt Engineering is 20%. Agentic Architecture at 27% is the real heavyweight.
❌
"I can skip Claude Code"
Claude Code is 20% of the exam. CLAUDE.md, hooks, worktrees, CI/CD — all fair game.

🔑 Key Takeaways

  • CCA-F: 60 questions, 120 minutes, 720/1000 to pass, live proctored, closed-book.
  • 5 domains with unequal weights — Domain 1 (Agentic Architecture) is king at 27%.
  • 6 scenarios, 4 randomly selected per sitting — study all six, no shortcuts.
  • Weight your study time like the exam weights its questions — more time on Domain 1.
  • Build first, study second — the exam tests decision-making from real experience, not recall.
