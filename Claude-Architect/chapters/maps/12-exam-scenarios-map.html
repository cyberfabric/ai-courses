The Big Idea

The exam favors deterministic, architectural answers over probabilistic, prompt-based ones. When stuck between two choices, pick the one that uses code-level enforcement (hooks, schemas, structured patterns) — not prompt-level guidance.

📊 Domain Priority When Short on Time

  1. Domain 1: Agentic Architecture (27%)

    Most questions, most points. Agent loops, hub-and-spoke, context isolation, hooks vs prompts.

  2. Domain 3: Claude Code (20%)

    Very specific, high-scoring. CLAUDE.md, settings, hooks, worktrees, claude -p.

  3. Domain 4: Prompt & Structured Output (20%)

    Appears across all scenarios. Few-shot, system prompts, constrained decoding, validation loops.

  4. Domain 2: Tool Design & MCP (18%)

    MCP details are testable. Transports, tool descriptions, .mcp.json configuration.

  5. Domain 5: Context & Reliability (15%)

    Fewest questions but tricky. Compaction, escalation handoffs, graceful degradation.

🧪 Quick Quiz

What context should each subagent receive in Multi-Agent Research?

Only the specific sub-question and relevant instructions. Never the full orchestrator history — that's context pollution. (Context isolation)

How should a $500 refund limit be enforced?

PreToolUse hook on the refund tool that validates the amount. Not system prompt (probabilistic), not tool description (informational only).

What transport for a shared team MCP server?

Streamable HTTP — supports remote, multi-client access. stdio is local-only. SSE is deprecated.

How should the agent loop decide to continue executing?

Check stop_reason === "tool_use". Never parse the response text or check confidence scores.

You're stuck between two answers. What's the tiebreaker?

"Which answer is more deterministic and architectural?" Code-level enforcement (hooks, schemas) beats prompt-level guidance every time.

📖 Key Terms

3-Pass Approach
Exam time strategy: easy (60 min) → medium (40 min) → review (20 min). Skip >2 min questions on first pass.
Scenario-Domain Matrix
Map of which domains each scenario tests. Domain 1 appears in the most scenarios regardless of which 4 are selected.
Context Isolation
Subagents receive only task-specific context, never the full orchestrator history.
Cold Handoff
Anti-pattern: escalating without summary/context. Always include actions taken + recommendation.
Deterministic vs Probabilistic
The exam's golden rule: hooks/schemas/code > prompts/confidence/sentiment for enforcement.
stop_reason
The only correct way to control agent loop continuation. Check for "tool_use", never parse text.
Fan-out/Fan-in
Parallel decomposition pattern. Synthesize results (don't concatenate). 3–7 sub-tasks is the sweet spot.

📌 Key Takeaways

  • 60 questions, 120 minutes, 720/1000 to pass. 4 of 6 scenarios randomly selected — study all 6.
  • Domain 1 (27%) is highest priority — appears in 2–3 scenarios regardless of selection.
  • The exam favors deterministic answers: hooks > prompts, schemas > freeform, stop_reason > text parsing.
  • 3-pass strategy: easy first (60 min), medium second (40 min), review last (20 min).
  • Never leave questions blank — no penalty for guessing, 25% odds beat 0%.
  • Top traps: sentiment escalation, full context to subagents, prompts for business rules, stdio for remote servers.
  • When stuck: "Which answer is more deterministic and architectural?" — that's the correct one.
