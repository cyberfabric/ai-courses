Chapter 6

MCP: The Universal Adapter

Memory Map
🔌

The Big Idea

MCP is USB for AI. One standard protocol replaces N×M custom integrations with N+M — build one server per tool, one client per app, and everything connects.

🏗️ The Three Participants

🖥️ Host
AI app (Claude Code, Desktop)
🔗 Client
One per server connection
MCP
Architecture
⚙️ Server
Provides tools, resources, prompts
📡 Transport
stdio or Streamable HTTP

🧩 The Three Primitives

🔧
Tools (Actions)
Executable functions the AI invokes. Verbs — "query_database", "send_message". Have name, description, inputSchema.
📄
Resources (Data)
Read-only data sources for context. Nouns — database schemas, config files, API docs. Identified by URI.
📝
Prompts (Templates)
Reusable prompt templates with parameters. Pre-built conversation starters like "analyze this table".

🚚 Transport Types

AspectstdioStreamable HTTP
🌍 LocationLocal processRemote / network
👥 ClientsSingle clientMultiple clients
🔐 AuthNone (inherits user perms)Bearer tokens, API keys, OAuth
⚡ SpeedFaster (no network)Network overhead
📦 ExamplesFilesystem, local DBSentry, Slack, cloud APIs
Exam signal words: "shared", "different machines", "team-wide" → Streamable HTTP. "Local", "same machine", "dev database" → stdio. SSE is deprecated — Streamable HTTP replaces it.

⚙️ Configuration: Two Levels

👥
.mcp.json
Project-level. Committed to VCS. Team-shared servers. Use ${VAR} for secrets.
👤
~/.claude.json
User-level. Not committed. Personal or experimental servers. Won't affect teammates.

🔐 Security Boundaries

🛡️
Least Privilege
Scope servers narrowly. Read-only DB access unless writes are required. Filesystem scoped to project dir.
🔑
Credentials
Always environment variables via ${VAR}. Never hardcode in config files.
🧱
Server Granularity
Prefer many small servers over one big one. Independent failure domains, precise permissions.

🧮 The N×M → N+M Reduction

❌
Without MCP
4 apps × 6 services = 24 custom integrations. Every app writes custom code for every tool.
✅
With MCP
4 apps + 6 servers = 10 integrations. Each tool builds one server, each app builds one client.

⚖️ MCP vs. Direct Tool Integration

Use MCP When...Use Direct Tools When...
Multiple AI apps need the same toolOnly one app uses the tool
Team needs to share tool configsTool is specific to one agent's role
Standard discovery & lifecycle neededTight control over tool behavior
Tool connects to external systemTool is pure computation (no I/O)

📖 Key Terms

MCP Host
The AI application that creates clients and coordinates connections (Claude Code, VS Code).
MCP Client
A component inside the host — one per server connection. Handles lifecycle and messaging.
MCP Server
A program exposing tools, resources, and/or prompts. Can be local or remote.
stdio Transport
Standard I/O streams for local servers. Fast, no auth, single client.
Streamable HTTP
HTTP POST + SSE streaming for remote servers. Multi-client, auth support.
JSON-RPC 2.0
The protocol format for all MCP messages — same regardless of transport.
Capability Negotiation
Client and server agree on supported features during initialization.
Notifications
Server pushes changes (e.g., new tools added) so clients adapt without restarting.

🎯 Scenario Connections

🎧
Customer Support
MCP servers for ticketing (tools: CRUD) and knowledge base (tools: search, resources: article index).
💻
Dev Productivity
Primary MCP scenario. Connect Claude to CI, deployment, monitoring via MCP servers.
📊
Data Extraction
MCP for data sources. Resources provide schemas; tools perform extraction.
🚀
CI/CD
MCP servers for test runners, linters, deploy tools. Config via .mcp.json in repo.

❓ Quick Quiz

An org has 5 AI apps and 8 external services. How many integrations without MCP vs. with MCP?

Without: 5 × 8 = 40. With: 5 + 8 = 13. MCP reduces N×M to N+M.

A team wants to share an MCP server across machines. Which transport?

Streamable HTTP. "Shared" + "across machines" = remote transport. stdio is for local, single-client only.

Where should a developer add an experimental MCP server that shouldn't affect teammates?

~/.claude.json (user-level). Not .mcp.json, which is committed and shared.

What are the three MCP primitives, and what type is each?

Tools (actions/verbs), Resources (data/nouns), Prompts (templates). A good server combines all three.

Should an MCP server for a production database have full read-write access?

No. Principle of least privilege — read-only unless writes are explicitly required. Credentials via env vars, never hardcoded.

🎯 Key Takeaways

  • MCP = USB for AI. Solves N×M with N+M. One protocol, everything connects.
  • Three participants: Host (app) → Client (connector) → Server (provider). One client per server.
  • Three primitives: Tools (verbs), Resources (nouns), Prompts (templates).
  • Two transports: stdio (local, fast, single client) vs. Streamable HTTP (remote, auth, multi-client).
  • Two config levels: .mcp.json (team, VCS) vs. ~/.claude.json (personal). Always use ${VAR} for secrets.
  • Security: least privilege, small focused servers, env vars for credentials, scope filesystem access.
  • Lifecycle: Initialize → Negotiate → Discover → Exchange → Terminate.
