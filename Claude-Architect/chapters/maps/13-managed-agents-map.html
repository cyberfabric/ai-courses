The Big Idea

Managed Agents decouples the brain (Claude reasoning) from the hands (container execution) with a durable session log in between. Each scales and recovers independently — turning agent harnesses from "pets" into "cattle."

📖 Key Terms

Four Pillars
Agent (config), Environment (container template), Session (running instance), Events (communication).
Brain/Hands Decoupling
Reasoning (stateless harness) separated from execution (containers). Each scales and recovers independently.
Session Log
Append-only, durable event store outside the context window. Supports positional queries for selective retrieval.
agent_toolset_20260401
Enables all built-in tools: bash, file ops, web search, web fetch. One tool type for the full set.
Pets vs Cattle
Analogy: traditional harnesses are "pets" (hand-maintained). Managed Agents makes them "cattle" (interchangeable, disposable).
Resource-Bundled Auth
Credentials wired into container setup (e.g., git tokens). Agent never handles them directly.
Vault-Proxied Access
OAuth tokens stored in external vaults. A proxy makes calls on the agent's behalf — tokens never enter the sandbox.
Event Steering
Sending additional user events mid-session to redirect or guide the agent while it's running.

🧪 Quick Quiz

A company has strict data residency requirements. Managed or self-hosted?

Self-hosted (Messages API + Agent SDK). Managed Agents processes data on Anthropic's infrastructure, which won't satisfy data residency compliance.

What happens when a container crashes during tool execution?

The failure is passed to Claude as a tool-call error. Claude can retry, reinitialize, or take an alternative approach. No information is lost — the session log is durable.

What's the primary benefit of brain/hands decoupling?

Independent scaling and failure recovery. Reasoning and execution scale separately. If the harness fails, a new instance picks up from the session log.

Sub-agents need different packages (Python vs Node.js). What's the correct approach?

Create separate environment templates for each agent type. Environments are templates — optimize each for its agent's needs.

Where do stable instructions go vs task-specific context?

Stable instructions (role, constraints, format) → agent system prompt. Task-specific context (the ticket, the diff) → session's first user event. Mirrors CLAUDE.md vs conversation in Claude Code.

📌 Key Takeaways

  • Four pillars: Agent (config) → Environment (container) → Session (instance) → Events (communication).
  • Brain/hands decoupling: reasoning and execution scale independently. Session log provides durability.
  • Managed for async/long-running/rapid prototyping. Self-hosted for custom loops, data residency, edge deployment.
  • All 5 exam domains map directly: orchestration, tools, Claude Code parallels, prompt design, reliability.
  • Security: never expose credentials to the agent. Use resource-bundled auth or vault-proxied access.
  • Container failures become tool-call errors — Claude decides to retry, reinitialize, or escalate.
  • Beyond the core: Memory now in public beta (file-backed, scoped); multi-agent and outcomes still research preview.
  • Beta header managed-agents-2026-04-01 required on every Managed Agents API call (SDKs set it automatically).
