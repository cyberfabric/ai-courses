The Big Idea

Claude Code isn't just an interactive tool — it's an operational runtime. Plan mode for thinking, worktrees for isolation, headless mode for CI/CD, and the dual-agent pattern for safe automation. Domain 3's workflow half lives here.

🧩 Core Workflow Concepts

🗺️
Plan Mode
Read-only analysis. Claude explores and proposes but cannot write or execute. Enter with --permission-mode plan or Shift+Tab×2.
🌳
Git Worktrees
Isolated copies of the repo for parallel sessions. --worktree name creates a branch at .claude/worktrees/. No file conflicts.
🤖
Headless Mode (-p)
Non-interactive execution. Output to stdout. The foundation of CI/CD integration. Always pair --max-turns with --max-budget-usd.
⚙️
CI/CD Integration
anthropics/claude-code-action@v1 in GitHub Actions. API key in Secrets. Triggers on PR events or cron schedules.
👥
Dual-Agent Pattern
Initializer (plan mode, read-only) creates tasks → Worker agents (isolated worktrees) execute in parallel. Hub-and-spoke for Claude Code.
⏰
Scheduled Agents
Routines (cloud, research preview), desktop tasks, GitHub Actions cron, or /loop. Routines combine scheduled, API, and GitHub triggers in one config.
💾
Session Management
--continue resumes last session. --resume name for named sessions. /branch forks a conversation like git branch.
💰
Cost Control
--max-turns caps iterations. Effort levels adjust thinking tokens. Model selection (Sonnet vs Opus) for cost/quality tradeoff.

👥 Dual-Agent CI/CD Pipeline

  1. 🔍 Initializer Agent (Plan Mode)

    Read-only. Analyzes the PR, categorizes changes, creates a structured task list. Broad read access, no write access.

  2. 🔀 Task Decomposition

    Initializer outputs JSON tasks: review agent, test agent, security agent — each with a focused prompt and scope.

  3. ⚡ Worker Agents (Parallel Worktrees)

    Each worker runs in its own worktree with isolation: worktree. Limited write access, focused scope. Runs simultaneously.

  4. 📋 Aggregator

    Combines findings, posts a unified PR comment, decides block/approve merge.

📖 Key Terms

Plan Mode
Permission mode where Claude can read and analyze but cannot write files or execute commands. --permission-mode plan.
Worktree
An isolated git working directory. Each Claude session gets its own branch and file copy at .claude/worktrees/.
Headless Mode (-p)
Non-interactive execution — processes prompt, outputs to stdout, no human in the loop. The CI/CD foundation.
--max-turns
Caps the number of agent loop iterations. Pair with --max-budget-usd (hard dollar ceiling per -p command) for unattended execution.
Routines
Saved Claude Code configs that run on Anthropic-managed cloud (research preview, April 2026). Three trigger types: scheduled cron, API endpoint, GitHub events. Manage at claude.ai/code/routines or via /schedule.
Dual-Agent Pattern
Initializer agent plans in read-only mode → spawns focused worker agents in isolated worktrees. Safer than one agent with full access.
claude-code-action@v1
Official GitHub Action that runs Claude Code in a CI runner. API key via ${{ secrets.ANTHROPIC_API_KEY }}.
Session Resume
--continue (last session) or --resume name (named session). Restores full conversation history and state.
/branch
Forks the current conversation into a new session — explore alternatives without losing the original thread.

🧪 Quick Quiz

A developer wants Claude to analyze a large codebase and propose a refactoring strategy without making changes. Which mode?

Plan mode (--permission-mode plan). The key phrase is "without making changes" — plan mode is read-only.

Multiple agents need to work on different features simultaneously in the same repo. What feature prevents conflicts?

Git worktrees (--worktree name). Each session gets an isolated copy. Subagents use isolation: worktree in frontmatter.

How should the API key be provided to Claude Code in GitHub Actions?

GitHub Secrets — ${{ secrets.ANTHROPIC_API_KEY }}. Never hardcode, commit to .env, or put in .mcp.json.

What flags prevent runaway costs when running Claude in CI/CD?

--max-turns caps agent loop iterations; --max-budget-usd caps the dollar amount for the command. Pair them for unattended execution.

In the dual-agent pattern, what's the key difference between the initializer and workers?

The initializer runs in plan mode (read-only, broad access) and produces a task list. Workers run in isolated worktrees with focused write access and limited scope.

🎯 Key Takeaways

  • Plan mode = read-only analysis. Use --permission-mode plan. The plan-first pattern is a best practice for complex changes.
  • Worktrees = parallel isolation. --worktree name for sessions, isolation: worktree for subagents. Auto-cleanup if no changes.
  • Headless mode (-p) = CI/CD foundation. Three output formats: text, json, stream-json. Pair --max-turns with --max-budget-usd.
  • CI/CD = anthropics/claude-code-action@v1 + API key in GitHub Secrets + --max-turns + --max-budget-usd for cost control.
  • Routines (research preview) = saved cloud configs with scheduled, API, or GitHub triggers — manage at claude.ai/code/routines or via /schedule.
  • Dual-agent pattern = initializer (plan mode) + workers (worktrees). Separates planning from execution for safer automation.
