The AI Family Tree — A Deeper Look

Decision boundaries, feature spaces, and what mathematically distinguishes each ML approach

Companion: This expands on AI-Intro Ch 1: What Is AI, Really?. If you haven’t read it yet, start there for the big-picture overview.

In AI-Intro, we painted the family tree of AI with broad strokes: AI contains Machine Learning, which contains Deep Learning, which contains LLMs. You got the what. Now let’s see the how — with actual numbers, step-by-step computations, and the math that turns out to be way less scary than it sounds.

By the end of this chapter, you’ll be able to classify a data point by hand, trace through a k-means clustering algorithm, and understand exactly what a “decision boundary” is. Let’s go.

The Family Tree, Revisited

Here’s the hierarchy again, but this time we’ll note what mathematically separates each level:

Level What It Is What Makes It Different
AI Any system that appears intelligent Can be hand-coded rules — no learning required
Machine Learning Systems that learn from data Learns a function from inputs to outputs
Deep Learning ML using neural networks with many layers Learns its own features automatically
LLMs DL models trained on text Autoregressive, sequence-to-sequence, transformer-based

The key insight: as you move down the tree, the model takes on more of the work. Classical ML needs you to hand-pick features. Deep learning discovers them on its own. LLMs discover features and generate output one token at a time.

Decision Boundaries: What a Classifier Actually Learns

When you train a classifier — say, “is this email spam or not?” — the model is learning to draw a line (or curve, or surface) through your data that separates one category from another. That line is called a decision boundary.

Decision Boundary

A decision boundary is the line (or surface) that a classifier draws through feature space to separate different categories. Everything on one side is “Category A”; everything on the other side is “Category B.”

In 2D, it’s a line. In 3D, it’s a plane. In higher dimensions, it’s a hyperplane — but the idea is exactly the same: draw a boundary, and classify based on which side you land on.

Let’s make this concrete. Imagine you’re classifying animals into “cat” or “dog” using just two features: weight (kg) and height (cm).

Animal Weight (kg) Height (cm) Label
Whiskers 4 25 Cat
Mittens 5 28 Cat
Rex 20 55 Dog
Buddy 25 60 Dog

If you plot these on a 2D grid (weight on the x-axis, height on the y-axis), the cats cluster in the bottom-left and the dogs cluster in the top-right. A straight line somewhere in the middle separates them perfectly. That line is the decision boundary.

A new animal comes in at weight 10, height 35. Which side of the line does it fall on? That’s the model’s prediction.

Feature Engineering vs. Learned Features

Here’s where classical ML and deep learning diverge:

Classical ML: You Pick the Features

You decide: “I’ll use weight and height to classify animals.” You compute those features yourself, organize them into a table, and feed them to the algorithm. If you pick bad features, the model fails — no matter how good the algorithm is.

Analogy: Giving someone a recipe and ingredients. The quality of the meal depends on how good your recipe is.

Deep Learning: The Model Finds Features

You feed in raw data (pixels, audio samples, text characters) and the network discovers useful features on its own. Early layers might detect edges; later layers detect shapes; even later layers detect objects. You never told it about edges — it figured that out.

Analogy: Giving someone thousands of finished dishes and saying “figure out the recipes yourself.”

This is the fundamental advantage of deep learning: it eliminates the feature engineering bottleneck. But it comes at a cost — you need much more data and compute.

Supervised Learning with Numbers

Let’s trace through the most common type of ML: supervised learning. The idea is simple — you show the model labeled examples, and it learns the pattern.

Here’s a tiny dataset with 2 features and 3 samples. We want to predict whether a student passes (1) or fails (0) based on hours studied and hours slept:

Student Hours Studied (x1) Hours Slept (x2) Passed? (y)
A 2 9 1 (yes)
B 1 4 0 (no)
C 3 7 1 (yes)

A linear classifier tries to find weights w1 and w2 and a bias b such that:

w1 × x1 + w2 × x2 + b > 0 → predict Pass
w1 × x1 + w2 × x2 + b ≤ 0 → predict Fail

The training process adjusts w1, w2, and b until these rules work for all training examples. We’ll see exactly how that adjustment works in Chapter 4. For now, let’s assume training found: w1 = 0.5, w2 = 0.3, b = -2.0.

Classifying a 2D Point with a Linear Boundary

Let’s verify our trained weights work on the training data. The rule: if w1×x1 + w2×x2 + b > 0, predict Pass.

Weights: w1 = 0.5, w2 = 0.3, b = -2.0

Student x1 x2 w1×x1 w2×x2 + b Sum Prediction Actual
A 2 9 1.0 2.7 -2.0 1.7 Pass (>0) Pass ✓
B 1 4 0.5 1.2 -2.0 -0.3 Fail (≤0) Fail ✓
C 3 7 1.5 2.1 -2.0 1.6 Pass (>0) Pass ✓

All three predictions match the actual labels. The line 0.5x1 + 0.3x2 - 2.0 = 0 is the decision boundary that separates passing from failing students in our 2D feature space.

What Do Those Weights Mean?

The weight w1 = 0.5 for “hours studied” and w2 = 0.3 for “hours slept” tell us: studying matters a bit more than sleeping for passing the exam (in this tiny dataset). The bias b = -2.0 sets the bar — you need enough of both to overcome that negative threshold.

Every linear classifier works this way: weights say how much each feature matters, the bias sets the baseline, and the sign of the result gives the prediction.

Quick Check

A new student studied 4 hours and slept 5 hours. Using our weights (w1=0.5, w2=0.3, b=-2.0), would the model predict pass or fail?

Answer: 0.5×4 + 0.3×5 - 2.0 = 2.0 + 1.5 - 2.0 = 1.5. Since 1.5 > 0, the model predicts Pass.

Unsupervised Learning: K-Means Step by Step

Supervised learning needs labeled data. But what if you have data with no labels and just want to find natural groups? That’s unsupervised learning, and the most intuitive algorithm is k-means clustering.

K-Means Clustering

K-means groups data points into k clusters by repeating two steps:

  1. Assign each point to the nearest cluster center (centroid)
  2. Update each centroid to the average of its assigned points

Repeat until the centroids stop moving. That’s it.

Let’s trace this with k = 2 clusters and 6 data points on a 2D plane:

Point x y
P112
P221
P323
P487
P598
P689

K-Means: Iteration by Iteration

Setup: We randomly initialize two centroids:

  • Centroid A = (1, 2) (happens to be P1)
  • Centroid B = (9, 8) (happens to be P5)

Iteration 1 — Assign: For each point, compute the distance to each centroid and assign to the nearest one. We’ll use the squared Euclidean distance (no need for square roots — the ordering is the same):

Point (x, y) Dist² to A (1,2) Dist² to B (9,8) Assigned
P1 (1, 2) 0 100 A
P2 (2, 1) 2 98 A
P3 (2, 3) 2 74 A
P4 (8, 7) 74 2 B
P5 (9, 8) 100 0 B
P6 (8, 9) 98 2 B

Example calculation for P2: dist² to A = (2-1)² + (1-2)² = 1 + 1 = 2

Iteration 1 — Update centroids:

  • Cluster A has P1(1,2), P2(2,1), P3(2,3) → new centroid = ((1+2+2)/3, (2+1+3)/3) = (1.67, 2.0)
  • Cluster B has P4(8,7), P5(9,8), P6(8,9) → new centroid = ((8+9+8)/3, (7+8+9)/3) = (8.33, 8.0)

Iteration 2 — Assign: Recalculate distances to the updated centroids. Since the two groups are clearly separated, every point stays in the same cluster. The centroids don’t move → converged!

Result: Cluster A = {P1, P2, P3} (the “low” group) and Cluster B = {P4, P5, P6} (the “high” group). The algorithm found two natural clusters with no labels at all.

What Just Happened?

K-means found structure in unlabeled data by simply measuring distances and averaging positions. Nobody told it “these are two groups” — it discovered the grouping on its own. This is the essence of unsupervised learning: finding patterns without being told what to look for.

Reinforcement Learning: The Reward Loop

There’s a third flavour of ML that’s neither supervised nor unsupervised. In reinforcement learning (RL), an agent takes actions in an environment and gets rewards or penalties. Over time, it learns which actions lead to the best outcomes.

Think of a tiny grid world:

Col 0 Col 1 Col 2
Row 0 Start    
Row 1   Pit (-10)  
Row 2     Goal (+10)

The agent starts at (0,0) and can move up, down, left, or right. Each move costs -1 (time penalty). The pit at (1,1) gives -10. The goal at (2,2) gives +10.

At first, the agent moves randomly. Sometimes it stumbles into the pit, sometimes it finds the goal. Over many episodes, it learns: “going right-right-down-down gets me to the goal fastest while avoiding the pit.” That’s reinforcement learning — trial, error, and reward signals.

Where RL Shows Up

RL is how game-playing AIs (AlphaGo, OpenAI Five) learned to play. It’s also a key ingredient in RLHF (Reinforcement Learning from Human Feedback), which is how ChatGPT and Claude learned to be helpful assistants. We’ll cover RLHF in detail in Chapter 12.

The Three Flavours: A Summary

Type Input Signal Goal Example
Supervised Data + labels Correct answers Learn the mapping Spam detection
Unsupervised Data only None Find patterns Customer segmentation
Reinforcement Environment Rewards/penalties Maximize reward Game playing, RLHF

What Makes Neural Networks “Deep”

The classifier we built earlier used a single line to separate data. That works when the boundary between categories is simple. But what about data that can’t be separated by a straight line?

The answer: stack layers. Each layer transforms the data, and by the time it reaches the output, even tangled, messy data has been reshaped into something linearly separable.

The deep learning trick: One layer draws simple boundaries. Two layers can combine them into curves. Three layers can carve out arbitrary regions. More layers = more complex decision surfaces. That’s what “deep” means — many layers of transformation.

The Feature Hierarchy

As data flows through the layers of a deep network, each layer builds on the previous one:

Layer 1

Detects edges, simple patterns

→

Layer 2

Combines edges into shapes, textures

→

Layer 3

Combines shapes into parts (eyes, ears)

→

Layer 4+

Combines parts into objects, concepts

For a text model, the progression is similar: character patterns → word fragments → grammar structures → meaning and context.

Nobody programmed these features. The network discovered them during training, layer by layer. This is why deep learning is so powerful — and why it needs so much data. It has to learn every level of abstraction from scratch.

Why LLMs Are a Specific Type of Deep Learning

Not all deep learning models are LLMs. What makes an LLM special?

Property General Deep Learning LLMs Specifically
Input Any data (images, audio, tabular) Sequences of text tokens
Architecture CNNs, RNNs, Transformers, etc. Transformer (decoder-only, usually)
Training objective Varies (classify, detect, generate) Predict the next token
Generation style Varies Autoregressive (one token at a time)
Scale Millions to billions of params Billions to trillions of params

An LLM is a deep learning model that:

  1. Is built on the Transformer architecture (Chapters 6-7)
  2. Is trained on massive text corpora to predict the next token
  3. Generates text autoregressively — one token at a time, each prediction feeding into the next

Every concept we cover in the next 12 chapters — neurons, layers, attention, training — applies to LLMs. They’re just deep learning models with a particular architecture and training goal.

Putting It All Together: The Full Classification Pipeline

Let’s see the full flow of how a machine learning model goes from raw data to a prediction. We’ll use our student pass/fail example:

1. Data

Collect examples with features and labels

→

2. Features

Pick (or learn) the right features

→

3. Train

Find weights that separate the data

→

4. Predict

Apply weights to new data

For classical ML, step 2 is manual. For deep learning, step 2 is learned. But the pipeline is the same.

The Most Common Mistake

Beginners often focus on the algorithm (step 3) and ignore the data (step 1) and features (step 2). In practice, better data and better features almost always beat a fancier algorithm. This is true even for deep learning — garbage in, garbage out, no matter how many layers you stack.
