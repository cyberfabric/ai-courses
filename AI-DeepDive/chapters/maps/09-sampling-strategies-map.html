Sampling Strategies — The Numbers Behind Creativity

Memory Map

The Big Idea

The model is always the same — same weights, same training. “Creativity” comes entirely from how you sample the probability distribution. Temperature, top-k, and top-p don’t change the model — they reshape which tokens can be selected.

💡 Why Top-p Adapts

💯
Confident Model
“Paris” = 92%. Top-p=0.9 keeps 1 token. Top-k=3 would keep 3 irrelevant extras.
🤔
Uncertain Model
4 emotions at 18–25% each. Top-p=0.9 keeps 4 tokens. Top-k=3 would cut a valid option.

🎒 Core Concepts

🔢
Logits
Raw scores from the model — not probabilities. One number per vocab token. Must go through softmax first.
📊
Softmax
elogit / sum(eall). Turns logits into probabilities (0–1, sum to 1). Exponential amplifies differences.
🌡️
Temperature
Divide logits by T before softmax. T<1 sharpens, T>1 flattens. T≈0 = greedy decoding.
🔁
Frequency Penalty
Subtract penalty × count from logit. More repeats = bigger penalty. Fights repetitive loops.
🚫
Presence Penalty
Binary: subtract fixed value if token appeared at all. Encourages topic diversity, not just word diversity.
🎲
Sampling
Randomly pick a token weighted by probability. Same distribution can produce different outputs each time.

🔢 Softmax Step by Step

  1. Exponentiate

    Compute elogit for each token. Bigger logits grow exponentially faster.

  2. Sum All

    Add up all the exponentials to get the normalizing denominator.

  3. Divide

    Each elogit ÷ sum = probability. Now they’re between 0 and 1, summing to 1.

📖 Key Terms

Logits
Raw, unnormalized scores from the model’s final layer — one per vocabulary token.
Softmax
Function that converts logits to probabilities: exponentiate, sum, divide.
Temperature
Divide logits by T before softmax. T<1 sharpens (deterministic), T>1 flattens (creative).
Top-k Sampling
Keep only the K highest-probability tokens, zero out the rest, renormalize.
Top-p (Nucleus) Sampling
Keep tokens until cumulative probability reaches P. Adapts to model confidence.
Min-p Sampling
Keep tokens whose probability ≥ P × top token’s probability. Simpler adaptive filter.
Frequency Penalty
Subtract penalty × token count from logit. Penalizes proportionally to repetition.
Presence Penalty
Subtract a fixed value from logit if token appeared at all. Binary on/off penalty.

❔ Quick Quiz

What does temperature actually do mathematically?

Divides all logits by T before softmax. This amplifies differences (T<1) or shrinks them (T>1), reshaping the probability distribution without changing which token has the highest probability.

Why does top-p adapt to model confidence but top-k doesn’t?

Top-k always keeps exactly K tokens. Top-p keeps tokens until cumulative probability reaches P — so when the model is very confident (one token dominates), fewer tokens survive; when uncertain, more pass through.

If all logits are equal, what does temperature change?

Nothing! Temperature amplifies differences between logits. Equal logits produce equal probabilities regardless of T.

Does higher temperature make the model smarter?

No. It makes the output more random. Sometimes random is creative; often it’s nonsense. Model quality comes from training, not sampling.

What’s the difference between frequency and presence penalty?

Frequency penalty scales with count (appeared 5 times → 5× penalty). Presence penalty is binary (appeared at all → fixed penalty). Frequency discourages word repetition; presence encourages new topics.

📌 Key Takeaways

  • Logits → softmax → probabilities. The model outputs raw scores, not probabilities. Softmax (exponentiate, sum, divide) converts them.
  • Temperature divides logits by T before softmax. T<1 sharpens (deterministic). T>1 flattens (creative). T≈0 = greedy.
  • Top-k = hard cutoff at rank K. Simple but doesn’t adapt to confidence.
  • Top-p = keep tokens until cumulative probability ≥ P. Adapts to confidence. The production default.
  • Pipeline order matters: temperature first, then top-k/top-p, then sample.
  • Frequency penalty fights repetition by penalizing proportionally to token count.
  • Same model, different sampling = different personality. “Creativity” is a sampling setting, not a model property.
