Embeddings — From Words to Geometry

The Big Idea

Embeddings turn words into vectors where meaning becomes geometry. Similar words land near each other, relationships become directions, and finding relevant documents becomes computing a dot product.

📐 Cosine Similarity in 3 Steps

  1. Dot Product

    Multiply element-wise, sum: A·B = a1b1 + a2b2 + …

  2. Magnitudes

    |A| = sqrt(a1² + a2² + …) for each vector.

  3. Divide

    cos(A,B) = dot / (|A| × |B|). Result: 1.0 = identical, 0 = unrelated, -1.0 = opposite.

🎒 Core Concepts

📑
Embedding Table
A vocab_size × dim matrix. Each row is one token’s learned vector. GPT-2: 50,257 × 768 = 38.6M parameters.
👥
Skip-Gram
Predict context words from center word. The weights become the embeddings. Words in similar contexts get similar vectors.
📐
Cosine Similarity
Measures angle between vectors, ignoring length. 1.0 = identical direction, 0 = unrelated, -1.0 = opposite.
🔎
Semantic Search
Embed query + docs, rank by cosine similarity. Finds meaning matches even with zero word overlap.
🗃
Vector Database
Specialized storage for fast approximate nearest-neighbor search. IVF, HNSW, PQ techniques. Powers RAG at scale.
⚠️
Bias in Embeddings
Embeddings reflect training data patterns, including biases. “doctor - man + woman” might give “nurse” instead of “doctor.”

📖 Key Terms

Embedding
A learned vector that maps a discrete token to a continuous space where geometry encodes meaning.
Embedding Table
A vocab_size × dim matrix. Row lookup converts token ID to vector. No computation, just indexing.
Skip-Gram
Word2Vec training method: predict context words from center word. Weights become embeddings.
Distributional Hypothesis
“You shall know a word by the company it keeps.” Words in similar contexts get similar vectors.
Cosine Similarity
dot(A,B) / (|A| × |B|). Measures angle, ignores length. The standard for embedding comparison.
Vector Arithmetic
king − man + woman ≈ queen. Relationships encoded as linear directions in embedding space.
Sentence Embedding
A single vector for a whole text. Simple: average word vectors. Better: use a transformer encoder.
Vector Database
Specialized storage for fast approximate nearest-neighbor search (IVF, HNSW, PQ). Powers RAG.

❔ Quick Quiz

Why can’t we just use token IDs as numbers?

Token IDs are arbitrary labels. ID 4821 (“cat”) and 4822 (“dog”) look numerically close, but that’s meaningless. Embeddings give each token a vector where distance reflects meaning.

How does skip-gram make “cat” and “dog” similar?

Both predict similar context words (“sat,” “the,” “on”). Training pushes words toward the context they predict. Same contexts → same neighborhood in embedding space.

Why use cosine similarity instead of Euclidean distance?

Cosine measures direction (what the text is about), ignoring magnitude. Longer documents produce longer vectors, but direction stays meaningful. [1,2,3] and [2,4,6] have cosine 1.0 but Euclidean distance 3.74.

What makes semantic search better than keyword search?

Semantic search finds meaning matches even with zero word overlap. “teach my pet commands” matches “teaching your dog tricks” because their embeddings point in the same direction.

Why might king − man + woman give “nurse” instead of “doctor”?

Embeddings reflect training data biases, not reality. If the training text associates “doctor” with “man” and “nurse” with “woman,” the vector arithmetic captures that bias.

📌 Key Takeaways

  • Embeddings = learned lookup table. Each token ID maps to a vector. No computation, just indexing a matrix.
  • Meaning becomes geometry. Similar words → nearby vectors. Relationships → directions.
  • Trained via context: skip-gram predicts neighbors. Same context = similar vectors. “You shall know a word by the company it keeps.”
  • Cosine similarity = dot product / magnitudes. Measures angle, ignores length. The standard measure.
  • king − man + woman ≈ queen. Vector arithmetic works because relationships are linear directions.
  • Semantic search: embed query + docs, rank by cosine. Finds meaning, not just keywords. This powers RAG.
  • Embeddings carry biases from training data. Useful but imperfect representations of meaning.
