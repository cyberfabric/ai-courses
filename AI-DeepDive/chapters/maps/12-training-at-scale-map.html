The Big Idea

Training a frontier model is a three-stage pipeline: pre-training gives knowledge (99% of cost), SFT gives instruction-following, and alignment gives good behavior. Pre-training is a blank slate reading the internet; alignment turns that into a useful assistant.

🎒 Core Concepts

📚
Pre-training
Next-token prediction on trillions of tokens. 99% of compute. Gives the model knowledge and language ability.
🎯
SFT
Train on curated (instruction, response) pairs. Transforms a text predictor into an instruction follower. Fast and cheap.
👍
RLHF
Humans pick preferred responses → train reward model → PPO optimizes the LLM to score higher. Teaches quality, not just correctness.
⚡
DPO
Simpler RLHF alternative. No reward model, no PPO. Directly optimize on preference pairs with a modified loss.
📈
Scaling Laws
Loss follows a power law: more compute = predictably lower loss. You can predict model quality before training.
💻
GPU Parallelism
Data parallel (split batches), tensor parallel (split layers), pipeline parallel (split depth). Real runs use all three.

📈 The Loss Curve

  1. Step 0 — Loss ~11.0

    Random predictions. Guessing from 50K+ tokens.

  2. 1% trained — Loss ~5.0

    Basic grammar and common words learned. Fast drop.

  3. 10% trained — Loss ~3.5

    Decent text generation. Syntax and frequent patterns.

  4. 50% trained — Loss ~2.8

    Strong understanding. Facts and reasoning emerging.

  5. 100% trained — Loss ~2.5

    Diminishing returns. Each halving of loss ≈ 10× more compute.

📖 Key Terms

Pre-training
Next-token prediction on internet-scale text. Gives the model general knowledge. 99%+ of total compute.
SFT (Supervised Fine-Tuning)
Train on curated instruction-response pairs. Transforms text predictor into instruction follower.
RLHF
Reinforcement Learning from Human Feedback. Train reward model on human preferences, then optimize via PPO.
DPO (Direct Preference Optimization)
Skip the reward model. Directly adjust probabilities: more likely for preferred responses, less for rejected.
Constitutional AI (RLAIF)
Replace human raters with AI self-critique against written principles. Scales better than human feedback.
Chinchilla Scaling
Optimal tokens ≈ 20× parameters. Under-training wastes model capacity. Over-training saves inference cost.
Mixed Precision
Use float16/bfloat16 for most ops, float32 for weight accumulation. Nearly 2× speedup.
Reward Hacking
Model games the reward model instead of genuinely improving. E.g., longer = higher score → unnecessary padding.

❔ Quick Quiz

Why can’t a pre-trained model have a conversation?

Pre-training only teaches next-token prediction. Given a question, the base model might generate more questions (that’s what comes next in web text). SFT teaches it to produce answers instead.

Why is pre-training 99% of the cost but SFT transforms the model more visibly?

Pre-training processes trillions of tokens (massive compute). SFT uses only ~100K examples but targets exactly the behavior change needed. Pre-training builds the foundation; SFT activates it for a specific purpose.

What does the Chinchilla ratio tell you?

Optimal training tokens ≈ 20× parameter count. GPT-3 (175B params, 300B tokens) was under-trained. Llama 3 (70B params, 15T tokens) is intentionally over-trained to be cheaper at inference.

What’s the advantage of DPO over RLHF?

DPO is simpler (no separate reward model, no PPO), more stable to train, and produces competitive quality. It directly optimizes on preference pairs with standard gradient descent.

What is reward hacking?

The model learns to game the reward model rather than genuinely improve. Example: if longer responses score higher, the model pads output unnecessarily. Requires careful reward model design.

📌 Key Takeaways

  • Three stages: Pre-training (read everything) → SFT (learn the job) → Alignment (learn preferences).
  • Pre-training = 99% of cost. Next-token prediction on 1–15T tokens. GPT-3 ≈ $4.6M. GPT-4 ≈ $100M.
  • Data pipeline: crawl → dedup → quality filter → safety filter → tokenize.
  • Chinchilla: optimal tokens ≈ 20× params. Over-training saves inference cost.
  • SFT transforms text predictor → instruction follower with ~100K curated examples.
  • RLHF = reward model + PPO. DPO = simpler, no reward model, equally competitive.
  • Loss improves predictably with more compute (scaling laws). Each halving costs ~10× more.
