Training Step by Step — Forward, Loss, Backward

Memory Map

The Big Idea

Training is a loop: predict → measure error → trace blame → nudge weights. Every gradient answers one question: “If I nudge this weight, does the loss go up or down?” Repeat millions of times and the network learns.

🔢 One Iteration in Numbers

  1. Forward Pass

    Input [1.0, 0.5] → hidden z=0.3, ReLU=0.3 → output z=0.18 → sigmoid = 0.545. Target: 1.0.

  2. Compute Loss

    MSE = (0.545 - 1.0)² = 0.207. The prediction is way off.

  3. Backpropagation

    Trace blame backward. Gradients: w₁=-0.135, w₂=-0.068, w₃=-0.068. All negative → all weights should increase.

  4. Weight Update (lr=0.1)

    w₁: 0.300 → 0.314. w₂: -0.200 → -0.193. w₃: 0.600 → 0.607. Tiny nudges in the right direction.

🎒 Core Concepts

📏
Loss Function
A single number measuring "how wrong." MSE = (predicted - actual)². Cross-entropy for classification. Lower = better.
🔙
Gradient
For each weight: "if I nudge it up, does the loss go up or down, and by how much?" Direction + magnitude.
🔗
Backpropagation
The "blame game" — trace the error signal backward through the network, layer by layer, using the chain rule.
🎯
Learning Rate
Controls step size. Too small = crawls. Too big = overshoots. Typical: 0.001–0.01. Adam adjusts it automatically.
🔁
Epoch
One pass through the entire training dataset. Models typically train for 10–100+ epochs.
⚠️
Overfitting
Training loss ↓ but validation loss ↑. The model memorized instead of learning. Fix: more data, dropout, early stopping.

🎮 Learning Rate: The Goldilocks Problem

Learning Rate Step Size Result
🧢 0.0001 Tiny Barely moves. Training takes forever.
✅ 0.001–0.01 Just right Steady progress. The sweet spot.
💥 0.5+ Huge Overshoots. Loss bounces or explodes.

📦 Batch Size Tradeoffs

Strategy Batch Size Speed Stability
Stochastic 1 Fast updates Noisy
Mini-batch 32–256 GPU-efficient Balanced
Full batch All data Slow Stable but can get stuck

📖 Key Terms

Loss / Cost Function
A number measuring prediction error. Training minimizes this. MSE and cross-entropy are common choices.
Gradient
The derivative of the loss with respect to a weight — tells direction and magnitude for the update.
Backpropagation
Algorithm for computing all gradients by tracing the error signal backward through the network via the chain rule.
Learning Rate (η)
Multiplier controlling step size in weight updates. Too small = slow; too big = unstable.
Epoch
One complete pass through all training examples.
Mini-batch
A subset of training examples (typically 32–256) used for one gradient update. Balances noise and efficiency.
Overfitting
When training loss is low but validation loss is high — the model memorized instead of generalizing.
Validation Loss
Loss measured on held-out data the model hasn't trained on — the true test of learning.

❔ Quick Quiz

A gradient is -0.5. Should the weight increase or decrease?

Increase. The update rule is w = w - lr × gradient. Subtracting a negative = adding. The negative gradient means increasing the weight would decrease the loss.

What are the four steps of one training iteration?

1) Forward pass (predict), 2) Compute loss (measure error), 3) Backward pass (compute gradients), 4) Update weights (nudge toward less error).

Training loss is 0.01, validation loss is 0.50. What’s wrong?

Overfitting. The model memorized the training data but can't generalize. The huge gap between training and validation loss is the telltale sign.

Why not update weights after every single example?

Too noisy — one weird example can push weights in the wrong direction. Mini-batches (32–256 examples) average out noise while keeping updates frequent and GPU-efficient.

What happens to hidden weight gradients when ReLU outputs zero?

They become zero too. No error signal flows through a dead neuron — those weights get no update. This is the "dead neuron" problem.

📌 Key Takeaways

  • Training loop: forward → loss → backward → update. Repeat millions of times.
  • Loss function = single number measuring "how wrong." MSE for regression, cross-entropy for classification.
  • Gradient = direction + magnitude for each weight. Negative gradient → increase weight. Positive → decrease.
  • Backpropagation traces blame backward through the network, layer by layer.
  • Update rule: new_weight = old_weight - learning_rate × gradient.
  • Learning rate is critical: 0.001–0.01 typical. Too small = crawls, too big = explodes.
  • Overfitting = memorizing instead of learning. Watch the gap between training and validation loss.
