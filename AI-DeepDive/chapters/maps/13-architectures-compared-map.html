The Big Idea

Different tasks need different architectures. Decoder-only won the chatbot race, but encoder-only dominates search, diffusion dominates image generation, and MoE gives you big-model knowledge at small-model speed. Know the landscape to pick the right tool.

📖 Key Terms

Encoder-Only (BERT)
Bidirectional attention. Predicts masked tokens. Best for understanding: classification, embeddings, search.
Decoder-Only (GPT)
Causal attention. Autoregressive generation. Universal interface for chat, code, reasoning.
Encoder-Decoder (T5)
Encoder reads full input, decoder generates output attending to encoder. Best for translation.
Mixture of Experts (MoE)
N expert FFNs per layer, router picks top-k. More total params, fewer active. Saves compute, not memory.
Diffusion Model
Add noise (forward), learn to remove noise (reverse). Text embedding guides denoising for text-to-image.
Vision Transformer (ViT)
Split image into patches, treat as token sequence, apply standard transformer. Bridges vision and language.
State-Space Model (SSM)
O(n) alternative to attention. Compressed running state instead of full token-to-token attention.
Multimodal
Project image/audio embeddings into text space. One transformer attends across all modalities.

❔ Quick Quiz

Why can’t BERT generate text?

BERT uses bidirectional attention — it sees tokens before AND after each position. Generation requires predicting the next token without seeing future tokens. BERT’s architecture doesn’t support this causal constraint.

Mixtral 8x7B has 47B params but only uses ~13B per token. Why?

MoE with 8 experts, top-2 routing. Each token activates only 2 of 8 expert FFNs. The shared layers + 2 experts ≈ 13B active parameters. The other 6 experts do zero work for that token.

How does a text prompt control image generation in diffusion?

The text is encoded into a vector (via CLIP). This vector guides each denoising step — the model removes noise in the direction of the text embedding, sculpting the image toward what was described.

Why would you use BERT instead of GPT-4 for a spam classifier?

BERT is smaller, faster, and purpose-built for classification (bidirectional understanding). GPT-4 can classify too, but costs 100× more per request with no quality benefit for simple classification.

What advantage do state-space models have over transformers?

O(n) vs O(n²) scaling. For very long sequences, SSMs avoid the quadratic attention cost. They maintain a compressed state instead of attending to every previous token.

📌 Key Takeaways

  • Three transformer branches: Encoder-only (BERT, understanding), Decoder-only (GPT, generation), Encoder-decoder (T5, translation).
  • Decoder-only won the chatbot race: simpler, universal, scales better, in-context learning.
  • MoE = many experts, activate few. More knowledge at the same inference speed. Saves compute, not memory.
  • Diffusion = start with noise, denoise step by step. Text guides the sculpting. Dominates image generation.
  • ViT = image patches as tokens. Same transformer, different input. Bridges vision and language.
  • SSMs (Mamba) = O(n) alternative to O(n²) attention. Compressed state. Great for very long sequences.
  • Match architecture to task: chat → decoder, search → encoder, images → diffusion, broad knowledge → MoE.
