The Big Idea

Every ML model learns a decision boundary — a line (or surface) that separates categories in feature space. As you go from classical ML to deep learning, the model does more of the work: picking features, stacking layers, and composing simple patterns into complex ones.

🎒 Core Concepts

✏️
Decision Boundary
The line/surface a classifier draws through feature space to separate categories. In 2D it's a line, in higher dimensions a hyperplane.
⚖️
Weights & Bias
Weights say how much each feature matters. Bias sets the baseline. The formula: w₁×x₁ + w₂×x₂ + b. Sign = prediction.
🔎
Feature Space
The multi-dimensional space where each axis is a feature. Data points live here; classifiers draw boundaries through it.
🧩
Feature Engineering vs. Learned Features
Classical ML: you pick features (recipe). Deep learning: the model discovers features from raw data (figures out the recipe).
🌱
Feature Hierarchy
Deep networks build features layer by layer: edges → shapes → parts → objects. Each layer composes simpler patterns into complex ones.
📚
K-Means Clustering
Unsupervised: assign points to nearest centroid, update centroid to cluster average, repeat. Finds groups without labels.

📖 Key Terms

Decision Boundary
Line/surface that separates categories in feature space.
Feature Space
The coordinate system where each axis is a data feature (height, weight, etc.).
Weights (w)
Numbers that determine how much each input feature influences the output.
Bias (b)
A constant that shifts the decision boundary — sets the baseline threshold.
K-Means
Unsupervised algorithm: assign to nearest centroid, recompute centroid, repeat.
Centroid
The center point of a cluster — the average of all points assigned to it.
Reinforcement Learning
Learning by trial and error, guided by reward/penalty signals from an environment.
Autoregressive
Generating output one step at a time, where each step depends on all previous steps.
Feature Hierarchy
Layers building progressively: edges → shapes → parts → objects → concepts.

❔ Quick Quiz

What is a decision boundary?

A line (or surface) that a classifier draws through feature space to separate different categories. Everything on one side is one class, everything on the other is the other class.

What's the key difference between classical ML and deep learning?

Classical ML requires you to manually select and engineer features. Deep learning discovers features automatically from raw data by stacking layers.

In a linear classifier, what role does the bias play?

The bias shifts the decision boundary. It sets the baseline threshold that the weighted sum of inputs must overcome to produce a positive prediction.

What are the two repeating steps of k-means?

1) Assign each point to the nearest centroid. 2) Update each centroid to the average of its assigned points. Repeat until convergence.

What makes LLMs a specific type of deep learning?

LLMs are transformer-based, trained on text to predict the next token, and generate output autoregressively (one token at a time).

📌 Key Takeaways

  • Every classifier learns a decision boundary — a line/surface separating categories in feature space.
  • Linear classifiers use weights × features + bias. The sign of the sum = the prediction.
  • Classical ML needs hand-picked features; deep learning discovers them automatically.
  • Three types of learning: supervised (labels), unsupervised (patterns), reinforcement (rewards).
  • "Deep" means many layers — each layer composes simple features into complex ones (edges → shapes → concepts).
  • LLMs = deep learning + transformers + text + autoregressive generation.
  • Better data beats a fancier algorithm. Garbage in, garbage out — always.
