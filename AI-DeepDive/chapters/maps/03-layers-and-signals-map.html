The Big Idea

A neural network is layers of matrix multiplications + activations, stacked in sequence. Data flows forward — each layer transforms it into a more useful representation. The math at every stage is the same: multiply, add bias, activate.

🎒 Core Concepts

🗂
Layer
A group of neurons sharing the same inputs. Each neuron has its own weights — 3 inputs, 4 neurons = 4 different outputs.
🔢
Weight Matrix
All neurons' weights packed into one table. Rows = inputs, columns = neurons. Matrix multiply computes all neurons at once.
✖
Matrix Multiplication
Dot product of input row with each weight column. It's the same w×x from Ch 2, just organized for parallel computation.
📈
Softmax
Turns raw scores into probabilities: exponentiate each score, divide by the sum. Outputs are positive and sum to 1.
🌱
Feature Hierarchy
Each layer builds on the last: edges → shapes → parts → objects. Discovered automatically, not programmed.
📏
Depth vs. Width
Width detects more patterns at one level. Depth composes patterns — that's how you go from pixels to "golden retriever."

📖 Key Terms

Layer
A group of neurons that all receive the same inputs; each applies its own weights.
Weight Matrix
All of a layer's weights in a 2D grid: rows = inputs, columns = neurons.
Forward Pass
Feeding data through the network layer by layer, from input to prediction.
Matrix Multiplication
Dot product of the input with each column of the weight matrix — computes all neurons at once.
Softmax
ez/Σez — converts raw scores to probabilities summing to 1.
Feature Hierarchy
Layers progressively detect more abstract features: pixels → edges → shapes → objects.
Parameter Count
Per layer: (inputs × neurons) + biases. Sum across layers for total.

❔ Quick Quiz

A layer has 5 inputs and 8 neurons. How many parameters?

(5 × 8) + 8 = 48 (40 weights + 8 biases).

What does softmax guarantee about its outputs?

All outputs are positive and they sum to exactly 1.0 — a valid probability distribution.

Why are GPUs important for neural networks?

Neural networks are built on matrix multiplications. GPUs are designed to do massive parallel matrix math, making them orders of magnitude faster than CPUs for this workload.

Why does depth help more than width for complex tasks?

Depth enables composition — building complex features from simple ones (edges → shapes → objects). Width just adds more detectors at the same abstraction level.

In our forward pass, Neuron 2 output 0. Why?

Its weighted sum was -0.50 (negative). ReLU killed it to zero. That neuron found nothing relevant in this particular input.

📌 Key Takeaways

  • A layer = many neurons sharing inputs. Each neuron computes its own weighted sum + activation.
  • Weight matrix organizes all neurons' weights. Matrix multiplication computes them all at once.
  • Forward pass: data flows input → hidden layers → output layer → prediction.
  • Softmax converts raw scores to probabilities (ez / Σez).
  • Depth enables composition — each layer builds more abstract features on top of the previous layer.
  • Parameter formula: (inputs × neurons) + biases per layer. Our 26-param toy uses the same math as GPT-4's 1.8T.
