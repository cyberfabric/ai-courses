The Big Idea

Every AI response is built one token at a time: predict → select → append → repeat. The model doesn’t plan ahead or remember past turns — it re-reads the entire conversation every time and generates the next token from a probability distribution.

🎲 Three Decoding Strategies

StrategyHow It PicksSpeedBest For
🎯 Greedy Always highest probability Fastest (1×) Deterministic output, but can loop
🔬 Beam Search Keep top B sequences, prune B× slower Translation, summarization
🎲 Sampling Random pick weighted by probability 1× Chat, creative writing

🎒 Core Concepts

🔁
Autoregressive
Each output becomes input for the next step. The model “feeds itself” its own predictions, one token at a time.
💬
Chat = Concatenation
Every turn re-sends the full history: system prompt + all user/assistant messages. “Memory” is an illusion — it’s just re-reading.
⏯
Streaming
Each token is sent as generated. What you see appearing is literally the model working in real time.
🚫
Stop Conditions
EOS token (natural end), max length (forced cutoff), or stop sequence (structured output). Mid-sentence cutoffs = max length hit.
🔄
Greedy Trap
Always picking the most probable token can cause repetitive loops. Local optima ≠ globally good sequence.
🔬
Beam Search
Keep B candidate sequences, expand all, prune to top B. Finds globally better sequences at B× the compute cost.

📖 Key Terms

Autoregressive Generation
Predict one token, append it to the input, repeat. Each output feeds back as input.
Greedy Decoding
Always pick the highest-probability token. Fast, deterministic, but prone to repetition.
Beam Search
Keep top B candidate sequences at each step. Better global results at B× compute cost.
Sampling
Randomly select from the probability distribution. Introduces variety and creativity.
EOS Token
Special <EOS> token that signals “stop generating.” The model learns when to produce it.
TTFT (Time to First Token)
Time to process the full prompt before generation starts. Grows with prompt length.
TPS (Tokens Per Second)
Generation speed once output begins. Roughly constant regardless of prompt length.
Context Window
Maximum tokens the model can process. Chat history is re-sent every turn, consuming window space.

❔ Quick Quiz

Does the model “remember” your previous messages?

No. The entire conversation history is concatenated and re-sent every turn. “Memory” is an illusion — the model re-reads everything from scratch.

Why can greedy decoding produce repetitive text?

Because the locally most-probable token can lead to a loop. Once the model outputs “the weather was nice,” the most probable next tokens may lead right back to the same phrase.

What’s the difference between TTFT and TPS?

TTFT = time to process the prompt (grows with prompt length). TPS = generation speed once output starts (roughly constant). Long prompts increase TTFT but not TPS.

Why does streaming work token by token?

Because the model literally generates one token at a time. Each token is complete and can be sent immediately. There’s no “draft” — what you see appearing is the model’s actual output process.

Beam search with B=4 is how much slower than greedy?

Approximately 4× slower. Each step requires B forward passes instead of 1, since every beam must be expanded independently.

📌 Key Takeaways

  • One token at a time: predict → select → append → repeat. Every AI response is built this way.
  • Greedy = always pick the top token. Fast but can loop. Beam search = keep B options alive. Better but B× slower.
  • Sampling adds controlled randomness for variety. Chapter 9 goes deep on temperature, top-k, top-p.
  • Chat = full history re-sent every turn. No real memory. Context window limits how much fits.
  • Streaming is real-time generation — each token sent as it’s produced.
  • Two latency phases: TTFT (prompt processing, varies) and TPS (generation speed, constant).
  • Stop conditions: EOS token, max length, or stop sequence.
