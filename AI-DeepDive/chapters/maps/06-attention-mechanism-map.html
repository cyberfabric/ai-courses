Attention Is All You Need

Memory Map

The Big Idea

Attention lets each token ask “who else matters to me?” by computing relevance scores against every other token. The result: each token’s output is a weighted blend of information from the most relevant tokens. One formula: softmax(Q×KT / √dk) × V.

🔢 Five Steps with Numbers

  1. Compute Q, K, V

    Multiply each token’s embedding by WQ, WK, WV. Three 4D embeddings → three 3D vectors each.

  2. Attention Scores (Q × KT)

    Dot product of each Q with every K. Produces a 3×3 score matrix. Higher = more relevant.

  3. Scale by √dk

    Divide all scores by √3 ≈ 1.73. Prevents softmax from becoming too “peaky.”

  4. Softmax (row by row)

    Each row becomes a probability distribution summing to 1. Row i = “how much should token i attend to each token?”

  5. Weighted Sum of V

    Blend value vectors using attention weights. Output = context-aware representation of each token.

🎒 Core Concepts

🔎
Self-Attention
Q, K, V all come from the same sequence. Each token attends to every other token in the same input.
👥
Multi-Head
Run attention N times in parallel with different weights. Each head captures a different relationship type (grammar, coreference, proximity…).
📏
Scaling (√dk)
Without it, large dimensions produce huge scores → softmax becomes a spike → only one token gets all attention.
📈
Softmax
Turns raw scores into probabilities (positive, sum to 1). Each token gets its own distribution over all tokens.
🔗
Cross-Attention
Q from one sequence, K/V from another. Used in translation: decoder queries the encoder’s output.
📈
O(n²) Cost
Every token attends to every other → n×n matrix. Double the sequence = 4× the compute. The fundamental bottleneck.

📖 Key Terms

Query (Q)
What a token is looking for — computed as embedding × WQ.
Key (K)
What a token advertises about itself — computed as embedding × WK.
Value (V)
The actual information a token carries — computed as embedding × WV.
Attention Score
Dot product of Q and K — measures how relevant token j is to token i.
Self-Attention
Q, K, V all derived from the same sequence. The sequence attends to itself.
Multi-Head Attention
Run attention N times in parallel with different weights, concatenate outputs. Each head captures different relationships.
Scaled Dot-Product
Divide scores by √dk before softmax to prevent extreme distributions.
O(n²) Complexity
Attention cost grows quadratically with sequence length — the main bottleneck for long contexts.

❔ Quick Quiz

Why divide attention scores by √dk?

Large dimensions produce large dot products. Without scaling, softmax becomes extremely peaky — one token gets ~100% attention. Scaling keeps the distribution moderate so the model can attend to multiple tokens.

What do the three matrices WQ, WK, WV do?

They project each token’s embedding into three different spaces: Q (what it’s searching for), K (what it offers), V (the information it carries). These matrices are learned during training.

Why use multiple heads instead of one big attention?

One head can only compute one attention distribution. Multiple heads let the model attend to different relationships simultaneously — grammar, meaning, coreference — then combine the perspectives.

If you double the context from 4K to 8K tokens, how does attention cost change?

It quadruples. Attention is O(n²): 4K² = 16M, 8K² = 64M. That’s 4× more computation.

What’s the difference between self-attention and cross-attention?

Self-attention: Q, K, V all come from the same sequence. Cross-attention: Q comes from one sequence (e.g., decoder), K and V from another (e.g., encoder output).

📌 Key Takeaways

  • Attention = softmax(Q×KT / √dk) × V. One formula, five steps.
  • Q, K, V are projections of each token via learned weight matrices.
  • Dot product of Q and K measures relevance. Softmax turns scores into probabilities.
  • Output = weighted blend of all tokens’ values. After attention, each token “knows about” every relevant token.
  • Scaling prevents softmax from collapsing to a spike on one token.
  • Multi-head captures multiple relationship types in parallel (grammar, coreference, proximity).
  • O(n²) cost is the fundamental reason context windows are limited.
