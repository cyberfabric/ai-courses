Chapter 6

Attention Is All You Need

Queries, keys, and values — computed with real numbers on a 3-token sequence

Companion: This expands on AI-Intro Ch 5: The Transformer. There you learned what attention does. Now you’ll compute every number yourself.

In Chapter 5, you learned how text becomes token IDs. In Chapters 2–4, you saw how neural networks process numbers through layers. But those networks treated every input equally — there was no concept of “this input is more relevant to me than that one.”

Attention changes everything. It’s the mechanism that lets a model decide, for each token, which other tokens matter most right now. It’s the single most important idea in modern AI, and in this chapter, you’ll compute it from scratch.

The Core Question

Consider this sentence:

“The cat sat on the mat because it was tired.”

When the model processes the word “it,” it needs to figure out: what does “it” refer to? The cat? The mat? The sitting? A simple neural network would treat all previous words equally. But attention lets the model compute a relevance score between “it” and every other word, and “cat” should score highest.

The question attention answers is: “When processing this token, how much should I pay attention to each other token?”

The Search Engine Analogy

Attention works like a search engine with three components:

ComponentSearch EngineAttention
Query (Q) What you type in the search bar What this token is looking for
Key (K) The title/summary of each web page What each token advertises about itself
Value (V) The actual content of each web page The actual information each token carries

You type a query, the search engine compares it to every page’s key (title), ranks them by relevance, and returns the values (content) of the best matches. Attention does the same thing, but for every token in a sequence, simultaneously.

Self-Attention in One Sentence

For each token: compute a query (what am I looking for?), compare it to every token’s key (what do you offer?), get relevance scores, and blend everyone’s values weighted by those scores.

Where Q, K, V Come From

Every token starts as an embedding vector — a list of numbers representing that token’s meaning (we’ll cover how embeddings are trained in Chapter 11). To get Q, K, and V, we multiply each embedding by three different learned weight matrices:

  • Q = embedding × WQ — “What am I looking for?”
  • K = embedding × WK — “What do I offer?”
  • V = embedding × WV — “What information do I carry?”

These weight matrices are learned during training (Chapter 4). The model discovers what to look for and what to advertise — nobody programs these rules by hand.

What do these vectors mean?

Each token now has three representations:

  • Q — what it’s searching for (its “question” to the other tokens)
  • K — what it offers to other tokens searching for relevance
  • V — the actual information it will contribute to the output

The same token plays all three roles simultaneously. When processing “cat,” the model uses cat’s Q to search, but also uses cat’s K and V when other tokens look at cat.

Step 2: Attention Scores (Q × K)

Now comes the core: compute how relevant each token is to every other token. We do this by taking the dot product of each Q with every K:

Dot Product = Relevance Score

The dot product of Qi and Kj measures how well token i’s “question” matches token j’s “advertisement.” High dot product = high relevance. Low dot product = not relevant.

Computing the 3×3 attention score matrix

Score(The, The) = QThe · KThe = (0.69×0.61) + (0.57×0.66) + (0.81×0.66)

= 0.42 + 0.38 + 0.53 = 1.33

Score(The, cat) = QThe · Kcat = (0.69×0.55) + (0.57×0.46) + (0.81×0.42)

= 0.38 + 0.26 + 0.34 = 0.98

Score(The, sat) = QThe · Ksat = (0.69×0.53) + (0.57×0.39) + (0.81×0.46)

= 0.37 + 0.22 + 0.37 = 0.96

Step 3: Scale the Scores

Before softmax, we divide every score by √dk, where dk is the dimension of the key vectors. Our keys are 3-dimensional, so we divide by √3 ≈ 1.73.

Scaling the scores

Q ↓   K →Thecatsat
The1.33/1.73 = 0.770.98/1.73 = 0.570.96/1.73 = 0.55
cat0.98/1.73 = 0.570.76/1.73 = 0.440.72/1.73 = 0.42
sat1.09/1.73 = 0.630.80/1.73 = 0.460.80/1.73 = 0.46

Why scale?

Without scaling, large key dimensions produce huge dot products. Huge numbers going into softmax create extremely “peaky” distributions where one token gets nearly 100% of the attention and everything else gets ~0%. Dividing by √dk keeps the scores in a moderate range so softmax produces a useful distribution — allowing the model to attend to multiple tokens at once.

Step 4: Softmax — Scores to Probabilities

Softmax turns each row of scores into a probability distribution (positive numbers that sum to 1). We apply it row by row — each token gets its own distribution over all tokens.

Softmax for “The”’s row: [0.77, 0.57, 0.55]

Step 1: Exponentiate each score:

e0.77 = 2.16,   e0.57 = 1.77,   e0.55 = 1.73

Step 2: Sum them: 2.16 + 1.77 + 1.73 = 5.66

Step 3: Divide each by the sum:

The: 2.16 / 5.66 = 0.38

cat: 1.77 / 5.66 = 0.31

sat: 1.73 / 5.66 = 0.31

Check: 0.38 + 0.31 + 0.31 = 1.00 ✓

Applying softmax to all three rows:

Reading the attention weights

Each row is a probability distribution. Row “The” says: when processing “The,” pay 38% attention to itself, 31% to “cat,” and 31% to “sat.” In this toy example the weights are fairly uniform because our random weight matrices haven’t been trained. In a real model, after billions of training steps, “it” in “the cat sat because it was tired” would have an attention weight of, say, 0.72 for “cat” and small weights for everything else.

Checkpoint

If the scaled attention scores for “sat” were [0.63, 0.46, 0.46], and you compute softmax, what property must the three resulting weights have?

Answer: They must all be positive and sum to exactly 1.0. They form a probability distribution over all tokens.

Step 5: Weighted Sum of Values

The final step: for each token, blend the value vectors using the attention weights. This produces the attention output — a new representation that incorporates information from the most relevant tokens.

Weights: The=0.38, cat=0.31, sat=0.31

Values: VThe=[0.58, 0.58, 0.89], Vcat=[0.56, 0.44, 0.40], Vsat=[0.26, 0.57, 0.63]

Output1 = (0.38×0.58) + (0.31×0.56) + (0.31×0.26) = 0.22 + 0.17 + 0.08 = 0.47

Output2 = (0.38×0.58) + (0.31×0.44) + (0.31×0.57) = 0.22 + 0.14 + 0.18 = 0.54

Output3 = (0.38×0.89) + (0.31×0.40) + (0.31×0.63) = 0.34 + 0.12 + 0.20 = 0.66

Attention output for “The” = [0.47, 0.54, 0.66]

What just happened?

Each token’s output is now a blend of all tokens’ values, weighted by relevance. “The” didn’t just look at its own value — it mixed in information from “cat” and “sat” based on how relevant they were. This is the magic: after attention, every token’s representation contains information from every other relevant token.

In our toy example the outputs look similar because the weights are untrained. In a real model, “it”’s output would be heavily influenced by “cat”’s value, encoding the knowledge that “it” refers to “cat.”

Self-Attention: Q, K, V from the Same Source

In our example, all three — Q, K, and V — came from the same sequence (“The cat sat”). This is called self-attention: the sequence is attending to itself.

This is what happens inside a transformer. Each token asks “who else in this sentence is relevant to me?” and the answer comes from the same sentence.

Cross-attention is different: Q comes from one sequence (e.g., the decoder), while K and V come from another (e.g., the encoder). This is used in translation: the decoder “queries” the input sentence for relevant source words. We’ll see this in Chapter 7.

Multi-Head Attention: Parallel Perspectives

A single attention computation captures one “type” of relationship. But language has many types: grammar, meaning, coreference, position… To capture multiple patterns, transformers run attention multiple times in parallel, each with different WQ, WK, WV matrices. Each run is called a head.

Multi-head attention: 4 heads example

If our model dimension is 12 and we use 4 heads:

  • Each head gets dimension 12 ÷ 4 = 3
  • Head 1 might learn to track grammar (subject–verb agreement)
  • Head 2 might learn coreference (“it” → “cat”)
  • Head 3 might learn proximity (nearby words are relevant)
  • Head 4 might learn semantic similarity (related concepts)

Each head runs the full Q×KT÷√dk + softmax + ×V pipeline independently.

After all heads finish:

  1. Concatenate: [head1_output | head2_output | head3_output | head4_output] = 12-dim vector
  2. Multiply by a final weight matrix WO (12×12) to combine the heads

Why multiple heads help

One head can only compute one set of attention weights. “It” can’t simultaneously give high weight to “cat” (for coreference) AND to “was” (for grammar) in a single attention distribution. With multiple heads, different heads can focus on different relationships. It’s like having multiple people read the same sentence, each looking for something different.

Real models use many heads: GPT-3 has 96 heads, GPT-4 likely has more. Each captures a different type of token-to-token relationship.

The Party Analogy

Imagine you’re at a party with 100 people. You can only listen to a few people at once. How do you decide who to listen to?

  1. You have a question (Query): “Who here knows about machine learning?”
  2. Each person wears a name tag (Key): describing what they know — “ML researcher,” “Chef,” “Dog trainer”
  3. You compare your question to each name tag (Q×K): the ML researcher scores high, the chef scores low
  4. You listen proportionally (Softmax): 70% attention to the ML researcher, 20% to the data engineer, 10% split across everyone else
  5. You absorb what they say (weighted V): your understanding is mostly shaped by the ML researcher’s words, plus a little from everyone else

Multi-head attention is like splitting yourself into 8 copies, each asking a different question: one asks about ML, another about food, another about hobbies. Each copy blends what they hear, and then you merge all 8 perspectives into your final understanding.

Attention Complexity: The Quadratic Problem

One critical detail: the Q×KT step computes a score for every pair of tokens. With n tokens, that’s an n×n matrix.

Sequence LengthAttention Matrix SizeOperations
100 tokens100 × 10010,000
1,000 tokens1,000 × 1,0001,000,000
10,000 tokens10,000 × 10,000100,000,000
128,000 tokens128K × 128K16.4 billion

This is the quadratic scaling of attention: double the sequence length and computation goes up 4×. This is why long context windows are expensive and why researchers are working on more efficient alternatives (we’ll touch on this in Chapter 10).

Why context length is limited

A 128K context window requires a 128K×128K attention matrix per head, per layer. With 96 heads and 96 layers, that’s an enormous amount of memory and compute. This quadratic cost is the fundamental reason context windows can’t just be “made bigger.”

Exercise: Compute Attention from Scratch

Your turn: 2-token attention

Given two tokens with these Q, K, V vectors (dimension 2):

TokenQKV
A[1.0, 0.5][0.8, 0.3][0.2, 0.9]
B[0.3, 0.7][0.6, 0.9][0.7, 0.1]

Compute:

  1. The 2×2 raw attention scores (Q×KT)
  2. The scaled scores (divide by √2 ≈ 1.41)
  3. Softmax of each row
  4. The attention output for token A (weighted sum of values)

Solution:

  1. Score(A,A) = (1.0×0.8)+(0.5×0.3) = 0.95

    Score(A,B) = (1.0×0.6)+(0.5×0.9) = 1.05

    Score(B,A) = (0.3×0.8)+(0.7×0.3) = 0.45

    Score(B,B) = (0.3×0.6)+(0.7×0.9) = 0.81

  2. Scaled: [0.67, 0.74; 0.32, 0.57]

  3. Softmax row A: e0.67=1.95, e0.74=2.10, sum=4.05 → [0.48, 0.52]

    Softmax row B: e0.32=1.38, e0.57=1.77, sum=3.15 → [0.44, 0.56]

  4. Output A = 0.48×[0.2, 0.9] + 0.52×[0.7, 0.1] = [0.10+0.36, 0.43+0.05] = [0.46, 0.48]

Token A pays slightly more attention to B (52%) than to itself (48%). Its output blends both tokens’ values accordingly.

Key Takeaways

  • Attention lets each token decide which other tokens are relevant by computing Q×K scores.
  • Q, K, V come from multiplying embeddings by three learned weight matrices.
  • Dot product of Q and K measures relevance. Higher = more relevant.
  • Scaling by √dk prevents softmax from becoming too peaky.
  • Softmax turns scores into a probability distribution: who gets my attention?
  • Weighted sum of V produces the output: a blend of all tokens’ information, weighted by relevance.
  • Self-attention: Q, K, V all come from the same sequence.
  • Multi-head attention: run attention N times in parallel, each head capturing a different relationship.
  • Quadratic cost: attention is O(n²) in sequence length — the fundamental bottleneck.
