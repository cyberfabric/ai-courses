Chapter 7

The Full Transformer Architecture

Layer norm, residual connections, FFN, and positional encoding — the complete picture beyond attention

Companion: This expands on AI-Intro Ch 5: The Transformer. There you saw the transformer as a black box. Now you’ll trace every operation inside a single transformer block.

In Chapter 6, you computed attention from scratch — the mechanism that lets tokens pay attention to each other. But attention is only one of four operations in a transformer block. The other three are just as essential, and without them, attention alone would collapse.

In this chapter, you’ll trace all four operations with real numbers and understand why each one exists. By the end, you’ll know exactly what happens inside one transformer “layer” — and GPT-4 has about 120 of them stacked in sequence.

That’s one transformer block. The input enters, passes through all four operations, and exits as a refined representation. Then it enters the next block. And the next. And the next — 96 times in GPT-3.

Let’s trace each operation with a 4-dimensional vector, starting from where Chapter 6 left off: the attention output for one token.

Residual Connections: Don’t Forget the Original

Residual Connection (Skip Connection)

Instead of replacing the input with the attention output, add the input back to the output. The formula is simply: output = input + attention(input). This “skip connection” lets information flow directly through the network without being forced through every transformation.

Why is this necessary? Deep networks suffer from the vanishing gradient problem: by the time error signals travel backward through 96 layers, they shrink to nearly zero, and early layers stop learning. Residual connections create a “highway” for gradients to flow directly to early layers.

Residual connection with numbers

Input to the block: [0.50, 0.30, 0.80, 0.10]

Attention output: [0.20, -0.10, 0.30, 0.40]

Add them element by element:

PositionInput+Attention=Result
10.50+0.20=0.70
20.30+-0.10=0.20
30.80+0.30=1.10
40.10+0.40=0.50

Residual output: [0.70, 0.20, 1.10, 0.50]

What the residual connection means

The original input is preserved in the output. Attention only needs to learn what to add to the representation — the refinement, the adjustment. It doesn’t have to learn to reproduce everything the input already knew. Think of it as editing a document: you keep the original draft and mark up changes, rather than rewriting from scratch each time.

Layer Normalization: Keeping Numbers Tame

Layer Normalization

Normalize a vector to have mean = 0 and variance = 1, then apply learnable scale (γ) and shift (β) parameters. This prevents numbers from growing too large or too small as they pass through dozens of layers.

After 96 transformer blocks, numbers could easily explode to millions or collapse to near-zero. Layer norm keeps them in a stable range at every step.

Layer normalization step by step

Input vector: [0.70, 0.20, 1.10, 0.50] (our residual output from above)

Step 1: Compute the mean

μ = (0.70 + 0.20 + 1.10 + 0.50) / 4 = 2.50 / 4 = 0.625

Step 2: Subtract the mean (center at zero)

Step 5: Scale and shift (learnable parameters γ and β)

Final = γ × normalized + β. Typically γ starts at 1 and β at 0, so the initial output equals the normalized values. During training, the model learns the best γ and β for each position.

Checkpoint

After layer normalization, the vector [0.70, 0.20, 1.10, 0.50] became [0.23, -1.30, 1.45, -0.38]. What’s the key property of the normalized output?

Answer: Mean ≈ 0 and variance ≈ 1. The values are centered and have consistent scale, regardless of what the input values were.

The Feed-Forward Network: Token-Level Processing

After attention and Add&Norm, each token passes through a feed-forward network (FFN) — independently. This is the same kind of neural network from Chapters 2–3: two linear layers with an activation function in between.

Feed-Forward Network (FFN)

FFN(x) = W2 × ReLU(W1 × x + b1) + b2

The hidden layer is typically 4× wider than the model dimension. If the model is 4D, the FFN hidden layer is 16D. This expansion lets the network process and transform the information from attention.

Why is this needed? Attention tells each token what to look at, but the FFN processes what to do with that information. Researchers have found that FFNs often store factual knowledge — “Paris is the capital of France” lives in the FFN weights.

FFN with numbers: 4D → 8D → 4D

Input (after layer norm): [0.23, -1.30, 1.45, -0.38]

Layer 1: Multiply by W1 (4×8 matrix) + bias, then ReLU.

We’ll show 3 of the 8 hidden neurons to keep things readable:

Hidden NeuronWeighted SumReLU
h10.820.82
h2-0.450.00 ← killed by ReLU
h31.231.23

All 8 hidden values after ReLU: [0.82, 0.00, 1.23, 0.55, 0.00, 0.91, 0.37, 0.00]

Layer 2: Multiply by W2 (8×4 matrix) + bias. Projects back to 4D.

FFN output: [0.41, -0.22, 0.53, 0.18]

Expand, process, compress

The FFN expands 4D → 8D, applies ReLU (killing some neurons), then compresses back to 4D. This “expand and compress” pattern lets the network explore a richer space of representations before projecting back. Three of the 8 hidden neurons were killed by ReLU — the FFN decided those “features” weren’t relevant for this token. The surviving neurons carry the useful information forward.

Second Add & Norm

After the FFN, we apply the same residual connection and layer normalization again:

Second residual + layer norm

Input to FFN (from first layer norm): [0.23, -1.30, 1.45, -0.38]

FFN output: [0.41, -0.22, 0.53, 0.18]

Residual connection (add):

[0.23+0.41, -1.30+(-0.22), 1.45+0.53, -0.38+0.18] = [0.64, -1.52, 1.98, -0.20]

Layer norm: Normalize to mean≈0, variance≈1 (same process as before).

Block output: [0.37, -1.26, 1.43, -0.54]

That’s the complete output of one transformer block for one token. This vector enters the next block, where the whole process repeats.

Positional Encoding: Where Are You in the Sentence?

There’s a problem we haven’t addressed: attention treats all tokens the same regardless of position. “The cat sat” and “sat cat the” would produce identical attention scores. But word order obviously matters!

Positional Encoding

Before the first transformer block, add a unique “position fingerprint” to each token’s embedding. This way, the model can distinguish between the same word at position 1 vs. position 100.

The original transformer paper used sinusoidal encoding: each position gets a vector of sine and cosine values at different frequencies.

Positional encoding vectors (4D, simplified)

PositionPE1PE2PE3PE4
0 (“The”)0.001.000.001.00
1 (“cat”)0.840.540.011.00
2 (“sat”)0.91-0.420.021.00
3 (“on”)0.14-0.990.031.00

Each position has a unique pattern. The sine/cosine waves at different frequencies create a unique “fingerprint” that the model can learn to decode.

Combining position with meaning:

Token embedding for “cat”: [0.30, 0.90, 0.70, 0.10]

Position encoding for position 1: [0.84, 0.54, 0.01, 1.00]

Sum: [1.14, 1.44, 0.71, 1.10]

This combined vector encodes both what the token is (cat) and where it is (position 1).

Why addition works

It seems crude to just add two vectors together — wouldn’t that scramble the meaning? In high dimensions (768+), there’s enough “room” for the model to separate the meaning signal from the position signal. Think of it like adding a watermark to an image: the watermark is always there, but it doesn’t destroy the original picture.

Modern alternative: Most current models use RoPE (Rotary Position Embedding) instead of sinusoidal encoding. RoPE rotates the Q and K vectors by amounts proportional to their position, making it easier for the model to extrapolate to longer sequences than it was trained on. The intuition is the same — encode position — but the mechanism is more flexible.

The Causal Mask: No Peeking at the Future

Language models like GPT and Claude generate text left to right. When predicting the next token, the model should only see tokens that came before the current position — not future tokens. This is enforced by a causal mask.

Causal mask for a 4-token sequence

Before softmax, we add a mask to the attention scores. Future positions get -∞, which softmax turns into 0:

“The” can only see itself. “cat” can see “The” and itself. “on” can see everything before it. Nobody can see the future.

Why the mask matters

Without the causal mask, the model could “cheat” during training by looking at the answer. If it’s supposed to predict “sat” after “The cat,” it shouldn’t already be able to see “sat” in its attention. The mask forces the model to make genuine predictions using only past context — exactly as it will need to during generation.

Encoder vs. Decoder

The original transformer paper had two halves. Modern models typically use only one:

TypeAttentionUsed ForExample Models
Encoder-only Bidirectional (sees all tokens) Understanding: classification, embeddings BERT, RoBERTa
Decoder-only Causal (only sees past tokens) Generation: chat, code, writing GPT-4, Claude, LLaMA
Encoder-Decoder Both + cross-attention Translation, summarization T5, original Transformer

GPT-4, Claude, and LLaMA are all decoder-only. They use the causal mask and generate tokens one at a time. We’ll explore why decoder-only won in Chapter 13.

Stacking Blocks: Why Depth Creates Intelligence

One transformer block refines the representation slightly. But stack 96 of them, and something remarkable happens:

ModelLayersParameters
GPT-2 Small12117M
GPT-2 Large36774M
GPT-396175B
LLaMA 2 70B8070B
GPT-4 (estimated)~120~1.8T

Each layer builds on the previous one’s output. Early layers tend to capture simple patterns (syntax, word proximity). Middle layers capture semantic relationships (what refers to what). Late layers capture high-level reasoning and task-specific behavior.

What each layer “learns”: Research on probing classifiers shows that layer 1 already knows parts of speech, layer 5 understands syntax trees, layer 20 grasps semantic roles, and the final layers specialize in the specific task (question answering, code completion, etc.). Each block refines the representation one step further.

The Editing Analogy

A transformer block is like one round of document editing:

  1. Attention = Research. “What information from other parts of this document is relevant here?”
  2. Residual + Layer Norm = Keep the original draft and format the notes.
  3. FFN = Rewriting. Process the gathered information and improve the text.
  4. Residual + Layer Norm = Merge the rewrite with the original, clean up formatting.

One round of editing makes the document a little better. After 96 rounds, it’s polished. Each round has access to the improvements from all previous rounds.

Notice: the FFN has 2/3 of the parameters in each block. Attention gets the fame, but FFN does most of the heavy lifting in terms of parameter count.

Checkpoint

In a transformer block, which component has more parameters: attention or the FFN?

Answer: The FFN. With the standard 4× expansion, the FFN has 8d² parameters vs. attention’s 4d². The FFN is 2/3 of the block.

Exercise: Residual Connection + Layer Norm

Your turn: apply Add & Norm

Input vector: [1.0, -0.5, 0.3, 0.8]

Attention output: [0.4, 0.2, -0.1, 0.3]

Compute:

  1. The residual output (add the two vectors)
  2. The mean of the residual output
  3. The centered values (subtract the mean)
  4. Verify the centered values sum to approximately 0

Solution:

  1. Residual: [1.0+0.4, -0.5+0.2, 0.3+(-0.1), 0.8+0.3] = [1.4, -0.3, 0.2, 1.1]
  2. Mean: (1.4 + (-0.3) + 0.2 + 1.1) / 4 = 2.4 / 4 = 0.6
  3. Centered: [1.4-0.6, -0.3-0.6, 0.2-0.6, 1.1-0.6] = [0.8, -0.9, -0.4, 0.5]
  4. Sum: 0.8 + (-0.9) + (-0.4) + 0.5 = 0.0 ✓

The centering step is the core of layer normalization — making the mean zero. You’d then divide by the standard deviation to make the variance 1.

Key Takeaways

  • A transformer block has four operations: Attention → Add&Norm → FFN → Add&Norm.
  • Residual connections add the input back to the output, creating gradient highways and letting each operation learn refinements.
  • Layer normalization keeps values centered (mean=0, variance=1) to prevent numerical explosion across deep stacks.
  • The FFN expands, processes with ReLU, and compresses back. It holds 2/3 of a block’s parameters and stores factual knowledge.
  • Positional encoding is added to embeddings before the first block so the model knows token order.
  • The causal mask prevents tokens from seeing the future — essential for autoregressive generation.
  • Stacking blocks builds from simple patterns to complex reasoning: 12 layers for basic tasks, 96+ for GPT-3-level capability.
