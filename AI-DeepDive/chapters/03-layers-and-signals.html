Chapter 3

How Networks Think — Layers, Weights & Signals

From one neuron to a full network — trace a complete forward pass with real numbers

Companion: This expands on AI-Intro Ch 2: Neural Networks Without the Math. You saw the big picture there. Now we’ll trace every multiplication.

In Chapter 2, you mastered the single neuron: multiply inputs by weights, add bias, apply activation. But one neuron can only draw one decision boundary. Real problems need hundreds, thousands, or millions of neurons working together.

In this chapter, we’ll connect neurons into layers, represent weights as matrices, and trace a complete forward pass through a multi-layer network — with every number visible. By the end, you’ll be able to compute what a neural network outputs for a given input, entirely by hand.

From One Neuron to a Layer

A layer is simply a group of neurons that all receive the same inputs. Each neuron has its own weights and bias, so each one computes something different from the same data.

Let’s build a layer with 3 inputs and 4 neurons:

What a Layer Does

Every neuron in the layer receives all 3 inputs. Each neuron multiplies those inputs by its own weights, adds its own bias, and applies the activation function. The result is 4 outputs — one per neuron.

3 inputs in, 4 outputs out. The layer has transformed the data from 3 dimensions to 4 dimensions.

How many individual operations? Each of the 4 neurons does 3 multiplications (one per input). That’s 4 × 3 = 12 multiplications, plus 4 bias additions, plus 4 activations. Not bad — but writing it neuron-by-neuron gets tedious fast. Enter matrices.

Weights as a Matrix

Instead of writing out each neuron’s weights separately, we pack them into a weight matrix. Each column is one neuron’s weights:

Neuron 1 Neuron 2 Neuron 3 Neuron 4
Input 1 0.2 -0.1 0.4 0.3
Input 2 0.5 0.3 -0.2 0.1
Input 3 -0.3 0.7 0.1 -0.5

This is a 3×4 matrix (3 rows for inputs, 4 columns for neurons). We’ll call it W.

The biases form a vector: b = [0.1, -0.2, 0.3, 0.0]

The inputs are also a vector: x = [1.0, 0.5, -0.5]

Why Matrices?

A matrix is just a compact way to write “every neuron’s weights, all at once.” The mathematical operation — matrix multiplication — computes all 4 neurons’ weighted sums in one step. This is why GPUs are so important for AI: they’re designed to do matrix math blazingly fast.

Matrix Multiplication: Row Times Column

Matrix multiplication sounds intimidating, but the rule is simple: each output is a dot product of the input row with one column of the weight matrix.

The Dot Product Rule

To get the output for neuron j:

  1. Take the input vector
  2. Take column j of the weight matrix
  3. Multiply element-by-element
  4. Sum all the products

That’s the same weighted sum from Chapter 2 — just organized differently.

Computing All 4 Neurons at Once

Inputs: x = [1.0, 0.5, -0.5]

Neuron 1 (column 1 of W):

z1 = (1.0 × 0.2) + (0.5 × 0.5) + (-0.5 × -0.3) + 0.1

z1 = 0.2 + 0.25 + 0.15 + 0.1 = 0.70

Neuron 2 (column 2 of W):

z2 = (1.0 × -0.1) + (0.5 × 0.3) + (-0.5 × 0.7) + (-0.2)

z2 = -0.1 + 0.15 + (-0.35) + (-0.2) = -0.50

Neuron 3 (column 3 of W):

z3 = (1.0 × 0.4) + (0.5 × -0.2) + (-0.5 × 0.1) + 0.3

z3 = 0.4 + (-0.1) + (-0.05) + 0.3 = 0.55

Neuron 4 (column 4 of W):

z4 = (1.0 × 0.3) + (0.5 × 0.1) + (-0.5 × -0.5) + 0.0

z4 = 0.3 + 0.05 + 0.25 + 0.0 = 0.60

Weighted sums: z = [0.70, -0.50, 0.55, 0.60]

Apply ReLU:

NeuronzReLU(z)
10.700.70
2-0.500.00
30.550.55
40.600.60

Layer output: h = [0.70, 0.00, 0.55, 0.60]

What Just Happened?

Three numbers went in, four numbers came out. The layer transformed our 3-dimensional input into a 4-dimensional representation. Neuron 2 output zero (ReLU killed its negative sum) — it found nothing relevant in this input. The other three neurons each detected different patterns and passed their signals forward.

Quick Check

How many individual multiplications did we just do?

Answer: 12 — each of the 4 neurons multiplied 3 inputs by 3 weights. This is why we call it a 3×4 matrix: 3 inputs, 4 neurons, 12 products.

The Forward Pass: Layer by Layer

A neural network is just layers stacked in sequence. The output of one layer becomes the input to the next. This process of feeding data forward through the network is called the forward pass.

We already computed the hidden layer output: h = [0.70, 0.00, 0.55, 0.60]. Now this becomes the input to the output layer.

The Output Layer

Our output layer has 2 neurons (for binary classification — say, “cat” vs. “dog”). It takes 4 inputs (from the hidden layer) and produces 2 raw scores.

Output layer weights (4×2 matrix):

Out 1 (“Cat”) Out 2 (“Dog”)
h1 0.6 -0.4
h2 -0.2 0.5
h3 0.3 0.1
h4 -0.1 0.7

Output biases: bout = [0.1, -0.1]

Full Forward Pass: Hidden to Output

Hidden output: h = [0.70, 0.00, 0.55, 0.60]

Output neuron 1 (“Cat”):

zcat = (0.70×0.6) + (0.00×-0.2) + (0.55×0.3) + (0.60×-0.1) + 0.1

zcat = 0.42 + 0.00 + 0.165 + (-0.06) + 0.1 = 0.625

Output neuron 2 (“Dog”):

zdog = (0.70×-0.4) + (0.00×0.5) + (0.55×0.1) + (0.60×0.7) + (-0.1)

zdog = -0.28 + 0.00 + 0.055 + 0.42 + (-0.1) = 0.095

Raw scores: [0.625, 0.095]

Softmax: Turning Scores into Probabilities

Raw scores are hard to interpret. Is 0.625 confident? The softmax function converts scores into probabilities that sum to 1.

Softmax

For each score, softmax does:

  1. Raise e to the power of each score
  2. Divide each result by the sum of all results

This guarantees all outputs are positive and they sum to exactly 1.0 — a valid probability distribution. We’ll dive deep into softmax in Chapter 6.

Softmax Step by Step

Raw scores: [0.625, 0.095]

Final prediction: 63% Cat, 37% Dog. The network thinks this input is more likely a cat.

The Full Journey

Let’s trace the complete path: 3 raw numbers went into the hidden layer, got transformed into 4 hidden values (one was zeroed by ReLU), then those 4 values were combined into 2 scores, and softmax turned those scores into a probability distribution. That’s a forward pass. Every prediction you’ve ever gotten from ChatGPT, Claude, or any neural network follows this same pattern — just with many more layers and neurons.

What Each Layer “Sees”

Each layer transforms its input into something more useful for the next layer. In a deep network, this creates a feature hierarchy:

Image Model

  • Layer 1: edges, gradients
  • Layer 2: textures, corners
  • Layer 3: eyes, ears, noses
  • Layer 4+: faces, objects

Text Model (LLM)

  • Layer 1: character patterns
  • Layer 2: word fragments
  • Layer 3: grammar, syntax
  • Layer 4+: meaning, context

Nobody programs these features. The network discovers them automatically during training (Chapter 4). Each layer builds on the previous one, composing simple patterns into complex concepts.

Why Depth Matters More Than Width

You might think: “Instead of 4 layers of 100 neurons, why not 1 layer of 400 neurons?” Mathematically, these are very different:

Width (more neurons in one layer) lets you detect more patterns at the same level of abstraction. Depth (more layers) lets you detect patterns built on top of other patterns.

A wide network might detect 400 different edges. A deep network detects edges, then builds shapes from edges, then builds objects from shapes. Depth creates composition — and composition is how you go from pixels to “that’s a golden retriever.”

The Factory Assembly Line

A multi-layer network is like a factory assembly line. Raw materials (inputs) enter at one end. Each station (layer) performs one transformation: cutting, shaping, polishing, assembling. No single station does something impressive on its own, but the sequence of stations turns raw metal into a finished product. Adding more workers to one station (width) helps, but adding more stations (depth) is what enables building complex things.

Counting Parameters

For our tiny network, let’s count every learnable parameter:

Layer Shape Weights Biases Total
Hidden (input → hidden) 3 × 4 12 4 16
Output (hidden → output) 4 × 2 8 2 10
Grand total 26

The formula per layer: (inputs × neurons) + neurons. The first term is the weight matrix; the second is the bias vector.

Quick Check

If you added a second hidden layer with 8 neurons (so the architecture becomes 3 → 4 → 8 → 2), how many total parameters would the network have?

Answer: Layer 1: (3×4)+4 = 16. Layer 2: (4×8)+8 = 40. Layer 3: (8×2)+2 = 18. Total: 16 + 40 + 18 = 74 parameters.

Why You Can’t Run GPT-4 on Your Laptop

1.8 trillion parameters stored as 16-bit floats = ~3.6 terabytes of memory just for the weights. That’s before you account for activations, gradients, and the KV cache (Chapter 10). This is why large models need clusters of GPUs with hundreds of gigabytes of memory each.

The Complete Forward Pass: Summary Table

Here’s our entire network’s computation in one view:

Stage Operation Values
Input — [1.0, 0.5, -0.5]
Hidden (weighted sum) x × W1 + b1 [0.70, -0.50, 0.55, 0.60]

Six stages. Three inputs became a prediction with confidence. That’s the forward pass.

  2. Apply ReLU to get hidden outputs (h1, h2, h3)
  3. Compute the output weighted sum
  4. Apply sigmoid to get the final prediction

Solution:

Hidden weighted sums:

  • z1 = (2×0.3) + (1×0.4) + 0.1 = 0.6 + 0.4 + 0.1 = 1.1
  • z2 = (2×-0.5) + (1×0.1) + 0.0 = -1.0 + 0.1 + 0.0 = -0.9
  • z3 = (2×0.2) + (1×-0.3) + (-0.1) = 0.4 - 0.3 - 0.1 = 0.0

Output:

zout = (1.1×0.5) + (0.0×-0.3) + (0.0×0.8) + 0.2 = 0.55 + 0 + 0 + 0.2 = 0.75

sigmoid(0.75) = 1/(1 + e-0.75) ≈ 0.68

Final prediction: 0.68 (68% confidence for the positive class).

What’s Next

You can now compute what a network outputs for any given input. But there’s a catch: we used weights that were given to us. Where do those weights come from? The answer is training — and training is a loop: make a prediction (forward pass), measure how wrong it is (loss), figure out which weights to blame (backward pass), and nudge them in the right direction (update).
