Chapter 2

Inside a Neuron — The Math That Isn’t Scary

Weights, bias, activation functions — trace a single neuron’s computation with real numbers

Companion: This expands on AI-Intro Ch 2: Neural Networks Without the Math. In AI-Intro, you met the neuron as a concept. Now you’ll meet it as a calculation.

In Chapter 1, we used weights, a bias, and a threshold to classify data points. That formula — w×x + b — is literally what a single artificial neuron computes. The neuron is the atom of neural networks. Everything else is just many neurons working together.

In this chapter, we’ll trace exactly what happens inside one neuron, with real numbers. The grand total of math required: multiplication, addition, and one simple rule. You can do it all on a napkin.

The Single Neuron: Anatomy

An artificial neuron takes in several inputs, multiplies each by a weight, adds them all up with a bias, and then applies an activation function. That’s it. Here’s the formula:

output = activation( w1×x1 + w2×x2 + w3×x3 + b )

Let’s name the parts:

Part Symbol What It Does
Inputs x1, x2, x3 The data coming in (features, or outputs from previous neurons)
Weights w1, w2, w3 How much each input matters (learned during training)
Bias b A constant that shifts the threshold (also learned)
Weighted sum z All inputs × weights, added together, plus bias
Activation function f(z) A simple rule applied to the sum (the “decision gate”)

The Weighted Sum: Step by Step

Let’s use concrete numbers. Our neuron has 3 inputs:

  • Inputs: x1 = 0.5, x2 = 0.8, x3 = 0.2
  • Weights: w1 = 0.4, w2 = -0.3, w3 = 0.7
  • Bias: b = 0.1

Computing the Weighted Sum

Step 1: Multiply each input by its weight:

Input Weight Product
x1 = 0.5 w1 = 0.4 0.5 × 0.4 = 0.20
x2 = 0.8 w2 = -0.3 0.8 × -0.3 = -0.24
x3 = 0.2 w3 = 0.7 0.2 × 0.7 = 0.14

Step 2: Add all products together plus the bias:

z = 0.20 + (-0.24) + 0.14 + 0.1 = 0.20

Result: The weighted sum is z = 0.20. This is the “raw score” before the activation function.

What Does the Weighted Sum Mean?

Each weight controls how much its input matters. The weight w1 = 0.4 says “input 1 is moderately important, in a positive way.” The weight w2 = -0.3 says “input 2 is somewhat important, but it pushes the answer down.” Negative weights suppress inputs; positive weights amplify them.

The bias b = 0.1 is like the neuron’s “default mood” — it shifts the output up or down regardless of the inputs.

Quick Check

If we changed w2 from -0.3 to +0.3, would z go up or down? By how much?

Answer: It would go up. The product changes from -0.24 to +0.24, a difference of 0.48. New z = 0.20 + 0.24 + 0.14 + 0.1 = 0.68. Flipping a weight’s sign is a big deal!

Why We Need Activation Functions

The weighted sum z is just a linear function — multiply and add. If we stacked 100 neurons that each just computed weighted sums with no activation function, the entire stack would collapse into a single weighted sum. Mathematically:

The Linear Collapse Problem

A linear function of a linear function is still a linear function. Without activation functions, a 100-layer network is no more powerful than a single layer. All those extra neurons would be wasted — you’d just be computing z = (some big weight) × x + (some big bias).

The activation function breaks this linearity. It introduces a non-linear twist that lets each layer create genuinely new patterns. Let’s meet the three most common ones.

ReLU: The Most Popular Activation

ReLU (Rectified Linear Unit)

The rule is embarrassingly simple:

  • If the input is positive, keep it as-is
  • If the input is negative, output 0

In math: ReLU(z) = max(0, z). That’s it. This is the most widely used activation function in modern neural networks.

Let’s apply ReLU to 5 different values to see the pattern:

ReLU in Action

Input (z) z > 0? ReLU(z) What Happened
2.5 Yes 2.5 Kept as-is
0.20 Yes 0.20 Kept as-is (this is our neuron’s z!)
0.0 No 0.0 On the boundary — outputs 0
-1.3 No 0.0 Negative → killed to zero
-5.0 No 0.0 Very negative → still just zero

For our neuron with z = 0.20: ReLU(0.20) = 0.20. The value passes through unchanged.

ReLU is popular because it’s fast (just a comparison), it doesn’t suffer from some problems that older activation functions have, and it works surprisingly well in practice. About 80% of neural networks use it or a variant of it.

Why “Killing” Negatives Is Useful

When a neuron outputs zero for a particular input, it effectively says: “This combination of inputs is not relevant — I have nothing to contribute.” This creates sparsity — many neurons stay silent for any given input, and only the relevant ones “fire.” This is efficient and helps the network learn distinct features.

Sigmoid: Squashing Into 0–1

Sigmoid

Sigmoid takes any number — positive, negative, huge, tiny — and squashes it into a value between 0 and 1. The formula is:

σ(z) = 1 / (1 + e-z)

Don’t panic about the e — it’s just a number (~2.718). The important thing is the behavior: large positive inputs → close to 1, large negative inputs → close to 0, zero input → exactly 0.5.

Let’s apply sigmoid to the same 5 values:

Sigmoid in Action

Input (z) e-z 1 + e-z σ(z) Interpretation
2.5 0.08 1.08 0.92 Strong positive → close to 1
0.20 0.82 1.82 0.55 Slightly positive → just above 0.5
0.0 1.00 2.00 0.50 Zero → exactly the midpoint
-1.3 3.67 4.67 0.21 Negative → closer to 0
-5.0 148.4 149.4 0.01 Very negative → nearly 0

For our neuron: σ(0.20) = 0.55. The sigmoid turns our raw score into something we can interpret as a probability — “55% confidence.”

When to Use Sigmoid

Sigmoid is the go-to for output neurons in binary classification: “Is this spam? Is this a cat?” The output is a probability between 0 and 1. You don’t see it much inside hidden layers anymore (ReLU won that battle), but it’s still king for yes/no output layers.

Tanh: Sigmoid’s Cousin

Tanh is like sigmoid but shifted to output between -1 and +1 instead of 0 and 1:

Input (z) tanh(z) Note
2.5 0.99 Close to +1
0.20 0.20 Nearly linear near zero
0.0 0.00 Zero-centered (unlike sigmoid)
-1.3 -0.86 Negative output is possible
-5.0 -1.00 Close to -1

The advantage of tanh: outputs are centered at zero, which means neurons can output negative values. This makes training slightly easier in some cases. You’ll see tanh in older networks and in some specific architectures (like LSTMs), but ReLU dominates modern practice.

Comparing All Three

Property ReLU Sigmoid Tanh
Output range [0, ∞) (0, 1) (-1, 1)

Quick Check

Our neuron computed z = 0.20. What would the output be with each activation function?

Answer: ReLU(0.20) = 0.20. Sigmoid(0.20) = 0.55. Tanh(0.20) = 0.20. Note how ReLU and tanh happen to give the same answer here — for small positive values, they’re nearly identical.

Why Non-Linearity Matters: A Visual Argument

Let’s make the “linear collapse” problem concrete with numbers.

Stacking Linear Layers Collapses

Imagine two layers, each with a single neuron (no activation function):

Layer 1: z1 = 2 × x + 1

Layer 2: z2 = 3 × z1 + 0.5

Substituting layer 1 into layer 2:

z2 = 3 × (2x + 1) + 0.5 = 6x + 3 + 0.5 = 6x + 3.5

The two layers collapsed into a single linear function: 6x + 3.5. A single neuron with weight 6 and bias 3.5 does the same thing. Two layers wasted!

Now add ReLU after layer 1:

  • If x = 1: z1 = 3 → ReLU = 3 → z2 = 3×3 + 0.5 = 9.5
  • If x = -1: z1 = -1 → ReLU = 0 → z2 = 3×0 + 0.5 = 0.5

Now the two-layer network behaves differently for positive and negative inputs — it has a “kink” that one layer can’t reproduce. That kink is where the power comes from.

The Key Insight

Each activation function creates a “bend” in the function the network computes. Stack enough bends together and you can approximate any shape: curves, corners, circles, anything. This is why deep networks are so expressive — each layer adds more bends, carving more complex decision surfaces.

The Bias as a “Shift”

The bias might seem like a minor detail, but it’s crucial. Without it, the neuron’s “activation threshold” is stuck at zero — it can only respond to inputs whose weighted sum is positive.

With a bias, you shift that threshold:

Scenario Bias Effect
Positive bias (b = 2) Shifts threshold left Neuron fires more easily — doesn’t need as much from inputs
Zero bias (b = 0) No shift Threshold is at zero
Negative bias (b = -3) Shifts threshold right Neuron needs strong input to fire — more selective

Think of the bias as the neuron’s baseline sensitivity. A neuron with a large negative bias is picky — it only fires for strong, confident signals. A neuron with a positive bias is eager — it fires at the slightest nudge.

One Neuron = One Tiny Classifier

Here’s something beautiful: a single neuron with sigmoid activation is already a classifier. It takes inputs, computes a score, and outputs a probability between 0 and 1. Sound familiar? It should — it’s exactly the linear classifier from Chapter 1, with an activation function bolted on.

Chapter 1’s Linear Classifier

w1×x1 + w2×x2 + b > 0?

Yes → Class A. No → Class B.

Binary output: just yes or no.

One Neuron with Sigmoid

σ(w1×x1 + w2×x2 + b)

Output: a probability from 0 to 1.

Smoother: “72% likely Class A.”

One neuron draws one decision boundary — a single line through feature space. To draw complex, curvy boundaries? You need many neurons. That’s Chapter 3.

The Talent Show Analogy

A neuron is like a judge at a talent show:

  • Each act (input) gets a weighted score — the judge cares more about some acts than others
  • The bias is the judge’s baseline mood — a grumpy judge (negative bias) is harder to impress
  • The weighted sum is the total impression — good acts push it up, bad acts push it down
  • The activation function is the judge’s final decision rule — ReLU says “if positive, clap; if negative, silence.” Sigmoid says “rate it from 0 to 1.”

That’s the Whole Thing

Multiply, add, apply one rule. Every neuron in every neural network — from a 3-neuron toy to GPT-4’s trillions of parameters — does exactly this computation. The magic isn’t in the individual neuron. It’s in what happens when you connect thousands of them together.

How Many Parameters?

One neuron with 3 inputs has:

  • 3 weights (one per input)
  • 1 bias
  • Total: 4 parameters

The general formula: a neuron with n inputs has n + 1 parameters (n weights + 1 bias).
