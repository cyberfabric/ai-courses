Chapter 4

Training Step by Step — Forward, Loss, Backward

One complete training iteration with real numbers — from wrong prediction to weight update

Companion: This expands on AI-Intro Ch 3: How AI Learns. There you saw the high-level loop. Now we’ll trace every number.

In Chapter 3, you traced a forward pass through a network and got a prediction. But we gave you the weights. In reality, a network starts with random weights — it knows absolutely nothing. Training is how it goes from random guessing to useful predictions.

In this chapter, we’ll walk through one complete training step: make a prediction, measure how wrong it is, figure out which weights to blame, and nudge them in the right direction. All with real numbers.

The Training Loop: Four Steps

1. Forward

Feed input, get prediction

→

2. Loss

Measure the error

→

3. Backward

Find who’s to blame

→

4. Update

Adjust the weights

Repeat these four steps thousands (or millions) of times, and the network gradually gets better. Let’s trace one iteration.

Step 1: The Forward Pass (Getting It Wrong)

We’ll use a tiny network with just 2 inputs, 1 hidden neuron, and 1 output — the simplest possible multi-layer network. This keeps the numbers manageable while showing the full training process.

Network:

  • Input: x = [1.0, 0.5]
  • Hidden weights: w1 = 0.3, w2 = -0.2, bias bh = 0.1
  • Activation: ReLU
  • Output weight: w3 = 0.6, bias bo = 0.0
  • Output activation: sigmoid

Target: The correct answer is 1.0 (the input belongs to the positive class).

Forward Pass

Hidden neuron:

zh = (1.0 × 0.3) + (0.5 × -0.2) + 0.1 = 0.3 - 0.1 + 0.1 = 0.3

h = ReLU(0.3) = 0.3

Output neuron:

zo = (0.3 × 0.6) + 0.0 = 0.18

ŷ = sigmoid(0.18) = 1/(1 + e-0.18) ≈ 0.545

Prediction: 0.545. Target: 1.0. The network is wrong — it’s barely above 50/50.

Step 2: Measuring the Error (Loss)

We need a single number that says “how wrong is this prediction?” That number is the loss.

Mean Squared Error (MSE)

The simplest loss function: take the difference between prediction and target, square it.

Loss = (predicted - actual)²

Squaring does two things: it makes the loss always positive (no negative errors), and it punishes big errors much more than small ones.

Computing MSE Loss

ValueNumber
Predicted (ŷ)0.545
Actual (y)1.0
Difference0.545 - 1.0 = -0.455
Loss(-0.455)² = 0.207

The loss is 0.207. Not great. A perfect prediction would give loss = 0.

Cross-Entropy Loss (The Other Option)

For classification tasks, cross-entropy is usually preferred over MSE. It uses logarithms to punish confident-but-wrong predictions especially hard:

Loss = -[y × log(ŷ) + (1 - y) × log(1 - ŷ)]

For our numbers: -[1.0 × log(0.545)] = -(-0.607) = 0.607. Cross-entropy gives a higher loss here because it’s more sensitive to how far the probability is from the target. We’ll stick with MSE for simplicity in this walkthrough.

Step 3: The Backward Pass (Who’s to Blame?)

We know the prediction is wrong. Now we need to figure out: which weights should we change, and in which direction?

This is backpropagation — and despite its scary name, the idea is beautifully simple.

Backpropagation: The Blame Game

Start at the output (where the error is). Ask: “Which weights in the last layer contributed to this error?” Then move one layer back and ask the same question. Keep going until you reach the first layer.

For each weight, you compute a gradient — a number that tells you:

  • Direction: should this weight go up or down?
  • Magnitude: how much does changing this weight affect the error?

The chain rule in plain English: The gradient of a weight tells you “if I nudge this weight up by a tiny amount, how much does the loss change?” If the loss goes up, the gradient is positive (weight should decrease). If the loss goes down, the gradient is negative (weight should increase).

Let’s compute gradients for our 3 weights. We won’t use calculus — we’ll use the intuition of “tracing the chain of influence.”

Gradient for w3 (output weight)

The output weight w3 directly affects the output. The chain of influence is:

w3 → zo → ŷ → Loss

Gradient of w3 Step by Step

We need three pieces, multiplied together:

1. How does Loss change with ŷ?

dLoss/dŷ = 2 × (ŷ - y) = 2 × (0.545 - 1.0) = -0.91

2. How does ŷ change with zo? (sigmoid derivative)

dŷ/dzo = ŷ × (1 - ŷ) = 0.545 × 0.455 = 0.248

3. How does zo change with w3?

dzo/dw3 = h = 0.3 (the input to this weight)

Chain them together:

gradientw3 = -0.91 × 0.248 × 0.3 = -0.068

What Does -0.068 Mean?

The gradient is negative, which means: increasing w3 would decrease the loss. That’s what we want! The weight should go up. It makes sense — the network predicted too low (0.545 vs. 1.0), so strengthening the connection from the hidden neuron to the output should push the prediction higher.

Gradients for w1 and w2 (hidden weights)

For hidden weights, the chain is longer — the error has to travel through more steps:

w1 → zh → h → zo → ŷ → Loss

Gradients of Hidden Weights

We already computed the “output error signal”:

dLoss/dzo = -0.91 × 0.248 = -0.226

This signal flows backward through w3:

dLoss/dh = -0.226 × w3 = -0.226 × 0.6 = -0.135

Through ReLU (since h was positive, the derivative is 1):

dLoss/dzh = -0.135 × 1 = -0.135

Now for each hidden weight:

WeightInputGradient
w1 x1 = 1.0 -0.135 × 1.0 = -0.135
w2 x2 = 0.5 -0.135 × 0.5 = -0.068

All gradients:

WeightCurrent ValueGradientDirection
w10.3-0.135Increase ↑
w2-0.2-0.068Increase ↑
w30.6-0.068Increase ↑

The Blame Report

All three gradients are negative, meaning all three weights should increase. This makes sense: the network predicted too low (0.545 instead of 1.0), so every weight in the chain needs to push harder. Weight w1 gets the biggest adjustment (-0.135) because it has the most direct influence — its input was 1.0, the largest signal in the network.

Quick Check

If the hidden neuron’s output h had been zero (killed by ReLU), what would happen to the gradients of w1 and w2?

Answer: They would be zero. ReLU’s derivative is 0 for negative inputs, so no error signal flows through a dead neuron. The hidden weights get no update — this is the “dead neuron” problem mentioned in Chapter 2.

Step 4: Updating the Weights

Now we have gradients. The update rule is beautifully simple:

new_weight = old_weight - learning_rate × gradient

The learning rate (usually written as η or lr) controls how big each step is. Let’s use lr = 0.1.

Weight Update with lr = 0.1

WeightOldGradientlr × gradientNew
w1 0.300 -0.135 -0.014 0.314
w2 -0.200 -0.068 -0.007 -0.193
w3 0.600 -0.068 -0.007 0.607

Every weight moved slightly in the direction that reduces the loss. The changes are small — that’s intentional. Training is a slow, careful process of many tiny adjustments.

If we ran the forward pass again with the updated weights, the prediction would be slightly closer to 1.0. After hundreds or thousands of iterations with many training examples, the network converges to weights that make accurate predictions.

The Learning Rate: Too Big vs. Too Small

The learning rate is one of the most important choices in training. Let’s see why with the same gradient for w1:

Learning Rate Update Amount New w1 Problem?
0.0001 (too small) 0.000014 0.300014 Barely moved. Training will take forever.
0.1 (good) 0.014 0.314 Nice, steady progress.
0.5 (too big) 0.068 0.368 Big jump. Might overshoot the sweet spot.
5.0 (way too big) 0.675 0.975 Wild swing. Loss could increase.

The Goldilocks Problem

Too small: training is painfully slow and may get stuck. Too big: training becomes unstable — the loss bounces around or even explodes. Typical starting learning rates: 0.001 to 0.01 for most networks. Modern optimizers like Adam adjust the rate automatically during training.

The Mixing Board Analogy

Training is like adjusting the dials on a mixing board while listening to music:

  • Each dial (weight) changes the sound a little
  • You listen to the output (prediction) and compare it to the target (the song you want)
  • The loss is how far off it sounds
  • You figure out which dials are most wrong (gradients)
  • You nudge each dial slightly (weight update)
  • Repeat. Hundreds of thousands of times.

With 175 billion dials (GPT-3), you need a computer. But the process is the same.

Epochs and Batches

One training step uses one example. But real datasets have millions of examples. Two important terms:

Epoch

One epoch = one pass through the entire training dataset. If you have 10,000 examples and train for 100 epochs, the model sees each example 100 times.

Batch Size: Finding the Middle Ground

You could update weights after every single example, but that’s noisy — one weird example might push weights in the wrong direction. You could also wait until you’ve seen the entire dataset, but that’s slow. The compromise: mini-batches.

Strategy Batch Size Pros Cons
Stochastic 1 Fast updates, can escape local minima Noisy, unstable gradients
Mini-batch 32-512 Good balance, efficient on GPUs Needs tuning
Full batch Entire dataset Stable, accurate gradients Slow, memory-heavy, can get stuck

In practice, batch sizes of 32 to 256 are most common. GPUs process batches in parallel, so a batch of 64 takes barely longer than a batch of 1. Free parallelism.

The Loss Curve: A Training Story

During training, you track the loss at each step. A healthy training run looks like this:

Overfitting: When the Model Memorizes

At epoch 200, training loss is almost zero (the model gets training examples nearly perfect), but validation loss has increased. The model memorized the training data instead of learning general patterns. It’s like a student who memorizes the textbook word for word but fails on new questions.

Healthy Training

  • Training loss: 0.15
  • Validation loss: 0.20
  • Small gap — model generalizes well

Overfitting

  • Training loss: 0.01
  • Validation loss: 0.50
  • Huge gap — model memorized, doesn’t generalize

Common fixes for overfitting: more training data, dropout (randomly zeroing neurons during training), early stopping (stop training when validation loss starts rising), and regularization (penalizing large weights).

Quick Check

After the weight update, would the prediction be closer to 1.0 or further from it?

Answer: Closer. Every weight moved in the direction that reduces the loss, so the next forward pass will produce a prediction slightly higher than 0.545, nudging toward the target of 1.0.

Exercise: One Training Step

Exercise: Compute a Weight Update

A single neuron (no hidden layer) has:

  • Input: x = 2.0
  • Weight: w = 0.5
  • Bias: b = 0.0
  • No activation function (linear output)
  • Target: y = 3.0
  • Loss: MSE = (predicted - actual)²
  • Learning rate: lr = 0.05

Tasks:

  1. Compute the prediction: ŷ = w × x + b
  2. Compute the MSE loss
  3. Compute the gradient: dLoss/dw = 2 × (ŷ - y) × x
  4. Update the weight: w_new = w - lr × gradient
  5. What would the prediction be with the updated weight?

Solution:

  1. ŷ = 0.5 × 2.0 + 0.0 = 1.0
  2. Loss = (1.0 - 3.0)² = 4.0
  3. gradient = 2 × (1.0 - 3.0) × 2.0 = 2 × (-2.0) × 2.0 = -8.0
  4. w_new = 0.5 - 0.05 × (-8.0) = 0.5 + 0.4 = 0.9
  5. ŷ_new = 0.9 × 2.0 = 1.8 — moved from 1.0 toward 3.0. Progress!
