Chapter 9

Sampling Strategies — The Numbers Behind Creativity

Temperature, top-k, top-p with actual probability distributions — see how each reshapes the output

Companion: This expands on AI-Intro Ch 7: Temperature & Friends. There you learned what the knobs do conceptually. Now you’ll see exactly how each one reshapes the probability distribution with real numbers.

In Chapter 8, you traced the autoregressive loop: predict → select → append → repeat. But we glossed over the most interesting part: how does “select” actually work?

The model doesn’t output probabilities directly. It outputs raw scores called logits — just big and small numbers with no particular range. Everything interesting happens between those raw scores and the final token selection. That’s where temperature, top-k, and top-p live.

By the end of this chapter, you’ll be able to take a set of logits, apply any combination of these strategies, and predict exactly which tokens can be selected and with what probability.

Starting Point: Raw Logits

After the transformer processes your prompt, the final layer produces one number per vocabulary token. For a vocabulary of 50,257 tokens, that’s 50,257 numbers. These are the logits.

Let’s work with a tiny vocabulary of 5 tokens to keep things traceable:

TokenLogit
the2.0
a1.0
cat0.5
my-1.0
banana-3.0

Higher logit = the model thinks this token is more likely. But 2.0 doesn’t mean 2x as likely as 1.0. Logits aren’t probabilities. To convert them, we need softmax.

Softmax: Turning Logits into Probabilities

The Softmax Function

Softmax takes a list of logits and converts them to probabilities that sum to 1. The formula for each token: P(token) = e^logit / sum(e^all_logits).

Two key properties: (1) every output is between 0 and 1, and (2) they all add up to exactly 1.0. Larger logits get exponentially larger probabilities.

Step 1: Exponentiate each logit

Step 2: Sum all exponentials

7.389 + 2.718 + 1.649 + 0.368 + 0.050 = 12.174

Step 3: Divide each by the sum

What Just Happened

The model is 60.7% confident the next token should be “the”. That’s the baseline probability distribution — before any sampling strategy is applied. Notice how the gap between logits 2.0 and 1.0 became a nearly 3:1 ratio in probability (60.7% vs 22.3%). That’s the exponential at work — small logit differences create big probability differences.

Temperature: The Creativity Dial

Temperature is the simplest and most powerful sampling control. The idea: divide all logits by T before applying softmax.

P(token) = e^(logit/T) / sum(e^(all_logits/T))

That’s it. One division changes everything.

What Temperature Does

T < 1 — Dividing by a fraction makes logits bigger. The differences between tokens get amplified. The top token dominates even more. Output becomes more deterministic.

T = 1 — No change. Standard softmax.

T > 1 — Dividing by a large number makes logits smaller. The differences shrink. Probabilities become more uniform. Rare tokens get a real chance.

Reading the Results

Look at “the” across all three temperatures: 86.7% → 60.7% → 36.2%. Same model, same logits, wildly different confidence.

At T=0.5, “the” is almost certain. At T=2.0, even “banana” has a 3% chance. This is why high temperature makes models “creative” — it’s literally giving unlikely tokens a fighting chance.

Temperature Near Zero

As T approaches 0, the distribution becomes a spike: 100% on the top token, 0% on everything else. This is equivalent to greedy decoding from Chapter 8.

Most APIs use T=0 (or a tiny value like 0.001) for “deterministic” output. Mathematically, dividing by near-zero makes the top logit approach infinity while others become negligible.

Watch out: Temperature 0 doesn’t mean the model is “more correct.” It means the model is more committed — it always picks its top choice, whether that choice is right or wrong. For factual Q&A, low temperature reduces hallucination by avoiding unlikely completions. For creative tasks, it produces bland, repetitive output.

Quick Check

If logits are [3.0, 3.0, 3.0] (all equal), what does any temperature setting do?

Answer: Nothing! All three tokens remain equally likely (33.3% each) regardless of temperature. Temperature amplifies or reduces differences between logits. If there’s no difference, there’s nothing to amplify.

Top-k Sampling: Keep the Best, Drop the Rest

Temperature reshapes the whole distribution. Top-k takes a more brutal approach: keep only the top K tokens, set everything else to zero, renormalize.

Top-k Sampling

  1. Sort tokens by probability (highest first)
  2. Keep only the top K tokens
  3. Set all other tokens’ probabilities to 0
  4. Renormalize the remaining K tokens so they sum to 1
  5. Sample from this filtered distribution

Top-k with k=3

Start with our T=1.0 probabilities:

What Top-k Gives You

Top-k is a hard cutoff: “I don’t care how unlikely these tokens are — if they’re not in the top K, they’re gone.” This prevents truly bizarre outputs. With k=3, “banana” can never be selected, even at high temperature.

But here’s the problem: K is fixed. When the model is very confident (one token dominates), k=40 keeps 39 irrelevant tokens alive. When the model is uncertain (many tokens are plausible), k=40 might cut off perfectly reasonable options. That’s the motivation for top-p.

Top-p (Nucleus) Sampling: Adaptive Filtering

Top-p solves top-k’s fixed-size problem with a simple insight: instead of keeping a fixed number of tokens, keep tokens until their cumulative probability reaches P.

Top-p (Nucleus) Sampling

  1. Sort tokens by probability (highest first)
  2. Walk down the list, accumulating probabilities
  3. Stop when the cumulative sum first exceeds P
  4. Drop all remaining tokens
  5. Renormalize and sample

Top-p with p=0.9

Start with our T=1.0 probabilities, sorted by probability:

cat0.1350.965✔ (first to exceed 0.9)
my0.030—✘ Dropped
banana0.004—✘ Dropped

Result: Same 3 tokens as top-k=3 in this case. But that’s a coincidence! Watch what happens with a different distribution.

The Adaptive Advantage

Top-p trusts the model’s confidence level. When the model is sure, top-p narrows the choices. When the model is uncertain, top-p widens them. Top-k always keeps exactly K tokens regardless of confidence. That’s why top-p has largely replaced top-k as the go-to strategy in production systems.

Repetition Penalties: Breaking the Loop

In Chapter 8, we saw greedy decoding get stuck in a repetitive loop. Even with sampling, models can repeat themselves. Two penalty mechanisms fight this:

Frequency Penalty

Subtract a value proportional to how many times the token has already appeared:

adjusted_logit = logit - frequency_penalty × count

Frequency Penalty in Action

Suppose the generated text so far is: “The cat sat on the mat. The cat...”

Token counts: the=3, cat=2, sat=1, on=1, mat=1

With frequency_penalty = 0.5:

TokenOriginal LogitPenaltyAdjusted Logit
the2.0-1.5 (0.5 × 3)0.5
cat1.8-1.0 (0.5 × 2)0.8
sat1.5-0.5 (0.5 × 1)1.0
ran1.20.0 (never used)1.2
played0.80.0 (never used)0.8

Result: “ran” now has the highest adjusted logit (1.2) even though “the” had the highest original logit (2.0). The model is now more likely to say “ran” than to repeat “the” for the fourth time.

Presence Penalty

A simpler version: subtract a fixed value if the token appeared at all, regardless of how many times.

adjusted_logit = logit - presence_penalty × (1 if token appeared, 0 if not)

The key difference: frequency penalty penalizes more for more repeats. Presence penalty is all-or-nothing. Use frequency penalty to discourage overuse of specific words. Use presence penalty to encourage the model to explore new topics.

Quick Check

If you set frequency_penalty = 2.0 and a token has appeared 5 times, how much is subtracted from its logit?

Answer: 2.0 × 5 = 10.0. That’s a massive penalty — the token is essentially eliminated. High frequency penalties can be too aggressive.

The Stereo Equalizer Analogy

Think of the probability distribution as a stereo equalizer with one slider per token. The height of each slider represents that token’s probability.

  • Temperature is the volume knob. Turn it down (T<1) and only the loudest frequencies are audible (top tokens dominate). Turn it up (T>1) and you hear the quiet frequencies too (rare tokens get a chance).
  • Top-k mutes all but the top K sliders. Everything below rank K is silenced.
  • Top-p mutes the sliders that contribute least to the total volume. If the top few are already loud enough (cumulative ≥ p), the quiet ones get cut.
  • Frequency penalty gradually turns down the slider each time that frequency plays. The more a note repeats, the quieter it gets.

Practical Settings: What to Use When

Every AI provider offers these parameters in their API. Here’s a guide based on what works in practice:

Use CaseTemperatureTop-pFrequency Pen.Why
Code generation 0.0–0.2 1.0 0.0 Code needs precision, not creativity. One correct answer.
Factual Q&A 0.0–0.3 0.9 0.0 Want the most likely (usually correct) answer.
Chat / conversation 0.7 0.9 0.3 Natural variety without being chaotic.
Creative writing 0.9–1.2 0.95 0.5 Surprise and variety are features, not bugs.
Brainstorming 1.0–1.5 0.95 0.8 Maximum diversity. Weird ideas welcome.
Tip: Notice that top-p is almost always 0.9–0.95 across all use cases. Temperature does most of the heavy lifting. Top-p is the safety net that catches the really implausible tail tokens. In most applications, you only need to tune temperature.

Min-p: The New Kid on the Block

There’s a newer strategy gaining popularity: min-p. The idea is elegantly simple.

Instead of keeping a fixed number of tokens (top-k) or a cumulative probability mass (top-p), min-p says: keep any token whose probability is at least P × the top token’s probability.

With min_p = 0.1 and a top token probability of 0.60, any token with probability ≥ 0.06 survives.

TokenProbThreshold (0.06)Kept?
the0.607—✔ (top token)
a0.2230.223 ≥ 0.06✔
cat0.1350.135 ≥ 0.06✔
my0.0300.030 < 0.06✘
banana0.0040.004 < 0.06✘

Like top-p, min-p adapts to the model’s confidence. When the model is sure, fewer tokens pass. When uncertain, more tokens pass. The advantage over top-p: it’s simpler to reason about and avoids the “always include the token that pushes cumulative over P” edge case.

Why the Same Model Can Write Code and Poetry

This is the key insight of this chapter. The model is always the same — same weights, same transformer blocks, same training. The “creativity” comes entirely from how you sample its outputs.

The model produces a probability distribution. The sampling strategy decides how to interpret that distribution. Low temperature says “trust the model’s top choice.” High temperature says “explore beyond the obvious.” It’s not that the model becomes creative — it’s that you’re allowing it to express uncertainty.

When you ask Claude to write code with T=0:

  • The model produces a distribution where def has 0.95 probability
  • T=0 picks def every time
  • Result: precise, deterministic, correct

When you ask Claude to write a poem with T=1.0:

  • The model produces a distribution where many words are plausible
  • T=1.0 lets any reasonably likely word be selected
  • Result: varied, surprising, creative

Same model. Different sampling strategy. Completely different output character.

Common Misconceptions

“Temperature 0 means no randomness.” — Almost true. Most APIs implement T=0 as greedy decoding (always pick the top token). But some implementations use a tiny T like 0.001, which can still occasionally vary. For true determinism, you also need to set a fixed random seed if the API supports it.
“Higher temperature = smarter.” — No! Higher temperature means more random. Sometimes the random choice is brilliant. Often it’s nonsense. The model’s quality comes from training, not from sampling settings.
“I should always set both top-k and top-p.” — Usually redundant. Top-p already adapts to the distribution shape. Adding top-k on top of top-p rarely changes anything unless you set k very low. Most production systems use only top-p.
