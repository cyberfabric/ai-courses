Security & Compliance With AI

Memory Map

The Big Idea

AI security is like giving your team Stack Overflow access — incredibly useful, but you still review what comes in before it goes to production. The difference: AI generates at 100x speed, so your defenses must scale accordingly.

🚨 The Five Security Risks

📤
1. Data Exfiltration
Your code sent to third-party servers. Free tiers may train on it. Defense: ZDR plans, SSO accounts, network controls.
🐛
2. Vulnerable Code
AI reproduces known vulnerabilities at scale. 2.7x higher vuln density. Defense: security rules in context files, SAST scanning.
💥
3. Prompt Injection
Malicious instructions hidden in PRs, issues, or docs manipulate AI behavior. 73% of systems are exposed. Defense: update tools, limit permissions.
🔑
4. Secrets in Context
AI ingests .env files, credentials, and keys from your project. Defense: .claudeignore / .cursorignore exclusion rules.
📦
5. Supply Chain Attacks
AI hallucinates package names, attackers register them. 4x increase since 2020. Defense: verify packages exist before installing.

🚨 Incident Response: 3 Severity Levels

  1. Level 1: Data Exposure (Low)

    Non-critical code sent to unapproved tool. Document, check retention policy, remind team. No blame.

  2. Level 2: Credential Exposure (Medium)

    Secrets in AI prompt. Rotate credentials immediately. Audit usage logs. Review ignore files.

  3. Level 3: Customer Data Exposure (High)

    PII sent to AI tool. Standard breach response. Notify DPO/legal. GDPR: 72-hour notification window.

📖 Key Terms

Zero-Data-Retention (ZDR)
Enterprise plan where the AI vendor processes but never stores your prompts or uses them for training.
Prompt Injection
Malicious instructions hidden in data (PRs, issues, docs) that manipulate AI behavior — the SQL injection of AI.
Supply Chain Attack
Attackers register hallucinated package names the AI suggests, planting malware for unsuspecting developers.
Content Exclusion
Configuring AI tools to ignore sensitive files (.env, keys, credentials) via .claudeignore or .cursorignore.
SAST/DAST
Static/Dynamic Application Security Testing. Automated scanners that catch vulnerabilities in CI/CD pipelines.
EU AI Act
EU regulation classifying AI systems by risk. High-risk enforcement begins August 2026 with major fines.
Business Associate Agreement (BAA)
Contract required under HIPAA when a vendor handles protected health information on your behalf.

🤔 Quick Quiz

What are the five AI security risks every tech lead should know?

(1) Data exfiltration (code sent to third parties), (2) vulnerable code generation (2.7x higher vuln density), (3) prompt injection (malicious instructions in data), (4) secrets in context (credentials ingested by AI), (5) supply chain attacks (hallucinated package names).

How much more vulnerable is AI-generated code compared to human-written code?

2.7x higher vulnerability density. 25.1% of AI code samples contain confirmed OWASP Top 10 vulnerabilities, and 41% include overly broad permission settings. The easiest fix: add security rules to your project context files (reduces vulns by 40–60%).

What is the single easiest step to reduce AI-generated security vulnerabilities?

Add security rules to your CLAUDE.md or .cursorrules. Rules like "always use parameterized queries" and "never hardcode secrets" reduce AI-generated vulnerabilities by 40–60%, because the AI follows the instructions you give it.

How do supply chain attacks exploit AI hallucinations?

AI suggests a package that doesn't exist (hallucinated name). Attackers register that exact name in the package registry with malicious code. Developers run the install command without checking. Always verify packages exist before installing.

When does the EU AI Act's high-risk enforcement begin, and what are the penalties?

August 2, 2026. Fines up to €35 million or 7% of global annual turnover, whichever is higher. If you serve EU customers, confirm your AI vendors have published compliance documentation.

📌 Key Takeaways

  • Five risks to manage: data exfiltration, vulnerable code, prompt injection, secrets in context, supply chain attacks.
  • AI-generated code has 2.7x higher vulnerability density. Add security rules to context files to cut this by 40–60%.
  • Prompt injection is real and growing (73% exposure, 340% YoY increase). Keep tools updated, limit agent permissions.
  • Configure .claudeignore / .cursorignore to exclude .env files, credentials, keys, and local databases from AI context.
  • Enterprise requirements: zero-data-retention, SSO, audit logs, SOC 2. Security enables AI adoption, it doesn't block it.
  • EU AI Act enforcement begins August 2, 2026 with fines up to €35M / 7% of turnover.
  • Build a "no blame" reporting culture. Fast incident reporting matters more than zero incidents.
