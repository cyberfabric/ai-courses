AI Cost Management & Budgeting

Memory Map
💡

The Big Idea

AI tool budgeting is like cloud cost management — easy to underestimate, easy to overspend, and someone needs to own the dashboard. Visible costs (seat licenses) are often less than half the TCO. Model tiering is your highest-leverage cost optimization.

💳 The Three Cost Models

🪑
Per-Seat Licensing
Predictable monthly fee per developer. Copilot $10–$39, Cursor $20–$40, Tabnine $12–$59. Easy to budget, adds up across teams.
🪙
Usage-Based (Tokens)
Pay per token processed. Variable, hard to predict. Power users report $500–$2,000/mo on API. 78% of IT leaders hit surprise charges.
🏢
Enterprise Agreements
Volume discounts (15–30%), committed annual spend, custom terms, zero data retention. Trigger: >$5K/mo with one vendor.

👻 Hidden Costs

💵 Reported: $3,200/mo (seat licenses only)
→
📊 Real: $11,400/mo (+ API overage + personal subs + training + productivity dip)

  1. 🎓 Training Time

    2–4 weeks of reduced productivity during onboarding per developer.

  2. 👤 Redundant Subscriptions

    Developers expensing personal AI accounts alongside team licenses.

  3. 📈 API Overages

    Usage-based costs that exceed budget. 78% of IT leaders report surprise charges.

  4. 🔧 Context Engineering

    Time writing CLAUDE.md, .cursorrules, prompt libraries. Invisible but real.

  5. 🔄 Rework Costs

    Fixing AI-generated code that doesn't meet standards.

📖 Key Terms

Token
The billing unit of AI APIs. ~0.75 words per token. Input tokens (your prompt) and output tokens (AI response) are priced separately.
Prompt Caching
Reusing previously sent context at 90% discount. The biggest cost saver for API-based tools.
Model Tiering
Using cheaper models (Haiku) for simple tasks and premium models (Opus) for complex ones. Highest-leverage cost optimization.
TCO (Total Cost of Ownership)
All costs: seat licenses + API tokens + training + rework + compliance. Often 2–3x the visible cost.
Headcount Avoidance
Features shipped without hiring additional developers. The ROI argument that CFOs understand.
Shadow AI
Developers using personal AI subscriptions for work. Creates redundant costs and security risks.
AI FinOps
The emerging discipline of managing AI tool costs — the same way FinOps manages cloud costs.

Quick Quiz

Why do output tokens cost 5x more than input tokens?

Generating text requires sequential computation (each token depends on the previous one), while reading input can be parallelized. Generation is computationally harder.

What discount does prompt caching give on repeated context?

90% — cached input tokens cost just 0.1x the base price. First write costs 1.25x, but every subsequent cache hit is 90% off.

When should you negotiate an enterprise agreement?

When you're spending more than $5,000/month with a single vendor. At $60K+ annual, most vendors offer meaningful discounts.

What's the "headcount avoidance" argument?

Instead of "we save 4 hours/dev/week," frame it as: "We shipped 3 features without hiring, saving $150K salary + $50K recruiting." It's the language finance speaks.

Why shouldn't you commit more than 50% of AI budget to one vendor?

The AI tool landscape changes too fast. A 3-year lock-in sounds great until 18 months later when a better tool exists and your contract won't let you switch.

Key Takeaways

  • Three cost models: per-seat (predictable), usage-based tokens (variable), and enterprise agreements (discounted).
  • Model tiering is the #1 cost lever — Haiku is 5x cheaper than Opus. Match the model to the task.
  • Prompt caching saves 90% on repeated context. Batch processing saves 50%. Stack them.
  • Hidden costs (training, rework, shadow AI) often double the visible tool spend.
  • The winning business case: headcount avoidance, not hours saved. "We shipped without hiring."
  • Never commit >50% of budget to one vendor. Negotiate at end of quarter for best terms.
  • AI budgeting = cloud cost management — build visibility, set guardrails, assign ownership.
