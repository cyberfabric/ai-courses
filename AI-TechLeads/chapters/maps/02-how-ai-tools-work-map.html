The Big Idea

LLMs are brilliant pattern matchers with zero judgment. They predict the most likely next token — they don't reason, verify, or understand your business. Better context in = better code out. That's the lever you control.

📂 The Context Window

🔢
1M–2M Tokens
Current top-tier context windows. ~750K words. Sounds huge, but a medium codebase can exceed it.
💸
More Tokens = More Cost
Don't dump your entire codebase. Focused context produces better results AND costs less.
👁
Attention Dilution
More irrelevant context = worse performance. Quality beats quantity every time.
🎯
Curated > Comprehensive
A well-crafted 10K-token prompt often beats a 500K-token dump.

🚀 The Three Levels of Context Engineering

  1. Level 1: Prompt Context

    What the developer types each time. Starts from zero every request. Where most teams are stuck.

  2. Level 2: Project Context

    Shared rules files (CLAUDE.md, .cursorrules). Tech stack, conventions, naming standards. Every request gets this automatically. This is the biggest ROI investment.

  3. Level 3: Tool Context (MCP)

    Live data via Model Context Protocol. 50+ official servers, 150+ community. AI queries real DB schemas, live API docs, deployment state.

📖 Key Terms

Token
~3/4 of a word or ~3–4 characters of code. The unit AI models think in and charge for.
Context Window
The model's working memory. Everything (prompt + code + history) must fit inside it.
Hallucination
AI confidently generating something wrong: fake APIs, wrong method signatures, deprecated patterns.
Temperature
Controls output randomness. Low = predictable code. High = creative brainstorming.
Knowledge Cutoff
The date when training data stops. The model doesn't know anything that happened after this date.
Context Engineering
Systematically providing AI with the right project info (rules files, conventions, architecture) to improve output.
MCP (Model Context Protocol)
Industry standard for connecting AI agents to live tools and data. The bridge between static models and dynamic projects.
Model Tiering
Using cheap models for simple tasks, expensive models for hard ones. The smart way to control AI costs.

🤔 Quick Quiz

Why do LLMs hallucinate?

Because they predict the most statistically likely next token, not the correct one. They have no fact-checking mechanism. Code that looks plausible may call APIs that don't exist or use deprecated patterns.

Why doesn't "paste the whole codebase" work even with 1M-token windows?

Three reasons: attention dilution (relevant info gets buried), cost (more tokens = more money), and quality (focused context beats comprehensive dumps).

What's the "brilliant intern" mental model?

AI is like an intern who's read millions of codebases but never shipped production code. Incredibly productive with clear instructions and code review. You wouldn't let them merge without review — same applies to AI.

What are the three levels of context engineering?

Level 1: Prompt context (per-request). Level 2: Project context (rules files, shared conventions). Level 3: Tool context (MCP servers, live data). Most teams are stuck at Level 1.

How can model tiering cut costs by 50–70%?

Use cheap models (Haiku, Flash) for simple completions, tests, and docs. Reserve expensive models (Opus) for complex architecture, security-sensitive code, and hard debugging.

📌 Key Takeaways

  • LLMs predict tokens, they don't reason. Pattern matchers, not thinkers. Always verify output.
  • Hallucinations are presented with 100% confidence. Train your team: confidence ≠ correctness.
  • Context quality trumps context quantity. A curated 10K-token prompt beats a 500K-token dump.
  • Context engineering is the #1 lever for improving AI output. Invest in rules files as a team asset.
  • Use the "brilliant intern" mental model: clear instructions, always review, build trust gradually.
  • Model tiering (cheap for simple, expensive for hard) cuts costs dramatically with minimal quality loss.
  • AI excels at patterns (boilerplate, tests, docs) and struggles with judgment (architecture, security, business logic).
