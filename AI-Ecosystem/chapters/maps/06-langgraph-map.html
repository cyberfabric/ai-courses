Chapter 6

LangGraph: Smarter Workflows

Memory Map
💡

The Big Idea

Chains are recipes (step 1, step 2, done). Graphs are flowcharts (decisions, loops, go back). LangGraph lets your AI workflow try something, check if it worked, and retry if it didn't. When chains hit a wall because they can't go backwards, LangGraph is the answer.

🧱 The 3 Building Blocks

📋
State
A shared Python object (TypedDict) holding all working data. Every node reads and updates it. The "working memory" of your graph.
⚙️
Nodes
Python functions that do work. Each takes the state, does one job, returns updates. Like individual UseCase classes in Android.
↗️
Edges
Normal: always A → B. Conditional: look at state and decide where to go next. Conditional edges enable loops and branching.

🔄 Patterns That Need Cycles

Pattern What Happens Example
Self-correction LLM checks output, retries if wrong Code generator that runs tests & fixes errors
Iterative refinement Each pass improves the output Research assistant that deepens its search
Tool retry Failed tool? Try a different approach Search with different keywords on failure
Multi-step planning Plan → execute → evaluate → replan Trip planner adjusting for availability
Validation loops Output must meet criteria before moving on JSON validated against a schema

The self-correcting agent is the #1 use case. It alone justifies learning LangGraph.

🤖 Agent Architectures

🔁 ReAct (Reason + Act)
Think → Act (call tool) → Observe result → Think again → repeat until answer. The most common agent loop.
📋 Plan-and-Execute
Create a task list first, then execute tasks one by one. Revise the plan after each step. Good for complex, multi-step problems.
🔧 Tool-Calling Agent
Simplest pattern: LLM decides which tool to call. Modern LLMs (GPT-4, Claude, Llama 3.1) support this natively.

⚡ Power Features

👤
Human-in-the-Loop
Pause the graph at any node, wait for human approval, then resume. Critical for actions like sending emails or making purchases.
💾
Checkpointing
Save state at every step. Resume after crashes, replay for debugging, pause for days then continue. Like git for your workflow.
⏪
Time Travel
Replay the graph from any previous state. Fork from a checkpoint to try different approaches. Invaluable for debugging.
🛡️
Fault Tolerance
Process crashes? Resume from last checkpoint. Long-running workflows survive restarts. Store in SQLite or Postgres for production.

⚖️ LangGraph vs LangChain

Scenario LangChain LangGraph
Simple RAG pipeline ✅ retrieve → prompt → answer ❌ Overkill
Chatbot with memory ✅ chain + memory Only if complex routing needed
Agent with tools Simple cases only ✅ Tool loops, error recovery
Self-correcting workflow ❌ No loops possible ✅ Sweet spot
Human approval steps ❌ Not built-in ✅ interrupt/resume is native
Complex branching ❌ Awkward ✅ Conditional edges

The rule: Straight line → LangChain. Loops/decisions/checkpoints → LangGraph. Most apps start as chains and graduate to graphs.

🧪 Quick Quiz

What's the fundamental limitation of chains that LangGraph solves?

Chains can't go backwards. They flow A → B → C, done. LangGraph adds cycles (loops), conditional branching, and state — so the workflow can retry, refine, and make decisions.

What are the 3 building blocks of a LangGraph workflow?

State (shared working memory), Nodes (Python functions that do work), and Edges (connections between nodes — normal or conditional).

What's the ReAct pattern?

Reason + Act. The agent thinks about what to do, takes an action (calls a tool), observes the result, and repeats until it has an answer. It's a loop: think → act → observe → think → ...

When is LangGraph overkill?

When your workflow is a straight line with no loops or decisions. Simple RAG, classification, summarization, translation — all fine with a chain or a single LLM call.

What does human-in-the-loop mean in LangGraph?

The graph pauses at a specified node and waits for human approval before continuing. Use interrupt_before to pause. Resume with the same thread ID. Critical before irreversible actions (sending emails, making purchases).

📌 Key Takeaways

  • LangGraph = flowcharts for AI. Chains go forward only; graphs can loop, branch, and make decisions.
  • 3 building blocks: State (working memory), Nodes (worker functions), Edges (connections — normal or conditional).
  • Conditional edges are the magic — they look at state and decide where to go next, enabling loops and branching.
  • Self-correcting agents are the #1 use case: try → check → retry if needed.
  • 3 agent architectures: ReAct (think/act loop), Plan-and-Execute, Tool-Calling.
  • Human-in-the-loop: pause for approval before irreversible actions. First-class feature, not a hack.
  • Checkpointing: save state at every step. Resume after crashes. Replay for debugging. Like git for workflows.
  • The rule: straight line → chain. Loops/decisions/checkpoints → LangGraph. Don't use a graph when a straight line works.
