Chapter 3

Ollama: Running AI on Your Machine

Memory Map
💡

The Big Idea

Ollama is Docker for AI models. One command to download, run, and chat with any model locally. No cloud, no API keys, no cost per request. Prototype locally, deploy to cloud.

🛡️ Five Reasons to Run Models Locally

🔒
Privacy
Data never leaves your machine. Critical for company code, customer data, legal docs.
💰
Cost
$0 per request. Iterate on prompts hundreds of times without billing surprises.
⚡
Speed
No network latency. Short requests can be faster than cloud round-trips.
✈️
Offline
Works without internet. Airplane, train, bad WiFi — doesn't matter.
♾️
No Rate Limits
Hammer it as hard as your hardware allows. Batch processing, stress testing — no throttling.

🗃️ Top Models to Know

llama3.1 (8B) — 🏆 Default choice
Best balance of quality, speed, and resource usage. ~5 GB RAM. Start here.
phi3 (3.8B) — 🪶 Lightweight champion
Shockingly good for its tiny size. ~2.5 GB RAM. Great for low-resource machines.
mistral (7B) — ⚖️ Strong all-rounder
Apache 2.0 license. ~4.5 GB RAM. Competitive quality.
codellama (7B/13B) — 💻 Code specialist
Optimized for code generation and understanding.
qwen2.5 (0.5B-72B) — 📏 Every size
Wide range from tiny to massive. Strong benchmarks across the board.

⚠️ Honest Limitations

📉
Quality Gap
Local 8B is noticeably worse than GPT-4/Claude for complex reasoning. Fine for simple tasks.
📏
Context Window
Local: 4K-8K tokens. Cloud: 128K-200K+. Long documents are a problem.
🖼️
Multimodal
Local vision/audio models exist but are less mature than cloud offerings.
👤
Single User
Your laptop serves one conversation well. Multiple users need a server or cloud.

🧪 Quick Quiz

What's the first Ollama command you should try?

ollama run llama3.1 — downloads and starts Meta's 8B model. If low on RAM, try ollama run phi3.

Why is Ollama's OpenAI-compatible API such a big deal?

Code written for OpenAI works with Ollama by changing one URL. Develop free locally, deploy to cloud for production — same codebase.

Should you judge an AI technique (like RAG) based only on a local model?

No! Local 7B models are weaker. If results are mediocre, the model might be the bottleneck, not the technique. Always validate with a cloud model.

What's Open WebUI?

A ChatGPT-like web interface that connects to Ollama. Private, free, great for non-technical teammates. Runs locally via Docker.

How does an Android app talk to Ollama during development?

Call http://10.0.2.2:11434 (emulator's localhost alias). Same REST API as OpenAI. Switch to cloud URL for production.

📌 Key Takeaways

  • Ollama = Docker for AI. One command: ollama run llama3.1. That's it.
  • Run locally for privacy, cost, speed, offline access, and no rate limits.
  • The API on localhost:11434 is OpenAI-compatible — swap one URL to switch local/cloud.
  • Match model to hardware: 7-8B for 16 GB RAM, 1-3B for 8 GB, 13B+ needs serious GPU.
  • GPU gives 5-10x speed over CPU. Apple Silicon is the sweet spot.
  • Best workflow: prototype locally, deploy to cloud.
  • Local models have real limitations: smaller context, lower quality, single user. Don't judge AI techniques by local-only results.
  • Open WebUI = private ChatGPT for your team, no cost, no data leaving your network.
