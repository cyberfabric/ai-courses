Chapter 10

n8n: AI Workflow Automation

Drag-and-drop AI workflows — connect LLMs to your business processes without writing code

The One-Sentence Version

n8n is an open-source workflow automation platform that lets you build AI-powered automations by connecting visual nodes — think Zapier, but self-hosted, free, and with native AI capabilities.

For nine chapters, we've been writing Python. LangChain pipelines, Gradio interfaces, fine-tuning scripts. But here's the thing: a huge number of AI use cases don't need custom code at all. They need automation — "when this happens, do this AI thing, then put the result there."

n8n lets you build these workflows visually. Drag a trigger node, connect it to an AI node, connect that to an output node. Done. No Python, no deployment, no framework debates.

As a developer, you might think visual tools are "beneath you." They're not. n8n is about leverage — automating the repetitive stuff so you can focus on the hard stuff. Classifying incoming support tickets? Summarizing meeting notes? Extracting data from emails? These are real tasks that eat hours every week, and n8n can automate them in an afternoon. Plus, it's self-hosted — your data never leaves your server.

Why n8n for AI?

There are plenty of automation platforms — Zapier, Make (formerly Integromat), Power Automate. What makes n8n stand out for AI workflows?

It's Open Source and Self-Hosted

n8n runs on your server. Your data never touches a third-party cloud. For AI workflows that process sensitive information — customer data, internal documents, financial records — this is a major advantage. You can connect n8n to your local Ollama instance (Chapter 3) and build an entirely private AI automation stack.

Native AI Nodes

n8n doesn't just "support" AI — it has purpose-built nodes for common AI tasks. These aren't generic HTTP request nodes; they're designed specifically for AI workflows with the right inputs, outputs, and configuration options.

It's Free (or Cheap)

The self-hosted version is free and unlimited. The cloud version has a free tier and paid plans. Compare that to Zapier, where AI features are premium-only and automation limits kick in fast.

Core Concepts

If you've never used a workflow automation tool, here are the basics:

  • Trigger: what starts the workflow (a new email, a webhook call, a schedule, a Slack message)
  • Nodes: the steps in your workflow (each node does one thing: transform data, call an API, run AI)
  • Connections: arrows between nodes that define the data flow
  • Executions: each time the workflow runs, it's one execution — you can see logs, inputs, and outputs for debugging

If you've used GitHub Actions or CI/CD pipelines, you already understand the concept. n8n is similar, but with a visual canvas instead of YAML.

The AI-Specific Nodes

This is where n8n gets interesting for our purposes. Here are the AI nodes you'll use most:

AI
AI Agent Node
A full agent with tools — it can make decisions, call tools (search, calculate, query databases), and iterate. This is LangGraph-level capability in a visual node.
LM
Chat Model Node
Connect to any LLM — OpenAI, Anthropic, Ollama, HuggingFace, Mistral. Configure the model, temperature, and system prompt visually.
VS
Vector Store Node
Connect to Pinecone, Chroma, Qdrant, or Supabase vectors. Build RAG workflows without writing a single line of retrieval code.
TC
Text Classifier Node
Categorize incoming text (support tickets, feedback, emails) into predefined categories using an LLM.
SA
Sentiment Analysis Node
Analyze the tone of text — positive, negative, neutral. Useful for customer feedback, reviews, and social media monitoring.
SE
Information Extractor Node
Extract structured data from unstructured text — names, dates, amounts, addresses. Like output parsing from Chapter 5, but visual.

Example Workflow 1: Smart Email Processor

Let's walk through a real workflow that would take hours to code but minutes to build in n8n:

Email Arrives → Classify (AI) → Extract Data (AI) → Update Database → Send Response

What it does: When a customer email arrives, AI classifies it (complaint, question, feature request, billing). Then AI extracts key details (customer name, order number, issue description). The data goes into your database. A templated response is drafted and sent (or routed to a human for review).

Nodes used: Gmail Trigger → Text Classifier → Information Extractor → Postgres/Airtable → Gmail Send (or Slack notification for human review)

Building this in code would mean: email API integration, prompt engineering, JSON parsing, database queries, and email sending. In n8n, each step is a visual node with configuration fields. The whole workflow takes 30-60 minutes to build.

Example Workflow 2: AI-Assisted Slack Bot

Slack Message → RAG Retrieval → AI Generates Response → Human Approves → Reply in Slack

What it does: Someone asks a question in your team's Slack channel. n8n searches your knowledge base (vector store with your docs), the AI generates an answer based on retrieved context, a human approves the response, and it's posted back to Slack.

This is a complete RAG chatbot (Chapter 4) deployed as a Slack integration — without writing Python. The same workflow that took us 10+ lines of LangChain code is now a visual pipeline.

The human approval step is key. For internal tools, AI can respond automatically. For customer-facing or sensitive contexts, always add a human checkpoint. n8n makes this trivial — just add a "Wait for Approval" node.

Example Workflow 3: Content Pipeline

RSS Feed → Summarize (AI) → Translate (AI) → Post to CMS → Notify Team

What it does: Monitors RSS feeds for industry news. AI summarizes each article into 2-3 sentences. Optionally translates to other languages. Posts summaries to your CMS or internal wiki. Sends a daily digest to your team on Slack.

This runs on a schedule (every hour, every day) with zero human intervention. It's the kind of automation that saves a content team hours per week.

My Recommendation

For AI workflows, n8n is the clear winner. Its native AI nodes, Ollama integration, and self-hosted privacy make it purpose-built for what we've been discussing in this guide. Zapier is fine for simple non-AI automations. Make is a solid middle ground. But if AI is the core of your workflow, n8n is the right choice.

The Privacy Stack: n8n + Ollama

This combination deserves special attention. When you connect n8n to your local Ollama instance:

  • Your workflows run on your server
  • Your LLM runs on your server
  • Your data never leaves your network
  • There are zero API costs per request

This is the fully private AI automation stack. It's perfect for companies in regulated industries (healthcare, finance, legal) or any organization that takes data privacy seriously. The AI quality won't match GPT-4, but for classification, extraction, and summarization tasks, local models are often good enough.

When to Code Instead

n8n is powerful, but it's not always the right tool. Here's when to write code instead:

  • Complex AI logic: if your AI pipeline has complex branching, self-correction loops, or multi-agent collaboration, code (LangGraph, Chapter 6) gives you more control.
  • Custom UI: if users need to interact with the AI feature through a custom interface, build it properly (or at least use Gradio, Chapter 8).
  • High performance: if you need sub-second response times or high throughput, a custom backend will be faster than a workflow engine.
  • Deep integration: if the AI feature is tightly integrated into your app's core logic, it belongs in your codebase, not in an external workflow tool.
The biggest risk with n8n (and any no-code tool): building workflows that become critical infrastructure but are hard to version-control, test, and maintain. Treat important n8n workflows like code — document them, back them up, and have a plan for when they break.

Getting Started

n8n is surprisingly easy to try. You don't need to commit to a server or sign up for anything:

# Option 1: Run locally with npx (requires Node.js)
npx n8n

# Option 2: Run with Docker
docker run -it --rm -p 5678:5678 n8nio/n8n

# Then open http://localhost:5678 in your browser

That's it. You get a visual canvas where you can start building workflows immediately. The AI nodes require you to configure an LLM connection (paste an OpenAI API key or point to your local Ollama at http://localhost:11434), but the rest works out of the box.

For a quick test: create a workflow with a Manual Trigger → Chat Model (Ollama or OpenAI) → output. Click "Test Workflow" and watch your first AI automation run. The visual feedback — seeing data flow through each node — makes debugging intuitive.

Templates: Don't Start From Scratch

n8n has a library of hundreds of pre-built workflow templates, including many AI-specific ones. Before building anything, search the template library. Common templates include:

  • AI chatbot with memory and RAG
  • Email classification and routing
  • Document summarization pipeline
  • Social media monitoring with sentiment analysis
  • Automated report generation
  • Customer feedback analysis

Templates save hours. Clone one, modify it for your use case, and you're running in minutes instead of building from scratch.

Connection to Android Development

n8n connects to your Android world in several practical ways:

  • Backend automation: Use n8n to automate tasks your app triggers — when a user submits feedback, n8n classifies it, creates a ticket, and notifies the right team. Your Android app just calls a webhook.
  • Push notification intelligence: n8n monitors data sources and decides when to send push notifications to users. AI determines which content is relevant to which user segments.
  • Content pipeline: n8n generates, curates, or translates content for your app. Your Android app fetches the results from your CMS or API — it doesn't know n8n exists.
  • Support automation: User sends a message through your app → n8n receives it via webhook → AI classifies and drafts a response → human reviews → response sent back to the app.
  • You need to automate repetitive tasks that involve AI (classification, extraction, summarization)
  • You want AI-powered workflows without writing Python
  • Data privacy matters — you need everything self-hosted
  • You're connecting AI to business tools (email, Slack, databases, CRMs)
  • You want to prototype AI workflows quickly before coding them
  • Non-technical team members need to build or modify AI automations
  • Your AI feature needs complex logic that's hard to express visually
  • You need a custom user interface (use Gradio or a real frontend)
  • Performance is critical — workflow engines add overhead
  • The AI feature is core to your app's logic and belongs in your codebase
  • You're already comfortable with Python and prefer code over visual tools
Design an n8n Workflow

Think about a repetitive task in your current job that could benefit from AI. It might be:

  • Classifying incoming bug reports or feature requests
  • Summarizing daily standup notes or meeting recordings
  • Monitoring competitor updates and generating brief summaries
  • Extracting structured data from unstructured emails or documents
  • Generating weekly status reports from project management data

Now sketch the workflow:

  1. What triggers it? (Email? Schedule? Webhook from your app?)
  2. What AI step is needed? (Classification? Extraction? Generation?)
  3. Where does the output go? (Database? Slack? Email? Your app's API?)
  4. Does it need a human approval step?
  5. Would you use cloud AI or local Ollama? (Consider privacy and quality.)

Bonus: install n8n (npx n8n or Docker) and try building it. The visual builder is surprisingly intuitive.
