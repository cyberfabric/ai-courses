Chapter 4

Your First Agent

Build a complete, working research agent from scratch — no frameworks, just code

The Plan

In Chapters 1–3 you learned what agents are, how they’re architected, and how to design tools. Now we build. By the end of this chapter, you’ll have a working research agent that can search the web, read pages, and synthesize findings — running on your machine in about 100 lines of Python.

No frameworks. No magic. Just the Anthropic SDK, a few tool functions, and the while True loop you saw in Chapter 1. Once you understand what’s happening at this level, every framework you encounter later will feel transparent instead of magical.

1. Setup → 2. Define Tools → 3. System Prompt → 4. Agent Loop → 5. Run It

Step 2: Define Your Tools

Our research agent needs three tools. Following the design rules from Chapter 3: verb names, rich descriptions, typed parameters.

import anthropic
import httpx
from bs4 import BeautifulSoup
import json

client = anthropic.Anthropic()

# --- Tool Definitions (what the LLM sees) ---

tools = [
    {
        "name": "search_web",
        "description": (

            "each with a title, URL, and snippet. Use specific queries "
            "for better results. Example: 'Python asyncio best practices 2026' "
            "instead of just 'Python'."
        ),
        "input_schema": {
            "type": "object",
            "properties": {
                "query": {
                    "type": "string",
                    "description": "The search query. Be specific and detailed."
                }
            },
            "required": ["query"]
        }
    },
    {
        "name": "read_page",
        "description": (
            "Fetch and read the text content of a web page. Returns the "
            "main text content, stripped of HTML tags. Use this after "
            "search_web to read full articles. Returns first 5000 chars "
            "to stay within limits."
        ),
        "input_schema": {
            "type": "object",
            "properties": {
                "url": {
                    "type": "string",
                    "description": "The full URL to read, e.g. 'https://example.com/article'"
                }
            },

That’s what the LLM sees. Now let’s write the actual implementations — the code that runs when the agent calls each tool.

Step 3: Implement the Tools

# --- Tool Implementations (what actually runs) ---

def search_web(query: str) -> str:
    """Search DuckDuckGo and return results as JSON."""
    try:
        url = "https://html.duckduckgo.com/html/"
        resp = httpx.post(url, data={"q": query}, timeout=10,
                          headers={"User-Agent": "ResearchAgent/1.0"})
        soup = BeautifulSoup(resp.text, "html.parser")
        results = []
        for r in soup.select(".result")[:5]:
            title_tag = r.select_one(".result__title a")
            snippet_tag = r.select_one(".result__snippet")
            if title_tag:
                results.append({
                    "title": title_tag.get_text(strip=True),
                    "url": title_tag.get("href", ""),
                    "snippet": snippet_tag.get_text(strip=True) if snippet_tag else ""
                })
        if not results:
            return json.dumps({"status": "no_results",
                               "suggestion": "Try different or broader search terms."})
        return json.dumps({"status": "success", "results": results})
    except Exception as e:
        return json.dumps({"status": "error", "message": str(e),
                           "suggestion": "Search failed. Try a different query."})

def read_page(url: str) -> str:
    """Fetch a web page and return its text content."""
    try:
        resp = httpx.get(url, timeout=15, follow_redirects=True,
                         headers={"User-Agent": "ResearchAgent/1.0"})
        soup = BeautifulSoup(resp.text, "html.parser")
        # Remove script and style elements
        for tag in soup(["script", "style", "nav", "footer", "header"]):
            tag.decompose()
        text = soup.get_text(separator="\n", strip=True)
        # Truncate to stay within context limits
        if len(text) > 5000:
            text = text[:5000] + "\n\n[...truncated — page too long]"
        return json.dumps({"status": "success", "content": text, "url": url})
    except Exception as e:
        return json.dumps({"status": "error", "message": str(e),
                           "suggestion": "Could not read page. Try a different URL."})

def save_report(filename: str, content: str) -> str:
    """Save content to a file."""
    try:
        with open(filename, "w") as f:
            f.write(content)
        return json.dumps({"status": "success",
                           "message": f"Report saved to {filename}"})
    except Exception as e:
        return json.dumps({"status": "error", "message": str(e)})

# Map tool names to functions
tool_functions = {
    "search_web": lambda inputs: search_web(inputs["query"]),
    "read_page": lambda inputs: read_page(inputs["url"]),
    "save_report": lambda inputs: save_report(inputs["filename"], inputs["content"]),
}

Notice how every tool returns structured JSON with a status field, exactly as we discussed in Chapter 3. Errors include suggestions so the agent can recover.

Step 4: The System Prompt

The system prompt turns a generic LLM into a research agent. This is where you encode the ReAct architecture from Chapter 2:

SYSTEM_PROMPT = """You are a research agent. Your job is to answer questions
thoroughly by searching the web and reading relevant sources.

## Process
1. Think about what information you need to answer the question.
2. Search the web with specific, targeted queries.
3. Read the most promising pages for detailed information.
4. If needed, search again with refined queries based on what you learned.
5. Synthesize your findings into a clear, well-sourced answer.

## Rules
- Always cite your sources with URLs.
- If sources disagree, note the conflicting information.
- If you cannot find reliable information, say so honestly.
- Be efficient — aim for 2-4 searches maximum.
- After gathering enough info, save a report using save_report.

## Quality Standards
- Prefer recent sources (2025-2026) over older ones.
- Cross-reference claims across multiple sources when possible.
- Distinguish between facts and opinions in your report."""

The System Prompt Blueprint

Every agent system prompt should have three sections:

  1. Identity — who the agent is and what it does (“You are a research agent”)
  2. Process — the step-by-step approach (your architecture, in plain English)
  3. Rules — constraints, quality standards, and guardrails

Why it works: the identity anchors the model’s behavior. The process gives it a strategy instead of letting it wing it. The rules prevent common failure modes. Skip any of these and the agent becomes unpredictable.

Step 5: The Agent Loop

This is the heart of every agent. It’s the while True loop from Chapter 1, but now with real tools, logging, and a step limit:

def run_agent(question: str, max_steps: int = 15) -> str:
    """Run the research agent on a question."""
    print(f"\n{'='*60}")
    print(f"Research question: {question}")
    print(f"{'='*60}\n")

    messages = [{"role": "user", "content": question}]

    for step in range(1, max_steps + 1):
        # Call the LLM
        response = client.messages.create(
            model="claude-sonnet-4-6-20250514",
            max_tokens=4096,
            system=SYSTEM_PROMPT,
            tools=tools,
            messages=messages,
        )

        # --- Handle tool use ---
        if response.stop_reason == "tool_use":
            # The response may contain text (thinking) + tool calls
            for block in response.content:
                if hasattr(block, "text"):
                    print(f"  [{step}] Think: {block.text[:150]}...")

            # Execute each tool call
            tool_results = []
            for block in response.content:
                if block.type == "tool_use":
                    print(f"  [{step}] Tool:  {block.name}({json.dumps(block.input)[:100]})")
                    result = tool_functions[block.name](block.input)
                    print(f"  [{step}] Result: {result[:100]}...")
                    tool_results.append({
                        "type": "tool_result",
                        "tool_use_id": block.id,
                        "content": result,
                    })

            # Feed results back into the conversation
            messages.append({"role": "assistant", "content": response.content})
            messages.append({"role": "user", "content": tool_results})

        # --- Handle final response ---
        else:
            final_text = ""
            for block in response.content:
                if hasattr(block, "text"):
                    final_text += block.text
            print(f"\n  [Done in {step} steps]")
            print(f"\n{'='*60}")
            print(final_text)
            return final_text

    print(f"\n  [Hit step limit of {max_steps}]")
    return "Agent reached maximum steps without completing."

The Agent Loop, Decoded

Every iteration of the loop follows the same three-phase pattern:

  1. Call the LLM — send the conversation history (including previous tool results) to the model
  2. Check the response — did the model produce a final answer (stop_reason == "end_turn") or a tool call (stop_reason == "tool_use")?
  3. Execute and feed back — if it’s a tool call, run the tool, add the result to the conversation, and loop

The conversation messages list is the agent’s memory. Every tool call and result gets appended, so the model sees the full history of what it’s done. This is the “short-term memory” we’ll expand on in Chapter 5.

Step 6: Run It!

# --- Main entry point ---

if __name__ == "__main__":
    question = input("What would you like me to research? > ")
    run_agent(question)

Debugging the Loop

When your agent misbehaves (and it will), the logs are your best friend. Here’s what to look for:

  • Agent calls the wrong tool → your tool descriptions are ambiguous. Clarify them.
  • Agent passes wrong parameters → add examples to parameter descriptions, use enums.
  • Agent loops without progress → it’s stuck. Add a step limit (we did) and check if tool results are useful.
  • Agent ignores tool results → the result might be too long. Truncate large outputs.
The context window trap. Every tool call and result gets appended to the messages list. A web page might return 5,000 characters. After 10 pages, that’s 50,000 characters of context. The model’s accuracy drops after ~32K tokens, and it suffers from the “lost in the middle” effect — information in the middle of the context gets ignored. Always truncate tool outputs to a reasonable size.

Handling Edge Cases

Infinite Loops

We added max_steps=15, but you can also detect when the agent is repeating itself:

def detect_loop(messages, window=4):
    """Check if the agent is repeating the same tool calls."""
    recent_tools = []
    for msg in messages[-window * 2:]:
        if isinstance(msg.get("content"), list):
            for block in msg["content"]:
                if hasattr(block, "name"):
                    recent_tools.append(f"{block.name}:{block.input}")
    # If we see the same call twice in the last window, we're stuck
    return len(recent_tools) != len(set(recent_tools))

Making It Interactive

Want to approve actions before the agent executes them? Add a confirmation gate:

# Inside the agent loop, before executing a tool:
for block in response.content:
    if block.type == "tool_use":
        if block.name == "save_report":  # Confirm writes
            print(f"\n  Agent wants to save: {block.input['filename']}")
            confirm = input("  Allow? (y/n) > ")
            if confirm.lower() != "y":
                tool_results.append({
                    "type": "tool_result",
                    "tool_use_id": block.id,
                    "content": '{"status": "denied", "message": "User declined."}',
                    "is_error": True,
                })
                continue
        # ... execute the tool normally

What You Just Built vs. What Frameworks Do

Your ~100-line agent does the same core thing that every agent framework does. The frameworks add:

Wiring Automatic tool registration, schema generation from types, result handling
Features Memory, streaming, retries, tracing, multi-agent support, guardrails
Ecosystem Pre-built tools, MCP integrations, deployment helpers, community

But the core is the same loop. Now when you use LangGraph, CrewAI, or Claude Agent SDK (Chapter 10), you’ll understand what they’re doing under the hood. You won’t be confused by abstractions because you’ve built the primitive yourself.

The Full Working Agent

All the code in this chapter fits in a single agent.py file of about 120 lines. The pieces: tool definitions (what the LLM sees), tool implementations (what actually runs), system prompt (the agent’s personality and strategy), and the loop (the engine that ties it all together). That’s it. That’s the whole agent.

Cost awareness. Each iteration of the loop is an API call. With Claude Sonnet, a typical research session of 5–8 steps costs roughly $0.02–0.05. That’s fine for development. But an agent stuck in a loop running 100 iterations can cost $1+ per question. Always set max_steps and monitor your usage at console.anthropic.com.
