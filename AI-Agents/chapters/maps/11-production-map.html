Production Deployment

The Big Idea

The production gap isn’t about model quality — it’s about engineering discipline. 88% of agent projects fail before reaching production, not because the agent doesn’t work, but because of cost overruns, security incidents, and missing observability.

🗂️ Core Production Concerns

💸
Cost Management
Route tasks to the right model tier (Opus/Sonnet/Haiku). Use prompt caching for 90% savings. Set token budgets at request, task, and monthly levels.
⚡
Latency Optimization
Run tool calls in parallel with asyncio. Stream responses for perceived speed. Target sub-10-second user-facing responses.
🔒
Security
Layered defense: input validation, privilege minimization, tool access control, output filtering, data sanitization, key isolation, behavioral monitoring.
📈
Scaling
Match scale to infra: serverless (<100 req/hr), task queues (100–1K), containers (>1K). Agent tasks are long-running — queues often win.
👁️
Observability
Track five metrics: success rate, cost per task, P50/P95 latency, error patterns, and token usage. You can’t fix what you can’t see.
🚦
Rate Limiting
Use a token bucket rate limiter shared across all agent instances. Prevents blowing through API quotas in multi-agent systems.

🚨 Production Incident Response

  1. 🔔 Alert Fires

    Success rate dropped from 94% to 71%. Cost/task spiked 4.5x. Automated monitoring catches it.

  2. 🔍 Investigate Traces

    Check agent execution traces. Agents are retrying search_web 3–4 times per task. The search API returns 429.

  3. 🎯 Root Cause

    A new feature doubled request volume. Search API rate limit is being hit consistently.

  4. 🛠️ Fix

    Deploy search result cache (TTL: 15 min). Route through cache before hitting API.

  5. ✅ Verify Recovery

    Success rate back to 93% within 10 minutes. Cost/task normalized. Cache hit rate: 62%.

📖 Key Terms

Model Routing
Directing tasks to different model tiers (Opus/Sonnet/Haiku) based on complexity — the biggest cost lever.
Prompt Caching
Reusing cached system prompts across requests for a 90% discount on input tokens. Cache persists ~5 minutes.
Token Budget
Hard limits on token usage at three levels: per-request, per-task, and per-month. Prevents runaway costs.
Canary Deployment
Routing a small percentage (e.g., 5%) of traffic to a new agent version before full rollout.
Token Bucket
A rate limiting algorithm that refills “tokens” over time — shared across agents to coordinate API usage.
Layered Defense
Security strategy combining input validation + tool access control + output filtering + monitoring. No single layer is enough.
Operations Playbook
Pre-defined incident → symptom → first response mappings for common production failures.
Eval Suite Gate
Running the agent’s evaluation benchmark in CI/CD — if accuracy drops or cost spikes, the deploy is blocked.

🧪 Quick Quiz

What’s the single biggest cost optimization lever for production agents?

Tiered model routing — using cheaper models (Haiku) for simple tasks and reserving expensive models (Opus) for complex reasoning. Teams report 60–90% savings from routing alone.

Why is prompt caching so effective for agents?

Agents send the same system prompt and tool definitions with every request. Caching gives a 90% discount on those repeated input tokens. For 1,000 requests with a 50K-token system prompt, that’s $135 saved.

Why are task queues often better than request-response for agents?

Agent tasks typically run 10–60 seconds — too long for synchronous HTTP. A task queue (Redis Queue, Celery, SQS) lets you process in background workers with built-in retries and timeouts.

What makes agent security fundamentally different from chatbot security?

Agents can act, not just talk. A prompt-injected agent can delete files, exfiltrate data, or send unauthorized messages. The blast radius is orders of magnitude larger than a chatbot saying something wrong.

📌 Key Takeaways

  • 88% of agent projects fail before production — the gap is engineering, not models.
  • Tiered model routing is the #1 cost lever: Opus for planning, Sonnet for work, Haiku for simple tasks.
  • Prompt caching saves 90% on repeated input tokens — essential for agents with large system prompts.
  • Security must be layered: input validation + tool access control + output filtering + monitoring. No single layer suffices.
  • Use task queues for agents (not request-response) — agent tasks are long-running by nature.
  • Track five production metrics: success rate, cost/task, latency (P50/P95), error patterns, token usage.
  • The CI/CD pipeline for agents includes an eval suite gate — if accuracy drops, the deploy is blocked.
  • Before shipping, answer: How much can it cost? What happens on failure? How will I know? What can it access? How do I roll back?
