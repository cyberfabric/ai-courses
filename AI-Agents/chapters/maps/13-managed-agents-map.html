Managed Agents: Deploy on Anthropic’s Infrastructure

Memory Map
💡

The Big Idea

Managed Agents is to self-hosted agents what Heroku was to EC2. You define what your agent does (model, tools, prompt). Anthropic runs it — containers, scaling, sandboxing, the tool-use loop. Production agents in ~40 lines of Python.

🧱 The Four Core Concepts

📋
Agent
The configuration: model, system prompt, tool definitions, MCP servers. A deployment manifest. Created once, referenced by ID.
📦
Environment
The container template: pip/npm/apt packages, networking rules (unrestricted or locked-down), file mounts.
▶️
Session
A running instance. Agent + Environment = Session. Each session is isolated with its own filesystem and state.
📡
Events
The communication channel. SSE stream for messages, tool calls, and status updates between your app and the agent.

Key insight: The session (your data) is decoupled from the container (execution). If a container crashes, a new one spins up with the same session state. This is why managed infrastructure can offer reliability guarantees that are hard to replicate yourself.

callable_agents
API field that lets an orchestrator agent delegate to specialist agents. Each runs in its own thread within the session.

⚠️ Common Pitfalls

💸
Zombie Sessions
Sessions bill while active. If your code crashes before closing, the agent keeps running and billing. Always use try/finally.
🔓
Unvalidated Custom Tools
Custom tool calls are untrusted input. An agent might craft dangerous SQL or unexpected API params. Validate everything.

🧪 Quick Quiz

What are the four core concepts of Managed Agents?

Agent (configuration), Environment (container template), Session (running instance), Events (SSE communication). An Agent runs inside an Environment, creating a Session, which communicates through Events.

Why is the Brain/Harness/Sandbox separation important?

The session (Brain) stores the event log independently of the container (Sandbox). If a container crashes, a new one spins up with the same state. The Harness (the while True loop) is stateless and replaceable. This decoupling enables reliability guarantees.

When should you choose self-hosted over Managed Agents?

When you need multi-provider model support, strict data residency requirements, deep customization beyond the API surface, or want to avoid vendor lock-in. Most teams don’t hit these needs. Start managed, migrate if needed.

Where do custom tools execute, and why does it matter?

Custom tools execute in YOUR application, not the agent’s sandbox. This means agent-crafted inputs hit your real systems (databases, APIs). You must validate inputs as strictly as any user-facing endpoint.

Can specialist agents delegate to other specialists?

No. Delegation is one level deep. The orchestrator can call specialists, but specialists cannot delegate further. This keeps execution predictable and costs bounded.

📌 Key Takeaways

  • Managed Agents = Heroku for agents. Define the config, Anthropic runs the infrastructure.
  • Four concepts: Agent (config) + Environment (container) + Session (instance) + Events (SSE stream).
  • Internal architecture: Brain (event log), Harness (the loop), Sandbox (execution) — decoupled for reliability.
  • Built-in tools: bash, file ops, glob/grep, web fetch/search. Individually toggleable.
  • Custom tools bridge the agent to your infrastructure. They run in YOUR app — validate inputs strictly.
  • Multi-agent via callable_agents. One-level delegation only. Specialists run in isolated threads.
  • Costs: $0.08/agent-hour + tokens + $10/1K searches. No base fee. Watch for zombie sessions.
  • Start managed, move to self-hosted only when you hit a real limitation. Most teams never do.
