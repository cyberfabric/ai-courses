The Big Idea

JSON schemas the LLM sees: name, description, parameters. This is the tool’s interface.
⚡
Tool Implementations
The actual code that runs: search_web(), read_page(), save_report(). Returns structured JSON with status.
📝
System Prompt
The agent’s personality and strategy. Three sections: Identity, Process, Rules.
🔁
The Loop
Call LLM → check response → execute tools → feed back → repeat until done or limit hit.

🔁 The Agent Loop, Decoded

  1. 📤 Call the LLM

    Send full conversation history (including all prior tool results) to the model.

  2. 🔍 Check the Response

    stop_reason == "tool_use" → agent wants a tool. stop_reason == "end_turn" → final answer.

  3. ⚡ Execute & Feed Back

    Run the tool, get the result, append both to messages, loop again.

  4. 🔁 Repeat or ✅ Finish

    Keep going until the agent responds (done) or hits max_steps (safety limit).

The messages list is the agent’s short-term memory — every tool call and result gets appended.

🐛 Debugging Cheat Sheet

Symptom Cause Fix
Calls wrong tool Ambiguous descriptions Clarify tool descriptions
Wrong parameters Vague param docs Add examples, use enums
Loops without progress Agent is stuck Set max_steps, check tool results
Ignores tool results Results too long Truncate large outputs
Quality degrades mid-run Context window overload Truncate tool outputs to ~5K chars

🛡️ Edge Cases & Safety

♾️
Infinite Loop Detection
Track recent tool calls. If the same call appears twice in a sliding window, the agent is stuck. Break the loop.
📏
Context Window Trap
Every result gets appended. 10 pages = 50K chars. Accuracy drops after ~32K tokens. Always truncate.
✋
Interactive Confirmation
Add approval gates before writes: prompt user, deny returns is_error: true, agent adapts.
💸
Cost Awareness
~$0.02–0.05 per session (5–8 steps). A stuck loop at 100 iterations can cost $1+. Always set max_steps.

The core is the same loop. Frameworks dress it up. Understanding the primitive makes every framework transparent.

📖 Key Terms

Agent Loop
The while True that calls the LLM, checks for tool calls, executes them, feeds results back, and repeats.
stop_reason
The API’s signal: "tool_use" means the model wants a tool; "end_turn" means it’s done.
messages
The conversation history list — the agent’s short-term memory. Every tool call and result is appended here.
max_steps
Safety limit on loop iterations. Prevents runaway execution and cost overruns. Default: 10–15 for research tasks.
tool_functions
A dictionary mapping tool names to implementation functions. The dispatcher between the LLM’s intent and your code.
Lost in the Middle
LLMs pay less attention to information in the middle of long contexts. Why you must truncate tool outputs.

🧪 Quick Quiz

What are the four pieces of an agent?

Tool definitions (JSON schemas), tool implementations (actual code), system prompt (identity + process + rules), and the loop (call LLM, check, execute, repeat).

How does the loop know when to stop?

Two conditions: stop_reason == "end_turn" (model is done) or step >= max_steps (safety limit reached).

Why truncate tool results to ~5K characters?

Every result is appended to the messages list. Large results bloat the context window, degrade accuracy (“lost in the middle”), and increase costs.

What three sections should every system prompt have?

Identity (who the agent is), Process (step-by-step approach), Rules (constraints and guardrails). Skip any and the agent becomes unpredictable.

Why build from scratch before using frameworks?

When you understand the primitive (~120 lines), every framework becomes transparent. You won’t be confused by abstractions because you’ve built the core yourself.

📌 Key Takeaways

  • A complete agent = tool definitions + tool implementations + system prompt + while True loop (~120 lines).
  • The loop: call LLM → check stop_reason → execute tools → feed results back → repeat.
  • System prompt blueprint: Identity (who), Process (how), Rules (constraints).
  • The messages list is your short-term memory. Everything gets appended — truncate to avoid the context window trap.
  • Always set max_steps and detect infinite loops. Cost per session: ~$0.02–0.05.
  • Build the primitive first. Frameworks add wiring, features, and ecosystem — the core loop is the same.
