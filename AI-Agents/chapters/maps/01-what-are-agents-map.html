Chapter 1

What Are AI Agents, Really?

Memory Map
💡

The Big Idea

An AI agent is an LLM in a loop — it observes, thinks, acts with tools, checks the result, and repeats until the job is done. Without the loop, it’s just a chatbot.

🔄 The Observe-Think-Act Loop

Every agent — from the simplest script to a multi-agent orchestra — runs this cycle:

👁️ Observe
→
🧠 Think
→
⚡ Act
→
✅ Check
→
🔁 Repeat or Stop

Like an event loop in a server or a game loop in game dev — the agent keeps running until done or hitting a limit.

🧩 The Five Agent Capabilities

🔧
Tool Use
Call external functions — search the web, read files, execute code, make API calls. Without tools, an LLM can only generate text.
📋
Planning
Break complex goals into steps and decide execution order. The agent strategizes, not just reacts.
🧠
Memory
Remember what’s done, what worked, what failed. From simple conversation history to vector databases.
🔄
Self-Correction
Detect errors, adjust approach, try again. A chatbot gives wrong answers; an agent notices and fixes them.
🚀
Autonomous Execution
Run multiple steps without human input at each step. You set the goal; the agent finds the path.
🔁
The Loop
The glue that makes it agentic. A single tool call is just function calling. The loop is what makes an agent.

📊 The Spectrum: Chatbot to Autonomous System

Type Who Steers? Uses Tools? Loops? Example
🗣️ Chatbot The conversation No / limited No Basic ChatGPT
🤝 Copilot The human Yes, human-initiated Rarely GitHub Copilot
🤖 Agent The agent (with guardrails) Yes, autonomously Yes Claude Code, Cursor
🌐 Autonomous The system itself Yes, minimal oversight Continuously Research bots, trading

Key distinction: Who decides what happens next? Chatbot = you type. Copilot = you accept/reject. Agent = the system decides, within boundaries you set.

⏰ Why Agents Became Possible (2024–2025)

  1. 🧠 Reasoning Models

    Claude Opus 4, GPT-4o, o1/o3/o4-mini — models that can plan multi-step solutions before acting.

  2. 🔧 Formalized Tool Use

    Native JSON schemas, guaranteed valid calls, multi-tool responses — no more parsing JSON from markdown.

  3. 🔌 MCP Standard

    The “USB-C for AI tools.” Universal tool connections, now under the Linux Foundation with industry-wide backing.

  4. 🏗️ Production Infrastructure

    Monitoring, tracing, guardrails, deployment tooling — agents moved from “cool demo” to deployable.

🧪 The “Bash Script Test”

✅
Use a Script
If a deterministic script can handle it — use the script. Faster, cheaper, more predictable.
🤔
Consider an Agent
If the script would be 10,000 lines of if/else — the LLM handles the judgment calls those branches encode.
🤖
Agent Territory
If the task requires understanding context and making judgment calls — this is what agents are for.

⚠️ Why Agents Need Guardrails

🌀
Hallucination + Actions
A chatbot gives wrong answers. An agent deletes the wrong file, calls the wrong API, commits broken code.
💸
Runaway Costs
A confused agent stuck in a loop burns hundreds of dollars in minutes. Always set token budgets and step limits.
💥
Blast Radius
The tools you give an agent define what it can break. Start with minimum tools needed — you can always add more.
😎
Confidence Problem
LLMs are confidently wrong as often as confidently right. Build “ask for help” behavior explicitly.

📖 Key Terms

AI Agent
A system using an LLM to observe, reason, act with tools, and loop until a goal is achieved.
Observe-Think-Act Loop
The core cycle every agent runs: perceive state, reason about next step, execute action, check result, repeat.
Tool Use
An agent’s ability to call external functions — search, read files, execute code, call APIs.
MCP (Model Context Protocol)
The “USB-C for AI tools” — a universal standard for connecting agents to tools, databases, and APIs.
Blast Radius
The scope of damage an agent can cause, defined by the tools and permissions you give it.
Hallucination
When an LLM generates confident but incorrect information. Especially dangerous in agents because they act on it.
Function Calling
The API mechanism that lets an LLM request tool execution with structured JSON parameters.
Context Window
The amount of text an LLM can process at once. 1M+ tokens let agents hold entire codebases in memory.

🎯 The Core Analogy

👨‍💻

An Agent = A New Developer on Your Team

You give them a task, they have skills (the LLM), access to tools (IDE, terminal, docs), and they work step by step. They make mistakes. They sometimes ask for help. And you definitely don’t give them sudo rm -rf / on day one.

🧪 Quick Quiz

What’s the key difference between a chatbot and an agent?

The loop. A chatbot responds once per message. An agent keeps going — observing, thinking, acting, and checking — until the goal is achieved.

Can a single API tool call make something an agent?

No. A single tool call is just function calling. An agent uses tool calls inside a loop, deciding which tools to call, evaluating results, and continuing until done.

When should you NOT use an agent?

When a deterministic script can handle the task. If it doesn’t require judgment, context understanding, or multi-step reasoning, use a simpler automation.

What is MCP and why does it matter?

Model Context Protocol — a universal standard for connecting agents to tools. It’s the “USB-C for AI tools,” backed by major players and now under the Linux Foundation.

Name the five core agent capabilities.

Tool Use, Planning, Memory, Self-Correction, and Autonomous Execution — all powered by the observe-think-act loop.

📌 Key Takeaways

  • An agent is an LLM in a loop — observe, think, act, check, repeat.
  • Five capabilities: tool use, planning, memory, self-correction, autonomous execution.
  • The spectrum runs: chatbot → copilot → agent → autonomous system. The key is who decides what happens next.
  • Use the “Bash script test” — only build an agent when the task requires judgment and context.
  • Agents need guardrails: token budgets, step limits, minimal tool access, human-in-the-loop.
  • The core agent loop is ~20 lines of code. Every framework is a dressed-up version of while True.
