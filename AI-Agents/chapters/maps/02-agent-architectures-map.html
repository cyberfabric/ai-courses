Chapter 2

Agent Architectures

Memory Map
💡

The Big Idea

The same LLM with the same tools behaves completely differently depending on how you structure the loop. Architecture is the management style — micromanaging, delegating, or full project planning.

🏗️ The Six Architectures at a Glance

🔁
Simple Tool-Use Loop
Call LLM → execute tool → feed result back → repeat. No reasoning structure. Good for 1–3 step tasks.
🧠
ReAct
Think → Act → Observe → Repeat. Forces the agent to reason before every action. The production workhorse.
📋
Plan-and-Execute
Make a full plan first, then execute steps one by one. Re-plan when things go wrong. Best for complex multi-step projects.
🪞
Reflexion
Act → Critique → Fix → Repeat. The agent reviews its own output and self-corrects. Doubles token cost but boosts quality.
🌳
Tree of Thoughts
Explore multiple reasoning paths in parallel, evaluate each, pick the best. Like a chess engine. Expensive, rarely needed.
🚦
Router
A lightweight agent routes requests to specialist agents. Gateway to multi-agent systems. Two flavors: handoff vs agent-as-tool.

🤖 Autonomy
🎯 Reliability
Pick Two

High autonomy + high reliability = expensive. High autonomy + low cost = unreliable. High reliability + low cost = low autonomy (more human oversight).

🖥️ The System Prompt = The Architecture

🎯

ReAct Is a Prompt Pattern, Not a Framework

You implement ReAct by telling the model how to think in the system prompt. Three questions your system prompt must answer: What’s the goal? How to approach? What NOT to do?

🧩 Key Patterns Worth Remembering

  1. 📋 Planning as a Tool

    A “create_plan” tool that’s a no-op — it just stores the plan in context. Context engineering: shaping what the model sees to improve behavior. Claude Code uses this internally.

  2. 🚦 Handoff vs Agent-as-Tool

    Handoff: specialist takes over the conversation (OpenAI style). Agent-as-tool: specialist handles a subtask and returns results to the router (Claude SDK style).

  3. 🔀 Hybrid Architectures

    Production agents rarely use one pure pattern. Typical: Plan-and-Execute outer loop with ReAct inside each step. Or Router dispatching to specialists using Reflexion.

  4. 🏭 The Production Rule

    Deterministic workflows with embedded AI decision points. Not pure autonomous agents — structured pipelines where the LLM handles judgment calls and regular code handles the rest.

📖 Key Terms

ReAct
Reasoning + Acting — interleave explicit thinking with tool calls. The default production architecture.
Plan-and-Execute
Create a full plan before executing. Re-plan when reality diverges from the plan.
Reflexion
Self-correction loop: produce output, critique it, fix issues, repeat. Doubles cost but boosts quality.
Tree of Thoughts
Explore multiple reasoning paths in parallel, evaluate each, pursue the best. Like a chess engine.
Router Pattern
A lightweight agent routes requests to specialist agents based on the task type.
Context Engineering
Shaping what’s in the model’s context window to improve its behavior — the plan tool is a prime example.
Extended Thinking
Claude’s thinking blocks — reasoning that doesn’t count against output tokens. Free audit trail for ReAct.
Trade-Off Triangle
Autonomy, Reliability, Cost — pick two. Every architecture is a different trade-off point.

🎭 The Core Analogy

👨‍💼
Micromanager
Simple Loop — do this, now do that, no thinking time, just execute.
🤝
Delegator with Check-ins
ReAct — think it through, act, report what you found, decide next step together.
📊
Project Planner
Plan-and-Execute — write the full plan first, then execute step by step, adjust if needed.
🔍
Quality Reviewer
Reflexion — do the work, then critique your own work, fix what’s wrong.

🧪 Quick Quiz

What makes ReAct different from a simple tool-use loop?

ReAct forces the agent to think out loud before every action. The explicit Thought step creates an audit trail, improves decisions, and makes debugging possible.

When should you use Reflexion, and what’s the catch?

Use it for high-stakes outputs where correctness matters (code gen, published content). The catch: it doubles token cost because every critique is another full LLM call.

What is “Planning as a Tool” and why does it work?

A no-op tool that writes a plan into the conversation context. It works because the plan stays visible in the context window, keeping the agent on track. It’s context engineering.

What are the three sides of the trade-off triangle?

Autonomy, Reliability, Cost — pick two. High autonomy + high reliability is expensive. High autonomy + low cost is unreliable. High reliability + low cost requires more human oversight.

What’s the production rule of thumb for choosing an architecture?

Start with the simplest architecture that could work. Add complexity only when you have evidence the simple version fails. Most production agents use ReAct or simple loops.

📌 Key Takeaways

  • Six architectures: Simple Loop, ReAct, Plan-and-Execute, Reflexion, Tree of Thoughts, Router.
  • ReAct is the default — think before every action. Implemented via system prompt, not framework magic.
  • Plan-and-Execute shines for complex multi-step tasks. The “plan tool” trick keeps agents on track via context engineering.
  • Trade-off triangle: Autonomy, Reliability, Cost — pick two.
  • Production agents use hybrids — deterministic workflows with embedded AI decision points.
  • The system prompt is your architecture in natural language. It must answer: goal, approach, and guardrails.
  • Start simple. Add complexity only when the simpler approach fails.
