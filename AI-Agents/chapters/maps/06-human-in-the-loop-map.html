Chapter 6

Human-in-the-Loop Patterns

Memory Map
🎯

The Big Idea

95% accuracy sounds great — until your agent runs 100 steps per day. That’s 5 mistakes daily. Human-in-the-loop isn’t a limitation — it’s the most critical pattern for production agents.

📊 The Autonomy Spectrum

Level Human Role Agent Freedom When to Use
🔴 Fully Manual Approves every action None Learning, debugging
🟡 Approval Gates Approves risky actions only Low-risk actions free Early production
🟢 Supervised Autonomy Monitors, intervenes on errors Most actions free Trusted production
🔵 Full Autonomy Reviews outputs after the fact All actions free Low-stakes, well-tested

Most production agents in 2026 live at Approval Gates or Supervised Autonomy. Full autonomy is rare.

🛡️ Five Human-in-the-Loop Patterns

🚪
1. Approval Gates
Categorize tools into three tiers: auto-approve (reads), needs-approval (writes), blocked (destructive). Pause before risky actions.
🆘
2. Escalation
Give the agent an ask_human tool. It escalates when <80% confident, stuck after 2 failures, or facing ambiguous requests.
💰
3. Budget Pattern
Set limits on tokens, tool calls, and time. Agent runs freely within budget, pauses for check-in when exhausted.
📜
4. Audit Trails
Log every tool call and decision to JSONL. Review after the fact for debugging, compliance, and trust-building.
🔔
5. Notifications
Alert humans (Slack, email, PagerDuty) without blocking execution. Agent continues; humans monitor and intervene if needed.

Never Default to “Allow” on Timeout

If the human isn’t responding, auto-approving is the worst option. Default to the safest choice: skip, log, continue with non-destructive work. Run skipped actions later with approval.

📈 Building Trust Gradually

  1. 🔴 Week 1: Approve Everything

    Pair on every action. The agent proves it understands the task and makes sensible tool choices.

  2. 🟠 Week 2: Auto-Approve Reads

    Searches, file reads, and memory recalls run freely. Writes still need approval.

  3. 🟡 Month 2: Auto-Approve Safe Writes

    Low-risk writes (saving reports, creating drafts) go through. Destructive actions still gated.

  4. 🟢 Month 6: Supervised Autonomy

    Agent runs with spot checks. Move tools from needs_approval to auto_approve based on track record.

Like onboarding a new team member: pair on everything → review all PRs → review only complex changes → autonomous with spot checks.

📖 Key Terms

Approval Gate
A checkpoint that pauses the agent and requires human approval before executing a sensitive tool call.
Escalation
When the agent voluntarily asks for human guidance via an ask_human tool — typically when confidence is low or the task is ambiguous.
Budget Pattern
Resource limits (tokens, tool calls, time) that let the agent run freely until exhausted, then require a human check-in.
Audit Trail
A JSONL log of every tool call, decision, and approval — used for post-hoc review, debugging, and compliance.
Bounded Autonomy
The agent runs freely within defined limits, with mandatory escalation for high-stakes decisions. The industry standard for production agents.
Graceful Degradation
Fallback behavior when the human doesn’t respond: timeout with safe default, queue for later, partial execution, or complete stop.
Permission Stack
Claude Code’s priority-based permission model: Deny (highest) → Ask → Allow (lowest), evaluated per-tool and per-argument.

🧪 Quick Quiz

Why is 95% accuracy not good enough for agents?

At 100 steps per day, 95% accuracy means 5 mistakes daily. If any involve destructive actions (deleting files, sending emails, modifying production data), the cost is severe.

What are the three tiers in an approval gate system?

Auto-approve (reads, searches — run freely), needs-approval (writes, sends — pause for human), blocked (destructive operations — always denied).

When should an agent escalate to a human?

When confidence is below ~80%, after 2 failed attempts at the same approach, when facing ambiguous instructions, or when encountering conflicting information from multiple sources.

How do budget and approval gates complement each other?

They work on different dimensions: budget prevents runaway costs (resource limits), while approval gates prevent dangerous actions (risk classification). Together they cover both cost and safety.

What should happen when a human doesn’t respond to an approval request?

Never auto-approve. Default to the safest option: skip the action, log it, continue with non-destructive work. Queue the skipped action for later approval.

📌 Key Takeaways

  • Full autonomy is rarely the right choice. Most production agents use approval gates or supervised autonomy.
  • Five patterns: approval gates, escalation, budgets, audit trails, and notifications — combine them.
  • Categorize tools into three tiers: auto-approve, needs-approval, blocked. Unknown tools default to “ask.”
  • Give agents an ask_human tool so they can escalate when uncertain instead of guessing.
  • Budget + approval gates is the most common production setup — cost control and risk control together.
  • Never auto-approve on timeout. Default to skip, log, and continue safely.
  • Build trust gradually: start tight, loosen permissions as the agent proves itself over weeks and months.
