The Big Idea

A system prompt is a job description for a personal assistant. Without it, your agent is a helpful stranger. With it, your agent knows your name, your allergies, your budget, and when to leave you alone. The best prompt is one you've iterated on for a month.

All six sections work together. Persona sets the vibe, context makes it personal, standing orders set limits, examples show format, priorities resolve conflicts.

🧩 Core Prompt Patterns

🎭
Persona Crafting
Define three dimensions: tone (casual vs formal), proactiveness (anticipate needs vs wait), and detail level (full recipe vs "how about tacos?").
🏠
Context Injection
The "new roommate" approach: tell your agent everything a new housemate needs to know — household, diet, schedule, budget, quirks.
🚧
Standing Orders
Always-on guardrails: "never spend without asking," "no reminders before 10 AM on weekends," "when unsure, ASK." Your agent's common sense, defined by you.
⚖️
Priority Hierarchy
When rules conflict: Safety > Budget > Preferences > Convenience > Quality. Tells the agent which instruction wins.
📸
Few-Shot Examples
Show, don't tell. Include 2-3 example interactions for your most common tasks. The agent mimics the format and detail level you demonstrate.
📐
Structured + Natural
Use bullet points for hard rules and facts. Use prose for personality and soft preferences. Best prompts blend both — like a resume.

📅 A Month of Prompt Evolution

  1. Week 1: Basics

    Name, household members, casual tone. "Help me, be casual." The minimum viable prompt.

  2. Week 2: Guardrails

    Agent suggests a $60 dinner → add budget limits. Agent buys something → add "ask before spending." Reactions to real mistakes.

  3. Week 3: Schedule & Context

    Agent schedules gym during book club → add your full weekly schedule. Add weekend morning quiet hours.

  4. Week 4: Fine-Tuning

    Add few-shot examples for grocery lists. Adjust tone. Your prompt is now battle-tested and deeply personal.

🔥 Debugging Bad Agent Behavior

Problem Likely Cause Prompt Fix
Too verbose No length guidance "Keep responses under 100 words"
Ignores diet Restrictions buried deep Move dietary info to the top
Suggests pricey options No budget context Add explicit limits + define "cheap"
Asks too many questions Overly cautious rule Narrow: "Ask before spending/deleting only"
Forgets preferences Only in past chats Put essentials in system prompt, not memory

9 out of 10 times, bad behavior = unclear instructions. Check the prompt before blaming the LLM.

📖 Key Terms

System Prompt
Hidden instructions that run before every conversation. Your agent's personality, boundaries, and knowledge about you. Lives in config.yaml.
User Prompt
The message you type in the chat. The system prompt shapes how the agent interprets and responds to every user prompt.
Standing Orders
Always-on rules the agent follows without being asked: "never spend without asking," "no weekend reminders before 10 AM."
Context Injection
Teaching your agent personal details — diet, schedule, budget, household — so every response is tailored to your life, not generic.
Priority Hierarchy
Ranked list for resolving conflicting instructions: Safety > Budget > Preferences > Convenience > Quality.
Few-Shot Examples
2-3 sample interactions in the system prompt that show the agent exactly how you want responses formatted.
Persona
Your agent's communication style across three dimensions: tone (casual/formal), proactiveness (anticipates/waits), detail level (full/brief).
Prompt Templates
Reusable instruction blocks for common tasks: meal planning, email triage, shopping lists, travel research.

🧪 Quick Quiz

Why is prompting an agent riskier than prompting a chatbot?

Because agents act. A vague instruction to a chatbot produces a mediocre answer. The same instruction to an agent might trigger real purchases, messages, or schedule changes.

What's the "new roommate" approach to context injection?

Tell your agent everything you'd tell a new roommate in their first week: dietary restrictions, household quirks, schedule, budget limits, pet needs — the details that prevent mistakes.

What's the single most important standing order?

"When unsure, ASK — don't guess." An agent that asks is mildly annoying but safe. An agent that guesses wrong causes real problems.

When should you use structured vs. natural language in prompts?

Structured (bullets, lists) for hard rules and facts (budgets, diet). Natural language (prose) for personality and soft preferences (tone, humor). Best prompts use both.

How many few-shot examples should you include?

2-3 examples covering your most common interaction types (lists, scheduling, meals). More than 5 bloats the system prompt.

📌 Key Takeaways

  • A system prompt is a job description — persona + context + standing orders + examples + priority hierarchy.
  • The "new roommate" test: if a new housemate would need to know it, put it in the prompt.
  • Most important standing order: "When unsure, ASK — don't guess."
  • Priority hierarchy resolves conflicts: Safety > Budget > Preferences > Convenience > Quality.
  • Don't write the perfect prompt on day one. Iterate weekly based on real mistakes.
  • Your system prompt is sensitive data — treat it like a diary. Back it up, don't share it.
