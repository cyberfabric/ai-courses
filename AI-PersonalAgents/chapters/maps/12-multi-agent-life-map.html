The Big Idea

One agent doing everything is like one person doing every job in a company. When your system prompt hits 200 lines and context starts leaking, it's time to specialize: focused agents per domain, coordinated by a CEO agent. To you it feels like one seamless assistant. Behind the scenes, five specialists collaborate.

🏠
Home Agent
Home Assistant, scenes, routines, energy, maintenance, security. Local LLM for device control privacy.
💰
Finance Agent
Expenses, budgets, shopping lists, subscriptions, deal finding, cost splitting. Local LLM for financial privacy.
❤️
Health Agent
Meals, workouts, medications, sleep, weight, weekly summaries. Local LLM — most sensitive data of all.
📅
Scheduler Agent
Calendar, email triage, social coordination, travel. Cloud LLM — email drafts need intelligence.
📚
Knowledge Agent
Books, media, learning goals, recommendations, Obsidian. Cloud LLM — recommendations need broad knowledge.

Schedule vs Health Scheduler: "Meeting at 7:30" Health: "That's gym time" Ask user; suggest 8:30 instead
Energy vs Comfort Home: "Eco mode" Health: "User is sick" Health overrides — comfort temp

🧠 Shared vs. Private Memory

🔒 Health data (private)
🔒 Finance data (private)
📋 Shared Memory (profile, preferences, standing orders)
🔒 Home data (private)
🔓 Coordinator (reads shared only)

Finance Agent can't see health metrics. Health Agent can't see expenses. Shared memory has universal rules only.

📈 The Evolution Path

  1. Single Agent

    One agent does everything. Use for 2-4 weeks. Don't split until you feel the pain.

  2. Split Health

    Most sensitive data, clearest boundaries. Move health prompt + data to a dedicated local agent.

  3. Split Finance

    Same reasoning — sensitive data, clear domain. Second agent to isolate.

  4. Add Coordinator

    With 3 agents, you need a CEO. Introduce CrewAI for routing and merging.

  5. Full Team

    Split Home, Scheduler, Knowledge as needed. Only pay the complexity cost when you get clear value.

📖 Key Terms

Coordinator Pattern
A master agent that routes your messages to specialist agents, collects their responses, resolves conflicts, and replies as one voice.
Specialist Agent
A focused agent with a short system prompt covering one domain (health, finance, home). Better results through focus.
CrewAI
Open-source framework for multi-agent orchestration — handles agent creation, task routing, and inter-agent communication.
Shared Memory
Data accessible to all agents: user profile, preferences, standing orders. NOT domain-specific sensitive data.
Private Memory
Domain-specific data accessible only to its specialist: health metrics (Health Agent only), expenses (Finance Agent only).
Priority Hierarchy
Safety > Health > Budget > Convenience > Quality. The coordinator uses this to break ties when agents conflict.
Agent-to-Agent Communication
Specialists querying each other: Health Agent asks Finance Agent for grocery budget before planning meals.
Privacy Creep (Multi-Agent)
More agents = more data flows, more API keys, more attack surface. The coordinator sees routing metadata even without data access.

🧪 Quick Quiz

Why split a single agent into multiple specialists?

When the system prompt is 200+ lines, the agent starts losing context — forgetting dietary restrictions during travel planning, for example. Specialists have focused prompts with only the context they need, giving better results AND better data isolation.

How does the coordinator resolve conflicts between agents?

Using the priority hierarchy: Safety > Health > Budget > Convenience > Quality. A sick user needing comfort temperature overrides the Home Agent's energy-saving mode. For ambiguous conflicts, the coordinator presents both options and lets you decide.

Why do sensitive agents (health, finance, home) use local LLMs?

These agents handle the most private data — medications, expenses, door lock codes. Running them on local models (Ollama) means that data never leaves your machine. Cloud models are reserved for agents needing broad intelligence (scheduler, knowledge).

When should you NOT split into multi-agent?

When your single agent handles everything well. Don't split preemptively — complexity has a cost. Split only when you feel the pain: system prompt too long, context being lost, or you need stricter data isolation between domains.

What's the "company analogy" for multi-agent systems?

A single agent is a one-person startup — decent at everything, excellent at nothing. Multiple specialists are departments: a finance team, a health team, a home team. The coordinator is the CEO who routes work and resolves disagreements.

🎯 Key Takeaways

  • One agent doing everything is like one person doing every job — specialize when the system prompt gets too long or context leaks.
  • The coordinator pattern: you talk to one agent, it routes to specialists, merges responses, and replies seamlessly.
  • Five specialists: Home, Finance, Health, Scheduler, Knowledge — each with focused prompts and only the data they need.
  • Sensitive agents (health, finance, home) run on local LLMs. Cloud intelligence goes to scheduler and knowledge agents.
  • Agents communicate: Health asks Finance for budget before meal planning. Scheduler tells Health about gym conflicts.
  • Conflicts resolve via priority hierarchy: Safety > Health > Budget > Convenience > Quality.
  • Don't split until you feel the pain. Start with single agent, split health first, then finance, then add coordinator.
  • Cost is manageable: ~$3/month cloud for 2 agents + $0 for local agents on a Pi 5 with 4 GB RAM.
