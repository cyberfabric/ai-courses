Chapter 3

Setting Up Your First Agent

Memory Map
💡

The Big Idea

Setting up a personal agent is like assembling IKEA furniture: short parts list, numbered steps, genuinely useful result. Clone, configure, connect, chat — 15 minutes from zero to a working agent in your messaging app.

🛠️ The Setup Journey

  1. Install OpenClaw

    git clone → npm install → cp config.example.yaml config.yaml → npm start. Agent is alive but deaf, blind, and mute.

  2. Connect an LLM (The Brain)

    Add your API key to config.yaml. Claude Sonnet recommended for the sweet spot of speed, cost, and smarts. Or go fully local with Ollama.

  3. Bridge a Messaging App (The Ears & Mouth)

    Telegram: get a token from @BotFather. WhatsApp: scan a QR code. Signal: use signal-cli. Set allowed_users to lock it down.

  4. First Conversation

    Send "Hello!" in your messaging app. If the agent responds, you're done. If not, check the troubleshooting section.

  5. First Useful Task

    Ask about the weather, set a reminder, create a shopping list. Prove the full loop: message → reasoning → action → response.

⚙️ The config.yaml Anatomy

🏷️
agent:
Name, language, timezone. Call your agent whatever you like — "Jarvis," "Friday," "Bob."
🧠
llm:
Provider, model, API key, temperature. Temperature 0.7 = good default. Lower for facts, higher for creativity.
🌉
bridges:
Telegram token, WhatsApp session, Signal config. Always set allowed_users to lock down access.
🔧
tools:
Weather, reminders, notes, lists. Each tool has its own API key and storage path. Enable only what you need.
💾
memory:
Local backend, max history. Keeps conversation context between sessions — the agent remembers what you said yesterday.
📋
logging:
Set to "info" normally, "debug" when troubleshooting. Logs at ./logs/openclaw.log.

📖 Key Terms

Bridge
The component connecting OpenClaw to a messaging app (Telegram, WhatsApp, Signal). Your agent's ears and mouth.
BotFather
Telegram's official bot for creating bots. Send /newbot to get a token for your agent.
config.yaml
OpenClaw's DNA file. Contains LLM settings, bridge tokens, tool configs, memory, and logging. Never commit to Git.
Temperature
LLM creativity dial. 0.0 = precise/factual, 1.0 = creative/varied. Default 0.7 for personal agents.
allowed_users
Whitelist of messaging usernames that can talk to your agent. Without it, anyone can access your personal data.
Ollama
Tool for running LLMs locally. ollama pull llama3.2 downloads a model; OpenClaw connects via localhost:11434.
Docker Compose
Container orchestration for "set and forget" deployment. restart: always keeps your agent alive through reboots.

🔥 Troubleshooting Quick Reference

🔌
Port 3700 in use
lsof -i :3700 to find the culprit. Kill it or change port in config.
🔑
API key 401 error
Check for extra spaces/quotes. Verify key is active. Confirm billing is set up.
🤖
Telegram bot silent
Send /start first. Check username in allowed_users (case-sensitive). One token per bot instance.
📸
WhatsApp QR expired
Delete ./data/whatsapp-session, restart, scan quickly. Have WhatsApp ready before restart.

🧪 Quick Quiz

What are the four commands to install and start OpenClaw?

git clone the repo, npm install, cp config.example.yaml config.yaml, npm start.

Why should you always set allowed_users in the bridge config?

Without it, anyone who finds your bot's username can talk to your agent and access your personal data. It's like leaving your front door open.

What does the temperature setting control?

How creative vs. predictable the LLM is. 0.7 is the default. Lower (0.3) for strict facts (finance). Higher (0.9) for creative suggestions (meals, travel).

Why start on your laptop but eventually move to Docker or a Pi?

Your laptop is easiest for testing, but the agent sleeps when the lid closes. Docker/Pi gives 24/7 availability — a personal agent should always be reachable.

What's the #1 security rule at setup time?

Never commit config.yaml to Git — it contains your API key. Add it to .gitignore immediately. Also set spending limits on your API dashboard.

📌 Key Takeaways

  • OpenClaw setup: clone → npm install → config → start. Takes ~15 minutes.
  • Start with Claude Sonnet as the LLM. Swap to local/other models later with one config line.
  • Start with Telegram for the easiest bridge. Migrate to WhatsApp/Signal once it works.
  • Always set allowed_users and never commit config.yaml to Git.
  • Use Docker with restart: always for a 24/7 agent that survives reboots.
  • Start on your laptop, prove it works, then move to a home server or Pi for always-on availability.
