Chapter 3

Setting Up Your First Agent

From zero to “Hello, agent” — installation, LLM connection, and your first conversation

Time to Get Your Hands Dirty

In the last two chapters you learned what personal AI agents are and which tools exist. Now we stop talking and start building. By the end of this chapter you will have a working OpenClaw agent running on your machine, connected to an LLM, bridged to your favorite messaging app, and ready to chat.

Think of it like assembling IKEA furniture. The parts list is short, the steps are numbered, and when you're done you have something genuinely useful — except this one won't wobble.

What You'll Have by the End of This Chapter

  • OpenClaw installed and running locally
  • An LLM connected (Claude, GPT, or a local model)
  • A messaging bridge to Telegram, WhatsApp, or Signal
  • A completed first conversation with your agent
  • A working weather-check-and-reminder task proving the full loop

Prerequisites: The Shopping List

Before we start installing anything, let's make sure you have everything you need. The good news: the list is short.

Setup Checklist

  • A computer — Mac, Linux, or Windows with WSL. Minimum 8 GB RAM (16 GB if you plan to run a local LLM later)
  • Node.js 20+ — OpenClaw runs on Node. Check with node --version
  • Git — to clone the repository. You're a developer, you have this
  • A messaging app — Telegram (easiest), WhatsApp, or Signal installed on your phone
  • An LLM API key — Claude (Anthropic) or GPT (OpenAI), or Ollama installed for a local model
  • 15 minutes — seriously, that's all this takes

Which Messaging App?

If you don't have a strong preference, start with Telegram. Its Bot API is the easiest to set up — no phone number scanning, no third-party bridges, just a simple token from BotFather. You can always add WhatsApp or Signal later.

Step 1: Installing OpenClaw

OpenClaw is open-source and lives on GitHub. Installation is three commands and a config file. Let's go.

Clone the Repository

git clone https://github.com/openclaw/openclaw.git
cd openclaw
↓

Install Dependencies

npm install
↓

Copy the Example Config

cp config.example.yaml config.yaml
↓

Start OpenClaw

npm start

If all goes well, you'll see a friendly startup message in your terminal:

🐾 OpenClaw v2.4.1
   Status: running
   LLM: not configured
   Bridges: none
   Dashboard: http://localhost:3700

OpenClaw is alive — but it's deaf, blind, and mute. It has no LLM brain and no way to talk to you. Let's fix that.

Windows Users

OpenClaw runs best on macOS or Linux. On Windows, use WSL 2 (Windows Subsystem for Linux). Install Ubuntu from the Microsoft Store, then follow the Linux instructions inside WSL. Native Windows support exists but the messaging bridges can be flaky.

Step 2: Connecting an LLM

An agent without an LLM is like a car without an engine. The LLM is the brain that understands your messages and decides what to do. OpenClaw supports three approaches:

Cloud API (Claude or GPT)

  • Smartest models available
  • No local compute needed
  • Costs a few cents per conversation
  • Requires internet connection
  • Your messages go to the API provider

Local Model (Ollama / LM Studio)

  • 100% private — nothing leaves your machine
  • Free after hardware cost
  • Works offline
  • Needs 16+ GB RAM
  • Slower, less capable models

Option A: Claude API (Recommended)

Claude is excellent at following nuanced personal instructions — "don't suggest sushi on weekdays because the good place is too far from the office" is the kind of thing Claude handles well. Here's how to connect it:

  1. Go to console.anthropic.com and create an account
  2. Navigate to API Keys and generate a new key
  3. Open config.yaml and add your key:
llm:
  provider: anthropic
  model: claude-sonnet-4-20250514
  api_key: sk-ant-your-key-here
  max_tokens: 4096

Which Claude Model?

Start with Claude Sonnet — it's fast, affordable, and smart enough for personal agent tasks. Move to Claude Opus if you need the agent to handle complex multi-step reasoning (like planning a two-week trip across three countries). For simple reminders and lists, even Claude Haiku works great and costs almost nothing.

Option B: OpenAI API

llm:
  provider: openai
  model: gpt-4o
  api_key: sk-your-key-here
  max_tokens: 4096

Option C: Local Model with Ollama

If privacy is your top priority (and it probably should be for personal data), you can run a model entirely on your machine:

  1. Install Ollama: curl -fsSL https://ollama.com/install.sh | sh
  2. Pull a model: ollama pull llama3.2
  3. Update config.yaml:
llm:
  provider: ollama
  model: llama3.2
  base_url: http://localhost:11434
  max_tokens: 4096

Restart OpenClaw (npm start) and you should now see:

🐾 OpenClaw v2.4.1
   Status: running
   LLM: anthropic/claude-sonnet (connected ✓)
   Bridges: none
   Dashboard: http://localhost:3700

The brain is online. Now let's give it a mouth and ears.

Step 3: Bridging a Messaging App

The whole point of a personal agent is that you talk to it where you already are — your messaging app. OpenClaw uses bridges to connect to Telegram, WhatsApp, Signal, and Discord. Let's set up the easiest one first.

Telegram (Easiest Setup)

1. Create a Bot

Open Telegram, search for @BotFather, and send /newbot. Give it a name like "My Personal Agent" and a username like my_life_agent_bot. BotFather gives you a token.

↓

2. Configure the Bridge

bridges:
  telegram:
    enabled: true
    token: "123456:ABC-your-bot-token"
    allowed_users:
      - your_telegram_username
↓

3. Restart & Test

Restart OpenClaw, open your bot in Telegram, and send "Hello!"

Lock Down Your Bot

Always set allowed_users in the config. Without it, anyone who finds your bot's username can talk to your agent — and it has access to your personal data. This is like leaving your front door open with a sign saying "free stuff inside."

WhatsApp (More Setup, More Familiar)

WhatsApp doesn't have a simple bot API, so OpenClaw uses the whatsapp-web.js bridge. This simulates a WhatsApp Web session:

  1. Enable the WhatsApp bridge in config.yaml:
bridges:
  whatsapp:
    enabled: true
    session_path: ./data/whatsapp-session
  1. Restart OpenClaw. A QR code appears in the terminal
  2. Open WhatsApp on your phone → Linked Devices → Scan the QR code
  3. Send a message to your own number (yes, it works) or create a group with just you and the agent

WhatsApp vs Telegram for Agents

Telegram wins for developer experience (proper bot API, no QR scanning, markdown support). WhatsApp wins if your whole family uses it and you want to add the agent to a family group later. Both work fine — pick whichever you'll actually use daily.

Signal (Privacy-First Choice)

Signal requires the signal-cli tool as a bridge. It's the most privacy-respecting option but the most complex to set up. If you chose Signal, follow the OpenClaw docs for the signal-cli bridge — it involves registering a secondary number and linking it.

Understanding the Config File

The config.yaml file is your agent's DNA. Let's look at the key sections and set sensible defaults for personal use:

# config.yaml — annotated for personal use

# Your agent's name and personality
agent:
  name: "Jarvis"              # Call it whatever you like
  language: "en"               # Primary language
  timezone: "Europe/Sofia"     # Your timezone for reminders

# LLM connection (configured in Step 2)
llm:
  provider: anthropic
  model: claude-sonnet-4-20250514
  api_key: sk-ant-your-key-here
  max_tokens: 4096
  temperature: 0.7             # 0.0 = precise, 1.0 = creative

# Messaging bridges (configured in Step 3)
bridges:
  telegram:
    enabled: true
    token: "your-token"
    allowed_users:
      - your_username

# Built-in tools
tools:
  weather:
    enabled: true
    provider: openweathermap
    api_key: "your-owm-key"    # Free tier is enough
  reminders:
    enabled: true
    storage: ./data/reminders.json
  notes:
    enabled: true
    storage: ./data/notes/
  lists:
    enabled: true
    storage: ./data/lists/

# Memory — the agent remembers context between conversations
memory:
  enabled: true
  backend: local               # Keeps everything on your machine
  max_history: 100             # Number of messages to remember

# Logging
logging:
  level: info                  # Use "debug" when troubleshooting
  file: ./logs/openclaw.log

The Temperature Setting

Temperature controls how creative vs. predictable the LLM is. For personal agents, 0.7 is a good starting point. Lower it to 0.3 if you want strict, factual answers (finance tracking). Raise it to 0.9 if you want more creative suggestions (meal planning, travel ideas).

Setting Up a Second Tool for Comparison

Remember from Chapter 2: it's smart to try at least two tools before committing. Let's quickly set up Jan.ai as a comparison — it takes about 5 minutes.

Jan.ai: The Fully Offline Option

  1. Download Jan from jan.ai — available for Mac, Windows, and Linux
  2. Install and launch. It downloads a default local model automatically
  3. Start chatting in the built-in interface — no API key, no bridge, no config

Jan is a great comparison because it's the opposite philosophy from OpenClaw: everything self-contained in a desktop app, no terminal required, fully offline. The tradeoff? No messaging integration, fewer automation features, and you have to open the app instead of texting your agent.

OpenClaw

  • Lives in your messaging app
  • Highly configurable
  • Plugin ecosystem for automation
  • Requires terminal setup
  • Always-on via server/Docker

Jan.ai

  • Desktop app you open when needed
  • Zero configuration
  • Limited to chat (no automation)
  • Download and run
  • Runs when you open it

For this course, we'll continue with OpenClaw as the primary tool — but now you know how easy it is to have a fallback.

Running Locally vs. Home Server

Your agent is running on your laptop right now. That's fine for testing, but what happens when you close the lid? The agent goes to sleep. For a personal agent, you want it available 24/7 — like a butler who doesn't take breaks.

Your Laptop

  • Easy to set up and debug
  • Agent sleeps when laptop sleeps
  • Uses your machine's resources
  • Great for testing and development

Home Server / Raspberry Pi

  • Runs 24/7 — always available
  • Low power consumption (~5W for a Pi)
  • Keeps personal data on your network
  • Requires initial setup and SSH access

A Raspberry Pi 5 (8 GB) runs OpenClaw beautifully for cloud LLM setups. If you want to run a local model too, consider a mini PC with 32 GB RAM. But don't overthink this now — start on your laptop and migrate later when you're sure this is a keeper.

Docker: Set It and Forget It

If you want your agent to survive reboots, run in the background, and be easy to update, Docker is your friend:

# docker-compose.yaml
version: "3.8"
services:
  openclaw:
    image: openclaw/openclaw:latest
    container_name: my-agent
    restart: always
    volumes:
      - ./config.yaml:/app/config.yaml
      - ./data:/app/data
    ports:
      - "3700:3700"
    environment:
      - TZ=Europe/Sofia

Then just run:

docker compose up -d

The restart: always flag means your agent comes back after reboots, crashes, or power outages. It's like setting up a cron job, except Docker handles all the messy details.

Life Hack: The Morning Weather Agent

Here's a quick win that proves the value of an always-on agent. Set up a scheduled message in your config:

schedules:
  morning_briefing:
    cron: "0 7 * * *"    # Every day at 7:00 AM
    action: "Check the weather and tell me if I need an umbrella or a jacket. Also remind me of anything on today's list."

Every morning at 7 AM, your Telegram buzzes with: "Good morning! 18°C and cloudy, rain expected after 2 PM — grab an umbrella. You have 3 items on today's list: dentist at 10, pick up dry cleaning, call Mom."

No app to open. No weather site to check. It just arrives, like a helpful text from a friend who wakes up earlier than you.

Troubleshooting: The Top 5 Setup Problems

Every developer knows the first run never works perfectly. Here are the most common issues and how to fix them:

1. Port 3700 Already in Use

# Find what's using the port
lsof -i :3700
# Either kill that process or change OpenClaw's port:
# In config.yaml: server.port: 3701

2. API Key Not Working

Symptoms: "401 Unauthorized" or "Invalid API key" in logs.

  • Check for extra spaces or quotes around the key in config.yaml
  • Verify the key is active in your provider's dashboard
  • Make sure you have billing set up (Anthropic and OpenAI require a payment method)

3. Telegram Bot Not Responding

  • Send /start to the bot first — Telegram requires this initial handshake
  • Check that allowed_users contains your exact Telegram username (case-sensitive)
  • Make sure no other bot instance is using the same token

4. WhatsApp QR Code Expired

The QR code refreshes every 30 seconds. If it keeps expiring:

  • Delete the session folder: rm -rf ./data/whatsapp-session
  • Restart OpenClaw to generate a fresh QR
  • Scan quickly — have WhatsApp open and ready before restarting

5. "Module not found" on npm start

# Clear and reinstall
rm -rf node_modules package-lock.json
npm install

When in Doubt, Check the Logs

Set logging.level: debug in your config and restart. The log file at ./logs/openclaw.log will tell you exactly what's going wrong. Read the last 20 lines: tail -20 logs/openclaw.log

Security Basics at Setup Time

Your agent is going to know your schedule, your shopping list, your health data, and your finances. Security isn't optional — it's table stakes. Here are the essentials to get right now, before you start adding personal data.

API Key Security

Your API key is essentially a password that costs money. Treat it accordingly:

  • Never commit it to Git. Add config.yaml to your .gitignore immediately
  • Use environment variables for production setups: api_key: ${ANTHROPIC_API_KEY}
  • Set spending limits in your API provider's dashboard. Start with $10/month — a personal agent rarely costs more than $2-3/month
  • Rotate keys quarterly. Old keys are like old passwords — replace them regularly
  • Monitor usage. If your bill spikes, someone might be using your key

Security Setup Checklist

  • API key stored in config.yaml (not hardcoded in code)
  • config.yaml added to .gitignore
  • Spending limit set on API provider dashboard
  • allowed_users configured for messaging bridge
  • OpenClaw dashboard not exposed to the internet (localhost only)
  • Firewall allows only local network access if running on a home server

Don't Expose the Dashboard

OpenClaw's web dashboard runs on localhost:3700 by default. If you're running it on a home server, do not port-forward 3700 to the internet. Access it via your local network or SSH tunnel. Exposing it means anyone on the internet can read your conversations and control your agent.

Yes to all three!
Done! Here's what I set up: 📋 Saturday to-do: "Barbecue" added 🛒 Shopping list "BBQ Supplies" created: • Charcoal • Burger patties • Hot dogs • Buns • Condiments • Drinks (Edit this list anytime by saying "update BBQ shopping list") ⏰ Reminder set: Friday 6:00 PM — "Prep for tomorrow's barbecue: marinate meat, check supplies, clean the grill" Anything else for the weekend?
