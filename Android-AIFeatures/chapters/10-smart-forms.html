Smart Forms & Data Extraction

Auto-fill from photos, receipt parsing, natural language input, and forms that fill themselves

Users Hate Forms

Nobody has ever said "I love filling out forms." Every field the user has to type is friction. Every dropdown they have to scroll through is a micro-frustration. And on mobile, where typing is slower and screens are smaller, forms are the #1 place users abandon tasks.

AI can fix this. Every field the AI fills is a field the user doesn't type. Every photo that becomes a form entry is 30 seconds of tedious data entry eliminated. Smart forms don't just collect data — they extract it from the world around the user.

67% Of mobile users abandon forms with more than 3 fields
5x Faster form completion with AI auto-fill vs manual entry
92% Accuracy of receipt parsing with OCR + LLM pipeline

Photo-to-Form: The "Snap and Done" Pattern

The Snap-and-Done Pattern

The most satisfying smart form experience: point the camera at a document, tap, and watch the form fill itself.

  1. User taps "Scan" — Camera opens (or photo picker)
  2. AI extracts data — OCR reads text, LLM parses it into structured fields
  3. Form pre-fills — All fields populate with extracted data, highlighted in blue to signal "AI filled this"
  4. User reviews and edits — Each field is editable. Low-confidence fields are highlighted in yellow
  5. User confirms — One tap saves the entry. Done.

Real-world example: Expensify lets you photograph a receipt and auto-fills merchant, date, amount, and category. Adobe Scan captures business cards and creates contacts. Both follow the same pattern: scan, extract, review, save.

Recipe: Receipt Photo to Expense Form

Goal: User photographs a receipt, app extracts merchant, date, total, and line items into an editable expense form.

Step 1: Extract raw text with ML Kit (on-device, free):

suspend fun extractText(bitmap: Bitmap): String {
    val recognizer = TextRecognition.getClient(TextRecognizerOptions.DEFAULT_OPTIONS)
    val image = InputImage.fromBitmap(bitmap, 0)
    return suspendCoroutine { cont ->
        recognizer.process(image)
            .addOnSuccessListener { result -> cont.resume(result.text) }
            .addOnFailureListener { cont.resume("") }
    }
}

Step 2: Parse the raw text with Gemini into structured JSON:

suspend fun parseReceipt(rawText: String): ReceiptData {
    val prompt = """
        Extract data from this receipt text. Return valid JSON only:
        {
          "merchant": "store name",
          "date": "YYYY-MM-DD",
          "total": 0.00,
          "currency": "USD",
          "items": [{"name": "item", "price": 0.00}],
          "confidence": {"merchant": "high|medium|low", "date": "high|medium|low",
                         "total": "high|medium|low"}
        }
        If a field cannot be determined, use null.
        Receipt text:
        $rawText
    """.trimIndent()

    val response = model.generateContent(prompt)
    return Json.decodeFromString(response.text ?: "{}")
}

Step 3: Display in an editable form with confidence indicators:

@Composable
fun ExpenseForm(receipt: ReceiptData, onSave: (ReceiptData) -> Unit) {
    var merchant by remember { mutableStateOf(receipt.merchant ?: "") }
    var date by remember { mutableStateOf(receipt.date ?: "") }
    var total by remember { mutableStateOf(receipt.total?.toString() ?: "") }

    Column(modifier = Modifier.padding(16.dp)) {
        SmartTextField(
            value = merchant, onValueChange = { merchant = it },
            label = "Merchant",
            confidence = receipt.confidence?.merchant
        )
        SmartTextField(
            value = date, onValueChange = { date = it },
            label = "Date",
            confidence = receipt.confidence?.date
        )
        SmartTextField(
            value = total, onValueChange = { total = it },
            label = "Total",
            confidence = receipt.confidence?.total,
            prefix = receipt.currency ?: "$"
        )
        // Line items list...
        Button(onClick = { onSave(/* updated data */) }) {
            Text("Save Expense")
        }
    }
}

The SmartTextField shows a blue border for high-confidence fields and a yellow border for low-confidence ones, signaling to the user which fields to double-check.

Scenario: Aisha just had lunch with a client. She photographs the crumpled receipt. The app processes it in 2 seconds: "Café Bella, March 31, 2026, Total: $47.50." The merchant and total are highlighted blue (high confidence). The date is yellow (the receipt was faded). Aisha confirms the date, taps "Save," and the expense is logged. Total time: 8 seconds instead of 2 minutes of manual typing.

Natural Language to Structured Data

Sometimes the user doesn't have a document to scan — they just want to type naturally and have the app figure out the structure.

Traditional Form

Event: [____________] Person: [____________] Date: [__/__/____] Time: [__:__] Location: [____________] Five fields. Five taps. Five chances to abandon.

Natural Language Input

User types: "lunch with Anna tomorrow at noon at the Italian place" AI extracts: Event: Lunch Person: Anna Date: 2026-04-01 Time: 12:00 Location: Italian restaurant One field. One sentence. Done.

suspend fun parseNaturalInput(input: String, today: LocalDate): EventData {
    val prompt = """
        Extract event details from this natural language input.
        Today's date is ${today}. Return valid JSON:
        {"event": "", "person": "", "date": "YYYY-MM-DD",
         "time": "HH:MM", "location": ""}
        Use null for fields not mentioned.
        Input: $input
    """.trimIndent()

    val response = model.generateContent(prompt)
    return Json.decodeFromString(response.text ?: "{}")
}

The UX works best as a confirm-before-save pattern: show the extracted fields in a card, let the user tap to edit any field, then confirm to save. Never auto-create the event without review.

Always pass today's date to the LLM. "Tomorrow," "next Tuesday," and "this weekend" are meaningless without a reference date. Include the current date in every prompt that involves relative time expressions.

Intelligent Autofill

Beyond scanning documents, AI can predict field values based on context — what screen the user is on, what they've entered before, and what makes sense.

Cross-Field Prediction

When the user fills one field, predict the others:

fun predictExpenseFields(merchant: String, history: List<Expense>): PredictedFields {
    // Find past expenses at this merchant
    val pastEntries = history.filter {
        it.merchant.contains(merchant, ignoreCase = true)
    }.sortedByDescending { it.date }

    if (pastEntries.isEmpty()) return PredictedFields()

    // Predict category from history
    val likelyCategory = pastEntries
        .groupBy { it.category }
        .maxByOrNull { it.value.size }?.key

    // Predict typical amount range
    val avgAmount = pastEntries.map { it.amount }.average()

    return PredictedFields(
        category = likelyCategory,
        estimatedAmount = avgAmount,
        paymentMethod = pastEntries.first().paymentMethod
    )
}

// When user types "Starbucks" in the merchant field:
// Category auto-selects "Coffee/Dining"
// Amount hint shows "Usually ~$5.50"
// Payment method pre-selects "Corporate Card"

This doesn't even require AI APIs — it's pattern matching on local history. Simple, instant, and incredibly useful for repeat expenses.

AI-Powered Input Validation

Traditional validation: "Does this match a regex?" AI validation: "Does this make sense?"

Validation Type Traditional AI-Powered
Email Regex pattern match Pattern + "Did you mean john@gmail.com?" (typo in domain)
Address Required fields filled Geocoding check: "Is this a real address?"
Phone number Digit count check Format detection: "Looks like a US number, formatting as (555) 123-4567"
Description Min/max length Content moderation: "Is this appropriate for public display?"
Amount Is it a number? "This is 10x your usual spend at this merchant. Confirm?"

AI validation supplements, never replaces. Always keep your traditional validation (regex, required fields, type checks) as the baseline. AI validation adds a layer of intelligence on top. If the AI is wrong about a "suspicious" amount, the user can still override it. If your regex is wrong, nothing can save you.

Handling Extraction Errors

AI will misread text and extract wrong values. The UX must make correction effortless.

The Confidence-Based UI

@Composable
fun SmartTextField(
    value: String, onValueChange: (String) -> Unit,
    label: String, confidence: String?,
    prefix: String? = null
) {
    val borderColor = when (confidence) {
        "high" -> MaterialTheme.colorScheme.primary          // blue
        "medium" -> Color(0xFFD97706)                        // amber
        "low" -> MaterialTheme.colorScheme.error              // red
        else -> MaterialTheme.colorScheme.outline             // default
    }

    OutlinedTextField(
        value = value, onValueChange = onValueChange,
        label = { Text(label) },
        prefix = prefix?.let { { Text(it) } },
        colors = OutlinedTextFieldDefaults.colors(
            focusedBorderColor = borderColor,
            unfocusedBorderColor = borderColor.copy(alpha = 0.6f)
        ),
        trailingIcon = {
            if (confidence == "low") {
                Icon(Icons.Default.Warning, "Low confidence",
                    tint = MaterialTheme.colorScheme.error)
            } else if (confidence != null) {
                Icon(Icons.Outlined.AutoAwesome, "AI filled",
                    tint = MaterialTheme.colorScheme.primary,
                    modifier = Modifier.size(18.dp))
            }
        }
    )
}

High-confidence fields get a subtle sparkle icon. Low-confidence fields get a warning icon and an amber/red border. The user's eye is naturally drawn to the fields that need attention, while confident fields can be quickly scanned and accepted.

Offline Form Intelligence

Expense tracking and document scanning often happen where there's no signal — on a plane, in a basement parking garage, or at a conference with overwhelmed Wi-Fi.

  • ML Kit OCR: Runs entirely on-device. Always available
  • Local prediction models: Cross-field suggestions from history work offline
  • Queue LLM parsing: If cloud parsing isn't available, save the photo and raw OCR text. Parse when back online
  • Basic local extraction: Use regex for common patterns (dates, currency amounts, email addresses) as an offline fallback for LLM parsing
suspend fun extractReceiptData(bitmap: Bitmap): ReceiptData {
    val rawText = extractText(bitmap)  // ML Kit, always works

    return if (isOnline()) {
        parseReceiptWithLlm(rawText)   // best quality
    } else {
        parseReceiptLocally(rawText)   // regex fallback
    }
}

fun parseReceiptLocally(text: String): ReceiptData {
    val datePattern = Regex("""\d{1,2}[/\-.]\d{1,2}[/\-.]\d{2,4}""")
    val amountPattern = Regex("""\$?\d+\.\d{2}""")

    return ReceiptData(
        date = datePattern.find(text)?.value,
        total = amountPattern.findAll(text).lastOrNull()?.value,  // last amount is usually total
        merchant = text.lines().firstOrNull()?.take(40),          // first line is often merchant
        confidence = mapOf("merchant" to "low", "date" to "medium", "total" to "medium")
    )
}

The offline experience should feel seamless. Don't show "AI unavailable" — just silently use the local fallback. The form fills with lower accuracy, but the user can correct the fields. When back online, silently re-process queued items for better accuracy and update the entries.

Data Privacy for Extracted Data

Receipts and documents contain sensitive information: financial data, personal contacts, business information. Handle with care:

  • Process on-device when possible: ML Kit OCR never sends data off-device. Use it as the first step
  • Minimize cloud data: Send the extracted text to the LLM, not the original image. Text is smaller and contains less incidental data (no background, no other documents visible)
  • Encrypt local storage: Expense data stored in Room should use SQLCipher or Android's EncryptedSharedPreferences for sensitive fields
  • Clear data retention policy: Tell users how long you keep extracted data, and actually delete it when the time comes
