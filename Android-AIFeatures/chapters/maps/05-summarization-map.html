The Big Idea

Search is a librarian — it finds what exists. Generation is a writer — it creates something new. The safest pattern: AI drafts, the user finishes. Editing a 70% draft beats staring at a blank page every time.

🧩 Core Concepts

📚
Summarization Stack
Three tiers: Gemini Nano (free, on-device, limited), Gemini Flash (cheap, fast, great quality), Gemini Pro (expensive, best quality). Flash is the sweet spot for most apps.
✨
The TL;DR Button
ViewModel with sealed state (Idle → Loading → Streaming → Complete → Error). Stream tokens with generateContentStream(). Show an "AI Summary" badge.
🎯
Prompt Engineering
Specify format (3 bullets), length (one sentence each), focus (action items), tone, and edge cases. "Summarize this" is never enough — structured prompts win.
📝
AI Draft Pattern
AI creates a 70-80% draft → user edits → user owns the result. Never auto-send. Gmail's "Help me write" is the gold standard.
🗂️
Map-Reduce Chunking
For long documents: split into chunks → summarize each → summarize the summaries. Smart chunking splits at paragraph/sentence boundaries, never mid-sentence.
💰
Cost Control
Cache summaries (60-80% fewer calls), use smaller models for simple tasks, batch requests, set per-user limits. Always run the cost formula before shipping.

🔄 The AI Draft Lifecycle

  1. 📋 User Provides Context

    "Reply to this email" / "Summarize my notes" / "Write a product description"

  2. 🤖 AI Generates Draft

    Streaming output with structured prompt — format, length, focus, tone all specified

  3. ✏️ User Edits

    Tweaks wording, adds personal touches, removes anything off-base

  4. ✅ User Sends / Saves

    The final version belongs to the user, not the AI. Ownership = trust.

💰 Cost Control Strategies

💾
Cache Summaries
Store with source hash. Regenerate only when source content changes. Saves 60-80% of API calls.
🔽
Right-Size Models
Flash for most tasks (5-10x cheaper than Pro). Nano for short text. Pro only for complex analysis.
📦
Batch Requests
Group multiple summarizations into batch API calls. Lower per-request overhead.
🚦
Per-User Limits
Set daily caps. Predictable monthly costs. Show friendly limit-reached UI.
🧮

The Cost Formula

Monthly cost = DAU × summaries/user/day × (input + output tokens) × price/token × 30
Run this before you ship. Then run it again at 10x your current DAU.

📖 Key Terms

Streaming
Displaying AI output token-by-token as it's generated — the ChatGPT-style typing effect that sets expectations for generation time.
Token Limit / Context Window
Maximum text a model can process. Flash: 1M tokens. GPT-4o-mini: 128K. On-device: often just 4K-8K. Know your limit.
Map-Reduce
Split long documents into chunks, summarize each independently, then summarize the summaries into a final result.
Hallucination
When the summary mentions facts not in the original text. Use a verification prompt for critical content (medical, financial, legal).
Source Attribution
Linking generated content back to its source ("Based on your meeting notes from March 28") to help users verify and build trust.
Template-Based Generation
Combining AI with structured templates (hook, features, CTA) so every generated description follows a consistent format.
AI Draft Pattern
AI creates a starting point (70-80% done), user edits and owns the final result. Editing beats writing from scratch.
Cache Invalidation
Storing a hash of source content with the cached summary. If the hash changes, the cache is stale — regenerate.

Quick Quiz

Why is "Summarize this" a bad prompt?

No format guidance, no length constraint, no focus area. You'll get inconsistent output — sometimes a paragraph, sometimes 10 bullets. Specify format (3 bullets), length (one sentence each), and focus (action items).

What's the map-reduce pattern for long documents?

Split the document into chunks at natural boundaries (paragraphs, sections) → summarize each chunk independently → summarize all the chunk summaries into a final result.

When should you use Gemini Nano vs Flash for summarization?

Nano: short text (<500 words), offline scenarios, instant results, free. Flash: longer documents, higher quality needed, any device with internet. Try Nano first, fall back to Flash.

How does caching reduce summarization costs by 60-80%?

Store the summary alongside a hash of the source content. On subsequent views, check the hash — if the source hasn't changed, serve the cached summary instantly. Only regenerate when content actually changes.

Why should AI-generated content never auto-send?

The AI Draft Pattern: AI creates a starting point, user edits and owns the final result. Auto-sending removes user control, breaks trust, and risks sending incorrect or inappropriate content.

Key Takeaways

  • Summarization and content generation are the most common generative AI features — and the easiest to get wrong.
  • Gemini Flash is the sweet spot for most summarization: fast, cheap, excellent quality. Try Nano first if the device supports it.
  • Structured prompts (format + length + focus + tone + edge cases) produce consistent, useful output. "Summarize this" does not.
  • The AI Draft Pattern is the safest generation approach: AI drafts, user edits, user owns the result.
  • Cache aggressively with source hashing — most summaries don't need regenerating. Run the cost formula before you ship.
  • Quality gates matter: hallucination checks for critical content, regenerate buttons for user control, source attribution for trust.
