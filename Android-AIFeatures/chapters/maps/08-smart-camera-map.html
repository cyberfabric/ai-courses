Smart Camera & Real-Time Processing

The Big Idea

Live camera AI has 50 milliseconds per frame. Cloud APIs are too slow — this is on-device only territory. CameraX captures frames, ML Kit / MediaPipe analyzes them, and Compose draws the results as overlays. Everything happens locally, in real time.

🏗️ The CameraX + AI Pipeline

  1. 📷 CameraX Preview

    PreviewView shows live camera feed. ImageAnalysis use case runs in parallel with STRATEGY_KEEP_ONLY_LATEST.

  2. 🔍 ML Inference

    Your ImageAnalysis.Analyzer receives ImageProxy frames on a background thread. Run ML Kit or MediaPipe detection.

  3. 📐 Coordinate Transform

    Scale bounding boxes from model space (e.g., 640×480) to screen space (preview view size). The #1 visual bug source.

  4. 🎨 Draw Overlay

    Compose Canvas stacked on top of PreviewView in a Box. Draw bounding boxes, labels, edge highlights.

⚠️

The #1 Bug

Always close imageProxy.close() — in addOnCompleteListener, not addOnSuccessListener. Forgetting this stalls the entire camera pipeline. No more frames arrive. Ever.

🎯 Frame Rate Management

Strategy How It Works When to Use
Skip N frames Analyze every 3rd frame, close the rest Default starting point. Simple and effective.
Async pipeline Process frame N while camera captures N+1. KEEP_ONLY_LATEST drops old frames. When model is fast enough (~30-50ms)
Adaptive rate Analyze more when camera moves / objects change, less when stable Extended scanning sessions. Saves battery.

Start with "every 3rd frame" — ~100ms per analysis while preview stays smooth at 30fps.

📖 Key Terms

ImageAnalysis.Analyzer
CameraX interface that receives frames as ImageProxy on a background thread. You must close the proxy when done.
STREAM_MODE
ML Kit Object Detection mode optimized for live camera — tracks objects across frames for smoother results.
STRATEGY_KEEP_ONLY_LATEST
CameraX backpressure strategy: drops old frames if analysis is slower than capture. Prevents queue buildup.
Coordinate Transformation
Scaling bounding boxes from model space (640×480) to screen space (preview view size). Scale X and Y independently.
Document Scanner API
Google Play Services API that handles edge detection, crop, dewarp, enhancement, and multi-page scanning. Free, no custom code needed.
Contextual Actions (Barcode)
Acting on barcode content by type: URL→Open, WiFi→Connect, Contact→Add, Product→Search. Don't just show raw data.
Thermal Throttling
After 2-3 min of camera AI, CPU slows to cool down. Inference time doubles, frame rate drops. Monitor and adapt proactively.
Frame Skipping
Analyzing every Nth frame instead of every frame. "Every 3rd frame" = ~100ms per analysis while maintaining smooth 30fps preview.

Quick Quiz

Why can't you use cloud APIs for live camera AI?

Live camera needs <50ms per frame for smooth 20fps overlays. A cloud API round-trip takes 500ms-3s. On-device inference is the only option for real-time camera features.

What's the #1 bug in CameraX + ML Kit pipelines?

Forgetting to call imageProxy.close(). If you don't close it, the camera pipeline stalls and no more frames arrive. Always close in addOnCompleteListener, not addOnSuccessListener, so it's closed even on failure.

Why start with "analyze every 3rd frame" instead of every frame?

Most ML Kit detectors take 30-50ms. The camera captures at 30fps (33ms/frame). Analyzing every frame would fall behind. Every 3rd frame gives ~100ms per analysis while keeping the preview smooth at 30fps.

What makes contextual barcode actions better than showing raw content?

Instead of displaying "https://example.com", show an "Open in Browser" button. WiFi code → "Connect" button. Product barcode → "View product info". Each content type triggers a different, useful action automatically.

Why ask for camera permission contextually instead of at launch?

Cold requests at launch lack context — users don't know why you need the camera and default to Deny. Contextual requests (user taps "Scan Receipt" → explain why → mention privacy → system dialog) get higher grant rates because the user understands the purpose.

Key Takeaways

  • Live camera AI = on-device only. You have <50ms per frame. Cloud APIs are too slow.
  • CameraX pipeline: Preview + ImageAnalysis → ML Kit/MediaPipe inference → Compose Canvas overlay. Always close ImageProxy.
  • Use Google's Document Scanner API for scanning — don't reinvent edge detection, crop, and dewarp.
  • Barcode scanning's power is contextual actions: URL→Open, WiFi→Connect, Product→Search.
  • Start with "every 3rd frame" for frame rate management. Monitor thermal state and reduce rate when the device heats up.
  • Camera permission: explain why before asking, mention privacy, never nag after permanent denial.
