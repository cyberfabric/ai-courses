Chapter 2

Choosing the Right AI Approach

Memory Map
💡

The Big Idea

There's no single "best" AI approach. Cloud APIs are taxis (powerful, expensive), on-device models are bicycles (free, limited), hybrid is a car (flexible, complex). Let your feature requirements choose the path — not hype.

🛤️ The Three Paths

🚕
Cloud AI APIs
Send data to a server, get results back. Powerful models, fast to ship, but the meter is always running. Needs internet.
🚲
On-Device Models
Run inference on the phone. Free at scale, works offline, private — but smaller, less capable models.
🚗
Hybrid
On-device for fast/private tasks, cloud for heavy lifting. Best of both worlds, most complex to build and maintain.

📐 The 5-Dimension Decision Matrix

Dimension ☁️ Cloud Wins 📱 On-Device Wins
⏱️ Latency >500ms OK (summaries, search) <100ms needed (autocomplete, camera)
🔒 Privacy Public / non-sensitive data Health, financial, personal data
💰 Cost at Scale <50K DAU (affordable) >100K DAU ($0 per inference)
📡 Offline Need Always connected Must work without internet
🧠 Capability Complex reasoning, generation Classification, detection, simple NLP

If your answers split across columns → go hybrid.

DAU × requests/user/day × tokens/request × price/token × 30
Always run this for current users AND 10x growth. If 10x breaks the bank, plan your fallback now.

⏱️ Latency Budgets

  1. <100ms — On-Device Only

    Autocomplete, keyboard predictions, live camera overlays. No network round-trip possible.

  2. 100-500ms — Fast Cloud or On-Device

    Search results, quick classifications. Cloud works if latency is optimized.

  3. 1-3 seconds — Cloud with Loading UI

    Summarization, content generation. Acceptable with streaming animation or progress indicator.

📖 Key Terms

Cloud AI API
Send data to a remote model (Gemini, OpenAI, Claude). Powerful but costs per request and needs internet.
On-Device Inference
Run the model on the phone itself (TFLite, ML Kit, Gemini Nano). Free, private, fast, but limited.
Hybrid Architecture
On-device for fast/private tasks, cloud for heavy lifting. Smart routing decides which path.
Tiered Intelligence
The proven pattern: on-device handles simple/frequent tasks, cloud handles complex/occasional ones.
Backend Proxy
Your server sits between the app and AI API. Keeps API keys safe, adds rate limiting, enables monitoring.
Firebase App Check
Verifies requests come from your real app, not a script abusing your API endpoint.
AiProvider Interface
Abstract your AI calls behind a Kotlin interface. Switch providers without rewriting features.
Graceful Degradation
When AI is unavailable, fall back to a usable (if less magical) experience. Never a dead end.

🧪 Quick Quiz

Test Your Understanding

When should you choose on-device over cloud?

When you need <100ms latency (camera, autocomplete), handle sensitive data (health, financial), need offline support, or have a large user base where API costs would be prohibitive.

Why should you never hardcode API keys in your Android APK?

APKs can be decompiled in minutes. Anyone can extract the key, abuse it, and run up your bill. Use a backend proxy instead — the key stays on your server, protected and revocable.

What's the cost formula for cloud AI features?

DAU × requests/user/day × tokens/request × price/token × 30. Always model for current users AND 10x growth.

Why abstract AI providers behind an interface?

To avoid vendor lock-in. If you call Gemini directly in 47 places and need to switch to OpenAI, you're stuck. With an AiProvider interface, switching is a one-line DI change. It also makes testing easy with a FakeAiProvider.

What's the "Tiered Intelligence" pattern?

Use on-device for fast, frequent, private tasks (classification, predictions) and cloud for heavy, occasional, high-value tasks (summarization, generation). A routing layer decides which path based on task complexity, connectivity, and user preferences.

Key Takeaways

  • Three paths: Cloud (powerful, costly), On-Device (free, limited), Hybrid (best of both, most complex).
  • Use the 5-dimension decision matrix: latency, privacy, cost, offline need, capability.
  • Always model costs at current scale AND 10x growth before committing to cloud.
  • Never put API keys in the APK — use a backend proxy + Firebase App Check.
  • Abstract AI behind an interface (AiProvider) to avoid vendor lock-in and simplify testing.
  • Every AI feature needs a fallback: degrade gracefully, never dead-end the user.
  • Latency is about perceived speed — streaming animations make 2s feel fast.
  • When data is sensitive, on-device is the safe default. No legal paperwork beats "data never left the phone."
