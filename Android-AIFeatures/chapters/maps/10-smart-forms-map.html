The Big Idea

Every field the AI fills is a field the user doesn't type. Smart forms don't just collect data — they extract it from the world: photos, documents, natural language. The goal is zero manual entry.

✅ Validation: Traditional vs AI-Powered

Field Traditional AI-Powered
Email Regex match "Did you mean john@gmail.com?" (typo detection)
Address Required fields filled Geocoding: "Is this a real address?"
Phone Digit count Format detection + auto-format
Amount Is it a number? "10x your usual spend here. Confirm?"

⚠️ AI validation supplements, never replaces. Always keep regex/type checks as the baseline. AI adds intelligence on top. If the AI flags a "suspicious" amount, the user can override. If your regex is broken, nothing saves you.

📄 Document Extraction by Type

  1. 🧾 Receipts

    Merchant, date, total, line items, currency. Send raw OCR text to LLM for structured JSON. Include confidence per field.

  2. 💼 Business Cards

    Name, title, company, phone, email, address, website. More structured = more reliable extraction. One-tap "Add to Contacts."

  3. 💬 Natural Language

    "Lunch with Anna tomorrow at noon" → structured event. Always pass today's date to resolve relative time expressions.

  4. 📊 Invoices & IDs

    Same pipeline, different prompts. The LLM prompt defines what fields to extract. Adapt the template for each document type.

📖 Key Terms

ML Kit Text Recognition
Google's on-device OCR. Free, fast, always works offline. The foundation of every extraction pipeline.
Structured Extraction
Using an LLM to convert raw OCR text into typed JSON fields (merchant, date, amount, etc.).
SmartTextField
A Compose component with confidence-colored borders and icons that guide the user to fields needing review.
Cross-Field Prediction
Predicting field values from other fields using local history. No API calls, just Room queries.
Confirm-Before-Save
Always show extracted data for user review before persisting. Never auto-create entries without confirmation.
SQLCipher
Encryption for Room databases. Essential when storing sensitive extracted data (financial, personal contacts).

🔒 Data Privacy Checklist

  • Process on-device first: ML Kit OCR never sends data off-device
  • Send text, not images: Text is smaller and contains less incidental data than the full photo
  • Encrypt local storage: Use SQLCipher or EncryptedSharedPreferences for sensitive fields
  • Clear retention policy: Tell users how long you keep extracted data — and actually delete it

🧪 Quick Quiz

What's the advantage of sending OCR text to the LLM instead of the original image?

Text is smaller (cheaper, faster) and contains less incidental data — no background objects, no other visible documents. Better for both cost and privacy.

Why must you always pass today's date when parsing natural language time expressions?

"Tomorrow," "next Tuesday," and "this weekend" are meaningless without a reference date. The LLM needs the current date to resolve relative expressions into absolute dates.

How does cross-field prediction work without any AI API calls?

It queries the local Room database for past entries matching the current merchant, then uses groupBy/maxBy to find the most common category, average amount, and last-used payment method. Pure local pattern matching.

What should happen when the user is offline and tries to scan a receipt?

ML Kit OCR works offline for text extraction. Use regex fallback to extract dates and amounts from the raw text. Queue the receipt for LLM re-processing when back online. Don't show "AI unavailable" — just silently use the lower-accuracy fallback.

Why does AI validation supplement rather than replace traditional validation?

Traditional validation (regex, type checks) is deterministic and reliable. AI validation adds intelligence (typo correction, anomaly detection) but can be wrong — and users can override it. If the regex baseline is broken, nothing catches the error.

Key Takeaways

  • The Snap-and-Done pipeline: Photo → ML Kit OCR → LLM structured extraction → editable form with confidence colors.
  • Use confidence-based UI (blue/amber/red borders) to guide users to fields that need review instead of checking everything.
  • Cross-field prediction from local history needs no API — typing a merchant auto-suggests category, amount, and payment method.
  • Natural language input ("lunch with Anna tomorrow") replaces multi-field forms. Always pass today's date to the LLM.
  • Build offline-first: ML Kit OCR + regex fallback always works. Queue LLM parsing for when connectivity returns.
