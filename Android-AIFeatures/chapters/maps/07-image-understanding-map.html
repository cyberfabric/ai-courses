    User photographs a receipt, business card, or document

  2. 🔤 ML Kit OCR (On-Device)

    Extract raw text strings — free, instant, no internet needed

  3. 🤖 LLM Parsing (Cloud)

    "Extract: merchant, date, total, line items as JSON" — costs fraction of a cent

📖 Key Terms

Image Labeling (ML Kit)
On-device classification into 400+ categories with confidence scores. Set threshold at 0.7+ for useful labels.
Multimodal Model
An AI that processes both images and text. Send a photo + a question, get a text answer. Gemini Vision, GPT-4o Vision.
CLIP (Contrastive Language-Image Pre-Training)
Model that embeds images and text into the same 512-dim vector space. Enables "find similar images" and text-based image search.
Smart OCR
ML Kit extracts raw text (free, on-device) → LLM parses into structured JSON. Two cheap steps that turn photos into data.
Image Embeddings
A fixed-length vector representing an image's visual content. Similar images have similar vectors (high cosine similarity).
Point and Search
The Google Lens pattern: photograph an object → AI identifies it → app searches for related content. Vision + search combined.
WorkManager Batch Processing
Background analysis of photo libraries in small batches (20 at a time), with battery constraints, using periodic work requests.
Two-Tier Accessibility
ML Kit labels as basic alt text (instant, free) + Gemini Vision for rich descriptions (cloud, on-demand or pre-generated).

Quick Quiz

When should you use ML Kit vs Gemini Vision for image analysis?

ML Kit: basic classification, tagging, labeling — free, fast, on-device, private. Gemini Vision: when you need understanding (describe scenes, extract structured data, answer questions about the image). Use ML Kit for everything it handles, Gemini for the rest.

What makes CLIP special for visual search?

CLIP embeds both images and text into the same vector space. You can search images using another image ("find similar") OR using text ("sunset at the beach") — same engine, two input types.

How does Smart OCR turn a receipt photo into structured data?

Two steps: (1) ML Kit text recognition extracts raw text strings (free, on-device), (2) an LLM parses the raw text into structured JSON with fields like merchant, date, total, items (cloud, fraction of a cent).

Why is image preprocessing important before sending to AI?

Resize (cloud charges by size), fix orientation (EXIF rotation), convert format (HEIC → JPEG), compress (80% quality saves bandwidth), strip GPS metadata (privacy). Without preprocessing, you get wrong results, higher costs, and privacy leaks.

What's the photo privacy rule of thumb?

On-device first (ML Kit, no data leaves phone). For cloud: explicit opt-in consent, strip metadata before upload, transient processing only (don't store copies on the server). Photos are deeply personal.

Key Takeaways

  • ML Kit labels images (400+ categories, <100ms, free). Gemini Vision understands them (descriptions, questions, structured extraction).
  • Visual search with CLIP embeds images into vectors — "find more like this" without keywords, and you can search by text too.
  • Smart OCR = ML Kit text extraction (free) + LLM parsing (cheap) — turns photos of receipts and documents into structured data.
  • Always preprocess images: resize, fix orientation, compress, strip metadata. Saves cost, improves results, protects privacy.
  • Photos are deeply personal — on-device first, explicit consent for cloud, strip GPS metadata, no server-side storage.
  • Cost-saving stack: ML Kit ($0) → Gemini Flash (~$0.002) → GPT-4o (~$0.01-0.03). Cache every result.
