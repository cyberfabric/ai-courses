In-App Chat & Conversational UI

Memory Map

The Big Idea

A chatbot is for conversations that need clarification, exploration, or multi-step reasoning. If a button or a one-shot AI feature solves the problem, a chatbot just adds friction. The best chatbots are scoped — clear domain, firm boundaries, consistent personality.

🤔 Chatbot or Not?

Scenario ✅ Chatbot Makes Sense ❌ Chatbot Is Overkill
Interaction type Multi-turn, needs clarification One-shot, binary, simple lookup
Examples Customer support, product discovery, tutoring Toggle settings, yes/no choices, weather card
User need Explore, refine, follow up Get one answer and move on
Better alternative — Button, form, or Chapter 5 TL;DR feature

🧩 Core Building Blocks

📊
Data Model
ChatMessage (role, content, status, timestamp) + Conversation (id, title, systemPrompt). Persisted with Room.
💬
Chat UI in Compose
LazyColumn for messages, ChatBubble with role-based styling, ChatInput with send button. Auto-scroll only when near bottom.
🌊
Streaming Responses
Use chat.sendMessageStream() to show tokens arriving in real time. Add a thin progress bar during streaming. Never show a dead spinner.
🧠
Conversation Memory
Under 50 messages: send full history. Longer: sliding window + summary of older messages. Keeps context without exploding costs.
🎯
Scoped System Prompt
Define domain, boundaries, personality, and output format. "Recipe assistant for CookMate" beats "helpful assistant" every time.
🔧
Function Calling
AI triggers app actions: "Find pasta recipes" → calls searchRecipes() → returns real data. Start with 2-3 tools, not 20.
🎨
Rich Messages
Sealed class with Text, RecipeCard, ActionChips, ImageMessage. Action chips guide conversation and reduce typing.
💰
Cost Control
Rate limits: 10/min, 50/day (free), 100/conversation. Global circuit breaker on cost. Chat is the most expensive AI feature.

⚠️ Edge Cases to Handle

👋
Empty State
Show a greeting + 3 tappable suggestion chips. A blank screen is intimidating — guide the user in.
📜
Long Conversations
200+ messages degrades LazyColumn. Paginate: load latest 50, load older on scroll-up.
📡
Network Errors
Keep partial streamed response. Show "Connection lost. Tap to retry." Never discard received tokens.
🛡️
Content Filtering
Non-negotiable. Use model safety settings + keyword filters + graceful "I can't help with that" responses.

💰 Rate Limits at a Glance

Limit Value Friendly Message
Per-minute 10 messages "Slow down — I need a moment to think!"
Per-day (free) 50 messages "Today's messages used. Resets at midnight."
Per-conversation 100 messages "Getting long. Start a new conversation?"
Circuit breaker Cost > $X/hour "Chat temporarily unavailable." (+ internal alert)

20-message chat with Gemini Flash ≈ $0.001-0.005. At 10K DAU × 3 chats/day = $30-150/month. Budget before you ship.

📖 Key Terms

MessageRole
USER, ASSISTANT, or SYSTEM — tells the AI who said what in the conversation history.
MessageStatus
SENDING, STREAMING, COMPLETE, or ERROR — drives the UI state for each bubble.
startChat() / sendMessageStream()
Gemini SDK methods. startChat() creates a session that tracks history; sendMessageStream() returns tokens as a Flow.
Sliding Window + Summary
Memory strategy: anchor (first messages) + compressed summary of middle + recent N messages verbatim.
Function Calling
AI decides to call your app's functions (search, get details) instead of just generating text. You execute locally, return results.
Action Chips
Tappable suggestion buttons after AI responses ("Show more", "Something easier"). Guide conversation, reduce typing.
Scoped Assistant
A chatbot with clear domain, firm boundaries, defined personality, and format rules. Predictable and trustworthy.
Circuit Breaker
Global cost limit that disables chat when API spend exceeds a threshold. Set it before you need it.

Quick Quiz

When should you use a chatbot vs a one-shot AI feature?

Chatbot: when the interaction needs clarification, exploration, or multi-step reasoning. One-shot: when a single ask-and-answer solves the problem (summarization, classification, settings).

Why is "You are a helpful assistant" a bad system prompt?

No domain, no boundaries, no personality, no format guidance. The chatbot will do anything — including going off-topic, being verbose, and hallucinating. A scoped prompt with domain + boundaries + personality + format produces consistent results.

How does function calling bridge conversation and app actions?

You define functions (searchRecipes, getDetails) with descriptions. The AI decides when to call them. Your code executes them locally against real data, returns results to the AI, and the AI formats a natural-language response with rich cards.

What's the sliding window + summary strategy for long conversations?

Keep the system prompt + first message as anchor, compress the middle messages into a summary, and include the last N messages verbatim. This preserves the full conversation arc without sending hundreds of messages per request.

Why is chat the most expensive AI feature to run?

Each message sends the entire conversation history as input tokens. Cost grows linearly with conversation length. Talkative users can generate hundreds of API calls per session. Rate limits and circuit breakers are essential.

Key Takeaways

  • Only build a chatbot when the problem needs multi-turn conversation — buttons and one-shot features are better for everything else.
  • The Scoped Assistant Pattern (domain + boundaries + personality + format) is the difference between a useful chatbot and a chaotic one.
  • Stream responses token-by-token — a 3-second spinner feels dead, streaming text feels alive.
  • Function calling lets your chatbot do things, not just talk — search databases, navigate screens, trigger actions.
  • Conversation memory needs a strategy: full history for short chats, sliding window + summary for long ones.
  • Chat is the most expensive AI feature. Set rate limits and a circuit breaker before you ship, not after you get the bill.
