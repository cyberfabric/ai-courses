Chapter 3

Designing AI UX That Users Trust

Memory Map
💡

The Big Idea

Technical implementation is 40% of a great AI feature. The other 60% is UX. Trust is earned by being right consistently, transparent when unsure, and honest when wrong. "I'm not sure" is always better than a confident hallucination.

📊 The Trust Equation

1️⃣
1 Bad Hallucination
Can destroy trust in a feature forever.
🔟
5-10 Correct Results
Needed to build initial trust with a user.
🚪
3 Bad Experiences
73% of users abandon after 3 failures.

Trust is like meeting a new colleague — they earn it by being right, transparent, and honest.

⏳ Three-Phase Loading

🟡 Initiating
"Analyzing your photo..."
→
🔵 Processing
Streaming text / skeleton / steps
→
🟢 Completing
Smooth fade-in of result

  1. ❌ Bad: Bare spinner

    User thinks the app is frozen. No context, no expectations.

  2. ⚠️ Better: "Loading..."

    Something is happening, but what?

  3. ✅ Good: "Analyzing your receipt..."

    User knows exactly what the AI is doing.

  4. 🌟 Great: Step-by-step progress

    "Reading text... Found merchant... Extracting total..." User can see it working.

🎚️ Confidence → Behavior (Not Numbers)

Confidence ❌ Don't Show ✅ Do Show
High (>90%) "Receipt (93.2% confidence)" "Receipt detected" — just show the result
Medium (70-90%) "Receipt (74.1% confidence)" "This looks like a receipt" — softer language
Low (50-70%) "Receipt (58.7% confidence)" "Is this a receipt?" — ask user to confirm
Very Low (<50%) "Unknown (32.4% confidence)" Show nothing or ask for clarification

🤝 The Suggestion-Confirmation Pattern

🤖 AI Suggests
Labeled, non-destructive
→
👁️ User Reviews
See, edit, or dismiss
→
✅ User Confirms
Explicit action to accept

AI should suggest, not act. The user confirms, the app executes.

🎮 Five Escape Hatches (Every Feature Needs All Five)

👋
Dismiss
Swipe away, tap outside, "not now." Low-friction rejection.
✏️
Edit
Modify the AI's suggestion before accepting it.
↩️
Undo
Reverse an AI action after accepting it.
👎
Feedback
"Not helpful" / thumbs down / "wrong answer."
🚫
Disable
Turn off AI suggestions for this feature entirely.

⚠️ The AI Error Taxonomy

Error Type What Happened What to Show
🔌 Infrastructure API down, timeout "AI temporarily unavailable. Try again."
🤷 Low confidence AI can't determine answer "Here are some possibilities..." or nothing
🛡️ Content filtered Safety filter triggered "I can't help with that." (no details)
⏱️ Rate limited Usage cap hit "Reached today's limit. Resets midnight."
💀 Wrong result Confidently incorrect Needs feedback mechanisms to catch

Implicit feedback (what users do) is more powerful than explicit feedback (what they click). Log it from day one.

🧅 Progressive Disclosure (Three Layers)

  1. Layer 1: Simple Action

    One-tap "Summarize" button, autocomplete, smart sort. Zero config, instant value. 85% of users stay here forever.

  2. Layer 2: Customization

    "Summarize as bullets / paragraph / key takeaways." Revealed after initial trust is built.

  3. Layer 3: Power User

    Prompt customization, model selection, advanced settings. Behind a gear icon for the 15% who want control.

🚫 Dark Patterns to Avoid

⏰
Fake Urgency
"AI detected this deal expires soon!" when AI didn't analyze anything.
🎭
Manufactured Authority
"Our AI recommends this" when it's actually a paid placement.
🪤
Engagement Traps
Artificially low free limits designed to frustrate, not reflect real costs.
👻
Invisible AI
AI filtering content without telling the user. They should know and be able to switch.

The journalist test: would you be embarrassed if someone wrote about how your AI feature works?

📖 Key Terms

Three-Phase Loading
Initiating ("Analyzing...") → Processing (streaming/skeleton) → Completing (smooth reveal). Never a bare spinner.
Confidence Tiers
Map model confidence to UX behavior (show result / soften language / ask user / show nothing). Never expose raw percentages.
AI Suggested Badge
The ✨ AutoAwesome sparkle icon labeling AI-generated content. Universal convention users already understand.
Suggestion-Confirmation
AI suggests → User reviews → User confirms. AI never auto-applies without explicit user action.
Implicit Feedback
Signals from user behavior (accept/dismiss, edit distance, re-queries) that reveal AI quality without asking.
Progressive Disclosure
Layer 1 (simple) → Layer 2 (customization) → Layer 3 (power user). Don't overwhelm on day one.
LiveRegion
Compose semantics property that tells screen readers about dynamic AI content. Use Polite for results, Assertive for errors.
Edit Distance
How much the user changed AI output. High edit distance = AI wasn't helpful enough.

🧪 Quick Quiz

Test Your Understanding

Why should you never show raw confidence scores to users?

Users don't have a mental model for what "67% confidence" means. It creates anxiety and undermines trust. Instead, translate confidence into behavior: show the result, soften the language, ask for confirmation, or show nothing.

What are the three phases of AI loading states?

Initiating ("Analyzing your photo..."), Processing (streaming text, skeleton screens, step indicators), and Completing (smooth fade-in/slide-up of the result). Each phase tells the user what's happening.

Why is "I'm not sure" better than a confident wrong answer?

A confident hallucination damages trust far more than an honest "I don't know." One bad hallucination can destroy trust forever, while showing uncertainty preserves credibility and gives the user a chance to provide more context.

What's the difference between explicit and implicit feedback?

Explicit: thumbs up/down, "was this helpful?" — requires user action. Implicit: accept vs dismiss, edit distance, re-queries, time to action — gathered from natural behavior. Implicit is more powerful and less annoying.

Name the five escape hatches every AI feature needs.

Dismiss (swipe away), Edit (modify before accepting), Undo (reverse after accepting), Feedback (thumbs down), Disable (turn off entirely). The user must always feel in control.

Key Takeaways

  • Trust is earned: 5-10 correct results to build, 1 bad hallucination to destroy.
  • Never show a bare spinner — use three-phase loading with contextual messages.
  • Translate confidence into behavior (language, options, silence) — not raw numbers.
  • AI suggests, user confirms — never auto-apply AI results without explicit user action.
  • Every AI feature needs five escape hatches: dismiss, edit, undo, feedback, disable.
  • "I'm not sure" is always better than a confident hallucination.
  • Log implicit feedback signals from day one — they're gold for improving quality later.
  • Progressive disclosure: start simple (Layer 1), reveal complexity only as trust builds.
