Chapter 8

Smart Camera & Real-Time Processing

Live object detection, document scanning, barcode reading, and AR overlays — AI meets the camera feed

From Photos to Live Video

In Chapter 7, you built features that analyze photos — images the user already has. This chapter is about analyzing what the camera sees right now. Live object detection overlays, real-time document scanning, barcode reading, and AR-style annotations.

This is fundamentally different from photo analysis. With a photo, you have all the time in the world. With a live camera, you have roughly 50 milliseconds per frame to analyze, process, and draw results before the next frame arrives. Cloud APIs are too slow. This is on-device AI territory.

20fps Minimum frame rate for smooth camera AI overlays
<50ms Max processing time per frame to maintain 20fps
0 Network calls allowed — everything must run on-device

Recipe: CameraX + ML Kit Object Detection Pipeline

Goal: Show real-time bounding boxes around detected objects in the camera feed.

Step 1: Add dependencies:

implementation("androidx.camera:camera-camera2:1.4.1")
implementation("androidx.camera:camera-lifecycle:1.4.1")
implementation("androidx.camera:camera-view:1.4.1")
implementation("com.google.mlkit:object-detection:17.0.2")

Step 2: Set up CameraX with ImageAnalysis:

fun startCamera(lifecycleOwner: LifecycleOwner, previewView: PreviewView) {
    val cameraProviderFuture = ProcessCameraProvider.getInstance(context)
    cameraProviderFuture.addListener({
        val provider = cameraProviderFuture.get()

        val preview = Preview.Builder().build().also {
            it.surfaceProvider = previewView.surfaceProvider
        }

        val analysis = ImageAnalysis.Builder()
            .setBackpressureStrategy(ImageAnalysis.STRATEGY_KEEP_ONLY_LATEST)
            .build()
            .also { it.setAnalyzer(analysisExecutor, objectAnalyzer) }

        provider.unbindAll()
        provider.bindToLifecycle(lifecycleOwner, CameraSelector.DEFAULT_BACK_CAMERA,
            preview, analysis)
    }, ContextCompat.getMainExecutor(context))
}

Step 3: Create the analyzer that processes each frame:

class ObjectAnalyzer(
    private val onResults: (List<DetectedObject>) -> Unit
) : ImageAnalysis.Analyzer {
    private val detector = ObjectDetection.getClient(
        ObjectDetectorOptions.Builder()
            .setDetectorMode(ObjectDetectorOptions.STREAM_MODE)
            .enableMultipleObjects()
            .enableClassification()
            .build()
    )

    @OptIn(ExperimentalGetImage::class)
    override fun analyze(imageProxy: ImageProxy) {
        val mediaImage = imageProxy.image ?: return imageProxy.close()
        val input = InputImage.fromMediaImage(mediaImage, imageProxy.imageInfo.rotationDegrees)
        detector.process(input)
            .addOnSuccessListener { objects -> onResults(objects) }
            .addOnCompleteListener { imageProxy.close() } // always close!
    }
}

Step 4: Draw bounding boxes on a Compose Canvas overlay:

@Composable
fun DetectionOverlay(objects: List<DetectedObject>, imageSize: Size, viewSize: Size) {
    Canvas(modifier = Modifier.fillMaxSize()) {
        val scaleX = viewSize.width / imageSize.width
        val scaleY = viewSize.height / imageSize.height
        objects.forEach { obj ->
            val rect = obj.boundingBox
            drawRect(
                color = Color.Green,
                topLeft = Offset(rect.left * scaleX, rect.top * scaleY),
                size = Size(rect.width() * scaleX, rect.height() * scaleY),
                style = Stroke(width = 3.dp.toPx())
            )
        }
    }
}

Stack the DetectionOverlay on top of the PreviewView in a Box, and you have real-time bounding boxes tracking objects as the user moves the camera.

Always close the ImageProxy. If you forget imageProxy.close(), the camera pipeline stalls and no more frames arrive. This is the #1 bug in CameraX + ML pipelines. Close it in addOnCompleteListener, not addOnSuccessListener, so it's closed even on failure.

Live Object Detection with MediaPipe

ML Kit Object Detection is great for general objects. For more specialized detection (specific product types, custom objects), MediaPipe gives you more control and supports custom models.

MP
MediaPipe Object Detector
Pre-built models for 80+ object categories (COCO dataset), or bring your own TFLite model. Optimized for real-time mobile inference with GPU acceleration.
MP
MediaPipe Hand/Pose/Face
Specialized models for body tracking: hand landmarks (21 points), body pose (33 points), face mesh (468 points). Perfect for gesture recognition and AR features.

MediaPipe models are typically faster than ML Kit for specialized tasks because they're optimized for a specific detection type rather than being general-purpose.

Document Scanning with AI

Document scanning is one of the most practical camera AI features. The user points the camera at a document, the app detects the edges in real time, and on tap it crops, dewarps, and extracts text.

Scenario: Sofia is an accountant who photographs receipts for expense reports. She opens the app, points her camera at a crumpled receipt on a café table. The camera feed shows a green outline snapping to the receipt's edges in real time. She taps the capture button. The app instantly crops, straightens, and enhances the image. Then ML Kit extracts the text, and a Gemini API call parses it into structured data: "Café Rosetta, March 28, 2026, Total: €14.50."

From photo to structured expense entry in under 3 seconds.

The Smart Scanner Pattern

A great document scanner has four stages, each providing visual feedback:

  1. Edge detection (live): As the user moves the camera, show a colored outline around the detected document edges. Green = good alignment, yellow = partial detection, no outline = no document found.
  2. Auto-capture or manual tap: Either capture automatically when edges are stable for 1 second, or let the user tap. Auto-capture feels more magical; manual tap gives more control.
  3. Perspective correction: Dewarp the image so a photo taken at an angle becomes a flat, rectangular document.
  4. Enhancement + OCR: Increase contrast, sharpen text, then extract and structure the content.

Real-world example: Google Drive's document scanner shows real-time edge detection with a blue overlay, auto-captures when stable, and produces a clean PDF. Adobe Scan adds AI-powered text cleanup.

Edge Detection with ML Kit

// Google's Document Scanner API handles the entire flow
val scanner = GmsDocumentScanning.getClient(
    GmsDocumentScannerOptions.Builder()
        .setGalleryImportAllowed(true)
        .setPageLimit(5)
        .setResultFormats(RESULT_FORMAT_JPEG, RESULT_FORMAT_PDF)
        .setScannerMode(SCANNER_MODE_FULL) // includes edge detection UI
        .build()
)

// Launch the scanner
val scannerLauncher = rememberLauncherForActivityResult(
    ActivityResultContracts.StartIntentSenderForResult()
) { result ->
    if (result.resultCode == RESULT_OK) {
        val scanResult = GmsDocumentScanningResult.fromActivityResultIntent(result.data)
        scanResult?.pages?.forEach { page ->
            val imageUri = page.imageUri
            // Now extract text with ML Kit or analyze with Gemini
            processScannedDocument(imageUri)
        }
    }
}

// Trigger scanning
scanner.getStartScanIntent(activity).addOnSuccessListener { intent ->
    scannerLauncher.launch(IntentSenderRequest.Builder(intent).build())
}

Don't reinvent the scanner. Google's Document Scanner API (via Play Services) provides the complete scanning experience: edge detection, perspective correction, image enhancement, and multi-page support. It's free and works on devices with Google Play Services. Build your own only if you need deep customization.

Barcode and QR Scanning

Barcode scanning is the most mature camera AI feature on Android. ML Kit handles it flawlessly, and the interesting part is what you do with the scanned content.

val barcodeScanner = BarcodeScanning.getClient(
    BarcodeScannerOptions.Builder()
        .setBarcodeFormats(Barcode.FORMAT_QR_CODE, Barcode.FORMAT_EAN_13)
        .build()
)

// In your ImageAnalysis.Analyzer
barcodeScanner.process(inputImage).addOnSuccessListener { barcodes ->
    barcodes.forEach { barcode ->
        when (barcode.valueType) {
            Barcode.TYPE_URL -> openUrl(barcode.url!!.url!!)
            Barcode.TYPE_WIFI -> connectToWifi(barcode.wifi!!)
            Barcode.TYPE_CONTACT_INFO -> addContact(barcode.contactInfo!!)
            Barcode.TYPE_PRODUCT -> searchProduct(barcode.rawValue!!)
            else -> showRawValue(barcode.rawValue!!)
        }
    }
}

The power move: contextual actions. Don't just show the barcode content — act on it. A URL gets an "Open" button. A Wi-Fi code gets "Connect." A product barcode gets "View product info." Each content type triggers a different, useful action.

AR-Style Annotations

Drawing labels and information anchored to detected objects in the camera feed creates an AR-lite experience without the complexity of ARCore.

@Composable
fun AnnotatedOverlay(detections: List<Detection>, imageSize: Size, viewSize: Size) {
    Canvas(modifier = Modifier.fillMaxSize()) {
        detections.forEach { detection ->
            val rect = scaleRect(detection.boundingBox, imageSize, viewSize)

            // Draw bounding box
            drawRoundRect(color = Color(0xFF2563EB), topLeft = Offset(rect.left, rect.top),
                size = Size(rect.width(), rect.height()),
                cornerRadius = CornerRadius(8f), style = Stroke(3.dp.toPx()))

            // Draw label background
            val labelHeight = 24.dp.toPx()
            drawRoundRect(color = Color(0xFF2563EB),
                topLeft = Offset(rect.left, rect.top - labelHeight),
                size = Size(rect.width(), labelHeight),
                cornerRadius = CornerRadius(8f, 8f, 0f, 0f))
        }
    }
    // Draw text labels using Compose Text (Canvas drawText is limited)
    detections.forEach { detection ->
        val rect = scaleRect(detection.boundingBox, imageSize, viewSize)
        Text(detection.label,
            modifier = Modifier.offset(x = rect.left.dp, y = (rect.top - 24).dp),
            color = Color.White, fontSize = 12.sp, fontWeight = FontWeight.Bold)
    }
}

The key challenge is coordinate transformation: the ML model works in image coordinates (e.g., 640x480), but the screen is a different size and aspect ratio. Always scale the bounding boxes to match the preview view size.

Frame Rate Management

Not every camera frame needs AI analysis. Smart frame management keeps the experience smooth without overloading the CPU.

Strategy 1: Analyze Every Nth Frame

class ThrottledAnalyzer(
    private val innerAnalyzer: ImageAnalysis.Analyzer,
    private val skipFrames: Int = 2 // analyze every 3rd frame
) : ImageAnalysis.Analyzer {
    private var frameCount = 0

    override fun analyze(imageProxy: ImageProxy) {
        if (frameCount++ % (skipFrames + 1) == 0) {
            innerAnalyzer.analyze(imageProxy)
        } else {
            imageProxy.close() // skip this frame
        }
    }
}

Strategy 2: Async Pipeline

Process frame N on a background thread while the camera captures frame N+1. Use STRATEGY_KEEP_ONLY_LATEST (which we already set) so old frames are automatically dropped if analysis is slower than capture.

Strategy 3: Adaptive Rate

Analyze more frequently when results are changing (camera moving, new objects appearing) and less frequently when stable (camera stationary, same objects). Measure the difference between consecutive results to decide.

Start with "every 3rd frame." Most ML Kit detectors run in 30-50ms, and the camera captures at 30fps (33ms per frame). Analyzing every 3rd frame gives you ~100ms per analysis while keeping the preview smooth at 30fps. Adjust based on your model's speed.

Camera Permission UX

A camera feature is useless without camera permission. How you ask for it determines whether users grant it.

Cold Permission Request

App launches → immediately shows system dialog: "Allow access to camera?" User thinks: Why? What will it do? Is this safe? → taps Deny

Contextual Permission Request

User taps "Scan Receipt" → app shows a friendly screen: "To scan receipts, we need your camera. Your photos are processed on-device and never uploaded." → then the system dialog User thinks: OK, that makes sense. → taps Allow

Always explain why before asking, and mention privacy ("processed on-device") if applicable. If the user denies permanently, show a helpful message with a button to open Settings — never nag.

Memory and Thermal Management

Continuous camera + ML inference is the most hardware-intensive feature you can build on Android. It drains the battery and heats up the phone.

  • Monitor thermal state: Use PowerManager.getThermalStatus() and reduce analysis frequency when the device is warm
  • Release camera on background: Unbind CameraX in onStop() — never run camera in the background
  • Use GPU delegation: MediaPipe and TFLite both support GPU acceleration, which is more power-efficient than CPU for ML inference
  • Limit resolution: Analyze at 640x480 or lower. Higher resolution doesn't improve most detection tasks but costs more battery
  • Show battery impact: If the feature will be used for extended periods, consider a subtle battery indicator

Thermal throttling is real. After 2-3 minutes of continuous camera AI, the phone may throttle CPU speed to cool down. Your inference time doubles, frame rate drops, and the experience degrades. Monitor and adapt — reduce analysis frequency before the user notices the slowdown.

Testing Camera Features

You can't hold a phone in front of a real document during automated tests. Here's how to test without a physical camera:

Test Type Approach What It Validates
Unit tests Pass static Bitmap images to your analyzer Detection logic, coordinate scaling, result filtering
Integration tests Feed pre-recorded video frames to ImageAnalysis Full pipeline from frame to detection to overlay
Screenshot tests Render overlay on a fixed image and compare Bounding box positioning, label rendering, styling
Manual QA Print test images and hold them in front of the camera Real-world conditions: lighting, angles, distance

The Test Image Library

Build a set of 20-30 test images that cover your feature's use cases: clean documents, crumpled receipts, barcodes at angles, low-light photos, blurry images. Use these consistently across unit and integration tests. When a user reports a bug, add their scenario to the library. Your test coverage grows with real-world edge cases.
