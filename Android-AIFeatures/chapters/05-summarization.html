Chapter 5

Summarization & Content Generation

TL;DR buttons, smart drafts, streaming text, and prompts that actually work

From Finding to Creating

In Chapter 4, you built search that finds content. Now you're going to build features that create it. Summarization and content generation are the most common generative AI features in apps today — and the most likely to impress users or embarrass you, depending on how you implement them.

Think of it as the difference between a librarian and a writer. Search is the librarian: "Here's the book you wanted." Generation is the writer: "Here's a summary of that book in three bullet points." Both are useful, but the writer has a much harder job — and much more room for error.

Summarization Use Cases

Summarization is everywhere in great apps. Here are the patterns you'll see (and build):

Use Case Input Output Example App
Article TL;DR Long article (1,000+ words) 3 bullet points News reader, RSS app
Email thread summary 10-message email chain Key decisions + action items Email client
Meeting notes Transcript or raw notes Structured agenda + takeaways Productivity app
Review highlights 50 user reviews "Users love X, complain about Y" Shopping, restaurant app
Notification batching 12 unread messages "5 from Anna about the deadline" Messaging app

Scenario: Karim opens his note-taking app after a long meeting. He has three pages of messy, stream-of-consciousness notes. He taps the sparkle icon and selects "Summarize." Two seconds later, a clean card appears with five bullet points: three key decisions, two action items with owners. Karim copies the action items to his task manager and moves on with his day.

Without AI, Karim would spend 10 minutes re-reading and reformatting. With AI, it took 2 seconds plus a quick review.

The Summarization Stack

You have three options, and they map directly to the decision framework from Chapter 2:

Nano Gemini Nano (on-device) — fast, free, limited quality, flagship devices only
Flash Gemini Flash (cloud) — fast, cheap, good quality for most tasks
Pro Gemini Pro / GPT-4o (cloud) — slower, expensive, highest quality

For most summarization features, Gemini Flash hits the sweet spot: fast enough for interactive use (1-2 seconds), cheap enough for scale, and good enough quality for bullet-point summaries. Use Pro/GPT-4o only for complex summarization (multi-document, nuanced analysis).

Building a "TL;DR" Button

Let's build the most common summarization feature: a button that condenses long text into a digestible summary.

Recipe: The Complete TL;DR Feature

Goal: User taps "Summarize," sees a streaming 3-bullet summary with an "AI Summary" badge.

Step 1: Create the summarization ViewModel:

class SummarizeViewModel(
    private val model: GenerativeModel
) : ViewModel() {
    private val _uiState = MutableStateFlow<SummaryState>(SummaryState.Idle)
    val uiState: StateFlow<SummaryState> = _uiState

    fun summarize(text: String) {
        viewModelScope.launch {
            _uiState.value = SummaryState.Loading("Reading the content...")
            try {
                val prompt = buildSummaryPrompt(text)
                var result = ""
                model.generateContentStream(prompt).collect { chunk ->
                    result += chunk.text.orEmpty()
                    _uiState.value = SummaryState.Streaming(result)
                }
                _uiState.value = SummaryState.Complete(result)
            } catch (e: Exception) {
                _uiState.value = SummaryState.Error("Couldn't summarize. Tap to retry.")
            }
        }
    }
}

Step 2: Craft the prompt (this matters more than you think):

fun buildSummaryPrompt(text: String): String = """
    Summarize the following text in exactly 3 bullet points.
    Each bullet should be one concise sentence.
    Focus on key decisions and action items.
    If there are no action items, focus on the main ideas.

    Text to summarize:
    $text
""".trimIndent()

Step 3: Build the Compose UI with all states:

@Composable
fun SummaryCard(state: SummaryState, onRetry: () -> Unit) {
    AnimatedVisibility(visible = state != SummaryState.Idle) {
        Card(modifier = Modifier.fillMaxWidth().padding(16.dp)) {
            Column(modifier = Modifier.padding(16.dp)) {
                // AI badge
                Row(verticalAlignment = Alignment.CenterVertically) {
                    Icon(Icons.Outlined.AutoAwesome, null, Modifier.size(16.dp))
                    Spacer(Modifier.width(4.dp))
                    Text("AI Summary", style = MaterialTheme.typography.labelSmall)
                }
                Spacer(Modifier.height(8.dp))
                when (state) {
                    is SummaryState.Loading ->
                        Text(state.message, color = MaterialTheme.colorScheme.outline)
                    is SummaryState.Streaming ->
                        Text(state.text, style = MaterialTheme.typography.bodyMedium)
                    is SummaryState.Complete ->
                        Text(state.text, style = MaterialTheme.typography.bodyMedium)
                    is SummaryState.Error ->
                        TextButton(onClick = onRetry) { Text(state.message) }
                    else -> {}
                }
            }
        }
    }
}

The result: a smooth card that slides in, streams text as it arrives, shows the "AI Summary" badge, and handles errors gracefully.

Prompt Engineering for Summarization

The prompt is the most underrated part of any generative AI feature. The difference between a good prompt and a bad one is the difference between "useful summary" and "useless wall of text."

Vague Prompt

Summarize this.

No format guidance, no length constraint, no focus. You'll get a different style every time — sometimes a paragraph, sometimes 10 bullets, sometimes a rewrite of the original.

Structured Prompt

Summarize the following text in exactly 3 bullet points. Each bullet should be one concise sentence (max 20 words). Focus on action items and decisions. If there are no action items, highlight the main conclusions.

Clear format (3 bullets), length constraint (one sentence each), focus area (action items), and a fallback instruction. Consistent output every time.

The Prompt Engineering Checklist

  • Format: Specify the output structure (bullets, paragraph, numbered list, JSON)
  • Length: Set explicit constraints (3 bullets, 2 sentences, max 50 words)
  • Focus: Tell the model what to prioritize (decisions, facts, opinions, action items)
  • Tone: Match your app's voice (professional, casual, technical)
  • Edge cases: What should the model do when input is too short, off-topic, or empty?

Test prompts with real data, not samples. Your prompt might work perfectly on a clean Wikipedia article but fall apart on a user's messy notes with abbreviations, typos, and half-sentences. Test with the ugliest real input you can find.

Content Generation: The "AI Draft" Pattern

Summarization takes long content and makes it short. Content generation does the opposite: takes a small input and creates something bigger.

The AI Draft Pattern

The safest and most useful content generation pattern: AI creates a starting point, the user finishes it.

  1. User provides context: "Reply to this email" / "Write a product description for this" / "Draft meeting notes"
  2. AI generates a draft: A full text that's 70-80% of the way there
  3. User edits: Tweaks wording, adds personal touches, removes anything off-base
  4. User sends/saves: The final version is the user's, not the AI's

Why this works: Editing is easier than writing from scratch. Users feel ownership of the final result. And if the AI gets something wrong, the user catches it during the editing step.

Real-world example: Gmail's "Help me write" feature generates a full email draft based on a short prompt. It always drops the user into the editor, never sends automatically. The AI is a writing assistant, not a ghost writer.

Template-Based Generation

For structured content (product descriptions, event listings, form letters), combine AI generation with templates. This constrains the AI's output for consistency:

fun buildProductDescriptionPrompt(
    name: String, category: String, features: List<String>
): String = """
    Write a product description for an e-commerce app.

    Product: $name
    Category: $category
    Key features: ${features.joinToString(", ")}

    Format:
    - Opening hook (1 sentence, under 15 words)
    - 3 feature highlights (1 sentence each)
    - Call to action (1 sentence)

    Tone: friendly, enthusiastic, not salesy.
    Total length: under 80 words.
""".trimIndent()

The template constrains the structure (hook, features, CTA) while the AI fills in the creative content. Every product description will follow the same format, making the feature predictable and consistent.

Token Limits and Chunking

Every AI model has a context window — a maximum amount of text it can process at once. When your user's document exceeds that limit, you need a chunking strategy.

Gemini Flash: 1M token context window (roughly 750K words — rarely a problem). GPT-4o-mini: 128K tokens. On-device models: Often just 4K-8K tokens. Know your model's limit before you build.

For documents that exceed the context window, use the map-reduce pattern:

Split into chunks
→
Summarize each chunk
→
Summarize the summaries
suspend fun summarizeLongDocument(text: String, model: GenerativeModel): String {
    val chunks = text.chunked(maxChunkSize) // split by paragraphs or sentences

    // Map: summarize each chunk independently
    val chunkSummaries = chunks.map { chunk ->
        val response = model.generateContent(
            "Summarize this section in 2 sentences:\n$chunk"
        )
        response.text.orEmpty()
    }

    // Reduce: summarize the summaries
    val combined = chunkSummaries.joinToString("\n")
    val finalResponse = model.generateContent(
        "Combine these section summaries into 3 key bullet points:\n$combined"
    )
    return finalResponse.text.orEmpty()
}

Smart chunking matters. Don't split mid-sentence or mid-paragraph. Split at natural boundaries: paragraph breaks, section headers, or sentence endings. A chunk that starts with "...continued from above" will confuse the model.

Content Quality Control

Generated content can be wrong, awkward, or off-brand. You need quality gates:

1. Hallucination Detection

If your summary mentions facts not in the original text, that's a hallucination. For critical features (medical, financial, legal), add a verification step:

suspend fun verifySummary(original: String, summary: String): Boolean {
    val response = model.generateContent("""
        Does this summary contain any claims not supported by the original text?
        Original: $original
        Summary: $summary
        Answer only YES or NO.
    """.trimIndent())
    return response.text?.trim()?.uppercase() == "NO"
}

2. The "Regenerate" Button

Sometimes the output just isn't great. Always offer a "Regenerate" option — it's cheap (one more API call) and gives the user control. Each regeneration should produce a different result, thanks to temperature settings.

3. Source Attribution

If the summary is based on a specific document, link back to the source. "Based on your meeting notes from March 28." This helps users verify and builds trust.

Caching Generated Content

Don't regenerate the same summary every time the user views it. Cache aggressively:

@Entity
data class CachedSummary(
    @PrimaryKey val sourceId: String,     // ID of the original content
    val summary: String,                   // the generated summary
    val sourceHash: String,                // hash of original content
    val generatedAt: Long                  // timestamp
)

// Check cache before generating
suspend fun getSummary(sourceId: String, content: String): String {
    val hash = content.hashCode().toString()
    val cached = dao.getSummary(sourceId)
    if (cached != null && cached.sourceHash == hash) {
        return cached.summary // content hasn't changed, use cache
    }
    // Content changed or no cache — generate fresh
    val summary = generateSummary(content)
    dao.upsert(CachedSummary(sourceId, summary, hash, System.currentTimeMillis()))
    return summary
}

The sourceHash is key: if the original content changes, invalidate the cache and regenerate. If it hasn't changed, serve the cached summary instantly.

Cost Management at Scale

Generative AI features can get expensive fast. Here's how to keep costs under control:

Strategy Savings Tradeoff
Cache summaries 60-80% fewer API calls Stale results if source changes
Use smaller models for simple tasks 5-10x cheaper per request Lower quality for complex content
Batch requests Lower overhead per request Slightly delayed results
Set per-user daily limits Predictable monthly costs Power users hit limits
Shorter prompts Fewer input tokens = lower cost Less context for the model

The Cost Formula You Need to Memorize

Monthly cost = DAU × summaries_per_user_per_day × (input_tokens + output_tokens) × price_per_token × 30

Run this before you ship. Then run it again for 10x your current DAU. If it's scary, add caching and limits before launch, not after you get the bill.

Gemini Nano vs. Cloud: A Practical Comparison

For summarization specifically, here's how on-device and cloud compare:

Gemini Nano (On-Device)

Speed: ~500ms for a short summary Cost: $0 Quality: Good for short text (under 500 words) Availability: Pixel 8+, Samsung S24+ only Offline: Yes

Great for quick summaries, notification previews, and short notes. Falls short on long or complex documents.

Gemini Flash (Cloud)

Speed: 1-2 seconds Cost: ~$0.0001 per summary Quality: Excellent for most content Availability: Any device with internet Offline: No

The workhorse for summarization. High quality, fast enough for interactive use, cheap enough for scale.

The smart approach: try Gemini Nano first (instant, free). If the device doesn't support it or the text is too long, fall back to Gemini Flash.

suspend fun smartSummarize(text: String): String {
    // Try on-device first (free, instant)
    if (isGeminiNanoAvailable && text.length < 2000) {
        return geminiNano.summarize(text)
    }
    // Fall back to cloud (better quality, costs money)
    return geminiFlash.summarize(text)
}
