Chapter 4

Vision Framework — See Like an iPhone

Text recognition, face detection, body pose, document scanning, and barcode reading — no custom models required

Apple Already Trained the Models for You

In Chapter 3 you converted a model, dragged it into Xcode, and wired up a camera pipeline. That's powerful — but it's also work. What if Apple already has a production-ready model for your task?

That's exactly what the Vision framework is. It's a collection of pre-trained computer vision models that Apple maintains, optimizes for each chip generation, and updates with every iOS release. You don't download model files, manage versions, or worry about Neural Engine compatibility. You create a request, run it, and read the results.

Vision Framework

Available since: iOS 11. Significantly expanded in iOS 14 (body pose), iOS 15 (live text), iOS 16 (person segmentation), iOS 18 (Swift concurrency API), and iOS 26 (RecognizeDocumentsRequest, lens smudge detection).

No model files needed. Apple bundles the models in the OS and optimizes them per device.

Key pattern: Create a request → Create a handler → Perform → Read observations.

The Vision Request Pattern

Every Vision feature follows the same three-line pattern you already know from Chapter 3:

// 1. Create the request
let request = VNRecognizeTextRequest { request, error in
    let results = request.results as? [VNRecognizedTextObservation]
    // Process results
}

// 2. Create the handler with your image
let handler = VNImageRequestHandler(cgImage: image, orientation: .up)

// 3. Perform
try handler.perform([request])

Change the request type, and you change what Vision looks for. That's the beauty of the framework — one pattern, dozens of capabilities.

Text Recognition (OCR)

Let's start with the most practical Vision feature: reading text from images. This is what powers Live Text across iOS.

func recognizeText(in image: CGImage) async throws -> [String] {
    let request = VNRecognizeTextRequest()
    request.recognitionLevel = .accurate  // .fast for real-time
    request.recognitionLanguages = ["en", "de", "fr"]
    request.usesLanguageCorrection = true

    let handler = VNImageRequestHandler(cgImage: image)
    try handler.perform([request])

    return request.results?.compactMap { observation in
        observation.topCandidates(1).first?.string
    } ?? []
}

That's a complete OCR implementation. Let's break down the key options:

Property Options When to Use
recognitionLevel .accurate / .fast .accurate for static images (receipts, documents). .fast for live camera feed.
recognitionLanguages Array of language codes Supports 18+ languages. Order matters — put the most likely language first.
usesLanguageCorrection Bool Improves accuracy using language models. Adds ~10ms latency.
Building a receipt scanner? Use .accurate mode with language correction on. Each VNRecognizedTextObservation includes bounding box coordinates (boundingBox), so you can map text to physical locations on the receipt — the total is usually at the bottom, the merchant name at the top.

RecognizeDocumentsRequest (iOS 26)

iOS 26 introduces RecognizeDocumentsRequest, a major upgrade over basic text recognition. Instead of returning flat text lines, it understands document structure:

import Vision

let request = RecognizeDocumentsRequest()
let handler = VNImageRequestHandler(cgImage: documentImage)
let results = try await handler.perform(request)

for document in results {
    // Structured elements — not just flat text
    for block in document.blocks {
        switch block {
        case .paragraph(let paragraph):
            print("Paragraph: \(paragraph.text)")
        case .table(let table):
            for row in table.rows {
                print("Row: \(row.cells.map { $0.text })")
            }
        case .list(let list):
            for item in list.items {
                print("Item: \(item.text)")
            }
        }
    }
    // Detected important information
    for info in document.importantInformation {
        print("\(info.type): \(info.value)")  // email, phone, URL
    }
}

What RecognizeDocumentsRequest Detects

  • Paragraphs — Lines grouped into coherent text blocks
  • Tables — Row/column structure with individual cell content
  • Lists — Bulleted and numbered items
  • Machine-readable codes — QR codes and barcodes embedded in documents
  • Important information — Emails, phone numbers, URLs, dates automatically extracted

26 languages supported. Runs entirely on-device.

Image Classification

In Chapter 3 you brought your own model for classification. Vision also has a built-in classifier that recognizes hundreds of scene and object categories:

let request = VNClassifyImageRequest()

let handler = VNImageRequestHandler(cgImage: photo)
try handler.perform([request])

let top5 = request.results?.prefix(5) ?? []
for observation in top5 {
    print("\(observation.identifier): \(observation.confidence)")
}
// outdoor, beach, sky, water, tropical — with confidence scores

Apple's built-in taxonomy includes scenes (beach, forest, office), objects (dog, car, food), and activities (running, cooking). It's less precise than a custom model for your specific categories, but it's free and requires zero setup.

Face Detection and Landmarks

Vision provides two levels of face analysis:

// Level 1: Just find the faces
let faceDetect = VNDetectFaceRectanglesRequest()

// Level 2: Find faces AND 76 landmark points
let faceLandmarks = VNDetectFaceLandmarksRequest()

let handler = VNImageRequestHandler(cgImage: groupPhoto)
try handler.perform([faceLandmarks])

for face in faceLandmarks.results ?? [] {
    let box = face.boundingBox  // Normalized rect (0..1)
    let landmarks = face.landmarks

    // Access specific features
    let leftEye = landmarks?.leftEye?.normalizedPoints
    let rightEye = landmarks?.rightEye?.normalizedPoints
    let nose = landmarks?.nose?.normalizedPoints
    let outerLips = landmarks?.outerLips?.normalizedPoints
    // 76 points total across all facial features
}

Practical uses: profile photo cropping (detect face, center crop around it), attention tracking (are the user's eyes looking at the screen?), AR overlays (place effects on specific facial landmarks), and face-based unlock UX.

Face detection is not face recognition. Vision tells you where faces are and what they look like. It does not identify who the person is. For that, you'd need a face embedding model — and significant privacy considerations.

Body Pose Detection

Want to track someone's body? Vision detects 19 joint points in real time:

let request = VNDetectHumanBodyPoseRequest()

let handler = VNImageRequestHandler(cgImage: frame)
try handler.perform([request])

for pose in request.results ?? [] {
    // Get specific joints
    if let rightWrist = try? pose.recognizedPoint(.rightWrist),
       rightWrist.confidence > 0.5 {
        let position = rightWrist.location  // Normalized coordinates
        print("Right wrist at: \(position)")
    }

    // Get all recognized points
    let allPoints = try? pose.recognizedPoints(forGroupKey: .all)
}

The 19 joints include: head, neck, shoulders, elbows, wrists, hips, knees, and ankles for both sides, plus the root (center of hips). Each point has a location (normalized 0..1 coordinates) and a confidence score.

Person Segmentation

Vision can separate people from backgrounds at the pixel level:

let request = VNGeneratePersonSegmentationRequest()
request.qualityLevel = .balanced  // .fast, .balanced, or .accurate

let handler = VNImageRequestHandler(cgImage: selfie)
try handler.perform([request])

if let mask = request.results?.first?.pixelBuffer {
    // mask is a CVPixelBuffer where white = person, black = background
    // Use it to replace the background, apply effects, etc.
    let ciMask = CIImage(cvPixelBuffer: mask)
    // Composite: original image where mask is white, new background where black
}

Three quality levels give you the speed/accuracy tradeoff:

Quality Resolution Speed Use Case
.fast Low ~5 ms Real-time video effects
.balanced Medium ~20 ms Good for most apps
.accurate Full ~80 ms Portrait mode, static photos

Barcode and QR Scanning

Vision's barcode detection is fast and supports 16+ symbologies:

let request = VNDetectBarcodesRequest()
// Optionally restrict to specific types for faster detection:
// request.symbologies = [.qr, .ean13, .code128]

let handler = VNImageRequestHandler(cgImage: image)
try handler.perform([request])

for barcode in request.results ?? [] {
    print("Type: \(barcode.symbology)")        // .qr, .ean13, etc.
    print("Value: \(barcode.payloadStringValue ?? "")")
    print("Bounds: \(barcode.boundingBox)")     // Normalized rect
}

Supported symbologies include QR, EAN-13, EAN-8, UPC-E, Code 128, Code 39, Code 93, PDF417, Aztec, Data Matrix, ITF-14, and more. For most apps, leave symbologies unset and Vision will detect all types.

Restrict symbologies for speed. If your app only needs QR codes, set request.symbologies = [.qr]. Vision can skip checking for other formats, making detection noticeably faster on each frame.

Combining Multiple Requests

Here's one of Vision's best features: you can run multiple requests on the same image in a single pass. The handler shares preprocessing work across all requests:

let textRequest = VNRecognizeTextRequest { request, error in
    // Handle text results
}
let barcodeRequest = VNDetectBarcodesRequest { request, error in
    // Handle barcode results
}
let faceRequest = VNDetectFaceRectanglesRequest { request, error in
    // Handle face results
}

let handler = VNImageRequestHandler(cgImage: image)
try handler.perform([textRequest, barcodeRequest, faceRequest])

Vision preprocesses the image once and feeds it to all three models. This is significantly faster than creating separate handlers for each request.

All requests run on the Neural Engine. Combining text + barcode + face detection in a single pass takes ~15 ms total — well within a 30fps camera budget.

VNImageRequestHandler vs. VNSequenceRequestHandler

Vision offers two handler types for different scenarios:

VNImageRequestHandler
One image, one or more requests
Create a new handler for each image
Use for: photos, single frames, document scanning. The handler you've been using throughout this chapter.
VNSequenceRequestHandler
Sequence of related frames (video)
Reuse the same handler across frames
Use for: object tracking across video frames. The handler maintains state between calls to track movement.

Region of Interest

If you know where to look, tell Vision. The regionOfInterest property limits detection to a specific area of the image, making requests faster and more focused:

let textRequest = VNRecognizeTextRequest()
// Only look at the bottom third of the image (where totals usually are)
textRequest.regionOfInterest = CGRect(x: 0, y: 0, width: 1.0, height: 0.33)
// Note: Vision uses normalized coordinates (0..1), origin at bottom-left
Vision's coordinate system is flipped. Vision uses normalized coordinates (0.0 to 1.0) with the origin at the bottom-left. UIKit's origin is top-left. When converting bounding boxes to UIKit coordinates, flip the Y axis: uiKitY = 1.0 - visionY - visionHeight.
