💡 The Core Insight

💡

Convert, integrate, classify — in one afternoon.

A working on-device AI feature is just 4 components: a Core ML model, an AVCaptureSession for camera frames, VNCoreMLRequest for inference, and a SwiftUI overlay for results. Three threads, zero cloud calls, ~3ms per prediction on the Neural Engine.

🔧 The 8-Step Build

  1. Create Xcode Project

    iOS App, SwiftUI, physical iPhone as run target.

  2. Convert Model (or Download)

    coremltools: PyTorch → .mlpackage. Or grab MobileNetV2 from Apple's gallery.

  3. Load Model in Swift

    MLModelConfiguration + VNCoreMLModel wrapper.

  4. Build VNCoreMLRequest

    Completion handler receives VNClassificationObservation results, sorted by confidence.

  5. Set Up Camera

    AVCaptureSession + AVCaptureVideoDataOutput on a background queue.

  6. Display Camera Preview

    UIViewRepresentable bridging AVCaptureVideoPreviewLayer into SwiftUI.

  7. Wire Everything Together

    ContentView: camera.onFrame → classifier.classify → @Published results → SwiftUI Text.

  8. Add Threading & Throttling

    Dedicated inference queue + isProcessing guard to skip frames during slow inference.

🏗️ The 4 Components

📷
CameraManager
AVCaptureSession + VideoDataOutput. Captures frames on a dedicated serial queue. Calls onFrame closure with each CVPixelBuffer.
🧠
ImageClassifier
Loads VNCoreMLModel once. Runs VNCoreMLRequest on inference queue. Publishes topLabel and confidence via @Published.
📺
CameraPreview
UIViewRepresentable bridging AVCaptureVideoPreviewLayer into SwiftUI. Handles layout with resizeAspectFill.
🎨
ContentView
ZStack: camera preview + classification overlay. Wires camera.onFrame to classifier.classify on appear.

📐 Three Threads, No Blocking

Thread Component Work
Camera Queue (serial) CameraManager Captures frames, delivers CVPixelBuffer
Inference Queue (serial) ImageClassifier VNCoreMLRequest + Neural Engine inference
Main Thread ContentView + CameraPreview SwiftUI rendering + camera preview layer

💡 The isProcessing guard ensures: if inference is still running, skip the current frame. Always classify the latest frame, never a stale one.

⚠️ The 6 Pitfalls Everyone Hits

💣
Crash on Launch
Missing NSCameraUsageDescription in Info.plist. Add it before your first run.
🔎
Model Not Found
Model file not added to app target. Check File Inspector → Target Membership.
🔄
Wrong Predictions
Image orientation not handled. Pass orientation: .right to VNImageRequestHandler. The #1 gotcha.
🔄
Dropped Frames
Inference blocking camera queue. Use a separate inference queue + isProcessing throttle.
🔢
Numeric Labels
Missing classifier labels during coremltools conversion. Re-convert with classifier_config.
🚫
Background Kill
Camera session not stopped. Call session.stopRunning() in onDisappear.

🤔 Quick Quiz

Test Yourself

Why do we wrap Core ML models in VNCoreMLModel instead of calling prediction() directly?

Vision's VNCoreMLRequest handles all image preprocessing automatically — resizing, normalizing, orientation. Without it, you'd do all that work manually for every frame.

What does imageCropAndScaleOption = .centerCrop do?

It tells Vision how to resize the camera frame (e.g., 1280×720) to the model's expected size (224×224). Center-crop takes the center square and scales it, which works well for classification.

Why set alwaysDiscardsLateVideoFrames = true?

If inference is slower than the camera frame rate, this drops old frames instead of queuing them. Without it, you'd process increasingly stale frames and eventually run out of memory.

Camera frames arrive in what orientation?

.right (landscape-right). If you don't tell Vision about this, the model sees a rotated image and predictions are wrong. Always pass orientation: .right to VNImageRequestHandler.

What are the scale and bias in coremltools ImageType?

ImageNet normalization constants. They transform raw pixel values (0–255) into the normalized range the PyTorch model was trained on. Core ML applies them automatically during preprocessing.

📋 Key Takeaways

If You Remember Nothing Else

  • Convert: torch.jit.trace → ct.convert() with ImageType → .mlpackage. Or download from Apple's gallery.
  • Load once: MLModelConfiguration(.all) → VNCoreMLModel. First load is slow (~100–500ms), predictions are fast (~3ms).
  • Use VNCoreMLRequest — Vision handles resize, normalize, and orientation automatically.
  • Camera: AVCaptureSession + VideoDataOutput + .hd1280x720 + alwaysDiscardsLateVideoFrames.
  • Three threads: camera queue, inference queue, main thread. Never block any of them.
  • Always pass orientation: .right for camera frames. The #1 gotcha in Core ML camera apps.
  • MobileNetV2 on Neural Engine: ~3ms. That's 10x faster than the camera's 30fps frame rate.
