Create ML — Training Your Own Models

Memory Map

💡 The Core Insight

💡

The "iMovie of Machine Learning" — drag in data, click Train, get a .mlmodel.

Transfer learning means 30 images per class is enough. Create ML reuses Apple's pre-trained feature extractor (ImageFeaturePrint) and only trains a tiny classification head on top. A 5-class image classifier trains in 45 seconds and weighs just 17 KB.

🛠️ Two Ways to Train

🎨
Create ML App (GUI)
Xcode → Open Developer Tool → Create ML. Drag-and-drop data, configure augmentations, click Train, export .mlmodel. No code required.
💻
Create ML Components (Swift)
Programmatic training in pure Swift. ImageFeaturePrint().appending(LinearClassifier()). Use for CI/CD, automated pipelines, or on-device training.

📚 Key Terms

Transfer Learning
Starting from a pre-trained model and fine-tuning the top layers on your data. Why 30 images is enough.
ImageFeaturePrint
Apple's built-in image feature extractor, shipped with every iPhone. Create ML classifiers ride on top of it.
Augmentations
Automatic transforms (crop, rotate, flip, blur, noise) that create training variations. 30 photos become 200+.
Confusion Matrix
Grid showing which classes the model confuses. Tells you exactly where to add more data.
MLUpdateTask
Core ML API for on-device model updates. Fine-tunes updatable layers with user data, data never leaves device.
Create ML Components
Swift framework for programmatic training. Composable pipeline: ImageFeaturePrint().appending(LinearClassifier()).
Overfitting
Model memorizes training data instead of learning patterns. Sign: 99% training accuracy, 70% validation accuracy.

🤔 Quick Quiz

Test Yourself

Why can Create ML train accurate models with just 30-50 images per class?

Transfer learning. Create ML starts from Apple's ImageFeaturePrint, which already understands edges, textures, shapes, and objects from millions of images. Your training only fine-tunes the top classification layers for your specific classes.

Why is a Create ML image classification model only 17 KB?

The model only contains the lightweight classification head. The heavy feature extractor (ImageFeaturePrint) ships with every iPhone as part of the OS. Core ML automatically chains them at runtime.

What's the difference between image classification and object detection?

Image classification labels the entire image ("this is a coffee mug"). Object detection finds where objects are with bounding boxes ("there's a coffee mug at coordinates (120,80) to (280,240)"). Object detection needs annotated training data with bounding box coordinates.

How does on-device personalization with MLUpdateTask work?

Ship a base model with updatable layers. As the user interacts with your app, collect labeled examples. Use MLUpdateTask to fine-tune only the top layers on-device. The model improves for that specific user, and their data never leaves the device.

What's the #1 training pitfall and how do you avoid it?

Background bias. If all training photos have the same background, the model learns the background, not the object. Fix: vary backgrounds aggressively, add a "negative" class, and enable augmentations (crop, rotation, exposure).
