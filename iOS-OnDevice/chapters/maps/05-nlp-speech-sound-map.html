Natural Language, Speech, and Sound

Memory Map

💡 The Core Insight

💡

Vision gives your app eyes. These three frameworks give it ears and a reading brain.

Natural Language analyzes text in microseconds. SpeechAnalyzer (iOS 26) transcribes voice with async/await. SoundAnalysis identifies 300+ sounds. All on-device, all private, no model files to manage.

🎨 Three Frameworks, Three Senses

📖
Natural Language
Text analysis: NER, sentiment, language detection, embeddings. Runs in microseconds — essentially free. No permissions needed.
🎤
Speech (iOS 26)
SpeechAnalyzer + SpeechTranscriber. Modular, async/await, offline-first. Matches mid-tier Whisper accuracy. Replaces SFSpeechRecognizer.
🔊
SoundAnalysis
300+ sound categories (dog bark, siren, applause, coughing). AVAudioEngine → SNAudioStreamAnalyzer → SNClassifySoundRequest.

📚 Key Terms

SpeechTranscriber
Speech-to-text module. Supports real-time (volatile + final results) and file transcription. Offline-first with system-managed language models.
SpeechDetector
Voice activity detection module. Identifies when someone is speaking in an audio stream.
Volatile Results
Partial transcriptions that update as the user speaks. Use for real-time UI ("live typing" effect). Final results are stable and ready for storage.
NLEmbedding
Turns words/sentences into 512-dimensional vectors. Similar meanings = close vectors. Enables semantic search without a server.
SNAudioStreamAnalyzer
Feeds real-time audio buffers from AVAudioEngine to sound classification requests. The bridge between microphone and classifier.
windowDuration
How much audio SoundAnalysis analyzes at once. 0.5s = fast but less accurate. 3s = accurate but delayed. Default: 1.5s.

🤔 Quick Quiz

Test Yourself

What NLTagger scheme do you use for sentiment analysis, and what's the output range?

.sentimentScore scheme. Returns a score from -1.0 (very negative) to +1.0 (very positive). Works best on paragraph-sized text, not individual words.

What's the difference between volatile and final results in SpeechTranscriber?

Volatile results are partial transcriptions that update as the user speaks — use for real-time UI. Final results are stable and complete — use for storage. Enable volatile results with .volatileResults reporting option.

How does SoundAnalysis receive audio data for real-time classification?

AVAudioEngine captures microphone input. You install a tap on the input node to get audio buffers, then feed each buffer to SNAudioStreamAnalyzer.analyze(_:atAudioFramePosition:). The analyzer runs attached requests and notifies observers.

Why should you check NLTagger.availableTagSchemes before running NER?

NER is only supported in ~7 languages. If you run NER on unsupported languages, the tagger silently returns no results. Check at runtime to avoid silent failures and fall back to basic tokenization.

What does NLEmbedding enable that simple string matching can't?

Semantic search — finding content by meaning, not exact text. "Happy" and "joyful" are close in embedding space even though they share no characters. Enables similar-item discovery, duplicate detection, and recommendations on-device.

📋 Key Takeaways

If You Remember Nothing Else

  • Three frameworks: Natural Language (text, microseconds), Speech/SpeechAnalyzer (voice, iOS 26), SoundAnalysis (300+ sounds). All on-device.
  • NLTagger is essentially free (~0.1–0.2ms). Run it on every keystroke, clipboard change, or message.
  • NLEmbedding = 512-dim vectors for semantic similarity. Search by meaning, not just text match.
  • SpeechAnalyzer (iOS 26) replaces SFSpeechRecognizer. Modular: SpeechTranscriber + SpeechDetector. Native async/await.
  • Volatile results = live UI updates. Final results = stable for storage. Enable with .volatileResults.
  • SoundAnalysis: AVAudioEngine → SNAudioStreamAnalyzer → SNClassifySoundRequest. windowDuration 1.5s is a good default.
  • NLTagger/NLEmbedding need no permissions. Speech and Sound need microphone + speech recognition permissions.
  • NER and sentiment only support ~7 languages. Always check availableTagSchemes at runtime.
