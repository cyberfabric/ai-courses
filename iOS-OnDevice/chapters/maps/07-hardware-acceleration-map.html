Chapter 7

Hardware Acceleration — Neural Engine and Metal

Memory Map

💡 The Core Insight

💡

Which engine runs your model makes a 15–20x difference in speed.

Your iPhone has three ML engines: CPU (slowest), GPU (middle), Neural Engine (fastest, most efficient). Core ML dispatches operations across all three. Use .all, warm up before measuring, benchmark for 60+ seconds, and monitor thermal state.

⚙️ The Three Engines

💻
CPU — The Generalist
Handles anything but nothing fast. Slowest for ML. Always available. Use as fallback with .cpuOnly.
🎨
GPU — The Parallel Processor
Thousands of cores for parallel math. Great for custom Metal shaders and image processing. 3–5x faster than CPU for ML.
🧠
ANE — The ML Specialist
16 cores purpose-built for tensor operations. 12–20x faster than CPU, most power-efficient. The default winner.

🔧 MLComputeUnits Options

.all (Default)
Core ML picks the best engine per operation. Usually optimal. Start here, only change after profiling.
.cpuAndNeuralEngine
Skip GPU. Use when GPU is busy (games, AR rendering). Avoids GPU contention.
.cpuAndGPU
Skip Neural Engine. Use when model has ANE-unsupported ops causing fallback overhead.
.cpuOnly
Slowest. Guaranteed available everywhere. Use as fallback for oldest devices or debugging.

⚠️ Four Things That Bite You

⏳
First Inference Is Slow
Model compilation, buffer allocation, NE init. 50–100x slower than steady-state. Always warm up with 3–5 dummy runs.
🔥
Thermal Throttling
After 30+ seconds of sustained ML, iPhone reduces clock speeds. Design for .serious thermal state, not .nominal.
🔄
ANE Fallbacks
Unsupported ops, dynamic shapes, or exotic layers fall back to CPU. Check in Core ML Instrument — fallbacks are your bottleneck.
📱
Device Diversity
iPhone SE (4ms) to iPhone 17 Pro (1.5ms) = 2.7x spread. For large models, the gap is worse. Benchmark on minimum device.

🌡️ Thermal State Management

  1. .nominal — Full Speed

    Cool device. Use your fastest model and highest frame rate.

  2. .fair — Slightly Warm

    Still OK. No changes needed yet.

  3. .serious — Getting Hot

    Reduce workload: switch to lightweight model, drop frame rate to 15fps.

  4. .critical — Throttling Hard

    Minimize ML work: pause inference, show cooldown message.

🤔 Quick Quiz

Test Yourself

Why is the first inference always slow?

First inference triggers model compilation: Core ML compiles for the specific hardware, allocates memory buffers, initializes the Neural Engine pipeline, and compiles Metal shaders. This is cached for subsequent runs. Always warm up with 3–5 dummy inferences.

When should you use .cpuAndNeuralEngine instead of .all?

When the GPU is busy with other work (games, AR rendering, heavy UI animations). .cpuAndNeuralEngine avoids GPU contention. But .all is usually best — only override after profiling.

What's the difference between p50 and p95 latency?

p50 (median) is the typical inference time. p95 is what 1 in 20 inferences takes. A high p95/p50 ratio means inconsistent performance. For real-time features, p95 matters more than average.

Why should you benchmark for 60+ seconds?

Thermal throttling kicks in after ~30 seconds of sustained ML inference. Short benchmarks show burst performance (.nominal thermal state), not real-world sustained performance (.serious state).

What's special about the A19 Pro's GPU Neural Accelerators?

The A19 Pro embeds dedicated tensor processing units inside each GPU core. The GPU itself can run ML natively, delivering ~3x peak GPU compute vs A18 Pro. Metal 4 exposes these through its tensor API.

📋 Key Takeaways

If You Remember Nothing Else

  • Three engines: CPU (slowest), GPU (middle), Neural Engine (fastest, 12–20x over CPU).
  • Use .all compute units by default. Only override after profiling with Instruments.
  • First inference is 50–100x slower (compilation). Warm up with 3–5 dummy runs.
  • Benchmark for 60+ seconds. Thermal throttling changes real-world performance after ~30s.
  • Monitor ProcessInfo.thermalState. Design for .serious, not .nominal.
  • Metal 4 = GPU-ML fusion with native tensors. Advanced (games, AR). Core ML handles 90%+ of cases.
  • Unified memory = no copies between CPU, GPU, ANE. Apple Silicon's secret ML advantage.
  • Always benchmark on your minimum supported device, not your newest one.
