Chapter 2

How ML Models Work on Apple Silicon

Memory Map

💡 The Core Insight

💡

An ML model is a compiled function you didn't write.

It's a file full of learned numerical weights. You feed it inputs, it returns outputs. You can't read the source code — but you don't need to. Core ML compiles it for your chip and runs it on the Neural Engine. Your job: preprocess inputs, call prediction(), interpret outputs.

🔄 The Inference Loop

The pattern behind every on-device AI feature:

📦 Load
.mlmodel → compile for chip
→
🔧 Preprocess
Resize, normalize, convert
→
🧠 Predict
Neural Engine forward pass
→
📈 Postprocess
Numbers → labels

💡 Loading is slow (100–500ms). Prediction is fast (3–15ms). Load once, predict many times.

🏗️ Model Architectures Cheat Sheet

📷
MobileNet
Image classification. 2–5 MB, <5ms. Your default "what is this?" model.
🖵️
YOLOv8
Object detection. Finds multiple objects with bounding boxes. 6–30 MB.
✂️
DeepLab
Segmentation. Labels every pixel. Powers portrait mode and background effects.
💬
BERT
Text understanding. Sentiment, entities, Q&A. Mobile variant: DistilBERT.
🌟
EfficientNet
Classification, more accurate than MobileNet. Bigger tradeoff: 20–80 MB.
🎤
Whisper
Speech-to-text. Multiple sizes. Convertible to Core ML via coremltools.

⚖️ The Size-Accuracy Tradeoff

💡 Start with MobileNetV3. Only upgrade if accuracy is insufficient for your specific categories.

⚙️ Compute Units

.all (default)
Core ML picks the best hardware per layer. Use this 99% of the time.
.cpuAndNeuralEngine
Neural Engine + CPU. Leaves GPU free for rendering.
.cpuAndGPU
GPU via Metal + CPU. When Neural Engine doesn't support your model's ops.
.cpuOnly
CPU only. For debugging or deterministic results.
Unified Memory
CPU, GPU, and Neural Engine share the same memory pool. No expensive data copies. This is Apple Silicon's secret weapon.

📖 Key Terms

Tensor
A multi-dimensional array of numbers. The universal language ML models speak.
MLShapedArray
Modern Swift type for tensors. Generic, subscriptable. Prefer over MLMultiArray in new code.
CVPixelBuffer
The image format Core ML expects. Vision's VNImageRequestHandler creates these for you.
.mlmodelc
Compiled model bundle. Xcode creates this at build time from .mlmodel/.mlpackage.
VNCoreMLRequest
Vision wrapper that auto-handles preprocessing (resize, normalize, orient) for image models.
Postprocessing
Converting raw model output (array of floats) into useful results (labels, bounding boxes, text).

🤔 Quick Quiz

Test Yourself

What's inside a .mlmodel file?

Millions of learned numerical weights (floats) plus a description of the layer architecture. It's essentially a "compiled function" that was learned from data, not hand-coded.

Why is the first model.prediction() call slower than subsequent ones?

Core ML compiles and optimizes the model for the specific device on first load (100–500ms). After that, each prediction runs in milliseconds. Load once, predict many.

What's a 3D tensor in iOS terms?

A color image (UIImage): Height × Width × Channels (RGB). A 224×224 color image is a tensor of shape [224, 224, 3].

Why should you use VNCoreMLRequest instead of calling model.prediction() directly?

Vision handles all preprocessing automatically — resizing to the model's expected dimensions, normalizing pixel values, and handling EXIF orientation. Without it, you must do all this manually.

When should you pick EfficientNet over MobileNet?

Only when MobileNet's accuracy isn't good enough for your specific categories and you can afford the extra size (20–80 MB) and latency (8–25ms). Start with MobileNet, upgrade only if needed.

📋 Key Takeaways

If You Remember Nothing Else

  • An ML model = a file of learned weights. Feed it inputs (tensors), get outputs (numbers you interpret).
  • The inference loop: Load → Preprocess → Predict → Postprocess. Load once, predict many.
  • .mlmodel (legacy single file) vs .mlpackage (modern bundle). Both compile to .mlmodelc in Xcode.
  • Use VNCoreMLRequest for image models — Vision handles preprocessing automatically.
  • Start with MobileNetV3 (5 MB, 4ms). Upgrade only if accuracy demands it.
  • Use .all compute units. Let Core ML pick the best hardware. Unified memory means no copy overhead.
  • Find models at Apple's Gallery, Hugging Face, or train your own with Create ML.
