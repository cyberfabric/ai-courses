Chapter 6

Foundation Models and Apple Intelligence

Memory Map

💡 The Core Insight

💡

Your app gets its own LLM — no cloud, no API key, no bill.

The Foundation Models framework gives you direct access to Apple's on-device ~3B parameter language model. Define a Swift struct with @Generable, and the model guarantees its output matches your type. No parsing. No hallucinated structure. Just typed Swift objects.

🧰 The Five Superpowers

🛠️
@Generable
Mark a Swift struct → model output matches your type. Supports String, Int, Float, Bool, Optional, Array, enums, nested structs.
📏
@Guide
Describe properties + add constraints. .range(1...10) for numbers, .count(4) for arrays. Model can't produce invalid output.
🌊
Streaming
streamResponse returns AsyncSequence of partial snapshots. Properties fill in one by one — bind directly to SwiftUI.
🗨️
Multi-Turn Sessions
LanguageModelSession maintains context. Each respond(to:) builds on prior exchanges. instructions: = system prompt.
🔧
Tool Calling
Define tools the model can invoke. Model decides when and what args. Like giving an assistant access to your phone's apps.
🎨
Image Playground
imagePlaygroundSheet for UI, ImageCreator for programmatic generation. Slow (10–30s) — always show progress.

🔎 Device Availability

  1. .available

    Good to go — start generating.

  2. .deviceNotEligible

    Hardware too old (pre-iPhone 15 Pro, pre-M1). Show fallback UI.

  3. .appleIntelligenceNotEnabled

    User hasn't enabled Apple Intelligence in Settings. Prompt them.

  4. .modelNotReady

    Model still downloading or updating. Show loading state.

⚠️ Always check SystemLanguageModel.default.availability before using Foundation Models. Both hardware AND Apple Intelligence setting must be enabled.

💭 Prompt Engineering for On-Device

✅
Do: One Task Per Prompt
"Summarize this email in one sentence." Short, focused, single-purpose.
✅
Do: Use @Generable
Let the type system enforce structure instead of asking for JSON in the prompt.
❌
Don't: Overload Prompts
"Summarize, analyze sentiment, suggest improvements, and translate" — too many tasks for ~3B model.
❌
Don't: Long Instructions
Keep system instructions short. "You summarize text. Be concise." beats a page of personality description.

📚 Key Terms

LanguageModelSession
The main entry point. Holds conversation context, instructions, and tools. Create new sessions for unrelated tasks.
@Generable
Macro that generates a compile-time schema from your struct. The model uses constrained decoding to match it exactly.
@Guide
Macro for property-level hints and constraints. .range() for numbers, .count() for arrays, description: for natural language hints.
Constrained Decoding
The model is forced to produce output matching your @Generable schema. Not "parse and hope" — structurally guaranteed.
Streaming Snapshots
Unlike token-by-token streaming, Foundation Models sends partially-filled structs. Optional properties become non-nil as the model generates.
Tool
A Swift type conforming to Tool protocol. Defines name, description, @Generable Arguments, and a call() function the model can invoke.
Writing Tools
System-wide Rewrite/Proofread/Summarize. Works automatically with standard text views. Customize with .writingToolsBehavior().

🤔 Quick Quiz

Test Yourself

What's the difference between @Generable and @Guide?

@Generable marks the struct itself, generating a compile-time schema. @Guide goes on individual properties to add descriptions and constraints (.range(), .count()). Together, they guarantee the model's output matches your type.

How does streaming work differently in Foundation Models vs ChatGPT?

Foundation Models streams snapshots — partially-filled structs where Optional properties become non-nil as the model generates them. ChatGPT streams raw token deltas. Snapshots are much easier to bind to SwiftUI views.

What happens if Foundation Models isn't available on the user's device?

Check SystemLanguageModel.default.availability. It can be unavailable because the device is too old (.deviceNotEligible), Apple Intelligence isn't enabled (.appleIntelligenceNotEnabled), or the model is still downloading (.modelNotReady). Always design a fallback.

When does a request go to Private Cloud Compute instead of on-device?

The system decides automatically based on task complexity. Simple tasks (summaries, rewrites) stay on-device. Complex tasks (multi-step reasoning, large context) may route to Apple's secure cloud. Users can opt out in Settings.

Why should you use @Generable instead of asking the model for JSON in your prompt?

@Generable uses constrained decoding to guarantee valid output matching your type. Asking for JSON in the prompt relies on the model to get the format right — it can hallucinate structure, add extra fields, or produce invalid JSON. With @Generable, invalid output is structurally impossible.

📋 Key Takeaways

If You Remember Nothing Else

  • Foundation Models = Apple's on-device ~3B LLM. import FoundationModels, no API key, no cloud, no cost.
  • Requires iPhone 15 Pro+ AND Apple Intelligence enabled. Always check SystemLanguageModel.default.availability.
  • @Generable + @Guide = typed output with constraints. The model can't produce invalid structure.
  • Stream with streamResponse — sends partial snapshots, not token deltas. Bind directly to SwiftUI.
  • Tool calling: define a Tool with @Generable Arguments. The model decides when to call it.
  • One task per prompt. Use @Generable for structure, not prompt text. Short instructions win.
  • ~30–40 tokens/sec on-device. Always stream — never wait for the full response before showing UI.
  • Writing Tools work automatically with standard text views. Image Playground for on-device image generation.
