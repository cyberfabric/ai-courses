💡 The Core Insight

💡

Apple already trained the models. Just pick a request.

Vision is a catalog of pre-trained computer vision models — OCR, faces, body pose, barcodes, segmentation, document parsing. No model files to manage. One pattern: create request → create handler → perform → read observations.

🔧 Two Handler Types

VNImageRequestHandler
One image, one or more requests. Create a new handler per image. Use for photos, single frames, documents.
VNSequenceRequestHandler
Reuse across a sequence of frames. Maintains state for object tracking across video. Use for continuous tracking.
regionOfInterest
Limit detection to a normalized CGRect (0..1). Faster and more focused. Bottom-left origin (not top-left!).

⚠️ Two Gotchas to Remember

🔄
Coordinate System
Vision origin = bottom-left. UIKit origin = top-left. Flip Y: uiKitY = 1.0 - visionY - height
👤
Detection ≠ Recognition
Face detection finds where faces are. It does not identify who the person is. Different task, different model.

🆕 New in iOS 26

  1. RecognizeDocumentsRequest

    Structured document parsing: paragraphs, tables, lists, machine-readable codes, and auto-extracted emails/phones/URLs. 26 languages.

  2. DetectLensSmudgeRequest

    Detects if the camera lens is smudged. Prompt users to clean before scanning.

  3. Swift Concurrency APIs

    Async/await versions of all Vision requests. Modern Swift patterns replacing completion handlers.

🤔 Quick Quiz

Test Yourself

How do you run text recognition and barcode detection on the same image efficiently?

Pass both requests to a single handler.perform([textRequest, barcodeRequest]) call. Vision preprocesses the image once and feeds it to both models.

What's the difference between .fast and .accurate recognition levels?

.fast (~8ms) is for real-time camera feeds. .accurate (~30ms) gives better results on static images like receipts and documents. Language correction adds ~10ms more.

Why does Vision use bottom-left origin coordinates?

Vision follows the Core Image / Core Graphics coordinate system (mathematical Y-up). UIKit uses top-left origin. Always flip Y when drawing Vision results in UIKit: y = 1.0 - visionY - height.

When should you use VNSequenceRequestHandler instead of VNImageRequestHandler?

When tracking objects across a sequence of video frames. VNSequenceRequestHandler maintains state between calls, enabling smooth object tracking. Use VNImageRequestHandler for single images or independent frames.

What does RecognizeDocumentsRequest do that VNRecognizeTextRequest doesn't?

It understands document structure: paragraphs, tables, lists, and auto-extracts important information like emails, phone numbers, and URLs. VNRecognizeTextRequest returns flat text lines without structural context.

📋 Key Takeaways

If You Remember Nothing Else

  • Vision = Apple's pre-trained CV models. No files to manage. One pattern: request → handler → perform → observations.
  • 7+ request types: text, documents, classification, faces (76 pts), body pose (19 joints), person segmentation, barcodes.
  • Combine multiple requests in one perform() call — Vision shares preprocessing. Text + barcode + face = ~15ms.
  • RecognizeDocumentsRequest (iOS 26) parses tables, lists, paragraphs, and auto-extracts emails/phones/URLs.
  • Vision coordinates: bottom-left origin, normalized 0..1. Flip Y for UIKit.
  • Face detection ≠ face recognition. Vision finds where, not who.
