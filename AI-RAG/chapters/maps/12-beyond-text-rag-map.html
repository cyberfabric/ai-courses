The Big Idea

The RAG pattern — retrieve relevant context, then generate — works far beyond plain text. Images, code, databases, knowledge graphs, and autonomous agents all use the same core idea. Your mental model expands from "text Q&A" to "any data, any modality, any reasoning."

Open Tabs Signal
Code assistants use open files as a strong relevance signal — opening the relevant file dramatically improves suggestions.
Symbol Retrieval
Find definitions of functions/types used near the cursor. Structure-aware, not just semantic similarity.
Fill-in-the-Middle (FIM)
LLM receives code before cursor (prefix) + after cursor (suffix) + retrieved context → generates what fits in between.

🗃️ Structured Data RAG

📊
Text-to-SQL
Natural language → SQL query → execute → explain results. Like having a data analyst on call. Best for aggregations, filters, joins.
🔍
Embed & Retrieve Rows
Embed database rows into a vector store, search by meaning. Best for free-text fields and conceptual queries ("find customers like X").
⚠️
Safety First!
Never let LLMs run arbitrary SQL. Whitelist SELECT only, use read-only connections, set timeouts, limit result sizes. DROP TABLE is syntactically correct too.

🤖 Agentic RAG

🔄
ReAct Loop
Reason → Act → Observe → Repeat. The agent dynamically chooses retrieval strategy based on the question and intermediate results.
🧰
RAG as a Tool
In agentic systems, RAG is one tool among many (APIs, databases, calculators, web). The agent decides when retrieval is needed.
🎯
Dynamic Strategy
Simple question → single vector search. Complex question → keyword + semantic + re-rank. The agent adapts, not the developer.
💬
Conversational RAG
Rewrite follow-up questions to be self-contained before searching. "How long does that take?" → "How long does the international refund process take?"

🧪 Quick Quiz

What are the three approaches to multimodal RAG?

Caption & Embed (vision model describes images → embed as text), Native Vision Embeddings (multimodal model embeds text + images together), and Late Interaction / ColPali (treat pages as images, match visually). Start with Caption & Embed for simplicity.

Why does opening a related file improve AI code suggestions?

Code assistants use open tabs as a strong retrieval signal. If the data model file is open, the assistant sees field names and types. Close it, and it has to guess or retrieve less relevant code from the index.

When does GraphRAG beat standard vector RAG?

When questions require understanding relationships between entities ("How are teams connected?") or high-level summaries across many documents ("Summarize Q1 initiatives"). For simple factual lookups, vector RAG is faster and cheaper.

What's the key difference between agentic RAG and traditional RAG?

Traditional RAG follows a fixed pipeline (embed → search → generate). Agentic RAG uses an autonomous agent that decides what to do at each step — choosing retrieval strategy, evaluating results, searching again if needed, and combining multiple sources dynamically.

How do you handle follow-up questions in conversational RAG?

Use an LLM to rewrite the follow-up question into a self-contained query using chat history. "How long does that take?" becomes "How long does the international refund process take?" Don't stuff the entire chat history into the search query — it dilutes the embedding.

📌 Key Takeaways

  • The core RAG pattern (retrieve → generate) applies to images, code, databases, graphs, and agents — not just text documents.
  • Multimodal RAG: start with "caption & embed" for simplicity, graduate to ColPali for complex PDFs with visual layouts.
  • Code RAG powers Cursor, Copilot, and Claude Code — it uses file structure, open tabs, and symbol graphs, not just semantic similarity.
  • Text-to-SQL lets users query databases in natural language — but never let LLMs run arbitrary SQL on production databases.
  • GraphRAG excels at relationship questions and cross-document reasoning (3.4x better) but costs 10-50x more to build than vector indexing.
  • Agentic RAG replaces fixed pipelines with dynamic agents that choose their own retrieval strategy — the future of knowledge systems.
  • RAG isn't always the answer: long context windows, fine-tuning, prompt caching, and direct API calls each have their sweet spot.
