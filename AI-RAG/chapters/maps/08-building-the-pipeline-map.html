Building a Production RAG Pipeline

Memory Map

The Big Idea

A production RAG system is two pipelines stitched together: an ingestion pipeline (kitchen prep — turning messy documents into searchable chunks) and a query pipeline (cooking to order — turning user questions into grounded answers). If your prep is sloppy, no amount of cooking skill saves the meal.

🧩 Key Production Concepts

📄
Document Loaders
PDFs (PyMuPDF/Unstructured), HTML (BeautifulSoup — strip nav, footer, ads), Markdown (easiest), enterprise sources (Confluence, Notion via API → clean → normalize).
🧹
Preprocessing
Collapse blank lines, remove invisible chars, strip page numbers, normalize whitespace. Garbage in = garbage out. Clean before chunking.
📝
RAG System Prompt
Force grounding: "Answer ONLY from context. Cite sources. Say 'I don't know' if context is insufficient." Without this, the LLM fills gaps from training data.
🎯
Context Engineering
What matters most isn't how you phrase instructions — it's the quality of context in the prompt. Mediocre prompt + great context beats brilliant prompt + garbage context.
📎
Source Attribution
Turn "the AI said so" into verifiable claims. Store rich metadata during ingestion (file, page, section, date). Link citations back to original documents.
📡
Streaming Responses
Use .stream() instead of .create(). Turns a 2-second wait into a fluid experience. Append source citations after stream completes.

⚡ Three Types of Caching

  1. 🔢 Embedding Cache

    Avoid re-embedding the same text during re-ingestion. Essential when you reprocess documents on a schedule.

  2. 💬 Query Cache

    Exact match: hash the question, return cached answer within a TTL (e.g., 1 hour). Redis or even a Python dict.

  3. 🧠 Semantic Cache

    Meaning match: embed the question, check if any cached question has cosine similarity > 0.95. Can reduce LLM calls by 30-50% for repetitive queries (e.g., customer support).

🚨 When Things Go Wrong

🕳️
Empty Retrieval
No relevant chunks found (or all below similarity threshold). Return a fallback: "I couldn't find relevant information. Could you rephrase?" Never let the LLM guess.
🔌
API Failures
Embedding or LLM API down. Retry with exponential backoff (base × 2^attempt). Max 3 retries, then fail fast with a clear error message.
🤖
LLM Ignores Context
Check in order: (1) System prompt enforces grounding? (2) Context before question? (3) Too many chunks burying relevant ones? (4) Reduce top-K from 10 to 3-5.

🏗️ Pipeline Orchestration: Build vs Framework

Approach Best For Trade-offs
🐍 Custom Python Simple pipelines, full control, small teams Easy to debug, ~200 lines. You build everything.
🦜 LangChain / LangGraph Complex agent workflows, multi-step reasoning Rich ecosystem. LangGraph adds stateful orchestration.
🦙 LlamaIndex Document-heavy RAG, complex data sources Best-in-class data connectors and index types.

Start custom, adopt a framework when you're reinventing its abstractions.

📖 Key Terms

Ingestion Pipeline
The offline process that loads, cleans, chunks, embeds, and stores documents in a vector database. Kitchen prep before the restaurant opens.
Query Pipeline
The real-time process that takes a user question, retrieves relevant chunks, builds a prompt, and generates a grounded answer.
Context Engineering
The practice of optimizing what context you put in front of the LLM, rather than just how you phrase instructions. Quality context > clever prompts.
Grounding
Constraining the LLM to answer only from provided context, not from training data. Enforced via the system prompt.
Source Attribution
Citing which documents/chunks the answer came from, so users can verify. Requires rich metadata stored during ingestion.
Semantic Cache
Caching by meaning similarity (cosine > 0.95) rather than exact string match. "What's the refund policy?" and "How do refunds work?" hit the same cache.
Batch Embedding
Embedding multiple chunks in a single API call (up to 2048 inputs). Makes ingestion 10-50x faster than one-at-a-time.

🧪 Quick Quiz

What are the two pipelines in every RAG system?

Ingestion pipeline (offline: load → preprocess → chunk → embed → store) and query pipeline (real-time: question → embed → retrieve → prompt → generate). Ingestion is kitchen prep; query is cooking to order.

Why should context come before the question in the prompt?

LLMs process text sequentially. Having context fresh in "working memory" when they encounter the question produces more grounded answers. Context: [...] Question: [...] outperforms the reverse.

When should you adopt LangChain/LlamaIndex instead of custom Python?

When you catch yourself reinventing their abstractions — writing your own retry logic, state management, connector library, or multi-step agent workflows. Most simple RAG systems are ~200 lines of custom Python. Don't add a framework for GitHub stars.

What's the difference between a query cache and a semantic cache?

Query cache: exact string match (hash the question). Semantic cache: meaning match (embed the question, check cosine similarity > 0.95). Semantic cache catches paraphrases like "refund policy" / "how do refunds work."

What should you check first when the LLM ignores retrieved context?

In order: (1) System prompt explicitly requires grounded answers? (2) Context placed before the question? (3) Too many chunks (reduce top-K to 3-5)? (4) Question ambiguous enough that the model's prior knowledge dominates?

⚠️ Watch Out

📑
PDFs Are Chaos
Tables get scrambled, columns merge, reading order breaks. Use Unstructured.io for complex PDFs. Convert tables to Markdown before chunking.
🏗️
The Framework Trap
Don't pick a framework for GitHub stars. Many production RAG systems are 200 lines of Python. Add a framework when complexity justifies it, not before.
📡
Streaming + Citations
You can't show source citations until the stream completes. Common pattern: stream answer text, append "Sources" section at the end.

📌 Key Takeaways

  • Every RAG system is two pipelines: ingestion (offline prep) and query (real-time answering).
  • Context engineering > prompt engineering — quality of retrieved context matters more than clever instructions.
  • The system prompt must enforce grounding: answer only from context, cite sources, say "I don't know" when needed.
  • Batch embed during ingestion (up to 2048 inputs/call) for 10-50x speedup.
  • Three caching layers: embedding cache (ingestion), query cache (exact match), semantic cache (meaning match, 30-50% LLM cost reduction).
  • Start custom Python, adopt LangChain/LlamaIndex only when you're reinventing their abstractions.
  • Always handle: empty retrieval (fallback message), API failures (retry with backoff), LLM ignoring context (reduce top-K, strengthen grounding).
