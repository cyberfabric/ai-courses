The Big Idea

Android has the deepest AI tooling integration of any mobile platform. Gemini in Android Studio for inline editing, Claude Code for multi-step workflows, and on-device AI (ML Kit, Gemini Nano) for user-facing features. Together, a solo dev delivers at the pace of a three-person team.

📐 The UiState Pattern (Document in CLAUDE.md)

Sealed UiState Interface
Loading, Success(data), Error(message) — exhaustive when-blocks force all states handled.
StateFlow
MutableStateFlow internally, exposed as read-only StateFlow. UI collects with collectAsStateWithLifecycle.
Pattern-Once Principle
Document the UiState pattern in CLAUDE.md once. AI follows it for every new ViewModel automatically.
Hilt Injection
@HiltViewModel + @Inject constructor. Repository as dependency. hiltViewModel() in composables.

🧪 Android Testing Layers

  1. Unit Tests (JVM)

    ViewModel tests with MockK + Turbine for StateFlow. DAO tests with Room's in-memory database. Fast, no device needed.

  2. Compose UI Tests

    createComposeRule + semantics matchers. Test all UI states: loading, success, error. Use waitUntil, never Thread.sleep.

  3. Instrumented Tests

    Run on device/emulator. End-to-end flows, hardware features (camera, sensors). Slower but validates real behavior.

  4. CI Pipeline

    GitHub Actions: unit tests → emulator UI tests (API 34) → Gradle cache → test report artifacts.

⚠️ Android-Specific Pitfalls

Hallucinated Compose APIs
AI may generate functions that don't exist in your Compose version. Verify every import against actual dependencies.
AsyncImage needs Coil
Not built-in. Icons.AutoMirrored needs Material 3 1.2+. collectAsStateWithLifecycle needs lifecycle-runtime-compose.
Room Migration Risks
AI migrations are usually syntactically correct but verify: column types, nullable defaults, indexes, version numbers.
Gemini Nano Fallbacks
Always implement fallback for unsupported devices. Never assume on-device LLM is available.

🧪 Quick Quiz

When should you use Gemini vs Claude Code for Android development?

Gemini: inline completions, quick code transforms, build error analysis — while editing in Android Studio. Claude Code: complex multi-file tasks, test generation, cross-module refactoring — autonomous multi-step workflows.

What's the "pattern-once principle" for ViewModels?

Document the sealed UiState + StateFlow pattern once in CLAUDE.md. AI follows it automatically for every new ViewModel without repeating instructions in each prompt. Context engineering applied to architecture.

Why must you always verify Compose API imports?

AI generates plausible-looking Compose functions that may not exist in your dependency version. AsyncImage requires Coil, Icons.AutoMirrored requires Material 3 1.2+. A generated import that compiles in the AI's mind may crash your build.

What are the three Android testing layers?

1) Unit tests (JVM) — ViewModel + DAO with MockK/Turbine. 2) Compose UI tests — createComposeRule + semantics matchers. 3) Instrumented tests — device/emulator for end-to-end flows.

When should you use ML Kit vs Gemini Nano?

ML Kit: vision/text tasks (barcode scanning, OCR, face detection) with broad device support. Gemini Nano: text generation, summarization, and classification — but only on supported hardware, always with a fallback.

Key Takeaways

  • Android has the deepest AI tooling of any mobile platform: Gemini in-IDE, on-device AI, and full Claude Code support.
  • Use Gemini for inline editing, Claude Code for multi-step autonomous tasks — they complement perfectly.
  • Compose UI generation delivers the biggest time savings. Describe a screen, get a working composable.
  • Document the sealed UiState pattern once in CLAUDE.md — AI follows it for every new ViewModel.
  • Room, Retrofit, and ViewModel generation are highly reliable due to their structured, pattern-based nature.
  • Always verify Compose API imports against your actual dependency versions — hallucinated APIs are common.
  • On-device AI: ML Kit for vision/text (broad support), Gemini Nano for LLM features (always needs fallback).
  • A complete, tested Android feature takes ~60 minutes with AI at every layer.
