Chapter 9

Agentic Coding Workflows

Memory Map
💡

The Big Idea

Give the AI a goal, not step-by-step instructions. The agent observes, plans, acts, verifies, and iterates autonomously. Your role shifts from driver to supervisor — you handle the 20% (spec, plan approval, review), the agent handles the 80%.

Chat AI responds to messages. Agentic AI takes initiative. It runs tests, reads errors, fixes bugs, and retries — all without you asking.

🛠️ Agent Capabilities

📖
Read & Search
Navigate entire codebase, find patterns, usages, and definitions across the project.
✏️
Edit & Create
Create new files, modify existing ones with precise diffs. Multi-file coordinated changes.
▶️
Run & Interpret
Execute builds, tests, linters. Read error messages and command output. Fix and retry.
🔀
Git & Branch
Stage changes, write commit messages, create branches. Deliver clean diffs ready for PR.

🎚️ The Trust Spectrum

Trust Level Agent Autonomy Best For
🔴 Full supervision Proposes, you approve each step DB migrations, security, production configs
🟡 Milestone review Runs freely, pauses at checkpoints New features, large refactors, API changes
🟢 Test-gated Runs freely; tests are the guardrail Bug fixes, test generation, documentation
🔵 Full autonomy Runs to completion, you review diff Formatting, linting fixes, boilerplate

Start supervised, loosen over time. Good CLAUDE.md + strong tests = more trust. Never auto-approve destructive operations.

🔐 Free guardrails worth turning on: --permission-mode auto (a classifier model gates approvals — routine work proceeds, scope escalation gets blocked) and /sandbox (OS-level filesystem and network isolation). Pair them on autonomous runs before you start trusting longer sessions.

⚡ Custom Commands (Skills)

What
Markdown files in .claude/commands/ that encode multi-step workflows. Invoke with /command-name.
$ARGUMENTS
Captures everything after the command name. /new-feature user-prefs → $ARGUMENTS = "user-prefs".
Key ingredients
Step-by-step instructions, pattern references, verification steps (tests + lint), git operations.
Team impact
New member runs /new-feature X on day one and gets convention-following code. Commit to repo for team-wide use.
Examples
/new-feature (full feature workflow), /fix-ci (diagnose & fix pipeline), /new-service (scaffold microservice).

🚨 Error Recovery

✅
Handles Well
Build failures (missing imports, type errors), test failures (assertion mismatches), linter errors (style, unused imports).
❌
Handles Poorly
Architectural misalignment (wrong pattern, keeps building on it), silent logic bugs (tests also wrong), scope drift (refactors unrelated files).
🔄
The 3-Attempt Rule
If the agent fails to fix an error after 3 attempts, intervene. Either provide missing context or redirect the approach entirely.
🗑️
Start Fresh > Fix Bad
When the agent goes off track, re-prompting with better constraints is faster than fixing 200 lines of misdirected code. Avoid sunk cost fallacy.

⏱️ Long-Running Agent Strategies

  1. Context Compaction

    Long sessions accumulate stale context. Use /compact to compress old context and free space for what matters now.

  2. Checkpointing

    Commit working intermediate states. If step 7 fails, revert to checkpoint at step 6 instead of starting over.

  3. Sub-Agents

    Main agent orchestrates; sub-agents handle independent tasks in parallel (migration, routes, tests). Like a tech lead delegating to team members.

  4. Agent Teams (since Opus 4.6)

    The same pattern productized: one team-lead session spawns teammates with their own context windows, coordinated via shared task list + messaging. Optional read-only plan mode for risky tasks. Token cost scales with active teammates.

🧪 Quick Quiz

What's the difference between chat-based AI and agentic AI?

Chat AI responds to messages. Agentic AI takes initiative — it reads files, plans, writes code, runs tests, fixes failures, and iterates autonomously. Your role shifts from driver to supervisor.

What are the four levels of the trust spectrum?

Full supervision (approve each step), Milestone review (pause at checkpoints), Test-gated (tests as guardrail), Full autonomy (review the diff). Match trust level to task risk and reversibility.

What are the five layers of safeguards for agentic coding?

Generation checks (CLAUDE.md rules) → Test suite (run after changes) → Pre-commit hooks (lint, secrets) → PR review (human) → CI pipeline (clean environment). Each catches different failure modes.

When should you start fresh instead of fixing the agent's output?

When the agent chose the wrong architecture/pattern and keeps building on it, or after 3 failed fix attempts. Re-prompting with better constraints is faster than fixing 200 lines of misdirected code.

📌 Key Takeaways

  • Agentic = goal-driven. Give the AI a goal, not step-by-step instructions. It observes, plans, acts, verifies, and iterates autonomously.
  • Match trust to risk. Full supervision for destructive ops, test-gated for bug fixes, full autonomy for formatting. Start supervised, loosen over time.
  • Five safeguard layers: CLAUDE.md rules → tests → pre-commit hooks → PR review → CI. No single layer is sufficient.
  • Custom commands scale team workflows. Encode patterns in .claude/commands/ with $ARGUMENTS. New members productive on day one.
  • 3-attempt rule: If the agent can't fix it in 3 tries, intervene. Start fresh beats fixing misdirected code.
  • For long sessions: use /compact, checkpoint with commits, and spawn sub-agents for parallel work.
