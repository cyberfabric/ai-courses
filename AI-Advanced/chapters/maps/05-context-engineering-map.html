Chapter 5

Context Engineering

Memory Map
💡

The Big Idea

The quality of AI output is determined by the quality of context you provide — not your prompt, not the model. Context engineering is curating the smallest set of high-signal tokens that give the model what it needs.

⚖️ Context Engineering vs Prompt Engineering

Aspect Prompt Engineering Context Engineering
Focus How you phrase the request What information surrounds the request
Scope Current message only Everything the model sees
Includes Instruction clarity, constraints, format CLAUDE.md, files, docs, project structure
Impact Important but overrated 10-50x more impactful
The proof Rich prompt + no context = generic code Simple prompt + rich context = project-perfect code

📝 CLAUDE.md: The Layered Architecture

🌍 Global

Personal prefs, style
→
📂 Project
project/CLAUDE.md
Stack, build, architecture
→
📁 Directory
src/api/CLAUDE.md
Module-specific rules

Each layer adds specificity. Project-level is the most important one.

Anatomy of a Great CLAUDE.md

📋
Project Overview
What it does, tech stack, repo layout. The 30-second pitch.
🔨
Build & Test
Exact commands to build, test, lint, deploy. Copy-pasteable.
🏗️
Architecture
Main modules, data flow, design patterns. Point to existing diagrams.
📏
Conventions
Naming, file org, testing approach, commit messages. What passes code review.
🚫
Important Rules
NEVER/ALWAYS rules. No runBlocking, no raw dicts, always add migrations.
📎
Example References
Point to files that exemplify your patterns. "Follow OrderService as template."

The "new developer" test: If a senior dev joining your team day one could write code that passes review using only your CLAUDE.md, it's good enough.

💰 The Context Budget

✅ Include ❌ Exclude
CLAUDE.md (auto-loaded) Entire codebases (let agent search)
Files directly relevant to the task Build artifacts, node_modules, compiled output
1-2 example files showing the pattern Long conversation threads on unrelated topics
Test files showing conventions Redundant docs already in CLAUDE.md
Full stack traces when debugging Marginally related info (decreases quality!)

Think of context like RAM — be intentional about what you load. More noise = worse output.

🦠 Context Rot

📅
Stale CLAUDE.md
Says "use LiveData" but you migrated to StateFlow 6 months ago. Review monthly, add to sprint retro.
📜
Long Conversations
Early messages describe a plan you abandoned. Start fresh sessions for new tasks. Use /compact.
⚔️
Contradictory Instructions
Directory CLAUDE.md vs project CLAUDE.md. Use directory-level files for monorepo differences (camelCase vs snake_case).
👻
Cached Understanding
AI read a file early, file changed since. Be explicit: "Re-read src/auth.py, it was updated."

⚡ Custom Commands (Skills)

What
Reusable prompt templates in .claude/commands/ — invoke with /command-name.
Why
Encode team workflows once. Ten developers running /generate-api produce consistent, convention-matching code.
How
Create a markdown file with step-by-step instructions. Reference existing files as patterns. Include verification steps.
Examples
/generate-api (new endpoint), /review-pr (code review), /new-service (scaffold microservice).
Team impact
Commit commands to the repo. Standardized AI workflows that scale across the team.

🧪 Quick Quiz

What has more impact: a perfectly phrased prompt or rich project context?

Rich context wins every time. A simple prompt with rich context ("Add caching like we did in OrderService") outperforms a detailed prompt with no context. Context has 10-50x more impact than prompt phrasing.

What are the three layers of CLAUDE.md files?

Global (~/.claude/CLAUDE.md — personal prefs), Project (project root — stack, build, architecture), Directory (subdirectory — module-specific rules). Project-level is most important.

What is context rot and how do you prevent it?

Context rot is when information becomes stale, contradictory, or misleading. Prevent it by reviewing CLAUDE.md monthly, starting fresh sessions for new tasks, using /compact, and being explicit when changing direction.

Why is "more context" not always better?

Stuffing the context window with marginally relevant info can decrease output quality. The model has to sort through noise to find signal. Curate aggressively: include what's necessary, exclude what's merely related.

What's the "new developer" test for CLAUDE.md?

Read your CLAUDE.md as if you were a senior dev joining the team on day one. Does it contain everything they'd need to write code that passes code review? If it's clear enough for a new hire, it's clear enough for AI.

📌 Key Takeaways

  • Context > Prompts — rich project context with a simple prompt beats a detailed prompt with no context, every time.
  • CLAUDE.md is your most impactful file. Include: overview, build commands, architecture, conventions, rules, example references.
  • Curate, don't dump. More tokens = more noise. Include what's necessary, exclude what's merely related.
  • Fight context rot — review CLAUDE.md monthly, start fresh sessions, use /compact, resolve contradictions.
  • Structure and docs are free context — clear directories help AI navigate; documentation has immediate ROI now.
  • Custom commands scale AI workflows across teams. Encode patterns once in .claude/commands/.
