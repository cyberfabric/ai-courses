Claude Skills, Subagents, Plugins & MCP — Extending Claude Code

Memory Map
💡

The Big Idea

Claude Code ships a file-based extension stack: five primitives plus two runtime modes plus plugins for distribution. Each piece is a Markdown file or a JSON setting. The leverage comes from using them in concert — matching the right primitive to what you want guaranteed.

🧰 The Five Primitives

📜
CLAUDE.md
Ambient instructions loaded every session. Keep it short.
🧩
Skills
.claude/skills/<n>/SKILL.md — on-demand domain knowledge or workflows.
🤖
Subagents
.claude/agents/<n>.md — isolated context, scoped tools, own model.
🔌
MCP Servers
claude mcp add — integrations to Notion, Linear, Figma, databases.
🪝
Hooks
.claude/settings.json — deterministic scripts at lifecycle points.
📦
Plugins
Installable bundles of all of the above. /plugin opens the marketplace.

🎯 Match the Primitive to the Guarantee

QuestionPrimitive
Persistent context?CLAUDE.md
Knowledge / workflow on demand?Skill
Focused task in its own context?Subagent
Access to an external system?MCP server
Action that must happen, every time?Hook
Bundle to ship to a team?Plugin

🛡️ The Three-Layer Defense

  1. Ambient (CLAUDE.md)

    Claude knows the rule. Advisory.

  2. On-Demand (Skill)

    Claude can apply the rule when a relevant task comes up.

  3. Guaranteed (Hook)

    The rule is applied. Deterministic. No model interpretation.

📖 Key Terms

Skill
A folder under .claude/skills/ with a SKILL.md describing a domain or workflow.
disable-model-invocation
Frontmatter flag that prevents Claude from auto-running a side-effect skill.
Subagent
A delegated worker with its own context window and tool whitelist.
MCP server
External system integration (Notion, Linear, Figma) installed via claude mcp add.
Hook
Deterministic script that runs at lifecycle points (pre-edit, pre-commit, post-tool).
Auto Mode
Permission mode where a Sonnet 4.6 classifier approves safe actions automatically.
Sandbox
OS-level filesystem and network isolation enabled with /sandbox.
Agent Teams
First-class orchestration of 3–5 teammate sessions with a team lead.
Plugin
Installable bundle of skills + subagents + hooks + MCP servers.

🚫 Anti-Patterns

📚
CLAUDE.md sprawl
Conditional knowledge bloats CLAUDE.md until rules get ignored. Move to Skills.
🔧
Subagent overuse
If a Skill works, use it. Subagents pay round-trip context cost.
🛒
Plugin sprawl
Audit plugins quarterly. Uninstall what you haven't used in 30 days.
💥
Auto Mode without sandbox
Pair with sandbox or fresh worktree. Don't run on uncommitted work.

🧠 Quick Quiz

You want to enforce that no commit ever lands with API keys. Skill or Hook?

Hook. Skills are advisory. Hooks are deterministic — the only fit for an invariant.

Your team has a 200-line API conventions doc. Where does it go?

A Skill. CLAUDE.md should stay short. The doc is conditional knowledge that loads on demand when API work comes up.

You're about to read 40 files to understand auth. Main session or Subagent?

Subagent. The 40 files would burn your main context. The subagent reads them in its own window and reports back a summary.

What three risk categories does Auto Mode's classifier check?

Scope escalation, untrusted infrastructure, and prompt-injection-driven actions.

What's the default size of an Agent Team?

3–5 teammates. Bigger teams produce more coordination overhead than parallelism gain.

Key Takeaways

  • Match the primitive to the guarantee: ambient (CLAUDE.md), on-demand (Skill), focused (Subagent), external (MCP), deterministic (Hook).
  • Skills go in .claude/skills/<n>/SKILL.md. Subagents go in .claude/agents/<n>.md. Hooks go in .claude/settings.json.
  • Subagents save context. They're one of the most powerful tools available because context is the fundamental constraint.
  • Auto Mode classifies intent; the sandbox enforces capability. Combine them for blast-radius control.
  • Agent Teams enable parallel sessions with a team lead. Writer/Reviewer is the canonical pattern.
  • Plugins are how you ship the stack to a team. Curate, don't sprawl.
  • Author one Skill, one Subagent, one Hook in your own project this week. That's the difference between knowing and using the stack.
