Chapter 4

Setting Up Your AI Dev Environment

Memory Map
💡

The Big Idea

You need at least one CLI agent and one IDE tool. Configure project-level context files (CLAUDE.md, .cursorrules) for each project — they're what make AI output match your codebase, not generic code.

🏗️ The Four-Layer Stack

  1. Layer 1: Always On — GitHub Copilot

    Tab completion in your IDE. Zero friction, passive assistance. Works in VS Code, JetBrains, Neovim.

  2. Layer 2: Active IDE — Cursor

    Inline edits (Cmd+K), chat panel (Cmd+L), Agent Mode for multi-file tasks. Best when you're deep in a file.

  3. Layer 3: Heavy Lifting — Claude Code

    CLI agent for multi-file features, test generation, refactoring, autonomous workflows. Your primary tool for complex tasks.

  4. Layer 4: Specialized — AS Gemini / v0 / Bolt

    Android-specific work (Gemini), UI prototyping (v0), full-stack scaffolds (Bolt). Reach for when their strength matches your task.

⚠️ The #1 Setup Mistake

📝

Don't Skip the CLAUDE.md

Working with Claude Code without a CLAUDE.md is like hiring a contractor without a brief. Generic code, wrong patterns, hours of corrections. Five minutes of setup saves hours of "no, not like that." Run claude /init, then customize with your team's conventions, architecture, and build commands.

🧪 Quick Quiz

What's the minimum viable AI tool setup?

One CLI agent (Claude Code) + one IDE-integrated tool (Cursor, Copilot, or Gemini). The CLI handles complex multi-file tasks; the IDE handles fast inline edits.

What's the first thing to do after installing Claude Code on a project?

Run claude /init to generate a CLAUDE.md, then review and customize it with your team's conventions, build commands, and architectural decisions.

Which config files should be committed to git, and which should be gitignored?

Commit: CLAUDE.md, .cursorrules, .github/copilot-instructions.md (team resources). Gitignore: .claude/settings.local.json, .cursor/settings.json, .cursor/chat/ (personal settings).

When should you use Copilot vs Claude Code vs Cursor?

Copilot: always-on tab completion (Layer 1). Cursor: inline edits and focused chat (Layer 2). Claude Code: multi-file tasks, test generation, autonomous workflows (Layer 3).

📌 Key Takeaways

  • Four-layer stack: Copilot (always on) → Cursor (active IDE) → Claude Code (heavy lifting) → Specialized tools.
  • CLAUDE.md is non-negotiable. Run claude /init, then customize. It's the foundation for every AI interaction in your project.
  • Project config files are team resources — commit CLAUDE.md, .cursorrules, and copilot-instructions.md to git.
  • Shell aliases speed up daily workflows — cc, cr, ct for quick Claude Code access.
  • Context consistency > tool consistency — standardize project config, let developers choose their preferred IDE tool.
