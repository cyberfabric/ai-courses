The Big Idea

AI participates at every phase of a project — from ideation to deployment. The skills compound: specs feed tests, tests validate implementation, patterns accelerate later features. A 10-week solo project becomes 3-4 weeks.

📋 What Happens at Each Phase

  1. 💭 Ideation & Research

    AI validates feasibility, analyzes competition, assesses scope. Turn a vague idea into a scoped MVP in 30 minutes.

  2. 📐 Architecture Design

    AI proposes architectures with trade-offs. You evaluate against real constraints: team size, budget, scalability needs.

  3. 🏗️ Project Scaffolding

    AI generates boilerplate, CI/CD, project structure, CLAUDE.md. A day's work in 2-4 hours. Biggest time savings.

  4. ⚙️ Feature Development

    Repeat the SDD cycle per feature: Spec → Tests → Implement → Review. Patterns compound — later features go faster.

  5. 🧪 Testing & QA

    Coverage audit, integration tests, security review. Most tests already built with features via TDD.

  6. 📝 Documentation

    AI generates README, API docs, and updates CLAUDE.md from the code it helped create. Accurate because it knows the codebase.

  7. 🚢 Deployment & Monitoring

    AI generates CI/CD pipelines, IaC, monitoring, health checks. From PR gates to production rollback.

🧪 Quick Quiz

Which phase gets the biggest AI speedup?

Scaffolding (5-10x) and documentation (5-10x) get the largest boosts because they're largely boilerplate generation. Feature development gets 2-3x because it still requires spec writing and review.

What's the SDD feature loop?

Spec → Tests → Implement → Review. Write the spec (15 min), AI generates tests (5 min), review tests (10 min), AI implements to pass tests (10 min), review and validate (10 min). ~50 minutes per complete feature.

How does AI-driven development change where your time goes?

From 60-70% typing code to 60-70% thinking: writing specs, reviewing AI output, making architecture decisions, and validating quality. Total time shrinks 2-3x, but the nature of work shifts dramatically.

Why do later features go faster than early ones?

Each feature establishes patterns the AI follows automatically. By Feature 5, conventions are ingrained in CLAUDE.md and the codebase structure. The compound effect accelerates development as the project matures.

Why should you validate AI architecture proposals?

AI lacks context about your team's operational maturity, infrastructure constraints, and complexity tolerance. It might suggest Kubernetes for a project that needs Docker Compose on a single VM. Filter through real-world constraints.

Key Takeaways

  • AI participates at all 7 phases: ideation, architecture, scaffolding, features, testing, docs, deployment.
  • Scaffolding and documentation get 5-10x speedups; feature development gets 2-3x.
  • The feature loop is Spec → Tests → Implement → Review — ~50 min per complete feature.
  • Skills compound: patterns from early features accelerate later ones automatically.
  • Your work shifts from 70% typing to 70% thinking — specs, reviews, architecture decisions.
  • A 10-week solo project becomes 3-4 weeks with AI at every phase.
  • Always validate AI architecture against real constraints: team size, budget, infra maturity.
  • Documentation is a milestone, not an afterthought — AI makes it fast enough to have no excuse.
