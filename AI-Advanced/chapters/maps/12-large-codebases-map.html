Chapter 12

Working with Large Codebases

Memory Map
💡

The Big Idea

Large codebases don't fit in any context window. The skill isn't loading more — it's loading the right 20% and trusting the agent to navigate the rest. Architecture documentation is your AI's map through the maze.

📊 The Context Problem at Scale

📏
1M Token Window
~25,000 lines of code. Sounds huge until your codebase has millions of lines.
📈
10-100x Overflow
Typical enterprise codebases exceed the context window by this factor.
🎯
80% Irrelevant
For any given task, 80% of the codebase is noise. The art is identifying the 20% that matters.
🤿
Drowning in Noise
Too much context is as bad as too little. The model loses focus when overwhelmed.

🏗️ The Three-Tier Context Strategy

  1. Tier 1: Always Loaded (CLAUDE.md)

    Architecture overview, conventions, build commands. Permanent context, 500-1000 lines max.

  2. Tier 2: Task-Relevant Files

    Files directly involved in the current task + immediate dependencies. Let the agent discover these via search.

  3. Tier 3: Reference Only

    Example files to follow by path reference ("follow the pattern in src/services/OrderService.ts") without loading contents.

🧭 Trust the agent's navigation. Give starting points and boundaries, not file dumps. Claude Code can search, read files on demand, and follow import chains — often better than manual pre-loading.

Wrap
AI creates adapter interfaces around legacy modules, exposing clean APIs.
Build
AI implements the new version behind the same interfaces.
Route
Feature flags toggle between legacy and new implementations.
Verify
Both run in parallel; compare outputs for parity.
Remove
Once parity confirmed, delete legacy code and adapters.

📦 Monorepo Strategies

🎯
Scope to Module
Navigate to the specific module first. Tell AI explicit boundaries: "Only look at code within this module."

Define module owners in root CLAUDE.md. Changes to payments need payments team review.
🚧
Cross-Module Rules
Services communicate only via API gateway or message queue. Never import another service's internals.

Module-specific instructions loaded when the agent enters that directory. Critical rules, architecture, testing setup.

🧪 Quick Quiz

What are the three tiers of context loading for large codebases?

Tier 1: Always loaded (CLAUDE.md). Tier 2: Task-relevant files discovered by the agent. Tier 3: Reference-only files pointed to by path without loading contents.

Why are ADRs (Architecture Decision Records) critical for AI?

ADRs explain why decisions were made. Without them, AI may "helpfully" refactor patterns back to approaches that were deliberately rejected — like switching from polling to WebSockets when your load balancer doesn't support sticky sessions.

What's the strangler fig pattern?

Incrementally replacing legacy code by: wrapping it with adapters → building new code behind the same interfaces → routing traffic with feature flags → verifying parity → removing the legacy. Both old and new run in parallel during migration.

Why should you refactor one module per commit?

If something breaks, you need to know which module's change caused it. One commit per module gives precise revert granularity. A single massive commit across the codebase makes debugging regressions nearly impossible.

What can AI do well with legacy code, and what can't it do?

AI excels at describing what code does: mapping data flow, listing functions, flagging risks. But it can't explain why business rules exist. Pair AI's analysis with human domain knowledge for complete documentation.

Key Takeaways

  • Large codebases exceed any context window — load the right 20%, not everything.
  • Use three-tier context: always-loaded CLAUDE.md, task-relevant files via agent search, reference-only paths.
  • Architecture documentation (ADRs, module maps, directory CLAUDE.md) is your AI's navigation map.
  • Refactor one module per commit with tests after each step — never a whole-codebase single commit.
  • The strangler fig pattern pairs perfectly with AI: wrap, build new, route traffic, verify, remove legacy.
  • AI is an excellent code archaeologist for undocumented modules — but verify business rules with humans.
  • In monorepos, scope AI to modules with explicit boundaries and ownership rules in CLAUDE.md.
  • Trust the agent to navigate — give starting points and constraints, not file dumps.
