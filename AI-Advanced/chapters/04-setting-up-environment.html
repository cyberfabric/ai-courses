Claude Code: Your Primary Agent

Claude Code is the tool you'll use most in this book. It's a CLI agent that reads your codebase, edits files, runs commands, and iterates autonomously. Available as a terminal CLI, desktop app (Mac/Windows), web app, and IDE extensions.

Installation

# Install via npm (requires Node.js 18+)
$ npm install -g @anthropic-ai/claude-code

# Verify installation
$ claude --version

# First run — authenticates via browser
$ claude

On first run, Claude Code opens a browser window for authentication. You'll need either an Anthropic API key (pay-per-use) or a Claude Max subscription (flat monthly rate with higher limits).

Which plan to choose: If you're experimenting, start with API credits — you'll pay only for what you use. If you're using Claude Code daily for work, the Max plan is almost always cheaper and removes the cognitive overhead of watching token costs.

Your First Command

Navigate to any project directory and run:

# Start Claude Code in your project
$ cd ~/projects/my-app
$ claude

# Or give it a direct task
$ claude "Explain the architecture of this project. What are the main
  modules and how do they interact?"

# Claude reads your files, understands the structure, and gives you
# a clear overview — a great way to verify it's working correctly.

Backend Try your first real task: claude "List all API endpoints in this project with their HTTP methods and a one-line description of each." This tests that Claude Code can navigate your codebase and produce useful output without changing anything.

Project Initialization

The most important step after installation: initialize Claude Code's project memory.

# Generate a CLAUDE.md file for your project
$ claude /init

# Claude analyzes your project structure, dependencies, build system,
# and conventions, then creates a CLAUDE.md with:
# - Project description and tech stack
# - Build and test commands
# - Code conventions it detected
# - File structure overview

Review and edit the generated CLAUDE.md. It's the foundation of everything Claude Code does in your project. Add your team's conventions, architectural decisions, and any rules that matter. We'll cover this in depth in Chapter 5.

Don't skip the CLAUDE.md. Working with Claude Code without a CLAUDE.md is like hiring a contractor without a brief. It'll produce generic code that doesn't match your project's patterns. Five minutes setting up CLAUDE.md saves hours of "no, not like that" corrections.

Key Configuration

# Set your preferred model (in ~/.claude/settings.json)
# Current top model: Opus 4.7 (released 2026-04-16).
# Other tiers: Sonnet 4.6 (claude-sonnet-4-6) for speed/cost,
# Haiku 4.5 (claude-haiku-4-5) for cheapest near-frontier work.
{
  "model": "claude-opus-4-7",
  "permissions": {
    "allow": ["read", "write", "execute"],
    "deny": ["delete"]
  }
}

# Project-level settings (in your-project/.claude/settings.json)
{
  "allowedTools": ["bash", "read", "write", "glob", "grep"],
  "mcpServers": {}
}
Permission levels: Claude Code asks for permission before risky actions by default. Start with the default permission mode and loosen it as you build trust. You can approve actions individually, by category, or set auto-approve for specific tool types. Two newer escape hatches worth knowing: auto mode (claude --permission-mode auto) lets a separate classifier model gate approvals — routine work proceeds, scope escalation gets blocked — and /sandbox wraps the session in OS-level filesystem and network isolation so Claude can move freely inside a defined boundary. See Best Practices for Claude Code for details.

Cursor: Your AI-First IDE

Cursor is a VS Code fork with deeply integrated AI. If VS Code is your editor of choice, Cursor gives you everything VS Code has plus AI superpowers.

Installation & Setup

  1. Download from cursor.com
  2. Import your VS Code settings (Cursor offers this on first launch)
  3. Sign in and choose your plan (free tier includes limited AI completions)
  4. Select your preferred model: Settings → Cursor → Models

Essential Configuration

Create a .cursorrules file in your project root. This is Cursor's equivalent of CLAUDE.md — project-specific instructions that guide AI behavior:

# .cursorrules — Project rules for Cursor AI
# Place in your project root

You are working on a Kotlin Android app using Jetpack Compose.

Tech stack:
- Kotlin 2.0, Jetpack Compose with Material 3
- Room for local database, Retrofit for networking
- Hilt for dependency injection
- MVVM architecture with UiState pattern

Conventions:
- Use StateFlow for UI state, not LiveData
- All composables go in ui/ package, grouped by feature
- ViewModels use sealed interfaces for UiState
- Repository pattern for data access
- Write unit tests for ViewModels, UI tests for screens

Do not:
- Use GlobalScope or runBlocking in production code
- Create god classes or god ViewModels
- Use deprecated Android APIs

Android Dev The .cursorrules file above is a real starting point for Android projects. Customize it with your actual package structure, naming conventions, and architectural patterns. The more specific you are, the better Cursor's suggestions will be.

Key Features to Learn

  • Tab completion — Accept inline suggestions with Tab. Cursor predicts your next edit based on context.
  • Cmd+K (inline edit) — Select code, press Cmd+K, describe the change. Cursor edits in place.
  • Cmd+L (chat panel) — Open the AI chat panel for longer conversations about your code.
  • Agent Mode — Let Cursor autonomously make multi-file changes. Access via the chat panel.
  • @ references — Mention files, symbols, or docs with @filename to add context to your prompts.

GitHub Copilot: The Universal Assistant

Copilot works in almost every editor and is the most widely adopted AI coding tool. It's strongest for tab completion and inline suggestions, with an evolving Agent Mode.

Setup in VS Code

  1. Install the "GitHub Copilot" extension from the VS Code marketplace
  2. Sign in with your GitHub account
  3. Enable Copilot: check the icon in the status bar

Setup in Android Studio / JetBrains

  1. Settings → Plugins → Marketplace → search "GitHub Copilot"
  2. Install and restart the IDE
  3. Sign in via Settings → GitHub Copilot

Project Configuration

Copilot supports project-level instructions via a .github/copilot-instructions.md file:

# .github/copilot-instructions.md

This is a Python FastAPI backend service.

Architecture:
- FastAPI with async handlers
- SQLAlchemy 2.0 with async sessions
- Pydantic v2 for request/response models
- Alembic for database migrations

Conventions:
- Type hints on all function signatures
- Docstrings on public functions (Google style)
- Tests use pytest with async fixtures
- Error responses follow RFC 7807 (Problem Details)

Full-Stack If you work across multiple repos with different stacks, Copilot's project-level instructions let you maintain different conventions per project. Your React frontend repo gets React rules; your Python backend repo gets Python rules. Same tool, different context.

Copilot + Claude Code together: Use Copilot for fast tab completions while typing (low friction, always on). Use Claude Code for complex multi-file tasks that need autonomous execution. They complement each other well — Copilot for the small moments, Claude Code for the big moves.

Android Studio Gemini

Android Dev If you build Android apps, Gemini is built into your primary IDE. No plugin needed — it has been part of Android Studio since Ladybug (2024.2). The current stable channel is Panda 4 (2025.3.4); the active Canary preview is Quail 1. The Otter Feature Drop in January 2026 brought the major Agent Mode improvements you'll see referenced in current Android tooling docs — if you're on anything older than Otter, update before you go further.

Enabling Gemini

  1. Update to Android Studio Panda 4 (stable) or pick up Quail 1 from the Canary channel for the bleeding edge
  2. Settings → Tools → Gemini → Enable
  3. Sign in with your Google account
  4. For Agent Mode: Settings → Tools → Gemini → Enable Agent Mode

What Gemini Does in Android Studio

  • Code completion — Context-aware suggestions as you type
  • Code transforms — Select code, right-click → Gemini → transform (refactor, explain, add docs)
  • Commit messages — Auto-generated commit messages from staged changes
  • Agent Mode — Multi-file autonomous tasks: "Add Room database support for a User entity with CRUD operations"
  • Build analysis — AI-powered suggestions for Gradle build errors and dependency conflicts
Gemini's Android advantage: Because Gemini is trained on Android documentation and integrated with the Android SDK, it understands Compose previews, Gradle DSL, manifest permissions, and Android lifecycle in ways that general-purpose models sometimes miss. Use it for Android-specific tasks, and Claude Code for cross-cutting concerns. See New Agentic Experiences for Android Studio for the latest capabilities.

Terminal Setup

A well-configured terminal makes AI-driven development smoother. These tweaks take minutes and pay off daily.

Shell Aliases

# Add to ~/.bashrc or ~/.zshrc

# Quick Claude Code access
alias cc="claude"
alias ccp="claude --print"       # Print mode (no file edits)
alias cci="claude /init"         # Initialize project

# Common AI-assisted tasks
alias cr="claude 'Review the staged changes for bugs, security
  issues, and style problems. Be concise.'"
alias ct="claude 'Run the test suite and fix any failures.'"
alias cdoc="claude 'Generate documentation for the public API
  in this module.'"

# Git + AI workflow
alias gcai="git add -A && claude 'Write a commit message for
  the staged changes. Follow conventional commits format.'"

DevOps Add aliases for your infrastructure workflow:

# DevOps-specific aliases
alias cinfra="claude 'Review this Terraform/Kubernetes config for
  security misconfigurations, missing resource limits, and
  cost optimization opportunities.'"
alias cpipe="claude 'Analyze this CI/CD pipeline for improvements:
  caching, parallelization, and unnecessary steps.'"

Shell Integration

Claude Code's shell integration connects your terminal to the AI. When enabled, Claude Code can see your terminal output and provide contextual help:

# Enable shell integration (added automatically on install)
# Verify it's working:
$ claude /doctor

# If missing, re-initialize:
$ claude /install-shell-integration

The Ideal Multi-Tool Setup

After setting up individual tools, here's how they fit together in practice:

The Recommended Stack

Layer 1 — Always on: GitHub Copilot for tab completion in your IDE. Zero-friction, passive assistance.
Layer 2 — Active IDE: Cursor for inline edits and focused chat when you're deep in a file.
Layer 3 — Heavy lifting: Claude Code for multi-file tasks, test generation, refactoring, and autonomous workflows.
Layer 4 — Specialized: Android Studio Gemini for Android-specific work. v0/Bolt for UI prototyping.

Team Lead When standardizing tools across your team, start with Claude Code + one IDE tool. Don't mandate the IDE choice (Cursor vs Copilot vs Windsurf) — let developers use what they're comfortable with. But do standardize the project configuration: a shared CLAUDE.md, a shared .cursorrules, and shared Copilot instructions. The consistency of context matters more than the consistency of tools.

Freelancer Your setup should be portable. Claude Code + Cursor covers 95% of use cases across any tech stack. Keep a personal template of CLAUDE.md and .cursorrules that you customize for each client project. The speed at which you set up AI tooling on a new project is a competitive advantage.

Project-Level Configuration Files

Every AI tool reads project-level configuration. Here's the complete picture of what goes where:

File / Directory Tool Purpose
CLAUDE.md Claude Code Project instructions, conventions, build commands, architecture notes
.claude/ Claude Code Settings, custom commands/skills, hooks, MCP server configs
.cursorrules Cursor Project rules and conventions for Cursor AI
.cursor/ Cursor Cursor project settings, prompt templates
.github/copilot-instructions.md GitHub Copilot Project instructions for Copilot suggestions
.gitignore additions All Exclude tool-specific local caches and personal settings
Commit your config files. CLAUDE.md, .cursorrules, and .github/copilot-instructions.md should be in version control. They're team resources, not personal preferences. When one developer discovers a convention that improves AI output, the whole team benefits.

What to Add to .gitignore

# AI tool local caches (add to .gitignore)

.cursor/settings.json
.cursor/chat/

Verify Your Setup

Run through this checklist to confirm everything is working:

  • Claude Code: Run claude --version — should print a version number
  • Claude Code: Run claude "What files are in this directory?" — should list your project files
  • CLAUDE.md: Run claude /init in your project — should generate a CLAUDE.md
  • Cursor: Open a file, press Cmd+K, type a change — should generate inline edits
  • Copilot: Start typing a function — should show ghost text suggestions
  • Gemini (Android): Open a Kotlin file, right-click → Gemini should appear in the menu

Exercise: Set Up Claude Code on a Real Project

Take an existing project you're actively working on and fully configure Claude Code:

  1. Install Claude Code if you haven't already: npm install -g @anthropic-ai/claude-code
  2. Navigate to your project and run claude /init
  3. Review the generated CLAUDE.md — edit it to add:
    • Your team's coding conventions
    • The build and test commands
    • Any architectural decisions or constraints
    • Patterns the AI should follow (reference existing files as examples)
  4. Run your first real task: Pick something small but useful:
    • "Write unit tests for [specific class]"
    • "Add input validation to [specific function]"
    • "Explain the architecture of the [specific module]"
  5. Review the output using the mindset from Chapter 3: check for correctness, style match, and architectural fit.
  6. Set up one shell alias from the terminal section that matches your daily workflow.

This exercise gets you from "installed" to "productive" with Claude Code. The CLAUDE.md you create here becomes the foundation for all the techniques in the remaining chapters.
