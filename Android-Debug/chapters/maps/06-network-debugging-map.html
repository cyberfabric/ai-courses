The Big Idea

Network bugs feel random because you can't see the wire. But with OkHttp logging, every request and response is visible. Capture the full HTTP conversation, paste it into Claude Code, and the bug that took an hour from a vague error message takes 5 minutes with the full log.

🔄 The Network Debug Flow

Capture HTTP logs

Get root cause + fix

38% of mobile app bugs involve the network layer. 60% are only intermittently reproducible. Logs are your flashlight.

📊 HttpLoggingInterceptor Levels

Level What It Logs When to Use
NONE Nothing Release builds
BASIC Method, URL, response code, timing Quick call pattern overview
HEADERS Basic + all headers Auth, caching, content-type issues
BODY Headers + full request/response bodies Debug builds only! ⚠️
⚠️ Never log bodies in release builds — they contain auth tokens, personal data, and payment info. Use Level.BODY only behind BuildConfig.DEBUG.

🪲 The Network Bug Hall of Fame

🔑
Token Refresh Race
Refresh returns 200 with new token, but retried request still sends the old token. Fix: rebuild the request with fresh token in Authenticator.
🔒
Certificate Pinning Failure
Server cert rotated but pins weren't updated. Always pin intermediate CA (not leaf) and keep a backup pin from a different provider.
⏱️
60-Second Timeouts
Users abandon after 8 seconds. Use aggressive connect timeout (10s), longer read timeout (30s), and add retry logic for transient failures.
📡
Duplicate API Calls
Multiple ViewModels call the same endpoint independently. Fix: repository-level deduplication with Deferred to share in-flight requests.
💾
No Caching
Same static data fetched every session. Configure OkHttp Cache (50-100 MB) and set Cache-Control per endpoint based on change frequency.
🤝
TLS Version Mismatch
SSLHandshakeException on older devices. Server requires TLS 1.3, device only supports 1.2. Fix: configure ConnectionSpec with compatible versions.

🔐 Thread-Safe Token Refresh: The Pattern

  1. Request gets 401

    OkHttp calls your Authenticator.authenticate().

  2. Synchronized check

    Compare the request's token with the stored token. If they match, nobody has refreshed yet.

  3. Refresh (or reuse)

    If stale → call refresh endpoint. If already refreshed by another thread → use the current stored token.

  4. Build NEW request

    response.request.newBuilder().header("Authorization", "Bearer $freshToken") — don't reuse the old request object!

  5. Prevent infinite loops

    Add a marker header (X-Retry-Auth). If it's already present, return null to stop retrying.

📖 Key Terms

HttpLoggingInterceptor
OkHttp interceptor that logs HTTP request/response details at configurable verbosity levels.
OkHttp 5.x
Current major line (5.0 shipped July 2025; 5.3.2 latest). Breaking changes vs 4.x: split JVM/Android artifacts, MockWebServer moved to mockwebserver3 coordinate & package, timeouts use kotlin.time.Duration.
Authenticator
OkHttp interface called automatically on 401 responses — your hook for token refresh and request retry.
Certificate Pinning
Hardcoding expected server certificate hashes in the app to prevent man-in-the-middle attacks. Pin intermediate CAs, not leaf certs.
Cache-Control
HTTP header controlling how responses are cached. no-store = never cache, max-age=N = cache for N seconds.
Request Deduplication
Sharing a single in-flight network request across multiple callers instead of making duplicate calls. Use Deferred in your repository.
ConnectionSpec
OkHttp class that configures which TLS versions and cipher suites are acceptable for a connection.
Interceptor
OkHttp middleware that can observe, modify, or short-circuit HTTP requests and responses. Application interceptors run first; network interceptors run last.

🧪 Quick Quiz

Your token refresh returns 200 but the next request still gets 401. What's wrong?

The retried request is using the old request object with the stale token. The Authenticator must build a new request via response.request.newBuilder() and set the fresh Authorization header.

Why should you pin the intermediate CA certificate instead of the leaf certificate?

Leaf certificates rotate frequently (often every 90 days with Let's Encrypt). Intermediate CA certs last years. Pinning the leaf means your app breaks on every certificate rotation.

Three ViewModels each call getUser() on app startup. How do you prevent 3 identical network requests?

Add in-flight request deduplication in the repository layer. Store the Deferred from the first request; subsequent callers await() the same Deferred instead of launching new requests.

What's the risk of using HttpLoggingInterceptor.Level.BODY in a release build?

Response bodies can contain auth tokens, personal data, and payment information. Logging them in release exposes sensitive data to anyone who can read logcat (other apps on rooted devices, ADB users, crash reporters).

Key Takeaways

  • Enable HttpLoggingInterceptor at Level.BODY in debug builds — the full HTTP log is your most powerful network debugging tool.
  • The #1 auth bug: token refresh succeeds but the retried request reuses the old token. Always rebuild the request in your Authenticator.
  • Pin intermediate CA certificates (not leaf), always have a backup pin, and use a pin from a second CA provider.
  • Set realistic timeouts (10s connect, 30s read) and add retry logic — 60-second timeouts just make users wait before failing.
  • Deduplicate in-flight requests and configure OkHttp caching — a real shopping app went from 127 to 52 API calls per session.
