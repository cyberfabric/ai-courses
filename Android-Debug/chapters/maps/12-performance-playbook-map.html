The Big Idea

Tools without a system are just a messy drawer. Performance budgets define "good enough" with real numbers. Benchmarks enforce them in CI. A debugging decision tree routes any symptom to the right skill in seconds. Now go build something — and when it breaks, you know exactly what to do.

🧪 Quick Quiz

What's the difference between a performance "wish" and a performance "budget"?

A wish is "the app should be fast." A budget is "cold start must be under 800ms on a Pixel 6." Budgets are concrete thresholds that can be measured and enforced in CI.

How does Macrobenchmark differ from Microbenchmark?

Macrobenchmark measures full app scenarios (startup, scroll) on a real device from the outside. Microbenchmark measures individual functions (e.g., PriceFormatter.format()) in-process. Use Macro for user-facing budgets, Micro for hot code paths.

A benchmark shows cold start went from 935ms to 1,247ms. What's the most likely cause?

A new SDK initialization running synchronously in Application.onCreate(). The +312ms regression is the classic sign of a "one-line" SDK integration that carries hidden startup cost. Fix: defer non-critical SDKs to background threads.

Your app's UI is janky. Which two skills might help, and how do you choose?

Use /analyze-recomposition (Ch 5) if you're using Compose — check recomposition counts and stability. Use /profile-cpu (Ch 9) for deeper analysis — method trace with frame timing breakdown. Start with recomposition if Compose, profiling if View-based.

Key Takeaways

  • A performance budget is a concrete number you commit to not exceeding — not "be fast" but "<800ms cold start."
  • Use Macrobenchmark for startup and scroll, Microbenchmark for hot functions. Run both in CI to catch regressions before merge.
  • The debugging decision tree routes any symptom to the right skill: crashes → /analyze-crash, jank → /profile-cpu, flaky tests → /diagnose-flaky.
  • You built 15 reusable skills across 12 chapters. Commit them to .claude/commands/ so your entire team benefits.
  • APK size creeps — use apkanalyzer to catch uncompressed PNGs, unused native libs, and bundled fonts.
  • Start the 30-day challenge: one debugging exercise per workday for 4 weeks. By the end, AI-assisted debugging is muscle memory.
