🚨 NSURLError Code Cheat Sheet

Code Name Common Cause
-1001 TimedOut Server not responding, DNS failure, too-short timeout
-1003 CannotFindHost DNS resolution failed. Typo in hostname or no network.
-1009 NotConnectedToInternet Device offline. Check reachability before retrying.
-1200 SecureConnectionFailed SSL/TLS handshake failed. Certificate or ATS issue.
-1202 ServerCertificateUntrusted Expired cert, wrong domain, missing intermediate.
-999 Cancelled Task cancelled. Often expected — check Task lifecycle if not.

🛠️ Traffic Capture Tools

📊
URLSessionTaskMetrics
Built-in. DNS time, TLS handshake, total duration, connection reuse. No third-party tools needed.
📝
os_log (Network)
Custom structured logging per request. Subsystem + category filtering. Survives app termination.
🦊
Proxyman
Native macOS/iOS app. Auto-configures simulator. JSON highlighting. Export as HAR for AI analysis.
🔵
Charles Proxy
Industry standard. Manual proxy config. SSL proxying with root cert. Map Local/Remote for mocking.

⚠️ async/await Networking Pitfalls

  1. 🧟 Forgotten Cancellation

    Unscoped Task { } lives forever. Fix: use .task modifier — auto-cancelled when view disappears.

  2. 🐌 Serial When Should Be Parallel

    Three await calls in sequence = 3 round trips. Fix: async let for independent requests.

  3. 🧊 Main-Thread JSON Decoding

    Large JSON decoded in @MainActor context = UI hitch. Fix: Task.detached for heavy decoding.

🔐 SSL Pinning Essentials

✅
Pin to SPKI Hash
Pin the public key hash, not the full certificate. Survives cert rotation if the key stays the same.
🔑
Always Have Backup Pins
Include current + next CA key hashes. If one expires, the backup keeps users connected.
🚨
Kill Switch
Implement a bypass URL that disables pinning in emergencies. Without it, a cert rotation bricks the app.
🐛
Debug Proxy Conflict
SSL pinning blocks Proxyman/Charles. Use #if DEBUG to disable during development.

📖 Key Terms

URLSessionTaskMetrics
Built-in API providing DNS, TLS, and total timing for every URLSession request.
App Transport Security (ATS)
iOS enforces HTTPS by default. Misconfigured ATS exceptions cause silent connection failures.
SSL Pinning
Validates server certificate against a known hash. Prevents MITM attacks but can brick your app if the cert rotates.
HAR (HTTP Archive)
Standard format for exporting captured network traffic from Proxyman or Charles. AI-readable structured data.
waitsForConnectivity
URLSession waits for network instead of failing immediately. Essential for mobile-first apps with spotty connectivity.
Retry storm
Timeout × retry count = total wait. Without exponential backoff, retries hammer an already-struggling server.
.task modifier
SwiftUI modifier that auto-cancels the async task when the view disappears. Use instead of unscoped Task { }.
async let
Runs async operations in parallel. Use for independent requests instead of sequential await calls.

🧪 Quick Quiz

What's wrong with a 60-second timeout and 3 retries?

60s × 4 attempts = 4 minutes of waiting before the user sees an error. Set timeouts to 15-30s for APIs, add exponential backoff between retries, and always support cancellation.

Why does SSL pinning break Proxyman and Charles Proxy?

The proxy inserts its own certificate to decrypt HTTPS traffic. SSL pinning rejects any certificate that doesn't match the pinned hash. Use #if DEBUG to disable pinning during development.

How do you parallelize two independent network calls in async/await?

Use async let: async let posts = fetchPosts(); async let notifications = fetchNotifications(); let (p, n) = try await (posts, notifications). Both fire immediately and you await both results together.

Why use .task instead of Task { } for network calls in SwiftUI?

.task is automatically cancelled when the view disappears. An unscoped Task { } lives forever — the request completes and tries to update state on a deallocated view.

What structured data should you feed to /analyze-network?

URLSession logs (request/response pairs with timing), Proxyman/Charles HAR exports, URLSessionTaskMetrics output, NSURLError codes, and the Swift source code of your networking layer.

📌 Key Takeaways

  • Make your network talk — URLSessionTaskMetrics, os_log, or Proxyman. Can't debug what you can't see.
  • NSURLError codes are your first diagnostic clue — AI classifies them instantly.
  • SSL pinning = silent app killer. Pin to SPKI hash, keep backup pins, have a kill switch.
  • Use .task for view-scoped network calls — automatic cancellation when views disappear.
  • Parallelize with async let — don't pay serial round-trip costs for independent requests.
  • Audit URLSessionConfiguration — 60s timeout and 7-day resource timeout are rarely what you want.
  • Build /analyze-network for bulk log analysis, retry storm detection, and config audits.
