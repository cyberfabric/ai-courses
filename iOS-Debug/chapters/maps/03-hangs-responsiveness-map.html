The Big Idea

Hangs are worse than crashes — users abandon frozen apps and never come back. The async keyword doesn't mean non-blocking: a synchronous call inside an async @MainActor method still freezes the UI.

🎯 The 6 Most Common Hang Patterns

  1. 🌐 Sync Network on Main Thread

    Data(contentsOf: url) or semaphore-wrapped URLSession. Use async/await or completion handlers.

  2. 🗄️ Heavy Core Data Fetch

    NSFetchRequest with no predicate on 100K+ rows, or sync migration in didFinishLaunching.

  3. 🔐 Sync Keychain Access

    SecItemCopyMatching can take seconds when device is locked or Keychain is large.

  4. 🖼️ Main Thread Image Decode

    Loading large JPEGs with UIImage(named:) — decode happens lazily on first render.

  5. 📐 Excessive Auto Layout

    Hundreds of constraints invalidated repeatedly in one layout pass. Common with dynamic table cells.

  6. 🔒 Lock Contention

    Main thread waits for a lock held by a background thread doing heavy work.

⚠️ Swift Concurrency Hang Traps

🧊
Sync Inside @MainActor async
Thread.sleep() or Data(contentsOf:) inside an async method still blocks. async = can suspend, not = every line is non-blocking.
⬇️
Task Priority Inversion
High-priority MainActor task awaits a .background priority task that keeps getting preempted.
🏴‍☠️
nonisolated Misuse
nonisolated method accessing @MainActor properties = data race. Swift 6 catches it; Swift 5.x hangs silently.

📖 Key Terms

Hang
UI is frozen — app is alive but unresponsive. Worse than a crash for user experience and retention.
Watchdog (0x8BADF00D)
iOS system monitor that terminates apps whose main thread is blocked beyond the threshold (~10-20s depending on state).
MXHangDiagnostic
MetricKit payload with symbolicated call stacks for hangs 250ms+ in production. Delivered once per day.
Thread Checker
Xcode runtime diagnostic that fires purple warnings when UI APIs are called off the main thread.
os_signpost
Instruments API to mark begin/end of code intervals. Shows labeled durations in the Instruments timeline.
@MainActor
Swift actor that runs code on the main thread. Blocking calls inside it freeze the UI even in async methods.
Priority inversion
High-priority task waits for a low-priority task that keeps getting preempted by other high-priority work.
Task.detached
Creates a task that doesn't inherit the caller's actor context. Use to move heavy work off @MainActor.

🧪 Quick Quiz

Why are hangs worse than crashes?

Users can't tell if the app is still working. They get frustrated, force-quit, and often don't come back. Hangs also don't show up in crash reports unless you're specifically looking (MetricKit, Xcode Organizer Hangs tab).

Does marking a method async prevent it from blocking the main thread?

No. async means the function can suspend at await points, but synchronous calls like Thread.sleep(), Data(contentsOf:), or SecItemCopyMatching inside it still block whichever thread/actor it's running on.

What's the watchdog timeout for app launch?

~20 seconds. If didFinishLaunchingWithOptions doesn't return in time (e.g., sync Core Data migration), iOS kills the app with code 0x8BADF00D.

How does MXHangDiagnostic differ from a watchdog crash?

MXHangDiagnostic catches shorter hangs (250ms+) that don't trigger the watchdog. Watchdog only kills for multi-second blocks. MetricKit delivers symbolicated call stacks daily — no dSYM needed.

What's the fix for a sync blocking call inside a @MainActor method?

Move the heavy work to Task.detached(priority:) so it runs off the MainActor. Show a loading state on the main thread, await the result, then update UI.

📌 Key Takeaways

  • Hangs are worse than crashes. 53% of users abandon an app after a 3-second freeze.
  • Watchdog (0x8BADF00D) timeouts: ~20s launch, ~10s foreground, ~5-10s background.
  • MetricKit MXHangDiagnostic catches hangs from 250ms+ in production with symbolicated stacks.
  • async ≠ non-blocking. Sync calls inside @MainActor async methods still freeze the UI.
  • Use Time Profiler to capture, /analyze-hang to diagnose, os_signpost to verify.
  • Enable Thread Checker — catch main-thread violations before users do.
