Chapter 8

Browser Apps: ChatGPT & Claude.ai

Simpler harnesses, same principles — Swiss Army knives vs the full workshop

We've spent two chapters deep in the machinery of coding agents — tools with 18+ functions, sub-agents, git worktrees, five-layer permission systems. Now let's step back and look at the tools billions of people use every day: ChatGPT, Claude.ai, and Gemini.

These are harnesses too. Simpler ones, but built on exactly the same principles. And once you see them through the harness lens, you'll understand why they're great at conversation but struggle with coding — and it has nothing to do with the model's intelligence.

Swiss Army Knives vs. the Full Workshop

Browser chat apps are like Swiss Army knives. Versatile, portable, always in your pocket. They have a blade (text generation), a screwdriver (code interpreter), a corkscrew (web search), and a few other tools. You can solve a surprising number of problems with them.

Coding agents are like a full workshop. Table saw, drill press, lathe, clamps, workbench. Specialized, powerful, built for serious projects. You wouldn't carry a workshop in your pocket, but you also wouldn't build a cabinet with a Swiss Army knife.

The difference isn't the blade's sharpness (the model). It's the tooling around it (the harness). Let's see exactly what each browser app has — and what it doesn't.

The Tool Evolution: From Text-Only to Multi-Tool

It's easy to forget that ChatGPT launched in November 2022 as a text-only harness — no web search, no code execution, no images, no file upload. Just a system prompt and a conversation window. Every tool you see today was added through harness engineering.

The evolution tells the story of harness engineering in miniature:

  • 2022 — Text-only chat (system prompt + conversation)
  • 2023 — Code Interpreter, web browsing, DALL-E, file upload (four major tools)
  • 2024 — Custom GPTs, Canvas, memory feature, improved search
  • 2025-2026 — Projects, voice, deeper API integrations

The model didn't change as dramatically as the harness did. GPT-4 was available throughout most of this evolution. What changed was the infrastructure around it — each new tool made the same model useful for new categories of tasks.

ChatGPT's Harness

ChatGPT is the most widely used AI interface in the world. Under the hood, it's a harness with a system prompt, a handful of tools, a memory system, and some interesting guardrails.

The System Prompt

ChatGPT has a detailed system prompt that shapes its behavior — tone, personality, tool usage rules, safety constraints. Leaked versions of this prompt reveal explicit instructions like "avoid web search for information already in training data" and detailed rules about when to use each tool.

The system prompt is what makes ChatGPT feel like "ChatGPT" rather than raw GPT-4. It defines the conversational style, the helpfulness, and the guardrails that prevent certain outputs. Change the system prompt, and the same model behaves completely differently — which is exactly what Custom GPTs do.

The Tools

ChatGPT has a small but powerful tool set:

Tool What It Does Limitation
Web Search Searches the web for current information Fires selectively, not on every query; English-only search
Code Interpreter Executes Python in a sandboxed container No HTTP requests; ephemeral environment; no filesystem
Canvas Browser-side Python via WebAssembly (Pyodide) Can make API calls but can't access uploaded files
DALL-E Generates images from text descriptions No fine-grained editing; style constraints
File Upload Upload files for analysis or processing Files only accessible in Code Interpreter sandbox

Compare this to Claude Code's 18+ tools with full filesystem access, terminal execution, and git integration. ChatGPT's toolbox is a fraction of the size — but it's enough for conversational tasks, data analysis, and quick prototyping.

Memory

ChatGPT has two memory systems, both harness-level:

  • Saved Memories — Facts the model extracts and stores across sessions ("user prefers Python," "user works at Acme Corp"). Injected into the system prompt as a "Memory Set Context" section. On by default.
  • Chat History — Past conversations searchable for context. Separate from saved memories — you can toggle each independently in settings.
ChatGPT's memory has a security implication worth knowing about. Researchers have demonstrated memory poisoning attacks where malicious content in uploaded files or web pages can plant memories that persist across all future sessions. A Google Doc or a web page ChatGPT browses could contain hidden instructions that write to your memory. This is a real vulnerability — review your saved memories periodically in Settings.

Claude.ai's Harness

Claude.ai uses the same underlying model as Claude Code but wraps it in a completely different harness. The browser experience is designed for conversation, research, and content creation — not for working in a codebase.

The Signature Feature: Artifacts

Claude.ai's most distinctive harness feature is Artifacts — a separate output panel for substantial, self-contained content. When Claude generates code, an interactive visualization, a diagram, or a long document, it renders in a dedicated window alongside the chat.

The system prompt determines when to create an artifact vs. inline text. The rule: if the content is significant (typically 15+ lines), self-contained, and something you'd likely want to reuse or edit, it becomes an artifact. This is a harness decision — the model doesn't "choose" to create artifacts; the system prompt tells it when to.

Research and Web Search

Claude.ai offers Research mode — a deep analysis mode that spends several minutes examining multiple sources before producing a comprehensive report. This is fundamentally different from ChatGPT's web search, which returns quick snippets. Research mode is slower but more thorough.

Standard web search is also available as a default-on toggle per conversation.

Projects

Projects in Claude.ai are collections of related chats with shared reference files. They function like a lightweight harness customization — you upload documents, set instructions, and every chat in the project inherits that context. It's the browser equivalent of a CLAUDE.md file.

What Browser Apps Can't Do (and Why)

Here's the critical gap. Browser apps lack the tools that make coding agents useful for actual development:

  • No filesystem access — can't read your project files or navigate directories
  • No terminal/shell — can't run build tools, package managers, or tests
  • No git integration — can't commit, branch, merge, or manage version history
  • No persistent environment — Code Interpreter containers are ephemeral
  • No multi-file projects — can't create connected files that reference each other
  • No debugging tools — no breakpoints, stack traces, or interactive debugging

This isn't a model limitation. GPT-4 and Claude are perfectly capable of reading files, editing code, and running tests. But the browser harness doesn't give them those tools. It's like asking a chef to cook without access to the kitchen — the talent is there, the infrastructure isn't.

The Harness Explains the Experience

When people say "ChatGPT is bad at coding," they usually mean "ChatGPT's browser harness doesn't have the tools for serious coding work." The same GPT-4 model, wrapped in Cursor's harness with file access, semantic search, and terminal execution, produces dramatically different results. The harness, not the model, explains the gap.

This is worth repeating because it's the most common misconception in AI: people evaluate the model based on the harness they experienced it through. If you only know Claude from claude.ai, you'd never guess it could autonomously refactor a 10-file codebase, run tests, and fix failures — because the browser harness doesn't have those tools. The model's capabilities are hidden behind the harness's limitations.

Custom GPTs and Projects: Mini-Harnesses

Custom GPTs are the most accessible example of harness engineering in the wild. When you create a Custom GPT, you're building a mini-harness:

  • System prompt — Custom instructions that shape the GPT's behavior and personality
  • Context — Uploaded knowledge files that get retrieved when relevant
  • Tools — Toggle web search, code interpreter, and image generation on/off
  • External tools — Connect third-party APIs via "Actions" (REST endpoints the model can call)

This is harness engineering at its simplest: you're configuring a system prompt, curating context, selecting tools, and potentially connecting external services. Every concept from this course applies.

Custom GPT as mini-harness (configuration)

Name: Code Review Assistant Instructions: You are a senior code reviewer. Review code for bugs, security issues, and style violations. Always suggest specific fixes. Use a numbered list format. Knowledge: [company-style-guide.md, security-checklist.md] Tools: Code Interpreter (on), Web Search (off), DALL-E (off) Actions: [POST to Jira API to create tickets]

System prompt + context files + tool selection + external API = a complete mini-harness. Same harness components, simpler configuration.

System Prompts in the Wild

One fascinating side effect of browser apps: their system prompts keep getting leaked. Through prompt injection techniques, researchers have extracted the full system prompts for ChatGPT, Claude.ai, and Gemini. These leaks are a goldmine for understanding harness decisions.

What the leaks reveal:

  • Tool selection rules — ChatGPT's system prompt explicitly says when to use web search and when not to ("avoid searching for information already in training data")
  • Personality engineering — Detailed instructions about tone, helpfulness, and how to handle sensitive topics
  • Safety constraints — Rules about what the model cannot generate or discuss
  • Memory injection — A section showing how saved memories are formatted and injected
  • Model awareness — The model is told which model it is and its training cutoff date

These leaked prompts confirm what this course has been teaching: the system prompt is the most powerful lever in the harness. The same model, with a different system prompt, would behave like a completely different product.

ChatGPT vs. Claude.ai vs. Gemini

ChatGPT

Tools: Web search, Code Interpreter (server-side Python), Canvas (browser Python), DALL-E, file upload.

Memory: Saved memories + searchable chat history. Persistent across sessions.

Customization: Custom GPTs with instructions, knowledge files, and API Actions. Projects for workspace organization.

Strength: Most tools. Code Interpreter is genuinely useful for data analysis.

Claude.ai

Tools: Web search, Research mode, Artifacts (interactive output panel), data analysis.

Memory: Project-level context via uploaded files. No cross-session memory like ChatGPT.

Customization: Projects with reference files and instructions.

Strength: Artifacts create the best output experience. Research mode for thorough analysis.

Gemini

Tools: Google Workspace extensions (Gmail, Drive, Docs, Calendar, Maps, YouTube). Image generation.

Memory: Gems (custom Gemini instances) with personal instructions.

Customization: Gems for personalized assistants. No custom API integrations like ChatGPT's Actions.

Strength: Deepest Google ecosystem integration. Live access to your actual Gmail, Drive, and Calendar data.

What They All Lack

No filesystem: Can't read your project, navigate directories, or persist files.

No terminal: Can't run builds, tests, or system commands.

No git: Can't commit, branch, or manage code versions.

No persistent code environment: Every session starts from scratch.

These missing tools are exactly what separates a chat harness from a coding harness.

The Simplicity Advantage

Fewer components means fewer failure modes. Browser apps don't deal with file permission conflicts, git merge disasters, or runaway bash commands. They don't need five-layer permission systems because they can't modify your filesystem.

This simplicity is why browser apps are the right tool for many tasks:

  • Brainstorming and ideation — no tools needed, pure conversation
  • Quick research — web search + conversation = fast answers
  • Data analysis — upload a CSV, let Code Interpreter crunch it
  • Content creation — drafts, summaries, translations
  • Learning and explanation — the model's strongest mode
  • Prototyping — test an idea in Code Interpreter before building it properly

The mistake is using a browser app for tasks that need a coding harness. Not because the model can't handle it — but because the harness doesn't provide the tools.

Inside the Harness: What Happens When ChatGPT "Browses the Web"

You ask ChatGPT: "What are the latest Next.js 15 features?"

  1. The harness evaluates whether to search. Web search doesn't fire on every question. The system prompt tells the model to search when the query involves current events, recent releases, or information likely beyond its training cutoff. "Next.js 15 features" qualifies.
  2. The model generates search queries. It might fire up to 5 parallel web queries, crafted to find relevant results. These are actual web searches, not pre-cached results.
  3. Results are scored and filtered. The harness scores results for freshness and relevance. Recent results get a bias boost. Results are returned as text snippets with source URLs.
  4. Search results enter the context window. The snippets become part of the model's context — they're tool results, just like file reads in Claude Code. The model now has current information it can reason about.
  5. The model synthesizes a response. It combines the search results with its training knowledge to generate an answer. Citations link back to sources.
  6. Memory may update. If the model determines you're interested in Next.js, it might save a memory: "user works with Next.js." This fact will appear in future conversations.

Same tool-use loop, simpler toolbox. The pattern is identical to Claude Code: the model requests a tool (web search), the harness executes it, results go back to the model. The difference is scope — web search vs. 18+ tools for file system manipulation, code editing, and terminal access.

Try It Yourself

Find the Harness Boundary

Ask ChatGPT or Claude.ai to do something that requires a tool it doesn't have. Observe how it handles the limitation:

  1. "Read the file src/app.ts in my project." — It can't. It has no filesystem access. Watch how it asks you to paste the code instead.
  2. "Run npm test and tell me what fails." — It can't. No terminal. It might offer to analyze test code you paste, but it can't execute tests.
  3. "Commit this change and push to main." — It can't. No git. It might give you the git commands to run yourself.
  4. "Create a login page, an API route, and a test file." — It can generate the code, but each piece comes as a separate code block you'd need to manually create files for. A coding agent would create the actual files.

Every "can't" in this exercise is a harness limitation, not a model limitation. The same model, wrapped in Claude Code's harness, would handle all four tasks directly. The browser harness simply doesn't have the tools.
