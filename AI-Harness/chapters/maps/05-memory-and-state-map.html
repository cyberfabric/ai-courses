Memory and State

Memory Map

The Big Idea

LLMs have zero memory — not limited, zero. Everything that feels like memory is the harness faking it. The filesystem is the most reliable memory system — if something matters, write it to a file.

🏢 The Office Storage Analogy

🖥️
Desk = Context Window
What's right in front of you. Limited space, instant access. Cleared when you go home (session ends).
🗄️
Filing Cabinet = Session State
Nearby, organized by project. Token counts, permissions, MCP connections. Lost if the process crashes.
🏭
Warehouse = Persistent Storage
Takes effort to retrieve, but survives everything. Files on disk, git history, CLAUDE.md.

⚡ Higher layers = faster but fragile   |   💎 Lower layers = slower but permanent

The harness replays the entire conversation with every API call. The model processes it from scratch each time.

📊 The Five Memory Layers

Layer What It Holds Lifetime Survives Crash?
🖥️ Context Window Conversation, tool results, system prompt Current turn (replayed) ❌ No
🗄️ Session State Token count, permissions, MCP connections Current session ❌ No
🧠 Auto Memory Learned preferences, patterns, insights Across sessions ✅ Yes
📄 Project Files CLAUDE.md, progress files, HANDOFF.md Permanent ✅ Yes
📦 Git History Commits, diffs, branch state Permanent (versioned) ✅ Yes

📁 The Filesystem IS the Memory

📋
Progress Files
claude-progress.txt with completed tasks, in-progress work, blockers, and key decisions. A new session reads it and picks up instantly.

CLAUDE.md
Permanent project context loaded every turn. The most reliable memory — you control exactly what's remembered.

📖 Key Terms

Working Memory
The context window. Full conversation replayed with every API call. Cleared on compaction.
Session State
Harness-tracked data (token count, permissions, modified files). Invisible to the model. Lost on crash.
Persistent Memory
Files on disk, git history, auto-memory systems. Survives crashes, sessions, and tool switches.
Conversation Replay
The harness sends the full conversation history with every message. The model "remembers" by re-reading everything.
Compaction
Summarizing old turns when context fills up. Preserves what happened; loses how it happened.
Progress File
A markdown file tracking completed tasks, current state, and blockers. Any new session can read it and resume.
HANDOFF.md
Session-to-session handoff document for multi-session agent work on feature branches.
Saved Memories (ChatGPT)
Facts extracted from conversations and stored externally. Injected into system prompt on next chat. Intentionally vague.

🧪 Quick Quiz

Do LLMs have memory?

No. Zero memory. Every API call starts from a blank slate. The harness creates the illusion by replaying the full conversation history with every request.

What's the most reliable memory system for coding agents?

The filesystem. Files survive crashes, compaction, new sessions, and even switching to a different AI tool. Git commits, CLAUDE.md, and progress files are the most durable memory layers.

What happens to your conversation when compaction triggers?

The harness summarizes the full history into a ~5,000-token summary. The model loses precise details from earlier turns but retains what tasks were completed, current state, and next steps. It can re-read files to recover specifics.

How does ChatGPT's "memory" differ from Claude Code's?

ChatGPT stores intentionally vague facts at the user level (across all projects). Claude Code uses file-based memory at the project level — more precise, scoped to the project, and under your control.

What's the recommended pattern for multi-session agent work?

Initializer agent creates progress files and makes initial commit. Working agents read progress + git log, do work, commit with descriptive messages, and update progress. HANDOFF.md records what was done and what's next.

📋 Key Takeaways

  • LLMs have zero memory. The harness fakes it by replaying the full conversation with every API call.
  • Five memory layers: Context Window → Session State → Auto Memory → Project Files → Git History.
  • Higher layers = fast but fragile. Lower layers = slow but permanent. Use all five.
  • The filesystem is the most reliable memory. If it matters, write it to a file.
  • Compaction trades detail for continuity. After compaction, the model knows what happened but not the specifics.
  • Work with compaction: keep conversations focused, write to files, commit often, use /compact at natural breaks.
  • Each tool handles cross-session memory differently: Claude Code (project files), ChatGPT (vague facts), Windsurf (autonomous memories), Cursor (git worktrees).
