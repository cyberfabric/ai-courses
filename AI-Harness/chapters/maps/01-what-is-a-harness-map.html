Chapter 1

What Is a Harness?

Memory Map
💡

The Big Idea

The harness is everything between you and the LLM — system prompt, tools, context, memory, guardrails, and persistence. Most of what you experience in an AI tool is the harness, not the model.

🎯 Key Analogies

💻
The Operating System
A harness is to an LLM what an OS is to a CPU. The CPU does raw math; the OS gives it a filesystem, memory management, permissions, and a UI.
🏎️
The Pit Crew
The LLM is the F1 driver. The harness is the pit crew, track layout, race rules, radio, and telemetry. A driver alone doesn't win races.
🏢
The Windowless Room
A raw LLM is a brilliant consultant locked in a room with no windows. Smart, but blind — no files, no tools, no project awareness.
🧳
The Suitcase
Context engineering is packing a suitcase — you can't take everything, so you pick what matters most for this specific trip.

🧩 The Six Harness Components

📜
1. System Prompt
Hidden instructions that shape the model's behavior before you say a word. Thousands of tokens you never see.
🔧
2. Tools
File read/write, terminal, web search, code execution. Without tools, the LLM can only talk about code.
📦
3. Context Engineering
Picking the right information for the task. 500K-line codebase, 200K-token window — what do you include?
🧠
4. Memory & State
LLMs have zero memory. The harness fakes continuity via conversation history, session state, and persistent notes.
🛡️
5. Guardrails
Permission boundaries, output validation, safety rails. What the model is allowed to do.
💾
6. Persistence
Saving progress, resuming sessions, handing off between conversations. Tomorrow picks up where today left off.

⚡ What Happens When You Press Enter

Your prompt captured
→
System prompt + CLAUDE.md loaded
→
Context assembled & tools registered
→
API call sent (your prompt = ~5%)
→
Model responds (maybe tool calls)
→
Harness executes tools, loops back
→
Final response displayed

  1. Capture & Assemble

    Your message is captured. The harness loads hidden system prompts, CLAUDE.md files, and conversation history.

  2. Register & Package

    Available tools are registered (file read, bash, grep, MCP). Everything is packaged into one API request.

  3. Model Thinks

    The model receives the full context. It may request tool calls: "read src/auth.ts" or "edit lines 42-58."

  4. Tool Loop

    The harness checks permissions, executes tools, sends results back. The model may call more tools. This loops until done.

  5. Display Result

    The harness formats and displays the final response. Your "simple" Enter triggered the entire pipeline.

⚖️ Raw API vs. Harnessed Tool

Capability Raw LLM API Harnessed Tool (e.g. Claude Code)
Read your files ❌ No ✅ Yes
Edit code ❌ No (only suggests) ✅ Yes (directly)
Run tests ❌ No ✅ Yes
Remember conversation ❌ No (stateless) ✅ Yes (harness manages history)
Know your project ❌ No ✅ Yes (CLAUDE.md, file indexing)
Ask permission ❌ No concept of it ✅ Yes (guardrails)
Prompt quality needed Very detailed, manual Simple — harness fills the gaps

📖 Key Terms

Harness
The complete software infrastructure that wraps an LLM — everything between user input and model output.
LLM (Large Language Model)
The AI model that generates text. Smart but blind without a harness.
System Prompt
Hidden instructions loaded before the user's first message — shapes behavior, tone, and rules.
Context Window
The fixed amount of text (measured in tokens) a model can process at once. Prime real estate.
Tool Use
The mechanism that lets an LLM request actions (file reads, edits, searches) executed by the harness.
Guardrails
Permission boundaries and safety checks that control what the model is allowed to do.
Persistence
Saving and resuming state across sessions so work isn't lost between conversations.
Harness Engineering
The practice of improving AI tools by optimizing the harness rather than (just) the model.

🧪 Quick Quiz

What is a harness in the context of AI tools?

The complete software infrastructure that wraps an LLM — system prompt, tools, context management, memory, guardrails, and persistence. Everything between you and the model.

Why does the same prompt get better results in Claude Code than a raw API call?

Because Claude Code's harness provides file access, project context (CLAUDE.md), tool execution, conversation memory, and permission management. The raw API only gets the text you send.

What percentage of an API request is typically your prompt vs. harness-provided context?

Your prompt is roughly 5% of what the model sees. The other ~95% is system prompt, project context, tool definitions, and conversation history — all provided by the harness.

Name the six components of a harness.

1) System Prompt, 2) Tools, 3) Context Engineering, 4) Memory & State, 5) Guardrails, 6) Persistence.

Why is harness engineering sometimes more impactful than model improvements?

Same model + better harness = dramatically better results. Cursor achieved 30% gains through harness tweaks alone. Most developers run great models through mediocre harnesses — that's the opportunity.

📋 Key Takeaways

  • A harness is everything between you and the LLM — it's what makes the model useful.
  • A raw LLM is a brilliant consultant locked in a windowless room — smart but blind.
  • The six components: System Prompt, Tools, Context, Memory, Guardrails, Persistence.
  • Your prompt is ~5% of what the model sees. The harness provides the other 95%.
  • Improving the harness often delivers more gains than improving the model itself.
  • Claude Code, ChatGPT, Cursor, Codex, and Windsurf are all harnesses around similar LLMs — the harness is what differentiates them.
