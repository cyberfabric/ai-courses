Browser Apps: ChatGPT & Claude.ai

Memory Map

The Big Idea

When people say "ChatGPT is bad at coding," they mean the browser harness lacks coding tools. The same model in a coding harness produces dramatically different results. People evaluate the model based on the harness they experienced it through.

🔧 Swiss Army Knife vs. Full Workshop

🇨🇭
Browser Apps = Swiss Army Knife
Versatile, portable, always in your pocket. A blade (text), a screwdriver (code interpreter), a corkscrew (web search). Solves many problems, but not heavy-duty work.
🏭
Coding Agents = Full Workshop
Table saw, drill press, lathe, clamps. Specialized, powerful, built for serious projects. You wouldn't build a cabinet with a Swiss Army knife.

The difference isn't the blade's sharpness (the model). It's the tooling around it (the harness).

📅 ChatGPT's Harness Evolution

  1. 2022 — Text Only

    System prompt + conversation window. No tools at all. Just raw text generation.

  2. 2023 — Four Major Tools

    Code Interpreter, web browsing, DALL-E, file upload. Same model, dramatically expanded capabilities.

  3. 2024 — Customization

    Custom GPTs (mini-harnesses), Canvas, memory feature, improved search.

  4. 2025-2026 — Projects & Integration

    Projects, voice, deeper API integrations. The harness evolved more than the model did.

🚫 What All Browser Apps Lack

📁
No Filesystem
Can't read your project files, navigate directories, or persist files across sessions.
💻
No Terminal
Can't run builds, tests, package managers, or system commands.
🔀
No Git
Can't commit, branch, merge, or manage version history.
🔗
No Multi-File Projects
Can't create connected files that reference each other. Code comes as separate blocks you copy manually.

Every "can't" is a harness limitation, not a model limitation. The same model in a coding harness handles all of these.

📖 Key Terms

Artifacts (Claude.ai)
Separate output panel for substantial, self-contained content — code, visualizations, documents. System prompt decides when to create one.
Code Interpreter (ChatGPT)
Sandboxed Python execution in a server-side container. No HTTP, no filesystem, ephemeral. Good for data analysis.
Custom GPTs
User-configurable mini-harnesses: custom system prompt + knowledge files + tool toggles + API Actions.
Research Mode (Claude.ai)
Deep analysis mode examining multiple sources over minutes. Slower but more thorough than standard web search.
Projects (Claude.ai)
Collections of related chats with shared reference files and instructions. Browser equivalent of CLAUDE.md.
Gems (Gemini)
Custom Gemini instances with personal instructions. Google's version of Custom GPTs.
Memory Poisoning
Attack where malicious content in files or web pages plants persistent memories in ChatGPT. Review saved memories periodically.
Actions (Custom GPTs)
REST API endpoints the model can call — the Custom GPT equivalent of MCP tools for connecting external services.

🧪 Quick Quiz

Why is ChatGPT "bad at coding" compared to Claude Code?

It's the harness, not the model. ChatGPT's browser harness lacks filesystem access, terminal, git, and multi-file project tools. The same GPT-4 model in Cursor's harness produces dramatically better coding results.

What are the four components of a Custom GPT mini-harness?

1) Custom instructions (system prompt), 2) Knowledge files (context), 3) Tool toggles (web search, code interpreter, DALL-E), 4) API Actions (external tool connections).

How does Claude.ai's Artifacts feature work?

The system prompt tells the model when to create an artifact vs inline text: if content is significant (15+ lines), self-contained, and reusable, it renders in a dedicated panel alongside the chat. It's a harness decision, not a model decision.

What's the "simplicity advantage" of browser apps?

Fewer components = fewer failure modes. No file permission conflicts, git merge disasters, or runaway bash commands. They're the right tool for brainstorming, research, data analysis, content creation, and learning.

What security risk comes with ChatGPT's memory feature?

Memory poisoning attacks. Malicious content in uploaded files or web pages can plant persistent memories that affect all future sessions. Review saved memories periodically in Settings.

📋 Key Takeaways

  • Browser apps are harnesses too — just simpler ones. Swiss Army knives vs. the full workshop.
  • People evaluate the model based on the harness. "ChatGPT is bad at coding" really means the browser harness lacks coding tools.
  • ChatGPT evolved from text-only (2022) to multi-tool (2025) — the harness changed more than the model.
  • Browser apps lack filesystem, terminal, git, and multi-file projects. These are harness limitations, not model limitations.
  • Custom GPTs are the most accessible harness engineering: system prompt + context + tool selection + API actions.
  • Leaked system prompts confirm: the system prompt is the most powerful lever. Same model + different prompt = different product.
  • Browser apps shine at brainstorming, research, data analysis, and learning. Use the right harness for the right task.
