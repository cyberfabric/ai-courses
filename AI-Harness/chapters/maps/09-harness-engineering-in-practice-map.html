The Big Idea

Improving the harness consistently delivers bigger gains than waiting for the next model. Constraining the agent's solution space — focused prompts, curated tools, restricted context, external verification — paradoxically produces better outcomes.

🎛️ The Four Tuning Levers

  1. 📜 System Prompt Iteration

    Specific, opinionated CLAUDE.md instructions. "Use vitest, not jest. Mock with msw, never jest.mock()." Test different instructions, measure success rates.

  2. 🔧 Tool Design

    Clear descriptions guide selection. "Prefer this over grep when the identifier name is unknown." One sentence prevents a category of errors.

  3. 📦 Context Curation

    What you exclude matters as much as what you include. Exclude node_modules, dist, generated files, lock files. Every exclusion frees tokens for what matters.

  4. 📋 Output Format Control

    Structured constraints reduce parsing errors. "Exactly one migration file. Include UP and DOWN. Run migrate:check after."

🚫 Anti-Patterns

📚
Context Stuffing
2,000-line CLAUDE.md = most instructions invisible. Keep top-level short; use directory-level files for details.
🔧
Tool Sprawl
Registering 200+ tools "just in case" overwhelms selection. Start small, add on demand (Stripe's approach).
📝
Over-Prompting
Contradictory instructions. "Always write tests" + "keep changes minimal" = the model silently picks one.
🪞
Self-Evaluation
Models can't reliably grade their own work. Use linters, type checkers, test suites, CI. External verification only.

📖 Key Terms

Hashline
Edit format where lines are tagged with content hashes. The model references lines by hash instead of reproducing text. 5–14 point improvement across 15 LLMs.
Blueprints (Stripe)
Workflows mixing deterministic steps ("checkout repo") with agentic steps ("figure out the fix"). Boring stuff always happens; creative stuff gets freedom.
AI Velocity Paradox
Teams ship faster with AI, but testing/security/deployment hasn't kept up. 69% of frequent users report deployment problems with AI code.
Self-Evaluation Fallacy
Models can't reliably evaluate their own work. They exhibit overconfidence even on broken output. Use external verification.
Context Curation
What you exclude matters as much as what you include. Noise actively degrades performance.
Model-Harness Co-Design
Training model and harness together via RL (Windsurf's SWE-1.5). The ceiling is much higher than treating them separately.
CursorBench
Eval suite from real engineering sessions. Shows harness quality can compensate for model size.

🧪 Quick Quiz

What did the Hashline technique prove about harness engineering?

That changing only the edit format — not the model, prompt, or tools — improved 15 different LLMs by 5–14 percentage points. Format is a harness decision worth more than a model upgrade. But it's language-dependent: measure your specific use case.

What's Stripe's key architectural insight with Blueprints?

Alternate between deterministic and agentic steps. Hardcode the boring stuff (checkout, lint, test). Give agentic freedom for creative work (figuring out the fix). This ensures consistency while allowing flexibility.

Why can't you trust a model to evaluate its own work?

The Self-Evaluation Fallacy: models exhibit overconfidence even on broken outputs. "Did you do this correctly?" almost always gets "yes." Use external verification: linters, type checkers, test suites, CI pipelines.

What are the four tuning levers?

1) System prompt iteration (specific, opinionated instructions), 2) Tool design (clear descriptions guide selection), 3) Context curation (exclude noise), 4) Output format control (structured constraints reduce errors).

Why does the 34% → 79% before/after matter?

Same model, same task. Four harness changes — focused prompt, 8 curated tools instead of 200+, restricted context, and external verification — more than doubled success. No model upgrade needed. Constraints eliminate whole categories of errors.

📋 Key Takeaways

  • Harness improvements consistently deliver bigger gains than model upgrades. Same model, better harness, dramatically better results.
  • Hashline: a format change improved 15 LLMs by 5–14 points. Format is a harness decision, not a model decision.
  • Stripe's Minions: 1,300 PRs/week via Blueprints (deterministic + agentic steps), curated tools, 2-round CI max, human review.
  • Four tuning levers: system prompt iteration, tool design, context curation, output format control.
  • Four anti-patterns: context stuffing, tool sprawl, over-prompting, self-evaluation. Constraints beat freedom.
  • Build your own evals. Public benchmarks overstate capability (81% Verified vs 23% Pro). Measure your actual tasks.
  • The boring improvements work: better error messages, clearer tool descriptions, smarter exclusions. Chase consistency, not novelty.
