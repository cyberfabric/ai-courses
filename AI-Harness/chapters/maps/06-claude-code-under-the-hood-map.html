The Big Idea

Claude Code's architecture is a deliberately "dumb" while-loop orchestrator. All intelligence lives in the model. Everything else — 110+ prompt fragments, 18 tools, five-layer permissions, hooks, sub-agents — is infrastructure around that loop.

The orchestrator is dumb. The model is smart. That's the design.

📋 CLAUDE.md Hierarchy

  1. 🌍 Global — ~/.claude/CLAUDE.md

    Personal defaults across all projects. Coding style, commit conventions, testing habits. Highest-leverage file you'll create.

  2. 📁 Project — ./CLAUDE.md

    Team-shared, committed to git. Stack, architecture, commands, project-specific rules.

  3. 📂 Directory — subdirectory CLAUDE.md files

    Area-specific overrides. "Files in src/legacy/ use CommonJS, not ES modules."

Most specific level wins on conflicts. All levels are combined into the context on every turn.

⚙️ Hooks: Deterministic in a Probabilistic System

✨
Auto-Format
PostToolUse on Edit/Write → run Prettier. Every edit is formatted, always.
🔒
Security Scan
PreToolUse on Bash → check for secrets before commits. Hard block, not a suggestion.
📢
Notification
PostToolUse → send Slack message when long tasks complete. Stay informed without watching.
🛑
Validation
PreToolUse → block edits to protected files. Production configs, migrations — untouchable.

The model might remember to format. A hook always formats. Hooks turn "should" into "will."

📖 Key Terms

Dumb Orchestrator
Claude Code's design philosophy: the while-loop is simple; all intelligence is in the model.
Prompt Fragments
110+ conditional pieces that assemble into the system prompt based on OS, mode, and available tools.
Hooks
Shell commands that run before (PreToolUse) or after (PostToolUse) tool execution. Deterministic behavior in a probabilistic system.
Sub-Agent
A separate agent instance with its own context window. Three types: Explore, Plan, General-Purpose.
Plan Mode
Read-only mode. The model explores and proposes a plan before any files are touched.
Worktree
A separate git working directory. Enables parallel agent work without file conflicts.
Deny Rules
Absolute permission blocks that can't be overridden, even in bypassPermissions mode.
Claude Agent SDK
The programmable library exposing Claude Code's harness infrastructure for building custom agents.

🧪 Quick Quiz

What is Claude Code's core architecture?

A deliberately "dumb" while-loop: while (response has tool calls) → execute → feed back → repeat. The loop terminates when the model produces plain text with no tool calls. All intelligence is in the model.

What's the difference between hooks and deny rules?

Hooks are custom shell commands that run before/after tool execution (auto-format, scan, notify). Deny rules are absolute blocks that prevent specific actions entirely, even in bypassPermissions mode. Hooks add behavior; deny rules remove capability.

When should you use plan mode?

When a change touches 3 or more files. Planning first catches architectural mistakes before they're coded. The model explores read-only, proposes a plan, you review, then it implements.

Why use sub-agents instead of doing everything in the main context?

Context isolation and parallelism. Explore sub-agents keep exploration tokens out of the main window. General-purpose sub-agents can work in parallel in isolated worktrees.

What's the most underused Claude Code feature?

The global CLAUDE.md at ~/.claude/CLAUDE.md. Most developers write project-level files but forget global preferences (coding style, commit conventions, testing habits) that apply everywhere.

📋 Key Takeaways

  • Claude Code is a dumb while-loop orchestrator. All intelligence is in the model, not the runtime.
  • The system prompt uses 110+ conditional fragments — you get a different prompt on different OS, modes, and tool configurations.
  • CLAUDE.md at three levels: global (personal defaults) → project (team rules) → directory (area overrides).
  • Five-layer permissions: Hooks → Deny Rules → Permission Mode → Allow Rules → User Prompt. Deny rules are absolute.
  • Hooks turn "should" into "will" — deterministic behavior (auto-format, security scans) in a probabilistic system.
  • Sub-agents (Explore, Plan, General) provide context isolation and parallel work in separate worktrees.
  • Settings follow a 5-level precedence: Enterprise → CLI flags → Local project → Shared project → Global user.
  • The Claude Agent SDK exposes this entire infrastructure as a library for building custom agents.
