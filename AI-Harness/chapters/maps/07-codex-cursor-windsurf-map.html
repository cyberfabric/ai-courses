Chapter 7

Codex, Cursor, and Windsurf

Memory Map
💡

The Big Idea

Same LLMs, different harnesses, vastly different results. Codex prioritizes security (cloud sandbox). Cursor prioritizes accuracy (per-model tuning). Windsurf prioritizes speed (model-harness co-design). The harness shapes the experience.

🚗 The Car Analogy

Three car manufacturers — same engines, steering, brakes. But a Tesla, a Toyota, and a Ferrari feel completely different to drive. The engineering around the engine makes the difference.

🏗️ Three Architectures, Three Philosophies

☁️
Codex — Cloud Sandbox
Code cloned into a cloud VM. Sandboxed with Landlock + seccomp. Agent can't access local files, network, or services. Strongest security isolation of any coding agent.
🖥️
Cursor — IDE-Native
Embedded in your editor. Sees open tabs, cursor position, recent edits. Per-model harness tuning — each model gets different instructions. 30% gains from harness tweaks alone.
🌊
Windsurf — Co-Designed
SWE-1.5 model trained with the Cascade harness via reinforcement learning. Real-time Flows track your actions continuously. ~14x faster than Claude Sonnet.

🔬 Same Task: "Add Rate Limiting"

  1. ☁️ Codex Approach

    Clones repo to cloud VM → reads AGENTS.md chain → writes code + tests in sandbox → returns PR diff. Can't access local Redis — must mock it.

  2. 🖥️ Cursor Approach

    Works in your IDE → semantic search finds routes → notices your open auth middleware tab → matches your project style. Local Redis accessible for testing.

  3. 🌊 Windsurf Approach

    Already knows you've been editing API routes (Flows). Creates checkpoint → writes code fast with SWE-1.5 → revert available if needed. Everything local.

Same task, same quality. Different paths: security (Codex) vs accuracy (Cursor) vs speed (Windsurf).

📖 Key Terms

Progressive Disclosure
Loading context incrementally — small root AGENTS.md, deeper detail in subdirectories. Used by Codex.
JSON-RPC
Codex's bidirectional communication protocol. Richer than MCP — supports streaming diffs, approval flows, thread persistence.
Per-Model Tuning
Cursor's approach of customizing system prompts and tool configs for each specific model. 30% performance gains.
CursorBench
Cursor's internal evaluation suite based on real engineering sessions. Better than public benchmarks at measuring real-world harness quality.
Cascade / Flows
Windsurf's real-time awareness system. Tracks edits, terminal, clipboard continuously to infer context and intent.
SWE-1.5
Windsurf's model co-designed with Cascade via reinforcement learning. 40% on SWE-Bench Pro, ~14x faster than Claude Sonnet.

Checkpoints (Windsurf)
Snapshots of file state before changes. Allows easy revert if an approach doesn't work — Windsurf's safety mechanism.

🧪 Quick Quiz

Why did Codex choose a cloud sandbox instead of local execution?

Security. Landlock + seccomp isolation means the agent physically cannot access your local filesystem, browser, or network. It eliminates entire categories of risk (accidental deletion, secret leaks, unauthorized access). The tradeoff: no local services or databases.

How did Cursor achieve a 30% performance improvement without changing the model?

Per-model harness tuning. They customized system prompts, tool names (Unix-aligned), instruction style (concise vs. rich), and mid-turn communication for each specific model. Same model, different harness instructions, dramatically different results.

What makes Windsurf's SWE-1.5 model unique?

It was co-designed with the Cascade harness via reinforcement learning. The model learned to use Cascade's specific tools, not just how to code generally. Like training a driver for one specific car instead of teaching a generic driver.

Why did OpenAI reject MCP for Codex's protocol?

MCP's tool-oriented model didn't support IDE-specific features like streaming diffs, approval workflows, and thread persistence. Codex needed richer interaction patterns, so they built JSON-RPC instead.

What are Windsurf's "Flows"?

Real-time awareness tracking. Cascade monitors your edits, terminal commands, clipboard, and cursor position continuously — updating context before you even send a message. The tradeoff: noise from unrelated actions during exploration.

📋 Key Takeaways

  • Same LLMs, different harnesses, vastly different experiences. The harness is what differentiates these tools.
  • Codex: cloud sandbox (strongest security), AGENTS.md progressive disclosure, JSON-RPC protocol, Skills system.
  • Cursor: IDE-native, per-model harness tuning (30% gains), 8 parallel agents, CursorBench for evaluation.
  • Windsurf: model-harness co-design via RL (SWE-1.5), real-time Flows awareness, checkpoints for safe rollback.
  • How you evaluate your harness matters as much as how you build it — CursorBench vs. SWE-Bench.
  • Standards don't always fit: Codex chose JSON-RPC over MCP because MCP lacked streaming diffs and approval flows.
  • Each tool's unique strength reflects its harness philosophy: security (Codex), accuracy (Cursor), speed (Windsurf).
