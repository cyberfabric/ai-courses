Chapter 4

Meet Windsurf: Your AI-Powered Workspace

Install it, open a folder, and get comfortable with the three panels you'll use every day

From Vocabulary to Action

In Chapter 3, you learned the words. Now let's meet the tool where most of your work will actually happen: Windsurf.

Windsurf is an IDE — remember, that's just an app for working with code files. But Windsurf has a twist: it comes with a built-in AI assistant called Cascade that you can chat with right inside the app. Think of it as having a knowledgeable colleague sitting next to you, ready to help at any moment.

If Google Docs is where you write documents, Windsurf is where developers write code. The difference? Windsurf also has a built-in chat assistant (Cascade) that can answer questions, generate files, and help you with tasks — like having Google Docs with a co-author who never sleeps.

You won't be writing code in Windsurf. You'll use it to:

  • Open and browse project folders
  • Edit spec files (plain-language descriptions of features)
  • Use Git visually (committing, branching — all with buttons, not commands)
  • Chat with Cascade to generate and refine specs
  • Run a couple of OpenSpec commands in the built-in terminal

Installing Windsurf on Your Mac

Let's get Windsurf on your machine. This is a one-time setup.

  1. Open Safari (or any browser)
  2. Go to windsurf.com
  3. Click the Download button — it should automatically detect that you're on a Mac
  4. Once the download finishes, open the .dmg file
  5. Drag the Windsurf icon into the Applications folder
  6. Open Applications in Finder and double-click Windsurf

The first time you open Windsurf, macOS might show a warning: "Windsurf is an app downloaded from the internet. Are you sure you want to open it?" Click Open. This is a standard Mac security prompt — it appears for any new app.

After installing, you can open Windsurf quickly anytime using Spotlight: press Cmd + Space, type Windsurf, and press Enter. Same trick you used for Terminal in Chapter 2.

Opening Windsurf for the First Time

When Windsurf launches, you'll see a Welcome screen. It may ask you to sign in or configure some settings. Follow the prompts — the defaults are fine for now.

Once you're past the welcome screen, you'll see a mostly empty window. This is normal. Windsurf needs you to open a folder before it has anything to show.

Windsurf has a lot of buttons, icons, and menus. Ignore most of them. You'll use about 5% of what's on screen. We'll point out exactly which parts matter for you and which you can safely pretend don't exist.

Opening a Folder

Remember the my-first-project folder you created in Chapter 2? Let's open it in Windsurf.

  1. In Windsurf, go to File → Open Folder…
  2. Navigate to Documents → my-first-project
  3. Click Open

Windsurf now shows your project. On the left side, you'll see a panel listing the files in your folder. If you followed Chapter 2, you should see notes.txt (or hello.txt) in that list. That's your project, loaded and ready.

Always open a folder, not an individual file. Windsurf is designed to work with entire project folders. If you open a single file, you'll miss most of its features (like the file explorer and Git integration).

The Three Panels You'll Actually Use

Windsurf's interface has many areas, but you only need to care about three panels. Let's meet them one by one.

1. File Explorer (Left Panel)

The left side of Windsurf shows the File Explorer — a tree view of every file and folder in your project.

It's Finder, but built into Windsurf. You can see your folders, expand them, and click on files to open them. The only difference is that everything stays inside the Windsurf window — no separate Finder window needed.

What you can do here:

  • Click a file to open it in the editor (center panel)
  • Right-click to see options: New File, New Folder, Rename, Delete
  • Expand/collapse folders by clicking the arrow next to them

If the File Explorer isn't visible, click the top icon in the left sidebar (it looks like two overlapping pages) or press Cmd + Shift + E.

2. Editor (Center Panel)

The big area in the center is the Editor. When you click on a file in the File Explorer, its contents appear here. This is where you read and edit files.

The editor works like any text editor you've used before:

  • Click to place your cursor
  • Type to add text
  • Cmd + S to save
  • Cmd + Z to undo

You'll notice the editor adds colors to certain words. That's called syntax highlighting — it color-codes different parts of a file to make them easier to read. You don't need to understand the colors; they're just there to help.

Click on notes.txt in the File Explorer. The file opens in the center panel. It's probably empty (you created it with touch, which makes empty files). That's fine — you'll add some text in a moment.

3. Cascade — Your AI Assistant (Right Panel or Chat)

This is the part that makes Windsurf special. Cascade is a built-in AI chat assistant. You can ask it questions, give it instructions, and it can even create and edit files for you.

To open Cascade, look for the chat icon in the right sidebar, or use the keyboard shortcut. You'll see a chat window where you can type messages, just like any messaging app.

Think of Cascade as Slack — but instead of messaging a colleague, you're messaging an AI that knows about your project. You can ask things like "What files are in this project?" or "Create a new file called ideas.txt with three bullet points." It reads your project files and responds accordingly.

You'll use Cascade extensively in the OpenSpec chapters (9–11), where you'll ask it to generate proposals and specs. For now, just know it's there.

The Layout at a Glance

Left File Explorer — browse files
Center Editor — read & edit files
Right Cascade — AI chat assistant

That's it. Three panels. Everything else on screen is extra features you can explore later if you're curious, but they're not needed for this course.

Creating a New File from Windsurf

In Chapter 2, you created files using the Terminal (touch). You can also create files directly in Windsurf — no Terminal needed.

  1. In the File Explorer (left panel), right-click on an empty area
  2. Select New File…
  3. Type a name: my-notes.txt
  4. Press Enter

The new file appears in the File Explorer and opens automatically in the editor. The cursor is blinking in the center panel, ready for you to type.

You can also create folders the same way — right-click and choose New Folder….

Editing and Saving a File

Let's put something in that new file:

  1. Click in the editor area (center panel)
  2. Type: This is my first file edited in Windsurf!
  3. Press Cmd + S to save

Look at the file's tab at the top of the editor. Before saving, you'll see a white dot next to the filename — that means there are unsaved changes. After pressing Cmd + S, the dot disappears. Your changes are saved.

Cmd + S is your best friend. Get into the habit of pressing it after every change. It's the same shortcut you use in Google Docs, Word, or any other app.

The Integrated Terminal (Bottom Panel)

There's one more area worth knowing about: the integrated terminal. It's a Terminal window built right into Windsurf, so you don't need to switch to the separate Terminal app.

To open it:

  • Press Ctrl + ` (the backtick key, above Tab on your keyboard)
  • Or go to View → Terminal

A panel slides up from the bottom of the window. It looks exactly like the Terminal app from Chapter 2 — same prompt, same commands. The difference? It's already pointed at your project folder, so you don't need to cd anywhere.

You won't need the integrated terminal much, but it'll come in handy when you run OpenSpec commands in Chapters 9–11 (things like openspec init). For now, just know how to open it.

If you accidentally open the terminal and it takes up too much space, just press Ctrl + ` again to hide it. Or drag the top edge of the terminal panel to resize it.

What You Can Safely Ignore

Windsurf has dozens of features designed for professional developers. You don't need most of them. Here's what to ignore:

  • Extensions / Plugins — extra add-ons for developers. Skip.
  • Debug panel — for testing code. Not your job.
  • Settings with hundreds of options — defaults are fine.
  • Keyboard shortcut lists — you only need Cmd+S, Cmd+Z, and Ctrl+`.

If you accidentally click something unfamiliar, just close the panel (click the X or press Escape) and go back to what you were doing. You can't break anything by clicking around.

Putting It All Together

Your mission: Set up Windsurf and create your first file.

  1. Open Windsurf (Cmd + Space, type "Windsurf", Enter)
  2. Open your project: File → Open Folder → navigate to Documents/my-first-project
  3. Verify you can see your files in the File Explorer (left panel)
  4. Right-click in the File Explorer and create a new file called ideas.txt
  5. Type a sentence in the editor: My first idea: learn OpenSpec
  6. Save with Cmd + S
  7. Open the integrated terminal with Ctrl + `
  8. Type ls in the terminal to confirm your files are there
  9. Close the terminal with Ctrl + `

If you can see your files in both the File Explorer and the terminal — you're all set. Windsurf is your workspace now.
