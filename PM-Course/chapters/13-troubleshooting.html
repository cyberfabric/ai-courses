Chapter 13

When Things Go Wrong: A Troubleshooting Guide

Common error messages, what they actually mean, and how to fix them without panicking

Errors Are Normal

Things will go wrong. Not because you're doing something wrong, but because that's how software works. Developers see error messages every single day. The difference between a beginner and an experienced user isn't that experienced users never see errors — it's that they know how to read them.

The Golden Rule of Troubleshooting

Read the error message out loud. Seriously. Error messages are written by developers to tell you what went wrong. They're not random gibberish — they're clues. Nine times out of ten, the message tells you exactly what to do.

Error messages are like the "check engine" light on your car. The light itself isn't the problem — it's telling you something needs attention. Some issues are minor (low washer fluid), some need a mechanic (engine misfire). The key is reading the message, not just panicking at the warning.

Terminal Errors

command not found

You typed a command and see:

zsh: command not found: opnespec

What it means: You mistyped the command, or the tool isn't installed.

How to fix it:

  • Check your spelling carefully. In the example above, it's opnespec instead of openspec — a simple typo.
  • If the spelling is correct and you still see this error, the tool might not be installed. Ask a developer to help you install it.

No such file or directory

You tried to navigate to a folder:

cd: no such file or directory: Documents/my-porject

What it means: The folder doesn't exist at that location, or you misspelled the name.

How to fix it:

  • Run ls to see what's actually in the current folder
  • Check the spelling (notice my-porject vs my-project)
  • Run pwd to make sure you're in the folder you think you're in

Permission denied

What it means: You're trying to do something your user account isn't allowed to do.

How to fix it: This is rare for the commands you'll use. If you see it, ask a developer — it usually requires an admin to adjust permissions.

None of the six Terminal commands from Chapter 2 (pwd, ls, cd, mkdir, touch, clear) can damage your computer. If you see an error, just read it, fix the typo or path, and try again.

Git Errors (in Windsurf)

"Nothing to commit, working tree clean"

What it means: You tried to commit, but Git doesn't see any changes.

Common causes:

  • You forgot to save the file (Cmd + S). Git tracks what's saved on disk, not what's displayed in the editor.
  • You already committed these changes.
  • You forgot to stage the changes before committing.

How to fix it: Save all files (Cmd + S), then check Source Control. If there are changes listed, stage them and try committing again.

"Please tell me who you are"

Git asks for your identity:

Please tell me who you are.

Run
  git config --global user.email "you@example.com"
  git config --global user.name "Your Name"

What it means: Git needs to know your name and email to label commits. This is a one-time setup.

How to fix it: Open the integrated terminal and type these two commands (replace with your actual name and email):

git config --global user.name "Your Name"
git config --global user.email "your.email@company.com"

After this, try committing again. You'll never see this message again.

This is one of the very few times you'll type a git command in the terminal. It's a one-time configuration, not a regular workflow step.

"Push rejected" / "Updates were rejected"

What it means: Someone else changed the remote branch since you last pulled. Your local version is behind.

How to fix it:

  • This is rare for your workflow because you work on your own branches
  • If it happens, click the pull or sync button in Windsurf to download the latest changes, then try pushing again
  • If it's still stuck, ask a developer — they can resolve it in minutes

"Detached HEAD"

What it means: You're looking at a specific point in history instead of being on a branch. It sounds scary, but it's harmless.

How to fix it: Click the branch indicator in the bottom-left corner of Windsurf and select the branch you want to be on (e.g., main or your feature branch). You're back to normal.

"Detached HEAD" is the most alarming-sounding message in all of Git. Despite its name, nothing is broken. You just need to switch back to a branch. Think of it like accidentally opening the "history" view in Google Docs — you're looking at an old version, not editing the current one.

Windsurf Issues

Files not showing in the File Explorer

Common cause: You opened the wrong folder, or you opened a single file instead of a folder.

How to fix it: Go to File → Open Folder and make sure you're selecting the project folder, not a file inside it.

Source Control not showing anything

Common cause: The folder isn't a Git repository yet.

How to fix it: Open Source Control (branch icon in sidebar) and click Initialize Repository. This is the same step from Chapter 5.

Cascade not responding

Common causes:

  • No internet connection — Cascade needs the internet to work
  • The AI service is temporarily down — wait a minute and try again
  • You need to sign in — check if Windsurf is asking for authentication

How to fix it: Check your internet connection first. If that's fine, try closing and reopening the Cascade panel, or restarting Windsurf.

OpenSpec Issues

"No active change" / "No change found"

What it means: You tried to run /opsx:propose or /opsx:apply, but there's no change directory set up.

How to fix it: Create a change first:

openspec new change mob123-your-feature

Then try the command again.

Specs not generating properly

Common cause: Your feature description was too vague.

How to fix it: Be more specific when describing the feature to Cascade. Instead of "add a search feature," try "add a search bar at the top of the notes grid that filters notes in real-time as the user types, matching against note titles and body text."

Specific (Works Well)

Add a search bar at the top of the notes grid. As the user types, notes that don't match the search text (in title or body) are hidden. When the search is cleared, all notes reappear.

Vague (Poor Results)

Add search to the app

BitBucket Issues

Can't create a Pull Request

Common causes:

  • You haven't pushed your branch yet — go back to Windsurf and push first
  • You're on the wrong repository — make sure you're looking at your fork, not the main repo
  • You're trying to create a PR from the main branch — you need to be on a feature branch

"No default reviewers" / can't add reviewers

How to fix it: Ask your team lead for the correct reviewers to add. Some repositories have default reviewers configured; others require you to add them manually.

Authentication errors on BitBucket

How to fix it: Try logging out and back in. If your password recently changed, you'll need to update it. Some teams use personal access tokens instead of passwords — ask a developer if you're unsure which to use.

When to Ask for Help

You should try to solve issues yourself first — most problems are typos, missing saves, or wrong folders. But there's absolutely no shame in asking for help. Here's a simple rule:

The Two-Attempt Rule

If you've read the error message, tried to fix it twice, and it's still not working — ask a developer. Don't spend 30 minutes stuck on something a colleague can fix in 2 minutes. Your time is valuable.

When you ask for help, share:

  • What you were trying to do ("I was trying to push my branch")
  • The exact error message (copy-paste it or take a screenshot)
  • What you already tried ("I checked the spelling and tried again")

Developers appreciate this level of detail. It helps them help you faster.

You're Prepared for the Unexpected

Your mission: Intentionally trigger an error and fix it.

  1. Open Terminal and type cd nonexistent-folder — read the error
  2. In Windsurf, try to commit without saving a file — observe what happens
  3. Run openspec status in a project that has no OpenSpec — read the message

The point isn't to memorize errors. It's to practice the skill of reading messages calmly and figuring out what they mean. That skill transfers to any tool you'll ever use.
