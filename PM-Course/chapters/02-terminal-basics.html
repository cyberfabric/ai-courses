Chapter 2

The Terminal: Your New Friendly Neighbor

Six commands. That's all you need. Let's meet the text-based side of your Mac.

What Is a Terminal, Anyway?

You already know how to talk to your Mac. You click on folders in Finder, drag files around, double-click to open things. That's the graphical way to interact with your computer — you point and click.

The Terminal is just another way to do the exact same things — except instead of clicking, you type. You type a short command, press Enter, and the computer does it.

Think of it like texting versus calling. Both get the message across. Texting (the Terminal) is sometimes faster because you can be very specific in a few words. Calling (Finder) is sometimes easier because you can see everything visually. Developers like texting. You'll learn just enough texting to get by.

You might hear people call it different names: CLI (Command Line Interface), command line, shell, or console. They all refer to the same thing — that window where you type commands. Don't let the multiple names confuse you. It's all the same app on your Mac: Terminal.

Opening Terminal on Your Mac

There are two ways to open Terminal. Pick whichever feels more natural:

Option A: Spotlight (the fast way)

  1. Press Cmd + Space on your keyboard
  2. A search bar appears in the center of your screen
  3. Type Terminal
  4. Press Enter

Option B: Finder (the scenic route)

  1. Open Finder
  2. Go to Applications → Utilities
  3. Double-click Terminal

You should now see a window with a mostly white (or black) background and some text near the top. There's a blinking cursor waiting for you. It might say something like: 

yourname@YourMac ~ %

That's the prompt. It's the Terminal's way of saying "I'm ready. What would you like me to do?"

The Terminal looks bare. No colorful icons, no friendly buttons. That's normal. It's designed to be minimal — just you and the computer having a text conversation. The blinking cursor is not judging you. It's just waiting patiently.

Understanding the Prompt

That line of text before the blinking cursor is called the prompt. Let's break it down: 

yourname@YourMac ~ %
PartWhat It Means
yourname Your Mac username (the name you log in with)
@YourMac The name of your computer
~ Your current location (we'll explain this in a moment)
% The Terminal is ready for your command

Your prompt might look slightly different — that's fine. The important part is that blinking cursor. That's where you type.

Command 1: pwd — "Where Am I?"

When you open a folder in Finder, you can see the folder name at the top of the window. In Terminal, there's no visual indicator of where you are. So the first command you'll learn answers the question: where am I right now?

Type this and press Enter: 

pwd

pwd stands for Print Working Directory. "Working directory" is just a fancy way of saying "the folder I'm currently in."

You should see something like:

/Users/yourname

This is your home directory — the folder that belongs to your user account. It's the same folder you see when you open Finder and click your name in the sidebar.

Remember that ~ in the prompt? That's a shortcut for your home directory. So ~ and /Users/yourname mean the same place. Developers are big on shortcuts.

A Quick Note on Paths

That /Users/yourname is called a path. It describes where something is on your computer, like an address.

Paths work like street addresses. Just as "123 Main Street, Apartment 4B" tells the postal service exactly where to deliver a letter, /Users/yourname/Documents/my-project tells the computer exactly where to find a folder. Each / is like going one level deeper — from the building, to the floor, to the apartment.

Command 2: ls — "What's in Here?"

Now that you know where you are, let's see what's in this folder. Type this and press Enter: 

ls

ls stands for list. It shows you everything in the current folder.

You should see a list of familiar folder names:

Applications  Desktop  Documents  Downloads  Library  Movies  Music  Pictures  Public

These are the same folders you see in Finder when you open your home directory. Try opening Finder right now and comparing — same folders, different view.

If you see a lot of items and they look cluttered, that's fine. ls just lists everything. It doesn't change anything — it's purely for looking around.

Command 3: cd — "Go to That Folder"

In Finder, you double-click a folder to open it. In Terminal, you type cd followed by the folder name. Let's go into your Documents folder: 

cd Documents

cd stands for change directory. "Directory" is just the developer word for "folder." Same thing, different name.

After pressing Enter, it might look like nothing happened. But check your prompt — it now shows Documents instead of ~. You moved!

Verify by running pwd again: 

pwd

You should see:

/Users/yourname/Documents

You're now inside the Documents folder. If you type ls, you'll see whatever files and folders live in your Documents.

Going Back Up

What if you want to go back to the folder you came from? In Finder, you'd click the back arrow. In Terminal, you type: 

cd ..

Those two dots (..) mean "the folder one level up." It's like pressing the back button. Try it now, then run pwd to confirm you're back in your home directory.

You can always jump straight back to your home directory from anywhere by typing just cd with nothing after it, or cd ~. Both take you home.

Command 4: mkdir — "Create a New Folder"

In Finder, you right-click and choose "New Folder." In Terminal: 

mkdir my-first-project

mkdir stands for make directory. It creates a new, empty folder with whatever name you give it.

After running this command, open Finder and navigate to the same folder where you ran the command. You'll see a brand-new folder called my-first-project. The Terminal created a real folder — you can see it, open it, rename it, do whatever you want with it in Finder.

Folder and file names in the Terminal are case-sensitive and shouldn't contain spaces. Use hyphens (-) instead of spaces. my-first-project is good. My First Project will cause headaches.

Command 5: touch — "Create a New Empty File"

Let's go into the folder you just created and make a file inside it: 

cd my-first-project
touch hello.txt

touch creates a new, empty file. In this case, an empty text file called hello.txt. That's it. The file exists now. It has no content yet, but it exists.

Switch to Finder and open the my-first-project folder. You should see hello.txt sitting there. It was created by your Terminal command.

You can verify this in Terminal too. Type ls and you'll see: 

hello.txt

Command 6: clear — "Clean Up the Screen"

After typing several commands, your Terminal window might look cluttered. Type: 

clear

This doesn't delete anything — it just clears the screen so you have a fresh view. Think of it like flipping to a clean page in a notebook. Everything you did is still done; you just can't see the old text anymore.

You can also press Cmd + K to clear the Terminal screen. It does the same thing, just faster.

Your Six Commands — The Complete List

Here they are, all in one place. This is everything you need for this course:

CommandWhat It DoesFinder Equivalent
pwd Shows which folder you're in Looking at the window title
ls Lists what's in the folder Looking at the files in a Finder window
cd folder Goes into a folder Double-clicking a folder
cd .. Goes up one level Clicking the back arrow
mkdir name Creates a new folder Right-click → New Folder
touch name Creates a new empty file No real equivalent (usually done inside apps)

That's it. Six commands. You will not need more than this for the rest of the course. Later, when you use Windsurf, the built-in terminal is there mainly for two OpenSpec commands — and we'll walk you through those step by step.

Why Do Developers Use the Terminal?

You might be wondering: if Finder does the same things, why bother with the Terminal at all?

Fair question. For you, the Terminal is a stepping stone. You need it for a few specific tasks (like initializing OpenSpec), and that's all. But developers love it because:

  • Speed — typing mkdir new-feature is faster than right-clicking and renaming
  • Precision — commands do exactly one thing, predictably, every time
  • Automation — you can chain commands together to do complex tasks in seconds

You don't need to love the Terminal. You just need to be comfortable enough to type a command when asked to. And after this chapter, you are.

Put It All Together

Your mission: Create a project folder with a file inside it, then verify everything in Finder.

  1. Open Terminal (Cmd + Space, type "Terminal", press Enter)
  2. Check where you are: pwd
  3. Navigate to your Documents folder: cd Documents
  4. Create a new folder: mkdir my-first-project
  5. Go into that folder: cd my-first-project
  6. Create a file: touch notes.txt
  7. Verify the file is there: ls
  8. Open Finder and navigate to Documents → my-first-project — you should see notes.txt

If you see notes.txt in both Terminal and Finder — congratulations, you just used the Terminal like a developer.

Bonus challenge: Without looking at the reference table above, try to:

  1. Go back up to the Documents folder
  2. Create another folder called practice
  3. Go into practice and create two files: file-a.txt and file-b.txt
  4. List the contents to confirm both files are there

If you got stuck, scroll up. No shame — this is practice, not an exam.

Common Questions

"I typed the command but nothing happened."

For commands like cd, mkdir, and touch, silence means success. The Terminal only talks back when there's a problem. No news is good news.

"I see 'No such file or directory.'"

This means you tried to go into a folder that doesn't exist at your current location. Run ls to see what folders are available, check the spelling, and try again.

"Can I break something by typing the wrong command?"

With the six commands you learned today? No. pwd and ls only look at things. cd only moves you around. mkdir and touch only create things. None of them delete or modify anything. You're safe.
