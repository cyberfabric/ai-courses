Refactoring Toward DDD with AI

Memory Map
🧭

The Big Idea

DDD gives you the vocabulary; AI gives you the throughput. Strategy stays human (contexts, subdomains, aggregate boundaries). Mechanics go to Claude (renames, extractions, repository scaffolds, test stubs). Mistake the split and you ship anti-patterns at scale; respect it and you collapse a quarter of work into a week.

🧪 The Verification Sandwich

  1. Run tests before each module

    Green baseline. If you start red, no rename will help.

  2. Claude executes one module at a time

    Module-scoped, not repo-scoped. Smaller blast radius.

  3. Run tests after each module

    Stop on red. Don't proceed to the next module until green.

  4. Claude generates new tests for moved logic

    One test class per aggregate method, using fakes (not mocks).

  5. Commit. Reset on bad runs.

    git reset --hard HEAD~1 is faster than untangling a half-bad refactor.

🚨 Where AI Will Fail You

🧭
Strategic boundaries
Bounded contexts, core vs supporting vs generic. Plausible suggestions, often wrong.
🤝
Talking to domain experts
PMs, support, ops know things the codebase doesn't. AI can encode it — can't elicit it.
⚖️
Trade-off calls
"Aggregate or value object?" depends on team size, deploy cadence, domain stability.
🪤
Pattern uniformity
Will apply repository pattern to a 5-row reference table. Reject mechanically-applied ceremony.

⚙️ Workflow Tactics

📦
One aggregate per PR
A 6-file PR is reviewable in 10 min. A 50-file PR is a horror.
🚧
Never two contexts in one PR
Mixing contexts hides cross-context coupling. Single-context PRs surface it as a missing import.
🔍
Diff every change
Read the diff — not Claude's summary, not the test result. The diff is the truth.
📚
Glossary as a guard
Claude proposes UserManager? Glossary says "Do not say: Manager." Reject.
⏸️
Pause every five PRs
Converging or making lateral moves? If lateral, re-read the context map before continuing.
⏪
Commit small, reset often
Each commit is a Claude-rollback checkpoint. git reset --hard HEAD~1 is the safety valve.

📖 Key Terms

Characterization test
A test pinning down current (possibly weird) behavior before you refactor — your safety net for thinly-tested legacy code.
Boundary wrapping
Wrapping primitives in value objects only at the edge (mappers, user input). Inside :domain, types are the contract.
Latent bug
An invariant that should be enforced but isn't enforced anywhere. Surfaces during step 3; fix as part of moving logic into the aggregate.
One aggregate per PR
The reviewability discipline. Keeps refactors small enough that humans can verify each diff in minutes, not hours.
Module-scoped run
Letting Claude execute changes one Gradle module at a time, with tests run between, so failures fail loud and small.
Glossary-as-guard
Using UBIQUITOUS_LANGUAGE.md's "do not say" list as a rejection criterion when reviewing AI-suggested class names.
Persistent artifact
A doc or skill committed to the repo that outlives any single refactor session: UBIQUITOUS_LANGUAGE.md, CONTEXT_MAP.md, .claude/commands/*.

❓ Quick Quiz

Test yourself

Why must steps 1–3 happen before steps 4–6?

Mechanizing renames before naming the domain bakes the wrong vocabulary into thousands of files. Steps 1–3 produce the strategic decisions (contexts, glossary, aggregate boundaries); steps 4–6 are mechanical executions of those decisions. Out of order, you scale anti-patterns instead of fixing them.

What does "AT THE BOUNDARY ONLY" mean for value-object extraction?

Wrap raw primitives in value objects only at the edges of :domain — in Retrofit DTO mappers, Room mappers, and user-input handlers. Once inside :domain, the value-object type is the contract. If you let Claude wrap raw strings inside :domain, you've just moved the validation problem.

What three things is AI genuinely bad at during a DDD refactor?

(1) Strategic boundary calls — what is and isn't a bounded context, what's core vs supporting vs generic. (2) Talking to domain experts — PMs and support know things the codebase doesn't. (3) Trade-off calls that depend on team size, deploy cadence, and domain stability.

What's the by-product of step 3 (aggregate proposal) that doubles as a free audit?

A list of invariants that aren't enforced anywhere in the code — latent bugs. Treat each as a ticket and fix it as you move logic into the aggregate. The refactor pays a bonus dividend in correctness.

Why is "one aggregate per PR / never two contexts per PR" load-bearing?

Reviewability and coupling visibility. A small, single-context PR is reviewable in 10 minutes and surfaces cross-context coupling as a missing import. A 50-file PR that touches multiple contexts hides coupling and is too large to verify line-by-line — so reviewers approve based on Claude's summary, which is the failure mode.

What artifacts persist past the refactor session?

UBIQUITOUS_LANGUAGE.md, CONTEXT_MAP.md, and the slash commands in .claude/commands/. You pay to create them once; they pay you back every time anyone (human or AI) opens the repo. They are the highest-leverage outputs of the entire course.

Key Takeaways

  • DDD gives you the vocabulary; AI gives you the throughput. Neither alone is enough.
  • Six steps in order: context map → glossary → aggregates → value objects → logic move → repositories.
  • Steps 1–3 are human-led (strategic). Steps 4–6 are AI-led (mechanical). Order matters — never mechanize before naming.
  • Wrap primitives in value objects "AT THE BOUNDARY ONLY." Inside :domain, the type is the contract.
  • Run tests between each module-scoped change, not at the end of the day. Stop on red. Reset on bad runs.
  • Step 3 produces a free latent-bug audit: invariants currently enforced nowhere are tickets, not commentary.
  • One aggregate per PR. Never two contexts per PR. Diff every change — not Claude's summary.
  • AI is bad at strategic boundary calls, talking to domain experts, and trade-off calls. Stay in the seat for those.
  • AI is excellent at codebase-wide renames, repository scaffolding, test stubs, and consistency enforcement. Hand off ruthlessly.
  • The persistent artifacts (UBIQUITOUS_LANGUAGE.md, CONTEXT_MAP.md, slash commands) outlive any session and pay back forever.
  • Walk one feature through the six steps in a week. The next feature is easier; the third is muscle memory.
