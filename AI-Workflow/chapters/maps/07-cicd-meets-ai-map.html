⚠️ Safety & Cost Rules

--tools "Read,Grep,Bash(git *)"
Restricts which tools are available — deny-by-default, the safest pattern for CI. --allowedTools only auto-approves; --disallowedTools explicitly blocks (and wins over allow rules).
Prompt Caching
Automatic in Claude Code — cache hits cost about 10% of input price. Repeated CI prompts (same review, many PRs) get dramatically cheaper. Current TTL: 5 minutes.
--max-turns
Cap iterations in claude_args (e.g. --max-turns 5). Prevents runaway loops; most CI prompts only need 1–3 turns.
Workflow Filters
Run AI review only on PRs to main, not every push to every branch. Use paths: filters to skip docs-only changes.
Concurrency Limits
concurrency: { group: review-${{ github.ref }}, cancel-in-progress: true } — stops parallel runs on the same PR from piling up.
Model Selection
Sonnet for review, Haiku for simple classification. Override per-job via claude_args: --model claude-haiku-4-5.

🧪 Quick Quiz

What flag makes Claude Code read-only in CI?

--tools "Read,Grep" (or --tools "") restricts which tools are available — the safest deny-by-default pattern. --allowedTools only auto-approves listed tools but doesn’t block others. --disallowedTools explicitly blocks specific tools.

What are the two ways to run Claude Code in GitHub Actions?

Either the official anthropics/claude-code-action@v1 (with prompt: and claude_args: inputs — posts PR comments natively), or the manual npm install + claude -p pattern when you need raw control over multiple Claude calls and output piping.

How does build failure diagnosis change the feedback loop?

Instead of 15 min scrolling through logs, the developer sees a PR comment with root cause, error location, and suggested fix. Debug time drops to 2 minutes.

What triggers the auto-changelog workflow?

Pushing a version tag (v*). It reads the git log between tags, generates grouped changelog entries, and commits to CHANGELOG.md.

Why should you be careful with CI costs?

API calls cost money. Use workflow filters (PRs to main only), concurrency limits, cheaper models for simple tasks, and monitor usage weekly.

  • Use --tools "Read,Grep,Bash(git *)" to restrict availability. --allowedTools only auto-approves; --disallowedTools explicitly blocks.
