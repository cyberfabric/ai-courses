Chapter 8

Documentation That Writes Itself

Memory Map
💡

The Big Idea

The problem isn’t writing docs — it’s keeping them updated. Code changes 20 times a day; docs change twice a quarter. Generate docs from code, flag staleness with hooks, and audit weekly with scheduled agents. Never write boilerplate documentation by hand.

🪦 The Documentation Problem

📉
60% Outdated
Most developers say their project docs don’t match the current code. Someone wrote v1; nobody updated.
⏱️
45 Min to Write
Average time to write a good README from scratch. AI does it in 30 seconds from actual code.
🚫
0 Min to Update
Time most devs spend updating docs after code changes. “I’ll do it later” = never.

🔗 Keeping Docs in Sync: Three Layers

  1. 🪝 Freshness Hook (Real-Time)

    PostToolUse hook maps source files to docs. Change an API file? Hook says “docs/API.md may need updating.” Claude fixes in same session.

  2. 📝 Regeneration Skills (On-Demand)

    Run /generate-readme or /generate-api-docs whenever you want a fresh version based on current code.

  3. ⏰ Weekly Freshness Audit (Scheduled)

    GitHub Action checks git log: when was each doc last modified vs. its source files? Creates issue if STALE or OUTDATED.

📐 ADR Template

Status
Accepted / Proposed / Deprecated / Superseded
Context
What problem or need prompted this decision?
Decision
What was decided, with enough detail to understand the approach.
Alternatives Considered
What else was evaluated and why it was rejected.
Consequences: Positive
What you gain from this decision.
Consequences: Negative
Trade-offs you’re accepting.
Consequences: Risks
What could go wrong and how you’d handle it.

Generate at decision time, not later. /generate-adr "Use Room for persistence" takes 30 seconds.

✨ The Documentation Golden Rule

Doc Type Approach Example
Can be generated from source Always generate README, API docs, KDoc, diagrams
Requires human judgment Generate skeleton, fill in ADRs, onboarding context, rationale
Boilerplate Never write by hand Setup steps, dependency lists, file descriptions

🧪 Quick Quiz

What’s the real problem with documentation?

Not writing it — keeping it updated. Code changes 20x/day, docs change 2x/quarter. The initial version gets written; updates never happen.

How does the freshness hook work?

A PostToolUse hook maps source files to doc files. When you edit an API file, it reminds Claude Code that docs/API.md may need updating — in the same session.

When should you generate an ADR?

At decision time, while context is fresh. Not later. /generate-adr takes 30 seconds and captures the alternatives and trade-offs you just discussed.

Why use Mermaid for architecture diagrams?

Mermaid renders natively on GitHub. The diagram is code in your markdown — no images to maintain, no external tools. Regenerate it when architecture changes.

What are the three layers of doc sync?

Real-time (freshness hook flags stale docs), on-demand (regeneration skills), scheduled (weekly audit creates GitHub issues for stale docs).

📌 Key Takeaways

  • The problem is updating, not writing. Generate docs from code so they’re always based on reality.
  • Six doc skills: /generate-readme, /generate-api-docs, /generate-kdoc, /generate-adr, /generate-onboarding, /generate-diagram.
  • Three sync layers: freshness hook (real-time), regeneration skills (on-demand), weekly audit (scheduled).
  • ADRs capture why — generate them at decision time while context is fresh, not later.
  • Mermaid diagrams render on GitHub — architecture as code, no images to maintain.
  • Golden rule: if it can be generated from source, generate it. Never write boilerplate docs by hand.
