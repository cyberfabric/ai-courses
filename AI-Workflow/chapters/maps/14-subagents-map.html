Chapter 14

Subagents — Specialized Agents for Specialized Jobs

Memory Map
🤖

The Big Idea

A subagent is a markdown file in .claude/agents/ that gives Claude a focused colleague: own context window, own tool allowlist, own model. Use one when a task would otherwise drown your main thread in logs you don’t need.

📄 Anatomy of a Subagent

🏷️
name (required)
The handle. Used in @agent-name mentions and the Task tool. Unique within the project.
📝
description (required)
The trigger Claude matches against your request. The most important field — vague descriptions = vague delegation.
🛠️
tools (optional)
Comma-separated allowlist. Omit to inherit parent thread’s tools. Whitelist for tight control.
🧠
model (optional)
haiku / sonnet / opus, full ID, or inherit. Right-sizing here is real money.

🏗️ Built-In Subagents

🔍
Explore
Read-only codebase search. Scope uncertain or many areas? Use Explore.
📋
Plan
Software-architect agent. Designs implementation plans before non-trivial work.
🤖
general-purpose
Default catch-all. Multi-step tasks that don’t fit Explore or Plan.

🎯 Three Ways to Invoke

  1. Auto-delegation by description

    Claude matches your request against every subagent’s description field and delegates. The magic path.

  2. Explicit @agent-name mention

    Type @code-reviewer in your prompt — deterministic routing. Typeahead added April 2026.

  3. Programmatic via Task tool

    From skills, commands, or the Agent SDK: spawn directly with the Task tool, passing subagent_type.

🤔 Subagent vs. Skill vs. Plugin

PrimitiveLives inBest for
Skill .claude/commands/*.md Reusable prompts you invoke as /name. Run in main context.
Subagent .claude/agents/<name>.md Tasks with heavy intermediate output. Reviewers, auditors, researchers.
Plugin Installed via /plugin install Distribution layer. Bundles skills + subagents + hooks + MCP.

Decision rule: does the task generate output you don’t want in the main thread? → subagent.

💰 Right-Sizing the Model

⚡
Haiku
Routine, structural work. Boilerplate, mechanical refactors, rote test scaffolding.
🎯
Sonnet
Reasoning, nuance, judgment. Code review, design feedback, ambiguity resolution. Daily driver.
🧠
Opus
Tricky novel work. Tough debugging, architecture design, security audits. Use sparingly.
🔄
inherit
Use whatever the main session is using. Good for orchestrators that match user’s quality bar.

⚠️ Common Mistakes

🔬
Over-Fragmentation
20 tiny agents = Claude has to read 20 descriptions. Aim for half a dozen strong specialists, not 30 weak ones.
🔓
Wrong Tool Allowlist
Reviewer with Write = quiet file mutator. Researcher without WebSearch = useless. Be deliberate.
🌫️
Vague Descriptions
“Use for reviews” loses to general-purpose. Specific, concrete descriptions win delegation.
📣
@-Mentioning Everywhere
If you’re always @-mentioning the same agent, the description is too weak. Tighten the description and let auto-delegation work.

📖 Key Terms

Subagent
A markdown file in .claude/agents/ defining a specialized agent with its own context, tools, and model.
Auto-delegation
Claude reads every available subagent’s description against your request and delegates if there’s a match.
@-mention
Explicit invocation: @code-reviewer in a prompt routes that work unconditionally to the named agent. Typeahead added April 2026.
Task tool
The programmatic interface for spawning subagents from skills, commands, or the Claude Agent SDK. Pass subagent_type.
Context isolation
The core benefit: a subagent’s search results, file reads, and tool outputs stay in its context window, not yours. Only the summary returns.
Project-level agent
Lives in .claude/agents/ in the repo. Committed. Team standard. Wins over user-level on name collision.
User-level agent
Lives in ~/.claude/agents/. Personal. Available across all your projects.
Right-sizing
Picking the cheapest model that does the job well. Haiku for rote, Sonnet for reasoning, Opus for novel/tricky work.

🧪 Quick Quiz

What are the two required frontmatter fields?

name and description. tools and model are optional — omitting tools inherits the parent’s allowlist; omitting model uses the parent’s model.

When should you reach for a subagent instead of a skill?

When the task generates a lot of intermediate output you don’t want polluting your main context — reading many files, running tools, parsing logs. Subagent does the work in its own context window and returns only the summary.

Why does the description field matter so much?

It’s the only signal Claude uses to decide whether to delegate. Vague descriptions = vague delegation. Use “use when” or “invoke before” phrasing with concrete triggers.

What model should you pick for routine boilerplate generation?

Haiku. Fast and cheap; on rote work it’s indistinguishable from Sonnet to a careful reviewer. Save Sonnet/Opus for tasks that need real reasoning.

Project-level vs. user-level: which wins on name collision?

Project-level. Teams should be able to override personal preferences with shared standards committed to the repo.

What’s the most common subagent design mistake?

Over-fragmentation. Twenty tiny agents make Claude read twenty descriptions to delegate one task. Aim for half a dozen strong specialists.

Key Takeaways

  • A subagent is a markdown file with YAML frontmatter (name, description, optional tools/model) and a system prompt. Lives in .claude/agents/.
  • The description field is the only signal Claude uses to decide whether to delegate — write specific, concrete triggers.
  • Use a subagent when the task would drown your main thread in logs you don’t need. The win is context isolation.
  • Three invocation paths: auto-delegation, @agent-name mention (typeahead since April 2026), or programmatic via the Task tool.
  • Right-size the model: Haiku for rote, Sonnet for reasoning, Opus for novel/tricky, or inherit from the parent.
  • Restrict tools deliberately: reviewers read-only, researchers get web access, never grant more than the role requires.
  • Prefer half a dozen strong specialists over thirty weak ones. Over-fragmentation kills auto-delegation.
  • Project-level agents (.claude/agents/, committed) are team standards; user-level (~/.claude/agents/) are personal. Project wins on name collision.
