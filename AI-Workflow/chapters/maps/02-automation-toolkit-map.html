The Big Idea

Claude Code has five distinct automation building blocks — CLAUDE.md, Skills, Hooks, Shell/CLI, and CI/CD. Each solves a different kind of problem. Know which tool to reach for, and you’ll never over-engineer or under-automate.

🧰 The Five Building Blocks

📄
CLAUDE.md
Markdown config file. Rules, conventions, context. 5 min to set up. Affects every session, every team member. Cheapest automation.
⚡
Skills (/Commands)
Markdown prompt templates in .claude/commands/. Filename = command name. You trigger manually. 10 min to write.
🪝
Hooks
JSON config in settings.json. Fire on events: PreToolUse, PostToolUse, Notification, Stop. Zero manual triggering.
🐚
Shell Scripts + CLI
claude -p for non-interactive mode. Pipe with git diff, jq, xargs. Full Unix composition power.
🔄
CI/CD (GitHub Actions)
YAML workflows with Claude Code steps. Triggered by push, PR, schedule, or webhook. Runs without any human present.

📄 CLAUDE.md: Four Scope Levels

  1. ~/.claude/CLAUDE.md — Global

    Personal preferences across all projects: coding style, your role, default behaviors.

  2. /project/CLAUDE.md — Project (shared)

    Checked into git. Team conventions, architecture rules, tech stack, review standards.

  3. /project/.claude/CLAUDE.md — Project (local)

    Not checked in. Personal project notes, WIP context, experiments.

  4. /project/subdir/CLAUDE.md — Directory

    Module-specific rules (e.g., API layer or UI layer conventions).

🪝 Hook Types at a Glance

Hook Type When It Fires Example
PreToolUse Before a tool runs Block force-push to main
PostToolUse After a tool runs Auto-lint after file edit
UserPromptSubmit Right after you press enter Inject context, rewrite prompts, guardrails
SessionStart / SessionEnd Session opens / terminates Setup & teardown automations
PreCompact Before context compaction Pin critical state to disk first
Stop / SubagentStop Turn or subagent finishes Summary, collect parallel results
Notification Claude wants to notify Send Slack message

🖥️ Key CLI Flags for Automation

-p "prompt"
Non-interactive mode — send prompt, get response, exit. The foundation of all scripted automation.
--output-format json
Machine-readable output for piping to jq and parsing in scripts.
--allowedTools
Auto-approve specific tools — they run without permission prompts. For CI runs that should proceed unattended.
--disallowedTools
Strip tools from Claude Code’s context entirely — the hard safety lever (e.g., forbid Bash(rm:*)).
--model
Choose the model — use faster models for simple tasks, powerful ones for complex reviews.
-c
Continue the last conversation — enables multi-step workflows across separate calls.

Aspect Project Skills User Skills

🐚 Shell Essentials for Automation

📥
Piping
git log | claude -p "..." — feed context into prompts via stdin.
🔧
jq
Parse JSON output from --output-format json. Essential for structured results.
✅
Exit Codes
claude -p returns 0 on success. Use in bash conditionals to gate pipeline steps.
⚠️
Quote Variables!
Always "$DIFF" not $DIFF. Unquoted vars with special chars break scripts.

🧪 Quick Quiz

What’s the cheapest, easiest automation to set up in Claude Code?

CLAUDE.md — a markdown file with rules and conventions that every session follows automatically.

What’s the difference between a skill and a hook?

A skill is triggered manually (you type the /command). A hook fires automatically in response to an event (file edit, tool use, task completion).

Which CLI flag makes Claude Code non-interactive for scripting?

-p (print mode) — send a prompt, get a response, exit. No interactive session.

Where do project-level hooks go?

In .claude/settings.json under the "hooks" key.

When should you use CI/CD instead of a local hook?

When the automation should run without you present — on every PR, on a schedule, or triggered by a webhook.

📌 Key Takeaways

  • Five building blocks: CLAUDE.md → Skills → Hooks → Shell/CLI → CI/CD. Each layer builds on the previous.
  • Start with CLAUDE.md — 5 minutes to set up, affects every session for every team member.
  • Skills = on-demand /commands you trigger. Hooks = automatic reactions to events.
  • claude -p is the key to non-interactive automation — pipe it, script it, cron it.
  • Use the decision tree: behavior rule → CLAUDE.md, manual trigger → skill, auto on event → hook, multi-tool → shell, unattended → CI/CD.
  • Don’t over-engineer: reach for the simplest tool that fits the job.
