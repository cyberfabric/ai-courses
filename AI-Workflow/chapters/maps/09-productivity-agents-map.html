The Big Idea

Meta-work — standups, triage, status checks, context switching — eats 1.5 hours every day. Combine skills and shell aliases into a single goodmorning command that replaces the 30-minute morning fog with a 2-minute briefing.

⏳ The Invisible Time Sink

🕐
1.5 Hours/Day
Average daily meta-work: Slack, GitHub, standups, CI checks, context loading. That’s before writing a single line of code.
🧠
23 Min to Refocus
Time to regain deep focus after each context switch. Every notification costs half an hour.
📅
375 Hours/Year
1.5 hrs × 250 workdays = nine working weeks per year lost to meta-work.

🧰 The Productivity Skill Arsenal

📋
/standup
Reads git log + open PRs. Outputs Yesterday/Today/Blockers. 10 min → 30 sec, perfect accuracy.
☀️
/morning-digest
Overnight repo activity, PRs waiting for you, your open work, suggested priorities. The complete morning briefing.
🏛️
/archaeology
Git log + blame + commit messages. Full history: who, when, why, is it safe to refactor?
📊
/pr-dashboard
Team PR overview: needs attention, ready to merge, blocked, stats. Flags aging (>3 days) and stale (>7 days).
🧹
/tech-debt-scan
TODOs, deprecated APIs, suppressed warnings, hardcoded values. Grouped by severity, sorted by effort.
🔔
/triage-notifications
Sorts GitHub notifications into P0 (act now) through P3 (archive). Stops notification overwhelm.

🔔 Notification Triage Levels

  1. 🔴 Act Now

    Review requests (you’re blocking someone), CI failures on your PRs, direct mentions.

  2. 🟡 Today

    Comments on your PRs, assigned issues, PR approvals you can merge.

  3. 🔵 This Week

    Team PR activity, repo issues, release notifications. Stay informed.

  4. ⚪ Archive

    Bot notifications (Dependabot), subscribed threads, anything >7 days old. Bulk mark read.

⚡ Shell Aliases: Quick One-Liners

explain
Pipe any error message into claude -p for an instant explanation and fix suggestions.
commitmsg
Generate a conventional commit message from staged changes. Output only the message.
tldr
Summarize any file in one sentence. cat file.kt | tldr
howto
Explain what a command does, step by step. Pipe any complex command into it.
myreview
Quick review of your branch changes against main. Brief, bug-focused.

Aliases are the gateway drug to automation. Start here, graduate to skills.

🧪 Quick Quiz

What is “meta-work” and how much time does it cost?

Work about the work: standups, triage, status checks, context loading. About 1.5 hours/day or 375 hours/year for an average developer.

Why is /standup more accurate than writing your update from memory?

It reads git log, open PRs, and current branch — every commit is accounted for. Memory at 9 AM is fuzzy; git remembers everything.

What’s the “goodmorning” pattern?

A shell script that chains morning-digest + standup + triage-notifications into one command. Replaces the 30-minute morning fog with a 2-minute briefing.

How do shell aliases relate to skills?

Aliases are quick one-liners for common tasks. When you need more structure or want to share with the team, graduate an alias to a full skill file.

What does /archaeology do that git blame alone doesn’t?

It combines git log, blame, and commit messages into a narrative: who wrote it, why, what major changes happened, is it safe to refactor, and who to ask for context.

📌 Key Takeaways

  • Meta-work costs 1.5 hrs/day (375 hrs/year). Automate it with productivity skills.
  • Six skills: /standup, /morning-digest, /archaeology, /pr-dashboard, /tech-debt-scan, /triage-notifications.
  • Shell aliases (explain, commitmsg, tldr) are the fastest automation — one-liners with claude -p.
  • Combine everything into a goodmorning script: digest + standup + triage in one command.
  • Capture codebase knowledge with /note before it disappears from your head.
  • Run tech debt scans monthly and track the trend — visibility creates accountability.
