The Big Idea

MCP is a USB port for Claude Code. Just like USB lets you plug any peripheral into your computer, MCP lets you plug any data source — Jira, databases, Slack, dashboards — into Claude Code. Zero copy-pasting, zero tab-switching.

🏢 Internal Tool / API

An MCP server declares tools, handles requests, and returns results — that’s it.

🧩 Anatomy of an MCP Server

🔧
Tools
Functions Claude Code can call. The automation power. get_ticket, query_database, check_deploy.
📄
Resources
Data Claude Code can read. API docs, schema definitions, config files.
💬
Prompts
Pre-built prompt templates. “Summarize this ticket”, “Debug this query”.
📦
MCP SDK
@modelcontextprotocol/sdk — the official TypeScript SDK. Handles protocol details for you.
📡
Transport
Local: stdio (your process). Remote: http or sse (vendor-hosted URL). Same protocol either way.
⚙️
Registration
Use claude mcp add <name> --scope project --transport stdio -- node server.js. Writes to .mcp.json at the project root — commit it to share with the team.

🛠️ Building Your First MCP Server

  1. Scaffold the project

    npm init, install @modelcontextprotocol/sdk, zod, and TypeScript.

  2. Create the server

    Instantiate McpServer, register tools with server.registerTool(name, { description, inputSchema }, handler) — inputSchema uses Zod (z.string(), z.number(), …).

  3. Connect transport

    Create StdioServerTransport and await server.connect(transport).

  4. Build

    npx tsc to compile TypeScript to JavaScript.

  5. Register with Claude Code

    claude mcp add deploy-status --scope project --transport stdio -- node ./dist/index.js — writes .mcp.json at the project root.

  6. Test interactively

    Start Claude Code and ask it to use your tools. Test happy paths and error handling.

🔒 Security Checklist

👁️
Read-Only First
Start with read-only tools. Add writes only when explicitly needed and approved.
🔑
Least Privilege
DB server gets SELECT only. Jira server gets read + comment, not admin.
🚫
No Production DB
Staging only. Always. No exceptions. Validate queries before executing.
🔐
Secrets in Env Vars
Never hardcode tokens. Use ${ENV_VAR} expansion in .mcp.json; keep actual values in your shell, .env, or OS keychain.
🛡️
Input Validation
Sanitize every input. Block non-SELECT queries at the server level. Prevent injection.
📝
Audit Logging
Log every tool call with timestamp, user, and parameters for security review.

📖 Key Terms

MCP (Model Context Protocol)
An open protocol that lets AI tools connect to external data sources and APIs through a standardized interface.
MCP Server
A small program that declares tools, handles requests from Claude Code, and returns results from your internal systems.
Tool (MCP)
A named function your server exposes that Claude Code can call — e.g., get_ticket, query_database.
Resource (MCP)
Read-only data your server makes available — API docs, schemas, config files.
StdioServerTransport
The communication layer for local servers Claude Code spawns — talks via stdin/stdout. Remote servers use HTTP or SSE instead.
.mcp.json
Canonical project-shared registration file at the repo root, with a top-level mcpServers key. Commit it; ${ENV_VAR} expansion is supported.
claude mcp add
CLI for registering servers. --scope local|project|user picks where the entry lands. Sibling commands: add-json, add-from-claude-desktop.
Hosted MCP server
Vendor-run HTTP endpoint (Stripe, Linear, Notion, Sentry, GitHub, …). One claude mcp add --transport http <name> <url> and you’re wired up — no install, no hosting.
Force Multiplier
MCP makes every existing skill and hook more powerful — your /standup can pull Jira, your /review can check deploys.

🧪 Quick Quiz

What analogy does the chapter use for MCP?

USB port. Just like USB lets you plug any peripheral into a computer, MCP lets you plug any data source into Claude Code.

What are the three main components of an MCP server?

Tools (functions Claude Code can call), Resources (data it can read), and Prompts (pre-built templates).

Where do you register MCP servers for team use?

In .mcp.json at the project root, with a top-level mcpServers key. The fastest way to write it is claude mcp add <name> --scope project --transport stdio -- <cmd>. Reference secrets via ${ENV_VAR} rather than hardcoding.

Do you always have to write your own MCP server?

No — many vendors (Stripe, Linear, Notion, Sentry, GitHub, Atlassian, Slack, …) ship hosted HTTP MCP servers. Add them with claude mcp add --transport http <name> <url>. Build your own only for internal tools the vendor list doesn’t cover.

Why should your database MCP server be read-only and staging-only?

Safety. Validate every query at the server level (only allow SELECT). Use a SELECT-only DB user. Never connect to production — staging only, always.

Why is MCP called a “force multiplier” for existing automations?

Every skill and hook gains access to every tool your MCP servers expose. Your standup skill can pull Jira data, your review skill can check deploys — possibilities multiply with each server.

Key Takeaways

  • MCP = USB port for Claude Code — plug in any data source or internal tool.
  • Two flavours: local (stdio, your own server) and remote (http/sse, vendor-hosted). Use hosted servers for Stripe/Linear/Notion/Sentry/etc.; build your own for internal tools.
  • Modern SDK API: server.registerTool(name, { description, inputSchema }, handler) with Zod schemas. Legacy server.tool() still works but is deprecated.
  • Register with claude mcp add <name> --scope project --transport stdio -- node server.js — writes to .mcp.json at the project root for team sharing.
  • .mcp.json supports ${ENV_VAR} expansion — keep secrets out of git.
  • Start read-only, enforce least privilege, staging DB only, log everything.
  • MCP is a force multiplier — every existing skill and hook gets access to every server’s tools.
