Controlling Output Format

The Big Idea

Format is like food delivery packaging — the answer might be perfect, but if it arrives in the wrong container, you waste time repacking it. One line of format instruction eliminates the "reformatting tax" entirely.

🛠️ The Six Techniques

📋
Schema Specification
For JSON: specify exact field names, data types, and allowed values. Don't say "return as JSON" — define the shape.
🚫
"Respond ONLY With..."
Strips all fluff. "Respond ONLY with the JSON. No explanation." Perfect when output goes into code or tools.
📏
Specific Length
"Exactly 3 bullet points" beats "be concise." Structural constraints (bullets, sentences) are more reliable than word counts.
📝
Section Headers
Write the ## headers yourself in the prompt. The AI fills them in, mirroring your structure exactly. You define, it fills.
🏷️
XML Delimiters
Use <tag>...</tag> to fence output sections. Especially reliable with Claude. Great for programmatic parsing.
🔁
Reinforcement
State the format instruction at the TOP and again at the BOTTOM of your prompt. AI weights beginnings and endings most.

🔧 When the AI Ignores Your Format

Format buried at the end
Move it to the top: "Format as JSON. Here's the task: ..."
Conflicting instructions
"Explain in detail" + "one sentence" = confusion. Pick one or split into sections.
Unwanted preamble ("Sure! Here's...")
Add: "No preamble, no greeting. Start directly with the [format]."
Markdown fences around JSON
"Return raw JSON without markdown code fences or backticks."
AI "improves" your format
"Follow this format EXACTLY. Do not modify or add to the structure."

Format Gotchas
AI loves to add "helpful" extras

❓ Quick Quiz

What's the "reformatting tax" and how do you avoid it?

Time spent reformatting AI output to fit your actual workflow. Avoid it by specifying the exact format you need in the prompt — one line of format instruction saves minutes of copy-paste surgery.

Why is "be concise" a bad length constraint?

It's subjective — the AI interprets it differently each time. Use specific structural constraints instead: "exactly 3 bullet points" or "one paragraph, max 3 sentences."

What's the Schema Specification technique?

When requesting JSON, specify exact field names, data types, and allowed values rather than just saying "return as JSON." The more precise the schema, the more consistent the output.

How does the Reinforcement technique work?

State the format instruction at the beginning AND end of your prompt. AI models weight the start and end more heavily than the middle, so sandwiching the instruction maximizes impact.

When should you use XML delimiters vs markdown headers?

XML tags for programmatic parsing (especially with Claude). Markdown headers for human-readable multi-section output. Both define structure — pick based on who/what will consume the output.

📌 Key Takeaways

  • Always specify format — "return as JSON", "present as table", "3 bullet points"
  • For JSON: define exact field names, types, and allowed values (schema specification)
  • "Respond ONLY with..." strips fluff for clean, parseable output
  • Structural length constraints (3 bullets) beat word counts (50 words)
  • Write section headers (##) yourself — AI mirrors your structure
  • XML tags are the most reliable delimiters, especially with Claude
  • When format breaks: move instruction to top, remove conflicts, reinforce at end
  • Tone is one keyword away: technical, casual, formal, ELI5, executive-summary
