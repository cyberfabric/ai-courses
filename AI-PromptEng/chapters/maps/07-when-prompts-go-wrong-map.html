Chapter 7

When Prompts Go Wrong

Memory Map
💡

The Big Idea

When the output is wrong, the prompt is the first suspect — not the AI, not the model, not bad luck. Prompt debugging works like code debugging: read the error, check your assumptions, fix the logic.

🚨 The Five Failure Modes

Failure Mode Symptom Diagnostic Question Fix
1. Misunderstood Task AI did the wrong thing entirely "Could someone interpret this differently?" Replace ambiguous words with specifics
2. Missing Context Answer looks right but is wrong for your case "What do I know that the AI doesn't?" Add project/domain/constraint details
3. Too Literal AI followed the letter, not the spirit "Did I specify what 'better' means?" Specify intent + what to preserve
4. Hallucination Confident but made-up facts "Can I verify this independently?" "If unsure, say so" + verify claims
5. Wrong Format Right content, wrong package "Did I specify format at the top?" Format at top + "ONLY" + reinforce at end

🛠️ Debugging Techniques

🔬
Explain Your Reasoning
Ask the AI to show its work. Like adding log statements — exposes where the logic breaks.
🚫
Negative Prompting
"Do NOT..." blocks unhelpful defaults, deprecated approaches, and scope creep.
🎯
Targeted Adjustment
Change one thing at a time, not four. If output improves, you know which fix helped.
📝
Prompt Versioning
Track what you tried: v1 (too vague) → v2 (added structure) → v3 (added examples) ✅

⚠️ Version numbers
⚠️ Library names & API params

🎭 Key Analogies

Debugging Code = Debugging Prompts
Don't blame the compiler (AI). Read the error (output), check assumptions, fix the logic (prompt).
The Obedient Genie
AI grants wishes literally. "Make it faster" may remove error handling. Specify what to change AND what to preserve.
"Explain Your Reasoning" = Log Statements
Asking AI to show its work lets you step through the logic and find exactly where reasoning breaks.
Temperature = Randomness Dial
0.0 = deterministic (code). 0.3-0.5 = slight variation (work tasks). 0.7-1.0 = creative. Doesn't fix bad prompts.

🌡️ Temperature Quick Reference

Setting Behavior Best For
0.0 Most deterministic Code, JSON, factual answers
0.3-0.5 Slight variation Technical writing, analysis
0.7-1.0 More creative Brainstorming, ideation

Temperature controls randomness, not quality. Fix the prompt first, then tune temperature.

❓ Quick Quiz

You asked "review this code" and got a style review instead of a security review. Which failure mode?

Misunderstood Task. "Review" is ambiguous. Fix: "Review this code for security vulnerabilities. Focus on SQL injection and authentication. Ignore code style."

The AI recommended a Kotlin library that doesn't exist. Which failure mode and how do you prevent it?

Hallucination. Prevent with: "Only recommend tools you're confident exist. If unsure, say so." Always verify library names, version numbers, and API parameters independently.

You asked for "5 test cases" and got 5 variations of the happy path. What went wrong?

Too Literal. The AI produced exactly 5 test cases — but you didn't specify diversity. Fix: "Include 1 happy path, 2 negative cases, 1 edge case, 1 security case."

When should you iterate vs. start over?

Iterate if the output is partially right (AI understood the general task). Start over if it's completely off-track. Rule: if 3 iterations haven't helped, the prompt structure is wrong — rewrite from scratch.

Why change one thing at a time instead of rewriting the whole prompt?

If you change 4 things and the output improves, you don't know which change helped. If it gets worse, you can't undo just one thing. Targeted adjustments = measurable improvement.

📌 Key Takeaways

  • When output is wrong, suspect the prompt first — not the AI
  • Five failure modes: misunderstood task, missing context, too literal, hallucination, wrong format
  • The ambiguity test: "Could a smart person interpret this differently?"
  • The assumption check: "What do I know that the AI doesn't?"
  • "Explain your reasoning" exposes flawed logic like log statements
  • Iterate with one targeted change at a time — 2-3 iterations is normal
  • Negative prompting ("Don't...") blocks unwanted defaults and scope creep
  • Version your important prompts — track what works and reuse it
