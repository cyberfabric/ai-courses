Chapter 2

Anatomy of a Great Prompt

Memory Map
💡

The Big Idea

Every great prompt is built from five blocks: Task, Context, Constraints, Format, Examples. When AI gives bad output, one of these is almost always what's missing.

🧱 The Five Building Blocks

🎯 Task
What to do
→
📋 Context
Background info
→
🚧 Constraints
Rules & limits
→
📐 Format
Output shape
→
📝 Examples
Show, don't tell

Like a work order: tell the contractor what, where, the rules, the deliverable, and show a photo.

🔍 Block by Block

🎯
Task
Start with an action verb: Write, Review, Debug, Compare, Generate. If you can't say it in one sentence, you have multiple tasks.
📋
Context
Your role, tech stack, situation, what you've tried, the audience. Ask: "What would a smart colleague need to know?"
🚧
Constraints
Scope limits, length, tone, exclusions. Don't forget to say what NOT to do — AI is a people-pleaser.
📐
Format
Table, JSON, bullet list, code block, checklist. Magic phrase: "Format your response as..."
📝
Examples
The most powerful block. 2-3 examples establish a pattern better than 100 words of description.

📊 The Optimal Block Order

  1. Context First

    Set the stage so everything that follows makes sense

  2. Task Second

    Now the AI knows what to do with the context

  3. Constraints Third

    Boundaries and rules narrow the output

  4. Format Fourth

    Shape the deliverable

  5. Examples Last

    The AI's freshest memory is the pattern to follow

Any order with all 5 blocks beats a prompt missing 3 of them.

⚡ The Instruction Hierarchy

Priority Layer Example
🥇 Highest System prompt .cursorrules, CLAUDE.md, API system message
🥈 User prompt What you type in the chat (your 5 blocks go here)
🥉 Conversation context Previous messages in the same thread
4th Training data AI's general knowledge — lowest priority

When instructions conflict, higher layers win.

📏 Key Rules & Anti-Patterns

1️⃣
One Prompt, One Task
Three separate prompts beat one mega-prompt. If your prompt says "and then" twice, split it up.
⚠️
Don't Bury the Task
Lead with your task or label it clearly. Three paragraphs of context before the question = lost focus.
🔀
No Conflicting Constraints
"Comprehensive analysis in 100 words" = both fail. Split into summary + detail sections instead.
👁️
Don't Assume Context
If the AI can't see it, it doesn't exist. Paste the code, the error, the ticket — don't reference it.

📋 The Prompt Template

Context:
I'm a [role] working on [project]. We use [tech/tools]. [Background].
Task:
[Action verb] [specific thing you want done].
Constraints:
What to avoid, length/scope limits, style/tone requirements.
Format:
Respond as [table / bullet list / code block / JSON / etc.].
Example:
Here's what a good output looks like: [example].

Save this as a snippet in your IDE. Type /prompt and go.

❓ Quick Quiz

What are the five building blocks of a great prompt?

Task, Context, Constraints, Format, and Examples.

Which block is the most powerful for eliminating ambiguity?

Examples. Showing 2-3 examples lets the AI infer the pattern better than a lengthy explanation.

What's the "one prompt, one task" rule?

You get better results from three separate prompts than one mega-prompt. Split complex requests into steps.

Why should you put Examples last in a prompt?

Due to the primacy-recency effect, the AI pays more attention to the end of a prompt. Putting examples last means the pattern is freshest in its "memory."

What's the #1 constraint mistake people make?

Forgetting to say what NOT to do. AI is a people-pleaser and will add features, error handling, and extras you never asked for.

📌 Key Takeaways

  • Five blocks: Task (action verb), Context (background), Constraints (rules), Format (output shape), Examples (patterns)
  • Examples are the most powerful block — 2-3 examples beat 100 words of explanation
  • Best order: Context → Task → Constraints → Format → Examples
  • System prompt overrides user prompt overrides conversation context
  • One prompt, one task — split mega-prompts into steps
  • Tell the AI what NOT to do — it's a people-pleaser
  • When output is bad, check which block you're missing — that's usually the fix
