Chapter 11

Model Differences and Cross-Platform Tips

Memory Map

The Big Idea

80% of good prompting works on every model. The same prompt can give brilliant results on one model and mediocre results on another — but if you follow the fundamentals (clear task, context, format, examples), your prompts are already portable. Optimize for the remaining 20% only when it matters.

⚡ Model Quirks & Fixes

🟣 Claude: The Honest Refusal
May say "I don't have enough info" instead of guessing. Fix: "Give your best assessment. Note your confidence level."
🟢 GPT: The Eager Helper
Tries to answer everything, even when it shouldn't. Fix: "If you're not confident this API exists, say so."
🔵 Gemini: The Google-Thinker
Defaults to Google solutions (Firebase, GCP, Jetpack). Fix: Be explicit about your tech stack.
🟠 Open-Source: The Instruction-Dropper
Smaller models (7-13B) "forget" instructions mid-response. Fix: Keep prompts shorter, break into multiple calls.

💰 Cost-Saving Techniques

✂️
Trim Context
Don't paste entire files when the AI only needs 20 lines. Every token costs money.
📏
Limit Output
"Respond in under 100 words." Output tokens cost 2-5x more than input tokens.
🔽
Right-Size the Model
Use Haiku/Flash for classification and formatting. Save Opus/Pro for complex analysis.
🔄
Cache & Batch
Use prompt caching for repeated system prompts. Batch similar requests into one call.
🔃
Break Long Chats
Start fresh instead of carrying 50 turns of context. The AI doesn't always need full history.

❓ Quick Quiz

Why does the same prompt give different results on different models?

Each model is trained on different data, with different techniques, and with different goals. These create real behavioral differences in how they interpret instructions, what they emphasize, and how they format output. Like different cars — same controls, different handling.

What percentage of good prompting is already portable across models?

About 80%. The fundamentals — clear task, context, format, examples, chain-of-thought — work on every major model. The remaining 20% is model-specific optimization that only matters at the extremes.

Why shouldn't you reference "Artifacts" or "Canvas" in your prompts?

These are product-specific UI features, not prompting techniques. They only work on specific platforms. Portable prompts use universal techniques like format specification, few-shot examples, and chain-of-thought that work on any model.

Why does the same model behave differently in chat vs API?

Chat interfaces have a hidden system prompt that makes the model friendly, conversational, and safe. The API gives you raw control — you set the system prompt, temperature, and token limits. Always test prompts in the environment where they'll actually run.

What's the most reliable cost-saving prompt technique?

Right-sizing the model. Use small/fast models (Haiku, Flash, 4o-mini) for simple tasks like classification and formatting, and reserve large/expensive models (Opus, Pro) for complex analysis and reasoning. Output tokens cost 2-5x more than input, so also limit response length.

📌 Key Takeaways

  • 80% of good prompting is already portable — clear task, context, format, examples work everywhere
  • Claude: best at instruction following and long context. GPT: broadest ecosystem and reasoning models. Gemini: Google integration and multimodal. Open-source: privacy and customization
  • Every model has quirks: Claude refuses honestly, GPT helps eagerly, Gemini thinks Google, open-source drops instructions
  • Don't reference model-specific features (Artifacts, Canvas, Deep Research) in portable prompts
  • API and chat interface produce different output from the same model — test where you deploy
  • System prompt adherence varies: Claude strongest, open-source weakest
  • Right-size your model: Haiku/Flash for simple tasks, Opus/Pro for complex analysis
  • For 80% of daily tasks, any major model works. Selection matters at the extremes.
