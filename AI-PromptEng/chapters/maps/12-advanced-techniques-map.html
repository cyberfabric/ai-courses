Advanced Prompt Engineering

Memory Map

The Big Idea

Advanced prompting is about composing simple techniques into systems. Instead of one big prompt, use chains of focused prompts where each step's output feeds the next. The whole becomes greater than the sum of its parts — like planning a five-course dinner instead of cooking one dish.

🧰 The Advanced Toolkit

🔗
Prompt Chaining
Break complex tasks into a sequence of focused prompts. Each step's output feeds the next. Assembly line for AI.
🪞
Meta-Prompting
Use AI to write prompts for you. Then ask it to critique and improve its own prompt. Prompts all the way down.
🔄
Self-Reflection
Generate → Review → Improve. AI catches 2-5 issues in its own output when given specific review criteria.
🌳
Tree-of-Thought
Generate 3 approaches, compare pros/cons, pick the best. Explores multiple paths before committing.
🏷️
Structured Reasoning
XML tags as "thinking slots" — forces systematic analysis. <assumptions> <arguments_for> <risks> <recommendation>
📚
Team Prompt Libraries
Shared, battle-tested templates for code review, bug reports, test cases, retros. Individual skill → team multiplier.

🔗 Designing a Chain

  1. Decompose

    Split the task into distinct sub-tasks. Each should have a single, clear goal.

  2. Define Handoffs

    What does each step need from the previous one? Specify output format for each step.

  3. Add Quality Gates

    Review output after each step before passing forward. One bad step poisons the rest.

  4. Keep Context Lean

    Only pass what the next step needs. Don't carry the full history forward.

When to chain: more than 2-3 sub-goals, output feeds another task, quality matters more than speed.

📁 The CLAUDE.md Pattern

What it is
A file in your repo that tells AI tools about your project's conventions, architecture, and rules — automatically.
What goes in it
Architecture (MVVM, Clean), language (Kotlin 2.0), testing (JUnit 5, Turbine), coding rules (Result type, KDoc).
Why it works
Persistent prompt engineering — your prompts live in the codebase and apply to every AI interaction without re-typing.
Who uses it
Claude Code, Cursor, GitHub Copilot, and other AI coding tools read project-level config files.

❓ Quick Quiz

When should you use prompt chaining instead of a single prompt?

When the task has more than 2-3 distinct sub-goals, when one task's output feeds another, or when quality matters more than speed. A 4-step chain takes longer but produces dramatically better results than one overloaded prompt.

What's the "meta-prompting power move"?

After AI generates a prompt, follow up with: "Now critique this prompt. What are its weaknesses? How could it fail? Improve it." The AI produces a second, better version that addresses its own blind spots.

Why does self-reflection need specific criteria?

Without criteria ("check for edge cases, thread safety, null pointers"), the AI tends to say "looks good" and make cosmetic changes. Specific review criteria force it to actually find real issues — typically 2-5 per reflection.

What's the key rule for automated AI pipelines?

Never let AI take irreversible actions automatically. AI can suggest (PR comments, draft changelogs) but should not merge PRs, publish releases, or deploy to production. Always keep a human in the loop for anything that goes live.

Will prompt engineering still matter as models get smarter?

The specific tricks may evolve (you might not need "think step by step"), but the core skill — articulating exactly what you want with the right constraints, format, and quality bar — is permanent. It's a communication skill that works with humans too.

📌 Key Takeaways

  • Prompt chaining: break big tasks into focused steps. Each output feeds the next. Quality over speed.
  • Meta-prompting: use AI to write prompts, then have it critique and improve them
  • Self-reflection: Generate → Review (with specific criteria) → Improve. Catches 2-5 issues per pass.
  • Tree-of-thought: generate 3 approaches, compare, pick the best. Great for architecture decisions.
  • XML tags create "thinking slots" that force balanced, systematic analysis
  • Team prompt libraries turn individual skill into a team multiplier
  • The CLAUDE.md pattern: persistent prompts embedded in your codebase
  • Automated pipelines: AI suggests, humans decide. Never automate irreversible actions.
  • Prompt engineering is a communication skill — it's permanent, even as models improve
